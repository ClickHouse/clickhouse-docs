Production Cutover Steps
1. Pre-Cutover Validation
Perform basic data validation checks:
Compare row counts between source and target.
Validate data in smaller and reference tables.
Spot-check critical tables for consistency.
2. Stop Writes on Source
Stop all application writes to the source database.
As an additional safeguard, set the database to read-only:
ALTER DATABASE piproductionepisodemetadata SET default_transaction_read_only = on;

ALTER DATABASE piproductionepisodemetadata SET default_transaction_read_only = off;
Confirm that no new writes are occurring.
3. Ensure Target is Fully Caught Up
Verify replication has fully completed.
Validate by checking the most recent row in a high-activity table on both source and target.
The latest record (e.g., max timestamp or ID) should match on both systems.
4. Enable Constraints and Indexes
Enable foreign key constraints.
Reset replication role:
ALTER ROLE postgres SET session_replication_role = origin;

Recreate:
UNIQUE constraints
Indexes
5. Reset Sequences
Reset all sequences to align with the current maximum values in their respective tables.
Reference script:
https://gist.github.com/saisrirampur/af4eec1421aff830bb13c8f9af5a3a82
6. Enable Application Traffic
Once sequences are validated and all constraints are in place:
Enable read traffic to Postgres managed by ClickHouse.
Enable write traffic.
Monitor closely for errors, constraint violations, and replication lag.
