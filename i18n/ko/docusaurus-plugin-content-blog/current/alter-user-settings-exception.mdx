## DB::Exception: Cannot update user `default` in users.xml because this storage is readonly. (ACCESS_STORAGE_READONLY) \{#dbexception-cannot-update-user-default-in-usersxml-because-this-storage-is-readonly-access_storage_readonly\}

사용자의 설정을 변경하려고 할 때 위와 같은 예외가 발생할 수 있습니다.

이 오류를 해결할 수 있는 몇 가지 방법은 다음과 같습니다.

## users.xml을 직접 편집하기 \{#edit-usersxml-directly\}

`/etc/clickhouse-server/users.d` 경로에 있는 `users.xml` 파일에서 특정 사용자에 대해 원하는 설정을 직접 편집하거나 추가할 수 있습니다.

`users.xml`에 대한 자세한 내용은 [여기](/operations/settings/settings-profiles)를 참고하십시오.

## 다른 사용자 생성 \{#create-another-user\}

지정한 설정으로 다른 사용자를 생성한 후, 새 사용자로 ClickHouse에 접속할 수 있습니다.

사용자 생성 방법은 [이 페이지](/sql-reference/statements/create/user)를 참고하십시오.

## SQL 기반 액세스 제어 활성화 \{#enable-sql-driven-access-control\}

기본 사용자에 대해 SQL 기반 액세스 제어와 계정 관리를 활성화할 수 있습니다. 이를 활성화하는 방법은 이 [페이지](/operations/access-rights#enabling-access-control)에 설명되어 있습니다.
