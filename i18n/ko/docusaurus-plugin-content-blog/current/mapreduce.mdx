## MapReduce 같은 것을 왜 사용하지 않을까요? \{#why-not-use-something-like-mapreduce\}

MapReduce와 같은 시스템은 `reduce` 연산이 분산 정렬에 기반한 분산 컴퓨팅 시스템으로 볼 수 있습니다. 이 부류에서 가장 일반적인 오픈 소스 솔루션은 [Apache Hadoop](http://hadoop.apache.org)입니다.

이러한 시스템은 지연 시간이 크기 때문에 온라인 쿼리에는 적합하지 않습니다. 다시 말해, 웹 인터페이스의 백엔드로 사용할 수 없습니다. 이 종류의 시스템은 실시간 데이터 업데이트에도 적합하지 않습니다. 분산 정렬은 연산 결과와 모든 중간 결과(있는 경우)가 단일 서버의 RAM에 위치하는 경우(온라인 쿼리에서 흔한 경우) `reduce` 연산을 수행하는 최선의 방법이 아닙니다. 이러한 경우에는 `reduce` 연산을 수행하는 최적의 방법이 해시 테이블입니다. `map-reduce` 작업을 최적화하는 일반적인 접근 방식은 RAM에서 해시 테이블을 사용한 사전 집계(부분 `reduce`)입니다. 이 최적화는 사용자가 수동으로 수행합니다. 분산 정렬은 단순한 `map-reduce` 작업을 실행할 때 성능 저하의 주요 원인 중 하나입니다.

대부분의 MapReduce 구현은 클러스터에서 임의의 코드를 실행할 수 있도록 지원합니다. 그러나 선언적 쿼리 언어가 실험을 빠르게 수행해야 하는 OLAP 용도에는 더 적합합니다. 예를 들어, Hadoop에는 Hive와 Pig가 있습니다. 또한 Spark용 Cloudera Impala 또는 Shark(구식), 그리고 Spark SQL, Presto, Apache Drill도 고려할 수 있습니다. 이러한 작업을 실행할 때의 성능은 특화된 시스템과 비교하면 매우 비효율적이고, 상대적으로 높은 지연 시간 때문에 이러한 시스템을 웹 인터페이스의 백엔드로 사용하는 것은 비현실적입니다.
