## `system.query_log` 테이블 사용 \{#using-the-systemquery_log-table\}

다음 유용한 쿼리는 실행된 쿼리 중 어떤 것이 메모리를 가장 많이 사용했는지 보여줍니다.

이 쿼리에 대한 몇 가지 설명은 다음과 같습니다.

* 결과는 지난 1일간(`now() - toIntervalDay(1))`)의 데이터로부터 계산되지만, 시간 간격은 쉽게 수정할 수 있습니다.
* `default`라는 이름의 클러스터가 있다고 가정합니다. 이는 [ClickHouse Cloud](https://console.clickhouse.cloud)에서 사용자의 클러스터 이름입니다. 사용 중인 클러스터 이름에 맞게 `default`를 변경하십시오.
* 클러스터가 없는 경우, 이 문서 맨 끝에 나와 있는 쿼리를 참고하십시오.

```sql
SELECT
    count() as nb_query,
    user,
    query,
    sum(memory_usage) AS memory,
    normalized_query_hash
FROM
    clusterAllReplicas(default, system.query_log)
WHERE
    (event_time >= (now() - toIntervalDay(1)))
    AND query_kind = 'Select'
    AND type = 'QueryFinish'
    and user != 'monitoring-internal'
GROUP BY
    normalized_query_hash,
    query,
    user
ORDER BY
    memory DESC;
```

응답은 다음과 같습니다:

```response
┌─nb_query─┬─user────┬─query─────────────────────────────────────────────────────────┬───memory─┬─normalized_query_hash─┐
│       11 │ default │ select version()                                              │ 46178924 │   7202516440347714159 │
│        2 │ default │ SELECT * FROM "system"."table_functions" LIMIT 31 OFFSET 0    │  8391544 │  12830067173062987695 │
└──────────┴─────────┴───────────────────────────────────────────────────────────────┴──────────┴───────────────────────┘
```

:::note
`system.query_log` 테이블이 없다면, 쿼리 로깅이 활성화되어 있지 않을 가능성이 큽니다. 활성화 방법에 대한 자세한 내용은 [`query_log` setting](/operations/server-configuration-parameters/settings#query_log)을 참조하십시오.
:::

:::note[`query_log`에는 제출한 쿼리만 포함되는 것이 아닙니다]
하나의 쿼리를 제출해도 여러 개의 행으로 나타날 수 있습니다. 즉, 초기 쿼리(`is_initial_query = 1`)에 더해, 분산 실행을 위한 보조 쿼리나 뷰를 평가할 때 사용되는 내부 쿼리처럼 파생되거나 내부적인 단계(`is_initial_query = 0`)도 함께 나타날 수 있습니다. 이러한 내부 행에는 ClickHouse가 할당한 자체 `query_id` 값이 있으며, 경우에 따라 `queryView...`와 같은 레이블이 포함되기도 하므로 모든 `query_id`가 클라이언트가 보고한 값과 일치한다고 가정해서는 안 됩니다. 실행한 그대로의 쿼리 기준으로 사용량을 귀속하려면 `is_initial_query = 1`로 필터링하거나 `query_id = initial_query_id`와 일치시키십시오. 자세한 내용은 [가장 비용이 큰 쿼리를 식별하는 방법](/knowledgebase/find-expensive-queries#initial_query_id-vs-query_id) 및 [`query_log` reference](/operations/system-tables/query_log)를 참조하십시오.
:::

클러스터가 없는 경우에는 단일 `system.query_log` 테이블에 직접 쿼리를 실행하면 됩니다:

```sql
SELECT
    count() as nb_query,
    user,
    query,
    sum(memory_usage) AS memory,
    normalized_query_hash
FROM
    system.query_log
WHERE
    (event_time >= (now() - toIntervalDay(1)))
    AND query_kind = 'Select'
    AND type = 'QueryFinish'
    and user != 'monitoring-internal'
GROUP BY
    normalized_query_hash,
    query,
    user
ORDER BY
    memory DESC;
```
