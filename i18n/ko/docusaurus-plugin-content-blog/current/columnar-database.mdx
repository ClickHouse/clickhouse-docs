## 소개 \{#introduction\}

컬럼형 데이터베이스는 각 컬럼의 데이터를 서로 독립적으로 저장합니다. 이렇게 하면 특정 쿼리에 사용되는 컬럼의 데이터만 디스크에서 읽으면 됩니다. 그 대신 전체 행에 영향을 주는 연산의 비용은 그에 비례해 더 커집니다. 컬럼형 데이터베이스는 컬럼 지향 데이터베이스 관리 시스템과 동일한 의미로 사용됩니다. ClickHouse는 이러한 시스템의 전형적인 예입니다.

## 컬럼형 데이터베이스의 장점 \{#advantages-of-a-columnar-database\}

컬럼형 데이터베이스의 주요 장점은 다음과 같습니다:

* 많은 컬럼 중 일부 컬럼만 사용하는 쿼리
* 대용량 데이터에 대한 집계 쿼리
* 컬럼 단위의 데이터 압축

## Row-oriented vs column-oriented databases \{#row-oriented-vs-column-oriented-databases\}

다음은 리포트를 생성할 때 전통적인 행 지향 시스템과 컬럼형 데이터베이스 간의 차이를 보여 주는 그림입니다:

**Traditional row-oriented**

**Columnar**

<Image img={ColumnOriented} alt="컬럼형 데이터베이스 저장 방식의 예시 그림" size="md" />

컬럼형 데이터베이스는 분석 애플리케이션에 가장 적합한 선택입니다. 테이블에 많은 컬럼을 필요할 경우를 대비해 두더라도, 쿼리를 실행할 때 사용되지 않는 컬럼은 읽지 않으므로 추가 읽기 비용이 들지 않기 때문입니다(전통적인 OLTP 데이터베이스는 데이터가 컬럼이 아닌 행 단위로 저장되므로 쿼리 시 모든 데이터를 읽습니다). 컬럼 지향 데이터베이스는 빅데이터 처리와 데이터 웨어하우징을 위해 설계되었습니다. 일반적으로 저가 하드웨어로 구성된 분산 클러스터를 사용해 네이티브하게 확장하면서 처리량을 높입니다. ClickHouse는 이를 [distributed](https://clickhouse.com/docs/engines/table-engines/special/distributed) 테이블과 [replicated](https://clickhouse.com/docs/engines/table-engines/mergetree-family/replication) 테이블을 조합하여 구현합니다.
