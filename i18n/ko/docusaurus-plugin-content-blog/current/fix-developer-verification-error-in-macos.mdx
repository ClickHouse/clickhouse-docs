## MacOS에서 개발자 확인 오류 해결 \{#fix-the-developer-verification-error-in-macos\}

`brew`를 사용하여 ClickHouse를 설치하면 MacOS에서 오류가 발생할 수 있습니다.
기본적으로 MacOS는 검증되지 않은 개발자가 만든 애플리케이션이나 도구를 실행하지 않습니다.

`clickhouse` 명령을 실행하려 할 때 다음과 같은 오류가 표시될 수 있습니다:

<Image img={dev_error} size="md" alt="MacOS 개발자 확인 오류 대화 상자" border />

이 개발자 확인 오류를 해결하려면 시스템 설정 창에서 관련 설정을 변경하거나, 터미널을 사용하거나, ClickHouse를 다시 설치하여 MacOS의 격리(Quarantine) 영역에서 해당 앱을 제거해야 합니다.

## 시스템 설정 절차 \{#system-settings-process\}

`clickhouse` 실행 파일을 격리 영역에서 제거하는 가장 쉬운 방법은 다음과 같습니다.

1. **System Settings(시스템 설정)**을 엽니다.

2. **Privacy &amp; Security(개인정보 보호 및 보안)**로 이동합니다.

   <Image img={privacy_default} size="md" alt="MacOS Privacy & Security settings default view" border />

3. 창 맨 아래로 스크롤하여 「&quot;clickhouse-macos-aarch64&quot; was blocked from use because it is not from an identified developer」라는 메시지를 찾습니다.

4. **Allow Anyway(어쨌든 허용)**를 클릭합니다.

   <Image img={privacy_allow} size="md" alt="MacOS Privacy & Security settings showing Allow Anyway button" border />

5. macOS 사용자 암호를 입력합니다.

이제 터미널에서 `clickhouse` 명령을 실행할 수 있습니다.

## Terminal process \{#terminal-process\}

때때로 `Allow Anyway` 버튼을 눌러도 이 문제가 해결되지 않는 경우가 있습니다. 이 경우 명령줄을 사용해 이 절차를 수행할 수도 있습니다.
또는 단순히 명령줄 사용을 선호할 수도 있습니다.

먼저 Homebrew가 `clickhouse` 실행 파일을 어디에 설치했는지 확인합니다:

```shell
which clickhouse
```

다음과 비슷한 결과가 출력됩니다:

```shell
/opt/homebrew/bin/clickhouse
```

이전 명령에서 얻은 경로를 인자로 지정하여 `xattr -d com.apple.quarantine` 명령을 실행해 격리(quarantine) 영역에서 `clickhouse`를 제거합니다:

```shell
xattr -d com.apple.quarantine /opt/homebrew/bin/clickhouse
```

이제 `clickhouse` 실행 파일을 문제 없이 실행할 수 있어야 합니다:

```shell
clickhouse
```

다음과 비슷한 결과가 출력되어야 합니다:

```shell
Use one of the following commands:
clickhouse local [args]
clickhouse client [args]
clickhouse benchmark [args]
...
```

## ClickHouse를 재설치하여 문제를 해결합니다 \{#fix-the-issue-by-reinstalling-clickhouse\}

Brew에는 애초에 설치된 바이너리가 격리되지 않도록 하는 명령줄 옵션이 있습니다.

먼저 ClickHouse를 제거하십시오:

```shell
brew uninstall clickhouse
```

이제 `--no-quarantine` 옵션을 사용하여 ClickHouse를 재설치하십시오:

```shell
brew install --no-quarantine clickhouse
```
