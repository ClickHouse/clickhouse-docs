## 쿼리에서 쿼리 캐시가 사용되고 있는지 어떻게 확인할 수 있습니까? \{#how-can-i-check-that-query-cache-is-being-used-in-my-query\}

[clickhouse client](https://clickhouse.com/docs/interfaces/cli)와 ClickHouse Cloud 서비스를 사용하는 다음 예제를 참고하십시오.

`query_cache_test` 테이블을 생성합니다.

## clickhouse-client 사용하기 \{#using-clickhouse-client\}

이제 `QueryCache`와 관련된 `TRACE` 로그의 차이를 다음과 같이 살펴봅니다.

첫 번째 실행:

```
[c-red-qc-36-server-0] 2023.05.29 12:06:10.543559 [ 454 ] {3754a7fd-b786-47c1-a258-dfbc75e35a04} \<Trace\> QueryCache: No entry found for query SELECT name FROM query_cache_test  WHERE age > 1000 FORMAT `Null` SETTINGS
[c-red-qc-36-server-0] 2023.05.29 12:06:10.547760 [ 454 ] {3754a7fd-b786-47c1-a258-dfbc75e35a04} \<Trace\> QueryCache: Stored result of query SELECT name FROM query_cache_test  WHERE age > 1000 FORMAT `Null` SETTINGS
```

두 번째 실행 시에는:

```
[c-red-qc-36-server-0] 2023.05.29 12:06:17.932043 [ 454 ] {a047527c-9d55-4e6b-9747-0ccad8787515} \<Trace\> QueryCache: Entry found for query SELECT name FROM query_cache_test  WHERE age > 1000 FORMAT `Null` SETTINGS
```

첫 번째 실행에서는 `No entry found for query SELECT...` 메시지에서 볼 수 있듯이 해당 엔트리가 쿼리 캐시에 존재하지 않았기 때문에, ClickHouse가 `Stored result of query SELECT...`와 같이 쿼리 결과를 캐시에 저장했습니다.

두 번째 실행에서는 `Entry found for query SELECT...` 메시지에서 알 수 있듯이, 이미 저장된 엔트리를 찾아 쿼리 캐시를 사용했습니다.

## SQL만 사용하기 \{#using-just-sql\}

`clickhouse client` trace 로그를 확인하지 않고 SQL 명령만으로도

관련 `system` 테이블을 확인하여 쿼리 캐시가 사용 중인지 검증할 수 있습니다:

```sql
clickhouse-cloud :) SELECT 1 SETTINGS use_query_cache=true;

SELECT 1
SETTINGS use_query_cache = 1

Query id: a5a078c7-61e5-4036-a6f0-4d602d5b72d2

┌─1─┐
│ 1 │
└───┘

1 row in set. Elapsed: 0.001 sec.

clickhouse-cloud :) SELECT 1 SETTINGS use_query_cache=true;

SELECT 1
SETTINGS use_query_cache = 1

Query id: 322ae001-b1ab-463f-ac8d-dc5ba346f3f9

┌─1─┐
│ 1 │
└───┘

1 row in set. Elapsed: 0.001 sec.

clickhouse-cloud :) SELECT * FROM clusterAllReplicas(default,system.query_cache);

SELECT *
FROM clusterAllReplicas(default, system.query_cache)

│ SELECT 1 SETTINGS  │         136 │     0 │      1 │          1 │ 2023-08-02 15:08:23 │ 12188185624808016954 │

1 row in set. Elapsed: 0.005 sec.

┌─event────────────┬─value─┬─description────────────────────────────────────────────────────────────────────────────────────────────┐

└──────────────────┴───────┴────────────────────────────────────────────────────────────────────────────────────────────────────────┘

2 rows in set. Elapsed: 0.006 sec.
```

마지막 결과에서 쿼리 `SELECT 1 SETTINGS use_query_cache=true;` 가 처음 실행되었을 때 `QueryCacheMisses` 가 1번 발생했고, 두 번째 쿼리 실행과 관련된 `QueryCacheHits` 이벤트가 있음을 확인할 수 있습니다.

또한 기본 최대 캐시 항목 크기는 1048576바이트(= 1 MiB)이며, 기본적으로 결과는 캐시에 60초 동안만 저장된다는 점도 기억해 두어야 합니다(예를 들어 `SETTINGS` 에서 `query_cache_ttl=300` 을 사용하면 쿼리 캐시 결과가 5분 동안 저장되도록 할 수 있습니다).

ClickHouse 쿼리 캐시(Query Cache)에 대한 더 자세한 정보는 [여기](https://clickhouse.com/docs/operations/query-cache)에서 확인할 수 있습니다.
