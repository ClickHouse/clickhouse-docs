## 2025년 12월 19일 \{#december-19-2025\}

* 이제 AWS ap-south-1에서 PCI 준수 서비스를 시작할 수 있습니다.
* **통합 사용자 ID 비공개 프리뷰**
  콘솔에서 데이터베이스 사용자를 관리하려는 고객은 SQL 콘솔용 새로운 인증 방식을 활성화할 수 있습니다.
  이를 통해 콘솔에 데이터베이스 사용자 관리 기능이 추가되기 전까지 새로운 인증 방식을 미리 사용해 볼 수 있습니다.
* **이제 S3 ClickPipes에서 비순차 모드를 사용할 수 있습니다**:
  이제 고객은 이벤트 기반 분석을 위해 Amazon S3에서 ClickHouse Cloud로 데이터를 순서에 관계없이 수집할 수 있습니다.
  더 이상 파일이 처리되기 위해 사전식 순서로 정렬되어 있을 필요가 없습니다. 자세한 내용은 [블로그](https://clickhouse.com/blog/clickpipes-s3-unordered-mode) 공지에서 확인할 수 있습니다.
* Fivetran 커넥터가 최근 베타로 전환되었습니다. Fivetran을 사용 중이며 ClickHouse를 대상으로 설정하려는 경우, 이 [문서](https://fivetran.com/docs/destinations/clickhouse/setup-guide)를 확인하십시오.

## 2025년 12월 12일 \{#december-12-2025\}

* **SAML SSO 셀프서비스 설정**

  이제 Enterprise 고객은 지원 티켓 없이도 콘솔에서 SAML 설정을 완료할 수 있습니다.
  또한 SAML 고객은 IdP(Identity Provider)를 통해 추가된 신규 사용자에게 할당할 기본 역할을 설정하고, 사용자 지정 세션 타임아웃도 설정할 수 있습니다.
  자세한 내용은 [docs](/cloud/security/saml-setup)를 참조하십시오.
* **Azure의 최대 레플리카 크기 및 스케일링 제한**

  이제 고객은 `eastus2`를 제외한 모든 Azure 지역에서 최대 레플리카 크기를 356 GiB로 설정할 수 있으며, `eastus2`에서 사용할 수 있는 최대 레플리카 크기는 120 GiB입니다.

## 2025년 11월 7일 \{#november-7-2025\}

* 이제 ClickHouse Cloud 콘솔에서 레플리카 크기를 1 vCPU, 4 GiB 단위로 구성할 수 있습니다.
  이 옵션은 새 서비스를 설정할 때와 설정 페이지에서 최소 및 최대 레플리카 크기를 지정할 때 모두 사용할 수 있습니다.
* 사용자 지정 하드웨어 프로필(Enterprise tier에서 사용 가능)도 이제 유휴 상태(idling)를 지원합니다.
* 이제 ClickHouse Cloud는 AWS Marketplace를 통해 더 간소화된 구매 환경을 제공하며, [종량제(pay-as-you-go)](https://aws.amazon.com/marketplace/pp/prodview-p4gwofrqpkltu?sr=0-2\&ref_=beagle\&applicationId=AWSMPContessa)와 [약정 지출 계약(committed spend contracts)](https://aws.amazon.com/marketplace/pp/prodview-4qyeihstyym2s?sr=0-3\&ref_=beagle\&applicationId=AWSMPContessa)을 각각 선택할 수 있습니다.
* 이제 ClickHouse Cloud의 ClickStack 사용자도 경고 기능을 사용할 수 있습니다.
  이제 추가 설정이나 추가 인프라 또는 서비스, 별도 구성 없이 로그, 메트릭, 트레이스 전반에 걸쳐 HyperDX UI에서 직접 경고를 생성하고 관리할 수 있습니다. 경고는 Slack, PagerDuty 등과 통합됩니다.
  자세한 내용은 [경고 문서](/use-cases/observability/clickstack/alerts)를 참조하십시오.

## 2025년 10월 17일 \{#october-17-2025\}

* **서비스 모니터링 - 리소스 사용량 대시보드**
  CPU 사용률 및 메모리 사용률 메트릭은 평균값 대신 특정 기간의 최대 사용률을 표시하도록 변경되어, 과소 프로비저닝된 인스턴스를 더 쉽게 파악할 수 있습니다.
  또한 CPU 사용률 메트릭에는 ClickHouse Cloud의 오토스케일러가 사용하는 메트릭과 더 유사한 Kubernetes 수준의 CPU 사용률 메트릭이 표시됩니다.
* **외부 버킷**
  이제 ClickHouse Cloud에서는 백업을 자체 클라우드 공급자 계정으로 직접 내보낼 수 있습니다.
  AWS S3, Google Cloud Storage 또는 Azure Blob Storage의 외부 스토리지 버킷을 연결하여 백업을 직접 관리하세요.

## 2025년 08월 29일 \{#august-29-2025\}

* [ClickHouse Cloud Azure Private Link](/cloud/security/azure-privatelink)에서 리소스를 식별할 때 Resource GUID 필터 대신 Resource ID 필터를 사용하도록 변경되었습니다. 기존 Resource GUID는 이전 버전과 호환되므로 계속 사용할 수 있지만, Resource ID 필터로 전환할 것을 권장합니다. migration에 대한 자세한 내용은 Azure Private Link용 [문서](/cloud/security/azure-privatelink#obtaining-private-endpoint-resourceid)를 참조하십시오.

## 2025년 8월 22일 \{#august-22-2025\}

* **AWS Glue용 ClickHouse Connector**
  이제 [AWS Marketplace](https://aws.amazon.com/marketplace/pp/prodview-eqvmuopqzdg7s)에서 제공되는 공식 [AWS Glue용 ClickHouse Connector](/integrations/glue)를 사용할 수 있습니다. 이 커넥터는 AWS Glue의 Apache
  Spark 기반 서버리스 엔진을 활용하여 ClickHouse와 다른 데이터 소스 간의 데이터 추출, 변환, 적재(ETL) 통합을 수행합니다. ClickHouse와 Spark 간에 테이블을 생성하고 데이터를 쓰고 읽는 방법은 안내
  [블로그 게시물](http://clickhouse.com/blog/clickhouse-connector-aws-glue)을 따라 시작하십시오.
* **서비스의 최소 레플리카 수 변경**
  스케일 업된 서비스는 이제 [다시 스케일 다운](/manage/scaling)하여 단일 레플리카를 사용할 수 있습니다(이전 최소값은 2개의 레플리카였습니다). 주의: 단일 레플리카 서비스는 가용성이 낮으므로 프로덕션 환경에서는 권장되지 않습니다.
* ClickHouse Cloud는 기본적으로 관리자 역할에 대해 서비스 스케일링 및 서비스 버전 업그레이드와 관련된 알림을 보내기 시작합니다. 관리자 역할 사용자는 알림 설정에서 해당 알림 기본 설정을 조정할 수 있습니다.

## 2025년 8월 13일 \{#august-13-2025\}

* **MongoDB CDC용 ClickPipes, 이제 비공개 프리뷰로 제공됩니다**
  이제 ClickPipes를 사용해 몇 번의 클릭만으로 MongoDB 데이터를 ClickHouse Cloud로 복제할 수 있으며,
  외부 ETL 도구 없이도 실시간 분석이 가능합니다. 이 커넥터는 지속적인
  복제와 일회성 마이그레이션을 모두 지원하며, MongoDB Atlas 및 셀프 호스팅 MongoDB
  배포 환경과 호환됩니다. MongoDB CDC 커넥터의 개요는 [블로그 게시물](https://clickhouse.com/blog/mongodb-cdc-clickhouse-preview)에서 확인할 수 있으며, [여기에서 얼리 액세스를 신청할 수 있습니다](https://clickhouse.com/cloud/clickpipes/mongodb-cdc-connector)!

## 2025년 8월 8일 \{#august-08-2025\}

* **알림**: 이제 서비스가 새 ClickHouse 버전으로 업그레이드되기 시작하면 UI 알림을 받습니다. 추가 Email 및 Slack 알림은 알림 센터를 통해 설정할 수 있습니다.
* **ClickPipes**: Azure Blob Storage (ABS) ClickPipes 지원이 ClickHouse Terraform 프로바이더에 추가되었습니다. ABS ClickPipe를 프로그래밍 방식으로 생성하는 방법은 프로바이더 문서의 예시를 참조하십시오.
  * [버그 수정] Null 엔진을 사용하는 대상 테이블에 쓰는 객체 스토리지 ClickPipes가 이제 UI에서 &quot;총 레코드&quot; 및 &quot;수집된 데이터&quot; 메트릭을 표시합니다.
  * [버그 수정] UI의 메트릭 &quot;기간&quot; 선택기는 선택한 기간과 관계없이 항상 기본값인 &quot;24시간&quot;으로 표시되는 문제가 있었습니다. 이제 이 문제가 수정되어 UI가 선택한 기간에 맞게 차트를 올바르게 업데이트합니다.
* **교차 리전 PrivateLink (AWS)** 는 이제 일반 제공됩니다. 지원되는 리전 목록은 [문서](/manage/security/aws-privatelink)를 참조하십시오.

## 2025년 07월 31일 \{#july-31-2025\}

**이제 ClickPipes에서 수직 스케일링을 사용할 수 있습니다**

[이제 스트리밍 ClickPipes에서 수직 스케일링을 사용할 수 있습니다](https://clickhouse.com/blog/clickpipes-flexible-scaling-monitoring).
이 기능을 사용하면 레플리카 수(수평 스케일링)뿐만 아니라 각
레플리카의 크기까지 제어할 수 있습니다. 이제 각 ClickPipe의 세부 정보 페이지에는
레플리카별 CPU 및 메모리 사용률도 표시되므로,
워크로드를 더 잘 이해하고 크기 재조정 작업을 보다 확신 있게 계획하는 데 도움이 됩니다.

## 2025년 7월 24일 \{#july-24-2025\}

**MySQL용 ClickPipes CDC가 이제 공개 베타로 제공됩니다**

ClickPipes의 MySQL CDC 커넥터가 이제 공개 베타로 널리 제공됩니다. 몇 번만 클릭하면
외부 의존성 없이 MySQL(또는 MariaDB) 데이터를 ClickHouse Cloud로 실시간 복제할 수 있습니다.
커넥터 개요는 [블로그 게시물](https://clickhouse.com/blog/mysql-cdc-connector-clickpipes-beta)에서 확인하고,
빠르게 시작하려면 [퀵스타트](https://clickhouse.com/docs/integrations/clickpipes/mysql)를 따르십시오.

## 2025년 07월 11일 \{#june-11-2025\}

* 이제 신규 서비스는 데이터베이스 및 테이블 메타데이터를 중앙 **SharedCatalog**에 저장합니다.
  이는 coordination과 객체 수명 주기를 위한 새로운 모델로, 다음을 가능하게 합니다:

  * 높은 동시성 환경에서도 **Cloud 규모의 DDL**
  * **안정적인 삭제 및 새로운 DDL 작업**
  * 이제 무상태 노드가 디스크 의존성 없이 시작되므로 **빠른 기동 및 재활성화**
  * Iceberg 및 Delta Lake를 포함해 **네이티브 형식과 오픈 형식 전반에서의 무상태 컴퓨트**

  SharedCatalog에 대한 자세한 내용은 [블로그](https://clickhouse.com/blog/clickhouse-cloud-stateless-compute)에서 확인하십시오

* 이제 GCP `europe-west4`에서 HIPAA 준수 서비스를 시작할 수 있습니다

## 2025년 06월 13일 \{#june-13-2025\}

* ClickHouse Cloud 대시보드가 이제 일반 제공됩니다. 대시보드를 사용하면 쿼리를 시각화하고, 필터와 쿼리 매개변수를 통해 데이터와 상호 작용하며, 공유를 관리할 수 있습니다.
* API 키 IP 필터: ClickHouse Cloud 사용 시 보안을 한층 강화할 수 있는 기능을 도입합니다. API 키를 생성할 때 API 키를 사용할 수 있는 위치를 제한하는 IP 허용 목록을 설정할 수 있습니다. 자세한 내용은 [문서](https://clickhouse.com/docs/cloud/security/setting-ip-filters)를 참조하십시오.

## 2025년 05월 30일 \{#may-30-2025\}

* ClickHouse Cloud에서 **ClickPipes for Postgres CDC**의 일반 제공을 시작했음을 알려드립니다. 이제 몇 번의 클릭만으로 Postgres
  데이터베이스를 복제하고 초고속 실시간 분석을 활용할 수 있습니다. 이 커넥터는
  더 빠른 데이터 동기화, 몇 초 수준까지 낮아진 지연 시간, 자동 schema 변경,
  완전한 보안 연결 등을 제공합니다. 자세한 내용은
  [블로그](https://clickhouse.com/blog/postgres-cdc-connector-clickpipes-ga)를
  참조하십시오. 시작하려면 [여기](https://clickhouse.com/docs/integrations/clickpipes/postgres)의 안내를 참조하십시오.

* SQL 콘솔 대시보드에 다음과 같은 개선 사항이 추가되었습니다:
  * 공유: 팀 구성원과 대시보드를 공유할 수 있습니다. 전역 또는 사용자별로 조정할 수 있는 4가지 액세스 수준을 지원합니다:
    * *쓰기 권한*: 시각화 추가/편집, 갱신 설정, 필터를 사용한 대시보드 상호작용.
    * *소유자*: 대시보드 공유, 대시보드 삭제, 그리고 &quot;쓰기 권한&quot; 사용자의 모든 기타 권한.
    * *읽기 전용 권한*: 대시보드를 보고 필터를 통해 상호작용
    * *액세스 없음*: 대시보드를 볼 수 없음
  * 이미 생성된 기존 대시보드의 경우, 조직 관리자는 해당 대시보드의 소유자를 자신으로 지정할 수 있습니다.
  * 이제 SQL 콘솔의 쿼리 보기에서 테이블이나 차트를 대시보드에 추가할 수 있습니다.

## 2025년 05월 16일 \{#may-16-2025\}

### ClickHouse Cloud 티어 변경 사항 \{#changes-to-clickhouse-cloud-tiers\}

ClickHouse는 끊임없이 변화하는 고객 요구 사항에 맞춰 제품을 발전시키는 데 전념하고 있습니다. 지난 2년간 일반 제공(GA) 이후 ClickHouse Cloud는 크게 발전했으며, 고객이 Cloud 서비스를 어떻게 활용하는지에 대한 매우 중요한 통찰도 얻었습니다.

워크로드에 맞는 ClickHouse Cloud 서비스의 사이징과 비용 효율성을 최적화하기 위해 새로운 기능을 도입하고 있습니다. 여기에는 **컴퓨트-컴퓨트 분리**, 고성능 머신 유형, 그리고 **단일 레플리카 서비스**가 포함됩니다. 또한 **자동 스케일링**과 관리형 업그레이드가 더 매끄럽고 신속하게 이루어지도록 개선하고 있습니다.

가장 까다로운 고객과 워크로드의 요구를 충족하기 위해 **새로운 Enterprise 티어**도 추가하고 있습니다. 이 티어는 산업별 보안 및 규정 준수 기능, 기본 하드웨어와 업그레이드에 대한 한층 더 세밀한 제어, 그리고 고급 재해 복구 기능에 중점을 둡니다.

이러한 변경을 지원하기 위해 현재의 **개발** 및 **프로덕션** 티어를, 변화하는 고객층의 사용 방식에 더 잘 맞도록 재구성하고 있습니다. 새로운 아이디어와 프로젝트를 시험하는 사용자를 위한 **Basic** 티어와, 대규모 프로덕션 워크로드 및 데이터를 다루는 사용자를 위한 **Scale** 티어를 도입하고 있습니다.

이러한 내용과 기타 기능 변경 사항은 이 [블로그](https://clickhouse.com/blog/evolution-of-clickhouse-cloud-new-features-superior-performance-tailored-offerings)에서 확인할 수 있습니다. 기존 고객은 [새 계획](https://clickhouse.com/pricing)을 선택하기 위해 조치를 취해야 합니다. 고객 대상 안내는 조직 관리자에게 이메일로 발송되었습니다.

### 단일 레플리카 서비스 \{#single-replica-services\}

독립 실행형 제공 방식과 웨어하우스 내 구성 모두에 대해 &quot;단일 레플리카 서비스&quot;라는 개념을 도입합니다. 독립 실행형 제공 방식에서는 단일 레플리카 서비스의 크기에 제한이 있으며, 소규모 테스트 워크로드에 사용하도록 설계되었습니다. 웨어하우스 내에서는 단일 레플리카 서비스를 더 큰 규모로 데프로이할 수 있으며, 재시작 가능한 ETL 작업처럼 대규모 고가용성이 필요하지 않은 워크로드에 활용할 수 있습니다.

### 수직 자동 스케일링 개선 사항 \{#vertical-auto-scaling-improvements\}

컴퓨트 레플리카를 위한 새로운 수직 스케일링 메커니즘인 &quot;Make Before Break&quot;(MBB)를 도입합니다. 이 방식은 기존 레플리카를 제거하기 전에 새 크기의 레플리카를 하나 이상 먼저 추가하므로, 스케일링 작업 중 용량 손실을 방지합니다. 기존 레플리카를 제거하고 새 레플리카를 추가하는 사이의 공백을 없애 MBB는 더 원활하고 중단 영향이 적은 스케일링 프로세스를 제공합니다. 특히 높은 리소스 사용률로 인해 추가 용량이 필요한 스케일 업 상황에서 더욱 효과적입니다. 레플리카를 너무 일찍 제거하면 리소스 제약이 오히려 더 심해지기 때문입니다.

### 관리형 업그레이드 개선 사항 \{#managed-upgrade-improvements\}

안전한 관리형 업그레이드는 데이터베이스에 새 기능이 추가되고 계속 발전하는 과정에서도 최신 상태를 유지할 수 있게 해 주므로 사용자에게 큰 가치를 제공합니다. 이번 롤아웃에서는 업그레이드에 &quot;make before break&quot;(또는 MBB) 방식을 적용하여 실행 중인 워크로드에 미치는 영향을 더욱 줄였습니다.

[Golang](https://github.com/ClickHouse/clickhouse-go/releases/tag/v2.30.1), [Python](https://github.com/ClickHouse/clickhouse-connect/releases/tag/v0.8.11), [NodeJS](https://github.com/ClickHouse/clickhouse-js/releases/tag/1.10.1) 클라이언트에 Dynamic, Variant, JSON 타입 지원이 추가되었습니다.

DBT는 이제 `1.8.7` 릴리스부터 [갱신 가능 구체화 뷰를 지원합니다](https://github.com/ClickHouse/dbt-clickhouse/releases/tag/v1.8.7).

### JWT 토큰 지원 \{#jwt-token-support\}

JDBC driver v2, clickhouse-java, [Python](https://github.com/ClickHouse/clickhouse-connect/releases/tag/v0.8.12), and[ NodeJS](https://github.com/ClickHouse/clickhouse-js/releases/tag/1.10.0) 클라이언트에 JWT 기반 인증 지원이 추가되었습니다.

JDBC / Java는 출시되면[ 0.8.0](https://github.com/ClickHouse/clickhouse-java/releases/tag/v0.8.0)에 포함될 예정이며, 출시 예상 시점은 아직 정해지지 않았습니다.

### Prometheus 통합 개선 사항 \{#prometheus-integration-improvements\}

Prometheus 통합에 몇 가지 기능이 개선되었습니다.

* **조직 수준 엔드포인트**. ClickHouse Cloud용 Prometheus 통합이 개선되었습니다. 이제 API는 서비스 수준 메트릭뿐 아니라 **조직 수준 메트릭**을 위한 엔드포인트도 제공합니다. 이 새로운 엔드포인트는 조직 내 모든 서비스의 메트릭을 자동으로 수집하므로, Prometheus collector로 메트릭을 내보내는 프로세스를 간소화합니다. 또한 이러한 메트릭은 Grafana 및 Datadog와 같은 시각화 도구와 연동하여 조직 성능을 더 종합적으로 파악하는 데 활용할 수 있습니다.

  이 기능은 현재 모든 사용자가 이용할 수 있습니다. 자세한 내용은 [여기](/integrations/prometheus)에서 확인할 수 있습니다.

* **필터링된 메트릭**. ClickHouse Cloud용 Prometheus 통합에서 필터링된 메트릭 목록을 반환하는 기능이 추가되었습니다. 이 기능을 사용하면 서비스 상태 모니터링에 중요한 메트릭에 집중할 수 있어 응답 payload 크기를 줄이는 데 도움이 됩니다.

  이 기능은 API의 선택적 쿼리 매개변수를 통해 사용할 수 있으므로, 데이터 수집을 더 쉽게 최적화하고 Grafana 및 Datadog와 같은 도구와의 통합도 간소화할 수 있습니다.

  필터링된 메트릭 기능은 현재 모든 사용자가 이용할 수 있습니다. 자세한 내용은 [여기](/integrations/prometheus)에서 확인할 수 있습니다.
