## 2026년 6월 3일 \{#2026-06-03\}

Open House 2026 주간에는 데이터가 있는 곳에서 ClickHouse를 실행하고, 에이전트형 AI 지원 워크플로를 플랫폼에 도입하는 데 중점을 둔 다양한 출시가 있었습니다.

### ClickPipes가 이제 Private Service Connect와 함께 GCP에서 네이티브로 실행됩니다 \{#clickpipes-gcp-psc\}

이제 5월 26일 이후 생성된 모든 신규 GCP 서비스에서 ClickPipes 인프라는 ClickHouse Cloud 서비스와 동일한 GCP 리전에서 실행되며, Private Service Connect(PSC)도 지원합니다. 이를 통해 클라우드 간 송신 비용을 없애고, 지연 시간을 줄이며, 데이터를 로컬에 유지하고, SSH 터널링 같은 우회 방식 대신 GCP 네이티브 프라이빗 네트워킹을 통해 GCP 소스에 연결할 수 있습니다. 기존 서비스는 ClickHouse 팀의 도움을 받아 마이그레이션할 수 있습니다.

자세한 내용은 [ClickPipes static IP list](/integrations/clickpipes#list-of-static-ips)를 참조하십시오.

### GCP Pub/Sub용 ClickPipes (비공개 프리뷰) \{#clickpipes-gcp-pubsub\}

이제 ClickPipes를 통해 GCP Pub/Sub topic의 데이터를 ClickHouse Cloud로 직접 스트리밍할 수 있으므로, GCP 네이티브 팀은 Cloud Logging, Cloud Monitoring, GKE 등의 데이터를 바로 분석할 수 있습니다. ClickPipes UI, OpenAPI, Terraform에서 사용할 수 있으며, 비공개 프리뷰 기간에는 [등록](https://clickhouse.com/cloud/clickpipes#pubsub-private-preview)이 필요합니다.

설정 및 구성에 대한 자세한 내용은 [GCP Pub/Sub용 ClickPipes 문서](/integrations/clickpipes/pubsub)를 참조하십시오.

### ClickHouse Agents: Claude 기반 에이전트 분석(공개 베타) \{#clickhouse-agents\}

ClickHouse Agents를 이제 [ai.clickhouse.cloud](http://ai.clickhouse.cloud)에서 공개 베타로 사용할 수 있습니다 — ClickHouse Cloud 데이터에 대해 대화형으로 스키마를 이해하는 분석을 수행할 수 있도록 Claude로 구동되는 ClickHouse의 에이전트 플랫폼입니다. 코드 없이 AgentBuilder로 사용자 지정 에이전트를 만들고, MCP 서버(Slack, Linear, Notion 등)를 연결하고, 하위 에이전트 워크플로를 구성하고, 대화 흐름을 벗어나지 않은 채 쿼리, 파이프라인, 앱을 생성할 수 있습니다. AWS Bedrock AgentCore 통합과 관리형 LibreChat 기반도 포함됩니다.

자세한 내용은 [ClickHouse Agents documentation](/cloud/features/ai-ml/agents)에서 확인하십시오.

### ClickHouse Postgres (공개 베타) \{#clickhouse-postgres-public-beta\}

로컬 NVMe storage를 사용하는 완전관리형 Postgres가 이제 공개 베타로 제공됩니다. AWS RDS보다 5배 이상 높은 TPS를 제공하며, 하나의 관리형 스택에서 OLTP와 OLAP를 아우릅니다. 네이티브 CDC 파이프라인은 중간 인프라 없이 데이터를 ClickHouse로 직접 스트리밍하며, 오픈소스 `pg_clickhouse` 확장 기능을 사용하면 표준 Postgres 세션에서 ClickHouse 테이블을 쿼리할 수 있습니다.

자세한 내용은 [Open House 2026 Day 1: Real-time data without lock-in and what teams can build next](https://clickhouse.com/blog/open-house-2026-day-1#postgres-announcements)에서 확인하십시오.

### ClickStack Cloud: 서버리스 관측성(비공개 프리뷰) \{#clickstack-cloud\}

ClickStack Cloud는 ClickHouse Cloud를 기반으로 하는 완전 관리형 서버리스 관측성 플랫폼입니다. OpenTelemetry Collector가 관리형 OTLP 엔드포인트를 가리키도록 설정하면 ClickStack UI에서 로그, 메트릭, 트레이스를 즉시 탐색할 수 있으며, 수집, 버퍼링, 스케일링, 스토리지, 쿼리 처리, 인증, RBAC는 모두 ClickStack Cloud가 처리합니다. 현재 AWS us-east-1에서 제한된 비공개 프리뷰로 제공됩니다.

자세한 내용은 [ClickStack Cloud private preview](https://clickhouse.com/blog/clickstack-cloud-private-preview) 및 [ClickStack Cloud waitlist](https://clickhouse.com/cloud/clickstack-cloud-waitlist)를 참고하십시오.

### Managed ClickStack용 AI Notebooks (베타) \{#clickstack-ai-notebooks\}

AI Notebooks는 Managed ClickStack에 AI 지원 인시던트 조사를 위한 영구적이고 투명한 워크스페이스를 제공합니다. 프롬프트, 쿼리, 차트, 추론으로 구성된 구조화된 흐름을 캡처하고, 여러 가설을 탐색할 수 있도록 분기하며, 조사 내용을 팀 협업용 아티팩트로 공유할 수 있습니다. 생성된 모든 쿼리와 결과는 표시되며 편집도 가능합니다. ClickStack UI의 왼쪽 탐색 메뉴에서 액세스할 수 있습니다.

자세한 내용은 다음을 참조하십시오.

* [ClickStack AI Notebooks 문서](/use-cases/observability/clickstack/notebooks)
* [베타 버전의 AI Notebooks(블로그)](https://clickhouse.com/blog/observability-mcp-server-ai-notebooks#ai-notebooks-in-beta)

### 수평 자동 스케일링(비공개 프리뷰) \{#horizontal-autoscaling\}

수평 자동 스케일링은 ClickHouse Cloud 클러스터의 레플리카 수를 자동으로 조정합니다. 수요가 급증하면 빠르게 확장하고, 부하가 줄어들면 점진적으로 축소하므로 변동이 크고 동시 쿼리 수가 많은 워크로드를 운영하는 팀은 더 이상 클러스터 규모를 수동으로 맞출 필요가 없습니다. 비공개 프리뷰에서는 동시 쿼리 수를 기준으로 스케일링하며, CPU 및 메모리 신호 기반 스케일링은 공개 베타에서 지원할 예정입니다. ClickHouse 26.2+를 사용하는 Enterprise 및 Scale tier 고객에게 제공됩니다.

### ClickStack MCP Server (오픈 소스) \{#clickstack-mcp-server\}

새로운 오픈 소스 ClickStack MCP Server는 ClickStack 및 ClickHouse를 기반으로 하는 구조화된 관측성 조사 도구를 외부 AI agents에 제공합니다. 이 서버는 logs, metrics, traces를 위한 시맨틱 프리미티브를 제공하며, 로그 패턴 분석, trace 이상치 조사, 상관관계 분석, 신호 간 debugging을 지원합니다. 또한 더 심층적인 탐색을 위한 네이티브 SQL 액세스를 유지하면서 dashboards, 검색, alerts를 영구 저장할 수 있습니다. Claude Code, Cursor, Codex 또는 사용자 지정 프레임워크 같은 agents를 연결하세요.

자세한 내용은 [ClickStack MCP server docs](/use-cases/observability/clickstack/mcp) 및 [Observability MCP server + AI Notebooks blog post](https://clickhouse.com/blog/observability-mcp-server-ai-notebooks#clickstack-mcp-server)를 참조하십시오.

### 실행형 사용자 정의 함수 \{#executable-udfs\}

{/* vale on */ }

이제 Python으로 함수를 작성해 zip으로 묶은 뒤 클러스터에 업로드하고, 다른 기본 제공 함수처럼 SQL에서 호출할 수 있습니다. ClickHouse는 오래 실행되는 샌드박스 프로세스 풀을 운영하며, 쿼리 속도로 행을 이 프로세스들에 전달하므로 SQL이 동작하는 곳이라면 어디서나 함수를 호출할 수 있습니다. 즉, 즉석 쿼리, 조인, 심지어 삽입이 일어날 때마다 실행되는 materialized view에서도 사용할 수 있습니다. 모델 코드는 데이터 바로 옆에 두고, 배포는 Cloud Console의 업로드 화면 한 번이면 끝납니다.

어떤 작업이 가능한지 보여드리기 위해 [demo](https://github.com/ClickHouse/stock-anomaly-udf)를 만들었습니다. StockHouse의 실시간 주식 거래 데이터를 바탕으로, 데이터가 수집되는 시점에 주식 거래 틱의 이상 징후를 점수화하는 소형 PyTorch 오토인코더를 학습시켰습니다. UDF 하나와 DDL 문 하나만 있으면 되고, 나머지는 모두 일반 SQL입니다. 또한 외부 HTTPS 호출을 할 수 있게 해 주는 네트워크 액세스 UDF(비공개 베타) 미리 보기도 함께 선보였습니다. 저희는 이를 활용해 뉴스와 SEC 공시를 가져오고, 특정 거래가 왜 이상 징후로 보이는지 Claude가 분류하도록 했습니다.

자세한 내용은 [announcement blog](https://clickhouse.com/blog/executable-udfs-clickhouse-cloud-beta) 또는 [docs](/cloud/features/user-defined-functions)를 참조하십시오.

## 2026년 5월 18일 \{#2026-05-18\}

### Python client v1.0.0 \{#python-client-v1\}

ClickHouse용 Python client인 [clickhouse-connect](/integrations/python)의 주요 버전 릴리스입니다. 핵심 변경 사항은 다음과 같습니다:

* ClickHouse 전용 SQL을 더 폭넓게 지원하는 완성도 높은 SQLAlchemy 방언
* 스트리밍을 포함해 모든 기능을 동일하게 제공하는 aiohttp 기반 네이티브 비동기 client
* DateTime, DateTime64, 고정 폭 숫자형 삽입, 맵, Decimals, BigDecimals의 성능 대폭 향상
* 새로운 `tz_mode` 및 `tz_source` 매개변수를 포함한 시간대 처리 방식 재설계
* 네이티브 Variant 컬럼 쓰기 지원
* 선택적 의존성(numpy, pandas, pyarrow, polars)에 대한 지연 import를 통해 기본 import 속도 4배 향상
* 실험적 Python 3.14 free-threading 지원

**호환성에 영향을 주는 변경 사항:** Python &lt; 3.9 및 Pandas 1.x 지원이 중단되었습니다. 시간대 관련 기타 호환성 변경 사항은 [release notes](https://github.com/ClickHouse/clickhouse-connect/releases)를 검토하십시오.

### ClickStack의 SQL 기반 차트 및 알림 \{#clickstack-sql-charting-alerting\}

[ClickStack](/use-cases/observability/clickstack)에서 이제 임의의 ClickHouse SQL 쿼리를 사용해 차트와 알림을 구성할 수 있습니다. 사용자는 대시보드와 알림 빌더에서 raw SQL을 직접 작성할 수 있으며, 윈도우 함수, CTE, 다단계 쿼리, ClickHouse 네이티브 분석 함수를 모두 지원합니다. SQL 알림을 사용하면 알림 판단 로직 전체를 쿼리 자체에 담을 수 있어 이상 탐지, 후행 평균, 통계적 스파이크 탐지를 구현할 수 있습니다. 모든 ClickStack 배포 환경(Cloud, Private, BYOC, OSS)에서 사용할 수 있습니다. 자세한 내용은 [SQL 기반 시각화](/use-cases/observability/clickstack/dashboards/sql-visualizations) 및 [SQL 알림 결과 해석](/use-cases/observability/clickstack/alerts#sql-result-interpretation)을 참조하십시오.

## 2026년 5월 8일 \{#2026-05-08\}

### Postgres 쿼리 인사이트 (비공개 프리뷰) \{#postgres-query-insights\}

이제 [ClickHouse Cloud Managed Postgres](/cloud/managed-postgres)에서 **Query insights** 탭을 사용할 수 있습니다(비공개 프리뷰). 이 탭은 모든 쿼리 패턴을 영향도별로 순위를 매기고, 각 패턴이 느린 이유를 다음 3가지 영역에서 보여줍니다.

* **개요** — 한 화면에서 확인하는 데이터베이스 상태
* **패턴** — 실행 시간, CPU, 오류, P95 기준으로 정렬할 수 있는 테이블
* **패턴별 플라이아웃** — 지연 시간 백분위수, CPU 대비 I/O, 캐시 대비 디스크, 임시 스필, 병렬 워커 부족, WAL 볼륨

오픈 소스 Postgres 확장 기능인 [pg&#95;stat&#95;ch](https://github.com/ClickHouse/pg_stat_ch)를 기반으로 합니다.

### Data Catalog 통합 확장 \{#data-catalog-integration-expansions\}

* [AWS Glue Catalog](/use-cases/data-lake/glue-catalog)에 대한 IAM 지원
* Data Catalog 통합을 위한 [BigLake 메타스토어](/use-cases/data-lake/biglake-catalog) 지원
* [Unity Catalog](/use-cases/data-lake/unity-catalog)에서 Delta Tables 지원

### 인덱스 세그먼트 분할 (비공개 프리뷰) \{#index-sharding\}

인덱스 세그먼트 분할을 이제 비공개 프리뷰로 사용할 수 있습니다. 이 기능은 인덱스 분석 단계를 여러 레플리카에 분산하여 각 레플리카의 메모리 사용량을 줄이고, 분산 병렬화를 통해 쿼리 성능을 향상시킵니다. 이는 특히 벡터 검색 및 전문 검색과 같은 보조 인덱스 비중이 큰 테이블에서 효과적입니다. 자세한 내용은 [블로그 게시물](https://clickhouse.com/blog/index-sharding-clickhouse-cloud-petabyte-scale-indexing)을 참조하십시오.

### ClickPipes용 Terraform 및 OpenAPI(일반 제공) \{#terraform-openapi-clickpipes-ga\}

ClickHouse [Terraform 프로바이더](https://registry.terraform.io/providers/ClickHouse/clickhouse/latest/docs)의 ClickPipes 리소스는 이제 안정 릴리스의 일부로 일반 제공되며, [OpenAPI 엔드포인트](https://clickhouse.com/docs/cloud/manage/api/swagger)도 더 이상 베타로 표시되지 않습니다. 여기에는 CDC 커넥터(Postgres, MySQL, MongoDB)에 대한 전면 지원이 포함됩니다. 자세한 내용은 [Terraform 문서](/integrations/clickpipes/programmatic-access/terraform), [OpenAPI 문서](/integrations/clickpipes/programmatic-access/openapi), [블로그 게시물](https://clickhouse.com/blog/terraform-ga)을 참조하십시오.

이제 고객은 Cloud Console의 청구 UI에서 모든 환불 내역을 직접 확인할 수 있습니다. 환불은 처리되면 표시되며, 신용카드 환불의 경우 은행에 따라 명세서에 반영되기까지 추가 시간이 소요될 수 있습니다.

### 조직 지출 알림(비공개 프리뷰) \{#org-spend-alerts\}

조직 지출 알림이 이제 비공개 프리뷰로 제공됩니다. 설정한 조직 지출 금액이 각 청구 기간 동안 50%, 75%, 100%에 도달하면 알림을 받을 수 있습니다. 이메일, Cloud Console 및/또는 Slack 알림 채널 중에서 선택할 수 있습니다. 알림은 조직의 총 사용량을 기준으로 하며, 매시간 발생하고 각 청구 기간마다 자동으로 재설정됩니다. Org Admin 또는 Billing Admin 역할이 필요합니다.

### AWS의 ClickPipes 지원 리전 확대 \{#clickpipes-aws-region-parity\}

이제 ClickPipes를 ClickHouse Cloud가 지원하는 모든 AWS 리전(퍼블릭 및 프라이빗)에서 사용할 수 있으며, 지원 리전이 6개에서 18개로 확대되었습니다. 여기에는 싱가포르(ap-southeast-1), 서울(ap-northeast-2), 도쿄(ap-northeast-1)와 같은 주요 리전이 포함됩니다. 새 리전은 2026년 4월 14일 이후에 생성된 서비스에서 사용할 수 있습니다. 자세한 내용은 [문서](/integrations/clickpipes#list-of-static-ips)를 참조하십시오.

## 2026년 4월 10일 \{#2026-04-10\}

### 중앙 집중식 마켓플레이스 과금 \{#centralized-marketplace-billing\}

이제 고객은 단일 클라우드 마켓플레이스 구독(AWS, Azure, GCP)을 통해 여러 조직의 과금을 통합할 수 있습니다. 주요 변경 사항은 다음과 같습니다.

* **마켓플레이스 구독 공유:** 조직의 과금을 신용 카드 결제에서 다른 조직의 기존 마켓플레이스 구독으로 전환할 수 있으므로 추가 마켓플레이스 구독을 설정할 필요가 없습니다. PAYG 사용량만 통합되며 선불 크레딧은 제외됩니다.
* **마켓플레이스 구독 업데이트:** 조직의 마켓플레이스 구독을 다른 조직에서 사용하는 구독으로 변경할 수 있습니다.
* **백업 결제 수단:** 마켓플레이스 과금을 사용하는 조직은 기본 마켓플레이스 구독이 취소되거나 만료될 경우에 대비해 예비 결제 수단으로 신용 카드를 추가할 수 있습니다.

자세한 내용은 [과금 문서](/manage/manage/billing/managing-payment-methods)를 참조하십시오.

## 2026년 4월 3일 \{#april-3-2026\}

* **두 개의 윈도우를 사용하는 더 스마트한 자동 확장 추천기:** ClickHouse Cloud는 이제 수직 자동 확장에 두 개의 조회 기간 윈도우를 결합한 방식을 사용합니다. 기존의 단일 30시간 윈도우 대신, 3시간 &quot;소형 윈도우&quot;와 30시간 &quot;대형 윈도우&quot;를 함께 사용합니다. 이를 통해 축소 지연 시간을 최대 30시간에서 최소 3시간까지 줄여, 확장 응답성은 유지하면서도 가변적인 워크로드의 인프라 비용을 낮출 수 있습니다. 자세한 내용은 [블로그 게시물](https://clickhouse.com/blog/smarter-auto-scaling)을 참조하십시오.
* **Cloud Console의 모니터링:** 새로운 개요 대시보드와 인프라 대시보드를 통해 ClickHouse 서버 동작에 대한 가시성이 향상되었습니다. 이제 관리자 사용자는 ClickHouse Cloud 사용 중 발생하는 일반적인 문제에 대해 이메일 알림을 받습니다. 자세한 내용은 [블로그 게시물](https://clickhouse.com/blog/clickhouse-cloud-new-monitoring-capabilities)을 참조하십시오.
* **컴퓨트-컴퓨트 분리(warehouses):** 기본 서비스(primary services, parent services라고도 함)에 대한 자동 유휴화 지원이 이제 공개 미리 보기로 순차 출시되고 있습니다. 이 기능이 아직 조직에 활성화되지 않았다면, 액세스 권한을 위해 [지원팀에 문의하십시오](https://clickhouse.com/support/program). 이 기능이 일반 제공 단계로 전환되면, 기존 기본 서비스에는 자동 유휴화를 켤 수 있는 옵션이 제공되고 새 기본 서비스에는 기본적으로 자동 유휴화가 활성화됩니다.
* **Data Catalog 통합:** ClickHouse Cloud는 이제 데이터 소스 UI에서 데이터 레이크 카탈로그 통합으로 Iceberg REST Catalog와 Microsoft OneLake를 지원합니다. 연결이 완료되면 카탈로그가 데이터베이스로 표시되므로, 데이터를 복제하지 않고도 Iceberg 테이블을 직접 읽을 수 있습니다.
* **GCP의 Bring Your Own Cloud (BYOC)가 이제 일반 제공됩니다:** ClickHouse BYOC는 이제 Google Cloud에서 일반 제공(GA)되며, 자체 GCP 프로젝트에 ClickHouse 서비스를 배포하고 엄격한 데이터 상주 요건을 충족할 수 있습니다. 자세한 내용은 [문서](/cloud/reference/byoc/overview)와 [블로그 게시물](https://clickhouse.com/blog/byoc-gcp-ga)을 참조하십시오. 액세스를 요청하려면 [문의하십시오](https://clickhouse.com/cloud/bring-your-own-cloud).
* **선불 크레딧 및 결제 수단 변경에 대한 청구 알림:** ClickHouse Cloud는 이제 약정 계약에 따라 조직에 선불 크레딧이 추가되거나 결제 수단(신용카드 또는 마켓플레이스 구독)이 업데이트될 때 알림을 전송합니다. 알림은 기본적으로 UI와 이메일에서 활성화되어 있으며, Slack으로도 전송되도록 구성할 수 있습니다.
