## 2022년 12월 20일 \{#december-20-2022\}

이번 릴리스에서는 관리자의 SQL 콘솔 로그인 환경이 간소화되었고, 콜드 읽기 시 읽기 성능이 향상되었으며, ClickHouse Cloud용 Metabase 커넥터가 개선되었습니다.

### Console 변경 사항 \{#console-changes-27\}

* 관리자 사용자가 SQL 콘솔을 원활하게 이용할 수 있도록 했습니다
* 새로 초대된 사용자의 기본 역할을 &quot;Administrator&quot;로 변경했습니다
* 온보딩 설문조사를 추가했습니다

### 신뢰성 및 성능 \{#reliability-and-performance-3\}

* 장시간 실행되는 삽입 쿼리가 네트워크 장애 발생 시 복구할 수 있도록 再試行 로직을 추가했습니다
* 콜드 읽기의 성능을 개선했습니다

### Integrations 변경 사항 \{#integrations-changes-26\}

* [Metabase plugin](/integrations/data-visualization/metabase-and-clickhouse.md)이 많은 기대를 모은 v0.9.1 대규모 업데이트를 진행했습니다. 이제 최신 Metabase 버전과 호환되며, ClickHouse Cloud 환경에서 철저한 테스트를 거쳤습니다.

## 2022년 12월 6일 - 정식 출시 \{#december-6-2022---general-availability\}

ClickHouse Cloud는 이제 SOC2 Type II 규정 준수, 프로덕션 workloads를 위한 가동 시간 SLA, 공개 상태 페이지를 통해 프로덕션 환경에서 사용할 수 있습니다. 이번 릴리스에는 AWS Marketplace 통합, ClickHouse 사용자를 위한 데이터 탐색 워크벤치인 SQL 콘솔, 그리고 ClickHouse Cloud에서 자기 주도형으로 학습할 수 있는 ClickHouse Academy 등 주요 신규 기능이 포함되어 있습니다. 자세한 내용은 이 [블로그](https://clickhouse.com/blog/clickhouse-cloud-generally-available)에서 확인하십시오.

### 프로덕션 환경에 적합 \{#production-ready\}

* SOC2 Type II 규정 준수(자세한 내용은 [블로그](https://clickhouse.com/blog/clickhouse-cloud-is-now-soc-2-type-ii-compliant) 및 [Trust Center](https://trust.clickhouse.com/) 참조)
* ClickHouse Cloud용 공개 [상태 페이지](https://status.clickhouse.com/)
* 프로덕션 사용 사례에 대한 가동 시간 SLA 제공
* [AWS Marketplace](https://aws.amazon.com/marketplace/pp/prodview-jettukeanwrfc)에서 제공

### 주요 신규 기능 \{#major-new-capabilities\}

* ClickHouse 사용자를 위한 데이터 탐색 워크벤치인 SQL 콘솔을 선보였습니다
* ClickHouse Cloud에서 자기 주도형으로 학습할 수 있는 [ClickHouse Academy](https://learn.clickhouse.com/visitor_class_catalog)를 출시했습니다

### 가격 및 사용량 계측 변경 사항 \{#pricing-and-metering-changes\}

* 체험 기간 30일로 연장
* 고정 용량과 낮은 월 비용의 개발 서비스(Development Services) 도입, 초기 프로젝트와 개발/스테이징 환경에 적합
* ClickHouse Cloud의 운영 및 확장 방식이 계속 개선됨에 따라 프로덕션 서비스(Production Services) 가격 인하
* 컴퓨팅 사용량 계측의 세분성과 정확도 향상

### Integrations 변경 사항 \{#integrations-changes-27\}

* ClickHouse Postgres / MySQL 통합 엔진 지원을 활성화했습니다
* SQL 사용자 정의 함수(UDF) 지원을 추가했습니다
* Kafka Connect sink를 베타로 승격했습니다
* 버전, 업데이트 상태 등 다양한 메타데이터를 추가하여 Integrations UI를 개선했습니다

### Console 변경 사항 \{#console-changes-28\}

* Cloud Console에서 다단계 인증 지원
* 모바일 기기용 Cloud Console 내비게이션 개선

### 문서 변경 사항 \{#documentation-changes\}

* ClickHouse Cloud용 전용 [문서](/cloud/overview) 섹션이 추가되었습니다

### 버그 수정 \{#bug-fixes-5\}

* 종속성 해결로 인해 백업에서 복원이 항상 수행되지 않던 알려진 문제를 해결했습니다

## 2022년 11월 29일 \{#november-29-2022\}

이번 릴리스에서는 SOC2 Type II 규정 준수를 지원하고, ClickHouse 버전을 22.11로 업데이트했으며, 여러 ClickHouse 클라이언트 및 통합 기능을 개선했습니다.

### 일반 변경 사항 \{#general-changes-4\}

* SOC 2 Type II 준수를 달성했습니다(자세한 내용은 [블로그](https://clickhouse.com/blog/clickhouse-cloud-is-now-soc-2-type-ii-compliant) 및 [Trust Center](https://trust.clickhouse.com)를 참조하세요)

### Console 변경 사항 \{#console-changes-29\}

* 서비스가 자동으로 일시 중지되었음을 나타내는 &quot;Idle&quot; 상태 표시기가 추가되었습니다

### ClickHouse 22.11 버전 업그레이드 \{#clickhouse-2211-version-upgrade\}

* Hudi 및 DeltaLake 테이블 엔진과 테이블 함수 지원 추가
* S3의 재귀 디렉터리 순회 개선
* 복합 시간 인터벌 구문 지원 추가
* 삽입 재시도를 통해 삽입 안정성 개선
* 전체 변경 사항은 [자세한 22.11 변경 로그](#clickhouse-2211-version-upgrade)를 참조하십시오

### Integrations \{#integrations-1\}

* Python client: v3.11 지원, 삽입 성능 개선
* Go client: DateTime 및 Int64 지원 문제 수정
* JS client: 상호 SSL 인증 지원
* dbt-clickhouse: DBT v1.3 지원

### 버그 수정 \{#bug-fixes-6\}

* 업그레이드 후 이전 ClickHouse 버전이 표시되던 버그를 수정했습니다
* &quot;default&quot; 계정의 권한 부여를 변경해도 더 이상 세션이 중단되지 않습니다
* 새로 생성된 비관리자 계정에는 더 이상 기본적으로 시스템 테이블(system table) 접근 권한이 부여되지 않습니다

### 이번 릴리스의 알려진 문제 \{#known-issues-in-this-release\}

* 종속성 해결 문제로 인해 백업에서 복원이 작동하지 않을 수 있습니다

## 2022년 11월 17일 \{#november-17-2022\}

이번 릴리스에서는 로컬 ClickHouse 테이블 및 HTTP 소스 기반 딕셔너리를 사용할 수 있게 되었으며, Mumbai 지역 지원이 추가되었고, Cloud Console의 사용자 경험이 개선되었습니다.

### 일반 변경 사항 \{#general-changes-5\}

* 로컬 ClickHouse 테이블 및 HTTP 소스의 [딕셔너리](/sql-reference/statements/create/dictionary) 지원이 추가되었습니다
* Mumbai [지역](/cloud/reference/supported-regions) 지원이 추가되었습니다

### Console 변경 사항 \{#console-changes-30\}

* 청구 인보이스 서식을 개선했습니다
* 결제 수단 입력을 위한 사용자 인터페이스를 간소화했습니다
* 백업 활동에 대한 로깅을 더욱 세분화했습니다
* 파일 업로드 중 오류 처리를 개선했습니다

### 버그 수정 \{#bug-fixes-7\}

* 일부 파트에 대용량 단일 파일이 있는 경우 백업이 실패할 수 있던 버그를 수정했습니다
* 액세스 목록 변경이 동시에 적용되면 백업 복원이 성공하지 않던 버그를 수정했습니다

### 알려진 문제 \{#known-issues\}

* 종속성 해결 문제로 인해 백업에서 복원이 작동하지 않을 수 있습니다

## 2022년 11월 3일 \{#november-3-2022\}

이번 릴리스에서는 요금에서 읽기 및 쓰기 단위를 제거하고(자세한 내용은 [가격 페이지](https://clickhouse.com/pricing)를 참조하십시오), ClickHouse 버전을 22.10으로 업데이트했으며, 셀프서비스 고객이 더 높은 수준의 수직 확장을 사용할 수 있도록 지원을 추가하고, 더 나은 기본값을 통해 안정성을 개선했습니다.

### 일반 변경 사항 \{#general-changes-6\}

* 가격 모델에서 읽기/쓰기 단위를 제거했습니다

### 설정 변경 \{#configuration-changes-2\}

* 안정성상의 이유로 `allow_suspicious_low_cardinality_types`, `allow_suspicious_fixed_string_types`, `allow_suspicious_codecs` 설정(기본값: false)은 이제 사용자 변경이 불가능합니다.

### Console 변경 사항 \{#console-changes-31\}

* 유료 고객이 셀프서비스로 수직 확장할 수 있는 최대 메모리를 720GB로 늘렸습니다
* 백업 복원 워크플로를 개선해 IP Access List 규칙과 비밀번호를 설정할 수 있도록 했습니다
* 서비스 생성 대화 상자에 GCP 및 Azure 대기자 등록 기능을 추가했습니다
* 파일 업로드 중 오류 처리 방식을 개선했습니다
* 청구 관리 워크플로를 개선했습니다

### ClickHouse 22.10 버전 업그레이드 \{#clickhouse-2210-version-upgrade\}

* 대용량 파트(최소 10 GiB)가 많이 있는 경우 「너무 많은 파트」 임계값을 완화하여 오브젝트 스토리지에서의 병합을 개선했습니다. 이를 통해 단일 테이블의 단일 파티션에서 최대 페타바이트 규모의 데이터를 처리할 수 있습니다.
* `min_age_to_force_merge_seconds` 설정으로 병합 제어를 개선하여, 지정한 시간 임계값이 지난 후 병합되도록 했습니다.
* 설정을 기본값으로 재설정하는 MySQL 호환 구문 `SET setting_name = DEFAULT`를 추가했습니다.
* Morton curve 인코딩, Java 정수 해시, 난수 생성을 위한 함수를 추가했습니다.
* 전체 변경 사항 목록은 [자세한 22.10 변경 로그](#clickhouse-2210-version-upgrade)를 참조하십시오.

## 2022년 10월 25일 \{#october-25-2022\}

이번 릴리스에서는 소규모 워크로드의 컴퓨트 사용량을 크게 줄이고, 컴퓨트 가격을 인하했으며(자세한 내용은 [pricing](https://clickhouse.com/pricing) 페이지를 참조하십시오), 기본값을 개선해 안정성을 높이고, ClickHouse Cloud 콘솔의 Billing 및 Usage 화면을 개선했습니다.

### 일반 변경 사항 \{#general-changes-7\}

* 서비스의 최소 메모리 할당량을 24G로 축소
* 서비스 idle 타임아웃을 30분에서 5분으로 단축

### 설정 변경 \{#configuration-changes-3\}

* `max_parts_in_total`을 100k에서 10k로 낮췄습니다. MergeTree 테이블에서 `max_parts_in_total` 설정의 기본값이 100,000에서 10,000으로 낮아졌습니다. 이렇게 변경한 이유는 데이터 파트 수가 많으면 Cloud에서 서비스 시작 시간이 느려질 가능성이 크다는 점이 확인되었기 때문입니다. 파트 수가 많다는 것은 대개 파티션 키를 지나치게 세분화해 선택했다는 뜻이며, 이는 보통 실수로 발생하므로 피해야 합니다. 이번 기본값 변경으로 이러한 사례를 더 일찍 감지할 수 있습니다.

### Console 변경 사항 \{#console-changes-32\}

* 체험판 사용자를 위해 Billing 보기의 크레딧 사용량 세부 정보를 강화했습니다
* Usage 보기의 툴팁과 도움말 텍스트를 개선하고, 가격 페이지 링크를 추가했습니다
* IP 필터링 옵션 전환 워크플로를 개선했습니다
* Cloud Console에 이메일 확인 메일 재전송 버튼을 추가했습니다

## 2022년 10월 4일 - 베타 \{#october-4-2022---beta\}

ClickHouse Cloud는 2022년 10월 4일에 공개 베타를 시작했습니다. 자세한 내용은 이 [블로그](https://clickhouse.com/blog/clickhouse-cloud-public-beta)에서 확인하십시오.

ClickHouse Cloud 버전은 ClickHouse core v22.10을 기반으로 합니다. 호환되는 기능 목록은 [Cloud Compatibility](/whats-new/cloud-compatibility) 가이드를 참조하십시오.
