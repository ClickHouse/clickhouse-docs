데이터 레이크하우스는 데이터 레이크 인프라에 데이터베이스의 원칙을 적용하면서도, 클라우드 스토리지 시스템의 유연성과 확장성을 유지하는 융합형 아키텍처입니다.

레이크하우스는 단순히 데이터베이스를 분해한 구조가 아니라, 전통적인 분석과 최신 AI·ML 워크로드를 하나의 통합된 플랫폼에서 지원하는 데 초점을 맞춘, 근본적으로 다른 기반(클라우드 객체 스토리지) 위에 데이터베이스와 유사한 기능을 구축한 것입니다.


## 데이터 레이크하우스의 구성 요소는 무엇입니까? \{#components-of-the-data-lakehouse\}

최신 데이터 레이크하우스 아키텍처는 데이터 웨어하우스와 데이터 레이크 기술이 융합된 형태로, 두 접근 방식의 장점을 결합합니다.
이 아키텍처는 유연하고 견고한 데이터 저장, 관리 및 분석 플랫폼을 제공하는, 서로 구분되지만 상호 연결된 여러 계층으로 구성됩니다.

이러한 구성 요소를 이해하는 일은 데이터 레이크하우스 전략을 도입하거나 최적화하려는 조직에 매우 중요합니다.
계층화된 접근 방식은 구성 요소의 교체와 각 계층의 독립적인 발전을 가능하게 하여, 아키텍처의 유연성과 장기적인 확장 가능성을 확보합니다.

이제 일반적인 데이터 레이크하우스 아키텍처의 핵심 구성 요소와, 이들이 어떻게 상호 작용하여 통합된 데이터 관리 플랫폼을 형성하는지 살펴보겠습니다.

| Component               | Description                                                                                                                                                                                                                                                                                                                                    |
|-------------------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **Data sources**        | 레이크하우스의 데이터 소스에는 운영 데이터베이스, 스트리밍 플랫폼, IoT 디바이스, 애플리케이션 로그 및 외부 공급자가 포함됩니다.                                                                                                                                                                                                                |
| **Query engine**        | 객체 스토리지에 저장된 데이터에 대해 분석 쿼리를 처리하며, 테이블 포맷 계층에서 제공하는 메타데이터와 최적화를 활용합니다. SQL 및 필요에 따라 기타 쿼리 언어를 지원하여 대용량 데이터를 효율적으로 분석할 수 있도록 합니다.                                                                                                              |
| **Metadata catalog**    | [data catalog](https://clickhouse.com/engineering-resources/data-catalog)는 메타데이터를 위한 중앙 저장소 역할을 하며, 테이블 정의와 스키마, 파티션 정보, 액세스 제어 정책을 저장하고 관리합니다. 레이크하우스 전반에서 데이터 검색, 데이터 계보(lineage) 추적, 거버넌스를 가능하게 합니다.                                            |
| **Table format layer**  | [table format layer](https://clickhouse.com/engineering-resources/open-table-formats)는 데이터 파일을 테이블로 논리적으로 구성하여, ACID 트랜잭션, 스키마 강제 및 변경, 시점 복구 기능, 데이터 스키핑 및 클러스터링과 같은 성능 최적화 등 데이터베이스와 유사한 기능을 제공합니다.                                                   |
| **Object storage**      | 이 계층은 모든 데이터 파일과 메타데이터에 대해 확장 가능하고 내구성이 뛰어나며 비용 효율적인 스토리지를 제공합니다. 데이터를 개방형 포맷으로 물리적으로 저장·보존하여, 여러 도구와 시스템에서 직접 접근할 수 있도록 합니다.                                                                                                                |
| **Client applications** | 레이크하우스에 연결하여 데이터를 쿼리하고 인사이트를 시각화하거나 데이터 제품을 구축하는 다양한 도구와 애플리케이션입니다. 여기에는 BI 도구, 데이터 사이언스 노트북, 커스텀 애플리케이션, ETL/ELT 도구 등이 포함될 수 있습니다.                                                                                                      |

## 데이터 레이크하우스의 이점은 무엇입니까? \{#benefits-of-the-data-lakehouse\}

데이터 레이크하우스 아키텍처는 전통적인 데이터 웨어하우스와 데이터 레이크 모두와 직접 비교할 때 여러 가지 중요한 이점을 제공합니다.

### 기존 데이터 웨어하우스와 비교 \{#compared-to-traditional-data-warehouses\}

| # | Benefit           | Description                                                                                                                                                                                                                                                                                                                                                                                                                                |
|---|-------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 1 | **비용 효율성**   | 레이크하우스는 독점 스토리지 형식 대신 저렴한 객체 스토리지를 활용하여, 통합 스토리지에 높은 비용을 부과하는 데이터 웨어하우스에 비해 스토리지 비용을 크게 절감합니다.                                                                                                                                                                                                                                               |
| 2 | **구성 요소 유연성 및 상호 교체 가능성** | 레이크하우스 아키텍처는 조직이 서로 다른 구성 요소로 대체할 수 있도록 합니다. 기존 시스템은 요구 사항이 변경되거나 기술이 발전하면 전체 시스템을 교체해야 하지만, 레이크하우스는 쿼리 엔진이나 테이블 포맷과 같은 개별 구성 요소를 교체하는 방식으로 점진적인 발전을 가능하게 합니다. 이러한 유연성은 벤더 종속성을 줄이고, 조직이 대규모 마이그레이션 없이 변화하는 요구 사항에 적응할 수 있도록 합니다. |
| 3 | **오픈 포맷 지원** | 레이크하우스는 Parquet와 같은 오픈 파일 포맷으로 데이터를 저장하여, 다양한 도구에서 벤더 종속성 없이 직접 접근할 수 있도록 합니다. 반면, 기존 데이터 웨어하우스의 독점 포맷은 해당 생태계 안에서만 접근을 허용하는 방식으로 사용을 제한합니다.                                                                                                                                                                         |
| 4 | **AI/ML 통합**    | 레이크하우스는 머신 러닝 프레임워크와 Python/R 라이브러리가 데이터에 직접 접근할 수 있도록 지원하지만, 데이터 웨어하우스는 고급 분석에 활용하기 전에 일반적으로 데이터를 별도로 추출해야 합니다.                                                                                                                                                                                                                       |
| 5 | **독립적인 확장성** | 레이크하우스는 스토리지와 컴퓨트를 분리하여 실제 요구 사항에 따라 각각을 독립적으로 확장할 수 있습니다. 이는 두 리소스를 함께 확장해야 하는 경우가 많은 데이터 웨어하우스와는 대조적입니다.                                                                                                                                                                                                                       |

### 데이터 레이크와 비교했을 때 \{#compared-to-data-lakes\}

| # | Benefit                     | Description                                                                                                                                                                                         |
|---|-----------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 1 | **쿼리 성능**               | 레이크하우스는 인덱싱, 통계, 데이터 레이아웃 최적화 등을 구현하여 SQL 쿼리가 데이터 웨어하우스에 필적하는 속도로 실행되도록 하며, 이로써 원시 데이터 레이크의 낮은 성능 문제를 극복합니다.            |
| 2 | **데이터 일관성**           | 레이크하우스는 ACID 트랜잭션을 지원하여 동시 작업 중에도 일관성을 보장함으로써, 파일 충돌로 인해 데이터가 손상될 수 있는 기존 데이터 레이크의 주요 한계를 해결합니다.                             |
| 3 | **스키마 관리**             | 레이크하우스는 스키마 검증을 강제하고 스키마 변경을 추적하여, 스키마 불일치로 인해 데이터 사용이 불가능해지는 데이터 레이크의 「데이터 늪(data swamp)」 문제를 방지합니다.                          |
| 4 | **거버넌스 기능**           | 레이크하우스는 행/컬럼 수준의 세분화된 액세스 제어와 감사 기능을 제공하여, 기본적인 데이터 레이크에서 제공되는 제한적인 보안 제어 기능의 한계를 보완합니다.                                     |
| 5 | **BI 도구 지원**            | 레이크하우스는 표준 BI 도구와 호환되도록 하는 SQL 인터페이스와 각종 최적화를 제공하며, 시각화 전에 추가 처리 계층이 필요한 원시 데이터 레이크와는 다릅니다.                                       |

## ClickHouse는 데이터 레이크하우스 아키텍처에서 어떤 역할을 하나요? \{#where-does-clickhouse-fit-in-the-data-lakehouse-architecture\}

ClickHouse는 현대적인 데이터 레이크하우스 생태계에서 강력한 분석용 쿼리 엔진입니다. 대규모 데이터 분석을 위해 조직에서 활용할 수 있는 고성능 옵션을 제공합니다. ClickHouse는 뛰어난 쿼리 속도와 효율성 덕분에 매우 매력적인 선택지입니다.

레이크하우스 아키텍처 내에서 ClickHouse는 기저 데이터와 유연하게 상호작용할 수 있는 특수화된 처리 계층으로 동작합니다. S3, Azure Blob Storage, Google Cloud Storage와 같은 클라우드 객체 스토리지 시스템에 저장된 Parquet 파일을 직접 쿼리할 수 있으며, 최적화된 열 지향 처리 기능을 활용하여 방대한 데이터셋에서도 매우 빠른 결과를 제공합니다. 이러한 직접 쿼리 기능을 통해 조직은 복잡한 데이터 이동이나 변환 과정 없이 데이터 레이크의 데이터를 분석할 수 있습니다.

ClickHouse는 Apache Iceberg, Delta Lake, Apache Hudi와 같은 오픈 테이블 포맷과 통합되어 더욱 고도화된 데이터 관리 요구 사항을 충족합니다. 이러한 통합을 통해 ClickHouse는 이들 포맷의 고급 기능을 활용하면서도, 잘 알려진 뛰어난 쿼리 성능을 그대로 제공합니다. 조직은 이러한 테이블 포맷을 직접 통합하거나 AWS Glue, Unity 등의 메타데이터 카탈로그 또는 기타 카탈로그 서비스와 연결하여 사용할 수 있습니다.

레이크하우스 아키텍처에서 쿼리 엔진으로 ClickHouse를 도입하면, 데이터 레이크에 대해 매우 빠른 분석 쿼리를 실행하면서도 레이크하우스 접근 방식의 핵심인 유연성과 개방성을 유지할 수 있습니다. 이 조합은 구성 요소 교체 가능성, 오픈 포맷, 통합된 데이터 관리 등 레이크하우스 모델의 핵심 이점을 유지하면서, 특수화된 분석 데이터베이스에 버금가는 성능 특성을 제공합니다.

## 하이브리드 아키텍처: 두 세계의 장점을 모두 활용하기 \{#hybrid-architecture-the-best-of-both-worlds\}

ClickHouse는 레이크하우스 구성 요소에 대한 쿼리에 뛰어날 뿐만 아니라, 고도로 최적화된 
스토리지 엔진을 통해 추가적인 이점도 제공합니다. 초저지연 쿼리가 요구되는 실시간 대시보드, 
운영 분석, 대화형 사용자 경험과 같은 사용 사례에서는 조직이 성능에 중요한 데이터를 
선별하여 ClickHouse의 네이티브 포맷으로 직접 저장할 수 있습니다. 이러한 하이브리드 방식은 
시간 민감형 분석을 위한 ClickHouse 특화 스토리지의 뛰어난 쿼리 속도와, 필요할 때 레이크하우스 
전반에 쿼리할 수 있는 유연성을 결합하여 두 세계의 장점을 모두 제공합니다.

이러한 이중 기능을 통해 조직은 계층형 데이터 전략을 구현할 수 있습니다. 즉, 빈번히 조회되는 
핫 데이터는 ClickHouse의 최적화된 스토리지에 저장하여 1초 미만의 쿼리 응답을 제공하면서, 
동시에 레이크하우스에 있는 전체 데이터 이력에는 끊김 없이 접근할 수 있습니다. 팀은 기술적 
제약이 아니라 성능 요구 사항을 기준으로 아키텍처를 결정할 수 있으며, 중요한 워크로드에는 
초고속 분석 데이터베이스로서 ClickHouse를 사용하고, 더 넓은 데이터 생태계를 위한 유연한 
쿼리 엔진으로도 활용할 수 있습니다.
