ClickHouse Agents는 계층형 권한 모델을 사용합니다. 역할은 사용자가 어떤 기능을 사용할 수 있는지 전반적으로 제어하고, 리소스 수준 액세스 목록은 특정 에이전트 또는 기타 리소스를 누가 사용할 수 있는지 제어하며, 관리자 권한은 전체 관리를 누가 수행할 수 있는지 제어합니다.

## 역할 \{#roles\}

역할은 하나 이상의 사용자에게 할당되는 기능별 권한 묶음입니다. 기본 역할은 다음과 같습니다.

* **User** — 에이전트를 실행하고 채팅할 수 있습니다. 에이전트를 빌드할 수 있는지, 스킬을 만들 수 있는지, 리소스를 공유할 수 있는지 등은 해당 역할의 기능별 토글 설정에 따라 달라집니다.
* **Admin** — User의 모든 권한에 더해, 사용자 관리, 마켓플레이스 구성, 권한 부여 등 조직 수준의 제어 기능을 사용할 수 있습니다.

Admin은 더 세분화된 제어를 위해 자체 토글이 포함된 사용자 지정 역할을 만들 수 있습니다.

## 리소스 공유 \{#resource-sharing\}

에이전트, 프롬프트, 스킬에는 각각 액세스 목록이 있습니다. 각 주체에는 3가지 액세스 수준이 있습니다:

* **Viewer** — 리소스를 사용할 수 있지만 수정할 수는 없습니다.
* **Editor** — 리소스의 구성(지침, 연결된 도구, 모델 등)을 변경할 수 있습니다.
* **Owner** — Editor 권한에 더해 리소스를 삭제하거나 공유 설정을 변경할 수 있습니다.

리소스를 생성하면 기본적으로 해당 리소스의 소유자가 됩니다. 명시적으로 공유하지 않은 항목은 본인만 사용할 수 있습니다.

## 주체 \{#principals\}

다음 대상에게 액세스 권한을 부여할 수 있습니다.

* **개별 사용자** — 한 번에 한 사용자씩 부여합니다.
* **그룹** — 팀 또는 조직 단위입니다. 사용자가 이후 그룹에 참여하거나 탈퇴하면 액세스 권한이 자동으로 업데이트됩니다.
* **조직 전체** — 조직의 모든 구성원이 리소스를 볼 수 있습니다. 이 설정은 이를 허용하는 역할 수준의 토글이 활성화된 경우에만 적용됩니다.

조직 전체에 공개하면 [마켓플레이스](/cloud/features/ai-ml/agents/marketplace)에서 해당 리소스를 찾을 수 있습니다.

## 에이전트 공유하기 \{#share-an-agent\}

Agent Builder에서 **Sharing** 섹션을 열고 **Add access**를 클릭하십시오. 주체를 선택하고(사용자, 그룹 또는 조직 전체), 액세스 수준을 지정한 후 저장하십시오. 공유받은 사용자는 다음에 목록을 갱신할 때 사용 가능한 에이전트 목록에서 해당 에이전트를 확인할 수 있습니다.

같은 패널에서 해당 행을 삭제하여 액세스를 철회할 수 있습니다. 그러면 공유받은 사용자는 즉시 액세스할 수 없게 됩니다. 진행 중인 대화는 계속 완료되지만, 새 대화는 시작할 수 없습니다.

## 기본 에이전트 \{#default-agent\}

에이전트 하나를 기본값으로 지정할 수 있습니다. 새 대화는 별도로 다른 항목을 선택하지 않으면 해당 에이전트로 열립니다. 관리자는 사용자가 직접 선택하지 않은 경우에 대비해 조직 전체의 기본값을 설정할 수도 있습니다.

## 관리자 작업 \{#admin-operations\}

관리자 전용 작업에는 다음이 포함됩니다.

* 역할을 생성하고 편집합니다.
* 마켓플레이스 카테고리를 구성하고 추천 에이전트를 선정합니다.
* 조직 수준에서 기능(메모리, 웹 검색, 코드 인터프리터, 마켓플레이스 등)을 활성화하거나 비활성화합니다.
* 다른 사용자에게 관리자 권한을 부여합니다.

ClickHouse Cloud에서는 조직 관리자에게 관리자 패널이 제공됩니다. 관리자 패널이 보이지 않으면 조직 관리자 또는 ClickHouse Cloud Support에 문의하여 액세스를 요청하십시오.

## 주의사항 \{#gotchas\}

* **에이전트를 삭제하면 모든 공유가 중단됩니다.** 수신자는 액세스 권한을 잃습니다. 에이전트와의 저장된 대화는 계속 읽을 수 있지만, 이어서 진행할 수는 없습니다.
* **공유된 에이전트를 수정하면 이를 사용하는 모든 사용자에게 영향을 줍니다.** 다른 사용자에게 영향을 주지 않고 실험해야 한다면 복사본을 사용하십시오.
* **사용자의 역할을 취소해도 해당 사용자가 만든 리소스는 삭제되지 않습니다.** 해당 사용자의 에이전트, 프롬프트, 스킬은 액세스 목록에 그대로 남아 있습니다. 완전히 오프보딩하기 전에 소유권을 이전하십시오.
