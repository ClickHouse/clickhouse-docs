## argAndMin \{#argAndMin\}

도입 버전: v1.1

최솟값인 `val`에 대해 `arg`와 `val` 값을 계산합니다.
최솟값인 `val`이 동일한 여러 행이 있는 경우, 어떤 `arg` 및 `val`이 반환될지는 결정적이지 않습니다.
`arg`와 `min` 두 요소 모두 [집계 함수](/sql-reference/aggregate-functions/index.md)처럼 동작하며, 처리 중에 둘 다 [`Null`을 건너뛰고](/sql-reference/aggregate-functions/index.md#null-processing), `Null`이 아닌 값이 존재하는 경우 `Null`이 아닌 값을 반환합니다.

:::note
`argMin`과의 유일한 차이점은 `argAndMin`이 인자와 값을 모두 반환한다는 점입니다.
:::

**관련 항목**

* [argMin](/sql-reference/aggregate-functions/reference/argMin.md)
* [Tuple](/sql-reference/data-types/tuple.md)

**구문**

```sql
argAndMin(arg, val)
```

**인수(Arguments)**

* `arg` — 최소값을 찾을 인수입니다. [`const String`](/sql-reference/data-types/string)
* `val` — 최소값입니다. [`(U)Int8/16/32/64`](/sql-reference/data-types/int-uint) 또는 [`Float*`](/sql-reference/data-types/float) 또는 [`Date`](/sql-reference/data-types/date) 또는 [`DateTime`](/sql-reference/data-types/datetime) 또는 [`Tuple`](/sql-reference/data-types/tuple)

**반환 값**

최소인 `val` 값에 대응하는 `arg` 값과 그 최소 `val` 값을 포함한 튜플을 반환합니다. [`Tuple`](/sql-reference/data-types/tuple)

**예시(Examples)**

**기본 사용법(Basic usage)**

```sql title=Query
SELECT argAndMin(user, salary) FROM salary;
```

```response title=Response
┌─argAndMin(user, salary)─┐
│ ('worker',1000)         │
└─────────────────────────┘
```

**NULL 처리를 포함한 확장 예제**

```sql title=Query
CREATE TABLE test
(
    a Nullable(String),
    b Nullable(Int64)
)
ENGINE = Memory AS
SELECT *
FROM VALUES((NULL, 0), ('a', 1), ('b', 2), ('c', 2), (NULL, NULL), ('d', NULL));

SELECT argMin(a,b), argAndMin(a, b), min(b) FROM test;
```

```response title=Response
┌─argMin(a, b)─┬─argAndMin(a, b)─┬─min(b)─┐
│ a            │ ('a',1)         │      0 │
└──────────────┴─────────────────┴────────┘
```

**튜플을 인수로 사용하기**

```sql title=Query
SELECT argAndMin(a, (b, a)), min(tuple(b, a)) FROM test;
```

```response title=Response
┌─argAndMin(a, (b, a))─┬─min((b, a))─┐
│ ('a',(1,'a'))        │ (0,NULL)    │
└──────────────────────┴─────────────┘
```
