## ULIDStringToDateTime \{#ULIDStringToDateTime\}

도입: v23.3

FUNCTION은 [ULID](https://github.com/ulid/spec)에서 타임스탬프를 추출합니다.

**구문**

```sql
ULIDStringToDateTime(ulid[, timezone])
```

**인수**

* `ulid` — 입력 ULID. [`String`](/sql-reference/data-types/string) 또는 [`FixedString(26)`](/sql-reference/data-types/fixedstring)
* `timezone` — 선택적입니다. 반환되는 값에 사용할 타임존 이름. [`String`](/sql-reference/data-types/string)

**반환 값**

밀리초 단위 정밀도의 타임스탬프입니다. [`DateTime64(3)`](/sql-reference/data-types/datetime64)

**예시**

**사용 예시**

```sql title=Query
SELECT ULIDStringToDateTime('01GNB2S2FGN2P93QPXDNB4EN2R')
```

```response title=Response
┌─ULIDStringToDateTime('01GNB2S2FGN2P93QPXDNB4EN2R')─┐
│                            2022-12-28 00:40:37.616 │
└────────────────────────────────────────────────────┘
```


## generateULID \{#generateULID\}

도입 버전: v23.2

[Universally Unique Lexicographically Sortable Identifier (ULID)](https://github.com/ulid/spec)를 생성합니다.

**구문**

```sql
generateULID([x])
```

**인수**

* `x` — 선택적 인수입니다. 지원되는 데이터 유형 중 어떤 것이든 결과로 생성하는 표현식입니다. 결과 값은 사용되지 않지만, 하나의 쿼리에서 이 함수를 여러 번 호출하는 경우 [공통 부분식 제거](/sql-reference/functions/overview#common-subexpression-elimination)를 회피하기 위해 표현식 자체는 사용됩니다. [`Any`](/sql-reference/data-types)

**반환 값**

ULID를 반환합니다. [`FixedString(26)`](/sql-reference/data-types/fixedstring)

**예시**

**사용 예시**

```sql title=Query
SELECT generateULID()
```

```response title=Response
┌─generateULID()─────────────┐
│ 01GNB2S2FGN2P93QPXDNB4EN2R │
└────────────────────────────┘
```

**하나의 행에서 여러 값을 생성해야 하는 경우의 사용 예**

```sql title=Query
SELECT generateULID(1), generateULID(2)
```

```response title=Response
┌─generateULID(1)────────────┬─generateULID(2)────────────┐
│ 01GNB2SGG4RHKVNT9ZGA4FFMNP │ 01GNB2SGG4V0HMQVH4VBVPSSRB │
└────────────────────────────┴────────────────────────────┘
```
