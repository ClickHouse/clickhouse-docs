## IPv4CIDRToRange \{#IPv4CIDRToRange\}

도입 버전: v20.1.0

Classless Inter-Domain Routing (CIDR) 접두사 길이가 지정된 IPv4 주소를 입력으로 받아, 해당 서브넷의 주소 범위를 그 서브넷의 첫 번째 주소와 마지막 주소에 해당하는 두 개의 IPv4 값으로 이루어진 튜플로 반환합니다.
IPv6 버전은 [`IPv6CIDRToRange`](#IPv4CIDRToRange)를 참고하십시오.

**구문**

```sql
IPv4CIDRToRange(ipv4, cidr)
```

**인수**

* `ipv4` — IPv4 주소. [`IPv4`](/sql-reference/data-types/ipv4) 또는 [`String`](/sql-reference/data-types/string)
* `cidr` — CIDR 값. [`UInt8`](/sql-reference/data-types/int-uint)

**반환 값**

서브넷 범위를 나타내는 두 개의 IPv4 주소로 구성된 튜플을 반환합니다. [`Tuple(IPv4, IPv4)`](/sql-reference/data-types/tuple)

**예제**

**사용 예제**

```sql title=Query
SELECT IPv4CIDRToRange(toIPv4('192.168.5.2'), 16);
```

```response title=Response
┌─IPv4CIDRToRange(toIPv4('192.168.5.2'), 16)─┐
│ ('192.168.0.0','192.168.255.255')          │
└────────────────────────────────────────────┘
```

## IPv4NumToString \{#IPv4NumToString\}

도입 버전: v1.1.0

32비트 정수를 IPv4 주소의 점으로 구분된 10진수 표기(A.B.C.D 형식) 문자열로 변환합니다.
입력값은 빅엔디언(Big-endian) 바이트 순서로 해석합니다.

**구문**

```sql
IPv4NumToString(num)
```

**별칭(Aliases)**: `INET_NTOA`

**인수(Arguments)**

* `num` — `UInt32` 숫자로 표현된 IPv4 주소. [`UInt32`](/sql-reference/data-types/int-uint)

**반환 값(Returned value)**

MAC 주소를 나타내는 숫자를 반환하며, 형식이 잘못되면 `0`을 반환합니다. [`String`](/sql-reference/data-types/string)

**예시(Examples)**

**사용 예시(Usage example)**

```sql title=Query
IPv4NumToString(3232235521)
```

```response title=Response
192.168.0.1
```

## IPv4ToIPv6 \{#IPv4ToIPv6\}

빅 엔디언(big endian) 32비트 정수를 IPv4 주소로 해석한 다음, 이 IPv4 주소에 대응하는 IPv6 주소를 `FixedString(16)` 형식으로 표현합니다.

**구문**


**인수**

* `x` — IPv4 주소. [`UInt32`](/sql-reference/data-types/int-uint)

**반환값**

IPv6 주소의 이진 표현을 반환합니다. [`FixedString(16)`](/sql-reference/data-types/fixedstring)

**예시**

**사용 예시**

```sql title=Query
SELECT IPv6NumToString(IPv4ToIPv6(IPv4StringToNum('192.168.0.1'))) AS addr;
```

```response title=Response
┌─addr───────────────┐
│ ::ffff:192.168.0.1 │
└────────────────────┘
```

* `ipv6` — IPv6 주소. [`IPv6`](/sql-reference/data-types/ipv6) 또는 [`String`](/sql-reference/data-types/string)
* `cidr` — CIDR 값. [`UInt8`](/sql-reference/data-types/int-uint)

**반환 값**

서브넷 범위를 나타내는 두 개의 IPv6 주소로 구성된 튜플을 반환합니다. [`Tuple(IPv6, IPv6)`](/sql-reference/data-types/tuple)

**예시**

**사용 예**

```sql title=Query

```

```response title=Response
┌─IPv6CIDRToRange(toIPv6('2001:0db8:0000:85a3:0000:0000:ac1f:8001'), 32)─┐
│ ('2001:db8::','2001:db8:ffff:ffff:ffff:ffff:ffff:ffff')                │
└────────────────────────────────────────────────────────────────────────┘
```

## IPv6StringToNum \{#IPv6StringToNum\}

도입 버전: v1.1.0

IPv6 주소를 표준 텍스트 표현에서 이진 형식(`FixedString(16)`)으로 변환합니다.
`::ffff:111.222.33.44.` 형식의 IPv4가 매핑된 IPv6 주소를 허용합니다.
IPv6 주소 형식이 올바르지 않으면 예외가 발생합니다.

입력 문자열에 유효한 IPv4 주소가 포함된 경우 해당 IPv4에 대응하는 IPv6 주소를 반환합니다.
16진수 표기(HEX)는 대문자와 소문자를 모두 허용합니다.

**문법**

```sql
IPv6StringToNum(string)
```

**별칭**: `INET6_ATON`

**인수**

* `string` — IPv6 주소 문자열입니다. [`String`](/sql-reference/data-types/string)

**반환 값**

IPv6 주소를 이진 형식으로 반환합니다. [`FixedString(16)`](/sql-reference/data-types/fixedstring)

**예시**

**기본 예제**

```sql title=Query
SELECT addr, cutIPv6(IPv6StringToNum(addr), 0, 0) FROM (SELECT ['notaddress', '127.0.0.1', '1111::ffff'] AS addr) ARRAY JOIN addr;
```

```response title=Response
┌─addr───────┬─cutIPv6(IPv6StringToNum(addr), 0, 0)─┐
│ notaddress │ ::                                   │
│ 127.0.0.1  │ ::ffff:127.0.0.1                     │
│ 1111::ffff │ 1111::ffff                           │
└────────────┴──────────────────────────────────────┘
```

## cutIPv6 \{#cutIPv6\}

도입된 버전: v1.1.0

이진 형식의 IPv6 주소를 담고 있는 `FixedString(16)` 값을 입력으로 받습니다.
지정된 바이트 수만큼 주소를 잘라낸 결과를 텍스트 형식의 문자열로 반환합니다.

**구문**

```sql
cutIPv6(x, bytesToCutForIPv6, bytesToCutForIPv4)
```

**인자**

* `x` — 이진 형식의 IPv6 주소입니다. [`FixedString(16)`](/sql-reference/data-types/fixedstring) 또는 [`IPv6`](/sql-reference/data-types/ipv6)
* `bytesToCutForIPv6` — IPv6에서 잘라낼 바이트 수입니다. [`UInt8`](/sql-reference/data-types/int-uint)
* `bytesToCutForIPv4` — IPv4에서 잘라낼 바이트 수입니다. [`UInt8`](/sql-reference/data-types/int-uint)

**반환 값**

지정된 바이트가 제거된 IPv6 주소의 텍스트 표현을 담은 문자열을 반환합니다. [`String`](/sql-reference/data-types/string)

**예제**

**사용 예제**

```sql title=Query
WITH
    IPv6StringToNum('2001:0DB8:AC10:FE01:FEED:BABE:CAFE:F00D') AS ipv6,
    IPv4ToIPv6(IPv4StringToNum('192.168.0.1')) AS ipv4
SELECT
    cutIPv6(ipv6, 2, 0),
    cutIPv6(ipv4, 0, 2)
```

```response title=Response
┌─cutIPv6(ipv6, 2, 0)─────────────────┬─cutIPv6(ipv4, 0, 2)─┐
│ 2001:db8:ac10:fe01:feed:babe:cafe:0 │ ::ffff:192.168.0.0  │
└─────────────────────────────────────┴─────────────────────┘
```

## isIPAddressInRange \{#isIPAddressInRange\}

도입 버전: v21.4.0

IP 주소가 [Classless Inter-Domain Routing (CIDR)](https://en.wikipedia.org/wiki/Classless_Inter-Domain_Routing) 표기법으로 표현된 네트워크에 포함되어 있는지 판별합니다.

이 FUNCTION은 문자열로 표현된 IPv4 및 IPv6 주소(및 네트워크)를 모두 지원합니다. 주소와 CIDR의 IP 버전이 일치하지 않으면 `0`을 반환합니다.

**구문**

```sql
isIPAddressInRange(address, prefix)
```

**인수**

* `address` — IPv4 또는 IPv6 주소입니다. [`String`](/sql-reference/data-types/string)
* `prefix` — CIDR 표기법의 IPv4 또는 IPv6 네트워크 프리픽스입니다. [`String`](/sql-reference/data-types/string)

**반환 값**

주소의 IP 버전과 CIDR이 일치하면 `1`, 그렇지 않으면 `0`을 반환합니다. [`UInt8`](/sql-reference/data-types/int-uint)

**예시**

**범위 내 IPv4 주소**

```sql title=Query
SELECT isIPAddressInRange('127.0.0.1', '127.0.0.0/8')
```

```response title=Response
1
```

**IPv4 주소가 유효한 범위에 있지 않습니다**

```sql title=Query
SELECT isIPAddressInRange('127.0.0.1', 'ffff::/16')
```

```response title=Response
0
```

**IPv6 주소가 범위를 벗어났습니다**

```sql title=Query
SELECT isIPAddressInRange('::ffff:192.168.0.1', '::ffff:192.168.0.4/128')
```

```response title=Response
0
```

## toIPv4 \{#toIPv4\}

도입 버전: v20.1.0

문자열 또는 IPv4 주소의 `UInt32` 형식을 IPv4 타입으로 변환합니다.
[`IPv4StringToNum`](/sql-reference/functions/ip-address-functions#IPv4StringToNum) 및 [`IPv4NumToString`](/sql-reference/functions/ip-address-functions#IPv4NumToString) 함수와 유사하지만, 입력 인수로 문자열과 부호 없는 정수 데이터 타입을 모두 지원합니다.

**Syntax**

```sql
toIPv4(x)
```

**인자**

* `x` — IPv4 주소인 [`String`](/sql-reference/data-types/string) 또는 [`UInt8/16/32`](/sql-reference/data-types/int-uint)

**반환값**

[`IPv4`](/sql-reference/data-types/ipv4) 형식의 IPv4 주소를 반환합니다.

**예시**

**사용 예시**

```sql title=Query
SELECT toIPv4('171.225.130.45');
```

```response title=Response
┌─toIPv4('171.225.130.45')─┐
│ 171.225.130.45           │
└──────────────────────────┘
```

**IPv4StringToNum 및 IPv4NumToString 함수와의 비교.**

```sql title=Query
WITH
    '171.225.130.45' AS IPv4_string
SELECT
    hex(IPv4StringToNum(IPv4_string)),
    hex(toIPv4(IPv4_string))
```

```response title=Response
┌─hex(IPv4StringToNum(IPv4_string))─┬─hex(toIPv4(IPv4_string))─┐
│ ABE1822D                          │ ABE1822D                 │
└───────────────────────────────────┴──────────────────────────┘
```

**정수형에서의 변환**

```sql title=Query
SELECT toIPv4(2130706433);
```

```response title=Response
┌─toIPv4(2130706433)─┐
│ 127.0.0.1          │
└────────────────────┘
```

## toIPv4OrDefault \{#toIPv4OrDefault\}

도입된 버전: v22.3.0

문자열 또는 IPv4 주소의 `UInt32` 형식을 [`IPv4`](../data-types/ipv4.md) 타입으로 변환합니다.
IPv4 주소의 형식이 잘못된 경우 `0.0.0.0`(값이 0인 IPv4) 또는 지정한 IPv4 기본값을 반환합니다.

**구문**

```sql
toIPv4OrDefault(string[, default])
```

**인수**

* `string` — 변환할 IP 주소를 나타내는 문자열입니다. [`String`](/sql-reference/data-types/string)
* `default` — 선택 사항입니다. 문자열이 유효한 IPv4 주소가 아닌 경우 반환할 값입니다. [`IPv4`](/sql-reference/data-types/ipv4)

**반환 값**

문자열을 IPv4 주소로 변환한 값을 반환하며, 변환에 실패하면 기본값을 반환합니다. [`IPv4`](/sql-reference/data-types/ipv4)

**예시**

**유효한 IPv4 문자열과 잘못된 IPv4 문자열**

```sql title=Query
WITH
    '192.168.1.1' AS valid_IPv4_string,
    '999.999.999.999' AS invalid_IPv4_string,
    'not_an_ip' AS malformed_string
SELECT
    toIPv4OrDefault(valid_IPv4_string) AS valid,
    toIPv4OrDefault(invalid_IPv4_string) AS default_value,
    toIPv4OrDefault(malformed_string, toIPv4('8.8.8.8')) AS provided_default;
```

```response title=Response
┌─valid─────────┬─default_value─┬─provided_default─┐
│ 192.168.1.1   │ 0.0.0.0       │ 8.8.8.8          │
└───────────────┴───────────────┴──────────────────┘
```
