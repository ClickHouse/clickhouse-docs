대기 중인 비동기 insert 작업 큐에 대한 정보를 포함합니다.

컬럼:

{/*AUTOGENERATED_START*/ }

* `query` ([String](../../sql-reference/data-types/)) — 쿼리 텍스트입니다.
* `database` ([String](../../sql-reference/data-types/)) — 데이터베이스 이름입니다.
* `table` ([String](../../sql-reference/data-types/)) — 테이블 이름입니다.
* `format` ([String](../../sql-reference/data-types/)) — 포맷 이름입니다.
* `first_update` ([DateTime64(6)](../../sql-reference/data-types/)) — 마이크로초 단위의 첫 삽입 시각입니다.
* `total_bytes` ([UInt64](../../sql-reference/data-types/)) — 큐에서 대기 중인 바이트의 총량입니다.
* `entries.query_id` ([Array(String)](../../sql-reference/data-types/)) — 큐에서 대기 중인 삽입 쿼리의 쿼리 ID 배열입니다.
* `entries.bytes` ([Array(UInt64)](../../sql-reference/data-types/)) — 큐에서 대기 중인 각 삽입 쿼리의 바이트 수 배열입니다.

{/*AUTOGENERATED_END*/ }

**예제**

쿼리:

```sql
SELECT * FROM system.asynchronous_inserts LIMIT 1 \G;
```

결과:

```text
Row 1:
──────
query:            INSERT INTO public.data_guess (user_id, datasource_id, timestamp, path, type, num, str) FORMAT CSV
database:         public
table:            data_guess
format:           CSV
first_update:     2023-06-08 10:08:54.199606
total_bytes:      133223
entries.query_id: ['b46cd4c4-0269-4d0b-99f5-d27668c6102e']
entries.bytes:    [133223]
```

**함께 보기**

* [system.query&#95;log](/operations/system-tables/query_log) — 쿼리 실행에 대한 공통 정보를 포함하는 `query_log` 시스템 테이블에 대한 설명입니다.
* [system.asynchronous&#95;insert&#95;log](/operations/system-tables/asynchronous_insert_log) — 수행된 비동기 insert 작업에 대한 정보를 포함하는 테이블입니다.
