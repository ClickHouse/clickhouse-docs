## 설명 \{#description\}

서버에서 지원하는 데이터 스키핑 인덱스 유형 목록과 각 유형에 대한 내장 문서가 포함되어 있습니다. 데이터 스키핑 인덱스 유형은 `CREATE TABLE` 쿼리의 `INDEX` 선언에서 `TYPE`에 지정하며, ClickHouse가 쿼리 조건을 만족할 수 없는 그래뉼을 건너뛸 수 있도록 합니다.

이 테이블은 사용 가능한 인덱스 *타입*을 나열하는 반면, [`system.data_skipping_indices`](/operations/system-tables/data_skipping_indices)는 기존 테이블에 정의된 인덱스 인스턴스를 나열합니다.

## 컬럼 \{#columns\}

{/*AUTOGENERATED_START*/ }

* `name` ([String](../../sql-reference/data-types/)) — INDEX 선언의 TYPE에 지정된 데이터 스키핑 인덱스 유형 이름입니다.
* `description` ([String](../../sql-reference/data-types/)) — 데이터 스키핑 인덱스 유형의 기능을 개괄적으로 설명합니다.
* `syntax` ([String](../../sql-reference/data-types/)) — CREATE TABLE 쿼리의 INDEX 절에서 인덱스를 선언하는 방법입니다.
* `examples` ([String](../../sql-reference/data-types/)) — 사용 예시입니다.
* `introduced_in` ([String](../../sql-reference/data-types/)) — 인덱스 유형이 처음 도입된 ClickHouse 버전이며, 형식은 major.minor입니다.
* `related` ([Array(String)](../../sql-reference/data-types/)) — 관련된 데이터 스키핑 인덱스 유형의 이름입니다.

{/*AUTOGENERATED_END*/ }

## 예시 \{#example\}

```sql title="Query"
SELECT name, syntax
FROM system.data_skipping_index_types
WHERE name IN ('minmax', 'set')
ORDER BY name
```

```text title="Response"
┌─name───┬─syntax───────────────────────────────────────┐
│ minmax │ INDEX name expr TYPE minmax GRANULARITY n     │
│ set    │ INDEX name expr TYPE set(max_rows) GRANULARITY n │
└────────┴──────────────────────────────────────────────┘
```
