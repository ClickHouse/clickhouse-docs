# system.background_schedule_pool \{#systembackground_schedule_pool\}

<SystemTableCloud />

## 설명 \{#description\}

백그라운드 스케줄 풀에 있는 작업에 대한 정보를 포함합니다. 백그라운드 스케줄 풀은 분산 전송, 버퍼 플러시, 메시지 브로커 작업과 같은 주기적인 작업을 실행하는 데 사용됩니다.

## 컬럼 \{#columns\}

{/*AUTOGENERATED_START*/ }

* `pool` ([String](../../sql-reference/data-types/)) — 풀 이름. 가능한 값: `schedule` — 범용 스케줄 풀, `buffer_flush` — Buffer 테이블 데이터 플러시용 풀, `distributed` — 분산 테이블 작업용 풀, `message_broker` — 메시지 브로커 작업용 풀.
* `database` ([String](../../sql-reference/data-types/)) — 데이터베이스 이름.
* `table` ([String](../../sql-reference/data-types/)) — 테이블 이름.
* `table_uuid` ([UUID](../../sql-reference/data-types/)) — 테이블 UUID.
* `query_id` ([String](../../sql-reference/data-types/)) — Query ID(현재 실행 중인 경우). 주의: 이는 실제 쿼리가 아니라 `system.text_log`에서 로그를 매칭하기 위해 무작위로 생성된 ID입니다.
* `elapsed_ms` ([UInt64](../../sql-reference/data-types/)) — 작업 실행 시간(현재 실행 중인 경우).
* `log_name` ([String](../../sql-reference/data-types/)) — 작업의 로그 이름.
* `deactivated` ([UInt8](../../sql-reference/data-types/)) — 작업이 비활성화되었는지 여부(비활성화된 작업은 풀에서 제거되므로 항상 false).
* `scheduled` ([UInt8](../../sql-reference/data-types/)) — 작업이 실행되도록 예약되었는지 여부.
* `delayed` ([UInt8](../../sql-reference/data-types/)) — 작업이 지연되어 예약되었는지 여부.
* `executing` ([UInt8](../../sql-reference/data-types/)) — 작업이 현재 실행 중인지 여부.

{/*AUTOGENERATED_END*/ }
