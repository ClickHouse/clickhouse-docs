## 설명 \{#description\}

사용자와 역할에 대한 역할 부여 정보를 포함합니다. 이 테이블에 항목을 추가하려면 `GRANT role TO user`를 사용합니다.

## 컬럼 \{#columns\}

{/*AUTOGENERATED_START*/ }

* `user_name` ([Nullable(String)](../../sql-reference/data-types/)) — 사용자명입니다.
* `role_name` ([Nullable(String)](../../sql-reference/data-types/)) — 역할명입니다.
* `granted_role_name` ([String](../../sql-reference/data-types/)) — `role_name` 역할에 부여된 역할명입니다. 한 역할을 다른 역할에 부여하려면 `GRANT role1 TO role2`를 사용하십시오.
* `granted_role_id` ([UUID](../../sql-reference/data-types/)) — 역할의 ID입니다.
* `granted_role_is_default` ([UInt8](../../sql-reference/data-types/)) — `granted_role`가 기본 역할인지 여부를 나타내는 플래그입니다. 가능한 값: • 1 — `granted_role`는 기본 역할입니다. • 0 — `granted_role`는 기본 역할이 아닙니다.
* `with_admin_option` ([UInt8](../../sql-reference/data-types/)) — `granted_role`에 `ADMIN OPTION` 특권이 있는지 여부를 나타내는 플래그입니다. 가능한 값: • 1 — 이 역할에는 `ADMIN OPTION` 특권이 있습니다. • 0 — 이 역할에는 `ADMIN OPTION` 특권이 없습니다.

{/*AUTOGENERATED_END*/ }
