# system.projection_parts \{#systemprojection_parts\}

이 테이블에는 MergeTree 계열 테이블의 프로젝션 파트에 대한 정보가 포함합니다.



## 컬럼 \{#columns\}



{/*AUTOGENERATED_START*/ }

* `partition` ([String](../../sql-reference/data-types/)) — 파티션의 이름입니다.
* `name` ([String](../../sql-reference/data-types/)) — 데이터 파트의 이름입니다.
* `part_type` ([String](../../sql-reference/data-types/)) — 데이터 파트의 저장 형식입니다. 가능한 값: Wide(컬럼별 파일)와 Compact(모든 컬럼을 하나의 파일에 저장하는 형식).
* `parent_name` ([String](../../sql-reference/data-types/)) — 원본(부모) 데이터 파트의 이름입니다.
* `parent_uuid` ([UUID](../../sql-reference/data-types/)) — 원본(부모) 데이터 파트의 UUID입니다.
* `parent_part_type` ([String](../../sql-reference/data-types/)) — 소스(부모) 데이터 파트가 사용하는 저장 포맷입니다.
* `active` ([UInt8](../../sql-reference/data-types/)) — 데이터 파트가 활성 상태인지 나타내는 플래그입니다. 데이터 파트가 활성 상태인 경우 테이블에서 사용됩니다. 그렇지 않으면 삭제 대기 상태입니다. 비활성 데이터 파트는 merge 및 mutation 작업 이후에 나타납니다.
* `marks` ([UInt64](../../sql-reference/data-types/)) — 마크의 개수입니다. 데이터 파트에 포함된 대략적인 행 수를 알아내려면 마크 개수에 인덱스 granularity(보통 8192)를 곱합니다. (adaptive granularity에서는 이 방법이 적용되지 않습니다.)
* `rows` ([UInt64](../../sql-reference/data-types/)) — 행의 수입니다.
* `bytes_on_disk` ([UInt64](../../sql-reference/data-types/)) — 모든 데이터 파트 파일의 총 크기(바이트 단위)입니다.
* `data_compressed_bytes` ([UInt64](../../sql-reference/data-types/)) — 데이터 파트의 압축된 데이터 전체 크기입니다. 마크 파일과 같은 모든 보조 파일은 포함되지 않습니다.
* `data_uncompressed_bytes` ([UInt64](../../sql-reference/data-types/)) — 데이터 파트의 압축되지 않은 데이터 총 크기입니다. 모든 보조 파일(예: 마크 파일)은 포함되지 않습니다.
* `marks_bytes` ([UInt64](../../sql-reference/data-types/)) — 마크 파일의 크기(바이트)입니다.
* `parent_marks` ([UInt64](../../sql-reference/data-types/)) — 소스(부모) 파트에 포함된 마크 개수입니다.
* `parent_rows` ([UInt64](../../sql-reference/data-types/)) — 소스(부모) 파트의 행 수입니다.
* `parent_bytes_on_disk` ([UInt64](../../sql-reference/data-types/)) — 모든 원본(부모) 데이터 파트 파일의 총 크기(바이트 단위)입니다.
* `parent_data_compressed_bytes` ([UInt64](../../sql-reference/data-types/)) — 소스(상위) 데이터 파트의 압축된 데이터 총 크기입니다.
* `parent_data_uncompressed_bytes` ([UInt64](../../sql-reference/data-types/)) — 소스(부모) 데이터 파트의 비압축 데이터 총 크기입니다.
* `parent_marks_bytes` ([UInt64](../../sql-reference/data-types/)) — 원본(상위) 데이터 파트에 있는 마크 파일의 크기입니다.
* `modification_time` ([DateTime](../../sql-reference/data-types/)) — 데이터 파트가 저장된 디렉터리가 마지막으로 수정된 시각입니다. 일반적으로 데이터 파트가 생성된 시각과 일치합니다.
* `remove_time` ([DateTime](../../sql-reference/data-types/)) — 데이터 파트가 비활성화된 시각입니다.
* `refcount` ([UInt32](../../sql-reference/data-types/)) — 데이터 파트가 사용되는 지점의 수입니다. 값이 2보다 크면 해당 데이터 파트가 쿼리나 머지 작업에서 사용 중임을 나타냅니다.
* `min_date` ([Date](../../sql-reference/data-types/)) — 데이터 파트 내 날짜 키의 최소값입니다.
* `max_date` ([Date](../../sql-reference/data-types/)) — 데이터 파트 내 날짜 키의 최댓값입니다.
* `min_time` ([DateTime](../../sql-reference/data-types/)) — 데이터 파트에서 날짜와 시간 키의 최소값입니다.
* `max_time` ([DateTime](../../sql-reference/data-types/)) — 데이터 파트 내 날짜 및 시간 키의 최댓값입니다.
* `partition_id` ([String](../../sql-reference/data-types/)) — 파티션의 ID입니다.
* `min_block_number` ([Int64](../../sql-reference/data-types/)) — 머지 이후 현재 파트를 구성하는 데이터 파트들 중 최소 번호입니다.
* `max_block_number` ([Int64](../../sql-reference/data-types/)) — 병합 후 현재 파트를 구성하는 데이터 파트들 중 최대 번호입니다.
* `level` ([UInt32](../../sql-reference/data-types/)) — MergeTree의 깊이입니다. 0이면 현재 파트는 다른 파트를 병합한 것이 아니라 INSERT 작업으로 생성되었음을 의미합니다.
* `data_version` ([UInt64](../../sql-reference/data-types/)) — 데이터 파트에 어떤 뮤테이션을 적용할지 결정하는 데 사용되는 숫자입니다(버전이 `data_version`보다 높은 뮤테이션이 적용됨).
* `primary_key_bytes_in_memory` ([UInt64](../../sql-reference/data-types/)) — 기본 키 값에 사용되는 메모리 크기(바이트 단위)입니다.
* `primary_key_bytes_in_memory_allocated` ([UInt64](../../sql-reference/data-types/)) — 프라이머리 키 값에 할당된 메모리 양(바이트 단위)입니다.
* `is_frozen` ([UInt8](../../sql-reference/data-types/)) — 파티션 데이터 백업이 존재하는지를 나타내는 플래그입니다. 1이면 백업이 존재하고, 0이면 백업이 존재하지 않습니다.
* `database` ([String](../../sql-reference/data-types/)) — 데이터베이스의 이름입니다.
* `table` ([String](../../sql-reference/data-types/)) — 테이블의 이름입니다.
* `engine` ([String](../../sql-reference/data-types/)) — 파라미터를 제외한 테이블 엔진의 이름입니다.
* `disk_name` ([String](../../sql-reference/data-types/)) — 데이터 파트를 저장하는 디스크의 이름입니다.
* `path` ([String](../../sql-reference/data-types/)) — 데이터 파트 파일들이 위치한 폴더의 절대 경로입니다.
* `hash_of_all_files` ([String](../../sql-reference/data-types/)) — 압축된 파일들의 sipHash128 값입니다.
* `hash_of_uncompressed_files` ([String](../../sql-reference/data-types/)) — 압축되지 않은 파일(마크 파일, 인덱스 파일 등)의 sipHash128 값입니다.
* `uncompressed_hash_of_compressed_files` ([String](../../sql-reference/data-types/)) — 압축된 파일의 데이터를 비압축된 것처럼 간주하여 계산한 sipHash128 값입니다.
* `delete_ttl_info_min` ([DateTime](../../sql-reference/data-types/)) — TTL DELETE 규칙에서 사용하는 날짜와 시간 키의 최소값입니다.
* `delete_ttl_info_max` ([DateTime](../../sql-reference/data-types/)) — TTL DELETE 규칙의 날짜·시간 키에 대한 최대값입니다.
* `move_ttl_info.expression` ([Array(String)](../../sql-reference/data-types/)) — TTL MOVE 규칙을 정의하는 표현식의 배열입니다.
* `move_ttl_info.min` ([Array(DateTime)](../../sql-reference/data-types/)) — 날짜와 시간 값으로 이루어진 배열입니다. 각 요소는 TTL MOVE 규칙에서 사용되는 최소 키 값을 나타냅니다.
* `move_ttl_info.max` ([Array(DateTime)](../../sql-reference/data-types/)) — 날짜와 시간 값의 배열입니다. 각 요소는 TTL MOVE 규칙의 최대 키 값을 나타냅니다.
* `default_compression_codec` ([String](../../sql-reference/data-types/)) — 이 데이터 파트를 압축하는 데 사용된 코덱의 이름입니다(컬럼에 대해 명시적인 코덱이 지정되지 않은 경우).
* `recompression_ttl_info.expression` ([Array(String)](../../sql-reference/data-types/)) — TTL 표현식입니다.
* `recompression_ttl_info.min` ([Array(DateTime)](../../sql-reference/data-types/)) — 이 파트에서 계산된 TTL 표현식의 최소값입니다. TTL이 만료된 행이 하나 이상 존재하는지 파악하는 데 사용됩니다.
* `recompression_ttl_info.max` ([Array(DateTime)](../../sql-reference/data-types/)) — 이 파트에서 계산된 TTL 표현식 값들 가운데 최대값입니다. 해당 파트의 모든 행에서 TTL이 만료되었는지 파악하는 데 사용됩니다.
* `group_by_ttl_info.expression` ([Array(String)](../../sql-reference/data-types/)) — TTL 표현식입니다.
* `group_by_ttl_info.min` ([Array(DateTime)](../../sql-reference/data-types/)) — 이 파트 내에서 계산된 TTL 표현식의 최소값입니다. TTL이 만료된 행이 하나 이상 있는지 파악하는 데 사용됩니다.
* `group_by_ttl_info.max` ([Array(DateTime)](../../sql-reference/data-types/)) — 이 파트에서 계산된 TTL 표현식 값 중 최대값입니다. 이 파트의 모든 행의 TTL이 만료되었는지 파악하는 데 사용됩니다.
* `rows_where_ttl_info.expression` ([Array(String)](../../sql-reference/data-types/)) — TTL 표현식입니다.
* `rows_where_ttl_info.min` ([Array(DateTime)](../../sql-reference/data-types/)) — 이 파트에서 계산된 TTL 식의 최소값입니다. 만료된 TTL이 설정된 행이 하나 이상 있는지 확인하는 데 사용됩니다.
* `rows_where_ttl_info.max` ([Array(DateTime)](../../sql-reference/data-types/)) — 이 파트 내에서 계산된 TTL 식의 최댓값입니다. TTL이 만료된 모든 행을 포함하고 있는지 파악하는 데 사용됩니다.
* `is_broken` ([UInt8](../../sql-reference/data-types/)) — PROJECTION 파트가 손상되었는지를 나타냅니다.
* `exception_code` ([Int32](../../sql-reference/data-types/)) — PROJECTION 파트의 비정상 상태를 설명하는 예외 메시지
* `exception` ([String](../../sql-reference/data-types/)) — PROJECTION 파트의 손상된 상태를 나타내는 예외 코드
