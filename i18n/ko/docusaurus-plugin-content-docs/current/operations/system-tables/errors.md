에러 코드와 해당 코드가 발생한 횟수를 포함합니다.

발생하지 않은 에러 코드를 포함하여 가능한 모든 에러 코드를 표시하려면 [system&#95;events&#95;show&#95;zero&#95;values](../settings/settings.md#system_events_show_zero_values) 설정을 1로 설정합니다.

컬럼:

{/*AUTOGENERATED_START*/ }

* `name` ([String](../../sql-reference/data-types/)) — 에러 이름입니다(errorCodeToName).
* `code` ([Int32](../../sql-reference/data-types/)) — 에러 코드 번호입니다.
* `value` ([UInt64](../../sql-reference/data-types/)) — 이 에러가 발생한 횟수입니다.
* `last_error_time` ([DateTime](../../sql-reference/data-types/)) — 마지막 에러가 발생한 시각입니다.
* `last_error_message` ([String](../../sql-reference/data-types/)) — 마지막으로 발생한 에러 메시지입니다.
* `last_error_format_string` ([String](../../sql-reference/data-types/)) — 마지막으로 발생한 에러의 형식 문자열입니다.
* `last_error_trace` ([Array(UInt64)](../../sql-reference/data-types/)) — 호출된 메서드가 저장된 물리적 주소 목록을 나타내는 스택 트레이스입니다.
* `remote` ([UInt8](../../sql-reference/data-types/)) — 원격 예외입니다(예: 분산 쿼리 중 하나에서 수신된 경우).
* `query_id` ([String](../../sql-reference/data-types/)) — 에러를 유발한 쿼리의 ID입니다(가능한 경우).

{/*AUTOGENERATED_END*/ }

:::note
일부 오류 카운터는 쿼리가 성공적으로 실행되는 동안에도 증가할 수 있습니다. 해당 오류가 오탐(false positive)이 될 수 없다는 확신이 없다면 이 테이블을 서버 모니터링 용도로 사용하는 것은 권장되지 않습니다.
:::

**예시**

```sql
SELECT name, code, value
FROM system.errors
WHERE value > 0
ORDER BY code ASC
LIMIT 1

┌─name─────────────┬─code─┬─value─┐
│ CANNOT_OPEN_FILE │   76 │     1 │
└──────────────────┴──────┴───────┘
```

```sql
WITH arrayMap(x -> demangle(addressToSymbol(x)), last_error_trace) AS all
SELECT name, arrayStringConcat(all, '\n') AS res
FROM system.errors
LIMIT 1
SETTINGS allow_introspection_functions=1\G
```
