## 설명 \{#description\}

`system.background_schedule_pool_log` 테이블은 [background&#95;schedule&#95;pool&#95;log](/operations/server-configuration-parameters/settings#background_schedule_pool_log) 서버 설정이 지정된 경우에만 생성됩니다.

이 테이블에는 백그라운드 스케줄 풀 작업 실행 이력이 포함됩니다. 백그라운드 스케줄 풀은 분산 전송, 버퍼 플러시, 메시지 브로커 작업과 같은 주기적 작업을 실행하는 데 사용됩니다.

## 컬럼 \{#columns\}

{/*AUTOGENERATED_START*/ }

* `hostname` ([LowCardinality(String)](/sql-reference/data-types/lowcardinality)) — 쿼리를 실행하는 서버의 호스트명입니다.
* `event_date` ([Date](/sql-reference/data-types/date)) — 이벤트가 발생한 날짜입니다.
* `event_time` ([DateTime](/sql-reference/data-types/datetime)) — 이벤트가 발생한 시간입니다.
* `event_time_microseconds` ([DateTime64(6)](/sql-reference/data-types/datetime64)) — 마이크로초 단위까지 포함한 이벤트 발생 시간입니다.
* `query_id` ([String](/sql-reference/data-types/string)) — 백그라운드 작업과 연관된 쿼리의 식별자입니다.
* `database` ([LowCardinality(String)](/sql-reference/data-types/lowcardinality)) — 데이터베이스 이름입니다.
* `table` ([LowCardinality(String)](/sql-reference/data-types/lowcardinality)) — 테이블 이름입니다.
* `table_uuid` ([UUID](/sql-reference/data-types/uuid)) — 백그라운드 작업이 속한 테이블의 UUID입니다.
* `log_name` ([LowCardinality(String)](/sql-reference/data-types/lowcardinality)) — 백그라운드 작업 이름입니다.
* `duration_ms` ([UInt64](/sql-reference/data-types/int-uint)) — 작업 실행 시간(밀리초)입니다.
* `error` ([UInt16](/sql-reference/data-types/int-uint)) — 발생한 예외의 오류 코드입니다.
* `exception` ([String](/sql-reference/data-types/string)) — 발생한 오류의 메시지 텍스트입니다.

{/*AUTOGENERATED_END*/ }

`system.background_schedule_pool_log` 테이블은 백그라운드 작업이 처음 실행된 후 생성됩니다.
