## 설명 \{#description\}

`system.histogram_metrics`의 이력입니다. `collect_interval_milliseconds`마다 스냅샷이 생성되어 디스크에 플러시됩니다.

## 컬럼 \{#columns\}

{/*AUTOGENERATED_START*/ }

* `hostname` ([LowCardinality(String)](/sql-reference/data-types/lowcardinality)) — 쿼리를 실행하는 서버의 호스트명입니다.
* `event_date` ([Date](/sql-reference/data-types/date)) — 이벤트 날짜입니다.
* `event_time` ([DateTime](/sql-reference/data-types/datetime)) — 이벤트 시각입니다.
* `event_time_microseconds` ([DateTime64(6)](/sql-reference/data-types/datetime64)) — 마이크로초 단위 정밀도의 이벤트 시각입니다.
* `metric` ([LowCardinality(String)](/sql-reference/data-types/lowcardinality)) — 메트릭 이름입니다.
* `labels` ([Map(LowCardinality(String), LowCardinality(String))](/sql-reference/data-types/map)) — 메트릭 레이블입니다.
* `histogram` ([Map(Float64, UInt64)](/sql-reference/data-types/map)) — 누적 히스토그램입니다. 버킷 상한을 해당 경계값 이하의 관측값 수에 매핑하며, 마지막 버킷으로 +inf를 포함합니다.
* `count` ([UInt64](/sql-reference/data-types/int-uint)) — 전체 관측값 수이며, histogram[+inf]와 같습니다.
* `sum` ([Float64](/sql-reference/data-types/float)) — 관측된 모든 값의 합계입니다.

{/*AUTOGENERATED_END*/ }

## 예시 \{#example\}

```sql
SELECT event_time, metric, labels, histogram
FROM system.histogram_metric_log
WHERE metric = 'keeper_response_time_ms'
ORDER BY event_time DESC
LIMIT 1
FORMAT Vertical;
```
