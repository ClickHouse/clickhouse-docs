## 설명 \{#description\}

현재 상태와 기타 속성을 포함한 모든 `BACKUP` 또는 `RESTORE` 작업의 목록을 제공합니다. 이 테이블은 지속되지 않으며 마지막 서버 재시작 이후에 실행된 작업만 표시합니다.

## 컬럼 \{#columns\}

{/*AUTOGENERATED_START*/ }

* `id` ([String](../../sql-reference/data-types/)) — 작업 ID입니다. `SETTINGS id=...`로 전달되거나 무작위로 생성된 UUID일 수 있습니다.
* `name` ([String](../../sql-reference/data-types/)) — 작업 이름이며, `Disk('backups', 'my_backup')`와 같은 문자열입니다.
* `base_backup_name` ([String](../../sql-reference/data-types/)) — 기준 백업 작업 이름이며, `Disk('backups', 'my_base_backup')`와 같은 문자열입니다.
* `query_id` ([String](../../sql-reference/data-types/)) — 백업을 시작한 쿼리의 Query id입니다.
* `status` ([Enum8(&#39;CREATING&#95;BACKUP&#39; = 0, &#39;BACKUP&#95;CREATED&#39; = 1, &#39;BACKUP&#95;FAILED&#39; = 2, &#39;RESTORING&#39; = 3, &#39;RESTORED&#39; = 4, &#39;RESTORE&#95;FAILED&#39; = 5, &#39;BACKUP&#95;CANCELLED&#39; = 6, &#39;RESTORE&#95;CANCELLED&#39; = 7)](../../sql-reference/data-types/)) — 백업 또는 복원 작업의 상태입니다.
* `error` ([String](../../sql-reference/data-types/)) — 오류가 발생한 경우의 오류 메시지입니다.
* `start_time` ([DateTime64(6)](../../sql-reference/data-types/)) — 작업이 시작된 시각입니다.
* `end_time` ([DateTime64(6)](../../sql-reference/data-types/)) — 작업이 완료된 시각입니다.
* `num_files` ([UInt64](../../sql-reference/data-types/)) — 백업에 저장된 파일 수입니다.
* `total_size` ([UInt64](../../sql-reference/data-types/)) — 백업에 저장된 파일의 총 크기입니다.
* `num_entries` ([UInt64](../../sql-reference/data-types/)) — 백업의 엔트리 수입니다. 즉, 백업이 폴더로 저장된 경우 폴더 내부의 파일 수입니다.
* `uncompressed_size` ([UInt64](../../sql-reference/data-types/)) — 백업의 비압축 크기입니다.
* `compressed_size` ([UInt64](../../sql-reference/data-types/)) — 백업의 압축 크기입니다.
* `files_read` ([UInt64](../../sql-reference/data-types/)) — 이 백업에서 RESTORE 중 읽은 파일 수입니다.
* `bytes_read` ([UInt64](../../sql-reference/data-types/)) — 이 백업에서 RESTORE 중 읽은 파일의 총 크기입니다.
* `ProfileEvents` ([Map(LowCardinality(String), UInt64)](../../sql-reference/data-types/)) — 이 작업 중 수집된 모든 profile events입니다.
