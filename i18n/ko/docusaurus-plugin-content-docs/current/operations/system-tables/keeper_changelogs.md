## 설명 \{#description\}

이 노드가 프로세스 내 ClickHouse Keeper를 실행하도록 구성되지 않은 경우 이 테이블(table)은 존재하지 않습니다. 이 테이블에는 프로세스 내 Keeper 로그 저장소가 추적하는 각 Raft changelog 파일(`changelog_<from>_<to>.bin[.zstd]`)마다 1개의 행(row)이 있으며, 현재 기록이 계속 추가되고 있는 활성 파일도 포함됩니다.

## 컬럼 \{#columns\}

{/*AUTOGENERATED_START*/ }

* `from_log_index` ([UInt64](../../sql-reference/data-types/int-uint.md)) — 파일에 있는 첫 번째 Raft 로그 인덱스입니다(포함).
* `to_log_index` ([UInt64](../../sql-reference/data-types/int-uint.md)) — 파일 이름이 포함하는 마지막 Raft 로그 인덱스입니다(포함). 활성 파일의 경우 이는 로테이션 대상이며 `last_entry_index`보다 클 수 있습니다.
* `last_entry_index` ([Nullable(UInt64)](../../sql-reference/data-types/int-uint.md)) — 실제로 이 파일에 추가된 가장 큰 로그 인덱스입니다. 활성 파일이 아직 어떤 엔트리도 받지 않았거나 파일이 손상된 경우 `NULL`입니다.
* `entries` ([UInt64](../../sql-reference/data-types/int-uint.md)) — 이 파일에 추가된 엔트리 수입니다. `ifNull(last_entry_index - from_log_index + 1, 0)`의 `ALIAS`입니다.
* `path` ([String](../../sql-reference/data-types/string.md)) — 디스크에 있는 파일 경로입니다.
* `disk_name` ([String](../../sql-reference/data-types/string.md)) — 파일이 저장된 디스크 이름입니다.
* `size_bytes` ([UInt64](../../sql-reference/data-types/int-uint.md)) — 디스크에 있는 파일의 크기입니다.
* `modification_time` ([DateTime](../../sql-reference/data-types/datetime.md)) — 파일의 마지막 수정 시간입니다.
* `is_compressed` ([Bool](../../sql-reference/data-types/boolean.md)) — 파일 payload가 zstd로 압축되어 있습니다.
* `active` ([Bool](../../sql-reference/data-types/boolean.md)) — 현재 이 파일에 엔트리가 추가되고 있습니다.
* `is_broken` ([Bool](../../sql-reference/data-types/boolean.md)) — 시작 시 후행 레코드가 손상된 것으로 확인되었습니다.

{/*AUTOGENERATED_END*/ }

예시:

```sql
SELECT from_log_index, to_log_index, entries, path, active FROM system.keeper_changelogs ORDER BY from_log_index;
```

```text
┌─from_log_index─┬─to_log_index─┬─entries─┬─path───────────────────────────┬─active─┐
│              1 │         1000 │    1000 │ changelog_1_1000.bin.zstd      │ false  │
│           1001 │         2000 │     537 │ changelog_1001_2000.bin.zstd   │ true   │
└────────────────┴──────────────┴─────────┴────────────────────────────────┴────────┘
```
