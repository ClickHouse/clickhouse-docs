# system.row_policies \{#systemrow_policies\}

하나의 특정 테이블에 대한 필터와, 이 ROW POLICY가 적용되어야 하는 역할 및/또는 사용자 목록을 포함합니다.

컬럼:

{/*AUTOGENERATED_START*/ }

* `name` ([String](../../sql-reference/data-types/)) — row policy의 이름입니다.
* `short_name` ([String](../../sql-reference/data-types/)) — row policy의 짧은 이름입니다. row policy의 이름은 예를 들어 `myfilter ON mydb.mytable`과 같이 복합으로 구성됩니다. 여기서 「myfilter ON mydb.mytable」은 row policy의 이름이고, 「myfilter」는 그 짧은 이름입니다.
* `database` ([String](../../sql-reference/data-types/)) — 데이터베이스 이름입니다.
* `table` ([String](../../sql-reference/data-types/)) — 테이블 이름입니다. 데이터베이스에 대한 policy인 경우 비어 있습니다.
* `id` ([UUID](../../sql-reference/data-types/)) — row policy ID입니다.
* `storage` ([String](../../sql-reference/data-types/)) — row policy가 저장되는 디렉터리 이름입니다.
* `select_filter` ([Nullable(String)](../../sql-reference/data-types/)) — `SELECT` 쿼리에서 필터링에 사용되는 식입니다.
* `is_restrictive` ([UInt8](../../sql-reference/data-types/)) — row policy가 행에 대한 액세스를 제한하는지 여부를 나타냅니다. 값: • 0 — row policy가 `AS PERMISSIVE` 절로 정의된 경우, • 1 — row policy가 AS RESTRICTIVE 절로 정의된 경우입니다.
* `apply_to_all` ([UInt8](../../sql-reference/data-types/)) — 모든 역할(role) 및/또는 사용자(user)에 대해 row policy가 설정되어 있음을 나타냅니다.
* `apply_to_list` ([Array(String)](../../sql-reference/data-types/)) — row policy가 적용되는 역할 및/또는 사용자 목록입니다.
* `apply_to_except` ([Array(String)](../../sql-reference/data-types/)) — 나열된 항목을 제외한 모든 역할 및/또는 사용자에 row policy가 적용됩니다.
