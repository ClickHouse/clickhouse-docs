## 설명 \{#description\}

현재 세그먼트 간에 이동 중인 파트와 그 진행 상태에 대한 정보를 포함합니다.

## 컬럼 \{#columns\}

{/*AUTOGENERATED_START*/ }

* `database` ([String](../../sql-reference/data-types/)) — 이동이 수행되는 데이터베이스 이름입니다.
* `table` ([String](../../sql-reference/data-types/)) — 이동이 수행되는 테이블 이름입니다.
* `task_name` ([String](../../sql-reference/data-types/)) — 이동 작업 이름입니다.
* `task_uuid` ([UUID](../../sql-reference/data-types/)) — 이동 작업의 식별자입니다.
* `create_time` ([DateTime](../../sql-reference/data-types/)) — 작업이 생성된 시각입니다.
* `part_name` ([String](../../sql-reference/data-types/)) — 이동 중인 파트의 이름입니다.
* `part_uuid` ([UUID](../../sql-reference/data-types/)) — 이동 중인 파트의 UUID입니다.
* `to_shard` ([String](../../sql-reference/data-types/)) — 대상 세그먼트의 이름입니다.
* `dst_part_name` ([String](../../sql-reference/data-types/)) — 결과 파트의 이름입니다.
* `update_time` ([DateTime](../../sql-reference/data-types/)) — 마지막 업데이트가 수행된 시각입니다.
* `state` ([String](../../sql-reference/data-types/)) — 이동의 현재 상태입니다.
* `rollback` ([UInt8](../../sql-reference/data-types/)) — 작업이 롤백되었는지 여부를 나타내는 플래그입니다.
* `num_tries` ([UInt32](../../sql-reference/data-types/)) — 작업을 완료하기까지의 시도 횟수입니다.
* `last_exception` ([String](../../sql-reference/data-types/)) — 예외가 발생한 경우 마지막 예외의 이름입니다.
