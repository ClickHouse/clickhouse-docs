## 설명 \{#description\}

모든 형식의 목록과 함께, 각 형식이 입력/출력에 적합한지와 병렬화를 지원하는지 여부를 포함합니다.

## 컬럼 \{#columns\}

{/*AUTOGENERATED_START*/ }

* `name` ([String](../../sql-reference/data-types/)) — 형식 이름입니다.
* `is_input` ([UInt8](../../sql-reference/data-types/)) — 해당 형식이 데이터 입력에 적합한지를 나타내는 플래그입니다.
* `is_output` ([UInt8](../../sql-reference/data-types/)) — 해당 형식이 데이터 출력에 적합한지를 나타내는 플래그입니다.
* `supports_parallel_parsing` ([UInt8](../../sql-reference/data-types/)) — 해당 형식이 병렬 파싱을 지원하는지를 나타내는 플래그입니다.
* `supports_parallel_formatting` ([UInt8](../../sql-reference/data-types/)) — 해당 형식이 병렬 포맷팅을 지원하는지를 나타내는 플래그입니다.
* `is_tty_friendly` ([UInt8](../../sql-reference/data-types/)) — 해당 형식이 일반적으로 터미널에 잘 표시되는지를 나타내는 플래그입니다. 다른 형식의 경우 CLI가 출력 전에 확인을 요청합니다.
* `content_type` ([String](../../sql-reference/data-types/)) — 출력 형식에 대응하는 HTTP Content-Type입니다. 현재 형식 설정에 따라 달라질 수 있습니다.
* `supports_random_access` ([UInt8](../../sql-reference/data-types/)) — 해당 형식이 입력에서 임의 접근을 지원하는지를 나타내는 플래그입니다.
* `has_schema_inference` ([UInt8](../../sql-reference/data-types/)) — 해당 형식은 데이터에서 schema를 동적으로 판별할 수 있습니다(내장된 헤더/메타데이터 또는 데이터 일부에서).
* `has_external_schema` ([UInt8](../../sql-reference/data-types/)) — 해당 형식은 고정된 schema를 가지거나, 자체 형식으로 사전 정의된 schema를 받을 수 있습니다.
* `prefers_large_blocks` ([UInt8](../../sql-reference/data-types/)) — 해당 형식은 출력 시 더 큰 블록을 기록하고 입력 시 더 큰 블록을 생성합니다.
* `supports_append` ([UInt8](../../sql-reference/data-types/)) — 이 형식은 단일 파일에 내용을 추가할 수 있습니다.
* `supports_subsets_of_columns` ([UInt8](../../sql-reference/data-types/)) — 입력 형식은 일부 컬럼이 생략된 경우를 인식할 수 있습니다.
