## 설명 \{#description\}

로컬 서버에 존재하는 [스케줄링 노드](/operations/workload-scheduling.md/#hierarchy)에 대한 정보와 상태를 제공합니다.
이 테이블은 모니터링용으로 사용할 수 있습니다. 테이블에는 각 스케줄링 노드마다 1개의 행이 포함됩니다.

## 컬럼 \{#columns\}

{/*AUTOGENERATED_START*/ }

* `resource` ([String](../../sql-reference/data-types/)) — 리소스 이름
* `path` ([String](../../sql-reference/data-types/)) — 이 리소스 스케줄링 계층 구조에서 스케줄링 노드까지의 경로
* `type` ([String](../../sql-reference/data-types/)) — 스케줄링 노드의 유형입니다.
* `weight` ([Float64](../../sql-reference/data-types/)) — 상위 `fair` 타입 노드에서 사용하는 노드의 가중치입니다.
* `priority` ([Int64](../../sql-reference/data-types/)) — 노드의 우선순위입니다. 상위 `priority` 타입 노드에서 사용되며, 값이 낮을수록 우선순위가 높습니다.
* `is_active` ([UInt8](../../sql-reference/data-types/)) — 이 노드가 현재 활성 상태인지, 즉 디큐할 리소스 요청이 있고 제약 조건이 충족되었는지를 나타냅니다.
* `active_children` ([UInt64](../../sql-reference/data-types/)) — 활성 상태인 자식 수입니다.
* `dequeued_requests` ([UInt64](../../sql-reference/data-types/)) — 이 노드에서 디큐된 리소스 요청의 총개수입니다.
* `canceled_requests` ([UInt64](../../sql-reference/data-types/)) — 이 노드에서 취소된 리소스 요청의 총개수입니다.
* `dequeued_cost` ([Int64](../../sql-reference/data-types/)) — 이 노드에서 디큐된 모든 요청의 비용(예: 바이트 단위 크기) 합계입니다.
* `throughput` ([Float64](../../sql-reference/data-types/)) — 현재 평균 처리량(초당 디큐 비용)입니다.
* `canceled_cost` ([Int64](../../sql-reference/data-types/)) — 이 노드에서 취소된 모든 요청의 비용(예: 바이트 단위 크기) 합계입니다.
* `busy_periods` ([UInt64](../../sql-reference/data-types/)) — 이 노드가 비활성화된 총횟수입니다.
* `vruntime` ([Nullable(Float64)](../../sql-reference/data-types/)) — `fair` 노드의 자식에만 해당합니다. SFQ 알고리즘이 max-min 공정 방식으로 다음에 처리할 자식을 선택할 때 사용하는 노드의 가상 런타임입니다.
* `system_vruntime` ([Nullable(Float64)](../../sql-reference/data-types/)) — `fair` 노드에만 해당합니다. 마지막으로 처리된 리소스 요청의 `vruntime`을 나타내는 가상 런타임입니다. 자식 활성화 시 `vruntime`의 새 값으로 사용됩니다.
* `queue_length` ([Nullable(UInt64)](../../sql-reference/data-types/)) — `fifo` 노드에만 해당합니다. 현재 큐에 있는 리소스 요청 수입니다.
* `queue_cost` ([Nullable(Int64)](../../sql-reference/data-types/)) — `fifo` 노드에만 해당합니다. 큐에 있는 모든 요청의 비용(예: 바이트 단위 크기) 합계입니다.
* `budget` ([Nullable(Int64)](../../sql-reference/data-types/)) — `fifo` 노드에만 해당합니다. 새 리소스 요청에 사용할 수 있는 &#39;cost units&#39;의 개수입니다. 리소스 요청의 추정 비용과 실제 비용이 서로 다를 경우(예: 읽기/쓰기 실패 후) 나타날 수 있습니다.
* `is_satisfied` ([Nullable(UInt8)](../../sql-reference/data-types/)) — 제약 조건 노드(예: `inflight_limit`)에만 해당합니다. 이 노드의 모든 제약 조건이 충족되면 `1`입니다.
* `inflight_requests` ([Nullable(Int64)](../../sql-reference/data-types/)) — `inflight_limit` 노드에만 해당합니다. 이 노드에서 디큐되어 현재 소비 상태에 있는 리소스 요청 수입니다.
* `inflight_cost` ([Nullable(Int64)](../../sql-reference/data-types/)) — `inflight_limit` 노드에만 해당합니다. 이 노드에서 디큐되어 현재 소비 상태에 있는 모든 리소스 요청의 비용(예: 바이트) 합계입니다.
* `max_requests` ([Nullable(Int64)](../../sql-reference/data-types/)) — `inflight_limit` 노드에만 해당합니다. `inflight_requests`가 이를 초과하면 제약 조건 위반이 발생하는 상한값입니다.
* `max_cost` ([Nullable(Int64)](../../sql-reference/data-types/)) — `inflight_limit` 노드에만 해당합니다. `inflight_cost`가 이를 초과하면 제약 조건 위반이 발생하는 상한값입니다.
* `max_speed` ([Nullable(Float64)](../../sql-reference/data-types/)) — `bandwidth_limit` 노드에만 해당합니다. 초당 토큰 수로 표현한 대역폭 상한값입니다.
* `max_burst` ([Nullable(Float64)](../../sql-reference/data-types/)) — `bandwidth_limit` 노드에만 해당합니다. token-bucket 스로틀러에서 사용할 수 있는 토큰 수의 상한값입니다.
* `throttling_us` ([Nullable(Int64)](../../sql-reference/data-types/)) — `bandwidth_limit` 노드에만 해당합니다. 이 노드가 스로틀링 상태에 있었던 총 마이크로초 수입니다.
* `tokens` ([Nullable(Float64)](../../sql-reference/data-types/)) — `bandwidth_limit` 노드에만 해당합니다. token-bucket 스로틀러에서 현재 사용할 수 있는 토큰 수입니다.
