# system.scheduler \{#systemscheduler\}

<SystemTableCloud />

로컬 서버에 존재하는 [scheduling node](/operations/workload-scheduling.md/#hierarchy)에 대한 정보와 상태를 제공합니다.
이 테이블은 모니터링용으로 사용할 수 있습니다. 테이블에는 각 scheduling node마다 1개의 행이 포함됩니다.

예:

```sql
SELECT *
FROM system.scheduler
WHERE resource = 'network_read' AND path = '/prio/fair/prod'
FORMAT Vertical
```

컬럼:

{/*AUTOGENERATED_START*/ }

* `resource` ([String](../../sql-reference/data-types/)) — 리소스 이름
* `path` ([String](../../sql-reference/data-types/)) — 이 리소스 스케줄링 계층 구조 내에서 스케줄링 노드까지의 경로
* `type` ([String](../../sql-reference/data-types/)) — 스케줄링 노드의 유형입니다.
* `weight` ([Float64](../../sql-reference/data-types/)) — 상위 `fair` 타입 노드에서 사용하는 노드의 가중치입니다.
* `priority` ([Int64](../../sql-reference/data-types/)) — 노드의 우선순위입니다. 「priority」 유형의 상위 노드에서 사용되며, 값이 낮을수록 우선순위가 높습니다.
* `is_active` ([UInt8](../../sql-reference/data-types/)) — 이 노드가 현재 활성 상태인지, 즉 디큐(dequeue) 대기 중인 리소스 요청이 있고 제약 조건이 충족되었는지를 나타냅니다.
* `active_children` ([UInt64](../../sql-reference/data-types/)) — 활성 상태인 자식 수입니다.
* `dequeued_requests` ([UInt64](../../sql-reference/data-types/)) — 이 노드에서 큐에서 제거(dequeue)된 리소스 요청의 총 개수입니다.
* `canceled_requests` ([UInt64](../../sql-reference/data-types/)) — 이 노드에서 취소된 리소스 요청의 총 횟수입니다.
* `dequeued_cost` ([Int64](../../sql-reference/data-types/)) — 이 노드에서 큐에서 제거(dequeue)된 모든 요청의 비용(예: 바이트 단위 크기)의 합입니다.
* `throughput` ([Float64](../../sql-reference/data-types/)) — 현재 평균 처리량(초당 디큐되는 비용).
* `canceled_cost` ([Int64](../../sql-reference/data-types/)) — 이 노드에서 취소된 모든 요청의 비용(예: 바이트 단위 크기)의 총합입니다.
* `busy_periods` ([UInt64](../../sql-reference/data-types/)) — 이 노드가 비활성화된 총 횟수입니다.
* `vruntime` ([Nullable(Float64)](../../sql-reference/data-types/)) — `fair` 노드의 자식 노드에만 해당합니다. SFQ 알고리즘이 max-min 공정 방식으로 다음에 처리할 자식 노드를 선택할 때 사용하는 노드의 가상 실행 시간입니다.
* `system_vruntime` ([널 허용(Float64)](../../sql-reference/data-types/)) — `fair` 노드에서만 사용됩니다. 마지막으로 처리된 리소스 요청의 `vruntime` 값을 나타내는 가상 런타임입니다. 자식이 활성화될 때 `vruntime`의 새 값으로 사용됩니다.
* `queue_length` ([널 허용(UInt64)](../../sql-reference/data-types/)) — `fifo` 노드에만 해당합니다. 현재 대기열에 있는 리소스 요청 수입니다.
* `queue_cost` ([널 허용(Int64)](../../sql-reference/data-types/)) — FIFO 노드에만 해당합니다. 대기열에 있는 모든 요청의 비용(예: 바이트 단위 크기)을 합산한 값입니다.
* `budget` ([널 허용(Int64)](../../sql-reference/data-types/)) — fifo 노드에만 사용됩니다. 새로운 리소스 요청에 사용할 수 있는 &#39;cost units&#39;의 개수입니다. 리소스 요청의 추정 비용과 실제 비용이 실제 비용과 차이가 발생하는 경우(예: 읽기/쓰기 실패 이후)에 나타날 수 있습니다.
* `is_satisfied` ([Nullable(UInt8)](../../sql-reference/data-types/)) — 제약 조건 노드(예: `inflight_limit`)에서만 사용됩니다. 이 노드의 모든 제약 조건이 충족되면 `1`입니다.
* `inflight_requests` ([Nullable(Int64)](../../sql-reference/data-types/)) — `inflight_limit` 노드에만 해당합니다. 이 노드에서 디큐된 리소스 요청 중 현재 소비 상태에 있는 요청의 개수입니다.
* `inflight_cost` ([널 허용(Nullable)(Int64)](../../sql-reference/data-types/)) — `inflight_limit` 노드에만 적용됩니다. 현재 소비 상태에 있는, 이 노드에서 디큐(dequeue)된 모든 리소스 요청의 비용(예: 바이트)의 합계입니다.
* `max_requests` ([Nullable(Int64)](../../sql-reference/data-types/)) — `inflight_limit` 노드에만 해당합니다. `inflight_requests`의 상한 값으로, 이를 초과하면 제약 조건 위반이 발생합니다.
* `max_cost` ([Nullable(Int64)](../../sql-reference/data-types/)) — `inflight_limit` 노드에만 사용합니다. CONSTRAINT 위반을 발생시키는 inflight&#95;cost의 상한입니다.
* `max_speed` ([널 허용(Float64)](../../sql-reference/data-types/)) — `bandwidth_limit` 노드에만 적용됩니다. 초당 토큰 수로 표현되는 대역폭의 상한값입니다.
* `max_burst` ([Nullable(Float64)](../../sql-reference/data-types/)) — `bandwidth_limit` 노드에만 해당합니다. 토큰 버킷 스로틀러에서 사용 가능한 토큰 수의 상한값입니다.
* `throttling_us` ([널 허용(Int64)](../../sql-reference/data-types/)) — `bandwidth_limit` 노드에만 해당합니다. 이 노드가 throttling 상태였던 총 마이크로초 수입니다.
* `tokens` ([Nullable(Float64)](../../sql-reference/data-types/)) — `bandwidth_limit` 노드에만 적용됩니다. token-bucket throttler에서 현재 사용 가능한 토큰 수입니다.
