## 설명 \{#description\}

`system.part_log` 테이블은 [part&#95;log](/operations/server-configuration-parameters/settings#part_log) 서버 설정이 지정된 경우에만 생성됩니다.

이 테이블에는 [MergeTree](../../engines/table-engines/mergetree-family/mergetree.md) 계열 테이블의 [데이터 파트](../../engines/table-engines/mergetree-family/custom-partitioning-key.md)에 대해 발생한 이벤트(데이터 추가, 병합 등)에 대한 정보가 포함됩니다.

`system.part_log` 테이블에는 다음 컬럼이 포함됩니다:

## 컬럼 \{#columns\}

{/*AUTOGENERATED_START*/ }

* `hostname` ([LowCardinality(String)](/sql-reference/data-types/lowcardinality)) — 쿼리를 실행 중인 서버의 호스트명.
* `query_id` ([String](/sql-reference/data-types/string)) — 이 데이터 파트를 생성한 INSERT 쿼리의 식별자입니다.
* `event_type` ([Enum8(&#39;NewPart&#39; = 1, &#39;MergeParts&#39; = 2, &#39;DownloadPart&#39; = 3, &#39;RemovePart&#39; = 4, &#39;MutatePart&#39; = 5, &#39;MovePart&#39; = 6, &#39;MergePartsStart&#39; = 7, &#39;MutatePartStart&#39; = 8)](/sql-reference/data-types/enum)) — 데이터 파트에 대해 발생한 이벤트의 유형입니다. 다음 값 중 하나를 가질 수 있습니다: NewPart — 새 데이터 파트 삽입, MergePartsStart — 데이터 파트 병합이 시작됨, MergeParts — 데이터 파트 병합이 완료됨, DownloadPart — 데이터 파트 다운로드, RemovePart — 데이터 파트를 제거하거나 [DETACH PARTITION](/sql-reference/statements/alter/partition#detach-partitionpart)을 사용해 분리함, MutatePartStart — 데이터 파트 뮤테이션이 시작됨, MutatePart — 데이터 파트 뮤테이션이 완료됨, MovePart — 데이터 파트를 한 디스크에서 다른 디스크로 이동함.
* `merge_reason` ([Enum8(&#39;NotAMerge&#39; = 1, &#39;RegularMerge&#39; = 2, &#39;TTLDeleteMerge&#39; = 3, &#39;TTLRecompressMerge&#39; = 4, &#39;TTLDropMerge&#39; = 5)](/sql-reference/data-types/enum)) — 유형이 MERGE&#95;PARTS인 이벤트의 발생 이유를 나타냅니다. 다음 값 중 하나를 가질 수 있습니다: NotAMerge — 현재 이벤트의 유형이 MERGE&#95;PARTS가 아닙니다, RegularMerge — 일반 병합, TTLDeleteMerge, TTLDropMerge — 만료된 데이터를 정리합니다. TTLRecompressMerge — 데이터 파트를 재압축합니다.
* `merge_algorithm` ([Enum8(&#39;Undecided&#39; = 0, &#39;Vertical&#39; = 1, &#39;Horizontal&#39; = 2)](/sql-reference/data-types/enum)) — 유형이 MERGE&#95;PARTS인 이벤트의 병합 알고리즘입니다. 다음 값 중 하나를 가질 수 있습니다: Undecided, Horizontal, Vertical
* `event_date` ([Date](/sql-reference/data-types/date)) — 이벤트 날짜.
* `event_time` ([DateTime](/sql-reference/data-types/datetime)) — 이벤트 발생 시간.
* `event_time_microseconds` ([DateTime64(6)](/sql-reference/data-types/datetime64)) — 마이크로초 단위 정밀도를 지원하는 이벤트 시간입니다.
* `duration_ms` ([UInt64](/sql-reference/data-types/int-uint)) — 이 작업의 총 소요 시간입니다.
* `database` ([String](/sql-reference/data-types/string)) — 데이터 파트가 있는 데이터베이스 이름입니다.
* `table` ([String](/sql-reference/data-types/string)) — 데이터 파트가 포함된 테이블의 이름입니다.
* `table_uuid` ([UUID](/sql-reference/data-types/uuid)) — 데이터 파트가 속해 있는 테이블의 UUID입니다.
* `part_name` ([String](/sql-reference/data-types/string)) — 데이터 파트 이름입니다.
* `partition_id` ([String](/sql-reference/data-types/string)) — 데이터 파트가 삽입된 파티션의 ID입니다. 파티셔닝 기준이 `tuple()`인 경우 이 컬럼의 값은 `all`입니다.
* `partition` ([String](/sql-reference/data-types/string)) — 파티션 이름입니다.
* `part_type` ([String](/sql-reference/data-types/string)) — part의 유형입니다. 가능한 값은 Wide와 Compact입니다.
* `part_storage_type` ([String](/sql-reference/data-types/string)) — DataPartStorage의 유형입니다. 가능한 값: Packed - 모든 파일이 단일 blob에 저장됩니다. Full - 각 파일이 별도의 blob에 저장됩니다.
* `disk_name` ([String](/sql-reference/data-types/string)) — 데이터 파트가 저장된 디스크의 이름입니다.
* `path_on_disk` ([String](/sql-reference/data-types/string)) — 데이터 파트 파일이 저장된 폴더의 절대 경로입니다.
* `rows` ([UInt64](/sql-reference/data-types/int-uint)) — 데이터 파트의 행 수입니다.
* `size_in_bytes` ([UInt64](/sql-reference/data-types/int-uint)) — 디스크에 저장된 데이터 파트의 크기(바이트)입니다.
* `merged_from` ([Array(String)](/sql-reference/data-types/array)) — 현재 파트가 만들어진 원본 파트 이름의 배열입니다.
* `bytes_uncompressed` ([UInt64](/sql-reference/data-types/int-uint)) — 결과로 생성된 part의 비압축 크기(바이트)입니다.
* `read_rows` ([UInt64](/sql-reference/data-types/int-uint)) — 병합 중에 읽힌 행 수입니다.
* `read_bytes` ([UInt64](/sql-reference/data-types/int-uint)) — 병합 중 읽은 바이트 수입니다.
* `peak_memory_usage` ([UInt64](/sql-reference/data-types/int-uint)) — 병합 중 사용된 RAM의 최대 사용량
* `deduplication_block_ids` ([Array(String)](/sql-reference/data-types/array)) — 이 데이터 part를 삽입할 때 중복 제거에 사용되는 block ID의 배열입니다.
* `error` ([UInt16](/sql-reference/data-types/int-uint)) — 발생한 예외의 오류 코드입니다.
* `exception` ([String](/sql-reference/data-types/string)) — 발생한 오류의 메시지 텍스트입니다.
* `mutation_ids` ([Array(String)](/sql-reference/data-types/array)) — 유형이 MUTATE&#95;PART&#95;START 및 MUTATE&#95;PART인 이벤트에서 소스 파트(merged&#95;from)에 적용된 mutation ID 배열입니다.
* `ProfileEvents` ([Map(LowCardinality(String), UInt64)](/sql-reference/data-types/map)) — 이 작업 중에 수집된 모든 profile events입니다.
* `projections_duration_ms` ([Map(LowCardinality(String), UInt64)](/sql-reference/data-types/map)) — 프로젝션별 병합/재구축에 걸린 시간(밀리초)입니다.

**별칭:**

* `ProfileEvents.Names` — `mapKeys(ProfileEvents)`의 별칭입니다.
* `ProfileEvents.Values` — `mapValues(ProfileEvents)`의 별칭입니다.
* `name` — `part_name`의 별칭입니다.

{/*AUTOGENERATED_END*/ }

`system.part_log` 테이블은 `MergeTree` 테이블에 처음으로 데이터를 삽입한 후에 생성됩니다.
