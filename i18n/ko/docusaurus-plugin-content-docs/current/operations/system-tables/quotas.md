## 설명 \{#description\}

[쿼터](../../operations/system-tables/quotas.md)에 대한 정보를 포함합니다.

## 컬럼 \{#columns\}

{/*AUTOGENERATED_START*/ }

* `name` ([String](../../sql-reference/data-types/)) — QUOTA 이름입니다.
* `id` ([UUID](../../sql-reference/data-types/)) — QUOTA ID입니다.
* `storage` ([String](../../sql-reference/data-types/)) — 쿼터의 저장 방식입니다. 가능한 값은 다음과 같습니다. users.xml 파일에 QUOTA가 설정된 경우 &#39;users.xml&#39;, SQL 쿼리로 QUOTA가 설정된 경우 &#39;disk&#39;입니다.
* `keys` ([Array(Enum8(&#39;user&#95;name&#39; = 1, &#39;ip&#95;address&#39; = 2, &#39;forwarded&#95;ip&#95;address&#39; = 3, &#39;client&#95;key&#39; = 4, &#39;normalized&#95;query&#95;hash&#39; = 7))](../../sql-reference/data-types/)) — 키는 QUOTA를 어떤 방식으로 공유할지 지정합니다. 두 연결이 동일한 QUOTA와 키를 사용하면 동일한 리소스 한도를 공유합니다. 값: [] — 모든 사용자가 동일한 QUOTA를 공유합니다, [&#39;user&#95;name&#39;] — 동일한 사용자 이름을 사용하는 연결이 동일한 QUOTA를 공유합니다, [&#39;ip&#95;address&#39;] — 동일한 IP에서 오는 연결이 동일한 QUOTA를 공유합니다. [&#39;client&#95;key&#39;] — 동일한 키를 사용하는 연결이 동일한 QUOTA를 공유합니다. 키는 클라이언트가 명시적으로 제공해야 합니다. clickhouse-client를 사용할 때는 `--quota_key` 매개변수로 키 값을 전달하거나 클라이언트 설정 파일에서 `quota_key` 매개변수를 사용하십시오. HTTP 인터페이스를 사용할 때는 X-ClickHouse-Quota header를 사용합니다, [&#39;user&#95;name&#39;, &#39;client&#95;key&#39;] — 동일한 client&#95;key를 사용하는 연결이 동일한 QUOTA를 공유합니다. 클라이언트가 키를 제공하지 않으면 QUOTA는 `user_name` 기준으로 추적됩니다, [&#39;client&#95;key&#39;, &#39;ip&#95;address&#39;] — 동일한 client&#95;key를 사용하는 연결이 동일한 QUOTA를 공유합니다. 클라이언트가 키를 제공하지 않으면 QUOTA는 ip&#95;address 기준으로 추적됩니다.
* `durations` ([Array(UInt32)](../../sql-reference/data-types/)) — 초 단위 시간 인터벌 길이입니다.
* `apply_to_all` ([UInt8](../../sql-reference/data-types/)) — 논리값입니다. QUOTA가 어떤 사용자에게 적용되는지 나타냅니다. 값: 0 — QUOTA는 apply&#95;to&#95;list에 지정된 사용자에게 적용됩니다. 1 — QUOTA는 apply&#95;to&#95;except에 나열된 사용자를 제외한 모든 사용자에게 적용됩니다.
* `apply_to_list` ([Array(String)](../../sql-reference/data-types/)) — QUOTA를 적용할 사용자 이름/역할 목록입니다.
* `apply_to_except` ([Array(String)](../../sql-reference/data-types/)) — QUOTA를 적용하지 않을 사용자 이름/역할 목록입니다.
