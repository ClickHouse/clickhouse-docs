# system.processes \{#systemprocesses\}

<SystemTableCloud />

이 시스템 테이블은 `SHOW PROCESSLIST` 쿼리를 구현하는 데 사용됩니다.

컬럼:

{/*AUTOGENERATED_START*/ }

* `is_initial_query` ([UInt8](../../sql-reference/data-types/)) — 이 쿼리가 사용자로부터 직접 전송된 것인지, 아니면 분산 쿼리 실행의 일환으로 ClickHouse 서버에 의해 실행된 것인지를 나타냅니다.
* `user` ([String](../../sql-reference/data-types/)) — 쿼리를 실행한 사용자입니다. 분산 처리를 사용하는 경우 쿼리는 기본 사용자로 원격 서버에 전송된다는 점에 유의하십시오. 이 필드에는 특정 쿼리 자체에 대한 사용자 이름이 포함되며, 이 쿼리가 실행을 시작한 다른 쿼리에 대한 사용자 이름은 포함되지 않습니다.
* `query_id` ([String](../../sql-reference/data-types/)) — 정의되어 있는 경우 쿼리 ID입니다.
* `address` ([IPv6](../../sql-reference/data-types/)) — 쿼리가 실행된 IP 주소입니다. 분산 처리인 경우에도 동일합니다. 분산 쿼리가 처음 어디에서 실행되었는지 추적하려면, 쿼리를 보낸 서버의 system.processes를 확인하십시오.
* `port` ([UInt16](../../sql-reference/data-types/)) — 쿼리를 실행한 클라이언트의 포트입니다.
* `initial_user` ([String](../../sql-reference/data-types/)) — 최초 쿼리를 실행한 사용자의 이름(분산 쿼리 실행 시 사용).
* `initial_query_id` ([String](../../sql-reference/data-types/)) — 초기 쿼리의 ID(분산 쿼리 실행에서 사용됨).
* `initial_address` ([IPv6](../../sql-reference/data-types/)) — 상위 쿼리가 실행된 IP 주소입니다.
* `initial_port` ([UInt16](../../sql-reference/data-types/)) — 부모 쿼리를 실행하는 데 사용된 클라이언트 포트입니다.
* `interface` ([UInt8](../../sql-reference/data-types/)) — 쿼리를 전송하는 데 사용된 인터페이스입니다. TCP = 1, HTTP = 2, GRPC = 3, MYSQL = 4, POSTGRESQL = 5, LOCAL = 6, TCP&#95;INTERSERVER = 7.
* `os_user` ([String](../../sql-reference/data-types/)) — clickhouse-client를 실행하는 운영 체제의 사용자 이름.
* `client_hostname` ([String](../../sql-reference/data-types/)) — clickhouse-client 또는 다른 TCP 클라이언트가 실행 중인 클라이언트 머신의 호스트명입니다.
* `client_name` ([String](../../sql-reference/data-types/)) — clickhouse-client 또는 다른 TCP 클라이언트의 이름입니다.
* `client_revision` ([UInt64](../../sql-reference/data-types/)) — clickhouse-client 또는 기타 TCP 클라이언트의 리비전입니다.
* `client_version_major` ([UInt64](../../sql-reference/data-types/)) — clickhouse-client 또는 다른 TCP 클라이언트의 주 버전입니다.
* `client_version_minor` ([UInt64](../../sql-reference/data-types/)) — clickhouse-client 또는 다른 TCP 클라이언트의 마이너 버전 번호입니다.
* `client_version_patch` ([UInt64](../../sql-reference/data-types/)) — clickhouse-client 또는 다른 TCP 클라이언트 버전의 패치 버전 구성 요소입니다.
* `http_method` ([UInt8](../../sql-reference/data-types/)) — 쿼리를 발생시킨 HTTP 메서드입니다. 가능한 값은 다음과 같습니다: 0 — 쿼리가 TCP 인터페이스에서 실행되었습니다. 1 — GET 메서드가 사용되었습니다. 2 — POST 메서드가 사용되었습니다.
* `http_user_agent` ([String](../../sql-reference/data-types/)) — HTTP 쿼리에서 전달되는 HTTP UserAgent 헤더입니다.
* `http_referer` ([String](../../sql-reference/data-types/)) — HTTP 쿼리로 전달된 Referer HTTP 헤더(쿼리를 발생시킨 페이지의 절대 또는 부분 주소를 포함함).
* `forwarded_for` ([String](../../sql-reference/data-types/)) — HTTP 쿼리와 함께 전송된 HTTP 헤더 X-Forwarded-For입니다.
* `quota_key` ([String](../../sql-reference/data-types/)) — QUOTA 설정에서 지정한 quota 키입니다(자세한 내용은 keyed 항목을 참조하십시오).
* `distributed_depth` ([UInt64](../../sql-reference/data-types/)) — 쿼리가 서버 노드 간에 내부적으로 다시 전송된 횟수입니다.
* `elapsed` ([Float64](../../sql-reference/data-types/)) — 요청 실행이 시작된 후 경과한 시간(초)입니다.
* `is_cancelled` ([UInt8](../../sql-reference/data-types/)) — 쿼리 취소 여부입니다.
* `is_all_data_sent` ([UInt8](../../sql-reference/data-types/)) — 모든 데이터가 클라이언트로 전송되었는지 여부입니다(즉, 서버에서 쿼리가 완료되었는지 여부입니다).
* `read_rows` ([UInt64](../../sql-reference/data-types/)) — 테이블에서 읽은 행의 개수입니다. 분산 처리인 경우 요청 서버에서는 모든 원격 서버에서 읽은 행 수의 합계입니다.
* `read_bytes` ([UInt64](../../sql-reference/data-types/)) — 테이블에서 읽은 압축되지 않은 바이트 수입니다. 분산 처리의 경우, 요청 서버에서는 모든 원격 서버의 값을 합산한 총합입니다.
* `total_rows_approx` ([UInt64](../../sql-reference/data-types/)) — 읽어야 할 총 행 수의 근사값입니다. 분산 처리의 경우, 요청을 수행하는 서버에서는 모든 원격 서버를 합한 전체 값입니다. 처리해야 할 새로운 소스가 알려지면, 요청 처리 중에 이 값이 갱신될 수 있습니다.
* `written_rows` ([UInt64](../../sql-reference/data-types/)) — 스토리지에 기록된 행의 수입니다.
* `written_bytes` ([UInt64](../../sql-reference/data-types/)) — 스토리지에 기록된 바이트 수입니다.
* `memory_usage` ([Int64](../../sql-reference/data-types/)) — 쿼리가 사용하는 RAM의 양입니다. 일부 유형의 전용 메모리는 포함되지 않을 수 있습니다.
* `peak_memory_usage` ([Int64](../../sql-reference/data-types/)) — 현재까지 기록된 최대 메모리 사용량입니다.
* `query` ([String](../../sql-reference/data-types/)) — 쿼리 텍스트입니다. INSERT문의 경우 삽입할 데이터는 포함하지 않습니다.
* `normalized_query_hash` ([UInt64](../../sql-reference/data-types/)) — 리터럴 값만 서로 다른 쿼리들에 대해서는 동일한 값을 갖는 숫자 해시입니다.
* `query_kind` ([String](../../sql-reference/data-types/)) — 쿼리의 유형입니다(SELECT, INSERT 등).
* `thread_ids` ([Array(UInt64)](../../sql-reference/data-types/)) — 이 쿼리에 참여한 모든 스레드의 식별자 목록입니다.
* `peak_threads_usage` ([UInt64](../../sql-reference/data-types/)) — 해당 쿼리를 동시에 실행하는 스레드의 최대 개수입니다.
* `ProfileEvents` ([Map(LowCardinality(String), UInt64)](../../sql-reference/data-types/)) — 이 쿼리에 대해 집계된 ProfileEvents입니다.
* `Settings` ([Map(LowCardinality(String), LowCardinality(String))](../../sql-reference/data-types/)) — 사용자 수준에서 변경된 설정의 목록입니다.
* `current_database` ([String](../../sql-reference/data-types/)) — 현재 데이터베이스 이름입니다.
* `is_internal` ([UInt8](../../sql-reference/data-types/)) — 내부적으로 실행되는 보조 쿼리인지 여부를 나타냅니다.

{/*AUTOGENERATED_END*/ }
