## 설명 \{#description\}

서버에서 지원하는 딕셔너리 소스 목록과 각 소스에 대한 내장 문서를 포함합니다. 딕셔너리 소스는 딕셔너리 데이터를 어디에서 불러올지 결정하며, `CREATE DICTIONARY` 쿼리의 `SOURCE` 절에 지정됩니다.

## 컬럼 \{#columns\}

{/*AUTOGENERATED_START*/ }

* `name` ([String](../../sql-reference/data-types/)) — SOURCE 절에 지정된 딕셔너리 소스의 이름입니다.
* `description` ([String](../../sql-reference/data-types/)) — 딕셔너리 소스가 수행하는 역할에 대한 개괄적인 설명입니다.
* `syntax` ([String](../../sql-reference/data-types/)) — 소스를 지정하는 데 사용하는 SOURCE 절의 구조입니다. 일부 소스는 딕셔너리를 DDL 쿼리로 생성할 때(서버 설정 파일에서 생성하는 경우와 달리) 액세스 제어(access control)가 적용될 수 있습니다. 자세한 내용은 각 소스의 `description`을 참조하십시오.
* `examples` ([String](../../sql-reference/data-types/)) — 사용 예시입니다.
* `introduced_in` ([String](../../sql-reference/data-types/)) — 소스가 처음 도입된 ClickHouse 버전으로, 형식은 major.minor입니다.
* `related` ([Array(String)](../../sql-reference/data-types/)) — 관련된 딕셔너리 소스의 이름입니다.

{/*AUTOGENERATED_END*/ }

## 예시 \{#example\}

```sql title="Query"
SELECT name, syntax
FROM system.dictionary_sources
WHERE name IN ('clickhouse', 'file')
ORDER BY name
```

```text title="Response"
┌─name───────┬─syntax─────────────────────────────────────────────────────────────────────────┐
│ clickhouse │ SOURCE(CLICKHOUSE(host 'host' port 9000 user 'default' password '' db 'db' table 'table')) │
│ file       │ SOURCE(FILE(path '/path/to/file' format 'CSV'))                                  │
└────────────┴────────────────────────────────────────────────────────────────────────────────┘
```
