하위 태그로 다음 설정을 구성할 수 있습니다:

| Setting                            | Description                                                                                                                                             | Default             | Note                                                                                                               |
|------------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------|---------------------|--------------------------------------------------------------------------------------------------------------------|
| `database`                         | 데이터베이스 이름입니다.                                                                                                                                |                     |                                                                                                                    |
| `table`                            | 시스템 테이블 이름입니다.                                                                                                                               |                     |                                                                                                                    |
| `engine`                           | 시스템 테이블에 대한 [MergeTree 엔진 정의](../../../engines/table-engines/mergetree-family/mergetree.md#table_engine-mergetree-creating-a-table)입니다. |                     | `partition_by` 또는 `order_by`가 정의된 경우 사용할 수 없습니다. 지정하지 않으면 기본적으로 `MergeTree`가 선택됩니다. |
| `partition_by`                     | 시스템 테이블에 대한 [사용자 정의 파티셔닝 키(Custom partitioning key)](../../../engines/table-engines/mergetree-family/custom-partitioning-key.md)입니다. |                     | 시스템 테이블에 `engine`을 지정하는 경우, `partition_by` 파라미터는 `engine` 설정 내부에 직접 지정해야 합니다.   |
| `ttl`                              | 테이블 [TTL](/engines/table-engines/mergetree-family/mergetree#table_engine-mergetree-ttl)을 지정합니다.                                              |                     | 시스템 테이블에 `engine`을 지정하는 경우, `ttl` 파라미터는 `engine` 설정 내부에 직접 지정해야 합니다.            |
| `order_by`                         | 시스템 테이블에 대한 [사용자 정의 정렬 키(Custom sorting key)](../../../engines/table-engines/mergetree-family/mergetree.md#order_by)입니다. `engine`이 정의된 경우 사용할 수 없습니다. |                     | 시스템 테이블에 `engine`을 지정하는 경우, `order_by` 파라미터는 `engine` 설정 내부에 직접 지정해야 합니다.      |
| `storage_policy`                   | 테이블에 사용할 스토리지 정책 이름입니다(선택 사항).                                                                                                   |                     | 시스템 테이블에 `engine`을 지정하는 경우, `storage_policy` 파라미터는 `engine` 설정 내부에 직접 지정해야 합니다. |
| `settings`                         | MergeTree 동작을 제어하는 [추가 파라미터](../../../engines/table-engines/mergetree-family/mergetree.md/#settings)입니다(선택 사항).                     |                     | 시스템 테이블에 `engine`을 지정하는 경우, `settings` 파라미터는 `engine` 설정 내부에 직접 지정해야 합니다.       |
| `flush_interval_milliseconds`      | 메모리의 버퍼에서 테이블로 데이터를 플러시하는 간격입니다.                                                                                             | `7500`              |                                                                                                                    |
| `max_size_rows`                    | 로그의 최대 크기(행 수)입니다. 플러시되지 않은 로그의 양이 `max_size_rows`에 도달하면 로그가 디스크로 덤프됩니다.                                    | `1048576`           |                                                                                                                    |
| `reserved_size_rows`               | 로그를 위해 미리 할당해 두는 메모리 크기(행 수)입니다.                                                                                                 | `8192`              |                                                                                                                    |
| `buffer_size_rows_flush_threshold` | 행 수에 대한 임계값입니다. 이 임계값에 도달하면 백그라운드에서 로그를 디스크로 플러시하는 작업이 시작됩니다.                                         | `max_size_rows / 2` |                                                                                                                    |
| `flush_on_crash`                   | 크래시 발생 시 로그를 디스크로 덤프할지 여부를 설정합니다.                                                                                             | `false`             |                                                                                                                    |
