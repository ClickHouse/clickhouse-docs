由于 ClickHouse 是开源的，它不仅接受来自 ClickHouse 员工的贡献，也接受来自社区的贡献。这些贡献的开发节奏往往不同；某些特性可能需要较长的原型设计阶段，或需要更多时间来收集足够的社区反馈并进行迭代，然后才能被视为正式可用（GA，Generally Available）。

由于特性何时能被归类为正式可用存在不确定性，我们将特性划分为两个类别：**Beta** 和 **Experimental**。

**Beta** 特性由 ClickHouse 团队正式提供支持。**Experimental** 特性是由 ClickHouse 团队或社区推动的早期原型，不属于正式支持范围。

下文各部分将详细说明 **Beta** 和 **Experimental** 特性的特征：



## Beta 功能 {#beta-features}

- 正在积极开发中,以实现正式发布 (GA)
- 主要已知问题可在 GitHub 上追踪
- 功能可能在未来发生变化
- 可能在 ClickHouse Cloud 中启用
- ClickHouse 团队为 Beta 功能提供支持

以下是 ClickHouse Cloud 中处于 Beta 阶段的功能,这些功能可在您的 ClickHouse Cloud 服务中使用。

注意:请确保使用最新版本的 ClickHouse [兼容性](/operations/settings/settings#compatibility)设置,以使用最近引入的功能。


## 实验性功能 {#experimental-features}

- 可能永远不会正式发布（GA）
- 可能会被移除
- 可能引入破坏性变更
- 功能可能会发生变化
- 需要显式启用
- ClickHouse 团队**不提供支持**
- 可能缺少重要功能和文档
- 无法在 ClickHouse Cloud 中启用

请注意：除上述列为 Beta 的功能外，ClickHouse Cloud 不允许启用任何其他实验性功能。
