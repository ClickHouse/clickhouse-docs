由于 ClickHouse 是开源软件，它不仅包含来自 ClickHouse 员工的贡献，也有大量社区贡献。这些贡献的开发进度往往不同；某些功能可能需要较长的原型设计阶段，或需要更多时间来获取充分的社区反馈并进行迭代，之后才会被视为“正式可用”（GA，Generally Available）。

由于功能达到“正式可用”状态的时间存在不确定性，我们将功能划分为两个类别：**测试版（Beta）**和**实验性（Experimental）**。

**测试版（Beta）**功能由 ClickHouse 团队正式支持。**实验性（Experimental）**功能是由 ClickHouse 团队或社区推动的早期原型，这些功能不属于正式支持范围。

下面各小节将明确说明 **测试版（Beta）** 和 **实验性（Experimental）** 功能的属性：

## Beta 功能 {#beta-features}

- 正在积极开发中，目标是达到正式发布（GA）
- 主要已知问题可在 GitHub 上跟踪
- 功能行为将来可能发生变化
- 可能在 ClickHouse Cloud 中启用
- ClickHouse 团队为 Beta 功能提供支持

以下是当前在 ClickHouse Cloud 中被视为 Beta，且可在你的 ClickHouse Cloud 服务中使用的功能。

注意：请确保你正在使用最新版本的 ClickHouse [compatibility](/operations/settings/settings#compatibility) 设置，以便使用最近引入的功能。

## 实验性特性 {#experimental-features}

* 可能永远不会进入 GA 阶段
* 可能会被移除
* 可能引入破坏性变更
* 该特性的功能可能随时发生变化
* 需要显式启用
* ClickHouse 团队**不为**实验性特性提供支持
* 可能缺少重要功能和文档
* 无法在云端启用

请注意：除上述列为 Beta 的特性外，不允许在 ClickHouse Cloud 中启用任何其他实验性特性。
