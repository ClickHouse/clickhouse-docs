Elasticsearch 和 ClickHouse 支持种类繁多的数据类型，但它们底层的存储和查询模型有本质差异。本节将常用的 Elasticsearch 字段类型映射到在 ClickHouse 中的对应类型（如有），并提供相关背景说明以帮助迁移决策。对于不存在对应类型的情况，将在注释中给出替代方案或补充说明。



| **Elasticsearch 类型**               | **ClickHouse 等效类型**                                                                                                                                                                                                                                                                          | **备注**                                                                                                                                                                                                                                                       |
| ---------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `boolean`                          | [`UInt8`](/sql-reference/data-types/int-uint) 或 [`Bool`](/sql-reference/data-types/boolean)                                                                                                                                                                                                  | 在较新版本中，ClickHouse 支持使用 `Boolean` 作为 `UInt8` 的别名。                                                                                                                                                                                                             |
| `keyword`                          | [`String`](/sql-reference/data-types/string)                                                                                                                                                                                                                                                 | 用于精确匹配的过滤、分组和排序。                                                                                                                                                                                                                                             |
| `text`                             | [`String`](/sql-reference/data-types/string)                                                                                                                                                                                                                                                 | ClickHouse 对全文搜索的支持有限；分词需要通过自定义逻辑实现，例如将 `tokens` 函数与数组相关函数组合使用。                                                                                                                                                                                              |
| `long`                             | [`Int64`](/sql-reference/data-types/int-uint)                                                                                                                                                                                                                                                | 64 位有符号整数。                                                                                                                                                                                                                                                   |
| `integer`                          | [`Int32`](/sql-reference/data-types/int-uint)                                                                                                                                                                                                                                                | 32 位有符号整数。                                                                                                                                                                                                                                                   |
| `短`                                | [`Int16`](/sql-reference/data-types/int-uint)                                                                                                                                                                                                                                                | 16 位有符号整数。                                                                                                                                                                                                                                                   |
| `byte`                             | [`Int8`](/sql-reference/data-types/int-uint)                                                                                                                                                                                                                                                 | 8 位有符号整数。                                                                                                                                                                                                                                                    |
| `unsigned_long`                    | [`UInt64`](/sql-reference/data-types/int-uint)                                                                                                                                                                                                                                               | 无符号 64 位整数。                                                                                                                                                                                                                                                  |
| `double`                           | [`Float64`](/sql-reference/data-types/float)                                                                                                                                                                                                                                                 | 64位浮点数。                                                                                                                                                                                                                                                      |
| `float`                            | [`Float32`](/sql-reference/data-types/float)                                                                                                                                                                                                                                                 | 32位浮点数。                                                                                                                                                                                                                                                      |
| `half_float`                       | [`Float32`](/sql-reference/data-types/float) 或 [`BFloat16`](/sql-reference/data-types/float)                                                                                                                                                                                                 | 最接近的等价类型。ClickHouse 不支持 16 位浮点数，但提供 `BFloat16`——它与 IEEE-754 半精度浮点数不同：半精度浮点数具有更高精度但数值范围较小，而 BFloat16 为了更大的数值范围牺牲了精度，因此更适合机器学习工作负载。                                                                                                                            |
| `scaled_float`                     | [`Decimal(x, y)`](/sql-reference/data-types/decimal)                                                                                                                                                                                                                                         | 存储定点数值。                                                                                                                                                                                                                                                      |
| `date`                             | [`DateTime`](/sql-reference/data-types/datetime)                                                                                                                                                                                                                                             | 具有秒级精度的等效日期类型。                                                                                                                                                                                                                                               |
| `date_nanos`                       | [`DateTime64`](/sql-reference/data-types/datetime64)                                                                                                                                                                                                                                         | ClickHouse 通过 `DateTime64(9)` 支持纳秒级精度。                                                                                                                                                                                                                       |
| `binary`                           | [`String`](/sql-reference/data-types/string), [`FixedString(N)`](/sql-reference/data-types/fixedstring)                                                                                                                                                                                      | 需要对二进制字段进行 Base64 解码。                                                                                                                                                                                                                                        |
| `ip`                               | [`IPv4`](/sql-reference/data-types/ipv4), [`IPv6`](/sql-reference/data-types/ipv6)                                                                                                                                                                                                           | 提供原生 `IPv4` 和 `IPv6` 类型。                                                                                                                                                                                                                                     |
| `object`                           | [`Nested`](/sql-reference/data-types/nested-data-structures/nested), [`Map`](/sql-reference/data-types/map), [`Tuple`](/sql-reference/data-types/tuple), [`JSON`](/sql-reference/data-types/newjson)                                                                                         | ClickHouse 可以使用 [`Nested`](/sql-reference/data-types/nested-data-structures/nested) 或 [`JSON`](/sql-reference/data-types/newjson) 来建模类似 JSON 的对象。                                                                                                            |
| `flattened`                        | [`String`](/sql-reference/data-types/string)                                                                                                                                                                                                                                                 | Elasticsearch 中的 `flattened` 类型会将整个 JSON 对象存储为单个字段，从而在无需完整映射定义的情况下，为嵌套键提供灵活的、无模式的访问。在 ClickHouse 中，可以使用 `String` 类型实现类似的功能，但需要在物化视图中完成相应处理。                                                                                                                  |
| `nested`                           | [`Nested（嵌套）`](/sql-reference/data-types/nested-data-structures/nested)                                                                                                                                                                                                                      | 在用户设置 `flatten_nested=0` 时，ClickHouse 的 `Nested` 列对成组的子字段提供了类似的语义。                                                                                                                                                                                           |

| `alias`                            | [`Alias`](/sql-reference/statements/create/table#alias) 列修饰符                                                                                                                                                                                                                                 | 可以通过字段修饰符来支持[别名](/sql-reference/statements/create/table#alias)。可以将函数应用于这些别名，例如 `size String ALIAS formatReadableSize(size_bytes)`                                                                                                                            |
| `range` 类型（`*_range`）              | [`Tuple(start, end)`](/sql-reference/data-types/tuple) 或 [`Array(T)`](/sql-reference/data-types/array)                                                                                                                                                                                       | ClickHouse 没有原生的区间类型，但数值和日期区间可以使用 [`Tuple(start, end)`](/sql-reference/data-types/tuple) 或 [`Array`](/sql-reference/data-types/array) 结构来表示。对于 IP 区间（`ip_range`），将 CIDR 值存为 `String`，并使用如 `isIPAddressInRange()` 之类的函数进行判断。或者，可以考虑基于 `ip_trie` 的查找字典以实现高效过滤。 |
| `aggregate_metric_double`          | [`AggregateFunction(...)`](/sql-reference/data-types/aggregatefunction) 和 [`SimpleAggregateFunction(...)`](/sql-reference/data-types/simpleaggregatefunction)                                                                                                                                | 使用聚合函数的状态和物化视图来建模预聚合度量。所有聚合函数都支持聚合函数状态。                                                                                                                                                                                                                      |
| `直方图`                              | [`Tuple(Array(Float64), Array(UInt64))`](/sql-reference/data-types/tuple)                                                                                                                                                                                                                    | 通过数组或自定义模式手动表示各桶及其计数。                                                                                                                                                                                                                                        |
| `annotated-text`                   | [`String`](/sql-reference/data-types/string)                                                                                                                                                                                                                                                 | 不原生支持实体感知搜索或标注。                                                                                                                                                                                                                                              |
| `completion`, `search_as_you_type` | 不适用                                                                                                                                                                                                                                                                                          | 没有原生的自动补全或提示引擎。可以通过 `String` 类型和[搜索函数](/sql-reference/functions/string-search-functions)实现类似功能。                                                                                                                                                              |
| `semantic_text`                    | 不适用                                                                                                                                                                                                                                                                                          | 没有原生语义搜索功能——需要生成嵌入向量并使用向量搜索。                                                                                                                                                                                                                                 |
| `token_count`                      | [`Int32`](/sql-reference/data-types/int-uint)                                                                                                                                                                                                                                                | 在摄取期间用于手动计算 token 数量，例如使用 `length(tokens())` 函数，可配合 Materialized 列使用                                                                                                                                                                                         |
| `dense_vector`                     | [`Array(Float32)`](/sql-reference/data-types/array)                                                                                                                                                                                                                                          | 使用数组存储向量嵌入                                                                                                                                                                                                                                                   |
| `sparse_vector`                    | [`Map(UInt32, Float32)`](/sql-reference/data-types/map)                                                                                                                                                                                                                                      | 使用 Map 模拟稀疏向量。不支持原生稀疏向量。                                                                                                                                                                                                                                     |
| `rank_feature` / `rank_features`   | [`Float32`](/sql-reference/data-types/float), [`Array(Float32)`](/sql-reference/data-types/array)                                                                                                                                                                                            | 不支持原生的查询时加权功能，但可以在评分逻辑中通过手动建模来实现。                                                                                                                                                                                                                            |
| `geo_point`                        | [`Tuple(Float64, Float64)`](/sql-reference/data-types/tuple) 或 [`Point`](/sql-reference/data-types/geo#point)                                                                                                                                                                                | 使用 (纬度，经度) 的元组。[`Point`](/sql-reference/data-types/geo#point) 是 ClickHouse 中可用的数据类型。                                                                                                                                                                         |
| `geo_shape`, `shape`               | [`Ring`](/sql-reference/data-types/geo#ring), [`LineString`](/sql-reference/data-types/geo#linestring), [`MultiLineString`](/sql-reference/data-types/geo#multilinestring), [`Polygon`](/sql-reference/data-types/geo#polygon), [`MultiPolygon`](/sql-reference/data-types/geo#multipolygon) | 原生支持地理形状和空间索引。                                                                                                                                                                                                                                               |
| `percolator`                       | 不适用                                                                                                                                                                                                                                                                                          | 没有对查询建立索引这一概念。请改用标准 SQL 和增量物化视图。                                                                                                                                                                                                                             |
| `version`                          | [`String`](/sql-reference/data-types/string)                                                                                                                                                                                                                                                 | ClickHouse 没有原生的版本类型。请将版本存储为字符串，并在需要时使用自定义 UDF 进行语义比较。如果需要执行范围查询，建议将其规范化为数值格式。                                                                                                                                                                               |



### 说明 {#notes}

- **数组（Arrays）**：在 Elasticsearch 中，所有字段原生支持数组。在 ClickHouse 中，数组必须显式定义（例如 `Array(String)`），其优势是可以按具体位置访问和查询，例如 `an_array[1]`。
- **多字段（Multi-fields）**：Elasticsearch 允许以[多种方式为同一字段建立索引](https://www.elastic.co/docs/reference/elasticsearch/mapping-reference/multi-fields#_multi_fields_with_multiple_analyzers)（例如同时作为 `text` 和 `keyword`）。在 ClickHouse 中，这种模式必须通过使用单独的列或视图来建模。
- **Map 和 JSON 类型** —— 在 ClickHouse 中，[`Map`](/sql-reference/data-types/map) 类型通常用于建模诸如 `resourceAttributes` 和 `logAttributes` 之类的动态键值结构。该类型通过允许在运行时添加任意键，实现灵活的无模式摄取，在概念上类似于 Elasticsearch 中的 JSON 对象。但需要注意以下重要限制：

  - **统一的值类型**：ClickHouse [`Map`](/sql-reference/data-types/map) 列必须具有一致的值类型（例如 `Map(String, String)`）。在不进行类型强制转换的情况下，不支持混合类型的值。
  - **性能开销**：访问 [`Map`](/sql-reference/data-types/map) 中任意键都需要将整个 map 加载到内存中，这在性能上可能不是最优。
  - **没有子列（subcolumns）**：与 JSON 不同，[`Map`](/sql-reference/data-types/map) 中的键不会表示为真正的子列，这限制了 ClickHouse 在索引、压缩和高效查询方面的能力。

  由于这些限制，ClickStack 正从 [`Map`](/sql-reference/data-types/map) 类型迁移，转而使用 ClickHouse 增强的 [`JSON`](/sql-reference/data-types/newjson) 类型。[`JSON`](/sql-reference/data-types/newjson) 类型解决了 `Map` 的许多不足：

  - **真正的列式存储**：每个 JSON 路径都存储为一个子列，从而实现高效的压缩、过滤和向量化查询执行。
  - **混合类型支持**：不同的数据类型（例如整数、字符串、数组）可以在同一路径下共存，无需进行强制转换或类型统一。
  - **文件系统可扩展性**：对动态键（`max_dynamic_paths`）和类型（`max_dynamic_types`）的内部限制，可防止在高基数键集合下磁盘上列文件数量爆炸式增长。
  - **紧凑存储**：null 和缺失值以稀疏方式存储，以避免不必要的开销。

    [`JSON`](/sql-reference/data-types/newjson) 类型特别适合可观测性工作负载，在保留无模式摄取灵活性的同时，提供了原生 ClickHouse 类型的性能和可扩展性——使其成为在动态属性字段中替代 [`Map`](/sql-reference/data-types/map) 的理想选择。

    有关 JSON 类型的更多详细信息，建议参阅 [JSON 指南](https://clickhouse.com/docs/integrations/data-formats/json/overview) 和 ["How we built a new powerful JSON data type for ClickHouse"](https://clickhouse.com/blog/a-new-powerful-json-data-type-for-clickhouse)。
