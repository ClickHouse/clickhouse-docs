[Model Context Protocol](https://www.anthropic.com/news/model-context-protocol)（MCP）是由 Anthropic 开发的一项标准，使 AI 助手能够与外部系统无缝集成。
该协议允许 AI 助手以安全且标准化的方式连接到数据源、API、数据库等。

MCP 在 AI 模型与各类服务之间创建了一个通用接口，免去了为每次集成单独实现自定义工具的需求。
你可以将它视为专为 AI 系统设计的通用 API 标准。

MCP 的关键优势在于，AI 库只需实现一次对该协议的支持。
之后，所有兼容 MCP 的服务即可立即使用，为 AI 库维护者节省大量时间。


## MCP 的架构是什么？ {#mcp-architecture}

MCP 遵循客户端-服务器架构：

- 客户端（如 Claude Desktop、Cursor 或 VS Code）与 MCP 服务器建立连接。您可以在 [awesome-mcp-clients](https://github.com/punkpeye/awesome-mcp-clients?tab=readme-ov-file#windsurf) GitHub 仓库中查看客户端集合。
- 服务器通过标准化接口暴露工具和能力。您可以在 [awesome-mcp-servers](https://github.com/punkpeye/awesome-mcp-servers) GitHub 仓库中查看服务器集合。
- AI 模型可以在需要时使用这些工具访问外部数据和功能

## ClickHouse 是否有 MCP 服务器? {#clickhouse-mcp-server}

有!
[ClickHouse MCP Server](https://github.com/ClickHouse/mcp-clickhouse) 提供以下工具:

- `run_select_query` - 在 ClickHouse 集群上执行 SQL 查询。
- `list_databases` - 列出 ClickHouse 集群上的所有数据库。
- `list_tables` - 列出数据库中的所有表。
