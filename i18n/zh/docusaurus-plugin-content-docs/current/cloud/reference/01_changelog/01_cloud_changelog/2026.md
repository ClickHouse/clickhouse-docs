## 2026 年 6 月 3 日 \{#2026-06-03\}

Open House 2026 活动周带来了一系列发布，重点围绕让 ClickHouse 在数据所在之处原生运行，并将智能体驱动、AI 辅助的工作流引入平台。

### ClickPipes 现已在 GCP 上原生运行，并支持 Private Service Connect \{#clickpipes-gcp-psc\}

对于 5 月 26 日之后创建的所有新 GCP 服务，ClickPipes 基础设施现已与您的 ClickHouse Cloud 服务运行在同一 GCP 区域，并支持 Private Service Connect (PSC)。这样可免除跨云出口成本、降低延迟、确保数据留在本地，并让您能够通过 GCP 原生私有网络连接到 GCP 数据源，而无需再使用 SSH 隧道等权宜之计。现有服务也可在我们团队的协助下迁移。

更多详情，请参阅 [ClickPipes static IP list](/integrations/clickpipes#list-of-static-ips)。

### 用于 GCP Pub/Sub 的 ClickPipes (私有预览) \{#clickpipes-gcp-pubsub\}

ClickPipes 现在可将 GCP Pub/Sub topic 中的数据直接流式传输到 ClickHouse Cloud，让以 GCP 为原生平台的团队能够直接分析来自 Cloud Logging、Cloud Monitoring、GKE 等的数据。可通过 ClickPipes UI、OpenAPI 和 Terraform 使用；在私有预览期间需要先[注册](https://clickhouse.com/cloud/clickpipes#pubsub-private-preview)。

有关设置和配置的详细信息，请参阅 [用于 GCP Pub/Sub 的 ClickPipes 文档](/integrations/clickpipes/pubsub)。

### ClickHouse 智能体：基于 Claude 的智能体分析 (Public Beta) \{#clickhouse-agents\}

ClickHouse 智能体现已在 [ai.clickhouse.cloud](http://ai.clickhouse.cloud) 开放 Public Beta——这是一个由 Claude 驱动的官方智能体平台，可直接基于你的 ClickHouse Cloud 数据进行对话式、具备 schema 感知能力的分析。你可以使用无代码的 AgentBuilder 构建自定义智能体，接入 MCP servers (Slack、Linear、Notion 等) ，编排子智能体工作流，并且无需离开当前对话即可生成查询、管道和应用。还包括 AWS Bedrock AgentCore 集成以及托管的 LibreChat 基础能力。

在 [ClickHouse Agents 文档](/cloud/features/ai-ml/agents) 中了解更多。

### ClickHouse Postgres (Public Beta) \{#clickhouse-postgres-public-beta\}

基于本地 NVMe 存储的全托管 Postgres 现已进入 Public Beta，TPS 比 AWS RDS 高出 5 倍以上，并在同一套托管技术栈中打通了 OLTP 与 OLAP。原生 CDC 管道可将数据直接流式传输到 ClickHouse，无需任何中间基础设施；开源 `pg_clickhouse` 扩展则允许你通过标准 Postgres 会话查询 ClickHouse 表。

更多信息，请参阅 [Open House 2026 Day 1: Real-time data without lock-in and what teams can build next](https://clickhouse.com/blog/open-house-2026-day-1#postgres-announcements)。

### ClickStack Cloud：无服务器可观测性 (私有预览) \{#clickstack-cloud\}

ClickStack Cloud 是一个由 ClickHouse Cloud 提供支持的全托管无服务器可观测性平台。将 OpenTelemetry Collector 指向托管的 OTLP 端点，即可立即在 ClickStack UI 中查看日志、指标和链路追踪——摄取、缓冲、扩缩容、存储、查询处理、身份验证和 RBAC 全部由平台代为完成。现已在 AWS us-east-1 面向有限用户开放私有预览。

更多详情，请参阅 [ClickStack Cloud 私有预览](https://clickhouse.com/blog/clickstack-cloud-private-preview) 和 [ClickStack Cloud 等候名单](https://clickhouse.com/cloud/clickstack-cloud-waitlist)。

### 面向托管 ClickStack 的 AI Notebooks (Beta) \{#clickstack-ai-notebooks\}

AI Notebooks 为托管 ClickStack 提供了一个持久、透明的工作区，用于开展 AI 辅助的事件调查。您可以记录由提示词、查询、图表和推理组成的结构化序列，通过分支探索多种假设，并将调查作为团队协作制品进行共享——且所有生成的查询与结果都清晰可见并可编辑。可从 ClickStack UI 的左侧导航中访问。

更多详情，请参阅：

* [ClickStack AI Notebooks 文档](/use-cases/observability/clickstack/notebooks)
* [处于 Beta 阶段的 AI Notebooks (博客) ](https://clickhouse.com/blog/observability-mcp-server-ai-notebooks#ai-notebooks-in-beta)

### 水平自动扩缩容 (私有预览) \{#horizontal-autoscaling\}

水平自动扩缩容会自动调整 ClickHouse Cloud 集群中的副本数量：当需求激增时快速横向扩展，在负载回落时再逐步缩减——这样一来，面对负载波动大且高并发的团队，就不再需要手动将集群调整到合适规模。私有预览阶段根据并发查询数进行扩缩容；计划在 Public Beta 中加入 CPU 和内存信号。适用于使用 ClickHouse 26.2+ 的 Enterprise 和 Scale 层级客户。

### ClickStack MCP Server (开源) \{#clickstack-mcp-server\}

全新的开源 ClickStack MCP Server 为外部 AI 智能体提供了基于 ClickStack 和 ClickHouse 构建的结构化可观测性调查工具。它提供适用于日志、指标和链路追踪的语义基本类型，支持日志模式分析、trace 异常调查、关联分析和跨信号调试；还可持久化保存仪表盘、搜索和告警，同时保留原生 SQL 访问能力，以便进行更深入的探索。可连接 Claude Code、Cursor、Codex 或自定义框架等智能体。

更多详情请参阅 [ClickStack MCP Server 文档](/use-cases/observability/clickstack/mcp) 和 [可观测性 MCP Server + AI Notebooks 博文](https://clickhouse.com/blog/observability-mcp-server-ai-notebooks#clickstack-mcp-server)。

### 可执行程序型用户自定义函数 \{#executable-udfs\}

{/* vale on */ }

现在，你可以用 Python 编写一个函数，将其打包成 zip，上传到你的集群中，并像调用任何内置函数一样在 SQL 中调用它。ClickHouse 会运行一组长期存活的沙箱进程，并以查询速度让数据行流经这些进程，因此你可以在 SQL 能用到的任何地方调用你的函数：即席查询、JOIN，甚至每次插入时都会触发的 materialized view。你的模型代码就和数据放在一起，而部署它在 Cloud 控制台里只需通过一个上传页面即可完成。

为了展示它的能力，我们构建了一个[演示](https://github.com/ClickHouse/stock-anomaly-udf)。基于 StockHouse 中的实时股票交易数据，我们训练了一个小型 PyTorch 自动编码器，在股票交易 tick 被摄取时对异常进行评分。它只需要一个 UDF 和一条 DDL 语句，其余全部都是普通 SQL。我们还顺带预览了支持网络访问的 UDF (private beta) ，它允许 UDF 发起对外 HTTPS 调用。我们用它来拉取新闻和 SEC 文件，并让 Claude 对某笔交易为何看起来异常进行分类。

更多详情，请阅读[公告博客](https://clickhouse.com/blog/executable-udfs-clickhouse-cloud-beta)或[文档](/cloud/features/user-defined-functions)

### 组织支出告警正式发布 \{#spend-alerts-ga\}

计算-计算分离的主服务空闲功能现已正式可用。

## 2026年5月18日 \{#2026-05-18\}

### Python client v1.0.0 \{#python-client-v1\}

[clickhouse-connect](/integrations/python) 的重大版本发布，这是 ClickHouse 的 Python 客户端。主要亮点包括：

* 一流的 SQLAlchemy dialect，更深入地支持 ClickHouse 特有的 SQL
* 基于 aiohttp 构建的原生异步客户端，具备完整的功能一致性，包括流式传输
* DateTime、DateTime64、定宽数值 insert、Map、Decimal 和 BigDecimal 的性能显著提升
* 重新设计了时区处理机制，引入新的 `tz_mode` 和 `tz_source` 参数
* 原生支持写入 Variant 列
* 为可选依赖项 (numpy、pandas、pyarrow、polars) 提供延迟导入，使基础导入速度提升 4 倍
* Experimental Python 3.14 free-threading 支持

**重大变更：**不再支持 Python &lt; 3.9 和 Pandas 1.x。请查看[发行说明](https://github.com/ClickHouse/clickhouse-connect/releases)，了解其他与时区相关的重大变更。

### ClickStack 中基于 SQL 的图表与告警 \{#clickstack-sql-charting-alerting\}

[ClickStack](/use-cases/observability/clickstack) 现已支持使用任意 ClickHouse SQL 查询构建图表和告警。用户可以直接在仪表板和告警构建器中编写原生 SQL，完整支持窗口函数、CTE、多阶段查询以及 ClickHouse 原生分析函数。SQL 告警可将整个告警判定逻辑直接编码到查询本身中，从而支持异常检测、滞后平均值和统计尖峰检测。该功能适用于所有 ClickStack 部署形态 (Cloud、Private、BYOC、OSS) 。更多详情，请参见 [基于 SQL 的可视化](/use-cases/observability/clickstack/dashboards/sql-visualizations) 和 [SQL 告警结果解读](/use-cases/observability/clickstack/alerts#sql-result-interpretation)。

### Postgres 查询洞察 (私有预览) \{#postgres-query-insights\}

[ClickHouse Cloud Managed Postgres](/cloud/managed-postgres) (私有预览) 现已新增 **查询洞察** 选项卡。它会按影响程度对每种查询模式进行排序，并通过以下三个视图说明各类查询变慢的原因：

* **概览** — 在单个页面查看数据库健康状况
* **模式** — 可按运行时长、CPU、错误和 P95 排序的表格
* **单个模式详情面板** — 延迟百分位、CPU 与 I/O、缓存与磁盘、临时溢写、并行工作线程不足以及 WAL 量

## 2026 年 4 月 10 日 \{#2026-04-10\}

### 集中管理 Marketplace 计费 \{#centralized-marketplace-billing\}

客户现在可以通过单个云 Marketplace 订阅 (AWS、Azure 和 GCP) ，整合多个组织的计费。主要更新包括：

* **Marketplace 订阅共享：** 可将某个组织从信用卡计费切换为另一个组织现有的 Marketplace 订阅，无需再搭建额外的 Marketplace 订阅。仅合并 PAYG 用量 (不包括预付费额度) 。
* **更新 Marketplace 订阅：** 可将某个组织的 Marketplace 订阅改为使用另一个组织正在使用的订阅。
* **备用付款方式：** 使用 Marketplace 计费的组织现在可以添加信用卡作为备用付款方式，以防主要 Marketplace 订阅被取消或到期。

更多详情请参阅[计费文档](/manage/manage/billing/managing-payment-methods)。

## 2026 年 4 月 3 日 \{#april-3-2026\}

* **采用双窗口推荐器的更智能自动扩缩容：** ClickHouse Cloud 现已在垂直自动扩缩容中采用双回看窗口方案，以 3 小时的“小窗口”和 30 小时的“大窗口”组合替代此前单一的 30 小时窗口。这将缩容延迟从最长 30 小时缩短到最短 3 小时，在不牺牲扩容响应速度的前提下，降低可变工作负载的基础设施成本。更多详情，请参阅[博客文章](https://clickhouse.com/blog/smarter-auto-scaling)。
* **Cloud Console 中的监控：** 全新的概览仪表板和基础设施仪表板可帮助您更好地了解 ClickHouse 服务端的运行情况。管理用户现在在使用 ClickHouse Cloud 时，会通过电子邮件收到常见问题通知。更多详情，请参阅[博客文章](https://clickhouse.com/blog/clickhouse-cloud-new-monitoring-capabilities)。
* **Compute-compute separation (warehouses) ：** 主服务 (也称为父服务) 上的自动空闲功能现已开始逐步向公开预览推出。如果您的组织尚未启用该功能，请[联系支持团队](https://clickhouse.com/support/program)申请访问权限。该功能正式可用后，默认行为将变为：现有主服务可选择开启自动空闲，而新建主服务将默认启用该功能。
* **数据目录集成：** ClickHouse Cloud 现已支持在数据源 UI 中将 Iceberg REST Catalog 和 Microsoft OneLake 作为数据湖目录集成。连接后，该目录会显示为一个数据库，使您无需复制数据即可直接读取 Iceberg 表。
* **GCP 上的自带云环境 (BYOC) 现已正式可用：** ClickHouse BYOC 现已在 Google Cloud 上正式可用，使您能够在自己的 GCP 项目中部署 ClickHouse 服务，并满足严格的数据驻留要求。更多详情，请参阅[文档](/cloud/reference/byoc/overview)和[博客文章](https://clickhouse.com/blog/byoc-gcp-ga)。您可以[联系我们](https://clickhouse.com/cloud/bring-your-own-cloud)申请访问权限。
* **预付费额度和付款方式变更的计费通知：** 现在，当您的组织因承诺消费合同而新增预付费额度，或付款方式发生更新时 (信用卡或 Marketplace 订阅) ，ClickHouse Cloud 会发送通知。默认情况下，通知会通过 UI 和电子邮件发送，也可配置为同时发送到 Slack。

## 2026年1月23日 \{#january-23-2026\}

* ClickPipes 现已在 AWS `eu-west-1` 区域上线。对于在该区域内自 1 月 20 日起新建的 ClickHouse Cloud 服务，ClickPipes 将使用对应区域。对于更早创建的服务，ClickPipes 默认使用 `eu-central-1`。更多详情，请参阅[文档](/integrations/clickpipes#list-of-static-ips)。

## 2026 年 1 月 16 日 \{#january-16-2026\}

* **通过 API 按服务标签筛选使用成本：**我们的 API 现已支持按特定服务标签筛选您组织的使用成本，让您能够更轻松地按更细粒度分析支出。
* **数据目录集成：**您现在可以将外部数据目录连接为数据源。ClickHouse Cloud 支持 AWS Glue 和 Unity Catalog (很快还会支持更多) 。连接后，该目录会显示为一个数据库，让您无需复制数据即可直接读取 Iceberg 表。如需开通，请联系支持团队申请访问权限。
* **ClickPipes：**
  * MongoDB 连接器已升级为 Public Beta，现已面向所有服务层级的新老 ClickHouse Cloud 客户开放。阅读[博客文章](https://clickhouse.com/blog/mongodb-cdc-connector-clickpipes-beta)了解新功能概览，并前往[文档](/integrations/clickpipes/mongodb)上手。
  * [S3 ClickPipe](/integrations/clickpipes/object-storage/s3/overview) 现已兼容 OVH Object Storage，即 OVHcloud 中与 S3 API 兼容的对象存储服务。
