## 2026年5月18日 \{#2026-05-18\}

### Python client v1.0.0 \{#python-client-v1\}

[clickhouse-connect](/integrations/python) 的重大版本发布，这是 ClickHouse 的 Python 客户端。主要亮点包括：

* 一流的 SQLAlchemy dialect，更深入地支持 ClickHouse 特有的 SQL
* 基于 aiohttp 构建的原生异步客户端，具备完整的功能一致性，包括流式传输
* DateTime、DateTime64、定宽数值 insert、Map、Decimal 和 BigDecimal 的性能显著提升
* 重新设计了时区处理机制，引入新的 `tz_mode` 和 `tz_source` 参数
* 原生支持写入 Variant 列
* 为可选依赖项 (numpy、pandas、pyarrow、polars) 提供延迟导入，使基础导入速度提升 4 倍
* Experimental Python 3.14 free-threading 支持

**重大变更：**不再支持 Python &lt; 3.9 和 Pandas 1.x。请查看[发行说明](https://github.com/ClickHouse/clickhouse-connect/releases)，了解其他与时区相关的重大变更。

### ClickStack 中基于 SQL 的图表与告警 \{#clickstack-sql-charting-alerting\}

[ClickStack](/use-cases/observability/clickstack) 现已支持使用任意 ClickHouse SQL 查询构建图表和告警。用户可以直接在仪表板和告警构建器中编写原生 SQL，完整支持窗口函数、CTE、多阶段查询以及 ClickHouse 原生分析函数。SQL 告警可将整个告警判定逻辑直接编码到查询本身中，从而支持异常检测、滞后平均值和统计尖峰检测。该功能适用于所有 ClickStack 部署形态 (Cloud、Private、BYOC、OSS) 。更多详情，请参见 [基于 SQL 的可视化](/use-cases/observability/clickstack/dashboards/sql-visualizations) 和 [SQL 告警结果解读](/use-cases/observability/clickstack/alerts#sql-result-interpretation)。

现已支持以下新的 ClickHouse Cloud 区域：

* AWS Mexico (mx-central-1) — 私有区域，PCI

完整列表请参见[支持的区域](/cloud/reference/supported-regions)以及[私有区域注意事项](/cloud/reference/supported-regions#private-regions)。

### Kafka ClickPipes 的 Protobuf 支持 \{#protobuf-kafka-clickpipes\}

[Kafka ClickPipes](/integrations/clickpipes/kafka) 现已支持 [**Protobuf**](/integrations/clickpipes/kafka/reference#protobuf) 格式，除了 Avro 和 JSON 之外，现还可使用 Protobuf。其特性包括：通过 Schema Registry 集成自动解析 schema、支持递归引用、众所周知类型以及原生 `oneof` 类型。Protobuf 支持已覆盖 Kafka ClickPipes 的所有 interface (UI、OpenAPI 和 Terraform) 。

## 2026 年 5 月 8 日 \{#2026-05-08\}

### Postgres 查询洞察 (私有预览) \{#postgres-query-insights\}

[ClickHouse Cloud Managed Postgres](/cloud/managed-postgres) (私有预览) 现已新增 **查询洞察** 选项卡。它会按影响程度对每种查询模式进行排序，并通过以下三个视图说明各类查询变慢的原因：

* **概览** — 在单个页面查看数据库健康状况
* **模式** — 可按运行时长、CPU、错误和 P95 排序的表格
* **单个模式详情面板** — 延迟百分位、CPU 与 I/O、缓存与磁盘、临时溢写、并行工作线程不足以及 WAL 量

由开源 Postgres 扩展 [pg&#95;stat&#95;ch](https://github.com/ClickHouse/pg_stat_ch) 提供支持。

### 数据目录集成扩展 \{#data-catalog-integration-expansions\}

我们通过数据源 UI 进一步扩展了对[数据目录集成](/manage/data-catalogs)的支持，新增了更多身份验证方式、表格式和数据目录类型：

* 支持 [AWS Glue Catalog](/use-cases/data-lake/glue-catalog) 的 IAM 身份验证
* 支持将 [BigLake Metastore](/use-cases/data-lake/biglake-catalog) 用于数据目录集成
* 支持 [Unity Catalog](/use-cases/data-lake/unity-catalog) 上的 Delta Tables

## 2026年4月25日 \{#2026-04-25\}

### 索引分片 (私有预览) \{#index-sharding\}

索引分片现已开放私有预览。此功能可将索引分析阶段分散到多个副本上执行，从而降低单个副本的内存占用，并借助分布式并行处理提升查询性能。对于使用大量辅助索引 (例如向量搜索和全文搜索) 的表，这项功能尤为有用。更多详情请阅读[博客文章](https://clickhouse.com/blog/index-sharding-clickhouse-cloud-petabyte-scale-indexing)。

### ClickPipes 的 Terraform 和 OpenAPI (正式可用) \{#terraform-openapi-clickpipes-ga\}

ClickHouse [Terraform provider](https://registry.terraform.io/providers/ClickHouse/clickhouse/latest/docs) 中的 ClickPipes 资源现已随稳定版本正式可用，[OpenAPI 端点](https://clickhouse.com/docs/cloud/manage/api/swagger) 也不再标记为 Beta。这包括对 CDC 连接器 (Postgres、MySQL、MongoDB) 的全面支持。有关更多详细信息，请参阅 [Terraform 文档](/integrations/clickpipes/programmatic-access/terraform)、[OpenAPI 文档](/integrations/clickpipes/programmatic-access/openapi) 和 [博客文章](https://clickhouse.com/blog/terraform-ga)。

## 2026年4月17日 \{#april-17-2026\}

### 可在账单 UI 中查看退款 \{#refunds-billing-ui\}

现在，客户可直接在 Cloud 控制台的账单 UI 中查看所有退款。退款会在处理完成后显示；信用卡退款显示在账单上的时间可能会更长，具体取决于银行。

### 组织支出提醒 (私有预览) \{#org-spend-alerts\}

组织支出提醒现已开放私有预览。当每个计费周期内配置的组织支出达到 50%、75% 和 100% 时，您将收到通知。可选择电子邮件、Cloud 控制台和/或 Slack 作为通知渠道。提醒基于组织总用量，每小时触发一次，并会在每个计费周期自动重置。需要具备 Org Admin 或 Billing Admin 角色。

### AWS 上的 ClickPipes 区域覆盖现已与 ClickHouse Cloud 持平 \{#clickpipes-aws-region-parity\}

ClickPipes 现已在 ClickHouse Cloud 支持的所有 AWS 区域 (公有和私有) 中可用，覆盖范围已从 6 个区域扩展到 18 个区域。其中包括新加坡 (ap-southeast-1) 、首尔 (ap-northeast-2) 和东京 (ap-northeast-1) 等重点区域。新区域适用于 2026 年 4 月 14 日之后创建的服务。更多详情请参阅[文档](/integrations/clickpipes#list-of-static-ips)。

## 2026 年 4 月 10 日 \{#2026-04-10\}

### 集中管理 Marketplace 计费 \{#centralized-marketplace-billing\}

客户现在可以通过单个云 Marketplace 订阅 (AWS、Azure 和 GCP) ，整合多个组织的计费。主要更新包括：

* **Marketplace 订阅共享：** 可将某个组织从信用卡计费切换为另一个组织现有的 Marketplace 订阅，无需再搭建额外的 Marketplace 订阅。仅合并 PAYG 用量 (不包括预付费额度) 。
* **更新 Marketplace 订阅：** 可将某个组织的 Marketplace 订阅改为使用另一个组织正在使用的订阅。
* **备用付款方式：** 使用 Marketplace 计费的组织现在可以添加信用卡作为备用付款方式，以防主要 Marketplace 订阅被取消或到期。

更多详情请参阅[计费文档](/manage/manage/billing/managing-payment-methods)。

### clickhousectl CLI (Public Beta) \{#clickhousectl-cli\}

[clickhousectl](https://github.com/ClickHouse/clickhousectl) 是 ClickHouse 官方提供的 CLI，用于管理本地安装、运行本地服务器以及操作 ClickHouse Cloud。立即安装：

```bash
curl https://clickhouse.com/cli | sh
```

## 2026 年 4 月 3 日 \{#april-3-2026\}

* **采用双窗口推荐器的更智能自动扩缩容：** ClickHouse Cloud 现已在垂直自动扩缩容中采用双回看窗口方案，以 3 小时的“小窗口”和 30 小时的“大窗口”组合替代此前单一的 30 小时窗口。这将缩容延迟从最长 30 小时缩短到最短 3 小时，在不牺牲扩容响应速度的前提下，降低可变工作负载的基础设施成本。更多详情，请参阅[博客文章](https://clickhouse.com/blog/smarter-auto-scaling)。
* **Cloud Console 中的监控：** 全新的概览仪表板和基础设施仪表板可帮助您更好地了解 ClickHouse 服务端的运行情况。管理用户现在在使用 ClickHouse Cloud 时，会通过电子邮件收到常见问题通知。更多详情，请参阅[博客文章](https://clickhouse.com/blog/clickhouse-cloud-new-monitoring-capabilities)。
* **Compute-compute separation (warehouses) ：** 主服务 (也称为父服务) 上的自动空闲功能现已开始逐步向公开预览推出。如果您的组织尚未启用该功能，请[联系支持团队](https://clickhouse.com/support/program)申请访问权限。该功能正式可用后，默认行为将变为：现有主服务可选择开启自动空闲，而新建主服务将默认启用该功能。
* **数据目录集成：** ClickHouse Cloud 现已支持在数据源 UI 中将 Iceberg REST Catalog 和 Microsoft OneLake 作为数据湖目录集成。连接后，该目录会显示为一个数据库，使您无需复制数据即可直接读取 Iceberg 表。
* **GCP 上的自带云环境 (BYOC) 现已正式可用：** ClickHouse BYOC 现已在 Google Cloud 上正式可用，使您能够在自己的 GCP 项目中部署 ClickHouse 服务，并满足严格的数据驻留要求。更多详情，请参阅[文档](/cloud/reference/byoc/overview)和[博客文章](https://clickhouse.com/blog/byoc-gcp-ga)。您可以[联系我们](https://clickhouse.com/cloud/bring-your-own-cloud)申请访问权限。
* **预付费额度和付款方式变更的计费通知：** 现在，当您的组织因承诺消费合同而新增预付费额度，或付款方式发生更新时 (信用卡或 Marketplace 订阅) ，ClickHouse Cloud 会发送通知。默认情况下，通知会通过 UI 和电子邮件发送，也可配置为同时发送到 Slack。

## 2026 年 3 月 20 日 \{#march-20-2026\}

* **Usage Breakdown 的自定义日期范围：** 您现在可以在 Usage Breakdown 页面中使用自定义日期范围，查看所有计费维度的使用成本。
  选择开始日期和结束日期 (含首尾两天) ，按特定时间段筛选成本，并将结果下载为 CSV。
