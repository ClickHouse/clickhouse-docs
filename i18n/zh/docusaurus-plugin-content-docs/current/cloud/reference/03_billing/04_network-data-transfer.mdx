ClickHouse Cloud 会监控并计量入站和出站的数据传输量。
这包括所有进出 ClickHouse Cloud 的数据，以及区域内和跨区域的数据传输。
此类用量在服务级别进行跟踪。基于这些用量，客户会产生数据传输费用，并计入其每月账单。

ClickHouse Cloud 对以下内容收费：

* 从 ClickHouse Cloud 到公网的数据出站传输，包括到其他云服务商其他区域的数据传输。
* 向同一云服务商中其他区域的数据出站传输。

对于区域内数据传输或使用 Private Link/Private Service Connect 及其相关数据传输，不收取费用。
但是，如果我们观察到的使用模式影响到我们对用户进行合理收费的能力，我们保留引入额外数据传输计费维度的权利。

注意：使用 Private Link/Private Service Connect 时，会产生云服务商（Cloud Service Provider，CSP）费用。请参考相应 CSP 的价格页面以获取最新信息。

数据传输费用会因云服务商（CSP）和区域不同而变化。
公网出站定价仅基于源区域。
区域间（或跨区域）定价取决于源区域和目标区域。

**降低数据传输成本的最佳实践**

在进行数据入站和出站时，有一些模式需要牢记，以尽量降低数据传输成本。

1. 从 ClickHouse Cloud 进行数据入站或出站时，在可能的情况下使用压缩，以减少传输数据量及相关成本。

2. 请注意，当通过原生协议、使用非内联值执行 INSERT 操作时（例如 `INSERT INTO [TABLE] FROM INFILE [FILE] FORMAT NATIVE`），ClickHouse
   客户端会从服务器拉取元数据以打包数据。如果元数据比 `INSERT` 负载更大，那么从服务器视角来看，出站流量可能反常地大于入站流量。
   如果这不可接受，请考虑使用 `VALUES` 语法将数据内联，或使用 HTTP 协议。
