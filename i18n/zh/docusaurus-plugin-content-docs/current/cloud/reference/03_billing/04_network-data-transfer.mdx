ClickHouse Cloud 会对入站和出站的数据传输进行计量。
这包括所有与 ClickHouse Cloud 之间的数据传输，以及区域内和跨区域的数据传输。
此类用量在服务级别进行跟踪。
基于这些用量，客户会产生数据传输费用，并计入其每月账单。

ClickHouse Cloud 就以下内容收费：

* 从 ClickHouse Cloud 到公共互联网的数据出站，包括到其他云服务商其他区域的数据。
* 到同一云服务商中另一个区域的数据出站。

对于区域内数据传输，或使用 Private Link/Private Service Connect 的数据传输，不收取费用。
但是，如果我们发现某些使用模式影响我们对用户进行合理计费的能力，我们保留引入额外数据传输计费维度的权利。

注意：使用 Private Link/Private Service Connect 时，会产生云服务商（CSP）的费用。
请参阅各 CSP 的定价页面以获取最新信息。

数据传输费用会因云服务商（CSP）和区域而异。
公共互联网出站定价仅基于源区域。
区域间（或跨区域）定价取决于源区域和目标区域。

**降低数据传输成本的最佳实践**

在进行数据入站和出站时，可以遵循一些模式来尽量降低数据传输成本。

1. 从 ClickHouse Cloud 进行数据入站或出站时，在可能的情况下使用压缩，以尽量减少传输数据量及相关费用。

2. 请注意，当通过原生协议并使用非内联值执行 `INSERT` 时（例如 `INSERT INTO [TABLE] FROM INFILE [FILE] FORMAT NATIVE`），ClickHouse 客户端会从服务器拉取元数据以打包数据。
   如果元数据比 `INSERT` 负载还大，从服务器视角看，你可能会反直觉地看到出站数据量大于入站数据量。
   如果无法接受这种情况，可以考虑使用 `VALUES` 语法将数据内联，或使用 HTTP 协议。
