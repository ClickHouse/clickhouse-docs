模型参数用于控制智能体使用哪个模型，以及该模型如何生成响应。请在 Agent Builder 的 **Model Parameters** 面板中进行配置。

## 提供商和模型 \{#provider-and-model\}

* **提供商** — 上游 LLM 提供商。
* **模型** — 该提供商提供的具体模型。不同模型各有优势：大型推理模型适合需要大量规划的任务，更快的小型模型适合常规查询。

提供商和模型为必填项。面板的其余部分会根据所选模型支持的功能自动调整。

## 上下文和输出限制 \{#context-and-output-limits\}

* **最大上下文标记数** — 限制智能体发送给模型的上下文总量。保留为 **System** 可使用模型的默认值。调低可降低成本；对于需要基于大量输入进行推理的智能体，可调高。
* **最大输出标记数** — 限制智能体响应的长度。**System** 使用模型的默认值。如果响应过长，请调低；如果响应被截断，请调高。
* **文件标记上限** — 限制单个上传文件可为上下文提供多少标记。在用户上传大文件且你不希望其挤占对话其余上下文时，这会很有用。

## 采样 \{#sampling\}

* **Temperature** — 随机性。值越高 (0.7–1.0) = 越随机；值越低 (0.0–0.3) = 越聚焦、越具确定性。我们建议调整这个参数或 Top P，但不要同时调整两者。
* **Top P** — 核采样。它会改变模型选择输出标记的方式。
* **Top K** — 将采样限制在每一步概率最高的前 K 个标记内。部分提供商支持；它从不同于 Temperature 的另一个维度控制确定性。

如果你不是为了某种特定行为进行调优，请将滑块保持在接近默认值的位置——这里的小幅改动通常很难带来明显变化，而大幅改动则可能降低输出质量。

## 推理控制 \{#reasoning-controls\}

适用于支持扩展推理的模型。具体可用的选项因提供商而异。

* **Thinking** — 切换模型的扩展推理模式。启用后，模型会先生成内部思考标记，再给出最终答案；这通常能提高处理高难度任务时的准确性，但会增加延迟和标记消耗。
* **Thinking Budget** — 思考阶段的标记预算。模型在消耗达到该标记数后会停止思考并给出答案。
* **Effort** — 高层级的推理强度设置 (**Auto**、low、medium、high) 。用于未直接提供思考标记预算的推理模型。
* **Thought Visibility** — 控制模型的思考过程是直接内联显示给用户、隐藏在折叠视图中，还是完全不显示。

## 对话行为 \{#conversation-behavior\}

* **重新发送文件** — 开启后，之前轮次中附带的文件会在之后的每一轮对话中再次发送，避免模型丢失对这些文件的上下文。如果对话较短，或者模型会在处理过程中逐步总结文件内容，可关闭此选项以节省标记。
* **使用提示缓存** — 当提供商支持时，会缓存提示中可复用的部分，以降低那些在多轮对话中重复使用指令和工具描述的场景下的成本和延迟。
* **Web Search** — 在受支持的模型上切换提供商原生的网页搜索功能。这与 [Web search tool](/cloud/features/ai-ml/agents/builder/web-search) 不同，后者作为智能体的工具之一运行，而不是作为提供商能力运行。

## 重置 \{#reset\}

面板底部的 **重置模型参数** 会将所有字段恢复为系统默认值。在进行多次尝试后，如果你想重新开始并回到一个干净的初始状态，请使用此选项。
