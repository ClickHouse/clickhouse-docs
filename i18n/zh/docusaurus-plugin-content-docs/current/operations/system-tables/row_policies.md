## 描述 \{#description\}

包含针对某个特定表的过滤器，以及应使用此ROW POLICY的角色和/或用户列表。

## 列 \{#columns\}

{/*AUTOGENERATED_START*/ }

* `name` ([String](../../sql-reference/data-types/)) — ROW POLICY的名称。
* `short_name` ([String](../../sql-reference/data-types/)) — ROW POLICY的短名称。ROW POLICY的名称是复合名称，例如：myfilter ON mydb.mytable。其中，&#39;myfilter ON mydb.mytable&#39; 是ROW POLICY的名称，&#39;myfilter&#39; 是其短名称。
* `database` ([String](../../sql-reference/data-types/)) — 数据库名称。
* `table` ([String](../../sql-reference/data-types/)) — 表名称。如果该策略针对数据库，则为空。
* `id` ([UUID](../../sql-reference/data-types/)) — ROW POLICY ID。
* `storage` ([String](../../sql-reference/data-types/)) — 存储该ROW POLICY的目录名称。
* `select_filter` ([Nullable(String)](../../sql-reference/data-types/)) — 用于在 SELECT 查询中过滤数据的表达式。
* `is_restrictive` ([UInt8](../../sql-reference/data-types/)) — 表示该ROW POLICY是否限制对行的访问。值：• 0 — 该ROW POLICY使用 `AS PERMISSIVE` 子句定义，• 1 — 该ROW POLICY使用 `AS RESTRICTIVE` 子句定义。
* `apply_to_all` ([UInt8](../../sql-reference/data-types/)) — 表示该ROW POLICY适用于所有角色和/或用户。
* `apply_to_list` ([Array(String)](../../sql-reference/data-types/)) — 应用该ROW POLICY的角色和/或用户列表。
* `apply_to_except` ([Array(String)](../../sql-reference/data-types/)) — 该ROW POLICY适用于除所列角色和/或用户之外的所有角色和/或用户。

{/*AUTOGENERATED_END*/ }
