## 描述 \{#description\}

该表包含向 ZooKeeper 服务器发起请求时的参数以及服务器响应相关的信息。

对于请求，只会填充包含请求参数的列，其余列填充默认值 (`0` 或 `NULL`) 。当收到响应时，会将响应中的数据补充到其他列中。

## 列 \{#columns\}

{/*AUTOGENERATED_START*/ }

* `hostname` ([LowCardinality(String)](/sql-reference/data-types/lowcardinality)) — 执行该查询的服务器主机名。
* `type` ([Enum8(&#39;Request&#39; = 1, &#39;Response&#39; = 2, &#39;Finalize&#39; = 3)](/sql-reference/data-types/enum)) — ZooKeeper 客户端中的事件类型。可以具有以下值之一：Request — 请求已发送，Response — 已收到响应，Finalize — 连接已断开，未收到响应。
* `event_date` ([Date](/sql-reference/data-types/date)) — 事件发生的日期。
* `event_time` ([DateTime64(6)](/sql-reference/data-types/datetime64)) — 事件发生时的日期和时间。
* `thread_id` ([UInt64](/sql-reference/data-types/int-uint)) — 执行此请求的线程 ID。
* `query_id` ([String](/sql-reference/data-types/string)) — 执行此请求时所属查询的 ID。
* `address` ([IPv6](/sql-reference/data-types/ipv6)) — 发起该请求时使用的 ZooKeeper 服务器的 IP 地址。
* `port` ([UInt16](/sql-reference/data-types/int-uint)) — 发起该请求时使用的 ZooKeeper 服务器端口。
* `session_id` ([Int64](/sql-reference/data-types/int-uint)) — ZooKeeper 服务器为每个连接设置的会话 ID。
* `duration_microseconds` ([UInt64](/sql-reference/data-types/int-uint)) — ZooKeeper 执行该请求所耗费的时间。
* `xid` ([Int64](/sql-reference/data-types/int-uint)) — 会话中的请求 ID。通常是按顺序递增的请求编号。对于请求行及其对应的响应/完成行，该值相同。
* `has_watch` ([UInt8](/sql-reference/data-types/int-uint)) — 请求是否已设置 watch。
* `op_num` ([Enum16(&#39;Close&#39; = -11, &#39;Error&#39; = -1, &#39;Watch&#39; = 0, &#39;Create&#39; = 1, &#39;Remove&#39; = 2, &#39;Exists&#39; = 3, &#39;Get&#39; = 4, &#39;Set&#39; = 5, &#39;GetACL&#39; = 6, &#39;SetACL&#39; = 7, &#39;SimpleList&#39; = 8, &#39;Sync&#39; = 9, &#39;Heartbeat&#39; = 11, &#39;List&#39; = 12, &#39;Check&#39; = 13, &#39;Multi&#39; = 14, &#39;Create2&#39; = 15, &#39;Reconfig&#39; = 16, &#39;CheckWatch&#39; = 17, &#39;RemoveWatch&#39; = 18, &#39;MultiRead&#39; = 22, &#39;Auth&#39; = 100, &#39;SetWatch&#39; = 101, &#39;SetWatch2&#39; = 105, &#39;AddWatch&#39; = 106, &#39;FilteredList&#39; = 500, &#39;CheckNotExists&#39; = 501, &#39;CreateIfNotExists&#39; = 502, &#39;RemoveRecursive&#39; = 503, &#39;CheckStat&#39; = 504, &#39;TryRemove&#39; = 505, &#39;FilteredListWithStatsAndData&#39; = 506, &#39;ListRecursive&#39; = 507, &#39;SessionID&#39; = 997)](/sql-reference/data-types/enum)) — 请求或响应类型。
* `path` ([String](/sql-reference/data-types/string)) — 请求中指定的 ZooKeeper 节点的路径；如果该请求无需指定路径，则为空字符串。
* `data` ([String](/sql-reference/data-types/string)) — 写入 ZooKeeper 节点的数据 (对于 SET 和 CREATE 请求，即请求要写入的内容；对于 GET 请求的响应，即读取到的内容) ，或空字符串。
* `is_ephemeral` ([UInt8](/sql-reference/data-types/int-uint)) — ZooKeeper 节点是否创建为临时节点。
* `is_sequential` ([UInt8](/sql-reference/data-types/int-uint)) — 是否以 sequential 模式创建 ZooKeeper 节点。
* `version` ([Nullable(Int32)](/sql-reference/data-types/nullable)) — 执行请求时所期望的 ZooKeeper 节点版本。此项适用于 CHECK、SET、REMOVE 请求 (如果请求不检查版本，则该值为 -1；对于其他不支持版本检查的请求，则为 NULL) 。
* `requests_size` ([UInt32](/sql-reference/data-types/int-uint)) — 多请求中包含的请求数 (这是一种特殊请求，由多个连续的普通请求组成，并以原子方式执行) 。多请求中包含的所有请求都将具有相同的 xid。
* `request_idx` ([UInt32](/sql-reference/data-types/int-uint)) — 多请求中所包含请求的编号 (对于多请求，先为 0，然后按顺序从 1 开始) 。
* `zxid` ([Int64](/sql-reference/data-types/int-uint)) — ZooKeeper 事务 ID。表示 ZooKeeper 服务器为成功执行的请求分配的序列号 (如果请求未执行/返回错误/客户端不知道请求是否已执行，则为 0) 。
* `error` ([Nullable(Enum8(&#39;ZNOWATCHER&#39; = -121, &#39;ZNOTREADONLY&#39; = -119, &#39;ZSESSIONMOVED&#39; = -118, &#39;ZNOTHING&#39; = -117, &#39;ZCLOSING&#39; = -116, &#39;ZAUTHFAILED&#39; = -115, &#39;ZINVALIDACL&#39; = -114, &#39;ZINVALIDCALLBACK&#39; = -113, &#39;ZSESSIONEXPIRED&#39; = -112, &#39;ZNOTEMPTY&#39; = -111, &#39;ZNODEEXISTS&#39; = -110, &#39;ZNOCHILDRENFOREPHEMERALS&#39; = -108, &#39;ZBADVERSION&#39; = -103, &#39;ZNOAUTH&#39; = -102, &#39;ZNONODE&#39; = -101, &#39;ZAPIERROR&#39; = -100, &#39;ZOUTOFMEMORY&#39; = -10, &#39;ZINVALIDSTATE&#39; = -9, &#39;ZBADARGUMENTS&#39; = -8, &#39;ZOPERATIONTIMEOUT&#39; = -7, &#39;ZUNIMPLEMENTED&#39; = -6, &#39;ZMARSHALLINGERROR&#39; = -5, &#39;ZCONNECTIONLOSS&#39; = -4, &#39;ZDATAINCONSISTENCY&#39; = -3, &#39;ZRUNTIMEINCONSISTENCY&#39; = -2, &#39;ZSYSTEMERROR&#39; = -1, &#39;ZOK&#39; = 0))](/sql-reference/data-types/nullable)) — 错误代码。可取很多值，这里仅列出其中一部分：ZOK — 请求已成功执行，ZCONNECTIONLOSS — 连接已断开，ZOPERATIONTIMEOUT — 请求执行已超时，ZSESSIONEXPIRED — 会话已过期，NULL — 请求已完成。
* `watch_type` ([Nullable(Enum8(&#39;NOTWATCHING&#39; = -2, &#39;SESSION&#39; = -1, &#39;CREATED&#39; = 1, &#39;DELETED&#39; = 2, &#39;CHANGED&#39; = 3, &#39;CHILD&#39; = 4))](/sql-reference/data-types/nullable)) — watch 事件类型 (对于 op&#95;num = Watch 的响应) ；其余响应为 NULL。
* `watch_state` ([Nullable(Enum16(&#39;AUTH&#95;FAILED&#39; = -113, &#39;EXPIRED&#95;SESSION&#39; = -112, &#39;CONNECTING&#39; = 1, &#39;ASSOCIATING&#39; = 2, &#39;CONNECTED&#39; = 3, &#39;READONLY&#39; = 5, &#39;NOTCONNECTED&#39; = 999))](/sql-reference/data-types/nullable)) — watch 事件的状态 (适用于 op&#95;num = Watch 的响应) ；其余响应为 NULL。
* `path_created` ([String](/sql-reference/data-types/string)) — 已创建的 ZooKeeper 节点的路径 (用于响应 CREATE 请求) ；如果该节点以 sequential 模式创建，则此路径可能与原始路径不同。
* `stat_czxid` ([Int64](/sql-reference/data-types/int-uint)) — 导致创建此 ZooKeeper 节点的修改对应的 zxid。
* `stat_mzxid` ([Int64](/sql-reference/data-types/int-uint)) — 最后一次修改此 ZooKeeper 节点的变更 zxid。
* `stat_pzxid` ([Int64](/sql-reference/data-types/int-uint)) — 最后一次修改此 ZooKeeper 节点子节点的事务 ID。
* `stat_version` ([Int32](/sql-reference/data-types/int-uint)) — 该 ZooKeeper 节点的数据变更次数。
* `stat_cversion` ([Int32](/sql-reference/data-types/int-uint)) — 此 ZooKeeper 节点子节点的变更次数。
* `stat_dataLength` ([Int32](/sql-reference/data-types/int-uint)) — 此 ZooKeeper 节点中数据字段的长度。
* `stat_numChildren` ([Int32](/sql-reference/data-types/int-uint)) — 此 ZooKeeper 节点的子节点数。
* `children` ([Array(String)](/sql-reference/data-types/array)) — 子 ZooKeeper 节点列表 (用于响应 LIST 请求) 。

{/*AUTOGENERATED_END*/ }
