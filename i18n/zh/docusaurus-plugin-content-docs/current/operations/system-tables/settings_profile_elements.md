## 描述 \{#description\}

描述SETTINGS PROFILE的内容：

* 约束。
* 该设置适用的角色和用户。
* 父级SETTINGS PROFILE。

## 列 \{#columns\}

{/*AUTOGENERATED_START*/ }

* `profile_name` ([Nullable(String)](../../sql-reference/data-types/)) — SETTINGS PROFILE 名称。
* `user_name` ([Nullable(String)](../../sql-reference/data-types/)) — 用户名。
* `role_name` ([Nullable(String)](../../sql-reference/data-types/)) — 角色名称。
* `index` ([UInt64](../../sql-reference/data-types/)) — SETTINGS PROFILE 元素的顺序编号。
* `setting_name` ([Nullable(String)](../../sql-reference/data-types/)) — 设置名称。
* `value` ([Nullable(String)](../../sql-reference/data-types/)) — 设置值。
* `min` ([Nullable(String)](../../sql-reference/data-types/)) — 设置的最小值。若未设置，则为 NULL。
* `max` ([Nullable(String)](../../sql-reference/data-types/)) — 设置的最大值。若未设置，则为 NULL。
* `writability` ([Nullable(Enum8(&#39;WRITABLE&#39; = 0, &#39;CONST&#39; = 1, &#39;CHANGEABLE&#95;IN&#95;READONLY&#39; = 2))](../../sql-reference/data-types/)) — 指示某项设置是否可修改的属性。
* `inherit_profile` ([Nullable(String)](../../sql-reference/data-types/)) — 此SETTINGS PROFILE 的父 profile。若未设置，则为 NULL。SETTINGS PROFILE 将从其父 profile 继承所有设置的值和约束 (min、max、readonly) 。

{/*AUTOGENERATED_END*/ }
