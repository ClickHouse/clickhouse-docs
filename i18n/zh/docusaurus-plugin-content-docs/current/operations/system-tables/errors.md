## 描述 \{#description\}

包含错误代码及其被触发的次数。

要显示所有可能的错误代码 (包括那些未被触发的) ，请将设置 [system&#95;events&#95;show&#95;zero&#95;values](../settings/settings.md#system_events_show_zero_values) 设为 1。

## 列 \{#columns\}

{/*AUTOGENERATED_START*/ }

* `name` ([String](../../sql-reference/data-types/)) — 错误名称 (errorCodeToName) 。
* `code` ([Int32](../../sql-reference/data-types/)) — 错误代码编号。
* `value` ([UInt64](../../sql-reference/data-types/)) — 此错误发生的次数。
* `last_error_time` ([DateTime](../../sql-reference/data-types/)) — 最近一次发生错误的时间。
* `last_error_message` ([String](../../sql-reference/data-types/)) — 最近一次错误的错误消息。
* `last_error_format_string` ([String](../../sql-reference/data-types/)) — 最近一次错误的格式字符串。
* `last_error_trace` ([Array(UInt64)](../../sql-reference/data-types/)) — 堆栈跟踪，表示一个物理地址列表，这些地址对应被调用方法的存储位置。
* `remote` ([UInt8](../../sql-reference/data-types/)) — 远程异常 (即在某次分布式查询期间接收到的异常) 。
* `query_id` ([String](../../sql-reference/data-types/)) — 导致错误的查询 ID (如果可用) 。

{/*AUTOGENERATED_END*/ }

:::note
某些错误计数器即使在查询成功执行时也可能增加。除非您确定相应错误不可能是误报，否则不建议将此表用于服务器监控。
:::

## 示例 \{#example\}

```sql title="Query"
SELECT name, code, value
FROM system.errors
WHERE value > 0
ORDER BY code ASC
LIMIT 1

┌─name─────────────┬─code─┬─value─┐
│ CANNOT_OPEN_FILE │   76 │     1 │
└──────────────────┴──────┴───────┘
```

```sql title="Response"
WITH arrayMap(x -> demangle(addressToSymbol(x)), last_error_trace) AS all
SELECT name, arrayStringConcat(all, '\n') AS res
FROM system.errors
LIMIT 1
SETTINGS allow_introspection_functions=1\G
```
