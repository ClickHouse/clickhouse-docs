## 描述 \{#description\}

包含所有 `BACKUP` 或 `RESTORE` 操作及其当前状态和其他属性的列表。注意，该表不是持久化的，只显示上次服务器重启之后执行的操作。

## 列 \{#columns\}

{/*AUTOGENERATED_START*/ }

* `id` ([String](../../sql-reference/data-types/)) — 操作 ID，可以通过 SETTINGS id=... 传入，也可以是随机生成的 UUID。
* `name` ([String](../../sql-reference/data-types/)) — 操作名称，即形如 `Disk('backups', 'my_backup')` 的字符串。
* `base_backup_name` ([String](../../sql-reference/data-types/)) — 基设备份操作名称，即形如 `Disk('backups', 'my_base_backup')` 的字符串。
* `query_id` ([String](../../sql-reference/data-types/)) — 启动备份的查询的查询 ID。
* `status` ([Enum8(&#39;CREATING&#95;BACKUP&#39; = 0, &#39;BACKUP&#95;CREATED&#39; = 1, &#39;BACKUP&#95;FAILED&#39; = 2, &#39;RESTORING&#39; = 3, &#39;RESTORED&#39; = 4, &#39;RESTORE&#95;FAILED&#39; = 5, &#39;BACKUP&#95;CANCELLED&#39; = 6, &#39;RESTORE&#95;CANCELLED&#39; = 7)](../../sql-reference/data-types/)) — 备份或恢复操作的状态。
* `error` ([String](../../sql-reference/data-types/)) — 错误消息 (如有) 。
* `start_time` ([DateTime64(6)](../../sql-reference/data-types/)) — 操作开始时间。
* `end_time` ([DateTime64(6)](../../sql-reference/data-types/)) — 操作完成时间。
* `num_files` ([UInt64](../../sql-reference/data-types/)) — 备份中存储的文件数。
* `total_size` ([UInt64](../../sql-reference/data-types/)) — 备份中存储的文件总大小。
* `num_entries` ([UInt64](../../sql-reference/data-types/)) — 备份中的条目数；如果备份以文件夹形式存储，则表示该文件夹中的文件数。
* `uncompressed_size` ([UInt64](../../sql-reference/data-types/)) — 备份的未压缩大小。
* `compressed_size` ([UInt64](../../sql-reference/data-types/)) — 备份的压缩后大小。
* `files_read` ([UInt64](../../sql-reference/data-types/)) — 返回从此备份执行 RESTORE 期间读取的文件数。
* `bytes_read` ([UInt64](../../sql-reference/data-types/)) — 返回从此备份执行 RESTORE 期间读取的文件总大小。
* `ProfileEvents` ([Map(LowCardinality(String), UInt64)](../../sql-reference/data-types/)) — 此操作期间捕获的所有 profile events。

{/*AUTOGENERATED_END*/ }
