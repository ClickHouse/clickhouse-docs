## 描述 \{#description\}

包含关于存储在 ClickHouse Keeper 或 ZooKeeper 中、属于 `ReplicatedMergeTree` 系列表的复制队列任务的信息。

## 列 \{#columns\}

{/*AUTOGENERATED_START*/ }

* `database` ([String](../../sql-reference/data-types/)) — 数据库的名称。
* `table` ([String](../../sql-reference/data-types/)) — 表的名称。
* `replica_name` ([String](../../sql-reference/data-types/)) — ClickHouse Keeper 中的副本名称。同一张表的不同副本名称各不相同。
* `position` ([UInt32](../../sql-reference/data-types/)) — 任务在队列中的位置。
* `node_name` ([String](../../sql-reference/data-types/)) — ClickHouse Keeper 中的节点名称。
* `type` ([String](../../sql-reference/data-types/)) — 队列中任务的类型，取值之一为：• GET&#95;PART — 从另一个副本获取 part；• ATTACH&#95;PART — 附加 part，可能来自当前副本 (如果在 detached 文件夹中找到) 。可以将其视为经过一些优化的 GET&#95;PART，因为两者几乎相同；• MERGE&#95;PARTS — 合并 parts；• DROP&#95;RANGE — 删除指定分区内、指定编号范围中的 parts。• CLEAR&#95;COLUMN — 注意：已弃用。从指定分区中删除特定列。• CLEAR&#95;INDEX — 注意：已弃用。从指定分区中删除特定索引。• REPLACE&#95;RANGE — 删除某个范围内的 parts，并用新的 parts 替换它们。• MUTATE&#95;PART — 对 part 应用一个或多个 mutations。• ALTER&#95;METADATA — 根据全局 /metadata 和 /columns 路径应用 alter 修改。
* `create_time` ([DateTime](../../sql-reference/data-types/)) — 提交任务执行时的日期和时间。
* `required_quorum` ([UInt32](../../sql-reference/data-types/)) — 等待该任务完成并确认完成的副本数量。此列仅与 GET&#95;PARTS 任务相关。
* `source_replica` ([String](../../sql-reference/data-types/)) — 源副本名称。
* `new_part_name` ([String](../../sql-reference/data-types/)) — 新 part 的名称。
* `parts_to_merge` ([Array(String)](../../sql-reference/data-types/)) — 要合并或更新的 parts 名称。
* `is_detach` ([UInt8](../../sql-reference/data-types/)) — 该标志表示队列中是否存在 DETACH&#95;PARTS 任务。
* `is_currently_executing` ([UInt8](../../sql-reference/data-types/)) — 该标志表示某个特定任务当前是否正在执行。
* `num_tries` ([UInt32](../../sql-reference/data-types/)) — 完成该任务的失败尝试次数。
* `last_exception` ([String](../../sql-reference/data-types/)) — 最近一次发生的错误消息 (如果有) 。
* `last_exception_time` ([DateTime](../../sql-reference/data-types/)) — 最近一次发生错误的日期和时间。
* `last_attempt_time` ([DateTime](../../sql-reference/data-types/)) — 最近一次尝试执行该任务的日期和时间。
* `num_postponed` ([UInt32](../../sql-reference/data-types/)) — 该操作被推迟的次数。
* `postpone_reason` ([String](../../sql-reference/data-types/)) — 任务被推迟的原因。
* `last_postpone_time` ([DateTime](../../sql-reference/data-types/)) — 最近一次推迟该任务的日期和时间。
* `merge_type` ([String](../../sql-reference/data-types/)) — 当前合并的类型。如果是 mutation，则为空。

{/*AUTOGENERATED_END*/ }
