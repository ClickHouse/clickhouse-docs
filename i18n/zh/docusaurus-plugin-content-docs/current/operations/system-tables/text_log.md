## 描述 \{#description\}

包含日志记录条目。写入该表的日志级别可以通过服务器设置 `text_log.level` 进行限制。

## 列 \{#columns\}

{/*AUTOGENERATED_START*/ }

* `hostname` ([LowCardinality(String)](/sql-reference/data-types/lowcardinality)) — 执行查询的服务器主机名。
* `event_date` ([Date](/sql-reference/data-types/date)) — 条目的日期。
* `event_time` ([DateTime](/sql-reference/data-types/datetime)) — 条目的时间。
* `event_time_microseconds` ([DateTime64(6)](/sql-reference/data-types/datetime64)) — 精确到微秒的条目时间。
* `thread_name` ([LowCardinality(String)](/sql-reference/data-types/lowcardinality)) — 记录日志的线程名称。
* `thread_id` ([UInt64](/sql-reference/data-types/int-uint)) — OS 线程 ID。
* `level` ([Enum8(&#39;Fatal&#39; = 1, &#39;Critical&#39; = 2, &#39;Error&#39; = 3, &#39;Warning&#39; = 4, &#39;Notice&#39; = 5, &#39;Information&#39; = 6, &#39;Debug&#39; = 7, &#39;Trace&#39; = 8, &#39;Test&#39; = 9)](/sql-reference/data-types/enum)) — 条目级别。可能的值：1 或 &#39;Fatal&#39;、2 或 &#39;Critical&#39;、3 或 &#39;Error&#39;、4 或 &#39;Warning&#39;、5 或 &#39;Notice&#39;、6 或 &#39;Information&#39;、7 或 &#39;Debug&#39;、8 或 &#39;Trace&#39;。
* `query_id` ([String](/sql-reference/data-types/string)) — 查询 ID。
* `logger_name` ([LowCardinality(String)](/sql-reference/data-types/lowcardinality)) — 日志记录器名称 (例如 DDLWorker) 。
* `message` ([String](/sql-reference/data-types/string)) — 消息本身。
* `revision` ([UInt32](/sql-reference/data-types/int-uint)) — ClickHouse 修订版本。
* `source_file` ([LowCardinality(String)](/sql-reference/data-types/lowcardinality)) — 记录日志时对应的源文件。
* `source_line` ([UInt64](/sql-reference/data-types/int-uint)) — 记录日志时对应的源代码行号。
* `message_format_string` ([LowCardinality(String)](/sql-reference/data-types/lowcardinality)) — 用于格式化消息的格式字符串。
* `value1` ([String](/sql-reference/data-types/string)) — 用于格式化消息的参数 1。
* `value2` ([String](/sql-reference/data-types/string)) — 用于格式化消息的参数 2。
* `value3` ([String](/sql-reference/data-types/string)) — 用于格式化消息的参数 3。
* `value4` ([String](/sql-reference/data-types/string)) — 用于格式化消息的参数 4。
* `value5` ([String](/sql-reference/data-types/string)) — 用于格式化消息的参数 5。
* `value6` ([String](/sql-reference/data-types/string)) — 用于格式化消息的参数 6。
* `value7` ([String](/sql-reference/data-types/string)) — 用于格式化消息的参数 7。
* `value8` ([String](/sql-reference/data-types/string)) — 用于格式化消息的参数 8。
* `value9` ([String](/sql-reference/data-types/string)) — 用于格式化消息的参数 9。
* `value10` ([String](/sql-reference/data-types/string)) — 用于格式化消息的参数 10。
