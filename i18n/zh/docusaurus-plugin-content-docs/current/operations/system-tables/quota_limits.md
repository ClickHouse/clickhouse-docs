## 描述 \{#description\}

包含所有配额各个时间区间的最大限制信息。一个配额可以对应任意数量的行，也可以为零行。

## 列 \{#columns\}

{/*AUTOGENERATED_START*/ }

* `quota_name` ([String](../../sql-reference/data-types/)) — 配额名称。
* `duration` ([UInt32](../../sql-reference/data-types/)) — 用于计算资源消耗的时间间隔长度，以秒为单位。
* `is_randomized_interval` ([UInt8](../../sql-reference/data-types/)) — 布尔值。表示该时间间隔是否随机化。如果未随机化，则时间间隔始终在固定时间开始。例如，1 分钟的时间间隔总是在整分钟开始 (即可以在 11:20:00 开始，但绝不会在 11:20:01 开始) ；1 天的时间间隔总是在 UTC 午夜开始。如果时间间隔已随机化，则第一个时间间隔会在随机时间开始，之后的时间间隔依次顺延。取值：0 — 时间间隔未随机化，1 — 时间间隔已随机化。
* `max_queries` ([Nullable(UInt64)](../../sql-reference/data-types/)) — 在指定时间段内允许执行的各类查询的最大数量。
* `max_query_selects` ([Nullable(UInt64)](../../sql-reference/data-types/)) — 在指定时间段内允许执行的 SELECT 查询的最大数量。
* `max_query_inserts` ([Nullable(UInt64)](../../sql-reference/data-types/)) — 在指定时间段内允许执行的 INSERT 查询的最大数量。
* `max_errors` ([Nullable(UInt64)](../../sql-reference/data-types/)) — 在指定时间段内允许返回错误的查询最大数量。
* `max_result_rows` ([Nullable(UInt64)](../../sql-reference/data-types/)) — 在指定时间段内所有查询的结果集中允许的总行数上限。
* `max_result_bytes` ([Nullable(UInt64)](../../sql-reference/data-types/)) — 在指定时间段内所有查询的结果集中允许的总字节数上限。
* `max_read_rows` ([Nullable(UInt64)](../../sql-reference/data-types/)) — 在指定时间段内执行所有查询时允许读取的最大行数。
* `max_read_bytes` ([Nullable(UInt64)](../../sql-reference/data-types/)) — 在指定时间段内执行所有查询时允许读取的最大字节数。
* `max_execution_time` ([Nullable(Float64)](../../sql-reference/data-types/)) — 在指定时间段内所有查询允许占用的最大执行时间 (以纳秒为单位)
* `max_written_bytes` ([Nullable(UInt64)](../../sql-reference/data-types/)) — 在指定时间段内执行所有查询时允许写入的最大字节数。
* `max_failed_sequential_authentications` ([Nullable(UInt64)](../../sql-reference/data-types/)) — 在指定时间段内允许的连续身份验证失败最大次数。
* `max_queries_per_normalized_hash` ([Nullable(UInt64)](../../sql-reference/data-types/)) — 在指定时间段内，任一单个规范化查询允许执行的最大次数。
