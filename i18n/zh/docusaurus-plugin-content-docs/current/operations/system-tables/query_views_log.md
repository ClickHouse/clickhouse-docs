## 描述 \{#description\}

包含在运行查询时执行的依赖视图的信息，例如视图类型或执行时间。

要开始记录日志：

1. 在 [query&#95;views&#95;log](../../operations/server-configuration-parameters/settings.md#query_views_log) 部分配置参数。
2. 将 [log&#95;query&#95;views](/operations/settings/settings#log_query_views) 设置为 1。

数据刷新的周期由服务器设置部分 [query&#95;views&#95;log](../../operations/server-configuration-parameters/settings.md#query_views_log) 中的 `flush_interval_milliseconds` 参数指定。要强制刷新，请使用 [SYSTEM FLUSH LOGS](/sql-reference/statements/system#flush-logs) 查询。

ClickHouse 不会自动从该表中删除数据。更多详情参见[简介](/operations/system-tables/overview#system-tables-introduction)。

可以使用 [log&#95;queries&#95;probability](/operations/settings/settings#log_queries_probability)) 设置来减少在 `query_views_log` 表中记录的查询数量。

## 列 \{#columns\}

{/*AUTOGENERATED_START*/ }

* `hostname` ([LowCardinality(String)](/sql-reference/data-types/lowcardinality)) — 执行该查询的服务器主机名。
* `event_date` ([Date](/sql-reference/data-types/date)) — 该视图最后一个事件发生的日期。
* `event_time` ([DateTime](/sql-reference/data-types/datetime)) — 该视图执行完成的日期和时间。
* `event_time_microseconds` ([DateTime64(6)](/sql-reference/data-types/datetime64)) — 该视图执行完成的日期和时间，精确到微秒。
* `view_duration_ms` ([UInt64](/sql-reference/data-types/int-uint)) — 视图执行耗时 (各阶段总和) ，单位为毫秒。
* `initial_query_id` ([String](/sql-reference/data-types/string)) — 初始查询的 ID (用于分布式查询执行) 。
* `view_name` ([String](/sql-reference/data-types/string)) — 视图名称。
* `view_uuid` ([UUID](/sql-reference/data-types/uuid)) — 视图的 UUID。
* `view_type` ([Enum8(&#39;Default&#39; = 1, &#39;Materialized&#39; = 2, &#39;Live&#39; = 3, &#39;Window&#39; = 4)](/sql-reference/data-types/enum)) — 视图类型。取值：&#39;Default&#39; = 1 — 默认视图，不应出现在此日志中；&#39;Materialized&#39; = 2 — materialized view；&#39;Live&#39; = 3 — Live view。
* `view_query` ([String](/sql-reference/data-types/string)) — 该视图执行的查询。
* `view_target` ([String](/sql-reference/data-types/string)) — 视图目标表的名称。
* `read_rows` ([UInt64](/sql-reference/data-types/int-uint)) — 读取的行数。
* `read_bytes` ([UInt64](/sql-reference/data-types/int-uint)) — 读取的字节数。
* `written_rows` ([UInt64](/sql-reference/data-types/int-uint)) — 写入的行数。
* `written_bytes` ([UInt64](/sql-reference/data-types/int-uint)) — 写入的字节数。
* `peak_memory_usage` ([Int64](/sql-reference/data-types/int-uint)) — 此视图上下文中已分配内存与已释放内存之间的最大差值。
* `ProfileEvents` ([Map(LowCardinality(String), UInt64)](/sql-reference/data-types/map)) — 用于度量不同指标的 ProfileEvents。其说明可在 `system.events` 表中找到。
* `status` ([Enum8(&#39;QueryStart&#39; = 1, &#39;QueryFinish&#39; = 2, &#39;ExceptionBeforeStart&#39; = 3, &#39;ExceptionWhileProcessing&#39; = 4)](/sql-reference/data-types/enum)) — 视图状态。取值：&#39;QueryStart&#39; = 1 — 视图执行成功启动，不应出现；&#39;QueryFinish&#39; = 2 — 视图执行成功结束；&#39;ExceptionBeforeStart&#39; = 3 — 在视图开始执行前发生异常；&#39;ExceptionWhileProcessing&#39; = 4 — 在视图执行期间发生异常。
* `exception_code` ([Int32](/sql-reference/data-types/int-uint)) — 异常代码。
* `exception` ([String](/sql-reference/data-types/string)) — 异常消息。
* `stack_trace` ([String](/sql-reference/data-types/string)) — 堆栈跟踪。如果查询已成功完成，则为空字符串。

**別名：**

* `ProfileEvents.Names` — `mapKeys(ProfileEvents)` 的别名。
* `ProfileEvents.Values` — `mapValues(ProfileEvents)` 的别名。

{/*AUTOGENERATED_END*/ }
