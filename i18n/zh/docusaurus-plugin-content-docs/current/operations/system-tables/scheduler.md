# system.scheduler

<SystemTableCloud />

包含本地服务器上[调度节点](/operations/workload-scheduling.md/#hierarchy)的信息和状态。
此表可用于监控。表中每个调度节点对应一行记录。

示例：

```sql
SELECT *
FROM system.scheduler
WHERE resource = 'network_read' AND path = '/prio/fair/prod'
FORMAT Vertical
```

<!--AUTOGENERATED_START-->

- `resource` ([String](../../sql-reference/data-types/)) — 资源名称
- `path` ([String](../../sql-reference/data-types/)) — 资源调度层次结构中调度节点的路径
- `type` ([String](../../sql-reference/data-types/)) — 调度节点的类型。
- `weight` ([Float64](../../sql-reference/data-types/)) — 节点的权重,由 `fair` 类型的父节点使用。
- `priority` ([Int64](../../sql-reference/data-types/)) — 节点的优先级,由 `priority` 类型的父节点使用(数值越低优先级越高)。
- `is_active` ([UInt8](../../sql-reference/data-types/)) — 此节点当前是否处于活动状态 - 具有待出队的资源请求且满足约束条件。
- `active_children` ([UInt64](../../sql-reference/data-types/)) — 处于活动状态的子节点数量。
- `dequeued_requests` ([UInt64](../../sql-reference/data-types/)) — 从此节点出队的资源请求总数。
- `canceled_requests` ([UInt64](../../sql-reference/data-types/)) — 从此节点取消的资源请求总数。
- `dequeued_cost` ([Int64](../../sql-reference/data-types/)) — 从此节点出队的所有请求的成本总和(例如字节大小)。
- `throughput` ([Float64](../../sql-reference/data-types/)) — 当前平均吞吐量(每秒出队成本)。
- `canceled_cost` ([Int64](../../sql-reference/data-types/)) — 从此节点取消的所有请求的成本总和(例如字节大小)。
- `busy_periods` ([UInt64](../../sql-reference/data-types/)) — 此节点停用的总次数。
- `vruntime` ([Nullable(Float64)](../../sql-reference/data-types/)) — 仅适用于 `fair` 节点的子节点。节点的虚拟运行时间,由 SFQ 算法使用,以最大最小公平方式选择下一个要处理的子节点。
- `system_vruntime` ([Nullable(Float64)](../../sql-reference/data-types/)) — 仅适用于 `fair` 节点。显示最后处理的资源请求的 `vruntime` 的虚拟运行时间。在子节点激活期间用作 `vruntime` 的新值。
- `queue_length` ([Nullable(UInt64)](../../sql-reference/data-types/)) — 仅适用于 `fifo` 节点。队列中当前的资源请求数量。
- `queue_cost` ([Nullable(Int64)](../../sql-reference/data-types/)) — 仅适用于 `fifo` 节点。队列中所有请求的成本总和(例如字节大小)。
- `budget` ([Nullable(Int64)](../../sql-reference/data-types/)) — 仅适用于 `fifo` 节点。新资源请求可用的"成本单位"数量。可能在资源请求的估计成本与实际成本不一致时出现(例如读/写失败后)。
- `is_satisfied` ([Nullable(UInt8)](../../sql-reference/data-types/)) — 仅适用于约束节点(例如 `inflight_limit`)。如果此节点的所有约束均已满足,则等于 `1`。
- `inflight_requests` ([Nullable(Int64)](../../sql-reference/data-types/)) — 仅适用于 `inflight_limit` 节点。从此节点出队且当前处于消费状态的资源请求数量。
- `inflight_cost` ([Nullable(Int64)](../../sql-reference/data-types/)) — 仅适用于 `inflight_limit` 节点。从此节点出队且当前处于消费状态的所有资源请求的成本总和(例如字节)。
- `max_requests` ([Nullable(Int64)](../../sql-reference/data-types/)) — 仅适用于 `inflight_limit` 节点。`inflight_requests` 的上限,超过将导致约束违规。
- `max_cost` ([Nullable(Int64)](../../sql-reference/data-types/)) — 仅适用于 `inflight_limit` 节点。`inflight_cost` 的上限,超过将导致约束违规。
- `max_speed` ([Nullable(Float64)](../../sql-reference/data-types/)) — 仅适用于 `bandwidth_limit` 节点。带宽的上限,以每秒令牌数表示。
- `max_burst` ([Nullable(Float64)](../../sql-reference/data-types/)) — 仅适用于 `bandwidth_limit` 节点。令牌桶限流器中可用令牌的上限。
- `throttling_us` ([Nullable(Int64)](../../sql-reference/data-types/)) — 仅适用于 `bandwidth_limit` 节点。此节点处于限流状态的总微秒数。
- `tokens` ([Nullable(Float64)](../../sql-reference/data-types/)) — 仅适用于 `bandwidth_limit` 节点。令牌桶限流器中当前可用的令牌数量。
<!--AUTOGENERATED_END-->
