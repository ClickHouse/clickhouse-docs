## 描述 \{#description\}

包含服务器支持的字典源列表，以及每个源的内置文档。字典源决定字典数据从哪里加载；在 `CREATE DICTIONARY` 查询的 `SOURCE` 子句中指定。

## 列 \{#columns\}

{/*自动生成开始*/ }

* `name` ([String](../../sql-reference/data-types/)) — 字典源的名称，如 SOURCE 子句中所指定。
* `description` ([String](../../sql-reference/data-types/)) — 对字典源用途的概述。
* `syntax` ([String](../../sql-reference/data-types/)) — 用于指定源的 SOURCE 子句结构。请注意，当字典是通过 DDL 查询创建时 (而不是通过服务器配置文件) ，某些源会受到访问控制的限制；详情请参见各个源的 `description`。
* `examples` ([String](../../sql-reference/data-types/)) — 使用示例。
* `introduced_in` ([String](../../sql-reference/data-types/)) — 该源首次引入时对应的 ClickHouse 版本，格式为 major.minor。
* `related` ([Array(String)](../../sql-reference/data-types/)) — 相关字典源的名称。

{/*AUTOGENERATED_END*/ }

## 示例 \{#example\}

```sql title="Query"
SELECT name, syntax
FROM system.dictionary_sources
WHERE name IN ('clickhouse', 'file')
ORDER BY name
```

```text title="Response"
┌─name───────┬─syntax─────────────────────────────────────────────────────────────────────────┐
│ clickhouse │ SOURCE(CLICKHOUSE(host 'host' port 9000 user 'default' password '' db 'db' table 'table')) │
│ file       │ SOURCE(FILE(path '/path/to/file' format 'CSV'))                                  │
└────────────┴────────────────────────────────────────────────────────────────────────────────┘
```
