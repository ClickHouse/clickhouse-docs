包含已执行 drop table 操作但尚未完成数据清理的表的相关信息。

列：

{/*AUTOGENERATED_START*/ }

* `index` ([UInt32](../../sql-reference/data-types/)) — 在 `marked_dropped_tables` 队列中的索引。
* `database` ([String](../../sql-reference/data-types/)) — 数据库名称。
* `table` ([String](../../sql-reference/data-types/)) — 表名。
* `uuid` ([UUID](../../sql-reference/data-types/)) — 表的 UUID。
* `engine` ([String](../../sql-reference/data-types/)) — 表引擎名称。
* `metadata_dropped_path` ([String](../../sql-reference/data-types/)) — 表的元数据文件在 `metadata_dropped` 目录中的路径。
* `table_dropped_time` ([DateTime](../../sql-reference/data-types/)) — 下次尝试删除表数据的预定时间。通常为表被删除时间加上 `database_atomic_delay_before_drop_table_sec`。

{/*AUTOGENERATED_END*/ }

**示例**

下面的示例展示了如何获取 `dropped_tables` 的相关信息。

```sql
SELECT *
FROM system.dropped_tables\G
```

```text
第 1 行:
──────
index:                 0
database:              default
table:                 test
uuid:                  03141bb2-e97a-4d7c-a172-95cc066bb3bd
engine:                MergeTree
metadata_dropped_path: /data/ClickHouse/build/programs/data/metadata_dropped/default.test.03141bb2-e97a-4d7c-a172-95cc066bb3bd.sql
table_dropped_time:    2023-03-16 23:43:31
```
