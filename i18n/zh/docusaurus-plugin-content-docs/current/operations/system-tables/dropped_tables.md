## 描述 \{#description\}

包含已执行 DROP TABLE 语句但尚未完成数据清理的表的信息。

## 列 \{#columns\}

{/*AUTOGENERATED_START*/ }

* `index` ([UInt32](../../sql-reference/data-types/)) — `marked_dropped_tables` 队列中的索引。
* `database` ([String](../../sql-reference/data-types/)) — 数据库名称。
* `table` ([String](../../sql-reference/data-types/)) — 表名。
* `uuid` ([UUID](../../sql-reference/data-types/)) — 表 UUID。
* `engine` ([String](../../sql-reference/data-types/)) — 表引擎名称。
* `metadata_dropped_path` ([String](../../sql-reference/data-types/)) — `metadata_dropped` 目录中该表元数据文件的路径。
* `table_dropped_time` ([DateTime](../../sql-reference/data-types/)) — 计划下一次尝试删除该表数据的时间。通常是该表被删除的时间加上 `database_atomic_delay_before_drop_table_sec`。

{/*AUTOGENERATED_END*/ }

## 示例 \{#example\}

以下示例演示如何获取关于 `dropped_tables` 的信息。

```sql
SELECT *
FROM system.dropped_tables\G
```
