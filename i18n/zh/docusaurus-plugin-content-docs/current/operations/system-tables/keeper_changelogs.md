## 说明 \{#description\}

如果此节点未配置为运行进程内 ClickHouse Keeper，则此表不存在。该表中每个由进程内 Keeper 日志存储跟踪的 Raft 更新日志文件 (`changelog_<from>_<to>.bin[.zstd]`) 对应一行，其中也包括当前正在追加写入的活动文件。

## 列 \{#columns\}

{/*AUTOGENERATED_START*/ }

* `from_log_index` ([UInt64](../../sql-reference/data-types/int-uint.md)) — 文件中的第一个 Raft 日志索引 (含) 。
* `to_log_index` ([UInt64](../../sql-reference/data-types/int-uint.md)) — 文件名所覆盖的最后一个 Raft 日志索引 (含) 。对于活动文件，这是轮转目标，可能会大于 `last_entry_index`。
* `last_entry_index` ([Nullable(UInt64)](../../sql-reference/data-types/int-uint.md)) — 实际追加到该文件中的最大日志索引。如果活动文件尚未接收任何条目，或文件已损坏，则为 `NULL`。
* `entries` ([UInt64](../../sql-reference/data-types/int-uint.md)) — 追加到该文件中的条目数。是 `ifNull(last_entry_index - from_log_index + 1, 0)` 的 `ALIAS`。
* `path` ([String](../../sql-reference/data-types/string.md)) — 磁盘上的文件路径。
* `disk_name` ([String](../../sql-reference/data-types/string.md)) — 存放该文件的磁盘名称。
* `size_bytes` ([UInt64](../../sql-reference/data-types/int-uint.md)) — 文件在磁盘上的大小。
* `modification_time` ([DateTime](../../sql-reference/data-types/datetime.md)) — 文件的最后修改时间。
* `is_compressed` ([Bool](../../sql-reference/data-types/boolean.md)) — 文件载荷已使用 zstd 压缩。
* `active` ([Bool](../../sql-reference/data-types/boolean.md)) — 当前正在向该文件追加内容。
* `is_broken` ([Bool](../../sql-reference/data-types/boolean.md)) — 启动时发现尾部记录已损坏。

{/*AUTOGENERATED_END*/ }

示例：

```sql
SELECT from_log_index, to_log_index, entries, path, active FROM system.keeper_changelogs ORDER BY from_log_index;
```
