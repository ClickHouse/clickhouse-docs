包含记录 AzureQueue 引擎所处理文件信息的日志条目。

可随时安全地截断或删除此表。

列：

{/*AUTOGENERATED_START*/ }

* `hostname` ([LowCardinality(String)](/sql-reference/data-types/lowcardinality)) — 主机名
* `event_date` ([Date](/sql-reference/data-types/date)) — 写入此日志行时的事件日期
* `event_time` ([DateTime](/sql-reference/data-types/datetime)) — 写入此日志行时的事件时间
* `database` ([String](/sql-reference/data-types/string)) — 队列表 (`S3Queue` 或 `AzureQueue`) 所在的数据库名称。
* `table` ([String](/sql-reference/data-types/string)) — 队列表 (`S3Queue` 或 `AzureQueue`) 的名称。
* `uuid` ([String](/sql-reference/data-types/string)) — 队列表 (`S3Queue` 或 `AzureQueue`) 的 UUID。
* `file_name` ([String](/sql-reference/data-types/string)) — 正在处理的文件名。
* `rows_processed` ([UInt64](/sql-reference/data-types/int-uint)) — 已处理的行数。
* `status` ([Enum8(&#39;Processed&#39; = 0, &#39;Failed&#39; = 1)](/sql-reference/data-types/enum)) — 正在处理的文件状态。
* `processing_start_time` ([Nullable(DateTime)](/sql-reference/data-types/nullable)) — 开始处理该文件的时间。
* `processing_end_time` ([Nullable(DateTime)](/sql-reference/data-types/nullable)) — 结束处理该文件的时间。
* `exception` ([String](/sql-reference/data-types/string)) — 如果发生异常，则为异常消息。
* `commit_id` ([UInt64](/sql-reference/data-types/int-uint)) — 提交此文件时所属事务的 ID。
* `commit_time` ([DateTime](/sql-reference/data-types/datetime)) — 在 Keeper 中提交该文件 (标记为失败或已处理) 的时间。
* `transaction_start_time` ([DateTime](/sql-reference/data-types/datetime)) — 整个处理事务开始的时间。
* `get_object_time_ms` ([UInt64](/sql-reference/data-types/int-uint)) — 在对象存储中查找该对象所花费的时间。

{/*AUTOGENERATED_END*/ }
