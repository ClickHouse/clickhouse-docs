## 描述 \{#purpose\}

包含关于 [MergeTree](../../engines/table-engines/mergetree-family/mergetree.md) 表已分离parts的信息。`reason` 列说明该部分被分离的原因。

对于用户手动分离的parts，该列为空。此类parts可以通过 [ALTER TABLE ATTACH PARTITION|PART](/sql-reference/statements/alter/partition#attach-partitionpart) 命令重新附加。

关于其他列的说明，请参阅 [system.parts](../../operations/system-tables/parts.md)。

如果部分名称无效，则某些列的值可能为 `NULL`。此类parts可以通过 [ALTER TABLE DROP DETACHED PART](/sql-reference/statements/alter/partition#drop-detached-partitionpart) 命令删除。

## 列 \{#columns\}

{/*AUTOGENERATED_START*/ }

* `database` ([String](../../sql-reference/data-types/)) — 此dataparts所属数据库的名称。
* `table` ([String](../../sql-reference/data-types/)) — 此dataparts所属表的名称。
* `partition_id` ([Nullable(String)](../../sql-reference/data-types/)) — 此dataparts所属分区的标识符。
* `name` ([String](../../sql-reference/data-types/)) — 此dataparts的名称。
* `bytes_on_disk` ([UInt64](../../sql-reference/data-types/)) — 所有dataparts文件的总大小，以字节为单位。
* `modification_time` ([DateTime](../../sql-reference/data-types/)) — 包含该dataparts的目录的修改时间。通常对应于执行 detach 的时间。
* `disk` ([String](../../sql-reference/data-types/)) — 存储此dataparts的磁盘名称。
* `path` ([String](../../sql-reference/data-types/)) — 此dataparts文件在磁盘上的路径。
* `reason` ([Nullable(String)](../../sql-reference/data-types/)) — 此dataparts被分离的原因。
* `min_block_number` ([Nullable(Int64)](../../sql-reference/data-types/)) — 合并后构成当前dataparts的各dataparts中的最小编号。
* `max_block_number` ([Nullable(Int64)](../../sql-reference/data-types/)) — 合并后构成当前dataparts的各dataparts中的最大编号。
* `level` ([Nullable(UInt32)](../../sql-reference/data-types/)) — 合并树的深度。0 表示当前dataparts是通过 insert 创建的，而不是通过合并其他dataparts创建的。

{/*AUTOGENERATED_END*/ }
