## 描述 \{#description\}

只有在指定了 [part&#95;log](/operations/server-configuration-parameters/settings#part_log) 服务器设置时，才会创建 `system.part_log` 表。

此表包含关于 [MergeTree](../../engines/table-engines/mergetree-family/mergetree.md) 系列表中[数据分片](../../engines/table-engines/mergetree-family/custom-partitioning-key.md)发生的事件的信息，例如数据的添加或合并。

`system.part_log` 表包含以下列：

## 列 \{#columns\}

{/*AUTOGENERATED_START*/ }

* `hostname` ([LowCardinality(String)](/sql-reference/data-types/lowcardinality)) — 执行该查询的服务器主机名。
* `query_id` ([String](/sql-reference/data-types/string)) — 用于创建此数据分区片段的 INSERT 查询的标识符。
* `event_type` ([Enum8(&#39;NewPart&#39; = 1, &#39;MergeParts&#39; = 2, &#39;DownloadPart&#39; = 3, &#39;RemovePart&#39; = 4, &#39;MutatePart&#39; = 5, &#39;MovePart&#39; = 6, &#39;MergePartsStart&#39; = 7, &#39;MutatePartStart&#39; = 8)](/sql-reference/data-types/enum)) — 与数据分区片段相关的事件类型。可以取以下值之一：NewPart — 插入新的数据分区片段，MergePartsStart — 数据分区片段合并已开始，MergeParts — 数据分区片段合并已完成，DownloadPart — 下载数据分区片段，RemovePart — 使用 [DETACH PARTITION](/sql-reference/statements/alter/partition#detach-partitionpart) 删除或分离数据分区片段。MutatePartStart — 数据分区片段变更已开始，MutatePart — 数据分区片段变更已完成，MovePart — 将数据分区片段从一个磁盘移动到另一个磁盘。
* `merge_reason` ([Enum8(&#39;NotAMerge&#39; = 1, &#39;RegularMerge&#39; = 2, &#39;TTLDeleteMerge&#39; = 3, &#39;TTLRecompressMerge&#39; = 4, &#39;TTLDropMerge&#39; = 5)](/sql-reference/data-types/enum)) — 类型为 `MERGE_PARTS` 的事件发生原因。可以具有以下值之一：NotAMerge — 当前事件的类型不是 `MERGE_PARTS`，RegularMerge — 某个常规合并，TTLDeleteMerge、TTLDropMerge — 清理过期数据。TTLRecompressMerge — 对数据分区片段进行重新压缩。
* `merge_algorithm` ([Enum8(&#39;Undecided&#39; = 0, &#39;Vertical&#39; = 1, &#39;Horizontal&#39; = 2)](/sql-reference/data-types/enum)) — 类型为 MERGE&#95;PARTS 的事件所使用的合并算法。可取以下值之一：Undecided、Horizontal、Vertical
* `event_date` ([Date](/sql-reference/data-types/date)) — 事件日期。
* `event_time` ([DateTime](/sql-reference/data-types/datetime)) — 事件时间。
* `event_time_microseconds` ([DateTime64(6)](/sql-reference/data-types/datetime64)) — 精确到微秒的事件时间。
* `duration_ms` ([UInt64](/sql-reference/data-types/int-uint)) — 此操作的耗时。
* `database` ([String](/sql-reference/data-types/string)) — 数据分区片段所在的数据库名称。
* `table` ([String](/sql-reference/data-types/string)) — 数据分区片段所在的表名称。
* `table_uuid` ([UUID](/sql-reference/data-types/uuid)) — 该数据分区片段所属的表的 UUID。
* `part_name` ([String](/sql-reference/data-types/string)) — 数据分区片段名称。
* `partition_id` ([String](/sql-reference/data-types/string)) — 数据分区片段插入到的分区 ID。如果采用 `tuple()` 进行分区，则该列的值为 `all`。
* `partition` ([String](/sql-reference/data-types/string)) — 分区名称。
* `part_type` ([String](/sql-reference/data-types/string)) — part 的类型。可选值：Wide 和 Compact。
* `part_storage_type` ([String](/sql-reference/data-types/string)) — DataPartStorage 的类型。可能值：Packed - 所有文件都存储在单个 blob 中；Full - 每个文件对应一个 blob。
* `disk_name` ([String](/sql-reference/data-types/string)) — 数据分区片段所在的磁盘名称。
* `path_on_disk` ([String](/sql-reference/data-types/string)) — 存放数据分区片段文件的文件夹的绝对路径。
* `rows` ([UInt64](/sql-reference/data-types/int-uint)) — 数据分区片段中的行数。
* `size_in_bytes` ([UInt64](/sql-reference/data-types/int-uint)) — 磁盘上数据分区片段的大小，以字节为单位。
* `merged_from` ([Array(String)](/sql-reference/data-types/array)) — 构成当前 part 的源 parts 名称数组。
* `bytes_uncompressed` ([UInt64](/sql-reference/data-types/int-uint)) — 结果 part 的未压缩大小 (以字节为单位) 。
* `read_rows` ([UInt64](/sql-reference/data-types/int-uint)) — 合并过程中读取的行数。
* `read_bytes` ([UInt64](/sql-reference/data-types/int-uint)) — 在合并期间读取的字节数。
* `peak_memory_usage` ([UInt64](/sql-reference/data-types/int-uint)) — 合并期间使用的 RAM 最大值
* `deduplication_block_ids` ([Array(String)](/sql-reference/data-types/array)) — 插入该 part 时用于去重的块 ID 数组。
* `error` ([UInt16](/sql-reference/data-types/int-uint)) — 该异常的错误代码。
* `exception` ([String](/sql-reference/data-types/string)) — 发生的错误的文本消息。
* `mutation_ids` ([Array(String)](/sql-reference/data-types/array)) — 对于类型为 MUTATE&#95;PART&#95;START 和 MUTATE&#95;PART 的事件，应用于源 part (`merged&#95;from`) 的 mutation ID 数组。
* `ProfileEvents` ([Map(LowCardinality(String), UInt64)](/sql-reference/data-types/map)) — 在此操作期间捕获的所有 profile events。
* `projections_duration_ms` ([Map(LowCardinality(String), UInt64)](/sql-reference/data-types/map)) — 各投影的合并/重建耗时，单位为毫秒。

**别名：**

* `ProfileEvents.Names` — `mapKeys(ProfileEvents)` 的别名。
* `ProfileEvents.Values` — `mapValues(ProfileEvents)` 的别名。
* `name` — `part_name` 的别名。

{/*AUTOGENERATED_END*/ }

在首次向 `MergeTree` 表插入数据后，会创建 `system.part_log` 表。
