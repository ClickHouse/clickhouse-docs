## 描述 \{#description\}

包含当前用户会话参数设置信息。

## 列 \{#columns\}

{/*AUTOGENERATED_START*/ }

* `name` ([String](../../sql-reference/data-types/)) — 设置名称。
* `value` ([String](../../sql-reference/data-types/)) — 设置值。
* `changed` ([UInt8](../../sql-reference/data-types/)) — 显示该设置是否已在配置中显式定义，或被显式修改。
* `description` ([String](../../sql-reference/data-types/)) — 设置的简短说明。
* `min` ([Nullable(String)](../../sql-reference/data-types/)) — 设置的最小值 (如果已通过约束设置) 。如果该设置没有最小值，则包含 NULL。
* `max` ([Nullable(String)](../../sql-reference/data-types/)) — 设置的最大值 (如果已通过约束设置) 。如果该设置没有最大值，则包含 NULL。
* `disallowed_values` ([Array(String)](../../sql-reference/data-types/)) — 不允许的值列表
* `readonly` ([UInt8](../../sql-reference/data-types/)) — 显示当前用户是否可以修改该设置：0 — 当前用户可以修改该设置，1 — 当前用户无法修改该设置。
* `type` ([String](../../sql-reference/data-types/)) — 可分配给该设置的值类型。
* `default` ([String](../../sql-reference/data-types/)) — 设置的默认值。
* `alias_for` ([String](../../sql-reference/data-types/)) — 如果该设置是另一个设置的别名，则为原始设置的名称。
* `is_obsolete` ([UInt8](../../sql-reference/data-types/)) — 显示该设置是否已过时。
* `tier` ([Enum8(&#39;Production&#39; = 0, &#39;Obsolete&#39; = 4, &#39;Experimental&#39; = 8, &#39;Beta&#39; = 12)](../../sql-reference/data-types/)) —
  此功能的支持级别。ClickHouse 功能按等级划分，具体取决于其当前的
  开发状态以及使用时可预期的表现：

- PRODUCTION：该功能稳定、安全可用，并且与其他 PRODUCTION 功能交互时不会出现问题。
- BETA：该功能稳定且安全。与其他功能一起使用时的结果尚不明确，且不保证正确性。欢迎测试并反馈。
- EXPERIMENTAL：该功能仍在开发中。仅适用于开发者和 ClickHouse 爱好者。该功能可能可用，也可能不可用，并且可能随时被移除。
- OBSOLETE：不再受支持。它要么已被移除，要么将在未来版本中被移除。

{/*AUTOGENERATED_END*/ }

## 示例 \{#example\}

下面的示例展示如何获取名称包含 `min_i` 的设置的信息。

```sql
SELECT *
FROM system.settings
WHERE name LIKE '%min_insert_block_size_%'
FORMAT Vertical
```
