## 描述 \{#description\}

`system.histogram_metrics` 的历史数据。每隔 `collect_interval_milliseconds` 采集一次快照，并写入磁盘。

## 列 \{#columns\}

{/*AUTOGENERATED_START*/ }

* `hostname` ([LowCardinality(String)](/sql-reference/data-types/lowcardinality)) — 执行该查询的服务器主机名。
* `event_date` ([Date](/sql-reference/data-types/date)) — 事件日期。
* `event_time` ([DateTime](/sql-reference/data-types/datetime)) — 事件时间。
* `event_time_microseconds` ([DateTime64(6)](/sql-reference/data-types/datetime64)) — 精确到微秒的事件时间。
* `metric` ([LowCardinality(String)](/sql-reference/data-types/lowcardinality)) — 指标名称。
* `labels` ([Map(LowCardinality(String), LowCardinality(String))](/sql-reference/data-types/map)) — 指标标签。
* `histogram` ([Map(Float64, UInt64)](/sql-reference/data-types/map)) — 累积直方图：将桶的上界映射为小于或等于该上界的观测值数量；最终桶包含 +inf。
* `count` ([UInt64](/sql-reference/data-types/int-uint)) — 观测值总数，等于 histogram[+inf]。
* `sum` ([Float64](/sql-reference/data-types/float)) — 所有观测值之和。

{/*AUTOGENERATED_END*/ }

## 示例 \{#example\}

```sql
SELECT event_time, metric, labels, histogram
FROM system.histogram_metric_log
WHERE metric = 'keeper_response_time_ms'
ORDER BY event_time DESC
LIMIT 1
FORMAT Vertical;
```
