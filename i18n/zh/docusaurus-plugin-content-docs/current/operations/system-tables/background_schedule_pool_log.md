## 描述 \{#description\}

仅当指定了 [background&#95;schedule&#95;pool&#95;log](/operations/server-configuration-parameters/settings#background_schedule_pool_log) 服务器设置时，才会创建 `system.background_schedule_pool_log` 表。

该表包含后台调度池中任务执行的历史记录。后台调度池用于执行周期性任务，例如分布式发送、缓冲区刷新以及消息代理操作。

## 列 \{#columns\}

{/*AUTOGENERATED_START*/ }

* `hostname` ([LowCardinality(String)](/sql-reference/data-types/lowcardinality)) — 执行查询的服务器主机名。
* `event_date` ([Date](/sql-reference/data-types/date)) — 事件日期。
* `event_time` ([DateTime](/sql-reference/data-types/datetime)) — 事件时间。
* `event_time_microseconds` ([DateTime64(6)](/sql-reference/data-types/datetime64)) — 精确到微秒的事件时间。
* `query_id` ([String](/sql-reference/data-types/string)) — 与后台任务关联的查询标识符。
* `database` ([LowCardinality(String)](/sql-reference/data-types/lowcardinality)) — 数据库名称。
* `table` ([LowCardinality(String)](/sql-reference/data-types/lowcardinality)) — 表名称。
* `table_uuid` ([UUID](/sql-reference/data-types/uuid)) — 后台任务所属表的 UUID。
* `log_name` ([LowCardinality(String)](/sql-reference/data-types/lowcardinality)) — 后台任务名称。
* `duration_ms` ([UInt64](/sql-reference/data-types/int-uint)) — 任务执行耗时 (毫秒) 。
* `error` ([UInt16](/sql-reference/data-types/int-uint)) — 发生异常时的错误代码。
* `exception` ([String](/sql-reference/data-types/string)) — 发生错误时的文本消息。

{/*AUTOGENERATED_END*/ }

`system.background_schedule_pool_log` 表会在首次执行后台任务后创建。
