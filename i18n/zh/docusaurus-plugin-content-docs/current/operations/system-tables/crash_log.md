## 描述 \{#description\}

包含有关致命错误的堆栈跟踪信息。该表在数据库中默认不存在，仅在发生致命错误时才会创建。

## 列 \{#columns\}

{/*AUTOGENERATED_START*/ }

* `hostname` ([LowCardinality(String)](/sql-reference/data-types/lowcardinality)) — 发生崩溃的主机名。
* `event_date` ([Date](/sql-reference/data-types/date)) — 崩溃发生的日期。
* `event_time` ([DateTime](/sql-reference/data-types/datetime)) — 崩溃发生的时间。
* `timestamp_ns` ([UInt64](/sql-reference/data-types/int-uint)) — 事件的纳秒级时间戳。
* `signal` ([Int32](/sql-reference/data-types/int-uint)) — 信号编号。
* `signal_code` ([Int32](/sql-reference/data-types/int-uint)) — 信号代码。提供有关信号原因的额外上下文。
* `thread_id` ([UInt64](/sql-reference/data-types/int-uint)) — 线程 ID。
* `query_id` ([String](/sql-reference/data-types/string)) — 查询 ID。
* `query` ([String](/sql-reference/data-types/string)) — 崩溃发生时正在执行的查询文本。
* `trace` ([Array(UInt64)](/sql-reference/data-types/array)) — 崩溃瞬间的堆栈跟踪。每个元素都是 ClickHouse 服务端进程内的一个虚拟内存地址。
* `trace_full` ([Array(String)](/sql-reference/data-types/array)) — 崩溃瞬间的堆栈跟踪。每个元素都包含 ClickHouse 服务端进程内一个被调用的方法。
* `fault_address` ([Nullable(UInt64)](/sql-reference/data-types/nullable)) — 导致故障的内存地址。
* `fault_access_type` ([String](/sql-reference/data-types/string)) — 导致故障的内存访问类型 (例如 &#39;read&#39;、&#39;write&#39;) 。
* `signal_description` ([String](/sql-reference/data-types/string)) — 基于 signal&#95;code 的易读描述 (例如 &#39;Address not mapped to object&#39;) 。
* `current_exception_trace_full` ([Array(String)](/sql-reference/data-types/array)) — 当前异常的完整堆栈跟踪，包含符号化后的栈帧 (仅适用于 std::terminate 崩溃) 。
* `version` ([String](/sql-reference/data-types/string)) — ClickHouse 服务端版本。
* `revision` ([UInt32](/sql-reference/data-types/int-uint)) — ClickHouse 服务端修订号。
* `build_id` ([String](/sql-reference/data-types/string)) — 由编译器生成的 Build ID。
* `git_hash` ([String](/sql-reference/data-types/string)) — ClickHouse 源代码的 Git 提交哈希。
* `architecture` ([String](/sql-reference/data-types/string)) — CPU 架构 (例如 x86&#95;64、aarch64) 。

{/*AUTOGENERATED_END*/ }
