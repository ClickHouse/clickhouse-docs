# system.resources \{#systemresources\}

包含位于本地服务器上的[资源](/operations/workload-scheduling.md#workload_entity_storage)的信息。该表中每个资源对应一行。

示例：

```sql
SELECT *
FROM system.resources
FORMAT Vertical
```

```text
Row 1:
──────
name:         io_read
read_disks:   ['s3']
write_disks:  []
create_query: CREATE RESOURCE io_read (READ DISK s3)

Row 2:
──────
name:         io_write
read_disks:   []
write_disks:  ['s3']
create_query: CREATE RESOURCE io_write (WRITE DISK s3)
```

列：

{/*AUTOGENERATED_START*/ }

* `name` ([String](../../sql-reference/data-types/)) — 资源名称。
* `read_disks` ([Array(String)](../../sql-reference/data-types/)) — 使用此资源执行读操作的磁盘名称列表。
* `write_disks` ([Array(String)](../../sql-reference/data-types/)) — 使用此资源执行写操作的磁盘名称列表。
* `unit` ([String](../../sql-reference/data-types/)) — 用于成本计量的资源单位。
* `create_query` ([String](../../sql-reference/data-types/)) — 该资源的 CREATE 查询语句。

{/*AUTOGENERATED_END*/ }
