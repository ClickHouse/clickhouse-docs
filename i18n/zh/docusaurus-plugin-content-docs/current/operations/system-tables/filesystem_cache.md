## 描述 \{#description\}

包含远程对象的文件系统缓存中所有条目的信息。

## 列 \{#columns\}

{/*AUTOGENERATED_START*/ }

* `cache_name` ([String](../../sql-reference/data-types/)) — 缓存对象的名称。可用于 `SYSTEM DESCRIBE FILESYSTEM CACHE <name>`、`SYSTEM DROP FILESYSTEM CACHE <name>` 命令语
* `cache_base_path` ([String](../../sql-reference/data-types/)) — 存储所有缓存文件 (属于由 `cache_name` 标识的缓存) 的基础目录路径。
* `cache_path` ([String](../../sql-reference/data-types/)) — 特定缓存文件的路径，对应源文件中的一个 File 段
* `key` ([String](../../sql-reference/data-types/)) — 该 File 段的缓存键
* `file_segment_range_begin` ([UInt64](../../sql-reference/data-types/)) — 对应 File 段范围起始位置的偏移量
* `file_segment_range_end` ([UInt64](../../sql-reference/data-types/)) — 对应 File 段范围结束位置 (含) 的偏移量
* `size` ([UInt64](../../sql-reference/data-types/)) — File 段的大小
* `state` ([String](../../sql-reference/data-types/)) — File 段状态 (DOWNLOADED、DOWNLOADING、PARTIALLY&#95;DOWNLOADED、...)
* `finished_download_time` ([DateTime](../../sql-reference/data-types/)) — File 段下载完成的时间。
* `cache_hits` ([UInt64](../../sql-reference/data-types/)) — 对应 File 段的缓存命中次数
* `references` ([UInt64](../../sql-reference/data-types/)) — 对应 File 段的引用数量。值为 1 表示当前没有对象在使用它 (唯一现有的引用仅来自缓存存储本身)
* `downloaded_size` ([UInt64](../../sql-reference/data-types/)) — File 段的已下载大小
* `kind` ([String](../../sql-reference/data-types/)) — File 段类型 (用于区分作为 &#39;Temporary data in cache&#39; 一部分添加的 File 段)
* `unbound` ([UInt8](../../sql-reference/data-types/)) — 内部实现标志
* `user_id` ([String](../../sql-reference/data-types/)) — 创建该 File 段的用户 ID
* `segment_type` ([String](../../sql-reference/data-types/)) — 分段的类型。用于区分数据文件 (`.json`、`.txt` 等) 和数据文件 (`.bin`、mark 文件) 。
* `file_size` ([Nullable(UInt64)](../../sql-reference/data-types/)) — 当前 File 段所属文件的文件大小

{/*AUTOGENERATED_END*/ }
