## 描述 \{#description\}

包含 `BACKUP` 和 `RESTORE` 操作相关信息的日志记录。

## 列 \{#columns\}

{/*AUTOGENERATED_START*/ }

* `hostname` ([LowCardinality(String)](/sql-reference/data-types/lowcardinality)) — 执行该查询的服务器主机名。
* `event_date` ([Date](/sql-reference/data-types/date)) — 条目的日期。
* `event_time` ([DateTime](/sql-reference/data-types/datetime)) — 条目的时间。
* `event_time_microseconds` ([DateTime64(6)](/sql-reference/data-types/datetime64)) — 精确到微秒的条目时间。
* `id` ([String](/sql-reference/data-types/string)) — 备份或恢复操作的标识符。
* `name` ([String](/sql-reference/data-types/string)) — 备份存储的名称 (即 FROM 或 TO 子句中的内容) 。
* `base_backup_name` ([String](/sql-reference/data-types/string)) — 增量备份时的基础备份名称。
* `query_id` ([String](/sql-reference/data-types/string)) — 与备份操作关联的查询 ID。
* `status` ([Enum8(&#39;CREATING&#95;BACKUP&#39; = 0, &#39;BACKUP&#95;CREATED&#39; = 1, &#39;BACKUP&#95;FAILED&#39; = 2, &#39;RESTORING&#39; = 3, &#39;RESTORED&#39; = 4, &#39;RESTORE&#95;FAILED&#39; = 5, &#39;BACKUP&#95;CANCELLED&#39; = 6, &#39;RESTORE&#95;CANCELLED&#39; = 7)](/sql-reference/data-types/enum)) — 操作状态。
* `error` ([String](/sql-reference/data-types/string)) — 操作失败时的错误消息 (成功操作则为空字符串) 。
* `start_time` ([DateTime64(6)](/sql-reference/data-types/datetime64)) — 操作开始时间。
* `end_time` ([DateTime64(6)](/sql-reference/data-types/datetime64)) — 操作结束时间。
* `num_files` ([UInt64](/sql-reference/data-types/int-uint)) — 备份中存储的文件数。
* `total_size` ([UInt64](/sql-reference/data-types/int-uint)) — 备份中存储文件的总大小。
* `num_entries` ([UInt64](/sql-reference/data-types/int-uint)) — 备份中的条目数。也就是说，如果备份以文件夹形式存储，则为文件夹中的文件数；如果备份以归档形式存储，则为归档中的文件数。如果是增量备份，或者包含空文件或重复文件，则该值与 `num_files` 不同。以下关系始终成立：`num_entries ≤ num_files`。
* `uncompressed_size` ([UInt64](/sql-reference/data-types/int-uint)) — 备份的未压缩大小。
* `compressed_size` ([UInt64](/sql-reference/data-types/int-uint)) — 备份的压缩大小。如果备份不是以归档形式存储，则其值等于 uncompressed&#95;size。
* `files_read` ([UInt64](/sql-reference/data-types/int-uint)) — 恢复操作期间读取的文件数。
* `bytes_read` ([UInt64](/sql-reference/data-types/int-uint)) — 恢复操作期间读取文件的总大小。

{/*AUTOGENERATED_END*/ }
