## 描述 \{#description\}

包含已执行查询的 [trace span](https://opentracing.io/docs/overview/spans/) 相关信息。

## 列 \{#columns\}

{/*AUTOGENERATED_START*/ }

* `hostname` ([LowCardinality(String)](/sql-reference/data-types/lowcardinality)) — 捕获此 span 的主机名。
* `trace_id` ([UUID](/sql-reference/data-types/uuid)) — 已执行查询的 trace ID。
* `span_id` ([UInt64](/sql-reference/data-types/int-uint)) — trace span 的 ID。
* `parent_span_id` ([UInt64](/sql-reference/data-types/int-uint)) — 父 trace span 的 ID。
* `operation_name` ([LowCardinality(String)](/sql-reference/data-types/lowcardinality)) — 操作名称。
* `kind` ([Enum8(&#39;INTERNAL&#39; = 0, &#39;SERVER&#39; = 1, &#39;CLIENT&#39; = 2, &#39;PRODUCER&#39; = 3, &#39;CONSUMER&#39; = 4)](/sql-reference/data-types/enum)) — 该 span 的 SpanKind。INTERNAL — 表示该 span 代表应用程序内部操作。SERVER — 表示该 span 涵盖对同步 RPC 或其他远程请求的服务器端处理。CLIENT — 表示该 span 描述向某个远程服务发出的请求。PRODUCER — 表示该 span 描述异步请求的发起方。这个父 span 通常会在对应的子 CONSUMER span 结束之前结束，甚至可能早于子 span 开始之前就已结束。CONSUMER - 表示该 span 描述异步 PRODUCER 请求的子级。
* `start_time_us` ([UInt64](/sql-reference/data-types/int-uint)) — trace span 的开始时间 (以微秒为单位) 。
* `finish_time_us` ([UInt64](/sql-reference/data-types/int-uint)) — trace span 的结束时间 (以微秒为单位) 。
* `finish_date` ([Date](/sql-reference/data-types/date)) — trace span 的结束日期。
* `status_code` ([Enum8(&#39;UNSET&#39; = 0, &#39;OK&#39; = 1, &#39;ERROR&#39; = 2)](/sql-reference/data-types/enum)) — span 的状态码。
* `status_message` ([LowCardinality(String)](/sql-reference/data-types/lowcardinality)) — 错误消息。
* `attribute` ([Map(LowCardinality(String), String)](/sql-reference/data-types/map)) — trace span 的属性。它们会根据 OpenTelemetry 标准中的建议进行填充。

**別名：**

* `attribute.names` — `mapKeys(attribute)` 的别名。
* `attribute.values` — `mapValues(attribute)` 的别名。

{/*AUTOGENERATED_END*/ }
