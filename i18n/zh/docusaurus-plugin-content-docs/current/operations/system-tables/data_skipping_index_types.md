## 说明 \{#description\}

包含服务器支持的数据跳过索引类型列表，以及每种类型的内置文档。数据跳过索引类型在 `CREATE TABLE` 查询中 `INDEX` 声明的 `TYPE` 里指定，可让 ClickHouse 跳过无法匹配查询条件的粒度。

请注意，此表列出的是可用的索引*类型*，而 [`system.data_skipping_indices`](/operations/system-tables/data_skipping_indices) 列出的是在现有表上定义的索引实例。

## 列 \{#columns\}

{/*AUTOGENERATED_START*/ }

* `name` ([String](../../sql-reference/data-types/)) — 数据跳过索引类型的名称，即 `INDEX` 声明中 `TYPE` 指定的名称。
* `description` ([String](../../sql-reference/data-types/)) — 对该数据跳过索引类型作用的概述。
* `syntax` ([String](../../sql-reference/data-types/)) — 在 `CREATE TABLE` 查询的 `INDEX` 子句中声明该索引的语法。
* `examples` ([String](../../sql-reference/data-types/)) — 使用示例。
* `introduced_in` ([String](../../sql-reference/data-types/)) — 首次引入该索引类型的 ClickHouse 版本，格式为 major.minor。
* `related` ([Array(String)](../../sql-reference/data-types/)) — 相关数据跳过索引类型的名称。

{/*AUTOGENERATED_END*/ }

## 示例 \{#example\}

```sql title="Query"
SELECT name, syntax
FROM system.data_skipping_index_types
WHERE name IN ('minmax', 'set')
ORDER BY name
```

```text title="Response"
┌─name───┬─syntax───────────────────────────────────────┐
│ minmax │ INDEX name expr TYPE minmax GRANULARITY n     │
│ set    │ INDEX name expr TYPE set(max_rows) GRANULARITY n │
└────────┴──────────────────────────────────────────────┘
```
