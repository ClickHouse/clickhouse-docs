## 说明 \{#description\}

包含有关[配额](../../operations/system-tables/quotas.md)的信息。

## 列 \{#columns\}

{/*AUTOGENERATED_START*/ }

* `name` ([String](../../sql-reference/data-types/)) — 配额名称。
* `id` ([UUID](../../sql-reference/data-types/)) — 配额 ID。
* `storage` ([String](../../sql-reference/data-types/)) — 配额的存储位置。可能的值：如果配额是在 users.xml 文件中配置的，则为 &#39;users.xml&#39;；如果配额是通过 SQL 查询配置的，则为 &#39;disk&#39;。
* `keys` ([Array(Enum8(&#39;user&#95;name&#39; = 1, &#39;ip&#95;address&#39; = 2, &#39;forwarded&#95;ip&#95;address&#39; = 3, &#39;client&#95;key&#39; = 4, &#39;normalized&#95;query&#95;hash&#39; = 7))](../../sql-reference/data-types/)) — 键指定配额的共享方式。如果两个连接使用相同的配额和键，则它们共享相同数量的资源。取值：[] — 所有用户共享同一个配额，[&#39;user&#95;name&#39;] — 用户名相同的连接共享同一个配额，[&#39;ip&#95;address&#39;] — 来自同一 IP 的连接共享同一个配额，[&#39;client&#95;key&#39;] — 具有相同键的连接共享同一个配额。键必须由客户端显式提供。使用 clickhouse-client 时，在 --quota&#95;key 参数中传入键值，或在客户端设置文件中使用 quota&#95;key 参数。使用 HTTP 接口时，使用 X-ClickHouse-Quota 头部信息，[&#39;user&#95;name&#39;, &#39;client&#95;key&#39;] — 具有相同 client&#95;key 的连接共享同一个配额。如果客户端未提供键，则会按 `user_name` 跟踪配额，[&#39;client&#95;key&#39;, &#39;ip&#95;address&#39;] — 具有相同 client&#95;key 的连接共享同一个配额。如果客户端未提供键，则会按 ip&#95;address 跟踪配额。
* `durations` ([Array(UInt32)](../../sql-reference/data-types/)) — 以秒为单位的时间间隔长度。
* `apply_to_all` ([UInt8](../../sql-reference/data-types/)) — 逻辑值。显示配额应用于哪些用户。取值：0 — 配额应用于 apply&#95;to&#95;list 中指定的用户。1 — 配额应用于除 apply&#95;to&#95;except 中列出的用户外的所有用户。
* `apply_to_list` ([Array(String)](../../sql-reference/data-types/)) — 应用此配额的用户名/角色列表。
* `apply_to_except` ([Array(String)](../../sql-reference/data-types/)) — 不应用此配额的用户名/角色列表。
