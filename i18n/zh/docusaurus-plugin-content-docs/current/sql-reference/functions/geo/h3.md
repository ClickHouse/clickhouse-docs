## H3 索引 {#h3-index}

[H3](https://h3geo.org/) 是一种地理索引系统,将地球表面划分为均匀的六边形网格单元。该系统采用层级结构,即顶层的每个六边形("父级")可以分割为七个更小但同样均匀的六边形("子级"),以此类推。

层级的级别称为 `resolution`(分辨率),取值范围为 `0` 到 `15`,其中 `0` 是 `base`(基础)级别,包含最大且粒度最粗的单元。

经纬度坐标对可以转换为 64 位 H3 索引,用于标识网格单元。

H3 索引主要用于位置分桶和其他地理空间操作。

H3 系统的完整说明可在 [Uber Engineering 网站](https://www.uber.com/blog/h3/)查阅。


## h3IsValid {#h3isvalid}

验证数字是否为有效的 [H3](#h3-index) 索引。

**语法**

```sql
h3IsValid(h3index)
```

**参数**

- `h3index` — 六边形索引数值。[UInt64](../../data-types/int-uint.md)。

**返回值**

- 1 — 该数字是有效的 H3 索引。[UInt8](../../data-types/int-uint.md)。
- 0 — 该数字不是有效的 H3 索引。[UInt8](../../data-types/int-uint.md)。

**示例**

查询：

```sql
SELECT h3IsValid(630814730351855103) AS h3IsValid;
```

结果：

```text
┌─h3IsValid─┐
│         1 │
└───────────┘
```


## h3GetResolution {#h3getresolution}

返回给定 [H3](#h3-index) 索引的分辨率。

**语法**

```sql
h3GetResolution(h3index)
```

**参数**

- `h3index` — 六边形索引编号。[UInt64](../../data-types/int-uint.md)。

**返回值**

- 索引分辨率。范围:`[0, 15]`。[UInt8](../../data-types/int-uint.md)。
- 如果索引无效,函数返回随机值。使用 [h3IsValid](#h3isvalid) 验证索引。[UInt8](../../data-types/int-uint.md)。

**示例**

查询:

```sql
SELECT h3GetResolution(639821929606596015) AS resolution;
```

结果:

```text
┌─resolution─┐
│         14 │
└────────────┘
```

## geoToH3 {#geotoh3}

返回指定分辨率的 [H3](#h3-index) 点索引 `(lat, lon)`。

**语法**

```sql
geoToH3(lat, lon, resolution)
```

**参数**

- `lat` — 纬度。类型：[Float64](../../data-types/float.md)。
- `lon` — 经度。类型：[Float64](../../data-types/float.md)。
- `resolution` — 索引分辨率。取值范围：`[0, 15]`。类型：[UInt8](../../data-types/int-uint.md)。

**返回值**

- 六边形索引编号。类型：[UInt64](../../data-types/int-uint.md)。
- 发生错误时返回 0。类型：[UInt64](../../data-types/int-uint.md)。

注意：在 ClickHouse v25.4 及更早版本中，`geoToH3()` 按 `(lon, lat)` 顺序接收参数值。从 ClickHouse v25.5 开始，输入值的顺序为 `(lat, lon)`。可以通过设置 `geotoh3_argument_order = 'lon_lat'` 来恢复之前的行为。

**示例**

查询：

```sql
SELECT geoToH3(55.71290588, 37.79506683, 15) AS h3Index;
```

结果：

```text
┌────────────h3Index─┐
│ 644325524701193974 │
└────────────────────┘
```

注意:在 ClickHouse v24.12 及更早版本中,`h3ToGeo()` 按 `(lon, lat)` 顺序返回值。从 ClickHouse v25.1 开始,返回值顺序为 `(lat, lon)`。可以通过设置 `h3togeo_lon_lat_result_order = true` 来恢复之前的行为。

**示例**

查询:

```sql
SELECT h3ToGeo(644325524701193974) AS coordinates;
```

结果:

```text
┌─coordinates───────────────────────────┐
│ (55.71290243145668,37.79506616830252) │
└───────────────────────────────────────┘
```

## h3PolygonToCells

返回给定几何对象（环或（多）多边形）在指定分辨率下所包含的六边形单元。

**语法**

```sql
h3PolygonToCells(geometry, resolution)
```

**参数**

* `geometry` 可以是以下任一[地理数据类型](../../data-types/geo.md)或其对应的基础类型：
  * [Ring](../../data-types/geo.md#ring)
  * [Polygon](../../data-types/geo.md#polygon)
  * [MultiPolygon](../../data-types/geo.md#multipolygon)
* `resolution` — 索引精度。取值范围：`[0, 15]`。[UInt8](../../data-types/int-uint.md)。

**返回值**

* 所包含的 H3 索引数组。[Array](../../data-types/array.md)([UInt64](../../data-types/int-uint.md))。

**示例**

查询：

```sql
SELECT h3PolygonToCells([(-122.4089866999972145,37.813318999983238),(-122.3544736999993603,37.7198061999978478),(-122.4798767000009008,37.8151571999998453)], 7) AS h3index;
```

结果：

```text

│ 608692971927240703 │
│ 608692970585063423 │
│ 608692970819944447 │
│ 608692970719281151 │
```
