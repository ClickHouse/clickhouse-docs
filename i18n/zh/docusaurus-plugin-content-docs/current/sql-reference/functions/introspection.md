# 内省函数

您可以使用本章介绍的函数对 [ELF](https://en.wikipedia.org/wiki/Executable_and_Linkable_Format) 和 [DWARF](https://en.wikipedia.org/wiki/DWARF) 进行内省,以实现查询性能分析。

:::note
这些函数运行速度较慢,且可能存在安全隐患。
:::

为确保内省函数正常运行:

- 安装 `clickhouse-common-static-dbg` 软件包。

- 将 [allow_introspection_functions](../../operations/settings/settings.md#allow_introspection_functions) 设置为 1。

        出于安全考虑,内省函数默认处于禁用状态。

ClickHouse 会将性能分析报告保存到 [trace_log](/operations/system-tables/trace_log) 系统表中。请确保该表和性能分析器配置正确。

## demangle {#demangle}

引入版本:v20.1

将符号转换为 C++ 函数名。
该符号通常由 `addressToSymbol` 函数返回。

**语法**

```sql
demangle(symbol)
```

**参数**

- `symbol` — 来自目标文件的符号。[`String`](/sql-reference/data-types/string)

**返回值**

返回 C++ 函数名,如果符号无效则返回空字符串。[`String`](/sql-reference/data-types/string)

**示例**

```sql title=Query
SELECT * FROM system.trace_log LIMIT 1 \G;
```

## isMergeTreePartCoveredBy {#isMergeTreePartCoveredBy}

引入版本:v25.6

该函数用于检查第一个参数的数据分片是否被第二个参数的数据分片所覆盖。

**语法**

```sql
isMergeTreePartCoveredBy(nested_part, covering_part)
```

**参数**

- `nested_part` — 预期被嵌套的数据分片名称。[`String`](/sql-reference/data-types/string)
- `covering_part` — 预期覆盖的数据分片名称。[`String`](/sql-reference/data-types/string)

**返回值**

如果被覆盖则返回 `1`,否则返回 `0`。[`UInt8`](/sql-reference/data-types/int-uint)

**示例**

**基本示例**

```sql title=查询
WITH 'all_12_25_7_4' AS lhs, 'all_7_100_10_20' AS rhs
SELECT isMergeTreePartCoveredBy(rhs, lhs), isMergeTreePartCoveredBy(lhs, rhs);
```

```response title=响应
┌─isMergeTreePartCoveredBy(rhs, lhs)─┬─isMergeTreePartCoveredBy(lhs, rhs)─┐
│                                  0 │                                  1 │
└────────────────────────────────────┴────────────────────────────────────┘
```


## logTrace {#logTrace}

引入版本：v20.12

为每个 [Block](/development/architecture/#block) 向服务器日志输出跟踪日志消息。

**语法**

```sql
logTrace(message)
```

**参数**

- `message` — 输出到服务器日志的消息。[`const String`](/sql-reference/data-types/string)

**返回值**

始终返回 `0`。[`UInt8`](/sql-reference/data-types/int-uint)

**示例**

**基本示例**

```sql title=Query
SELECT logTrace('logTrace message');
```

```response title=Response
┌─logTrace('logTrace message')─┐
│                            0 │
└──────────────────────────────┘
```


## mergeTreePartInfo {#mergeTreePartInfo}

引入版本：v25.6

该函数用于从 `MergeTree` 数据分片名称中提取有用的值。

**语法**

```sql
mergeTreePartInfo(part_name)
```

**参数**

- `part_name` — 要解析的数据分片名称。[`String`](/sql-reference/data-types/string)

**返回值**

返回一个包含以下子列的元组：`partition_id`、`min_block`、`max_block`、`level`、`mutation`。[`Tuple`](/sql-reference/data-types/tuple)

**示例**

**基本示例**

```sql title=查询
WITH mergeTreePartInfo('all_12_25_7_4') AS info
SELECT info.partition_id, info.min_block, info.max_block, info.level, info.mutation;
```

```response title=响应
┌─info.partition_id─┬─info.min_block─┬─info.max_block─┬─info.level─┬─info.mutation─┐
│ all               │             12 │             25 │          7 │             4 │
└───────────────────┴────────────────┴────────────────┴────────────┴───────────────┘
```

**返回值**

**示例**

**使用示例**

```sql title=查询
SELECT tid();
```
