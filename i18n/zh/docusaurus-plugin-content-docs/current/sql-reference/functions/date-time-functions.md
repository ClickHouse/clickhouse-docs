# 日期和时间函数

本节中的大多数函数接受可选的时区参数,例如 `Europe/Amsterdam`。此时将使用指定的时区,而非本地(默认)时区。

**示例**

```sql
SELECT
    toDateTime('2016-06-15 23:00:00') AS time,
    toDate(time) AS date_local,
    toDate(time, 'Asia/Yekaterinburg') AS date_yekat,
    toString(time, 'US/Samoa') AS time_samoa
```

```text
┌────────────────time─┬─date_local─┬─date_yekat─┬─time_samoa──────────┐
│ 2016-06-15 23:00:00 │ 2016-06-15 │ 2016-06-16 │ 2016-06-15 09:00:00 │
└─────────────────────┴────────────┴────────────┴─────────────────────┘
```

## UTCTimestamp {#UTCTimestamp}

引入版本：v22.11

返回查询分析时的当前日期和时间。该函数是一个常量表达式。

此函数返回与 `now('UTC')` 相同的结果。它仅为支持 MySQL 而添加。建议使用 [`now`](#now)。

**语法**

```sql
UTCTimestamp()
```

**别名**：`UTC_timestamp`

**参数**

- 无。

**返回值**

返回查询分析时的当前日期和时间。[`DateTime`](/sql-reference/data-types/datetime)

**示例**

**获取当前 UTC 时间戳**

```sql title=查询
SELECT UTCTimestamp()
```

```response title=响应
┌──────UTCTimestamp()─┐
│ 2024-05-28 08:32:09 │
└─────────────────────┘
```

## addInterval {#addInterval}

引入版本:v22.11

将一个时间间隔添加到另一个时间间隔或时间间隔元组。

:::note
相同类型的时间间隔将合并为单个时间间隔。例如,如果传入 `toIntervalDay(1)` 和 `toIntervalDay(2)`,则结果为 `(3)` 而非 `(1,1)`。
:::

**语法**

```sql
addInterval(interval_1, interval_2)
```

**参数**

- `interval_1` — 第一个时间间隔或时间间隔元组。[`Interval`](/sql-reference/data-types/int-uint) 或 [`Tuple(Interval)`](/sql-reference/data-types/tuple)
- `interval_2` — 要添加的第二个时间间隔。[`Interval`](/sql-reference/data-types/int-uint)

**返回值**

返回时间间隔元组 [`Tuple(Interval)`](/sql-reference/data-types/tuple)

**示例**

**添加时间间隔**

```sql title=Query
SELECT addInterval(INTERVAL 1 DAY, INTERVAL 1 MONTH);
SELECT addInterval((INTERVAL 1 DAY, INTERVAL 1 YEAR), INTERVAL 1 MONTH);
SELECT addInterval(INTERVAL 2 DAY, INTERVAL 1 DAY)
```

```response title=Response
┌─addInterval(toIntervalDay(1), toIntervalMonth(1))─┐
│ (1,1)                                             │
└───────────────────────────────────────────────────┘
┌─addInterval((toIntervalDay(1), toIntervalYear(1)), toIntervalMonth(1))─┐
│ (1,1,1)                                                                │
└────────────────────────────────────────────────────────────────────────┘
┌─addInterval(toIntervalDay(2), toIntervalDay(1))─┐
│ (3)                                             │
└─────────────────────────────────────────────────┘
```

## age {#age}

引入版本:v23.1

返回 `startdate` 和 `enddate` 之间差值的指定单位分量。
差值计算精度为 1 纳秒。

例如,2021-12-29 和 2022-01-01 之间的差值,按天单位计算为 3 天,
按月单位计算为 0 个月,按年单位计算为 0 年。

如需 age 的替代函数,请参阅 [`dateDiff`](#dateDiff)。

**Syntax**

```sql
age('unit', startdate, enddate[, timezone])
```

**参数**

- `unit` — 结果的时间间隔单位。

| 单位        | 可能的值                                  |
| ----------- | ---------------------------------------- |
| nanosecond  | `nanosecond`, `nanoseconds`, `ns`        |
| microsecond | `microsecond`, `microseconds`, `us`, `u` |
| millisecond | `millisecond`, `milliseconds`, `ms`      |
| second      | `second`, `seconds`, `ss`, `s`           |
| minute      | `minute`, `minutes`, `mi`, `n`           |
| hour        | `hour`, `hours`, `hh`, `h`               |
| day         | `day`, `days`, `dd`, `d`                 |
| week        | `week`, `weeks`, `wk`, `ww`              |
| month       | `month`, `months`, `mm`, `m`             |
| quarter     | `quarter`, `quarters`, `qq`, `q`         |
| year        | `year`, `years`, `yyyy`, `yy`            |

- `startdate` — 起始时间值(被减数)。[`Date`](/sql-reference/data-types/date) 或 [`Date32`](/sql-reference/data-types/date32) 或 [`DateTime`](/sql-reference/data-types/datetime) 或 [`DateTime64`](/sql-reference/data-types/datetime64)
- `enddate` — 结束时间值(减数)。[`Date`](/sql-reference/data-types/date) 或 [`Date32`](/sql-reference/data-types/date32) 或 [`DateTime`](/sql-reference/data-types/datetime) 或 [`DateTime64`](/sql-reference/data-types/datetime64)
- `timezone` — 可选。时区名称。如果指定,将同时应用于 startdate 和 enddate。如果未指定,则使用 startdate 和 enddate 各自的时区。如果两者时区不同,结果将不确定。[`String`](/sql-reference/data-types/string)

**返回值**

返回以指定单位表示的 enddate 和 startdate 之间的差值。[`Int32`](/sql-reference/data-types/int-uint)

**示例**

**计算以小时为单位的时间差**

```sql title=Query
SELECT age('hour', toDateTime('2018-01-01 22:30:00'), toDateTime('2018-01-02 23:00:00'))
```

```response title=Response
┌─age('hour', toDateTime('2018-01-01 22:30:00'), toDateTime('2018-01-02 23:00:00'))─┐
│                                                                                24 │
└───────────────────────────────────────────────────────────────────────────────────┘
```

**计算不同单位的时间差**

```sql title=Query
SELECT
    toDate('2022-01-01') AS e,
    toDate('2021-12-29') AS s,
    age('day', s, e) AS day_age,
    age('month', s, e) AS month_age,
    age('year', s, e) AS year_age
```

```response title=Response
┌──────────e─┬──────────s─┬─day_age─┬─month_age─┬─year_age─┐
│ 2022-01-01 │ 2021-12-29 │       3 │         0 │        0 │
└────────────┴────────────┴─────────┴───────────┴──────────┘
```

## dateDiff {#dateDiff}

引入版本:v23.4

返回 `startdate` 和 `enddate` 之间跨越的指定 `unit` 边界的数量。
差值使用相对单位计算。例如,2021-12-29 和 2022-01-01 之间,对于 day 单位的差值是 3 天
(参见 [`toRelativeDayNum`](#toRelativeDayNum)),对于 month 单位的差值是 1 个月(参见 [`toRelativeMonthNum`](#toRelativeMonthNum)),对于 year 单位的差值是 1 年
(参见 [`toRelativeYearNum`](#toRelativeYearNum))。

如果指定了 `week` 单位,则 `dateDiff` 假定周从星期一开始。
请注意,此行为与函数 `toWeek()` 不同,后者默认周从星期日开始。

作为 `dateDiff` 的替代方案,请参见函数 [`age`](#age)。

**语法**

```sql
dateDiff(unit, startdate, enddate[, timezone])
```

**别名**:`timestampDiff`、`TIMESTAMP_DIFF`、`DATE_DIFF`、`date_diff`、`timestamp_diff`

**参数**

- `unit` — 结果的时间间隔类型。

| 单位        | 可能的值                                  |
| ----------- | ---------------------------------------- |
| nanosecond  | `nanosecond`、`nanoseconds`、`ns`        |
| microsecond | `microsecond`、`microseconds`、`us`、`u` |
| millisecond | `millisecond`、`milliseconds`、`ms`      |
| second      | `second`、`seconds`、`ss`、`s`           |
| minute      | `minute`、`minutes`、`mi`、`n`           |
| hour        | `hour`、`hours`、`hh`、`h`               |
| day         | `day`、`days`、`dd`、`d`                 |
| week        | `week`、`weeks`、`wk`、`ww`              |
| month       | `month`、`months`、`mm`、`m`             |
| quarter     | `quarter`、`quarters`、`qq`、`q`         |
| year        | `year`、`years`、`yyyy`、`yy`            |

- `startdate` — 第一个时间值(被减数)。[`Date`](/sql-reference/data-types/date) 或 [`Date32`](/sql-reference/data-types/date32) 或 [`DateTime`](/sql-reference/data-types/datetime) 或 [`DateTime64`](/sql-reference/data-types/datetime64)
- `enddate` — 第二个时间值(减数)。[`Date`](/sql-reference/data-types/date) 或 [`Date32`](/sql-reference/data-types/date32) 或 [`DateTime`](/sql-reference/data-types/datetime) 或 [`DateTime64`](/sql-reference/data-types/datetime64)
- `timezone` — 可选。时区名称。如果指定,则同时应用于 `startdate` 和 `enddate`。如果未指定,则使用 `startdate` 和 `enddate` 各自的时区。如果两者时区不同,则结果未定义。[`String`](/sql-reference/data-types/string)

**返回值**

返回以 `unit` 表示的 `enddate` 和 `startdate` 之间的差值。[`Int64`](/sql-reference/data-types/int-uint)

**示例**

**计算以小时为单位的日期差值**

```sql title=查询
SELECT dateDiff('hour', toDateTime('2018-01-01 22:00:00'), toDateTime('2018-01-02 23:00:00')) AS res
```

```response title=响应
┌─res─┐
│  25 │
└─────┘
```

**计算不同单位的日期差值**

```sql title=查询
SELECT
    toDate('2022-01-01') AS e,
    toDate('2021-12-29') AS s,
```

此函数等同于 `dateDiff('second', startdate, enddate)`。

如需计算其他单位(小时、天、月等)的时间差,请改用 [`dateDiff`](#dateDiff) 函数。

**语法**

```sql
timeDiff(startdate, enddate)
```

**参数**

- `startdate` — 第一个时间值(减数)。[`Date`](/sql-reference/data-types/date) 或 [`Date32`](/sql-reference/data-types/date32) 或 [`DateTime`](/sql-reference/data-types/datetime) 或 [`DateTime64`](/sql-reference/data-types/datetime64)
- `enddate` — 第二个时间值(被减数)。[`Date`](/sql-reference/data-types/date) 或 [`Date32`](/sql-reference/data-types/date32) 或 [`DateTime`](/sql-reference/data-types/datetime) 或 [`DateTime64`](/sql-reference/data-types/datetime64)

**返回值**

返回 `enddate` 和 `startdate` 之间的差值,以秒为单位。[`Int64`](/sql-reference/data-types/int-uint)

**示例**

**计算以秒为单位的时间差**

```sql title=查询
SELECT timeDiff(toDateTime('2018-01-01 22:00:00'), toDateTime('2018-01-02 23:00:00')) AS res
```

```response title=响应
┌───res─┐
│ 90000 │
└───────┘
```

**计算时间差并转换为小时**

```sql title=查询
SELECT timeDiff(toDateTime('2018-01-01 22:00:00'), toDateTime('2018-01-02 23:00:00')) / 3600 AS hours
```

```response title=响应
┌─hours─┐
│    25 │
└───────┘
```

**等同于使用秒单位的 dateDiff**

```sql title=查询
SELECT
    timeDiff(toDateTime('2021-12-29'), toDateTime('2022-01-01')) AS time_diff_result,
    dateDiff('second', toDateTime('2021-12-29'), toDateTime('2022-01-01')) AS date_diff_result
```

```response title=响应
┌─time_diff_result─┬─date_diff_result─┐
│           259200 │           259200 │
└──────────────────┴──────────────────┘
```

## timeSlots {#timeSlots}

引入版本:v1.1

对于从 `StartTime` 开始并持续 `Duration` 秒的时间间隔,该函数返回一个时间点数组,数组由该间隔内按 `Size` 秒向下取整的时间点组成。`Size` 是一个可选参数,默认值为 1800(30 分钟)。

例如,在相应会话中搜索页面浏览量时需要使用此函数。

对于 `DateTime64`,返回值的精度可能与 `StartTime` 的精度不同。将采用所有给定参数中的最高精度。

**语法**

```sql
timeSlots(StartTime, Duration[, Size])
```

**参数**

- `StartTime` — 时间间隔的起始时间。[`DateTime`](/sql-reference/data-types/datetime) 或 [`DateTime64`](/sql-reference/data-types/datetime64)
- `Duration` — 时间间隔的持续时长(以秒为单位)。[`UInt32`](/sql-reference/data-types/int-uint) 或 [`DateTime64`](/sql-reference/data-types/datetime64)
- `Size` — 可选。时间槽的大小(以秒为单位)。默认值为 1800(30 分钟)。[`UInt32`](/sql-reference/data-types/int-uint) 或 [`DateTime64`](/sql-reference/data-types/datetime64)

**返回值**

返回一个 DateTime/DateTime64 数组(返回类型与 `StartTime` 的类型匹配)。对于 DateTime64,返回值的精度可能与 `StartTime` 的精度不同 - 将采用所有给定参数中的最高精度。[`Array(DateTime)`](/sql-reference/data-types/array) 或 [`Array(DateTime64)`](/sql-reference/data-types/array)

**示例**

**为时间间隔生成时间槽**

```sql title=查询
SELECT timeSlots(toDateTime('2012-01-01 12:20:00'), toUInt32(600));
SELECT timeSlots(toDateTime('1980-12-12 21:01:02', 'UTC'), toUInt32(600), 299);
SELECT timeSlots(toDateTime64('1980-12-12 21:01:02.1234', 4, 'UTC'), toDecimal64(600.1, 1), toDecimal64(299, 0))
```

```response title=响应
┌─timeSlots(toDateTime('2012-01-01 12:20:00'), toUInt32(600))─┐
│ ['2012-01-01 12:00:00','2012-01-01 12:30:00']               │
└─────────────────────────────────────────────────────────────┘
┌─timeSlots(toDateTime('1980-12-12 21:01:02', 'UTC'), toUInt32(600), 299)─┐
│ ['1980-12-12 20:56:13','1980-12-12 21:01:12','1980-12-12 21:06:11']     │
└─────────────────────────────────────────────────────────────────────────┘
┌─timeSlots(toDateTime64('1980-12-12 21:01:02.1234', 4, 'UTC'), toDecimal64(600.1, 1), toDecimal64(299, 0))─┐
│ ['1980-12-12 20:56:13.0000','1980-12-12 21:01:12.0000','1980-12-12 21:06:11.0000']                        │
└───────────────────────────────────────────────────────────────────────────────────────────────────────────┘
```

- `toStartOfInterval(t, INTERVAL 1 YEAR)` 返回与 `toStartOfYear(t)` 相同的结果，
- `toStartOfInterval(t, INTERVAL 1 MONTH)` 返回与 `toStartOfMonth(t)` 相同的结果，
- `toStartOfInterval(t, INTERVAL 1 DAY)` 返回与 `toStartOfDay(t)` 相同的结果，
- `toStartOfInterval(t, INTERVAL 15 MINUTE)` 返回与 `toStartOfFifteenMinutes(t)` 相同的结果。

计算相对于特定时间点执行：

| 间隔        | 起始点                 |
| ----------- | ---------------------- |
| YEAR        | 公元 0 年              |
| QUARTER     | 1900 Q1                |
| MONTH       | 1900 年 1 月           |
| WEEK        | 1970 年第 1 周 (01-05) |
| DAY         | 1970-01-01             |
| HOUR        | (\*)                   |
| MINUTE      | 1970-01-01 00:00:00    |
| SECOND      | 1970-01-01 00:00:00    |
| MILLISECOND | 1970-01-01 00:00:00    |
| MICROSECOND | 1970-01-01 00:00:00    |
| NANOSECOND  | 1970-01-01 00:00:00    |

(\*) 小时间隔比较特殊：计算始终相对于当天的 00:00:00（午夜）执行。因此，只有 1 到 23 之间的小时值有效。

如果指定了 `WEEK` 单位，`toStartOfInterval` 假定周从星期一开始。请注意，此行为与 `toStartOfWeek` 函数不同，后者默认周从星期日开始。

第二个重载分别模拟了 TimescaleDB 的 `time_bucket()` 函数和 PostgreSQL 的 `date_bin()` 函数。

**语法**

```sql
toStartOfInterval(value, INTERVAL x unit[, time_zone])
toStartOfInterval(value, INTERVAL x unit[, origin[, time_zone]])
```

**别名**：`date_bin`、`time_bucket`

**参数**

- `value` — 要向下舍入的日期或日期时间值。[`Date`](/sql-reference/data-types/date) 或 [`DateTime`](/sql-reference/data-types/datetime) 或 [`DateTime64`](/sql-reference/data-types/datetime64)
- `x` — 间隔长度数值。- `unit` — 间隔单位：YEAR、QUARTER、MONTH、WEEK、DAY、HOUR、MINUTE、SECOND、MILLISECOND、MICROSECOND、NANOSECOND。- `time_zone` — 可选。时区名称字符串。- `origin` — 可选。计算的起始点（仅限第二个重载）。

**返回值**

返回包含输入值的间隔的起始时间。[`DateTime`](/sql-reference/data-types/datetime)

**示例**

**基本间隔舍入**

```sql title=Query

```

```response title=Response
│               2023-01-01 │

```

**使用起始点**


## toWeek {#toWeek}

引入版本:v20.1

此函数返回日期或日期时间的周数。`toWeek()` 的双参数形式允许您指定一周从周日还是周一开始,以及返回值的范围是 `0` 到 `53` 还是 `1` 到 `53`。

[`toISOWeek()`](#toWeek) 是一个兼容性函数,等同于 `toWeek(date,3)`。

下表描述了 mode 参数的工作方式。

| 模式 | 一周的第一天 | 范围 | 第 1 周是第一个...  |
| ---- | ----------------- | ----- | ----------------------------- |
| 0    | 周日            | 0-53  | 本年中包含周日的周    |
| 1    | 周一            | 0-53  | 本年中有 4 天或更多天的周 |
| 2    | 周日            | 1-53  | 本年中包含周日的周    |
| 3    | 周一            | 1-53  | 本年中有 4 天或更多天的周 |
| 4    | 周日            | 0-53  | 本年中有 4 天或更多天的周 |
| 5    | 周一            | 0-53  | 本年中包含周一的周    |
| 6    | 周日            | 1-53  | 本年中有 4 天或更多天的周 |
| 7    | 周一            | 1-53  | 本年中包含周一的周    |
| 8    | 周日            | 1-53  | 包含 1 月 1 日            |
| 9    | 周一            | 1-53  | 包含 1 月 1 日            |

对于含义为"本年中有 4 天或更多天"的模式值,周数按照 ISO 8601:1988 标准编号:

- 如果包含 1 月 1 日的周在新年中有 4 天或更多天,则为第 1 周。
- 否则,它是上一年的最后一周,下一周为第 1 周。

对于含义为"包含 1 月 1 日"的模式值,包含 1 月 1 日的周即为第 1 周。
该周在新年中包含多少天并不重要,即使只包含一天也是如此。
也就是说,如果 12 月的最后一周包含下一年的 1 月 1 日,它将是下一年的第 1 周。

第一个参数也可以指定为 [`String`](../data-types/string.md) 类型,格式需符合 [`parseDateTime64BestEffort()`](type-conversion-functions.md#parsedatetime64besteffort) 支持的格式。支持字符串参数仅是为了与 MySQL 兼容,某些第三方工具需要此功能。由于字符串参数支持将来可能依赖于新的 MySQL 兼容性设置,并且字符串解析通常较慢,因此不建议使用。

**语法**

```sql
toWeek(datetime[, mode[, time_zone]])
```

**别名**:`week`

**参数**

- `datetime` — 要获取周数的日期或日期时间。[`Date`](/sql-reference/data-types/date) 或 [`DateTime`](/sql-reference/data-types/datetime)
- `mode` — 可选。模式 `0` 到 `9` 决定一周的第一天和周数的范围。默认值为 `0`。
- `time_zone` — 可选。时区。[`String`](/sql-reference/data-types/string)

**返回值**

根据指定的模式返回周数。[`UInt32`](/sql-reference/data-types/int-uint)

**示例**

**使用不同模式获取周数**

```sql title=Query
SELECT toDate('2016-12-27') AS date, toWeek(date) AS week0, toWeek(date,1) AS week1, toWeek(date,9) AS week9
```

```response title=Response
┌───────date─┬─week0─┬─week1─┬─week9─┐
│ 2016-12-27 │    52 │    52 │     1 │
└────────────┴───────┴───────┴───────┘
```
