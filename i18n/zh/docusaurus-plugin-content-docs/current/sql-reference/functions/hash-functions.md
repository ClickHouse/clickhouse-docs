# 哈希函数

哈希函数可用于对元素进行确定性的伪随机打乱。

Simhash 是一种哈希函数,它会为相近(相似)的参数返回相近的哈希值。

大多数哈希函数可接受任意数量和任意类型的参数。

:::note
NULL 的哈希值为 NULL。要获取可空列的非 NULL 哈希值,请将其包装在元组中:

```sql
SELECT cityHash64(tuple(NULL))
```

:::

:::note
要计算表中全部内容的哈希值,请使用 `sum(cityHash64(tuple(*)))`(或其他哈希函数)。`tuple` 确保不会跳过包含 NULL 值的行。`sum` 确保行的顺序不影响结果。
:::

## BLAKE3 {#BLAKE3}

引入版本：v22.10

计算 BLAKE3 哈希值并将结果字节集以 FixedString 类型返回。
此加密哈希函数通过 BLAKE3 Rust 库集成到 ClickHouse 中。
该函数速度较快，与 SHA-2 相比性能提升约两倍，同时生成与 SHA-256 相同长度的哈希值。
它返回一个类型为 FixedString(32) 的 BLAKE3 哈希值字节数组。

**语法**

```sql
BLAKE3(message)
```

**参数**

- `message` — 要进行哈希计算的输入字符串。[`String`](/sql-reference/data-types/string)

**返回值**

返回输入字符串的 32 字节 BLAKE3 哈希值，以固定长度字符串形式返回。[`FixedString(32)`](/sql-reference/data-types/fixedstring)

**示例**

**哈希**

```sql title=Query
SELECT hex(BLAKE3('ABC'))
```

```response title=Response
┌─hex(BLAKE3('ABC'))───────────────────────────────────────────────┐
│ D1717274597CF0289694F75D96D444B992A096F1AFD8E7BBFA6EBB1D360FEDFC │
└──────────────────────────────────────────────────────────────────┘
```

## URLHash {#URLHash}

引入版本:v1.1

一个快速、高质量的非加密哈希函数,用于对经过规范化处理的 URL 字符串进行哈希计算。

此哈希函数有两种模式:

| 模式              | 描述                                                                                                                                                                                  |
| ----------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `URLHash(url)`    | 计算字符串的哈希值,如果末尾存在 `/`、`?` 或 `#` 等尾随符号,则先将其去除。                                                                                   |
| `URLHash(url, N)` | 计算字符串在 URL 层级结构中直到第 N 层的哈希值,如果末尾存在 `/`、`?` 或 `#` 等尾随符号,则先将其去除。层级定义与 `URLHierarchy` 中相同。 |

**语法**

```sql
URLHash(url[, N])
```

**参数**

- `url` — 要进行哈希计算的 URL 字符串。[`String`](/sql-reference/data-types/string)
- `N` — 可选。URL 层级结构中的层级。[`(U)Int*`](/sql-reference/data-types/int-uint)

**返回值**

返回 `url` 计算得到的哈希值。[`UInt64`](/sql-reference/data-types/int-uint)

**示例**

**使用示例**

```sql title=查询
SELECT URLHash('https://www.clickhouse.com')
```

```response title=响应
┌─URLHash('htt⋯house.com')─┐
│     13614512636072854701 │
└──────────────────────────┘
```

**指定层级的 URL 哈希值**

```sql title=查询
SELECT URLHash('https://www.clickhouse.com/docs', 0);
SELECT URLHash('https://www.clickhouse.com/docs', 1);
```

```response title=响应
-- https://www.clickhouse.com 的哈希值
┌─URLHash('htt⋯m/docs', 0)─┐
│     13614512636072854701 │
└──────────────────────────┘
-- https://www.clickhouse.com/docs 的哈希值
┌─URLHash('htt⋯m/docs', 1)─┐
│     13167253331440520598 │
└──────────────────────────┘
```


## cityHash64 {#cityHash64}

引入版本：v1.1

生成一个 64 位 [CityHash](https://github.com/google/cityhash) 哈希值。

这是一个快速的非加密哈希函数。
对于字符串参数，它使用 CityHash 算法；对于其他数据类型的参数，它使用特定实现的快速非加密哈希函数。
该函数使用 CityHash 组合器获取最终结果。

:::info
Google 在 CityHash 被添加到 ClickHouse 之后更改了该算法。
换句话说，ClickHouse 的 cityHash64 与 Google 上游的 CityHash 现在会产生不同的结果。
ClickHouse cityHash64 对应 CityHash v1.0.2 版本。
:::

:::note
对于不同参数类型的相同输入值,计算出的哈希值可能相等。
例如,这会影响不同大小的整数类型、具有相同数据的命名和未命名 `Tuple`、`Map` 以及具有相同数据的对应 `Array(Tuple(key, value))` 类型。
:::

**语法**

```sql
cityHash64(arg1[, arg2, ...])
```

**参数**

- `arg1[, arg2, ...]` — 用于计算哈希的可变数量输入参数。[`Any`](/sql-reference/data-types)

**返回值**

返回输入参数计算得到的哈希值。[`UInt64`](/sql-reference/data-types/int-uint)

**示例**

**调用示例**

```sql title=Query
SELECT cityHash64(array('e','x','a'), 'mple', 10, toDateTime('2019-06-15 23:00:00')) AS CityHash, toTypeName(CityHash) AS type;
```

```response title=Response
┌─────────────CityHash─┬─type───┐
│ 12072650598913549138 │ UInt64 │
└──────────────────────┴────────┘
```

**计算整个表的校验和(精确到行顺序)**

```sql title=Query
CREATE TABLE users (
    id UInt32,
    name String,
    age UInt8,
    city String
)
ENGINE = MergeTree
ORDER BY tuple();

INSERT INTO users VALUES
(1, 'Alice', 25, 'New York'),
(2, 'Bob', 30, 'London'),
(3, 'Charlie', 35, 'Tokyo');

SELECT groupBitXor(cityHash64(*)) FROM users;
```

```response title=Response
┌─groupBitXor(⋯age, city))─┐
│     11639977218258521182 │
└──────────────────────────┘
```

## halfMD5 {#halfMD5}

引入版本：v1.1

将所有输入参数[解释](/sql-reference/functions/type-conversion-functions#reinterpretasstring)为字符串，并计算每个参数的 MD5 哈希值。然后合并这些哈希值，取结果字符串哈希的前 8 个字节，并将其按大端字节序解释为 [UInt64](/sql-reference/data-types/int-uint)。该函数性能相对较慢（每个处理器核心每秒可处理 500 万个短字符串）。

建议考虑使用 [`sipHash64`](#sipHash64) 函数替代。

该函数接受可变数量的输入参数。
参数可以是任何受支持的数据类型。
对于某些数据类型，即使参数类型不同，相同值计算出的哈希函数值也可能相同（例如：不同大小的整数、具有相同数据的命名和未命名元组、Map 以及具有相同数据的对应 Array(Tuple(key, value)) 类型）。

**语法**

```sql
halfMD5(arg1[, arg2, ..., argN])
```

**参数**

- `arg1[, arg2, ..., argN]` — 用于计算哈希的可变数量参数。[`Any`](/sql-reference/data-types)

**返回值**

返回给定输入参数计算出的半 MD5 哈希值，以大端字节序的 `UInt64` 形式返回。[`UInt64`](/sql-reference/data-types/int-uint)

**示例**

**使用示例**

```sql title=Query

```


## sipHash64 {#sipHash64}

引入版本:v1.1

生成一个 64 位的 [SipHash](https://en.wikipedia.org/wiki/SipHash) 哈希值。

这是一个加密哈希函数。其运行速度至少比 [`MD5`](#MD5) 哈希函数快三倍。

该函数将所有输入参数[解释](/sql-reference/functions/type-conversion-functions#reinterpretasstring)为字符串,并计算每个参数的哈希值。
然后使用以下算法组合这些哈希值:

1. 将第一个和第二个哈希值连接成一个数组并进行哈希运算。
2. 以类似的方式对先前计算的哈希值和第三个输入参数的哈希值进行哈希运算。
3. 对原始输入的所有剩余哈希值重复此计算过程。

:::note
对于不同参数类型的相同输入值,计算出的哈希值可能相等。
例如,这会影响不同大小的整数类型、具有相同数据的命名和未命名 `Tuple`、`Map` 以及具有相同数据的相应 `Array(Tuple(key, value))` 类型。
:::

**语法**

```sql
sipHash64(arg1[, arg2, ...])
```

**参数**

- `arg1[, arg2, ...]` — 可变数量的输入参数。[`Any`](/sql-reference/data-types)

**返回值**

返回输入参数计算得到的哈希值。[`UInt64`](/sql-reference/data-types/int-uint)

**示例**

**使用示例**

```sql title=Query
SELECT sipHash64(array('e','x','a'), 'mple', 10, toDateTime('2019-06-15 23:00:00')) AS SipHash, toTypeName(SipHash) AS type;
```

```response title=Response
┌──────────────SipHash─┬─type───┐
│ 11400366955626497465 │ UInt64 │
└──────────────────────┴────────┘
```
