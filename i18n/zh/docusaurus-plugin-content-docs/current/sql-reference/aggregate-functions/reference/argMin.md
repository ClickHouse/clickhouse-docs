## argMin \{#argMin\}

引入版本：v1.1

计算最小 `val` 值所对应的 `arg` 值。如果存在多行具有相同的最小 `val` 值，则返回哪一个关联的 `arg` 值是非确定性的。
`arg` 和 `min` 这两部分都作为[聚合函数](/sql-reference/aggregate-functions/index.md)工作，在处理过程中都会[跳过 `Null`](/sql-reference/aggregate-functions/index.md#null-processing)，并在存在非 `Null` 值时返回非 `Null` 值。

**另请参阅**

* [Tuple](/sql-reference/data-types/tuple.md)

**语法**

```sql
argMin(arg, val)
```

**参数**

* `arg` — 其值将与最小值对应并被返回的参数。[`const String`](/sql-reference/data-types/string)
* `val` — 最小值。[`(U)Int8/16/32/64`](/sql-reference/data-types/int-uint) 或 [`Float*`](/sql-reference/data-types/float) 或 [`Date`](/sql-reference/data-types/date) 或 [`DateTime`](/sql-reference/data-types/datetime) 或 [`Tuple`](/sql-reference/data-types/tuple)

**返回值**

返回与最小 `val` 值相对应的 `arg` 值。类型与 `arg` 的类型一致。

**示例**

**基础用法**

```sql title=Query
SELECT argMin(user, salary) FROM salary;
```

```response title=Response
┌─argMin(user, salary)─┐
│ worker               │
└──────────────────────┘
```

**包含 NULL 处理的扩展示例**

```sql title=Query
CREATE TABLE test
(
    a Nullable(String),
    b Nullable(Int64)
)
ENGINE = Memory AS
SELECT *
FROM VALUES((NULL, 0), ('a', 1), ('b', 2), ('c', 2), (NULL, NULL), ('d', NULL));

SELECT argMin(a, b), min(b) FROM test;
```

```response title=Response
┌─argMin(a, b)─┬─min(b)─┐
│ a            │      0 │
└──────────────┴────────┘
```

**在参数中使用 Tuple 类型**

```sql title=Query
SELECT argMin(a, (b, a)), min(tuple(b, a)) FROM test;
```

```response title=Response
┌─argMin(a, tuple(b, a))─┬─min(tuple(b, a))─┐
│ d                      │ (NULL,NULL)      │
└────────────────────────┴──────────────────┘
```
