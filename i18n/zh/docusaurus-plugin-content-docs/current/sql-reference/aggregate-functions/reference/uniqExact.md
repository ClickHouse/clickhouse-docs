## uniqExact \{#uniqExact\}

引入版本：v1.1

计算不同参数取值的精确数量。

:::warning
`uniqExact` 函数比 `uniq` 使用更多内存，因为随着不同值数量的增加，其状态大小会无限增长。
如果确实需要得到精确结果，请使用 `uniqExact` 函数。
否则请使用 [`uniq`](https://clickhouse.com/docs/sql-reference/aggregate-functions/reference/uniq) 函数。
:::

**语法**

```sql
uniqExact(x[, ...])
```

**参数**

* `x` — 函数可以接受可变数量的参数，每个参数可以是 [`Tuple(T)`](/sql-reference/data-types/tuple)、[`Array(T)`](/sql-reference/data-types/array)、[`Date`](/sql-reference/data-types/date)、[`DateTime`](/sql-reference/data-types/datetime)、[`String`](/sql-reference/data-types/string)、[`(U)Int*`](/sql-reference/data-types/int-uint)、[`Float*`](/sql-reference/data-types/float) 或 [`Decimal`](/sql-reference/data-types/decimal)。

**返回值**

返回不同参数取值的精确计数，返回类型为 UInt64。[`UInt64`](/sql-reference/data-types/int-uint)

**示例**

**基本用法**

```sql title=Query
CREATE TABLE example_data
(
    id UInt32,
    category String
)
ENGINE = Memory;

INSERT INTO example_data VALUES
(1, 'A'), (2, 'B'), (3, 'A'), (4, 'C'), (5, 'B'), (6, 'A');

SELECT uniqExact(category) as exact_unique_categories
FROM example_data;
```

```response title=Response
┌─exact_unique_categories─┐
│                       3 │
└─────────────────────────┘
```

**多参数**

```sql title=Query
SELECT uniqExact(id, category) as exact_unique_combinations
FROM example_data;
```

```response title=Response
┌─exact_unique_combinations─┐
│                         6 │
└───────────────────────────┘
```
