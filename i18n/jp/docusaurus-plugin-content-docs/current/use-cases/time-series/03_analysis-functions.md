# 時系列分析関数

ClickHouse では、標準的な SQL の集約関数やウィンドウ関数を使って時系列分析を行うことができます。
時系列データを扱う際には、一般的に次の 3 種類のメトリクスが登場します。

* 時間とともに単調に増加していくカウンターメトリクス（ページビュー数やイベントの累計など）
* 時点の値を表し、増減するゲージメトリクス（CPU 使用率や温度など）
* 観測値をサンプリングし、バケットごとに件数を集計するヒストグラム（リクエストレイテンシやレスポンスサイズなど）

これらのメトリクスに対する代表的な分析パターンとしては、期間間での値の比較、累積合計の計算、変化率の算出、分布の分析などがあります。
これらはすべて、集約関数の組み合わせに加え、`sum() OVER` のようなウィンドウ関数や、`histogram()` のような専用関数を組み合わせることで実現できます。



## 期間比較による変化 {#time-series-period-over-period-changes}

時系列データを分析する際、期間間での値の変化を理解する必要がよくあります。
これはゲージメトリクスとカウンターメトリクスの両方において不可欠です。
[`lagInFrame`](/docs/sql-reference/window-functions/lagInFrame) ウィンドウ関数を使用すると、前期間の値にアクセスしてこれらの変化を計算できます。

次のクエリは、"Weird Al" Yankovic の Wikipedia ページの閲覧数における日次変化を計算することでこれを実証しています。
trend 列は、前日と比較してトラフィックが増加したか(正の値)、減少したか(負の値)を示し、異常なスパイクやアクティビティの低下を特定するのに役立ちます。

```sql
SELECT
    toDate(time) AS day,
    sum(hits) AS h,
    lagInFrame(h) OVER (ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING) AS p,
    h - p AS trend
FROM wikistat
WHERE path = '"Weird_Al"_Yankovic'
GROUP BY ALL
LIMIT 10;
```

## 累積値 {#time-series-cumulative-values}

カウンターメトリクスは時間の経過とともに自然に累積されます。
この累積的な増加を分析するには、ウィンドウ関数を使用して累計を計算します。

以下のクエリは、累計を作成する`sum() OVER`句を使用した例です。`bar()`関数は増加を視覚的に表現します。

```sql
SELECT
    toDate(time) AS day,
    sum(hits) AS h,
    sum(h) OVER (ROWS BETWEEN UNBOUNDED PRECEDING AND 0 FOLLOWING) AS c,
    bar(c, 0, 50000, 25) AS b
FROM wikistat
WHERE path = '"Weird_Al"_Yankovic'
GROUP BY ALL
ORDER BY day
LIMIT 10;
```

## レート計算 {#time-series-rate-calculations}

時系列データを分析する際、単位時間あたりのイベント発生率を把握することが有用です。
このクエリは、1時間ごとの合計値を1時間の秒数(3600)で除算することで、ページビューの秒あたりのレートを計算します。
視覚的なバーは、アクティビティのピーク時間帯を特定するのに役立ちます。

```sql
SELECT
    toStartOfHour(time) AS time,
    sum(hits) AS hits,
    round(hits / (60 * 60), 2) AS rate,
    bar(rate * 10, 0, max(rate * 10) OVER (), 25) AS b
FROM wikistat
WHERE path = '"Weird_Al"_Yankovic'
GROUP BY time
LIMIT 10;
```

## ヒストグラム {#time-series-histograms}

時系列データの一般的なユースケースとして、追跡されたイベントに基づいてヒストグラムを構築することが挙げられます。
例えば、総ヒット数が10,000を超えるページのみを対象として、総ヒット数に基づくページ数の分布を把握したい場合を考えてみましょう。
`histogram()` 関数を使用すると、ビン数に基づいて適応型ヒストグラムを自動生成できます:

```sql
SELECT
    histogram(10)(hits) AS hist
FROM
(
    SELECT
        path,
        sum(hits) AS hits
    FROM wikistat
    WHERE date(time) = '2015-06-15'
    GROUP BY path
    HAVING hits > 10000
)
FORMAT Vertical;
```

次に、[`arrayJoin()`](/docs/sql-reference/functions/array-join) を使用してデータを整形し、`bar()` で可視化できます:

```sql
WITH histogram(10)(hits) AS hist
SELECT
    round(arrayJoin(hist).1) AS lowerBound,
    round(arrayJoin(hist).2) AS upperBound,
    arrayJoin(hist).3 AS count,
    bar(count, 0, max(count) OVER (), 20) AS b
FROM
(
    SELECT
        path,
        sum(hits) AS hits
    FROM wikistat
    WHERE date(time) = '2015-06-15'
    GROUP BY path
    HAVING hits > 10000
);
```
