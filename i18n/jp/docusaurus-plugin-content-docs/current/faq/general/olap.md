# OLAPとは？ {#what-is-olap}

[OLAP](https://en.wikipedia.org/wiki/Online_analytical_processing)はOnline Analytical Processingの略称です。技術的な観点とビジネス的な観点という2つの視点から捉えることができる広範な用語ですが、最も基本的なレベルでは、この言葉を逆から読み解くことができます：

Processing（処理）
: ソースデータを処理し...

Analytical（分析）
: ...分析レポートと洞察を生成し...

Online（オンライン）
: ...リアルタイムで提供します。


## ビジネスの観点から見たOLAP {#olap-from-the-business-perspective}

近年、ビジネスパーソンはデータの価値を認識し始めています。データに基づかず盲目的に意思決定を行う企業は、競合他社に後れを取ることが少なくありません。成功している企業のデータ駆動型アプローチでは、ビジネス上の意思決定に少しでも役立つ可能性のあるすべてのデータを収集し、それらをタイムリーに分析する仕組みが必要となります。ここでOLAPデータベース管理システム(DBMS)の出番となります。

ビジネスの観点では、OLAPは企業が業務活動を継続的に計画、分析、報告することを可能にし、効率を最大化し、コストを削減し、最終的に市場シェアを獲得します。これは社内システムで実現することも、Web/モバイル分析サービス、CRMサービスなどのSaaSプロバイダーに外部委託することも可能です。OLAPは多くのBIアプリケーション(ビジネスインテリジェンス)を支える技術です。

ClickHouseはOLAPデータベース管理システムであり、ドメイン固有のデータを分析するためのこれらのSaaSソリューションのバックエンドとして頻繁に使用されています。しかし、一部の企業はサードパーティプロバイダーとデータを共有することに依然として消極的であり、社内データウェアハウスのシナリオも有効な選択肢となります。


## 技術的観点から見たOLAP {#olap-from-the-technical-perspective}

すべてのデータベース管理システムは、OLAP(Online **Analytical** Processing)とOLTP(Online **Transactional** Processing)の2つのグループに分類できます。前者は大量の履歴データに基づくレポート作成に重点を置いていますが、その実行頻度はそれほど高くありません。一方、後者は通常、トランザクションの連続的なストリームを処理し、データの現在の状態を常に更新します。

実際には、OLAPとOLTPは明確なカテゴリーというよりも、スペクトラムのようなものです。ほとんどの実システムは通常、どちらか一方に重点を置いていますが、反対の種類のワークロードが必要な場合には、何らかのソリューションや回避策を提供します。この状況により、企業は複数のストレージシステムを統合して運用することを余儀なくされることが多く、それ自体は大きな問題ではないかもしれませんが、システムが増えるほど保守コストが高くなります。そのため、近年のトレンドはHTAP(**Hybrid Transactional/Analytical Processing**)であり、単一のデータベース管理システムで両方の種類のワークロードを同等に処理できます。

DBMSが純粋なOLAPまたは純粋なOLTPとして始まったとしても、競合に対抗するためにHTAPの方向に進まざるを得ません。ClickHouseも例外ではなく、当初は[可能な限り高速なOLAPシステム](../../concepts/why-clickhouse-is-so-fast.mdx)として設計されており、現在でも本格的なトランザクションサポートはありませんが、一貫性のある読み取り/書き込みや、データの更新/削除のためのミューテーションなどの機能を追加する必要がありました。

OLAPシステムとOLTPシステムの間の根本的なトレードオフは依然として存在します:

- 分析レポートを効率的に作成するには、列を個別に読み取ることができることが重要であるため、ほとんどのOLAPデータベースは[カラム型](../../faq/general/columnar-database.md)です。
- 一方、列を個別に保存すると、追加やインプレース変更などの行に対する操作のコストが列数に比例して増加します(システムがイベントのすべての詳細を念のため収集しようとする場合、列数は膨大になる可能性があります)。そのため、ほとんどのOLTPシステムは行単位でデータを保存します。
