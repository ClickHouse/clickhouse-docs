## 説明 \{#description\}

サーバーがサポートするデータスキッピングインデックスの型の一覧と、各型に埋め込まれたドキュメントが含まれています。データスキッピングインデックスの型は、`CREATE TABLE` クエリ内の `INDEX` 宣言の `TYPE` で指定します。これにより、ClickHouse はクエリ条件に一致しないグラニュールをスキップできます。

このテーブルには利用可能な索引の*型*が一覧表示されています。一方、[`system.data_skipping_indices`](/operations/system-tables/data_skipping_indices) には、既存のテーブルに定義されている索引インスタンスが一覧表示されています。

## カラム \{#columns\}

{/*AUTOGENERATED_START*/ }

* `name` ([String](../../sql-reference/data-types/)) — `INDEX` 宣言の `TYPE` で指定される、データスキッピングインデックスの型名。
* `description` ([String](../../sql-reference/data-types/)) — そのデータスキッピングインデックスの型が何を行うかを示す概要説明。
* `syntax` ([String](../../sql-reference/data-types/)) — `CREATE TABLE` クエリの `INDEX` 句でその索引をどのように宣言するか。
* `examples` ([String](../../sql-reference/data-types/)) — 使用例。
* `introduced_in` ([String](../../sql-reference/data-types/)) — その索引型が最初に導入された ClickHouse のバージョン。形式は major.minor。
* `related` ([Array(String)](../../sql-reference/data-types/)) — 関連するデータスキッピングインデックスの型名。

{/*AUTOGENERATED_END*/ }

## 例 \{#example\}

```sql title="Query"
SELECT name, syntax
FROM system.data_skipping_index_types
WHERE name IN ('minmax', 'set')
ORDER BY name
```

```text title="Response"
┌─name───┬─syntax───────────────────────────────────────┐
│ minmax │ INDEX name expr TYPE minmax GRANULARITY n     │
│ set    │ INDEX name expr TYPE set(max_rows) GRANULARITY n │
└────────┴──────────────────────────────────────────────┘
```
