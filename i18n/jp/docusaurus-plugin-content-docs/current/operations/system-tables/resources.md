# system.resources

ローカルサーバー上に存在する[リソース](/operations/workload-scheduling.md#workload_entity_storage)に関する情報を含みます。このテーブルには、各リソースごとに1行が格納されます。

例:

```sql
SELECT *
FROM system.resources
FORMAT Vertical
```

```text
Row 1:
──────
name:         io_read
read_disks:   ['s3']
write_disks:  []
create_query: CREATE RESOURCE io_read (READ DISK s3)

Row 2:
──────
name:         io_write
read_disks:   []
write_disks:  ['s3']
create_query: CREATE RESOURCE io_write (WRITE DISK s3)
```

カラム:

<!--AUTOGENERATED_START-->

- `name` ([String](../../sql-reference/data-types/)) — リソースの名前。
- `read_disks` ([Array(String)](../../sql-reference/data-types/)) — 読み取り操作でこのリソースを使用するディスク名のリスト。
- `write_disks` ([Array(String)](../../sql-reference/data-types/)) — 書き込み操作でこのリソースを使用するディスク名のリスト。
- `unit` ([String](../../sql-reference/data-types/)) — コスト測定に使用されるリソース単位。
- `create_query` ([String](../../sql-reference/data-types/)) — リソースのCREATEクエリ。
<!--AUTOGENERATED_END-->
