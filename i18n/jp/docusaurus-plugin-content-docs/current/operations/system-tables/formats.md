## 説明 \{#description\}

すべてのフォーマットの一覧に加え、各フォーマットが入力/出力に適しているかどうか、および並列化をサポートしているかどうかを示すフラグが含まれます。

## カラム \{#columns\}

{/*AUTOGENERATED_START*/ }

* `name` ([String](../../sql-reference/data-types/)) — フォーマット名。
* `is_input` ([UInt8](../../sql-reference/data-types/)) — このフォーマットがデータ入力に適しているかどうかを示すフラグ。
* `is_output` ([UInt8](../../sql-reference/data-types/)) — このフォーマットがデータ出力に適しているかどうかを示すフラグ。
* `supports_parallel_parsing` ([UInt8](../../sql-reference/data-types/)) — このフォーマットが並列解析をサポートしているかどうかを示すフラグ。
* `supports_parallel_formatting` ([UInt8](../../sql-reference/data-types/)) — このフォーマットが並列フォーマット処理をサポートしているかどうかを示すフラグ。
* `is_tty_friendly` ([UInt8](../../sql-reference/data-types/)) — このフォーマットが通常ターミナルで適切に表示されるかどうかを示すフラグ。その他のフォーマットでは、出力前に CLI が確認を求めます。
* `content_type` ([String](../../sql-reference/data-types/)) — 出力フォーマットに対応する HTTP Content-Type。現在のフォーマット設定に応じて変わる場合があります。
* `supports_random_access` ([UInt8](../../sql-reference/data-types/)) — このフォーマットが入力時のランダムアクセスをサポートしているかどうかを示すフラグ。
* `has_schema_inference` ([UInt8](../../sql-reference/data-types/)) — このフォーマットは、データから動的に schema を推定できます (埋め込まれたヘッダー/メタデータ、またはデータの一部から) 。
* `has_external_schema` ([UInt8](../../sql-reference/data-types/)) — このフォーマットは、固定の schema を持つか、独自の形式で事前定義された schema を受け入れます。
* `prefers_large_blocks` ([UInt8](../../sql-reference/data-types/)) — このフォーマットは、出力時により大きなブロックを書き込み、入力時にもより大きなブロックを生成します。
* `supports_append` ([UInt8](../../sql-reference/data-types/)) — このフォーマットでは、単一のファイルに追記できます。
* `supports_subsets_of_columns` ([UInt8](../../sql-reference/data-types/)) — この入力フォーマットは、一部のカラムが省略されている場合を認識できます。
