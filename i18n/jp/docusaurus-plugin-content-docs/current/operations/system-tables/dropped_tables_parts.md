## 説明 \{#description\}

[system.dropped&#95;tables](./dropped_tables.md) に含まれる削除済みの [MergeTree](../../engines/table-engines/mergetree-family/mergetree.md) テーブルのパーツに関する情報を保持します。

このテーブルのスキーマは [system.parts](./parts.md) と同じです。

## カラム \{#columns\}

{/*AUTOGENERATED_START*/ }

* `partition` ([String](../../sql-reference/data-types/)) — パーティション識別子 (テーブルのパーティションキー式から導出された文字列表現) 。
* `name` ([String](../../sql-reference/data-types/)) —
  データパーツの名前。パーツの命名規則から、データの配置や取り込み、マージのパターンに関するさまざまな情報を把握できます。パーツ名のフォーマットは次のとおりです。

```text
<partition_id>_<minimum_block_number>_<maximum_block_number>_<level>_<data_version>
```

* 定義:
  * `partition_id` - パーティションキーを識別します
  * `minimum_block_number` - パーツ内の最小ブロック番号を識別します。ClickHouse では常に連続したブロックがマージされます
  * `maximum_block_number` - パーツ内の最大ブロック番号を識別します
  * `level` - パーツに対して追加のマージが行われるたびに 1 ずつ増加します。レベル 0 は、まだマージされていない新しいパーツであることを示します。ClickHouse のすべてのパーツは常に不変であることを覚えておくことが重要です
  * `data_version` - オプションの値で、パーツが変更されると増加します (繰り返しになりますが、パーツは不変であるため、変更されたデータは常に新しいパーツにのみ書き込まれます)

* `uuid` ([UUID](../../sql-reference/data-types/)) — データパートのUUID。

* `part_type` ([String](../../sql-reference/data-types/)) — データ part の保存フォーマット。設定可能な値: `Wide` — 各カラムは別々のファイルに保存されます。`Compact` — すべてのカラムが 1 つのファイルに保存されます。データの保存フォーマットは、MergeTree テーブルの `min_bytes_for_wide_part` および `min_rows_for_wide_part` の設定で制御されます。

* `active` ([UInt8](../../sql-reference/data-types/)) — データパーツがアクティブかどうかを示すフラグです。データパーツがアクティブな場合は、テーブルで使用されます。そうでない場合は、まもなく削除されます。非アクティブなデータパーツは、マージおよびミューテーション操作の後に現れます。

* `marks` ([UInt64](../../sql-reference/data-types/)) — マーク数。データパート内のおおよその行数を求めるには、`marks` にインデックス粒度 (通常は 8192) を掛けます (この目安は adaptive granularity では機能しません) 。

* `rows` ([UInt64](../../sql-reference/data-types/)) — 行の数。

* `files` ([UInt64](../../sql-reference/data-types/)) — データパート内のファイル数。

* `bytes_on_disk` ([UInt64](../../sql-reference/data-types/)) — すべてのデータパートファイルの合計サイズ (バイト) 。

* `data_compressed_bytes` ([UInt64](../../sql-reference/data-types/)) — データパート内の圧縮データの合計サイズです。マークファイルなどの補助ファイルは含まれません。

* `data_uncompressed_bytes` ([UInt64](../../sql-reference/data-types/)) — データパート内の非圧縮データの合計サイズ。マークファイルなどの補助ファイルは含まれません。

* `primary_key_size` ([UInt64](../../sql-reference/data-types/)) — ディスク上の primary.idx/cidx ファイル内の主キーの値が使用するメモリ量 (バイト単位) 。

* `marks_bytes` ([UInt64](../../sql-reference/data-types/)) — マークファイルのサイズ。

* `secondary_indices_compressed_bytes` ([UInt64](../../sql-reference/data-types/)) — データパート内のセカンダリ索引に対する圧縮データの合計サイズ。補助ファイル (たとえば、マークファイルなど) は含まれません。

* `secondary_indices_uncompressed_bytes` ([UInt64](../../sql-reference/data-types/)) — データパート内のセカンダリ索引の非圧縮データの合計サイズです。補助ファイル (たとえば、マークを含むファイルなど) は含まれません。

* `secondary_indices_marks_bytes` ([UInt64](../../sql-reference/data-types/)) — セカンダリ索引のマークファイルのサイズ。

* `modification_time` ([DateTime](../../sql-reference/data-types/)) — データパートを含むディレクトリが変更された時刻です。通常、これはデータパートの作成時刻に対応します。

* `remove_time` ([DateTime](../../sql-reference/data-types/)) — データパートが非アクティブになった日時。

* `refcount` ([UInt32](../../sql-reference/data-types/)) — データパートが使用されている箇所の数です。値が 2 を超える場合、そのデータパートはクエリやマージ処理で使用されています。

* `min_date` ([Date](../../sql-reference/data-types/)) — データパート内の date キーの最小値。

* `max_date` ([Date](../../sql-reference/data-types/)) — データパートにおける日付キーの最大値。

* `min_time` ([DateTime](../../sql-reference/data-types/)) — データパートにおける日時キーの最小値。

* `max_time` ([DateTime](../../sql-reference/data-types/)) — データパート内の日付時刻キーの最大値。

* `partition_id` ([String](../../sql-reference/data-types/)) — パーティションのID。

* `min_block_number` ([Int64](../../sql-reference/data-types/)) — マージ後に現在のパーツを構成するデータパーツの最小番号。

* `max_block_number` ([Int64](../../sql-reference/data-types/)) — マージ後に現在のパーツを構成するデータパーツ数の最大値。

* `level` ([UInt32](../../sql-reference/data-types/)) — マージツリーの深さ。ゼロは、現在のパーツが他のパーツのマージではなく、insert によって作成されたことを意味します。

* `data_version` ([UInt64](../../sql-reference/data-types/)) — データパートにどの mutation を適用するかを判定するために使用される数値です (`data_version` より大きいバージョンの mutation) 。

* `primary_key_bytes_in_memory` ([UInt64](../../sql-reference/data-types/)) — 主キーの値が使用するメモリ量 (バイト単位) 。`primary_key_lazy_load` が有効で、キーが読み込まれていない場合は 0 になります。

* `primary_key_bytes_in_memory_allocated` ([UInt64](../../sql-reference/data-types/)) — 主キーの値のために確保されているメモリ量 (バイト単位) 。`primary_key_lazy_load` が有効で、キーが読み込まれていない場合は 0 です。

* `index_granularity_bytes_in_memory` ([UInt64](../../sql-reference/data-types/)) — 索引の粒度の値が使用するメモリ量 (バイト単位)  (primary&#95;key&#95;lazy&#95;load=1 かつ use&#95;primary&#95;key&#95;cache=1 の場合は 0 です) 。

* `index_granularity_bytes_in_memory_allocated` ([UInt64](../../sql-reference/data-types/)) — 索引の粒度値用に確保されたメモリ量 (バイト単位)  (primary&#95;key&#95;lazy&#95;load=1 かつ use&#95;primary&#95;key&#95;cache=1 の場合は 0) 。

* `is_frozen` ([UInt8](../../sql-reference/data-types/)) — パーティションデータのバックアップの有無を示すフラグです。1 はバックアップが存在することを、0 はバックアップが存在しないことを示します。詳細については、FREEZE PARTITION を参照してください。

* `database` ([String](../../sql-reference/data-types/)) — データベース名。

* `table` ([String](../../sql-reference/data-types/)) — テーブル名。

* `engine` ([String](../../sql-reference/data-types/)) — パラメータを含まないテーブルエンジン名。

* `disk_name` ([String](../../sql-reference/data-types/)) — データパートを保存するディスクの名前。

* `path` ([String](../../sql-reference/data-types/)) — データパートのファイルが格納されているフォルダーへの絶対パス。

* `hash_of_all_files` ([String](../../sql-reference/data-types/)) — 圧縮ファイルの sipHash128 値。

* `hash_of_uncompressed_files` ([String](../../sql-reference/data-types/)) — マークを含むファイルや索引ファイルなど、非圧縮ファイルの sipHash128。

* `uncompressed_hash_of_compressed_files` ([String](../../sql-reference/data-types/)) — 圧縮ファイル内のデータを、あたかも非圧縮であるかのように扱った場合の sipHash128。

* `delete_ttl_info_min` ([DateTime](../../sql-reference/data-types/)) — 有効期限 (TTL) の DELETE ルールにおける日時キーの最小値。

* `delete_ttl_info_max` ([DateTime](../../sql-reference/data-types/)) — TTL DELETE ルールにおける日時キーの最大値。

* `move_ttl_info.expression` ([Array(String)](../../sql-reference/data-types/)) — 式の配列。各式で有効期限 (TTL) MOVE ルールを定義します。

* `move_ttl_info.min` ([Array(DateTime)](../../sql-reference/data-types/)) — 日付と時刻の値の配列です。各要素は、TTL MOVE ルールにおける最小のキー値を表します。

* `move_ttl_info.max` ([Array(DateTime)](../../sql-reference/data-types/)) — 日付と時刻の値の配列。各要素は、TTL MOVE ルールにおける最大キー値を表します。

* `default_compression_codec` ([String](../../sql-reference/data-types/)) — このデータパートの圧縮に使用されるコーデック名です (カラムに明示的なコーデックが指定されていない場合) 。

* `recompression_ttl_info.expression` ([Array(String)](../../sql-reference/data-types/)) — 有効期限 (TTL) 式。

* `recompression_ttl_info.min` ([Array(DateTime)](../../sql-reference/data-types/)) — このデータパート内で計算された TTL 式の最小値。TTL の有効期限が切れた行が少なくとも 1 行あるかどうかを判断するために使用されます。

* `recompression_ttl_info.max` ([Array(DateTime)](../../sql-reference/data-types/)) — このパート内で計算された TTL 式の最大値。TTL の有効期限が切れたすべての行が含まれているかどうかを確認するために使用されます。

* `group_by_ttl_info.expression` ([Array(String)](../../sql-reference/data-types/)) — 有効期限 (TTL) の式。

* `group_by_ttl_info.min` ([Array(DateTime)](../../sql-reference/data-types/)) — このパート内で計算された TTL 式の最小値。少なくとも 1 行に有効期限 (TTL) 切れのデータがあるかどうかを確認するために使用されます。

* `group_by_ttl_info.max` ([Array(DateTime)](../../sql-reference/data-types/)) — このパート内で計算された TTL 式の最大値です。有効期限 (TTL) が切れたすべての行を含んでいるかどうかを判断するために使用されます。

* `rows_where_ttl_info.expression` ([Array(String)](../../sql-reference/data-types/)) — 有効期限 (TTL) 式。

* `rows_where_ttl_info.min` ([Array(DateTime)](../../sql-reference/data-types/)) — このパート内で計算された有効期限 (TTL) 式の最小値です。有効期限 (TTL) が切れた行が少なくとも 1 つ存在するかどうかを判断するために使用されます。

* `rows_where_ttl_info.max` ([Array(DateTime)](../../sql-reference/data-types/)) — このパート内で計算された有効期限 (TTL) 式の最大値です。有効期限 (TTL) が切れたすべての行が揃っているかどうかを判断するために使用します。

* `projections` ([Array(String)](../../sql-reference/data-types/)) — この part について計算されたプロジェクション名の一覧。

* `visible` ([UInt8](../../sql-reference/data-types/)) — このパートがSELECTクエリで可視かどうかを示すフラグ。

* `creation_tid` ([Tuple(UInt64, UInt64, UUID)](../../sql-reference/data-types/)) — このオブジェクトを作成した、または作成しようとしているトランザクションのID。

* `removal_tid_lock` ([UInt64](../../sql-reference/data-types/)) — removal&#95;tid のハッシュ値。オブジェクトを削除のためにロックする際に使用されます。

* `removal_tid` ([Tuple(UInt64, UInt64, UUID)](../../sql-reference/data-types/)) — このオブジェクトを削除した、または削除を試みているトランザクションの ID

* `creation_csn` ([UInt64](../../sql-reference/data-types/)) — このオブジェクトを作成したトランザクションのCSN

* `removal_csn` ([UInt64](../../sql-reference/data-types/)) — このオブジェクトを削除したトランザクションのCSN

* `has_lightweight_delete` ([UInt8](../../sql-reference/data-types/)) — そのパートに論理削除マスクがあるかどうかを示すフラグ。

* `last_removal_attempt_time` ([DateTime](../../sql-reference/data-types/)) — サーバーがこのパートの削除を最後に試みた時刻。

* `removal_state` ([String](../../sql-reference/data-types/)) — パート削除プロセスの現在の状態。

{/*AUTOGENERATED_END*/ }
