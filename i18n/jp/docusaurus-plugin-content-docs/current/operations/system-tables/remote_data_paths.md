## 説明 \{#description\}

ローカルのメタデータパスとリモートのブロブパスの対応関係を含め、リモートディスク (例: S3、Azure Blob Storage) に保存されたデータファイルに関する情報を格納します。

各行は、データファイルに関連付けられた 1 つのリモートブロブオブジェクトを表します。

## カラム \{#columns\}

{/*AUTOGENERATED_START*/ }

* `disk_name` ([String](../../sql-reference/data-types/)) — ディスク名。
* `path` ([String](../../sql-reference/data-types/)) — ディスクのパス。
* `cache_base_path` ([String](../../sql-reference/data-types/)) — キャッシュファイルのベースディレクトリ。
* `local_path` ([String](../../sql-reference/data-types/)) — ClickHouse ファイルのパスで、メタデータのパスとしても使用されます。
* `remote_path` ([String](../../sql-reference/data-types/)) — ClickHouse ファイルに関連付けられた、オブジェクトストレージ内の ブロブ パス。
* `size` ([UInt64](../../sql-reference/data-types/)) — ファイルのサイズ (圧縮後) 。
* `common_prefix_for_blobs` ([String](../../sql-reference/data-types/)) — オブジェクトストレージ内の ブロブ の共通プレフィックス。
* `cache_paths` ([Array(String)](../../sql-reference/data-types/)) — 対応する ブロブ のキャッシュファイル。

{/*AUTOGENERATED_END*/ }

## 例 \{#example\}

```sql
SELECT * FROM system.remote_data_paths LIMIT 1 FORMAT Vertical;
```

```text
Row 1:
──────
disk_name:              s3
path:                   /var/lib/clickhouse/disks/s3/
cache_base_path:        /var/lib/clickhouse/disks/s3_cache/
local_path:             store/123/1234abcd-1234-1234-1234-1234abcd1234/all_0_0_0/data.bin
remote_path:            abc123/all_0_0_0/data.bin
size:                   1048576
common_prefix_for_blobs:
cache_paths:            ['/var/lib/clickhouse/disks/s3_cache/a1/b2/c3d4e5f6']
```
