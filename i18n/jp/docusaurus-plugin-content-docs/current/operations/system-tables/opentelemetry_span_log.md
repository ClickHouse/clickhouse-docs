## 説明 \{#description\}

実行されたクエリに対する [トレーススパン](https://opentracing.io/docs/overview/spans/) に関する情報を含みます。

## カラム \{#columns\}

{/*AUTOGENERATED_START*/ }

* `hostname` ([LowCardinality(String)](/sql-reference/data-types/lowcardinality)) — このスパンが収集されたホスト名。
* `trace_id` ([UUID](/sql-reference/data-types/uuid)) — 実行されたクエリのトレース ID。
* `span_id` ([UInt64](/sql-reference/data-types/int-uint)) — トレーススパンの ID。
* `parent_span_id` ([UInt64](/sql-reference/data-types/int-uint)) — 親トレーススパンの ID。
* `operation_name` ([LowCardinality(String)](/sql-reference/data-types/lowcardinality)) — オペレーション名。
* `kind` ([Enum8(&#39;INTERNAL&#39; = 0, &#39;SERVER&#39; = 1, &#39;CLIENT&#39; = 2, &#39;PRODUCER&#39; = 3, &#39;CONSUMER&#39; = 4)](/sql-reference/data-types/enum)) — スパンの SpanKind。INTERNAL — スパンがアプリケーション内の内部操作を表すことを示します。SERVER — スパンが同期 RPC またはその他のリモートリクエストのサーバー側処理を表すことを示します。CLIENT — スパンが何らかのリモートサービスへのリクエストを表すことを示します。PRODUCER — スパンが非同期リクエストのイニシエーターを表すことを示します。この親スパンは、対応する子 CONSUMER スパンより先に終了することが多く、場合によっては子スパンが開始する前に終了することもあります。CONSUMER - スパンが非同期の PRODUCER リクエストの子を表すことを示します。
* `start_time_us` ([UInt64](/sql-reference/data-types/int-uint)) — トレーススパンの開始時刻 (マイクロ秒) 。
* `finish_time_us` ([UInt64](/sql-reference/data-types/int-uint)) — トレーススパンの終了時刻 (マイクロ秒) 。
* `finish_date` ([Date](/sql-reference/data-types/date)) — トレーススパンの終了日。
* `status_code` ([Enum8(&#39;UNSET&#39; = 0, &#39;OK&#39; = 1, &#39;ERROR&#39; = 2)](/sql-reference/data-types/enum)) — スパンのステータスコード。
* `status_message` ([LowCardinality(String)](/sql-reference/data-types/lowcardinality)) — エラーメッセージ。
* `attribute` ([Map(LowCardinality(String), String)](/sql-reference/data-types/map)) — トレーススパンに応じた属性です。これらは OpenTelemetry 標準の推奨事項に従って設定されます。

**別名:**

* `attribute.names` — `mapKeys(attribute)` のエイリアス。
* `attribute.values` — `mapValues(attribute)` のエイリアス。

{/*AUTOGENERATED_END*/ }
