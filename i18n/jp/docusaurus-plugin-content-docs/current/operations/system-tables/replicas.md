## 説明 \{#description\}

ローカルサーバー上に存在するレプリケートテーブルの情報とステータスを含みます。
このテーブルは監視用途に使用できます。テーブルには、すべての `Replicated*` テーブルごとに 1 行が格納されます。

## カラム \{#columns\}

{/*AUTOGENERATED_START*/ }

* `database` ([String](../../sql-reference/data-types/)) — データベースの名前。
* `table` ([String](../../sql-reference/data-types/)) — テーブルの名前。
* `uuid` ([UUID](../../sql-reference/data-types/)) — テーブルのUUID。
* `engine` ([String](../../sql-reference/data-types/)) — テーブルエンジンの名前。
* `is_leader` ([UInt8](../../sql-reference/data-types/)) — レプリカがリーダーかどうかを示します。複数のレプリカが同時にリーダーである場合があります。レプリカがリーダーにならないようにするには、merge&#95;tree 設定 `replicated&#95;can&#95;become&#95;leader` を使用します。リーダーはバックグラウンドマージのスケジューリングを担当します。なお、リーダーかどうかにかかわらず、利用可能で、かつ ZK にセッションを持つ任意のレプリカに書き込むことができます。
* `can_become_leader` ([UInt8](../../sql-reference/data-types/)) — レプリカがリーダーになれるかどうか。
* `is_readonly` ([UInt8](../../sql-reference/data-types/)) — レプリカが読み取り専用モードかどうかを示します。このモードは、設定に ClickHouse Keeper のセクションがない場合、ClickHouse Keeper でセッションを再初期化する際に不明なエラーが発生した場合、または ClickHouse Keeper でのセッション再初期化中に有効になります。
* `readonly_start_time` ([Nullable(DateTime)](../../sql-reference/data-types/)) — レプリカが読み取り専用モードに移行した時点のタイムスタンプ。レプリカが読み取り専用モードでない場合は Null です。
* `is_session_expired` ([UInt8](../../sql-reference/data-types/)) — ClickHouse Keeper とのセッションの有効期限が切れているかどうか。基本的には `is_readonly` と同じです。
* `future_parts` ([UInt32](../../sql-reference/data-types/)) — まだ実行されていない INSERT またはマージの結果として生成されるデータパーツの数。
* `parts_to_check` ([UInt32](../../sql-reference/data-types/)) — 検証キュー内のデータパーツ数。パーツは、破損している疑いがある場合に検証キューに追加されます。
* `zookeeper_name` ([String](../../sql-reference/data-types/)) — テーブルのメタデータが保存されている[Zoo]Keeperクラスター (補助クラスターの場合もあります) の名前
* `zookeeper_path` ([String](../../sql-reference/data-types/)) — ClickHouse Keeper内にあるテーブルデータへのパス。
* `replica_name` ([String](../../sql-reference/data-types/))  — ClickHouse Keeper 内のレプリカ名です。同じテーブルの各レプリカには異なる名前が付けられます。
* `replica_path` ([String](../../sql-reference/data-types/)) — ClickHouse Keeper 内のレプリカデータへのパス。&#39;zookeeper&#95;path/replicas/replica&#95;path&#39; を連結したものと同一です。
* `columns_version` ([Int32](../../sql-reference/data-types/)) — テーブル構造のバージョン番号です。ALTER が何回実行されたかを示します。レプリカ間でバージョンが異なる場合は、一部のレプリカでまだすべての ALTER が反映されていないことを意味します。
* `queue_size` ([UInt32](../../sql-reference/data-types/)) — 実行待ちの操作が入るキューのサイズ。操作には、データブロックの挿入、マージ、その他の特定の処理が含まれます。通常は future&#95;parts と一致します。
* `inserts_in_queue` ([UInt32](../../sql-reference/data-types/)) — 実行する必要があるデータブロックの挿入数。挿入内容は通常、すぐにレプリケートされます。この数が大きい場合は、何らかの問題が発生していることを意味します。
* `merges_in_queue` ([UInt32](../../sql-reference/data-types/)) — 実行を待機しているマージの数です。マージに時間がかかることがあるため、この値が長時間 0 より大きいままになる場合があります。
* `part_mutations_in_queue` ([UInt32](../../sql-reference/data-types/)) — 実行待ちのmutation数。
* `queue_oldest_time` ([DateTime](../../sql-reference/data-types/)) — `queue_size` が 0 より大きい場合、キュー内の最も古い操作が追加された時刻を示します。
* `inserts_oldest_time` ([DateTime](../../sql-reference/data-types/)) — `queue_oldest_time` を参照してください。
* `merges_oldest_time` ([DateTime](../../sql-reference/data-types/)) — `queue_oldest_time` を参照してください。
* `part_mutations_oldest_time` ([DateTime](../../sql-reference/data-types/)) — `queue_oldest_time`を参照。
* `oldest_part_to_get` ([String](../../sql-reference/data-types/)) — レプリケーションキュー内で最も古い GET&#95;PARTS エントリに基づいて、他のレプリカから取得するパーツの名前。
* `oldest_part_to_merge_to` ([String](../../sql-reference/data-types/)) — レプリケーションキュー内の最も古い MERGE&#95;PARTS エントリから取得された、マージ先の結果パーツの名前。
* `oldest_part_to_mutate_to` ([String](../../sql-reference/data-types/)) — レプリケーションキュー内で最も古い MUTATE&#95;PARTS エントリから取得した、変異後の結果パーツ名。
* `log_max_index` ([UInt64](../../sql-reference/data-types/)) — 一般的なアクティビティのログにおける最大エントリ番号です。このカラムと次の3つ (`log_pointer`、`total_replicas`、`active_replicas`) は、ClickHouse Keeper とのアクティブなセッションが存在する場合にのみ、ゼロ以外の値になります。
* `log_pointer` ([UInt64](../../sql-reference/data-types/)) — レプリカが一般アクティビティのログから実行キューにコピーした最大エントリ番号に 1 を加えた値です。log&#95;pointer が log&#95;max&#95;index より大幅に小さい場合は、何らかの問題が発生しています。
* `last_queue_update` ([DateTime](../../sql-reference/data-types/)) — キューが最後に更新された日時。
* `absolute_delay` ([UInt64](../../sql-reference/data-types/)) — 現在のレプリカが何秒遅延しているか。
* `total_replicas` ([UInt32](../../sql-reference/data-types/)) — このテーブルについて認識されているレプリカの総数。
* `active_replicas` ([UInt32](../../sql-reference/data-types/)) — ClickHouse Keeper にセッションを持つ、このテーブルのレプリカ数 (つまり、正常に稼働しているレプリカの数) 。
* `lost_part_count` ([UInt64](../../sql-reference/data-types/)) — テーブル作成以降、すべてのレプリカでそのテーブルにおいて失われたパーツの累計数です。値は ClickHouse Keeper に永続化され、増加することしかありません。
* `last_queue_update_exception` ([String](../../sql-reference/data-types/)) — キューに破損したエントリが含まれている場合。特に、ClickHouse がバージョン間の後方互換性を損ない、新しいバージョンによって書き込まれたログエントリを古いバージョンで解析できない場合に重要です。
* `zookeeper_exception` ([String](../../sql-reference/data-types/)) — ClickHouse Keeper から情報を取得する際にエラーが発生した場合の、最後の例外メッセージ。
* `replica_is_active` ([Map(String, UInt8)](../../sql-reference/data-types/)) — レプリカ名と、各レプリカがアクティブかどうかの対応関係。

{/*AUTOGENERATED_END*/ }

**別名:**

* `readonly_duration` — `if(isNull(readonly_start_time), NULL, now() - readonly_start_time)` のエイリアスです。

FROM system.replicas
WHERE table = 'test_table'
FORMAT Vertical
