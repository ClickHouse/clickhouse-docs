## 説明 \{#description\}

リモートオブジェクト用のファイルシステムキャッシュ内のすべてのエントリに関する情報が含まれます。

## カラム \{#columns\}

{/*AUTOGENERATED_START*/ }

* `cache_name` ([String](../../sql-reference/data-types/)) — キャッシュオブジェクトの名前。`SYSTEM DESCRIBE FILESYSTEM CACHE <name>`、`SYSTEM DROP FILESYSTEM CACHE <name>` コマンドで使用できます
* `cache_base_path` ([String](../../sql-reference/data-types/)) — すべてのキャッシュファイル (`cache_name` で識別されるキャッシュ) が格納されるベースディレクトリのパス。
* `cache_path` ([String](../../sql-reference/data-types/)) — ソースファイル内のファイルセグメントに対応する、個々のキャッシュファイルのパス
* `key` ([String](../../sql-reference/data-types/)) — ファイルセグメントのキャッシュキー
* `file_segment_range_begin` ([UInt64](../../sql-reference/data-types/)) — ファイルセグメント範囲の先頭に対応するオフセット
* `file_segment_range_end` ([UInt64](../../sql-reference/data-types/)) — ファイルセグメント範囲の末尾 (この値を含む) に対応するオフセット
* `size` ([UInt64](../../sql-reference/data-types/)) — ファイルセグメントのサイズ
* `state` ([String](../../sql-reference/data-types/)) — ファイルセグメントの状態 (DOWNLOADED、DOWNLOADING、PARTIALLY&#95;DOWNLOADED、...)
* `finished_download_time` ([DateTime](../../sql-reference/data-types/)) — ファイルセグメントのダウンロードが完了した時刻。
* `cache_hits` ([UInt64](../../sql-reference/data-types/)) — 対応するファイルセグメントのキャッシュヒット数
* `references` ([UInt64](../../sql-reference/data-types/)) — 対応するファイルセグメントへの参照数。値が 1 の場合、現時点ではどこからも使用されていません (唯一存在する参照はキャッシュストレージ自体によるものです)
* `downloaded_size` ([UInt64](../../sql-reference/data-types/)) — ファイルセグメントのダウンロード済みサイズ
* `kind` ([String](../../sql-reference/data-types/)) — ファイルセグメントの種別 (&#39;Temporary data in cache&#39; の一部として追加されたファイルセグメントを区別するために使用されます)
* `unbound` ([UInt8](../../sql-reference/data-types/)) — 内部実装フラグ
* `user_id` ([String](../../sql-reference/data-types/)) — ファイルセグメントを作成したユーザーの ID
* `segment_type` ([String](../../sql-reference/data-types/)) — セグメントの種類。データファイル (`.json`、`.txt` など) とデータファイル (`.bin`、マークファイル) を区別するために使用されます。
* `file_size` ([Nullable(UInt64)](../../sql-reference/data-types/)) — 現在のファイルセグメントが属するファイルのサイズ

{/*AUTOGENERATED_END*/ }
