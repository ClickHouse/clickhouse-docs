## 説明 \{#purpose\}

[MergeTree](../../engines/table-engines/mergetree-family/mergetree.md) テーブルのデタッチされたパーツに関する情報を含みます。`reason` 列は、そのパーツがデタッチされた理由を示します。

ユーザーによってデタッチされたパーツの場合、`reason` 列は空です。このようなパーツは [ALTER TABLE ATTACH PARTITION|PART](/sql-reference/statements/alter/partition#attach-partitionpart) コマンドで再度アタッチできます。

他の列の説明については、[system.parts](../../operations/system-tables/parts.md) を参照してください。

パーツ名が不正な場合、いくつかの列の値は `NULL` になることがあります。このようなパーツは [ALTER TABLE DROP DETACHED PART](/sql-reference/statements/alter/partition#drop-detached-partitionpart) で削除できます。

## カラム \{#columns\}

{/*AUTOGENERATED_START*/ }

* `database` ([String](../../sql-reference/data-types/)) — このパーツが属するデータベース名。
* `table` ([String](../../sql-reference/data-types/)) — このパーツが属するテーブル名。
* `partition_id` ([Nullable(String)](../../sql-reference/data-types/)) — このパーツが属するパーティションの識別子。
* `name` ([String](../../sql-reference/data-types/)) — このパーツの名前。
* `bytes_on_disk` ([UInt64](../../sql-reference/data-types/)) — すべてのデータパーツファイルの合計サイズ (バイト単位) 。
* `modification_time` ([DateTime](../../sql-reference/data-types/)) — データパーツを含むディレクトリが変更された時刻。通常、これは detach が発生した時刻に対応します。
* `disk` ([String](../../sql-reference/data-types/)) — このデータパーツを保存しているディスクの名前。
* `path` ([String](../../sql-reference/data-types/)) — このデータパーツのファイルへのディスク上のパス。
* `reason` ([Nullable(String)](../../sql-reference/data-types/)) — このパーツが detach された理由。
* `min_block_number` ([Nullable(Int64)](../../sql-reference/data-types/)) — マージ後の現在のパーツを構成するデータパーツの最小番号。
* `max_block_number` ([Nullable(Int64)](../../sql-reference/data-types/)) — マージ後の現在のパーツを構成するデータパーツの最大番号。
* `level` ([Nullable(UInt32)](../../sql-reference/data-types/)) — マージツリーの深さ。0 は、現在のパーツが他のパーツのマージではなく insert によって作成されたことを意味します。

{/*AUTOGENERATED_END*/ }
