## 説明 \{#description\}

[クォータ](../../operations/system-tables/quotas.md)に関する情報を含みます。

## カラム \{#columns\}

{/*AUTOGENERATED_START*/ }

* `name` ([String](../../sql-reference/data-types/)) — クォータ名。
* `id` ([UUID](../../sql-reference/data-types/)) — クォータ ID。
* `storage` ([String](../../sql-reference/data-types/)) — クォータの保存先。指定可能な値: users.xml ファイルでクォータが設定されている場合は &#39;users.xml&#39;、SQLクエリでクォータが設定されている場合は &#39;disk&#39;。
* `keys` ([Array(Enum8(&#39;user&#95;name&#39; = 1, &#39;ip&#95;address&#39; = 2, &#39;forwarded&#95;ip&#95;address&#39; = 3, &#39;client&#95;key&#39; = 4, &#39;normalized&#95;query&#95;hash&#39; = 7))](../../sql-reference/data-types/)) — キーは、クォータをどのように共有するかを指定します。2 つの接続が同じクォータとキーを使用する場合、それらは同じリソース量を共有します。値: [] — すべてのユーザーが同じクォータを共有します、[&#39;user&#95;name&#39;] — 同じユーザー名の接続は同じクォータを共有します、[&#39;ip&#95;address&#39;] — 同じ IP アドレスからの接続は同じクォータを共有します、[&#39;client&#95;key&#39;] — 同じキーを持つ接続は同じクォータを共有します。キーはクライアントが明示的に指定する必要があります。clickhouse-client を使用する場合は、--quota&#95;key パラメータにキーの値を渡すか、クライアントの設定ファイルで quota&#95;key パラメータを使用します。HTTP インターフェイスを使用する場合は、X-ClickHouse-Quota ヘッダーを使用します、[&#39;user&#95;name&#39;, &#39;client&#95;key&#39;] — 同じ client&#95;key を持つ接続は同じクォータを共有します。クライアントからキーが指定されていない場合、クォータは `user_name` ごとに追跡されます、[&#39;client&#95;key&#39;, &#39;ip&#95;address&#39;] — 同じ client&#95;key を持つ接続は同じクォータを共有します。クライアントからキーが指定されていない場合、クォータは ip&#95;address ごとに追跡されます。
* `durations` ([Array(UInt32)](../../sql-reference/data-types/)) — 秒単位の時間インターバルの長さ。
* `apply_to_all` ([UInt8](../../sql-reference/data-types/)) — 論理値。クォータがどのユーザーに適用されるかを示します。値: 0 — クォータは apply&#95;to&#95;list で指定されたユーザーに適用されます。1 — クォータは apply&#95;to&#95;except にリストされたユーザーを除くすべてのユーザーに適用されます。
* `apply_to_list` ([Array(String)](../../sql-reference/data-types/)) — クォータを適用するユーザー名/ロールの一覧。
* `apply_to_except` ([Array(String)](../../sql-reference/data-types/)) — クォータを適用しないユーザー名/ロールの一覧。
