## 説明 \{#description\}

すべての `BACKUP` または `RESTORE` 操作と、その現在の状態およびその他のプロパティの一覧を含みます。なお、このテーブルは永続化されず、最後にサーバーを再起動して以降に実行された操作のみが表示されます。

## カラム \{#columns\}

{/*AUTOGENERATED_START*/ }

* `id` ([String](../../sql-reference/data-types/)) — 操作 ID。`SETTINGS id=...` で指定することも、ランダムに生成された UUID を使用することもできます。
* `name` ([String](../../sql-reference/data-types/)) — 操作名。`Disk('backups', 'my_backup')` のような文字列です。
* `base_backup_name` ([String](../../sql-reference/data-types/)) — ベースバックアップ操作名。`Disk('backups', 'my_base_backup')` のような文字列です。
* `query_id` ([String](../../sql-reference/data-types/)) — バックアップを開始したクエリの Query ID。
* `status` ([Enum8(&#39;CREATING&#95;BACKUP&#39; = 0, &#39;BACKUP&#95;CREATED&#39; = 1, &#39;BACKUP&#95;FAILED&#39; = 2, &#39;RESTORING&#39; = 3, &#39;RESTORED&#39; = 4, &#39;RESTORE&#95;FAILED&#39; = 5, &#39;BACKUP&#95;CANCELLED&#39; = 6, &#39;RESTORE&#95;CANCELLED&#39; = 7)](../../sql-reference/data-types/)) — バックアップまたは復元操作のステータス。
* `error` ([String](../../sql-reference/data-types/)) — エラーメッセージ (存在する場合) 。
* `start_time` ([DateTime64(6)](../../sql-reference/data-types/)) — 操作の開始時刻。
* `end_time` ([DateTime64(6)](../../sql-reference/data-types/)) — 操作の終了時刻。
* `num_files` ([UInt64](../../sql-reference/data-types/)) — バックアップに保存されているファイル数。
* `total_size` ([UInt64](../../sql-reference/data-types/)) — バックアップに保存されているファイルの合計サイズ。
* `num_entries` ([UInt64](../../sql-reference/data-types/)) — バックアップ内のエントリ数。つまり、バックアップがフォルダーとして保存されている場合は、そのフォルダー内のファイル数です。
* `uncompressed_size` ([UInt64](../../sql-reference/data-types/)) — バックアップの非圧縮サイズ。
* `compressed_size` ([UInt64](../../sql-reference/data-types/)) — バックアップの圧縮サイズ。
* `files_read` ([UInt64](../../sql-reference/data-types/)) — このバックアップから RESTORE 中に読み取られたファイル数を返します。
* `bytes_read` ([UInt64](../../sql-reference/data-types/)) — このバックアップから RESTORE 中に読み取られたファイルの合計サイズを返します。
* `ProfileEvents` ([Map(LowCardinality(String), UInt64)](../../sql-reference/data-types/)) — この操作中に収集されたすべての profile events。

{/*AUTOGENERATED_END*/ }
