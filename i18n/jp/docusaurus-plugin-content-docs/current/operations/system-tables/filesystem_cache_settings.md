## 説明 \{#description\}

すべてのファイルシステムキャッシュの設定に関する情報を含みます

## カラム \{#columns\}

{/*AUTOGENERATED_START*/ }

* `cache_name` ([String](../../sql-reference/data-types/)) — キャッシュ名
* `path` ([String](../../sql-reference/data-types/)) — キャッシュディレクトリのパス
* `max_size` ([UInt64](../../sql-reference/data-types/)) — キャッシュの最大サイズ
* `max_elements` ([UInt64](../../sql-reference/data-types/)) — キャッシュ要素の最大数。たとえばファイルセグメント (ファイルシステム上のファイル数を制限)
* `max_file_segment_size` ([UInt64](../../sql-reference/data-types/)) — 1 つのファイルセグメントの最大サイズ
* `boundary_alignment` ([UInt64](../../sql-reference/data-types/)) — ファイルセグメントのアラインメント
* `cache_on_write_operations` ([UInt8](../../sql-reference/data-types/)) — ライトスルーキャッシュを有効にします (INSERT および MERGE 時にキャッシュ)
* `cache_policy` ([String](../../sql-reference/data-types/)) — キャッシュのエビクションポリシー
* `slru_size_ratio` ([Float64](../../sql-reference/data-types/)) — 保護要素と試用要素のサイズ比を表す SLRU キャッシュポリシーの比率
* `background_download_threads` ([UInt64](../../sql-reference/data-types/)) — バックグラウンドダウンロードスレッドの数。値 0 はバックグラウンドダウンロードを無効にします
* `background_download_queue_size_limit` ([UInt64](../../sql-reference/data-types/)) — バックグラウンドダウンロードキューのサイズ。値 0 はバックグラウンドダウンロードを無効にします
* `background_download_max_file_segment_size` ([UInt64](../../sql-reference/data-types/)) — バックグラウンドダウンロードでダウンロードできる最大サイズ
* `load_metadata_threads` ([UInt64](../../sql-reference/data-types/)) — サーバー起動時にキャッシュメタデータを読み込むスレッド数。値 0 はメタデータの非同期読み込みを無効にします
* `load_metadata_asynchronously` ([UInt8](../../sql-reference/data-types/)) — サーバー起動時のメタデータの非同期読み込みを有効にします
* `keep_free_space_size_ratio` ([Float64](../../sql-reference/data-types/)) — キャッシュがバックグラウンドで維持しようとする空き容量の比率
* `keep_free_space_elements_ratio` ([Float64](../../sql-reference/data-types/)) — キャッシュがバックグラウンドで維持しようとする空き要素数の比率
* `keep_free_space_remove_batch` ([UInt64](../../sql-reference/data-types/)) — 空き容量/空き要素比率を維持するバックグラウンドスレッドによるキャッシュ要素削除のバッチサイズ
* `enable_filesystem_query_cache_limit` ([UInt8](../../sql-reference/data-types/)) — クエリ内で書き込めるキャッシュの最大サイズ制限を有効にします
* `cache_hits_threshold` ([UInt64](../../sql-reference/data-types/)) — 非推奨の設定
* `enable_bypass_cache_with_threshold` ([UInt8](../../sql-reference/data-types/)) — 文書化されていません。使用は推奨されません
* `bypass_cache_threshold` ([UInt64](../../sql-reference/data-types/)) — 文書化されていません。使用は推奨されません
* `write_cache_per_user_id_directory` ([UInt8](../../sql-reference/data-types/)) — ClickHouse Cloud の内部設定
* `allow_dynamic_cache_resize` ([UInt8](../../sql-reference/data-types/)) — ファイルシステムキャッシュの動的リサイズを許可します
* `max_size_ratio_to_total_space` ([Float64](../../sql-reference/data-types/)) — 総ディスク容量に対する `max_size` の比率
* `skip_cache_on_disk_failure` ([UInt8](../../sql-reference/data-types/)) — true の場合、ディスク IO エラー時にファイルシステムキャッシュ操作を静かにバイパスします。false (デフォルト) の場合、ディスク IO エラーは起動失敗として伝播されます。
* `use_split_cache` ([UInt8](../../sql-reference/data-types/)) — system/data 用にファイルを分離して使用します。
* `split_cache_ratio` ([Float64](../../sql-reference/data-types/)) — split&#95;cache における、キャッシュ総サイズに対する system セグメントの比率。
* `overcommit_eviction_evict_step` ([UInt64](../../sql-reference/data-types/)) — overcommit eviction ポリシーにおけるバイト単位のエビクションステップ。keep&#95;free&#95;space&#95;*&#95;ratio 設定で使用されます
* `check_cache_probability` ([Float64](../../sql-reference/data-types/)) — デバッグまたは sanitizer build でのみ動作します。キャッシュ全体を走査して各キャッシュ要素の状態を確認し、キャッシュの整合性を検証します
* `is_initialized` ([UInt8](../../sql-reference/data-types/)) — キャッシュが正常に初期化されたかどうかを示します
* `current_size` ([UInt64](../../sql-reference/data-types/)) — 現在のキャッシュサイズ
* `current_elements_num` ([UInt64](../../sql-reference/data-types/)) — 現在のキャッシュ要素数 (ファイルセグメント)

{/*AUTOGENERATED_END*/ }
