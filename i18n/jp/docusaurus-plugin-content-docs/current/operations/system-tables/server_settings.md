## 説明 \{#description\}

`config.xml` に指定されているサーバーのグローバル設定に関する情報を保持します。
現在、このテーブルに表示されるのは `config.xml` の最上位階層にある設定のみで、入れ子になった設定 (例: [logger](../../operations/server-configuration-parameters/settings.md#logger)) には対応していません。

## カラム \{#columns\}

{/*AUTOGENERATED_START*/ }

* `name` ([String](../../sql-reference/data-types/)) — サーバー設定の名前。
* `value` ([String](../../sql-reference/data-types/)) — サーバー設定の値。
* `default` ([String](../../sql-reference/data-types/)) — サーバー設定のデフォルト値。
* `changed` ([UInt8](../../sql-reference/data-types/)) — config.xml で設定が指定されているかどうかを示します
* `description` ([String](../../sql-reference/data-types/)) — サーバー設定の簡潔な説明。
* `type` ([String](../../sql-reference/data-types/)) — サーバー設定の値の型。
* `changeable_without_restart` ([Enum8(&#39;No&#39; = 0, &#39;IncreaseOnly&#39; = 1, &#39;DecreaseOnly&#39; = 2, &#39;Yes&#39; = 3)](../../sql-reference/data-types/)) — 設定を実行時に変更できるかどうかを示します。
* `is_obsolete` ([UInt8](../../sql-reference/data-types/)) — 設定が廃止済みかどうかを示します。

{/*AUTOGENERATED_END*/ }

## 例 \{#example\}

次の例では、名前に `thread_pool` を含むサーバー設定の情報を取得する方法を示します。

```sql
SELECT *
FROM system.server_settings
WHERE name LIKE '%thread_pool%'
```
