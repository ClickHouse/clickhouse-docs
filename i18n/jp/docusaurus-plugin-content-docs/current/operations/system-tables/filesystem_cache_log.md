リモートファイルシステム上のオブジェクトに対するファイルシステムキャッシュで発生したすべてのイベントの履歴が記録されます。

このテーブルは、いつでも TRUNCATE または DROP しても安全です。

カラム:

{/*AUTOGENERATED_START*/ }

* `hostname` ([LowCardinality(String)](/sql-reference/data-types/lowcardinality)) — ホスト名
* `event_date` ([Date](/sql-reference/data-types/date)) — イベント日付
* `event_time` ([DateTime](/sql-reference/data-types/datetime)) — イベント時刻
* `query_id` ([String](/sql-reference/data-types/string)) — クエリの ID
* `source_file_path` ([String](/sql-reference/data-types/string)) — ファイルシステム上のファイルセグメントのパス
* `file_segment_range` ([Tuple(UInt64, UInt64)](/sql-reference/data-types/tuple)) — ファイルセグメントの範囲
* `total_requested_range` ([Tuple(UInt64, UInt64)](/sql-reference/data-types/tuple)) — 読み込み対象の全範囲
* `key` ([String](/sql-reference/data-types/string)) — ファイルセグメントのキー
* `offset` ([UInt64](/sql-reference/data-types/int-uint)) — ファイルセグメントのオフセット
* `size` ([UInt64](/sql-reference/data-types/int-uint)) — 読み込みサイズ
* `read_type` ([String](/sql-reference/data-types/string)) — 読み込みタイプ: READ&#95;FROM&#95;CACHE, READ&#95;FROM&#95;FS&#95;AND&#95;DOWNLOADED&#95;TO&#95;CACHE, READ&#95;FROM&#95;FS&#95;BYPASSING&#95;CACHE
* `read_from_cache_attempted` ([UInt8](/sql-reference/data-types/int-uint)) — キャッシュからの読み込みが試行されたかどうか
* `read_buffer_id` ([String](/sql-reference/data-types/string)) — 内部実装の読み込みバッファ ID
* `user_id` ([String](/sql-reference/data-types/string)) — ファイルセグメントを作成したユーザーの ID

{/*AUTOGENERATED_END*/ }
