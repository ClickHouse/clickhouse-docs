## 説明 \{#description\}

エラーコードと、それぞれが発生した回数を含みます。

まだ発生していないものも含めて、想定されるすべてのエラーコードを表示するには、設定 [system&#95;events&#95;show&#95;zero&#95;values](../settings/settings.md#system_events_show_zero_values) を 1 に設定します。

## カラム \{#columns\}

{/*AUTOGENERATED_START*/ }

* `name` ([String](../../sql-reference/data-types/)) — エラー名 (errorCodeToName) 。
* `code` ([Int32](../../sql-reference/data-types/)) — エラーコード。
* `value` ([UInt64](../../sql-reference/data-types/)) — このエラーが発生した回数。
* `last_error_time` ([DateTime](../../sql-reference/data-types/)) — 最後にエラーが発生した時刻。
* `last_error_message` ([String](../../sql-reference/data-types/)) — 最後に発生したエラーのメッセージ。
* `last_error_format_string` ([String](../../sql-reference/data-types/)) — 最後に発生したエラーのフォーマット文字列。
* `last_error_trace` ([Array(UInt64)](../../sql-reference/data-types/)) — 呼び出されたメソッドが格納されている物理アドレスの一覧を表すスタックトレース。
* `remote` ([UInt8](../../sql-reference/data-types/)) — リモート例外 (つまり、分散クエリのいずれかで受信した例外) 。
* `query_id` ([String](../../sql-reference/data-types/)) — エラーの原因となったクエリの ID (利用可能な場合) 。

{/*AUTOGENERATED_END*/ }

:::note
一部のエラーでは、クエリが正常に実行されている場合でもカウンターが増加することがあります。対応するエラーで誤検知が発生しないと確信できる場合を除き、このテーブルをサーバー監視の用途に使用することは推奨されません。
:::

## 例 \{#example\}

```sql title="Query"
SELECT name, code, value
FROM system.errors
WHERE value > 0
ORDER BY code ASC
LIMIT 1

┌─name─────────────┬─code─┬─value─┐
│ CANNOT_OPEN_FILE │   76 │     1 │
└──────────────────┴──────┴───────┘
```

```sql title="Response"
WITH arrayMap(x -> demangle(addressToSymbol(x)), last_error_trace) AS all
SELECT name, arrayStringConcat(all, '\n') AS res
FROM system.errors
LIMIT 1
SETTINGS allow_introspection_functions=1\G
```
