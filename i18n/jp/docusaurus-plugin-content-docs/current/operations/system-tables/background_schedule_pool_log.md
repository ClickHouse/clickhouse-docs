## 説明 \{#description\}

`system.background_schedule_pool_log` テーブルは、[background&#95;schedule&#95;pool&#95;log](/operations/server-configuration-parameters/settings#background_schedule_pool_log) サーバー設定が指定されている場合にのみ作成されます。

このテーブルには、バックグラウンドスケジュールプールで実行されたタスクの履歴が含まれます。バックグラウンドスケジュールプールは、分散送信、バッファフラッシュ、メッセージブローカーの操作といった定期タスクの実行に使用されます。

## カラム \{#columns\}

{/*AUTOGENERATED_START*/ }

* `hostname` ([LowCardinality(String)](/sql-reference/data-types/lowcardinality)) — クエリを実行するサーバーのホスト名。
* `event_date` ([Date](/sql-reference/data-types/date)) — イベントの日付。
* `event_time` ([DateTime](/sql-reference/data-types/datetime)) — イベントの時刻。
* `event_time_microseconds` ([DateTime64(6)](/sql-reference/data-types/datetime64)) — マイクロ秒精度のイベント時刻。
* `query_id` ([String](/sql-reference/data-types/string)) — バックグラウンドタスクに関連付けられたクエリの識別子。
* `database` ([LowCardinality(String)](/sql-reference/data-types/lowcardinality)) — データベース名。
* `table` ([LowCardinality(String)](/sql-reference/data-types/lowcardinality)) — テーブル名。
* `table_uuid` ([UUID](/sql-reference/data-types/uuid)) — バックグラウンドタスクが属するテーブルの UUID。
* `log_name` ([LowCardinality(String)](/sql-reference/data-types/lowcardinality)) — バックグラウンドタスクの名前。
* `duration_ms` ([UInt64](/sql-reference/data-types/int-uint)) — タスクの実行時間 (ミリ秒) 。
* `error` ([UInt16](/sql-reference/data-types/int-uint)) — 発生した例外のエラーコード。
* `exception` ([String](/sql-reference/data-types/string)) — 発生したエラーのテキストメッセージ。

{/*AUTOGENERATED_END*/ }

`system.background_schedule_pool_log` テーブルは、初回のバックグラウンドタスク実行後に作成されます。
