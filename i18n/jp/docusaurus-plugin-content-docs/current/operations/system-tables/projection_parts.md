## 説明 \{#description\}

このテーブルには、MergeTree ファミリーのテーブルにおけるプロジェクションパーツに関する情報が格納されています。

## 列 \{#columns\}

{/*AUTOGENERATED_START*/ }

* `partition` ([String](../../sql-reference/data-types/)) — パーティション名。
* `name` ([String](../../sql-reference/data-types/)) — データパートの名前。
* `part_type` ([String](../../sql-reference/data-types/)) — データパートの格納フォーマット。設定可能な値: Wide (カラムごとに1つのファイル) および Compact (すべてのカラムを1つのファイルに格納) 。
* `parent_name` ([String](../../sql-reference/data-types/)) — ソース (親) データパートの名前。
* `parent_uuid` ([UUID](../../sql-reference/data-types/)) — ソース (親) のデータパートの UUID。
* `parent_part_type` ([String](../../sql-reference/data-types/)) — ソース (親) データパートの格納フォーマット。
* `active` ([UInt8](../../sql-reference/data-types/)) — データパーツがアクティブかどうかを示すフラグです。データパーツがアクティブな場合、そのパーツはテーブルで使用中です。そうでない場合は、まもなく削除されます。非アクティブなデータパーツは、マージおよびミューテーション操作の後に現れます。
* `marks` ([UInt64](../../sql-reference/data-types/)) — マークの数。データパート内のおおよその行数を求めるには、marks に index granularity (通常は 8192) を掛けます (この方法は adaptive granularity では機能しません) 。
* `rows` ([UInt64](../../sql-reference/data-types/)) — 行の数。
* `bytes_on_disk` ([UInt64](../../sql-reference/data-types/)) — すべてのデータパートファイルの総サイズ (バイト単位) 。
* `data_compressed_bytes` ([UInt64](../../sql-reference/data-types/)) — data part 内の圧縮データの合計サイズです。補助ファイル (たとえば、mark ファイル) は含まれません。
* `data_uncompressed_bytes` ([UInt64](../../sql-reference/data-types/)) — データパート内の非圧縮データの合計サイズです。マークファイルなどの補助ファイルは含まれません。
* `marks_bytes` ([UInt64](../../sql-reference/data-types/)) — マークを含むファイルのサイズ。
* `parent_marks` ([UInt64](../../sql-reference/data-types/)) — ソース (親) частьに含まれる마크の数。
* `parent_rows` ([UInt64](../../sql-reference/data-types/)) — 元の (親) パート内の行数。
* `parent_bytes_on_disk` ([UInt64](../../sql-reference/data-types/)) — すべての元 (親) データパートファイルの合計サイズ (バイト単位) 。
* `parent_data_compressed_bytes` ([UInt64](../../sql-reference/data-types/)) — ソース (親) データパートの圧縮データの合計サイズ。
* `parent_data_uncompressed_bytes` ([UInt64](../../sql-reference/data-types/)) — ソース (親) のデータパート内にある非圧縮データの合計サイズ。
* `parent_marks_bytes` ([UInt64](../../sql-reference/data-types/)) — ソース (親) データパート内のマークファイルのサイズ。
* `modification_time` ([DateTime](../../sql-reference/data-types/)) — データパートが格納されているディレクトリが更新された時刻です。通常、これはデータパートの作成時刻に対応します。
* `remove_time` ([DateTime](../../sql-reference/data-types/)) — データパートが非アクティブになった日時。
* `refcount` ([UInt32](../../sql-reference/data-types/)) — データパートが使用されている箇所の数。値が 2 より大きい場合、そのデータパートはクエリまたはマージ処理で使用されています。
* `min_date` ([Date](../../sql-reference/data-types/)) — データパート内の日付キーの最小値。
* `max_date` ([Date](../../sql-reference/data-types/)) — データパートにおける日付キーの最大値。
* `min_time` ([DateTime](../../sql-reference/data-types/)) — データパートにおける日時キーの最小値。
* `max_time` ([DateTime](../../sql-reference/data-types/)) — data part内の日時キーの最大値。
* `partition_id` ([String](../../sql-reference/data-types/)) — パーティションのID。
* `min_block_number` ([Int64](../../sql-reference/data-types/)) — マージ後に現在のパーツを構成するデータパーツのうち、最小の番号。
* `max_block_number` ([Int64](../../sql-reference/data-types/)) — マージ後の現在のパーツを構成するデータパーツの最大番号。
* `level` ([UInt32](../../sql-reference/data-types/)) — マージツリーの深さ。0 は、現在のパーツが他のパーツのマージではなく、insert によって作成されたことを意味します。
* `data_version` ([UInt64](../../sql-reference/data-types/)) — データパートに適用すべき mutation (`data_version` より大きいバージョンの mutation) を判定するために使用される番号。
* `primary_key_bytes_in_memory` ([UInt64](../../sql-reference/data-types/)) — 主キーの値が使用するメモリ量 (バイト単位) 。
* `primary_key_bytes_in_memory_allocated` ([UInt64](../../sql-reference/data-types/)) — 主キーの値のために確保されたメモリ量 (バイト単位) 。
* `is_frozen` ([UInt8](../../sql-reference/data-types/)) — パーティションデータのバックアップが存在するかどうかを示すフラグです。1 はバックアップが存在することを示し、0 はバックアップが存在しないことを示します。
* `database` ([String](../../sql-reference/data-types/)) — データベース名。
* `table` ([String](../../sql-reference/data-types/)) — テーブル名。
* `engine` ([String](../../sql-reference/data-types/)) — パラメータを含まないテーブルエンジン名。
* `disk_name` ([String](../../sql-reference/data-types/)) — データパートを格納するディスクの名前。
* `path` ([String](../../sql-reference/data-types/)) — データパーツのファイルが格納されているフォルダーへの絶対パス。
* `hash_of_all_files` ([String](../../sql-reference/data-types/)) — 圧縮ファイルに対する sipHash128。
* `hash_of_uncompressed_files` ([String](../../sql-reference/data-types/)) — 非圧縮ファイル (マークを含むファイル、索引ファイルなど) の sipHash128。
* `uncompressed_hash_of_compressed_files` ([String](../../sql-reference/data-types/)) — 圧縮ファイル内のデータを、展開後のデータとみなして計算した sipHash128。
* `delete_ttl_info_min` ([DateTime](../../sql-reference/data-types/)) — TTL DELETE ルールにおける日付と時刻キーの最小値。
* `delete_ttl_info_max` ([DateTime](../../sql-reference/data-types/)) — 有効期限 (TTL) の DELETE ルールにおける日付と時刻のキーの最大値。
* `move_ttl_info.expression` ([Array(String)](../../sql-reference/data-types/)) — 式の配列。各式は TTL MOVE ルールを表します。
* `move_ttl_info.min` ([Array(DateTime)](../../sql-reference/data-types/)) — 日時の値の配列。各要素は、TTL MOVE ルールの最小キー値を表します。
* `move_ttl_info.max` ([Array(DateTime)](../../sql-reference/data-types/)) — 日付と時刻の値の配列。各要素は、有効期限 (TTL) MOVE ルールにおける最大キー値を表します。
* `default_compression_codec` ([String](../../sql-reference/data-types/)) — このデータパートの圧縮に使用されるコーデックの名前です (カラムに明示的なコーデックが指定されていない場合) 。
* `recompression_ttl_info.expression` ([Array(String)](../../sql-reference/data-types/)) — 有効期限 (TTL) の式。
* `recompression_ttl_info.min` ([Array(DateTime)](../../sql-reference/data-types/)) — このパート内で計算された有効期限 (TTL) 式の最小値。有効期限 (TTL) が切れた行が少なくとも1行存在するかどうかを判断するために使用されます。
* `recompression_ttl_info.max` ([Array(DateTime)](../../sql-reference/data-types/)) — このパート内で計算されたTTL式の最大値です。有効期限 (TTL) が切れたすべての行が含まれているかどうかを確認するために使用されます。
* `group_by_ttl_info.expression` ([Array(String)](../../sql-reference/data-types/)) — 有効期限 (TTL) の式。
* `group_by_ttl_info.min` ([Array(DateTime)](../../sql-reference/data-types/)) — このパート内で計算された TTL 式の最小値。有効期限 (TTL) が切れた行が少なくとも 1 行あるかどうかを判断するために使用されます。
* `group_by_ttl_info.max` ([Array(DateTime)](../../sql-reference/data-types/)) — この part 内で計算された TTL 式の最大値です。有効期限 (TTL) が切れたすべての行が含まれているかどうかを確認するために使用されます。
* `rows_where_ttl_info.expression` ([Array(String)](../../sql-reference/data-types/)) — 有効期限 (TTL) の式。
* `rows_where_ttl_info.min` ([Array(DateTime)](../../sql-reference/data-types/)) — このパート内で計算された有効期限 (TTL) 式の最小値です。少なくとも1つの行の有効期限 (TTL) が期限切れになっているかどうかを把握するために使用されます。
* `rows_where_ttl_info.max` ([Array(DateTime)](../../sql-reference/data-types/)) — このパート内で計算された TTL 式の最大値です。有効期限 (TTL) が切れたすべての行を含んでいるかどうかを判断するために使用されます。
* `is_broken` ([UInt8](../../sql-reference/data-types/)) — プロジェクション part が破損しているかどうか
* `exception_code` ([Int32](../../sql-reference/data-types/)) — プロジェクションパートの破損状態を示す例外メッセージ
* `exception` ([String](../../sql-reference/data-types/)) — プロジェクションパートの破損状態を示す例外コード

{/*AUTOGENERATED_END*/ }

**別名:**

* `bytes` — `bytes_on_disk` のエイリアスです。
* `marks_size` — `marks_bytes` のエイリアスです。
* `part_name` — `name` のエイリアスです。
