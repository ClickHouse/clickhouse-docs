## 説明 \{#description\}

サーバーでサポートされている Dictionary ソースの一覧と、各ソースに関する組み込みドキュメントが含まれています。Dictionary ソースは、Dictionary のデータをどこから読み込むかを決めるもので、`CREATE DICTIONARY` クエリの `SOURCE` 句で指定します。

## カラム \{#columns\}

{/*AUTOGENERATED_START*/ }

* `name` ([String](../../sql-reference/data-types/)) — SOURCE 句で指定する Dictionary ソースの名前。
* `description` ([String](../../sql-reference/data-types/)) — Dictionary ソースの役割の概要。
* `syntax` ([String](../../sql-reference/data-types/)) — ソースを指定するための SOURCE 句の構造。なお、一部の Dictionary ソースは、Dictionary をサーバー設定ファイルではなく DDL クエリから作成する場合、アクセス制御の対象となります。詳細については、各ソースの `description` を参照してください。
* `examples` ([String](../../sql-reference/data-types/)) — 使用例。
* `introduced_in` ([String](../../sql-reference/data-types/)) — そのソースが最初に導入された ClickHouse のバージョン (major.minor 形式) 。
* `related` ([Array(String)](../../sql-reference/data-types/)) — 関連する Dictionary ソースの名前。

{/*AUTOGENERATED_END*/ }

## 例 \{#example\}

```sql title="Query"
SELECT name, syntax
FROM system.dictionary_sources
WHERE name IN ('clickhouse', 'file')
ORDER BY name
```

```text title="Response"
┌─name───────┬─syntax─────────────────────────────────────────────────────────────────────────┐
│ clickhouse │ SOURCE(CLICKHOUSE(host 'host' port 9000 user 'default' password '' db 'db' table 'table')) │
│ file       │ SOURCE(FILE(path '/path/to/file' format 'CSV'))                                  │
└────────────┴────────────────────────────────────────────────────────────────────────────────┘
```
