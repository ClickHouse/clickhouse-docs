## 説明 \{#description\}

`system.part_log` テーブルは、[part&#95;log](/operations/server-configuration-parameters/settings#part_log) サーバー設定が指定されている場合にのみ作成されます。

このテーブルには、データの追加やマージなど、[MergeTree](../../engines/table-engines/mergetree-family/mergetree.md) ファミリーテーブルの[データパーツ](../../engines/table-engines/mergetree-family/custom-partitioning-key.md)で発生したイベントに関する情報が含まれています。

`system.part_log` テーブルには次のカラムが含まれます:

## カラム \{#columns\}

{/*AUTOGENERATED_START*/ }

* `hostname` ([LowCardinality(String)](/sql-reference/data-types/lowcardinality)) — クエリを実行しているサーバーのホスト名。
* `query_id` ([String](/sql-reference/data-types/string)) — このデータパートを作成した INSERT クエリの識別子。
* `event_type` ([Enum8(&#39;NewPart&#39; = 1, &#39;MergeParts&#39; = 2, &#39;DownloadPart&#39; = 3, &#39;RemovePart&#39; = 4, &#39;MutatePart&#39; = 5, &#39;MovePart&#39; = 6, &#39;MergePartsStart&#39; = 7, &#39;MutatePartStart&#39; = 8)](/sql-reference/data-types/enum)) — データパーツに対して発生したイベントの種類。取り得る値は次のとおりです: NewPart — 新しいデータパーツの挿入、MergePartsStart — データパーツのマージ開始、MergeParts — データパーツのマージ完了、DownloadPart — データパーツのダウンロード、RemovePart — [DETACH PARTITION](/sql-reference/statements/alter/partition#detach-partitionpart) を使用したデータパーツの削除またはデタッチ、MutatePartStart — データパーツのミューテーション開始、MutatePart — データパーツのミューテーション完了、MovePart — データパーツをあるディスクから別のディスクへ移動。
* `merge_reason` ([Enum8(&#39;NotAMerge&#39; = 1, &#39;RegularMerge&#39; = 2, &#39;TTLDeleteMerge&#39; = 3, &#39;TTLRecompressMerge&#39; = 4, &#39;TTLDropMerge&#39; = 5)](/sql-reference/data-types/enum)) — 型 `MERGE&#95;PARTS` のイベントが発生した理由を表します。次のいずれかの値を取ります: NotAMerge — 現在のイベントの型は `MERGE&#95;PARTS` 以外です。RegularMerge — 通常のマージです。TTLDeleteMerge、TTLDropMerge — 有効期限が切れたデータをクリーンアップします。TTLRecompressMerge — データパーツを再圧縮します。
* `merge_algorithm` ([Enum8(&#39;Undecided&#39; = 0, &#39;Vertical&#39; = 1, &#39;Horizontal&#39; = 2)](/sql-reference/data-types/enum)) — type が MERGE&#95;PARTS であるイベントのマージアルゴリズム。値には次のいずれかを指定できます: Undecided, Horizontal, Vertical
* `event_date` ([Date](/sql-reference/data-types/date)) — イベントの日付。
* `event_time` ([DateTime](/sql-reference/data-types/datetime)) — イベントの時刻。
* `event_time_microseconds` ([DateTime64(6)](/sql-reference/data-types/datetime64)) — マイクロ秒精度のイベント時刻。
* `duration_ms` ([UInt64](/sql-reference/data-types/int-uint)) — この操作の実行時間。
* `database` ([String](/sql-reference/data-types/string)) — データパートが含まれるデータベース名。
* `table` ([String](/sql-reference/data-types/string)) — データパートが格納されているテーブル名。
* `table_uuid` ([UUID](/sql-reference/data-types/uuid)) — データパートが属するテーブルのUUID。
* `part_name` ([String](/sql-reference/data-types/string)) — データパート名。
* `partition_id` ([String](/sql-reference/data-types/string)) — データパートが挿入されたパーティションの ID です。パーティション化が `tuple()` の場合、このカラムの値は `all` になります。
* `partition` ([String](/sql-reference/data-types/string)) — パーティションの名前。
* `part_type` ([String](/sql-reference/data-types/string)) — part のタイプです。設定可能な値: Wide および Compact。
* `part_storage_type` ([String](/sql-reference/data-types/string)) — DataPartStorage のタイプ。設定可能な値: Packed - すべてのファイルが 1 つのブロブに格納される、Full - ファイルごとに 1 つのブロブが格納される。
* `disk_name` ([String](/sql-reference/data-types/string)) — データパートが存在するディスクの名前。
* `path_on_disk` ([String](/sql-reference/data-types/string)) — データパートのファイルが格納されたフォルダーへの絶対パス。
* `rows` ([UInt64](/sql-reference/data-types/int-uint)) — データパート内の行数。
* `size_in_bytes` ([UInt64](/sql-reference/data-types/int-uint)) — ディスク上のデータパートのサイズ (単位：バイト) 。
* `merged_from` ([Array(String)](/sql-reference/data-types/array)) — 現在のパーツの元となったパーツ名の配列。
* `bytes_uncompressed` ([UInt64](/sql-reference/data-types/int-uint)) — 生成されたパートの非圧縮サイズ (バイト単位) 。
* `read_rows` ([UInt64](/sql-reference/data-types/int-uint)) — マージ中に読み込まれた行の数。
* `read_bytes` ([UInt64](/sql-reference/data-types/int-uint)) — マージ処理中に読み取られたバイト数。
* `peak_memory_usage` ([UInt64](/sql-reference/data-types/int-uint)) — マージ処理中に使用されたRAMの最大量
* `deduplication_block_ids` ([Array(String)](/sql-reference/data-types/array)) — このパートの挿入時に重複排除に使用されるブロックIDの配列。
* `error` ([UInt16](/sql-reference/data-types/int-uint)) — 発生した例外のエラーコード。
* `exception` ([String](/sql-reference/data-types/string)) — 発生したエラーのメッセージ。
* `mutation_ids` ([Array(String)](/sql-reference/data-types/array)) — type が MUTATE&#95;PART&#95;START および MUTATE&#95;PART のイベントで、ソースパート (`merged&#95;from`) に適用された mutation ID の配列。
* `ProfileEvents` ([Map(LowCardinality(String), UInt64)](/sql-reference/data-types/map)) — この操作中に記録されたすべての profile events。
* `projections_duration_ms` ([Map(LowCardinality(String), UInt64)](/sql-reference/data-types/map)) — プロジェクションごとのマージ/再構築にかかった時間 (ミリ秒) 。

**別名:**

* `ProfileEvents.Names` — `mapKeys(ProfileEvents)` のエイリアスです。
* `ProfileEvents.Values` — `mapValues(ProfileEvents)` のエイリアスです。
* `name` — `part_name` のエイリアスです。

{/*AUTOGENERATED_END*/ }

`system.part_log` テーブルは、`MergeTree` テーブルに最初のデータを挿入した後に作成されます。
