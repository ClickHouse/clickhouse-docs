## 説明 \{#description\}

ユーザーおよびロールに対するロール付与情報を保持します。このテーブルに行を追加するには、`GRANT role TO user` を使用します。

## カラム \{#columns\}

{/*AUTOGENERATED_START*/ }

* `user_name` ([Nullable(String)](../../sql-reference/data-types/)) — ユーザー名。
* `role_name` ([Nullable(String)](../../sql-reference/data-types/)) — ロール名。
* `granted_role_name` ([String](../../sql-reference/data-types/)) — `role_name` ロールに付与されたロール名。あるロールを別のロールに付与するには、`GRANT role1 TO role2` を使用します。
* `granted_role_id` ([UUID](../../sql-reference/data-types/)) — ロールの ID。
* `granted_role_is_default` ([UInt8](../../sql-reference/data-types/)) — `granted_role` がデフォルトロールかどうかを示すフラグ。設定可能な値: • 1 — `granted_role` はデフォルトロールです。 • 0 — `granted_role` はデフォルトロールではありません。
* `with_admin_option` ([UInt8](../../sql-reference/data-types/)) — `granted_role` が `ADMIN OPTION` 特権を持つロールかどうかを示すフラグ。設定可能な値: • 1 — このロールは `ADMIN OPTION` 特権を持ちます。• 0 — このロールは `ADMIN OPTION` 特権を持ちません。

{/*AUTOGENERATED_END*/ }
