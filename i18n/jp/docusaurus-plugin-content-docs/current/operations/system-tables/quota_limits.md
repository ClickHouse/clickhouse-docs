## 説明 \{#description\}

すべてのクォータのすべてのインターバルに対する上限に関する情報を含みます。1 つのクォータに対して、対応する行数は任意 (0 行も可) です。

## カラム \{#columns\}

{/*AUTOGENERATED_START*/ }

* `quota_name` ([String](../../sql-reference/data-types/)) — クォータ名。
* `duration` ([UInt32](../../sql-reference/data-types/)) — リソース消費量の計算に使用する時間インターバルの長さ (秒) 。
* `is_randomized_interval` ([UInt8](../../sql-reference/data-types/)) — 真偽値です。このインターバルがランダム化されているかどうかを示します。ランダム化されていない場合、インターバルは常に同じ時刻に開始されます。たとえば、1 分のインターバルは常に分単位のちょうどの時刻に開始されます (つまり、11:20:00 に開始することはありますが、11:20:01 に開始することはありません) 。また、1 日のインターバルは常に UTC の午前 0 時に開始されます。インターバルがランダム化されている場合、最初のインターバルはランダムな時刻に開始され、その後のインターバルは順に開始されます。値: 0 — インターバルはランダム化されていません、1 — インターバルはランダム化されています。
* `max_queries` ([Nullable(UInt64)](../../sql-reference/data-types/)) — 実行できるすべての種類のクエリの最大数。
* `max_query_selects` ([Nullable(UInt64)](../../sql-reference/data-types/)) — 実行できる SELECT クエリの最大数。
* `max_query_inserts` ([Nullable(UInt64)](../../sql-reference/data-types/)) — 実行できる INSERT クエリの最大数。
* `max_errors` ([Nullable(UInt64)](../../sql-reference/data-types/)) — 指定した期間内に許容される、エラーになったクエリの最大数。
* `max_result_rows` ([Nullable(UInt64)](../../sql-reference/data-types/)) — 指定した期間内に許容される、すべてのクエリの結果セットに含まれる行の総数の最大値。
* `max_result_bytes` ([Nullable(UInt64)](../../sql-reference/data-types/)) — 指定した期間内に許容される、すべてのクエリの結果セットに含まれるバイト総数の最大値。
* `max_read_rows` ([Nullable(UInt64)](../../sql-reference/data-types/)) — 指定した期間内に許容される、すべてのクエリの実行中に読み取る行数の最大値。
* `max_read_bytes` ([Nullable(UInt64)](../../sql-reference/data-types/)) — 指定した期間内に許容される、すべてのクエリの実行中に読み取るバイト数の最大値。
* `max_execution_time` ([Nullable(Float64)](../../sql-reference/data-types/)) — 指定した期間内に、すべてのクエリの実行に許容される時間の最大値 (ナノ秒) 。
* `max_written_bytes` ([Nullable(UInt64)](../../sql-reference/data-types/)) — 指定した期間内に許容される、すべてのクエリの実行中に書き込まれるバイト数の最大値。
* `max_failed_sequential_authentications` ([Nullable(UInt64)](../../sql-reference/data-types/)) — 指定した期間内に許容される、連続する認証失敗の最大数。
* `max_queries_per_normalized_hash` ([Nullable(UInt64)](../../sql-reference/data-types/)) — 指定した期間内に許容される、単一の正規化されたクエリごとの実行回数の最大値。
