## 説明 \{#description\}

非同期インサートに関する情報を保持します。各エントリは、非同期インサート用にバッファリングされたINSERT クエリを表します。

ロギングを開始するには、[asynchronous&#95;insert&#95;log](../../operations/server-configuration-parameters/settings.md#asynchronous_insert_log) セクションでパラメータを設定します。

データのフラッシュ周期は、[asynchronous&#95;insert&#95;log](../../operations/server-configuration-parameters/settings.md#asynchronous_insert_log) サーバー設定セクションの `flush_interval_milliseconds` パラメータで設定します。フラッシュを強制するには、[SYSTEM FLUSH LOGS](/sql-reference/statements/system#flush-logs) クエリを使用します。

ClickHouse はテーブルからデータを自動的に削除しません。詳細は [Introduction](/operations/system-tables/overview#system-tables-introduction) を参照してください。

## カラム \{#columns\}

{/*AUTOGENERATED_START*/ }

* `hostname` ([LowCardinality(String)](/sql-reference/data-types/lowcardinality)) — クエリを実行したサーバーのホスト名。
* `event_date` ([Date](/sql-reference/data-types/date)) — 非同期 INSERT が行われた日付。
* `event_time` ([DateTime](/sql-reference/data-types/datetime)) — 非同期 INSERT の実行が完了した日時。
* `event_time_microseconds` ([DateTime64(6)](/sql-reference/data-types/datetime64)) — 非同期 INSERT の実行が完了した日時 (マイクロ秒精度) 。
* `query` ([String](/sql-reference/data-types/string)) — クエリ文字列。
* `database` ([LowCardinality(String)](/sql-reference/data-types/lowcardinality)) — テーブルが含まれるデータベースの名前。
* `table` ([LowCardinality(String)](/sql-reference/data-types/lowcardinality)) — テーブル名。
* `format` ([LowCardinality(String)](/sql-reference/data-types/lowcardinality)) — フォーマット名。
* `query_id` ([String](/sql-reference/data-types/string)) — 最初のクエリの ID。
* `bytes` ([UInt64](/sql-reference/data-types/int-uint)) — INSERT されたバイト数。
* `rows` ([UInt64](/sql-reference/data-types/int-uint)) — INSERT された行数。
* `exception` ([String](/sql-reference/data-types/string)) — 例外メッセージ。
* `status` ([Enum8(&#39;Ok&#39; = 0, &#39;ParsingError&#39; = 1, &#39;FlushError&#39; = 2)](/sql-reference/data-types/enum)) — INSERT のステータス。値: &#39;Ok&#39; = 0 — INSERT 成功、&#39;ParsingError&#39; = 1 — データの解析時に発生した例外、&#39;FlushError&#39; = 2 — データのフラッシュ時に発生した例外。
* `data_kind` ([Enum8(&#39;Parsed&#39; = 0, &#39;Preprocessed&#39; = 1)](/sql-reference/data-types/enum)) — データの種類。値: &#39;Parsed&#39; および &#39;Preprocessed&#39;。
* `flush_time` ([DateTime](/sql-reference/data-types/datetime)) — フラッシュが行われた日時。
* `flush_time_microseconds` ([DateTime64(6)](/sql-reference/data-types/datetime64)) — フラッシュが行われた日時 (マイクロ秒精度) 。
* `flush_query_id` ([String](/sql-reference/data-types/string)) — フラッシュクエリの ID。
* `timeout_milliseconds` ([UInt64](/sql-reference/data-types/int-uint)) — このエントリに対して計算された適応型タイムアウト。

{/*AUTOGENERATED_END*/ }
