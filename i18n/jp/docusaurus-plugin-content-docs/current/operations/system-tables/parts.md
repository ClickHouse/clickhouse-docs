## 説明 \{#description\}

[MergeTree](../../engines/table-engines/mergetree-family/mergetree.md) テーブルのパーツに関する情報を格納します。

各行は 1 つのデータパーツを表します。

## カラム \{#columns\}

{/*AUTOGENERATED_START*/ }

* `partition` ([String](../../sql-reference/data-types/)) — パーティション識別子 (テーブルのパーティションキー式から導出された文字列表現) 。
* `name` ([String](../../sql-reference/data-types/)) —
  データパーツの名前。パーツ名の構造から、データ、取り込み、およびマージパターンに関する多くの側面を把握できます。パーツ名の形式は次のとおりです。

```text
<partition_id>_<minimum_block_number>_<maximum_block_number>_<level>_<data_version>
```

* 定義:
  * `partition_id` - パーティションキーを識別します
  * `minimum_block_number` - パーツ内の最小ブロック番号を識別します。ClickHouse は常に連続したブロックをマージします
  * `maximum_block_number` - パーツ内の最大ブロック番号を識別します
  * `level` - パーツに対するマージが 1 回行われるたびに 1 ずつ増加します。レベル 0 は、まだマージされていない新しいパーツであることを示します。ClickHouse のすべてのパーツは常に不変 (イミュータブル) であることを理解しておくことが重要です
  * `data_version` - オプションの値で、パーツに対してミューテーションが実行されると増加します (繰り返しになりますが、パーツは不変であるため、ミューテーションされたデータは常に新しいパーツにのみ書き込まれます)

* `uuid` ([UUID](../../sql-reference/data-types/)) — データパートの UUID。

* `part_type` ([String](../../sql-reference/data-types/)) — data part の保存形式。設定可能な値: `Wide` — 各カラムは個別のファイルに保存されます。`Compact` — すべてのカラムが 1 つのファイルに保存されます。データの保存形式は、MergeTree テーブルの `min_bytes_for_wide_part` および `min_rows_for_wide_part` の設定によって制御されます。

* `active` ([UInt8](../../sql-reference/data-types/)) — データパーツがアクティブかどうかを示すフラグです。データパーツがアクティブな場合、そのパーツはテーブルで使用されています。そうでない場合は、まもなく削除されます。非アクティブなデータパーツは、マージおよびミューテーション操作の後に現れます。

* `marks` ([UInt64](../../sql-reference/data-types/)) — マークの数。data part 内のおおよその行数を求めるには、marks に索引の粒度 (通常は 8192) を掛けます (この方法は adaptive granularity では機能しません) 。

* `rows` ([UInt64](../../sql-reference/data-types/)) — 行の数。

* `files` ([UInt64](../../sql-reference/data-types/)) — データパート内のファイル数。

* `bytes_on_disk` ([UInt64](../../sql-reference/data-types/)) — すべてのデータパートファイルの合計サイズ (単位: バイト) 。

* `data_compressed_bytes` ([UInt64](../../sql-reference/data-types/)) — データパート内の圧縮データの合計サイズです。マークファイルなどの補助ファイルはすべて含まれません。

* `data_uncompressed_bytes` ([UInt64](../../sql-reference/data-types/)) — データパート内の非圧縮データの合計サイズです。マークファイルなどの補助ファイルは含まれません。

* `primary_key_size` ([UInt64](../../sql-reference/data-types/)) — ディスク上の primary.idx/cidx ファイル内で主キーの値が使用するメモリ量 (バイト単位) 。

* `marks_bytes` ([UInt64](../../sql-reference/data-types/)) — マークファイルのサイズ。

* `secondary_indices_compressed_bytes` ([UInt64](../../sql-reference/data-types/)) — データパート内のセカンダリ索引の圧縮データの合計サイズです。マークファイルなどの補助ファイルは含まれません。

* `secondary_indices_uncompressed_bytes` ([UInt64](../../sql-reference/data-types/)) — データパート内のセカンダリ索引における非圧縮データの合計サイズです。マークファイルなどの補助ファイルは含まれません。

* `secondary_indices_marks_bytes` ([UInt64](../../sql-reference/data-types/)) — セカンダリ索引のマークを含むファイルのサイズ。

* `modification_time` ([DateTime](../../sql-reference/data-types/)) — データパートを含むディレクトリの更新時刻です。通常、これはデータパートの作成時刻に対応します。

* `remove_time` ([DateTime](../../sql-reference/data-types/)) — データパートが非アクティブになった時刻。

* `refcount` ([UInt32](../../sql-reference/data-types/)) — データパートの使用箇所数。値が 2 を超える場合、そのデータパートがクエリまたはマージで使用されていることを示します。

* `min_date` ([Date](../../sql-reference/data-types/)) — データパート内の日付キーの最小値。

* `max_date` ([Date](../../sql-reference/data-types/)) — データパート内の日付キーの最大値。

* `min_time` ([DateTime](../../sql-reference/data-types/)) — データパート内の日時キーの最小値。

* `max_time` ([DateTime](../../sql-reference/data-types/)) — データパートにおける日時キーの最大値。

* `partition_id` ([String](../../sql-reference/data-types/)) — パーティション ID。

* `min_block_number` ([Int64](../../sql-reference/data-types/)) — マージ後に現在のパーツを構成するデータパーツのうち、最小の番号。

* `max_block_number` ([Int64](../../sql-reference/data-types/)) — マージ後に現在のパーツを構成するデータパーツの最大番号。

* `level` ([UInt32](../../sql-reference/data-types/)) — マージツリーの深さ。0 は、現在のパーツが他のパーツのマージではなく、insert によって作成されたことを意味します。

* `data_version` ([UInt64](../../sql-reference/data-types/)) — データパートに適用すべき mutation (data&#95;version より大きいバージョンの mutation) を判定するために使用される数値。

* `primary_key_bytes_in_memory` ([UInt64](../../sql-reference/data-types/)) — 主キーの値が使用するメモリ量 (バイト単位) 。`primary_key_lazy_load` が有効で、かつキーが読み込まれていない場合は 0 になります。

* `primary_key_bytes_in_memory_allocated` ([UInt64](../../sql-reference/data-types/)) — 主キーの値用に確保されているメモリ量 (バイト単位) 。`primary_key_lazy_load` が有効で、キーが読み込まれていない場合は 0 です。

* `index_granularity_bytes_in_memory` ([UInt64](../../sql-reference/data-types/)) — 索引粒度の値が使用するメモリ量 (バイト単位)  (primary&#95;key&#95;lazy&#95;load=1 かつ use&#95;primary&#95;key&#95;cache=1 の場合は 0 になります) 。

* `index_granularity_bytes_in_memory_allocated` ([UInt64](../../sql-reference/data-types/)) — 索引グラニュラリティ値用に予約されたメモリ量 (バイト単位)  (primary&#95;key&#95;lazy&#95;load=1 かつ use&#95;primary&#95;key&#95;cache=1 の場合は 0 になります) 。

* `is_frozen` ([UInt8](../../sql-reference/data-types/)) — パーティションデータのバックアップが存在するかどうかを示すフラグです。1 はバックアップが存在すること、0 は存在しないことを示します。詳細については、FREEZE PARTITION を参照してください。

* `database` ([String](../../sql-reference/data-types/)) — データベース名。

* `table` ([String](../../sql-reference/data-types/)) — テーブルの名前。

* `engine` ([String](../../sql-reference/data-types/)) — パラメータを含まないテーブルエンジン名。

* `disk_name` ([String](../../sql-reference/data-types/)) — データパートを格納するディスクの名前。

* `path` ([String](../../sql-reference/data-types/)) — データパートのファイルが格納されているフォルダの絶対パス。

* `hash_of_all_files` ([String](../../sql-reference/data-types/)) — 圧縮されたファイルの sipHash128。

* `hash_of_uncompressed_files` ([String](../../sql-reference/data-types/)) — 非圧縮ファイル (マーク付きファイル、索引ファイルなど) の sipHash128。

* `uncompressed_hash_of_compressed_files` ([String](../../sql-reference/data-types/)) — 圧縮ファイル内のデータを、あたかも非圧縮であるかのように扱った場合の sipHash128。

* `delete_ttl_info_min` ([DateTime](../../sql-reference/data-types/)) — TTL DELETE ルールの日時キーの最小値。

* `delete_ttl_info_max` ([DateTime](../../sql-reference/data-types/)) — TTL DELETE ルールにおける日時キーの最大値。

* `move_ttl_info.expression` ([Array(String)](../../sql-reference/data-types/)) — 式の配列です。各式で、有効期限 (TTL) の MOVE ルールを定義します。

* `move_ttl_info.min` ([Array(DateTime)](../../sql-reference/data-types/)) — 日付と時刻の値の配列。各要素は、TTL MOVE ルールにおける最小のキー値を表します。

* `move_ttl_info.max` ([Array(DateTime)](../../sql-reference/data-types/)) — 日付と時刻の値の配列。各要素は、TTL MOVE ルールにおける最大キー値を表します。

* `default_compression_codec` ([String](../../sql-reference/data-types/)) — このデータパートの圧縮に使用されるコーデック名です (カラムに明示的なコーデックが指定されていない場合) 。

* `recompression_ttl_info.expression` ([Array(String)](../../sql-reference/data-types/)) — 有効期限 (TTL) の式。

* `recompression_ttl_info.min` ([Array(DateTime)](../../sql-reference/data-types/)) — このパート内で計算された TTL 式の最小値。少なくとも1行で有効期限 (TTL) が切れているかどうかを判断するために使用されます。

* `recompression_ttl_info.max` ([Array(DateTime)](../../sql-reference/data-types/)) — このパート内で計算された TTL 式の最大値です。TTL が期限切れのすべての行が含まれているかどうかを判断するために使用されます。

* `group_by_ttl_info.expression` ([Array(String)](../../sql-reference/data-types/)) — TTL 式。

* `group_by_ttl_info.min` ([Array(DateTime)](../../sql-reference/data-types/)) — このデータパート内で計算された TTL 式の最小値です。有効期限 (TTL) が切れた行が少なくとも1つ存在するかどうかを把握するために使用されます。

* `group_by_ttl_info.max` ([Array(DateTime)](../../sql-reference/data-types/)) — このパート内で計算された TTL 式の最大値です。有効期限 (TTL) が切れた行がすべて揃っているかどうかを把握するために使用されます。

* `rows_where_ttl_info.expression` ([Array(String)](../../sql-reference/data-types/)) — TTL 式。

* `rows_where_ttl_info.min` ([Array(DateTime)](../../sql-reference/data-types/)) — このパート内で計算された有効期限 (TTL) 式の最小値です。有効期限 (TTL) が切れた行が少なくとも 1 行あるかどうかを判断するために使用されます。

* `rows_where_ttl_info.max` ([Array(DateTime)](../../sql-reference/data-types/)) — このパート内で計算された TTL 式の最大値です。有効期限 (TTL) が切れたすべての行が揃っているかどうかを把握するために使用されます。

* `projections` ([Array(String)](../../sql-reference/data-types/)) — このパートについて計算されたプロジェクション名の一覧。

* `visible` ([UInt8](../../sql-reference/data-types/)) — このパートが SELECT クエリから見えるかどうかを示すフラグ。

* `creation_tid` ([Tuple(UInt64, UInt64, UUID)](../../sql-reference/data-types/)) — このオブジェクトを作成した、または作成しようとしているトランザクションの ID。

* `removal_tid_lock` ([UInt64](../../sql-reference/data-types/)) — removal&#95;tid のハッシュ。削除時にオブジェクトをロックするために使用されます。

* `removal_tid` ([Tuple(UInt64, UInt64, UUID)](../../sql-reference/data-types/)) — このオブジェクトを削除した、または削除しようとしているトランザクションの ID

* `creation_csn` ([UInt64](../../sql-reference/data-types/)) — このオブジェクトを作成したトランザクションのCSN

* `removal_csn` ([UInt64](../../sql-reference/data-types/)) — このオブジェクトを削除したトランザクションの CSN

* `has_lightweight_delete` ([UInt8](../../sql-reference/data-types/)) — パートに論理削除用のマスクがあるかどうかを示すフラグ。

* `last_removal_attempt_time` ([DateTime](../../sql-reference/data-types/)) — サーバーがこのパートの削除を最後に試みた日時。

* `removal_state` ([String](../../sql-reference/data-types/)) — part の削除プロセスの現在の状態。

{/*AUTOGENERATED_END*/ }

**別名:**

* `bytes` — `bytes_on_disk` のエイリアス。
* `marks_size` — `marks_bytes` のエイリアス。
* `part_name` — `name` のエイリアス。
