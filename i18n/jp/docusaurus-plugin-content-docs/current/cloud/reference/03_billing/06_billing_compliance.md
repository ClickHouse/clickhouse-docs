# ClickHouse Cloud の課金コンプライアンス \{#clickhouse-cloud-billing-compliance\}

## 請求に関するコンプライアンス \{#billing-compliance\}

ClickHouse Cloud を利用するには、組織に有効な請求方法が設定されている必要があります。30 日間のトライアルが終了するか、トライアルクレジットを使い切った時点の、いずれか早い方に達した後も ClickHouse Cloud の利用を継続するには、次のいずれかの請求オプションを選択してください。

| Billing option                                       | Description                                                                             |
|------------------------------------------------------|-----------------------------------------------------------------------------------------|
| [Direct PAYG](#direct-payg)                          | 組織に有効なクレジットカードを追加して、従量課金 (Pay-As-You-Go) を利用します           |
| [Marketplace PAYG](#cloud-marketplace-payg)          | サポートされているクラウドマーケットプレイスプロバイダー経由で従量課金サブスクリプションを設定します |
| [Committed spend contract](#committed-spend-contract) | サポートされているクラウドマーケットプレイスを通じて、または直接、コミットメント契約を締結します |

トライアルが終了した時点で組織にいずれの請求オプションも設定されていない場合、すべてのサービスは停止されます。さらに 2 週間経過しても請求方法が設定されない場合、すべてのデータは削除されます。

ClickHouse は組織単位でサービス料金を請求します。現在の請求方法で支払い処理ができない場合は、サービス中断を避けるために、上記 3 つのいずれかのオプションに更新する必要があります。選択した請求方法ごとの支払い要件の詳細については、以下を参照してください。

### クレジットカードによる従量課金請求 \{#direct-payg\}

ClickHouse Cloud の利用料金は、クレジットカードを使用して月次の後払いで支払うことができます。
クレジットカードを追加するには、次の[手順](#add-credit-card)に従ってください。

ClickHouse の月次請求サイクルは、組織のティア (Basic、Scale、または Enterprise) が選択され、組織内で最初のサービスが作成された日に開始されます。

登録済みのクレジットカードには通常、月次請求サイクルの終了時に課金されますが、サイクル途中の未決済額が 10,000 USD に達した場合、課金処理が前倒しされます（支払いしきい値の詳細は[こちら](/cloud/billing/payment-thresholds)を参照してください）。

登録済みのクレジットカードは有効であり、有効期限切れでなく、請求金額をカバーできる十分な利用可能枠がある必要があります。何らかの理由で請求額全額を課金できなかった場合、未払い請求書に対して次の制限が直ちに適用されます。

* レプリカあたり 120 GiB までしかスケールアップできません
* 停止中のサービスを起動できません
* 新しいサービスを起動または作成できません

当社は、最大 30 日間、組織に設定されている請求方法で支払い処理を試行します。14 日間経過しても支払いが成功しない場合、組織内のすべてのサービスは停止されます。この 30 日間の期間の終了までに支払いが行われず、かつ当社が延長を認めていない場合、組織に関連するすべてのデータとサービスは削除されます。

### クラウドマーケットプレイスによる従量課金請求 \{#cloud-marketplace-payg\}

従量課金 (Pay-As-You-Go) 請求は、サポート対象のクラウドマーケットプレイス
(AWS、GCP、または Azure) のいずれかを通じて組織に対して課金するように設定することもできます。Marketplace PAYG 請求に登録するには、次の
[手順](#marketplace-payg)に従ってください。

Direct PAYG による請求と同様に、Marketplace PAYG を利用した ClickHouse での月次請求サイクルは、組織のティア (Basic、Scale、または Enterprise) が選択され、組織内で最初のサービスが作成された日に開始されます。

ただし、マーケットプレイスの要件により、従量課金利用分の料金は 1 時間単位で報告されます。請求書は、通常は暦月単位の請求サイクルとなる、そのマーケットプレイスとの契約条件に従って発行されます。

例えば、1 月 18 日に最初の組織サービスを作成した場合、ClickHouse Cloud における最初の利用請求サイクルは 1 月 18 日から 2 月 17 日の終わりまでとなります。ただし、クラウドマーケットプレイスからの最初の請求書は、2 月初旬に発行される場合があります。

一方で、PAYG マーケットプレイスサブスクリプションが解約された場合、または自動更新に失敗した場合、（存在する場合は）組織に登録されているクレジットカードでの請求にフォールバックされます。クレジットカードを追加するには、[サポートにお問い合わせ](/about-us/support)ください。有効なクレジットカードが提供されていない場合、[Direct PAYG](#direct-payg) に対して前述した未払い請求書の制限と同じ制限が適用されます。これにはサービスの一時停止および最終的なデータ削除が含まれます。

### コミットメント契約による請求 \{#committed-spend-contract\}

コミットメント契約を通じて、組織向けのクレジットを購入することができます。

1. 営業担当に連絡してクレジットを直接購入します。支払い方法には ACH や
   電信送金などが含まれます。支払い条件は、適用される注文書に定められます。
2. 営業担当に連絡して、当社がサポートしているクラウドマーケットプレイス
   （AWS、GCP、または Azure）のいずれかで、サブスクリプションを通じてクレジットを購入します。
   料金は、プライベートオファーの受諾時およびその後はオファー条件に従って
   該当するマーケットプレイスに報告されますが、請求はそのマーケットプレイスとの
   契約条件に従って行われます。マーケットプレイス経由で支払うには、
   これらの[手順](#marketplace-payg)に従ってください。

組織に適用されたクレジット（コミットメント契約や返金によるものなど）は、
注文書または受諾されたプライベートオファーに記載された期間中、利用可能です。
クレジットは、最初の組織のティア（Basic、Scale、または Enterprise）が選択された日付を基準とした
請求期間において、クレジット付与日から消費が開始されます。

組織がクラウドマーケットプレイスのコミットメント契約 **ではない** 場合に
クレジットを使い切るか有効期限が切れたときは、その組織は自動的に
従量課金（Pay-As-You-Go、PAYG）課金に切り替わります。この場合、
組織に登録されているクレジットカードがあれば、それを使用して支払い処理を試行します。

組織がクラウドマーケットプレイスのコミットメント契約 **である** 場合に
クレジットを使い切ったときも、サブスクリプション期間の残りについては
同じマーケットプレイス経由で自動的に PAYG 課金に切り替わります。
ただし、サブスクリプションが更新されずに有効期限切れとなった場合には、
その後は組織に登録されているクレジットカードがあれば、それを使用して
支払い処理を試行します。

いずれのケースでも、設定済みのクレジットカードへの請求ができない場合は、
クレジットカードによる [Pay-as-you-go (PAYG)](#direct-payg)
課金に関して上記で説明した未払い請求書の制限が適用されます。
これにはサービスの一時停止が含まれます。コミットメント契約から PAYG 課金に
移行する詳細については、[利用規約](https://clickhouse.com/legal/agreements/terms-of-service)
の「Overconsumption」セクションを参照してください。
ただし、コミットメント契約のお客様については、データ削除を開始する前に、
未払い請求書についてご連絡します。データが時間経過によって自動的に削除されることはありません。

既存のクレジットが有効期限を迎える前、または使い果たされる前に
クレジットを追加したい場合は、[お問い合わせ](https://clickhouse.com/company/contact)
ください。

### クレジットカードで支払う方法 \{#add-credit-card\}

ClickHouse Cloud UI の Billing セクションに移動し、「Add Credit Card」
ボタン（下図）をクリックしてセットアップを完了します。ご不明な点がある場合は、
[サポートへのお問い合わせ](/about-us/support) からサポートを受けてください。

## マーケットプレイス経由での支払い方法 \{#marketplace-payg\}

サポート対象のマーケットプレイス（AWS、GCP、または Azure）のいずれかを通じて支払う場合は、
[こちら](/cloud/marketplace/marketplace-billing)の手順に従ってください。
クラウドマーケットプレイスでの課金・請求に関するご質問は、クラウドサービスプロバイダーに直接お問い合わせください。

マーケットプレイスの請求に関する問題解決に役立つリンク:
* [AWS 請求に関する FAQ](https://aws.amazon.com/aws-cost-management/aws-billing/faqs/)
* [GCP 請求に関する FAQ](https://cloud.google.com/compute/docs/billing-questions)
* [Azure 請求に関する FAQ](https://learn.microsoft.com/en-us/azure/cost-management-billing/cost-management-billing-faq)
