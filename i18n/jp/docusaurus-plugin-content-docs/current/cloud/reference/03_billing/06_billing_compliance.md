# ClickHouse Cloud の課金に関するコンプライアンス



## 請求コンプライアンス {#billing-compliance}

ClickHouse Cloudを使用するには、組織に有効な請求方法を設定する必要があります。30日間のトライアル期間が終了するか、トライアルクレジットが使い果たされた場合（いずれか早い方）、ClickHouse Cloudを継続して使用するには以下の請求オプションがあります：

| 請求オプション                                        | 説明                                                                             |
| ----------------------------------------------------- | --------------------------------------------------------------------------------------- |
| [Direct PAYG](#direct-payg)                           | 組織に有効なクレジットカードを追加して従量課金制を利用                           |
| [Marketplace PAYG](#cloud-marketplace-payg)           | サポートされているクラウドマーケットプレイスプロバイダー経由で従量課金制サブスクリプションを設定          |
| [Committed spend contract](#committed-spend-contract) | 直接またはサポートされているクラウドマーケットプレイス経由でコミット支出契約を締結 |

トライアルが終了し、組織に請求オプションが設定されていない場合、すべてのサービスが停止されます。2週間経過しても請求方法が設定されていない場合、すべてのデータが削除されます。

ClickHouseはサービスの料金を組織レベルで請求します。現在の請求方法で支払いを処理できない場合、サービスの中断を避けるために、上記の3つのオプションのいずれかに更新する必要があります。選択した請求方法に基づく支払いコンプライアンスの詳細については、以下を参照してください。

### クレジットカードによる従量課金制 {#direct-payg}

クレジットカードを使用して、ClickHouse Cloudの使用料金を月次後払いで支払うことができます。クレジットカードを追加するには、こちらの[手順](#add-credit-card)に従ってください。

ClickHouseの月次請求サイクルは、組織ティア（Basic、Scale、またはEnterprise）が選択され、組織内で最初のサービスが作成された日から開始されます。

登録されているクレジットカードは通常、月次請求サイクルの終了時に請求されますが、サイクル内の請求額が10,000米ドルに達した場合、支払い請求が前倒しされます（支払い閾値の詳細については[こちら](/cloud/billing/payment-thresholds)を参照してください）。

登録されているクレジットカードは有効で、有効期限が切れておらず、請求書の合計金額をカバーするのに十分な利用可能枠が必要です。何らかの理由で請求額の全額を請求できない場合、以下の未払い請求書制限が直ちに適用されます：

- レプリカあたり最大120 GiBまでしかスケールアップできません
- 停止したサービスを起動できません
- 新しいサービスを起動または作成できません

組織に設定された請求方法を使用して、最大30日間支払い処理を試みます。14日経過しても支払いが成功しない場合、組織内のすべてのサービスが停止されます。この30日間の期間が終了しても支払いが受領されず、延長が認められていない場合、組織に関連するすべてのデータとサービスが削除されます。

### クラウドマーケットプレイスの従量課金制 {#cloud-marketplace-payg}

従量課金制は、サポートされているクラウドマーケットプレイス（AWS、GCP、またはAzure）のいずれかを通じて組織に請求するように設定することもできます。Marketplace PAYG請求に登録するには、こちらの[手順](#marketplace-payg)に従ってください。

Direct PAYGによる請求と同様に、Marketplace PAYG下でのClickHouseの月次請求サイクルは、組織ティア（Basic、Scale、またはEnterprise）が選択され、組織内で最初のサービスが作成された日から開始されます。

ただし、マーケットプレイスの要件により、従量課金制の使用料金は1時間単位で報告されます。請求書は、そのマーケットプレイスとの契約条件に従って発行されることに注意してください。通常は暦月単位の請求サイクルとなります。

例えば、1月18日に最初の組織サービスを作成した場合、ClickHouse Cloudでの最初の請求使用サイクルは1月18日から2月17日の終日まで実行されます。ただし、クラウドマーケットプレイスからの最初の請求書は2月の初めに受け取る可能性があります。

ただし、PAYGマーケットプレイスサブスクリプションがキャンセルされた場合、または自動更新に失敗した場合、請求は組織に登録されているクレジットカード（存在する場合）にフォールバックされます。クレジットカードを追加するには、[サポートに連絡](/about-us/support)してサポートを受けてください。有効なクレジットカードが提供されていない場合、[Direct PAYG](#direct-payg)で上述した未払い請求書制限が同様に適用されます。これにはサービスの停止と最終的なデータ削除が含まれます。

### コミット契約請求 {#committed-spend-contract}

コミット契約を通じて組織のクレジットを購入するには、以下の方法があります：


1. 営業担当者に連絡してクレジットを直接購入する。支払いオプションにはACHまたは電信送金が含まれます。支払い条件は該当する注文書に記載されます。
2. 営業担当者に連絡して、サポートされているクラウドマーケットプレイス(AWS、GCP、またはAzure)のいずれかでサブスクリプションを通じてクレジットを購入する。料金はプライベートオファーの承認時に該当するマーケットプレイスに報告され、その後はオファー条件に従って報告されますが、請求はそのマーケットプレイスとの契約条件に従って行われます。マーケットプレイスを通じて支払う場合は、こちらの[手順](#marketplace-payg)に従ってください。

組織に適用されたクレジット(コミット契約や返金による付与など)は、注文書または承認されたプライベートオファーで指定された期間中、利用可能です。
クレジットは、最初の組織ティア(Basic、Scale、またはEnterprise)が選択された日付を基準とする請求期間において、クレジットが付与された日から消費されます。

組織がクラウドマーケットプレイスのコミット契約を**利用していない**場合、クレジットが不足または期限切れになると、組織は自動的に従量課金制(PAYG)請求に切り替わります。この場合、組織に登録されているクレジットカードがあれば、そのカードを使用して支払い処理を試みます。

組織がクラウドマーケットプレイスのコミット契約を**利用している**場合、クレジットが不足すると、サブスクリプションの残り期間について、同じマーケットプレイス経由で自動的にPAYG請求に切り替わります。ただし、サブスクリプションが更新されずに期限切れになった場合は、組織に登録されているクレジットカードがあれば、そのカードを使用して支払い処理を試みます。

いずれの場合も、設定されたクレジットカードへの請求ができない場合、クレジットカードによる[従量課金制(PAYG)](#direct-payg)請求について上記で説明した未払い請求書の制限が適用されます。これにはサービスの停止が含まれます。
コミット契約からPAYG請求への移行の詳細については、[利用規約](https://clickhouse.com/legal/agreements/terms-of-service)の「過剰消費」セクションを参照してください。
ただし、コミット契約をご利用のお客様については、データ削除を開始する前に、未払い請求書に関してご連絡いたします。データは一定期間経過後に自動的に削除されることはありません。

既存のクレジットが期限切れまたは消費される前に追加のクレジットを購入したい場合は、[お問い合わせ](https://clickhouse.com/company/contact)ください。
