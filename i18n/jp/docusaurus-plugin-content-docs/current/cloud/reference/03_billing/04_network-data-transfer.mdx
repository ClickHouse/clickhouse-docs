ClickHouse Cloud は、受信および送信されるデータ転送量を計測します。
これには、ClickHouse Cloud との間のすべての入出力データに加え、リージョン内および
リージョン間のデータ転送が含まれます。この利用状況はサービス単位で追跡されます。
この利用状況に基づいてデータ転送料金が発生し、その料金が月次請求に加算されます。

ClickHouse Cloud では、次の項目について料金が発生します:

* ClickHouse Cloud からパブリックインターネットへのデータ送信。
  他クラウドプロバイダーの他リージョン宛ての送信も含みます。
* 同一クラウドプロバイダー内の別リージョンへのデータ送信。

リージョン内のデータ転送や Private Link/Private Service Connect の利用および
それに伴うデータ転送には料金は発生しません。ただし、ユーザーに適切な料金を
請求する能力に影響を与える利用パターンが見られた場合は、追加のデータ転送に関する
料金体系を導入する権利を留保します。

注: Private Link/Private Service Connect の利用には、クラウドサービスプロバイダー (CSP) 側の料金が適用されます。最新情報については CSP の料金ページを参照してください。

データ転送料金は、クラウドサービスプロバイダー (CSP) およびリージョンによって異なります。
パブリックインターネットへの送信料金は送信元リージョンのみに基づきます。
リージョン間 (クロスリージョン) の料金は、送信元および送信先
リージョンの両方に依存します。

**データ転送コストを最小化するためのベストプラクティス**

データを送受信する際に、データ転送コストを最小化するため留意すべき
いくつかのパターンがあります。

1. ClickHouse Cloud との間でデータを送受信する際は、可能な限り圧縮を使用して、
   転送されるデータ量およびそれに伴うコストを最小化してください。

2. ネイティブプロトコル上で非インライン値を用いた `INSERT` を行う場合
   (例: `INSERT INTO [TABLE] FROM INFILE [FILE] FORMAT NATIVE`)、
   ClickHouse クライアントはデータをパックするためにサーバーからメタデータを
   取得します。メタデータが `INSERT` ペイロードより大きい場合、サーバー側の
   観点では、受信量より送信量の方が多くなるという直感に反する状況が発生する
   ことがあります。これが許容できない場合は、`VALUES` 構文でデータを
   インライン化するか、HTTP プロトコルの利用を検討してください。
