ClickHouse Cloud は、転送されるデータのインバウンドおよびアウトバウンドを計測します。
これには、ClickHouse Cloud との間のすべての入出力データ、および同一リージョン内と
リージョン間のデータ転送が含まれます。この利用状況はサービス単位で追跡されます。
この利用状況に基づき、顧客にはデータ転送料金が発生し、その後月次の請求に加算されます。

ClickHouse Cloud の課金対象:

* ClickHouse Cloud からパブリックインターネットへのデータの送信（他社クラウド
  プロバイダーの他リージョンへの送信を含む）。
* 同一クラウドプロバイダー内の別リージョンへのデータの送信。

同一リージョン内のデータ転送や、Private Link / Private Service Connect の利用および
それらを経由したデータ転送については課金されません。ただし、適切な料金設定に
影響を与える利用パターンが確認された場合には、追加のデータ転送料金の区分を
導入する権利を留保します。

注意: Private Link / Private Service Connect の利用には、Cloud Service Provider (CSP) による料金が適用されます。最新情報については、CSP の料金ページを参照してください。

データ転送料金は、Cloud Service Provider (CSP) とリージョンによって異なります。
パブリックインターネットへの送信料金は、送信元リージョンのみを基準に決定されます。
リージョン間（クロスリージョン）の料金は、送信元と送信先の両方のリージョンに
依存します。

**データ転送コストを最小限に抑えるためのベストプラクティス**

データのインバウンドおよびアウトバウンド時に、データ転送コストを最小限に抑えるために
留意すべきパターンがいくつかあります。

1. ClickHouse Cloud へのデータのインバウンドまたはアウトバウンド時には、可能な限り
   圧縮を使用し、転送データ量とそれに伴うコストを最小限に抑えてください。

2. ネイティブプロトコルを使用し、かつ非インライン値で `INSERT` を実行する場合
   （例: `INSERT INTO [TABLE] FROM INFILE [FILE] FORMAT NATIVE`）、ClickHouse
   クライアントはサーバーからメタデータを取得してデータをパックします。
   メタデータが `INSERT` ペイロードよりも大きい場合、サーバー側の観点では
   インバウンドよりもアウトバウンドが多く発生するという、直感に反する結果が
   見られることがあります。これが受け入れられない場合は、`VALUES` 構文でデータを
   インライン化するか、HTTP プロトコルの使用を検討してください。
