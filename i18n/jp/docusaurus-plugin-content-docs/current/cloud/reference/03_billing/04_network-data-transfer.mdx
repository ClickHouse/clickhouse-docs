ClickHouse Cloud は、転送されるデータのイングレスとエグレスを計測します。
これには、ClickHouse Cloud への入出力データだけでなく、同一リージョン内および
リージョン間のデータ転送が含まれます。この利用状況はサービス単位で追跡されます。
この利用状況に基づき、お客様にはデータ転送料金が発生し、月次請求書に加算されます。

ClickHouse Cloud では、以下に対して料金が発生します:

* ClickHouse Cloud からパブリックインターネットへのデータエグレス
  （他クラウドプロバイダーの別リージョンへのデータ転送を含む）。
* 同一クラウドプロバイダー内の別リージョンへのデータエグレス。

同一リージョン内のデータ転送や、Private Link/Private Service Connect の利用および
データ転送には料金は発生しません。ただし、ユーザーに対して適切に課金する能力に
影響する利用パターンが確認された場合、追加のデータ転送料金要素を導入する権利を
当社は留保します。

注意: Private Link/Private Service Connect の利用には、クラウドサービスプロバイダー
(CSP) の料金が適用されます。最新の情報については、CSP の料金ページを参照してください。

データ転送料金は、クラウドサービスプロバイダー (CSP) とリージョンによって異なります。
パブリックインターネットへのエグレスの料金は、発信元リージョンのみに基づきます。
リージョン間（またはクロスリージョン）の料金は、発信元リージョンと宛先リージョンの
両方に依存します。

**データ転送コストを最小化するためのベストプラクティス**

データ転送コストを最小化するために、イングレスおよびエグレス時に考慮すべき
パターンがいくつかあります。

1. ClickHouse Cloud との間でデータをイングレスまたはエグレスする際には、
   可能な限り圧縮を使用して、転送されるデータ量と関連コストを最小化してください。

2. ネイティブプロトコルを使い、インライン化されていない値を用いて `INSERT` を実行する場合
   （例: `INSERT INTO [TABLE] FROM INFILE [FILE] FORMAT NATIVE`）、ClickHouse
   クライアントは、データをパックするためにサーバーからメタデータを取得します。
   このメタデータが `INSERT` ペイロードよりも大きい場合、サーバーの視点では
   イングレスよりもエグレスの方が多くなるという、直感に反する状況が発生する
   可能性があります。これが許容できない場合は、`VALUES` 構文でデータをインライン化するか、
   HTTP プロトコルの使用を検討してください。
