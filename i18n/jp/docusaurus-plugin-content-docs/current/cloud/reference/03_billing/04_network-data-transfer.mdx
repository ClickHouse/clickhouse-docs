ClickHouse Cloud は、インバウンドおよびアウトバウンドのデータ転送量を計測します。
これには、ClickHouse Cloud への入出力データに加えて、同一リージョン内および
リージョン間のデータ転送が含まれます。この利用状況はサービス単位で追跡されます。
この利用状況に基づき、お客様にはデータ転送料金が発生し、その後月次請求に加算されます。

ClickHouse Cloud では、以下について料金が発生します:

* ClickHouse Cloud からパブリックインターネットへのデータ送信（他クラウドプロバイダの
  他リージョンへの送信を含む）
* 同一クラウドプロバイダ内の別リージョンへのデータ送信

同一リージョン内のデータ転送や Private Link/Private Service Connect の利用および
データ転送には料金は発生しません。ただし、ユーザーに対して適切に課金することに
影響を与える利用パターンが見られた場合には、追加のデータ転送料金の要素を導入する
権利を留保します。

注意: Private Link/Private Service Connect の利用には、クラウドサービスプロバイダ (CSP)
の料金が適用されます。最新の情報については、CSP の料金ページを参照してください。

データ転送料金は、クラウドサービスプロバイダ (CSP) とリージョンによって異なります。
パブリックインターネットへの送信料金は、送信元リージョンのみに基づきます。
リージョン間（またはクロスリージョン）の料金は、送信元と送信先の両方のリージョンに
依存します。

**データ転送コストを最小化するためのベストプラクティス**

データのインバウンドおよびアウトバウンド時に、データ転送コストを最小化するために
考慮すべきいくつかのパターンがあります。

1. ClickHouse Cloud にデータを取り込んだり ClickHouse Cloud からデータを送信したりする際は、
   可能な限り圧縮を使用して、転送されるデータ量とそれに伴うコストを最小化してください。

2. ネイティブプロトコルで非インライン値を用いて `INSERT` を実行する場合
   （例: `INSERT INTO [TABLE] FROM INFILE [FILE] FORMAT NATIVE`）、ClickHouse
   クライアントはデータをパックするためにサーバーからメタデータを取得します。
   メタデータが `INSERT` ペイロードよりも大きい場合、サーバーの観点では
   受信量よりも送信量の方が多くなるという、直感に反する結果が見られることが
   あります。これが許容できない場合は、`VALUES` 構文でデータをインライン化するか、
   HTTP プロトコルの使用を検討してください。
