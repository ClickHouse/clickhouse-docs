## 2025年12月19日 \{#december-19-2025\}

* AWS ap-south-1 で、PCI 準拠サービスを起動できるようになりました。
* **統合ユーザーアイデンティティのプライベートプレビュー**
  コンソールからデータベースユーザーを管理したいお客様は、SQLコンソール向けの新しい認証方式を有効にできます。
  これにより、コンソールでのデータベースユーザー管理機能の提供に向けた作業を継続する間、新しい認証方式を試すことができます。
* **S3 ClickPipes で順不同モードが利用可能に**:
  お客様は、イベント駆動型分析のために、Amazon S3 から ClickHouse Cloud にデータを任意の順序で取り込めるようになりました。
  取り込み対象のファイルを辞書順に並べる必要はなくなりました。詳細は発表[ブログ](https://clickhouse.com/blog/clickpipes-s3-unordered-mode)をご覧ください。
* Fivetran コネクタは最近ベータに移行しました。Fivetran を使用していて、ClickHouse を宛先としてセットアップしたい場合は、こちらの[ドキュメント](https://fivetran.com/docs/destinations/clickhouse/setup-guide)を参照してください。

## 2025年12月12日 \{#december-12-2025\}

* **SAML SSO のセルフサービス設定**

  Enterprise のお客様は、サポートチケットを発行しなくても、コンソールで SAML 設定を完了できるようになりました。
  さらに、SAML を利用しているお客様は、アイデンティティプロバイダー経由で追加される新規ユーザーに割り当てるデフォルトロールを設定できるほか、カスタムのセッションタイムアウト設定も行えます。
  詳細については、[ドキュメント](/cloud/security/saml-setup)を参照してください。
* **Azure における最大レプリカサイズとスケーリング制限**

  すべての Azure リージョンで、最大レプリカサイズを 356 GiB に設定できるようになりました。ただし、`eastus2` では利用可能な最大レプリカサイズは 120 GiB です。

## 2025年11月21日 \{#november-21-2025\}

* ClickHouse Cloud が **AWS Israel (Tel Aviv) — il-central-1** で利用可能になりました
* ClickHouse 組織で、マーケットプレイスの従量課金サブスクリプションまたはプライベートオファーに課金できるようにするため、マーケットプレイスのオンボーディング体験を改善しました。

## 2025年11月14日 \{#november-14-2025\}

* **ClickHouse Cloud** が、**2つの新しいパブリックリージョン** で利用可能になったことをお知らせします。

  * **GCP Japan (asia-northeast1)**
  * **AWS Seoul (Asia Pacific, ap-northeast-2)** — **ClickPipes** でもサポートされるようになりました

  これらのリージョンは以前は **プライベートリージョン** として提供されていましたが、現在は **すべてのユーザーに開放** されています。
* Terraform と API で、サービスへのタグ追加と、タグによるサービスの絞り込みがサポートされるようになりました。

## 2025年11月7日 \{#november-7-2025\}

* ClickHouse Cloudコンソールから、レプリカサイズを 1 vCPU、4 GiB 刻みで設定できるようになりました。
  これらのオプションは、新しいサービスのセットアップ時にも、設定ページでレプリカの最小サイズと最大サイズを設定する際にも利用できます。
* カスタムハードウェアプロファイル (Enterprise ティアで利用可能) がアイドル状態に対応しました。
* ClickHouse Cloud で、AWS Marketplace を通じた購入手続きが簡素化され、[従量課金](https://aws.amazon.com/marketplace/pp/prodview-p4gwofrqpkltu?sr=0-2\&ref_=beagle\&applicationId=AWSMPContessa) と [コミットメント支出契約](https://aws.amazon.com/marketplace/pp/prodview-4qyeihstyym2s?sr=0-3\&ref_=beagle\&applicationId=AWSMPContessa) の個別オプションが提供されるようになりました。
* ClickHouse Cloud の ClickStack ユーザーが、アラート機能を利用できるようになりました。
  追加のセットアップやインフラ、サービス、設定なしで、ログ、メトリクス、トレース全体にわたるアラートを HyperDX UI で直接作成および管理できるようになりました。アラートは Slack、PagerDuty などと連携します。
  詳細は、[アラートのドキュメント](/use-cases/observability/clickstack/alerts)を参照してください。

## 2025年10月17日 \{#october-17-2025\}

* **サービス監視 - リソース使用状況ダッシュボード**
  CPU 使用率とメモリ使用率のメトリクス表示は、平均値ではなく、特定の期間における最大使用率を表示するように変更されます。これにより、リソース割り当て不足のインスタンスをより見つけやすくなります。
  さらに、CPU 使用率メトリクスには、ClickHouse Cloud のオートスケーラーで使用されるメトリクスにより近い、Kubernetes レベルの CPU 使用率メトリクスが表示されます。
* **外部バケット**
  ClickHouse Cloud では、バックアップをご利用のクラウドサービスプロバイダーのアカウントに直接エクスポートできるようになりました。
  AWS S3、Google Cloud Storage、または Azure Blob Storage の外部ストレージバケットを接続して、バックアップ管理を自分で行えます。

## 2025年8月29日 \{#august-29-2025\}

* [ClickHouse Cloud Azure Private Link](/cloud/security/azure-privatelink) では、リソースの識別に使用するフィルターが、Resource GUID から Resource ID に変更されました。従来の Resource GUID も引き続き使用でき、後方互換性も維持されていますが、Resource ID フィルターへの切り替えを推奨します。移行の詳細については、Azure Private Link の[ドキュメント](/cloud/security/azure-privatelink#obtaining-private-endpoint-resourceid)を参照してください。

## 2025年8月22日 \{#august-22-2025\}

* **AWS Glue 向け ClickHouse Connector**
  [AWS Marketplace](https://aws.amazon.com/marketplace/pp/prodview-eqvmuopqzdg7s) で提供されている公式の [ClickHouse Connector for AWS Glue](/integrations/glue) を利用できるようになりました。AWS Glue の Apache
  Spark ベースのサーバーレスエンジンを使用して、ClickHouse と他のデータソース間のデータ連携における抽出・変換・ロードを行います。テーブルの作成方法や、ClickHouse と Spark 間でのデータの書き込みと読み取り方法については、
  発表 [ブログ記事](http://clickhouse.com/blog/clickhouse-connector-aws-glue) を参照してください。
* **サービスの最小レプリカ数の変更**
  スケールアップされたサービスは、[スケールダウン](/manage/scaling) して 1 レプリカで運用できるようになりました (以前の最小値は 2 レプリカでした) 。注意: 単一レプリカのサービスは可用性が低下するため、プロダクション用途には推奨されません。
* ClickHouse Cloud は、サービスのスケーリングおよびサービスバージョンのアップグレードに関する通知を、管理者ロールに対してデフォルトで送信するようになります。通知設定で通知の受信設定を調整できます。

## 2025年8月13日 \{#august-13-2025\}

* **MongoDB CDC 向け ClickPipes がプライベートプレビューで提供開始**
  ClickPipes を使用して、数クリックで MongoDB から ClickHouse Cloud へデータをレプリケーションできるようになりました。これにより、
  外部の ETL ツールを使わずにリアルタイム分析を実現できます。このコネクタは継続的な
  レプリケーションに加えて、一度限りの移行にも対応しており、MongoDB Atlas とセルフホスト型の MongoDB
  デプロイの両方で利用できます。MongoDB CDC コネクタの概要については[ブログ記事](https://clickhouse.com/blog/mongodb-cdc-clickhouse-preview)をご覧ください。[こちらから早期アクセスにお申し込みいただけます](https://clickhouse.com/cloud/clickpipes/mongodb-cdc-connector)。

## 2025年8月8日 \{#august-08-2025\}

* **通知**: サービスで新しい ClickHouse バージョンへのアップグレードが開始されると、UI 通知を受け取れるようになりました。メール通知と Slack 通知は、通知センターから追加できます。
* **ClickPipes**: Azure Blob Storage (ABS) ClickPipes のサポートが ClickHouse Terraform プロバイダーに追加されました。ABS ClickPipe をプログラムで作成する方法の例については、Terraform プロバイダーのドキュメントを参照してください。
  * [バグ修正] Null エンジンを使用する宛先テーブルに書き込む Object Storage 用 ClickPipes で、UI に &quot;Total records&quot; と &quot;Data ingested&quot; のメトリクスが表示されるようになりました。
  * [バグ修正] UI のメトリクスの &quot;Time period&quot; セレクターは、選択した期間に関係なく &quot;24 hours&quot; が既定で表示されていました。現在はこの問題が修正され、選択した期間に応じて UI のチャートが正しく更新されます。
* **クロスリージョン private link (AWS)** が一般提供になりました。サポートされているリージョンの一覧については、[ドキュメント](/manage/security/aws-privatelink)を参照してください。

## 2025年7月31日 \{#july-31-2025\}

**ClickPipes で垂直スケーリングが利用可能に**

[ストリーミング ClickPipes で垂直スケーリングが利用可能になりました](https://clickhouse.com/blog/clickpipes-flexible-scaling-monitoring)。
この機能により、レプリカ数 (水平スケーリング) に加えて、各レプリカの
サイズも制御できます。各 ClickPipe の詳細ページには、
レプリカごとの CPU とメモリの使用率も表示されるようになり、これによって
ワークロードをより正確に把握し、安心してリサイズを計画できるようになります。

## 2025年7月24日 \{#july-24-2025\}

**MySQL CDC 向け ClickPipes がパブリックベータ版に**

ClickPipes の MySQL CDC コネクタが、パブリックベータ版として広くご利用いただけるようになりました。数回クリックするだけで、
外部依存なしに、MySQL (または MariaDB) のデータを ClickHouse Cloud にリアルタイムで直接レプリケーションできます。
コネクタの概要については[ブログ記事](https://clickhouse.com/blog/mysql-cdc-connector-clickpipes-beta)をご覧ください。[クイックスタート](https://clickhouse.com/docs/integrations/clickpipes/mysql)
に沿って、すぐに利用を開始できます。

## 2025年7月11日 \{#june-11-2025\}

* 新しいサービスでは、データベースおよびテーブルのメタデータを中央の**SharedCatalog**に保存するようになりました。これは、調整とオブジェクトのライフサイクル管理のための新しいモデルであり、次のことを可能にします:

  * 高い同時実行性でも**CloudスケールのDDL**
  * **耐障害性に優れた削除と新しいDDL操作**
  * ステートレスノードがディスクに依存せず起動できるようになったことで、**高速なスピンアップとウェイクアップ**
  * IcebergおよびDelta Lakeを含む、**ネイティブ形式とオープン形式の両方にまたがるステートレスコンピュート**

  SharedCatalogの詳細は[ブログ](https://clickhouse.com/blog/clickhouse-cloud-stateless-compute)をご覧ください

* GCP `europe-west4` で HIPAA 準拠のサービスを起動できるようになりました

## 2025年6月27日 \{#june-27-2025\}

* データベース特権を管理するための Terraform プロバイダーが正式にサポートされるようになりました。
  このプロバイダーはセルフマネージド環境のデプロイにも対応しています。詳細については、
  [blog](https://clickhouse.com/blog/new-terraform-provider-manage-clickhouse-database-users-roles-and-privileges-with-code)
  および [docs](https://registry.terraform.io/providers/ClickHouse/clickhousedbops/latest/docs)
  を参照してください。
* Enterprise ティアのサービスでは、[slow release channel](/manage/updates/#slow-release-channel-deferred-upgrades) に登録できるようになりました。これにより、
  通常のリリース後に実施されるアップグレードを 2 週間延期できるため、
  テストのための追加時間を確保できます。

## 2025年6月13日 \{#june-13-2025\}

* ClickHouse Cloud ダッシュボードが一般提供になったことをお知らせします。ダッシュボードでは、クエリを可視化し、フィルターやクエリパラメータを使用してデータを操作し、共有を管理できます。
* API key の IP フィルター: ClickHouse Cloud とのやり取りをさらに保護するための追加の保護レイヤーを導入します。API key の生成時に IP 許可リストを設定すると、その API key を使用できる場所を制限できます。詳細は [ドキュメント](https://clickhouse.com/docs/cloud/security/setting-ip-filters) を参照してください。

## 2025年5月30日 \{#may-30-2025\}

* ClickHouse Cloud における **Postgres CDC 向け ClickPipes** の**一般提供**を発表します。数回クリックするだけで、Postgres
  データベースをレプリケートし、超高速のリアルタイム分析を実現できるようになりました。このコネクタは、より高速なデータ同期、最短数秒のレイテンシ、schema 変更への自動対応、
  完全に安全な接続などを提供します。詳細は
  [ブログ](https://clickhouse.com/blog/postgres-cdc-connector-clickpipes-ga)を参照してください。利用を開始するには、[こちら](https://clickhouse.com/docs/integrations/clickpipes/postgres)の手順を参照してください。

* SQLコンソール のダッシュボードに新たな改善を導入しました:
  * 共有: ダッシュボードをチームメンバーと共有できるようになりました。4 つのアクセスレベルをサポートしており、全体設定とユーザー単位の両方で調整できます:
    * *書き込みアクセス*: 可視化の追加/編集、更新設定の変更、フィルターを介したダッシュボードの操作。
    * *オーナー*: ダッシュボードの共有、ダッシュボードの削除、および「書き込みアクセス」を持つユーザーのその他すべての権限。
    * *読み取り専用アクセス*: フィルターを介したダッシュボードの表示と操作
    * *アクセスなし*: ダッシュボードを表示できません
  * すでに作成されている既存のダッシュボードについては、Organization Administrators が自分自身をオーナーとして割り当てられるようになりました。
  * クエリビューから、SQLコンソール 内のテーブルまたはチャートをダッシュボードに追加できるようになりました。

* AWS および GCP 向けの [分散キャッシュ](https://clickhouse.com/cloud/distributed-cache-waitlist) の
  プレビュー参加者を募集しています。詳しくは[ブログ](https://clickhouse.com/blog/building-a-distributed-cache-for-s3)をご覧ください。

## 2025年5月16日 \{#may-16-2025\}

* ClickHouse Cloud のサービスで使用されているリソースを確認できる
  リソース使用状況ダッシュボードを導入しました。以下のメトリクス
  は system テーブル からスクレイプされ、このダッシュボードに表示されます:
  * メモリ &amp; CPU: `CGroupMemoryTotal` (割り当てメモリ) 、`CGroupMaxCPU` (割り当て CPU) 、
    `MemoryResident` (使用メモリ) 、および `ProfileEvent_OSCPUVirtualTimeMicroseconds` (使用 CPU) のグラフ
  * データ転送: ClickHouse Cloud へのデータの流入と流出を示すグラフ。詳細は[こちら](/cloud/manage/network-data-transfer)をご覧ください。
* 新しい ClickHouse Cloud Prometheus/Grafana mix-in の提供を開始しました。
  これは ClickHouse Cloud サービスの監視を簡素化するために
  構築されています。
  この mix-in は、Prometheus 互換の API エンドポイントを使用して、
  ClickHouse のメトリクスを既存の Prometheus および Grafana 環境にシームレスに統合します。あわせて、
  サービスの健全性とパフォーマンスをリアルタイムで可視化できる
  事前設定済みのダッシュボードも含まれます。詳しくは公開[ブログ](https://clickhouse.com/blog/monitor-with-new-prometheus-grafana-mix-in)をご覧ください。

## 2025年4月18日 \{#april-18-2025\}

* 新しい組織レベルのロール **Member** と、2つの新しいサービスレベルの
  ロール **Service Admin** および **Service Read Only** を導入しました。
  **Member** は、SAML SSO ユーザーにデフォルトで割り当てられる組織レベルのロールで、
  サインインとプロファイル更新のみを許可します。1つ以上のサービスに対する **Service Admin**
  および **Service Read Only** ロールは、**Member**、**Developer**、または **Billing Admin** ロールを持つユーザーに
  割り当てることができます。詳細については、
  [&quot;ClickHouse Cloud のアクセス管理&quot;](https://clickhouse.com/docs/cloud/security/cloud-access-management/overview)
  を参照してください
* ClickHouse Cloud は現在、**Enterprise** のお客様向けに、以下のリージョンで **HIPAA** および **PCI** サービスを
  提供しています: AWS eu-central-1、AWS eu-west-2、AWS us-east-2。
* **ClickPipes 向けのユーザー向け通知機能** を導入しました。この機能では、
  ClickPipes の障害に関するアラートがメール、ClickHouse Cloud UI、
  および Slack を通じて自動的に送信されます。メールおよび UI による通知は
  デフォルトで有効になっており、パイプラインごとに設定できます。**Postgres CDC ClickPipes** では、
  アラートの対象にレプリケーションスロットのしきい値 (**設定** タブで設定可能) 、特定のエラー
  種別、および障害を解決するためのセルフサービス手順も含まれます。
* **MySQL CDC プライベートプレビュー** の提供を開始しました。これにより、お客様は MySQL
  データベースを数クリックで ClickHouse Cloud にレプリケートでき、高速分析を実現し、
  外部 ETL ツールが不要になります。このコネクタは、MySQL がクラウド上 (RDS、
  Aurora、Cloud SQL、Azure など) にある場合でもオンプレミスにある場合でも、継続的な
  レプリケーションと1回限りの移行の両方をサポートします。プライベート
  プレビューへの登録は、[こちらのリンク](https://clickhouse.com/cloud/clickpipes/mysql-cdc-connector) から行えます。
* **ClickPipes 向け AWS PrivateLink** を導入しました。AWS PrivateLink を使用すると、
  VPC、AWS サービス、オンプレミス
  システム、および ClickHouse Cloud の間で安全な接続を確立できます。これにより、
  Postgres、MySQL、AWS 上の MSK などのソースからデータを移動する際も、トラフィックを
  パブリックインターネットにさらすことなく利用できます。さらに、VPC サービスエンドポイントを介したクロスリージョンアクセスにも対応しています。
  PrivateLink 接続のセットアップは現在、[完全にセルフサービスで](https://clickhouse.com/docs/integrations/clickpipes/aws-privatelink)
  ClickPipes を通じて行えるようになりました。

## 2025年4月4日 \{#april-4-2025\}

* ClickHouse Cloud用のSlack通知: ClickHouse Cloud で、コンソール内通知およびメール通知に加えて、課金、スケーリング、ClickPipes イベントに関する Slack 通知がサポートされるようになりました。これらの通知は ClickHouse Cloud Slack アプリケーションを介して送信されます。組織管理者は、通知の送信先となる Slack チャンネルを指定して、通知センターからこれらの通知を設定できます。
* Production サービスおよび Development サービスを利用しているユーザーには、請求書に ClickPipes とデータ転送の利用料金が表示されるようになりました。

## 2025年3月21日 \{#march-21-2025\}

* AWS の区域間 Private Link 接続は、現在ベータ版として提供されています。セットアップ方法の詳細とサポートされている区域の一覧については、
  ClickHouse Cloud private link [ドキュメント](/manage/security/aws-privatelink)を参照してください。
* AWS 上のサービスで利用できるレプリカの最大サイズは、現在 236 GiB RAM に設定されています。
  これにより、バックグラウンドプロセス用のリソースを
  確保しつつ、効率的に活用できます。

## 2025年1月27日 \{#january-27-2025\}

### ClickHouse Cloud のティアの変更 \{#changes-to-clickhouse-cloud-tiers\}

当社は、絶えず変化するお客様の要件に対応できるよう、製品の改善に継続的に取り組んでいます。GA として提供を開始してからこの 2 年の間に、ClickHouse Cloud は大きく進化してきました。また、お客様が当社の Cloud オファリングをどのように活用しているかについて、非常に有益な知見も得られました。

お客様のワークロードに対して、ClickHouse Cloud サービスのサイジングとコスト効率を最適化するため、新機能を導入します。これには、**compute-compute separation**、高性能なマシンタイプ、**単一レプリカサービス**が含まれます。さらに、自動スケーリングとマネージドアップグレードも、よりシームレスかつ迅速に実行できるよう進化させています。

最も要求の厳しいお客様やワークロードのニーズに対応するため、**新しい Enterprise ティア** を追加します。重点領域には、業界固有のセキュリティ機能とコンプライアンス機能、基盤ハードウェアやアップグレードに対するより高度な制御、さらに強化されたディザスタリカバリ機能が含まれます。

これらの変更を支えるため、現在の **Development** ティアと **Production** ティアを、進化するお客様基盤の実際の利用状況により合うよう再構成します。新しいアイデアやプロジェクトを試しているユーザー向けの **Basic** ティアと、プロダクションワークロードや大規模データを扱うユーザーに適した **Scale** ティアを導入します。

これらおよびその他の機能上の変更については、この[ブログ](https://clickhouse.com/blog/evolution-of-clickhouse-cloud-new-features-superior-performance-tailored-offerings)をご覧ください。既存のお客様は、[新しいプラン](https://clickhouse.com/pricing)を選択するための対応が必要です。お客様向けの案内は、組織管理者に電子メールで送信されています。

### 単一レプリカサービス \{#single-replica-services\}

「単一レプリカサービス」という概念を、独立実行型の提供形態としても、warehouse 内のサービスとしても導入します。独立実行型の提供形態では、単一レプリカサービスはサイズが制限されており、小規模なテスト用ワークロードでの利用を想定しています。warehouse 内では、単一レプリカサービスをより大きなサイズでデプロイでき、再起動可能な ETL ジョブのように、大規模な高可用性を必要としないワークロードに利用できます。

### 垂直オートスケーリングの改善 \{#vertical-auto-scaling-improvements\}

コンピュートレプリカ向けに、「Make Before Break」 (MBB) と呼ぶ新しい垂直スケーリングの仕組みを導入します。この方式では、古いレプリカを削除する前に、新しいサイズのレプリカを1つ以上追加するため、スケーリング操作中のキャパシティ低下を防げます。既存のレプリカを削除してから新しいレプリカを追加するまでの空白をなくすことで、MBBはよりシームレスで影響の少ないスケーリングプロセスを実現します。これは特に、リソース使用率の上昇によって追加キャパシティが必要になるスケールアップ時に有効です。こうした状況でレプリカを早期に削除すると、リソース制約をさらに悪化させるだけだからです。

### 水平スケーリング (GA) \{#horizontal-scaling-ga\}

水平スケーリングが一般提供になりました。API および Cloud Console から追加のレプリカを追加して、サービスをスケールアウトできるようになりました。詳しくは、[ドキュメント](/cloud/features/autoscaling/horizontal#manual-horizontal-scaling)を参照してください。

### Prometheus integration の改善 \{#prometheus-integration-improvements\}

Prometheus integration に、いくつかの機能強化を追加しました。

* **組織レベルのエンドポイント**。ClickHouse Cloud用の Prometheus integration が強化され、サービスレベルのメトリクスに加えて、API で **組織レベルのメトリクス** 用エンドポイントも利用できるようになりました。この新しいエンドポイントは、組織内のすべてのサービスのメトリクスを自動的に収集するため、Prometheus collector へのメトリクスのエクスポートを効率化できます。これらのメトリクスは、Grafana や Datadog などの可視化ツールと統合でき、組織全体のパフォーマンスをより包括的に把握できます。

  この機能は現在、すべてのユーザーが利用できます。詳細は[こちら](/integrations/prometheus)をご覧ください。

* **フィルタリングされたメトリクス**。ClickHouse Cloud用の Prometheus integration で、フィルタリングされたメトリクスの一覧を返す機能が追加されました。この機能により、サービスの健全性の監視に重要なメトリクスに絞り込めるため、レスポンスのペイロードサイズを削減できます。

  この機能は、API のオプションのクエリパラメータを使用して利用できます。これにより、データ収集を最適化し、Grafana や Datadog などのツールとの統合をより簡単に進められます。

  フィルタリングされたメトリクス機能は現在、すべてのユーザーが利用できます。詳細は[こちら](/integrations/prometheus)をご覧ください。
