## 2022年12月20日 \{#december-20-2022\}

このリリースでは、管理者がSQL Consoleにシームレスにログインできるようになったほか、コールドリードのパフォーマンスが向上し、ClickHouse Cloud用のMetabaseコネクタも改善されました。

### コンソールの変更点 \{#console-changes-27\}

* 管理者ユーザーが SQL Console にスムーズにアクセスできるようになりました
* 新規招待ユーザーのデフォルトロールを &quot;Administrator&quot; に変更しました
* オンボーディングアンケートを追加しました

### 信頼性とパフォーマンス \{#reliability-and-performance-3\}

* ネットワーク障害時に復旧できるよう、長時間実行される insert クエリに再試行ロジックを追加しました
* コールドリード時の読み取り性能を改善しました

### インテグレーションの変更点 \{#integrations-changes-26\}

* [Metabase plugin](/integrations/data-visualization/metabase-and-clickhouse.md) が、待望の v0.9.1 への大型アップデートを実施しました。これにより、最新バージョンの Metabase と互換性を持ち、ClickHouse Cloud で十分にテストされています。

## 2022年12月6日 - 一般提供 \{#december-6-2022---general-availability\}

ClickHouse Cloud は、SOC2 Type II 準拠、プロダクションワークロード向けのアップタイム SLA、公開ステータスページの提供により、本番運用に対応しました。今回のリリースには、AWS Marketplace 連携、ClickHouse ユーザー向けのデータ探索ワークベンチである SQL Console、さらに ClickHouse Cloud を自分のペースで学べる ClickHouse Academy など、主要な新機能が含まれています。詳細は、この[ブログ](https://clickhouse.com/blog/clickhouse-cloud-generally-available)をご覧ください。

### 本番運用向け \{#production-ready\}

* SOC2 Type II 準拠 (詳細は[ブログ](https://clickhouse.com/blog/clickhouse-cloud-is-now-soc-2-type-ii-compliant)および[Trust Center](https://trust.clickhouse.com/)を参照)
* ClickHouse Cloud の公開[ステータスページ](https://status.clickhouse.com/)
* 本番利用のユースケース向けの稼働率 SLA を提供
* [AWS Marketplace](https://aws.amazon.com/marketplace/pp/prodview-jettukeanwrfc)で提供

### 主な新機能 \{#major-new-capabilities\}

* ClickHouseユーザー向けのデータ探索ワークベンチであるSQL Consoleを導入
* ClickHouse Cloudでセルフペース学習ができる[ClickHouse Academy](https://learn.clickhouse.com/visitor_class_catalog)を公開

### 料金およびメータリングの変更 \{#pricing-and-metering-changes\}

* トライアル期間を30日に延長
* 固定容量で月額費用を抑えた Development Services を導入。スタータープロジェクトや開発／ステージング環境に適しています
* ClickHouse Cloud の運用とスケーリングの継続的な改善に伴い、Production Services の価格を新たに引き下げ
* コンピュートのメータリングにおける粒度と精度を向上

### インテグレーションの変更点 \{#integrations-changes-27\}

* ClickHouse Postgres / MySQL インテグレーションエンジンのサポートを有効化
* SQL ユーザー定義関数 (UDF) への対応を追加
* Kafka Connect sink をベータに移行
* バージョンや更新状況などに関する豊富なメタデータを追加し、インテグレーション UI を改善

### コンソールの変更点 \{#console-changes-28\}

* クラウドコンソールで多要素認証に対応
* モバイル端末でのクラウドコンソールのナビゲーションを改善

### ドキュメントの更新 \{#documentation-changes\}

* ClickHouse Cloud用の専用[ドキュメント](/cloud/overview)セクションを追加しました

### バグ修正 \{#bug-fixes-5\}

* 依存関係の解決に起因して、バックアップからの復元が正常に動作しないことがある既知の問題を修正しました

## 2022年11月29日 \{#november-29-2022\}

このリリースでは、SOC 2 Type II準拠への対応、ClickHouseのバージョン22.11への更新、および複数のClickHouseクライアントとインテグレーションの改善が行われました。

### 全般的な変更 \{#general-changes-4\}

* SOC 2 Type II に準拠しました (詳細は[ブログ](https://clickhouse.com/blog/clickhouse-cloud-is-now-soc-2-type-ii-compliant)および[Trust Center](https://trust.clickhouse.com)をご覧ください)

### コンソールの変更点 \{#console-changes-29\}

* サービスが自動的に一時停止されたことを示す &quot;Idle&quot; ステータスインジケーターを追加しました

### ClickHouse 22.11 バージョンアップグレード \{#clickhouse-2211-version-upgrade\}

* Hudi および DeltaLake のテーブルエンジンとテーブル関数のサポートを追加
* S3 のディレクトリの再帰的走査を改善
* 複合時間インターバル構文のサポートを追加
* 挿入時の再試行により、挿入処理の信頼性を向上
* 変更点の完全な一覧については、[詳細な 22.11 の変更履歴](#clickhouse-2211-version-upgrade)を参照してください

### 連携 \{#integrations-1\}

* Python クライアント: v3.11 をサポートし、insert パフォーマンスを改善
* Go クライアント: DateTime と Int64 のサポートを修正
* JS クライアント: 相互 SSL 認証をサポート
* dbt-clickhouse: DBT v1.3 をサポート

### バグ修正 \{#bug-fixes-6\}

* アップグレード後に古い ClickHouse バージョンが表示される不具合を修正しました
* &quot;default&quot; アカウントの権限を変更しても、セッションが中断されなくなりました
* 新規作成された非管理者アカウントでは、デフォルトで system テーブルにアクセスできなくなりました

### このリリースの既知の問題 \{#known-issues-in-this-release\}

* 依存関係の解決の影響で、バックアップからの復元が機能しない場合があります

## 2022年11月17日 \{#november-17-2022\}

このリリースでは、ローカルのClickHouseテーブルおよびHTTPソースからの辞書が利用可能になり、ムンバイ区域のサポートが追加され、クラウドコンソールのユーザー体験が改善されました。

### 一般変更 \{#general-changes-5\}

* ローカルの ClickHouse テーブルおよび HTTP ソースの[辞書](/sql-reference/statements/create/dictionary)のサポートを追加
* Mumbai [リージョン](/cloud/reference/supported-regions)のサポートを追加

### コンソールの変更点 \{#console-changes-30\}

* 請求書の表示形式を改善
* 支払い方法入力のユーザーインターフェイスを簡素化
* バックアップのアクティビティログをより詳細に記録するよう改善
* ファイルのアップロード時のエラー処理を改善

### バグ修正 \{#bug-fixes-7\}

* 一部のパーツにサイズの大きい単一ファイルが含まれている場合に、バックアップが失敗する可能性がある不具合を修正しました
* アクセスリストの変更が同時に適用されると、バックアップからの復元に成功しない不具合を修正しました

### 既知の問題 \{#known-issues\}

* 依存関係の解決の影響で、バックアップからの復元が動作しない場合があります

## 2022年11月3日 \{#november-3-2022\}

このリリースでは、料金体系から読み取りおよび書き込みユニットを廃止し (詳細は[料金ページ](https://clickhouse.com/pricing)を参照してください) 、ClickHouseのバージョンを22.10に更新するとともに、セルフサービスのお客様向けに、より大規模な垂直スケーリングをサポートし、基本値の改善によって信頼性を向上させています。

### 一般的な変更 \{#general-changes-6\}

* 料金モデルから読み取り/書き込みユニットを削除しました

### 設定の変更 \{#configuration-changes-2\}

* 安定性のため、設定 `allow_suspicious_low_cardinality_types`、`allow_suspicious_fixed_string_types`、`allow_suspicious_codecs` (デフォルトは false) は、ユーザーが変更できなくなりました。

### コンソールの変更点 \{#console-changes-31\}

* 有料顧客向けに、垂直スケーリングのセルフサービスで利用できる上限をメモリ720GBまで引き上げました
* バックアップからの復元ワークフローを改善し、IP Access Listルールとパスワードを設定できるようにしました
* サービス作成ダイアログに、GCP と Azure のウェイトリストを追加しました
* ファイルのアップロード時のエラー処理を改善しました
* 請求管理のワークフローを改善しました

### ClickHouse 22.10 バージョンのアップグレード \{#clickhouse-2210-version-upgrade\}

* オブジェクトストア利用時のマージを改善しました。大きなパーツ (少なくとも 10 GiB) が多数存在する場合に、「パーツが多すぎる」のしきい値を緩和しています。これにより、単一テーブルの単一パーティションで最大 PB 級のデータを扱えるようになりました。
* `min_age_to_force_merge_seconds` 設定によってマージの制御を改善し、一定時間のしきい値経過後にマージを実行できるようにしました。
* 設定をリセットするための MySQL 互換構文 `SET setting_name = DEFAULT` を追加しました。
* Morton 曲線エンコーディング、Java 整数ハッシュ、乱数生成のための関数を追加しました。
* 変更の完全な一覧については、[詳細な 22.10 変更履歴](#clickhouse-2210-version-upgrade)を参照してください。

## 2022年10月25日 \{#october-25-2022\}

このリリースでは、小規模なワークロードのコンピュート消費量を大幅に削減し、コンピュート料金を引き下げ (詳細は[料金](https://clickhouse.com/pricing)ページを参照) 、より適切なデフォルト設定によって安定性を向上させるとともに、ClickHouse Cloud コンソールの Billing および Usage ビューを強化しました。

### 全般的な変更 \{#general-changes-7\}

* サービスの最小メモリ割り当てを24Gに削減
* サービスのアイドルタイムアウトを30分から5分に短縮

### 設定の変更 \{#configuration-changes-3\}

* max&#95;parts&#95;in&#95;total を 100k から 10k に引き下げました。MergeTree テーブルにおける `max_parts_in_total` 設定のデフォルト値を、100,000 から 10,000 に引き下げました。この変更は、データパーツ数が多いとクラウド上のサービスの起動時間が長くなる可能性が高いことが確認されたためです。通常、パーツ数が多いのは、パーティションキーが細かすぎることを示しており、多くの場合は意図せずそのように設定されているため、避けるべきです。デフォルト値を変更することで、このようなケースをより早い段階で検出できるようになります。

### コンソールの変更点 \{#console-changes-32\}

* トライアルユーザー向けに、Billingビューでクレジット使用状況の詳細情報を拡充
* Usageビューのツールチップとヘルプテキストを改善し、料金ページへのリンクを追加
* IPフィルタリングの設定切り替え時の操作フローを改善
* クラウドコンソールに確認メールの再送ボタンを追加

## 2022年10月4日 - ベータ \{#october-4-2022---beta\}

ClickHouse Cloud のパブリックベータは、2022年10月4日に開始されました。詳しくは、こちらの[ブログ](https://clickhouse.com/blog/clickhouse-cloud-public-beta)をご覧ください。

ClickHouse Cloud のバージョンは、ClickHouse core v22.10 をベースにしています。互換性のある機能の一覧については、[Cloud Compatibility](/whats-new/cloud-compatibility)ガイドを参照してください。
