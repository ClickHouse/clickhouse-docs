ClickHouse Cloudは、データ転送のインフラとエグレスを測定します。
これには、ClickHouse Cloudへの入出データ、および地域内データ転送や地域間データ転送が含まれます。
この使用状況はサービスレベルで追跡されます。この使用に基づいて、顧客はデータ転送料金が発生し、それが月次請求書に追加されます。

ClickHouse Cloudは以下の項目に対して料金を請求します：
- ClickHouse Cloudから公共インターネットへのデータエグレス、他のクラウドプロバイダーの他の地域へのデータを含む。
- 同じクラウドプロバイダー内の別の地域へのデータエグレス。

地域内データ転送やPrivate Link/Private Service Connectの使用およびデータ転送には料金が発生しません。
ただし、私たちはユーザーに適切に料金を請求する能力に影響を与える使用パターンが見られた場合、追加のデータ転送料金の次元を実施する権利を留保します。

データ転送料金はクラウドサービスプロバイダー（CSP）や地域によって異なります。
公共インターネットエグレスの価格は、発信地域のみに基づいています。
地域間（またはクロスリージョン）価格は、発信地域と宛先地域の両方に依存します。

:::note
ClickHouse Cloudは、発信地域と宛先地域に応じて、Tier 1からTier 4までの層で地域間使用を測定します。以下の表には、地域間データ転送の各組み合わせに対するテールが示されています。ClickHouse Cloudの請求使用画面では、データ転送使用量が層別に表示されます。

:::note
ネイティブプロトコルを使用して非インライン値（例：INSERT INTO [TABLE] FROM INFILE [FILE] FORMAT NATIVE）のINSERTを行うと、ClickHouseクライアントがメタデータをサーバーから取得してデータをパッキングします。メタデータがINSERTペイロードより大きい場合、サーバーの視点からは、直感に反して、入出のエグレスが入るよりも多くなることがあります。これが受け入れられない場合は、VALUES構文を使用してデータをインラインにするか、HTTPプロトコルを使用することを検討してください。
:::
