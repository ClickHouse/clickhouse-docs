モデルパラメータでは、エージェントが使用するモデルや、そのモデルによる応答の生成方法を制御します。
**エージェントビルダー** タブで、**モデル** 見出しの下にある **モデルを選択** をクリックします。
これにより、**モデルパラメータ** ウィンドウが開きます。

## プロバイダーとモデル \{#provider-and-model\}

* **プロバイダー** - 利用する上流のLLMプロバイダーを選択します。
* **モデル** - そのプロバイダーの特定のモデルを選択します。モデルごとに得意分野が異なります。たとえば、大規模な推論モデルは計画重視のタスクに適しており、小規模で高速なモデルは定型的な問い合わせに適しています。

**プロバイダー** と **モデル** はどちらも必須です。パネル内のその他の項目は、選択したモデルでサポートされる内容に応じて変わります。

## コンテキストと出力の上限 \{#context-and-output-limits\}

* **最大コンテキストトークン数** - エージェントがモデルに送信するコンテキスト全体の上限を設定します。モデルのデフォルト値を使う場合は **System** のままにします。コストを抑えるには小さくし、大きな入力に対する推論が必要なエージェントには大きくします。
* **最大出力トークン数** - エージェントの応答サイズの上限を設定します。**System** を選ぶとモデルのデフォルト値が使われます。応答が長すぎる場合は小さくし、途中で切れてしまう場合は大きく設定します。
* **ファイルのトークン上限** - アップロードされた単一ファイルがコンテキストに含められるトークン数の上限を設定します。ユーザーが大きなファイルを添付したときに、会話の他の部分を圧迫しないようにしたい場合に便利です。

## サンプリング \{#sampling\}

* **Temperature** - ランダム性。値が高いほど (0.7–1.0) 出力はよりランダムになり、低いほど (0.0–0.3) より焦点が絞られ、決定的になります。調整する場合は、これか Top P のどちらか一方にすることを推奨します。
* **Top P** - nucleus sampling。モデルが出力するトークンの選び方を変更します。
* **Top K** - 各ステップで、サンプリング対象を最も確率の高い上位 K 個のトークンに制限します。一部のプロバイダーでサポートされており、temperature とは異なる軸で決定性を制御します。

特定の挙動を狙って調整しているのでなければ、スライダーはデフォルト値付近のままにしてください。小さな変更で効果が大きく変わることはまれで、大きく変更すると出力品質が低下する場合があります。

## 推論の制御 \{#reasoning-controls\}

拡張推論に対応したモデルで利用できます。利用可能な項目は プロバイダー によって異なります。

* **Thinking** - モデルの拡張推論モードを切り替えます。有効にすると、モデルは最終回答の前に内部的な思考 トークン を生成します。これにより、難しいタスクでの精度は通常向上しますが、その分レイテンシと トークン 消費が増加します。
* **Thinking Budget** - 思考フェーズに割り当てる トークン 予算です。モデルはこの数の トークン を消費すると、思考を停止して回答します。
* **Effort** - 推論の強さを大まかに調整する設定です (**Auto**、low、medium、high) 。思考 トークン 予算を直接指定できない推論モデルで使用されます。
* **Thought Visibility** - モデルの思考をユーザーにインライン表示するか、折りたたみ表示の中に隠すか、完全に表示しないかを制御します。

## 会話の動作 \{#conversation-behavior\}

* **ファイルを再送信** — オンにすると、前のターンで添付したファイルが以降のすべてのターンでも再送信されるため、モデルがそれらを見失わずに済みます。会話が短い場合や、モデルが途中でファイルを順次要約していく場合は、トークンを節約するためオフにしてください。
* **プロンプトのキャッシュを使用** — プロバイダーが対応している場合、プロンプトの再利用可能な部分をキャッシュして、ターンをまたいで指示やツールの説明が繰り返される会話でコストとレイテンシを削減します。
* **ウェブ検索** — 対応モデルで、プロバイダー標準のウェブ検索のオン/オフを切り替えます。これは [Web search tool](/cloud/features/ai-ml/agents/builder/web-search) とは異なり、エージェントのツールの 1 つとして実行されるのではなく、プロバイダー側の機能として動作します。

## リセット \{#reset\}

パネル下部の**モデルパラメータをリセット**をクリックすると、すべてのフィールドがシステムのデフォルト値に戻ります。いろいろ試したあとで、まっさらな状態からやり直したい場合に使用してください。
