# 日付と時刻を扱う関数

このセクションのほとんどの関数は、オプションのタイムゾーン引数を受け付けます（例：`Europe/Amsterdam`）。この場合、ローカル（デフォルト）のタイムゾーンではなく、指定されたタイムゾーンが使用されます。

**例**

```sql
SELECT
    toDateTime('2016-06-15 23:00:00') AS time,
    toDate(time) AS date_local,
    toDate(time, 'Asia/Yekaterinburg') AS date_yekat,
    toString(time, 'US/Samoa') AS time_samoa
```

```text
┌────────────────time─┬─date_local─┬─date_yekat─┬─time_samoa──────────┐
│ 2016-06-15 23:00:00 │ 2016-06-15 │ 2016-06-16 │ 2016-06-15 09:00:00 │
└─────────────────────┴────────────┴────────────┴─────────────────────┘
```

## UTCTimestamp {#UTCTimestamp}

導入バージョン: v22.11

クエリ解析時点の現在の日付と時刻を返します。この関数は定数式です。

この関数は`now('UTC')`と同じ結果を返します。MySQL互換性のためにのみ追加されました。[`now`](#now)の使用を推奨します。

**構文**

```sql
UTCTimestamp()
```

**エイリアス**: `UTC_timestamp`

**引数**

- なし

**戻り値**

クエリ解析時点の現在の日付と時刻を返します。[`DateTime`](/sql-reference/data-types/datetime)

**例**

**現在のUTCタイムスタンプを取得**

```sql title=クエリ
SELECT UTCTimestamp()
```

```response title=結果
┌──────UTCTimestamp()─┐
│ 2024-05-28 08:32:09 │
└─────────────────────┘
```

## addInterval {#addInterval}

導入バージョン: v22.11

インターバルを別のインターバルまたはインターバルのタプルに加算します。

:::note
同じ型のインターバルは単一のインターバルに統合されます。例えば、`toIntervalDay(1)` と `toIntervalDay(2)` が渡された場合、結果は `(1,1)` ではなく `(3)` になります。
:::

**構文**

```sql
addInterval(interval_1, interval_2)
```

**引数**

- `interval_1` — 最初のインターバルまたはインターバルのタプル。[`Interval`](/sql-reference/data-types/int-uint) または [`Tuple(Interval)`](/sql-reference/data-types/tuple)
- `interval_2` — 加算する2番目のインターバル。[`Interval`](/sql-reference/data-types/int-uint)

**戻り値**

インターバルのタプル [`Tuple(Interval)`](/sql-reference/data-types/tuple) を返します。

**例**

**インターバルの加算**

```sql title=クエリ
SELECT addInterval(INTERVAL 1 DAY, INTERVAL 1 MONTH);
SELECT addInterval((INTERVAL 1 DAY, INTERVAL 1 YEAR), INTERVAL 1 MONTH);
SELECT addInterval(INTERVAL 2 DAY, INTERVAL 1 DAY)
```

```response title=レスポンス
┌─addInterval(toIntervalDay(1), toIntervalMonth(1))─┐
│ (1,1)                                             │
└───────────────────────────────────────────────────┘
┌─addInterval((toIntervalDay(1), toIntervalYear(1)), toIntervalMonth(1))─┐
│ (1,1,1)                                                                │
└────────────────────────────────────────────────────────────────────────┘
┌─addInterval(toIntervalDay(2), toIntervalDay(1))─┐
│ (3)                                             │
└─────────────────────────────────────────────────┘
```

## age {#age}

導入バージョン: v23.1

`startdate`と`enddate`の差分を指定された単位で返します。
差分は1ナノ秒の精度で計算されます。

例えば、2021-12-29と2022-01-01の差分は、日単位では3日、
月単位では0ヶ月、年単位では0年となります。

ageの代替として、関数[`dateDiff`](#dateDiff)を参照してください。

**構文**

```sql
age('unit', startdate, enddate[, timezone])
```

**引数**

- `unit` — 結果の時間単位。

| 単位        | 指定可能な値                          |
| ----------- | ---------------------------------------- |
| nanosecond  | `nanosecond`, `nanoseconds`, `ns`        |
| microsecond | `microsecond`, `microseconds`, `us`, `u` |
| millisecond | `millisecond`, `milliseconds`, `ms`      |
| second      | `second`, `seconds`, `ss`, `s`           |
| minute      | `minute`, `minutes`, `mi`, `n`           |
| hour        | `hour`, `hours`, `hh`, `h`               |
| day         | `day`, `days`, `dd`, `d`                 |
| week        | `week`, `weeks`, `wk`, `ww`              |
| month       | `month`, `months`, `mm`, `m`             |
| quarter     | `quarter`, `quarters`, `qq`, `q`         |
| year        | `year`, `years`, `yyyy`, `yy`            |

- `startdate` — 減算される最初の時刻値(被減数)。[`Date`](/sql-reference/data-types/date)、[`Date32`](/sql-reference/data-types/date32)、[`DateTime`](/sql-reference/data-types/datetime)、または[`DateTime64`](/sql-reference/data-types/datetime64)
- `enddate` — 減算元となる2番目の時刻値(減数)。[`Date`](/sql-reference/data-types/date)、[`Date32`](/sql-reference/data-types/date32)、[`DateTime`](/sql-reference/data-types/datetime)、または[`DateTime64`](/sql-reference/data-types/datetime64)
- `timezone` — オプション。タイムゾーン名。指定された場合、startdateとenddateの両方に適用されます。指定されない場合、startdateとenddateのタイムゾーンが使用されます。それらが異なる場合、結果は未定義です。[`String`](/sql-reference/data-types/string)

**戻り値**

enddateとstartdateの差分を指定された単位で表現した値を返します。[`Int32`](/sql-reference/data-types/int-uint)

**例**

**時間単位での差分を計算**

```sql title=クエリ

```

```response title=結果
┌─age('hour', toDateTime('2018-01-01 22:30:00'), toDateTime('2018-01-02 23:00:00'))─┐
│                                                                                24 │
└───────────────────────────────────────────────────────────────────────────────────┘
```

**異なる単位での差分を計算**

```sql title=クエリ
SELECT
    toDate('2022-01-01') AS e,
    toDate('2021-12-29') AS s,
    age('day', s, e) AS day_age,
    age('month', s, e) AS month_age,
    age('year', s, e) AS year_age
```

```response title=結果
┌──────────e─┬──────────s─┬─day_age─┬─month_age─┬─year_age─┐
│ 2022-01-01 │ 2021-12-29 │       3 │         0 │        0 │
└────────────┴────────────┴─────────┴───────────┴──────────┘
```

## dateDiff {#dateDiff}

導入バージョン: v23.4

`startdate`と`enddate`の間で指定された`unit`の境界を越えた回数を返します。
差分は相対単位を使用して計算されます。例えば、2021-12-29と2022-01-01の差分は、単位dayの場合は3日
([`toRelativeDayNum`](#toRelativeDayNum)を参照)、単位monthの場合は1ヶ月([`toRelativeMonthNum`](#toRelativeMonthNum)を参照)、単位yearの場合は1年
([`toRelativeYearNum`](#toRelativeYearNum)を参照)となります。

単位に`week`が指定された場合、`dateDiff`は週が月曜日から始まると仮定します。
この動作は、デフォルトで週が日曜日から始まる関数`toWeek()`とは異なることに注意してください。

`dateDiff`の代替として、関数[`age`](#age)を参照してください。

**構文**

```sql
dateDiff(unit, startdate, enddate[, timezone])
```

**エイリアス**: `timestampDiff`, `TIMESTAMP_DIFF`, `DATE_DIFF`, `date_diff`, `timestamp_diff`

**引数**

- `unit` — 結果の間隔の種類。

| 単位        | 使用可能な値                          |
| ----------- | ---------------------------------------- |
| nanosecond  | `nanosecond`, `nanoseconds`, `ns`        |
| microsecond | `microsecond`, `microseconds`, `us`, `u` |
| millisecond | `millisecond`, `milliseconds`, `ms`      |
| second      | `second`, `seconds`, `ss`, `s`           |
| minute      | `minute`, `minutes`, `mi`, `n`           |
| hour        | `hour`, `hours`, `hh`, `h`               |
| day         | `day`, `days`, `dd`, `d`                 |
| week        | `week`, `weeks`, `wk`, `ww`              |
| month       | `month`, `months`, `mm`, `m`             |
| quarter     | `quarter`, `quarters`, `qq`, `q`         |
| year        | `year`, `years`, `yyyy`, `yy`            |

- `startdate` — 減算される最初の時刻値(被減数)。[`Date`](/sql-reference/data-types/date)、[`Date32`](/sql-reference/data-types/date32)、[`DateTime`](/sql-reference/data-types/datetime)、または[`DateTime64`](/sql-reference/data-types/datetime64)


```response title=レスポンス

│  25 │
└─────┘
```

```sql title=クエリ
    toDate('2021-12-29') AS s,
```

## formatDateTime {#formatDateTime}

## timeSlots {#timeSlots}

導入バージョン: v1.1

`StartTime`から始まり`Duration`秒間続く時間間隔に対して、この間隔内の時点を`Size`秒単位で切り捨てた時刻の配列を返します。`Size`はオプションパラメータで、デフォルト値は1800(30分)です。

これは、例えば対応するセッション内のページビューを検索する際に必要となります。

`DateTime64`の場合、戻り値のスケールは`StartTime`のスケールと異なることがあります。すべての引数の中で最も大きいスケールが採用されます。

**構文**

```sql
timeSlots(StartTime, Duration[, Size])
```

**引数**

- `StartTime` — 間隔の開始時刻。[`DateTime`](/sql-reference/data-types/datetime)または[`DateTime64`](/sql-reference/data-types/datetime64)
- `Duration` — 間隔の長さ(秒単位)。[`UInt32`](/sql-reference/data-types/int-uint)または[`DateTime64`](/sql-reference/data-types/datetime64)
- `Size` — オプション。タイムスロットのサイズ(秒単位)。デフォルト値は1800(30分)。[`UInt32`](/sql-reference/data-types/int-uint)または[`DateTime64`](/sql-reference/data-types/datetime64)

**戻り値**

DateTime/DateTime64の配列を返します(戻り値の型は`StartTime`の型と一致します)。DateTime64の場合、戻り値のスケールは`StartTime`のスケールと異なることがあります - すべての引数の中で最も大きいスケールが採用されます。[`Array(DateTime)`](/sql-reference/data-types/array)または[`Array(DateTime64)`](/sql-reference/data-types/array)

**例**

**間隔のタイムスロットを生成**

```sql title=Query
SELECT timeSlots(toDateTime('2012-01-01 12:20:00'), toUInt32(600));
SELECT timeSlots(toDateTime('1980-12-12 21:01:02', 'UTC'), toUInt32(600), 299);
SELECT timeSlots(toDateTime64('1980-12-12 21:01:02.1234', 4, 'UTC'), toDecimal64(600.1, 1), toDecimal64(299, 0))
```

```response title=Response
┌─timeSlots(toDateTime('2012-01-01 12:20:00'), toUInt32(600))─┐
│ ['2012-01-01 12:00:00','2012-01-01 12:30:00']               │
└─────────────────────────────────────────────────────────────┘
┌─timeSlots(toDateTime('1980-12-12 21:01:02', 'UTC'), toUInt32(600), 299)─┐
│ ['1980-12-12 20:56:13','1980-12-12 21:01:12','1980-12-12 21:06:11']     │
└─────────────────────────────────────────────────────────────────────────┘
┌─timeSlots(toDateTime64('1980-12-12 21:01:02.1234', 4, 'UTC'), toDecimal64(600.1, 1), toDecimal64(299, 0))─┐
│ ['1980-12-12 20:56:13.0000','1980-12-12 21:01:12.0000','1980-12-12 21:06:11.0000']                        │
└───────────────────────────────────────────────────────────────────────────────────────────────────────────┘
```

## toStartOfInterval {#toStartOfInterval}

導入バージョン: v20.1

この関数は、`toStartOfInterval(date_or_date_with_time, INTERVAL x unit [, time_zone])` 構文により、他の `toStartOf*()` 関数を一般化したものです。

例:

- `toStartOfInterval(t, INTERVAL 1 YEAR)` は `toStartOfYear(t)` と同じ結果を返します
- `toStartOfInterval(t, INTERVAL 1 MONTH)` は `toStartOfMonth(t)` と同じ結果を返します
- `toStartOfInterval(t, INTERVAL 1 DAY)` は `toStartOfDay(t)` と同じ結果を返します
- `toStartOfInterval(t, INTERVAL 15 MINUTE)` は `toStartOfFifteenMinutes(t)` と同じ結果を返します

計算は特定の時点を基準として実行されます:

| 間隔        | 開始点                  |
| ----------- | ---------------------- |
| YEAR        | 0年                 |
| QUARTER     | 1900 Q1                |
| MONTH       | 1900年1月           |
| WEEK        | 1970年、第1週 (01-05) |
| DAY         | 1970-01-01             |
| HOUR        | (\*)                   |
| MINUTE      | 1970-01-01 00:00:00    |
| SECOND      | 1970-01-01 00:00:00    |
| MILLISECOND | 1970-01-01 00:00:00    |
| MICROSECOND | 1970-01-01 00:00:00    |
| NANOSECOND  | 1970-01-01 00:00:00    |

(\*) 時間間隔は特殊です: 計算は常に当日の 00:00:00 (深夜0時) を基準として実行されます。そのため、1から23までの時間値のみが有用です。

単位 `WEEK` が指定された場合、`toStartOfInterval` は週が月曜日から始まると仮定します。この動作は、デフォルトで週が日曜日から始まる関数 `toStartOfWeek` とは異なることに注意してください。

2番目のオーバーロードは、TimescaleDBの `time_bucket()` 関数、およびPostgreSQLの `date_bin()` 関数をエミュレートします。

**構文**

```sql
toStartOfInterval(value, INTERVAL x unit[, time_zone])
toStartOfInterval(value, INTERVAL x unit[, origin[, time_zone]])
```

**エイリアス**: `date_bin`, `time_bucket`

**引数**

- `value` — 切り捨てる日付または日時の値。[`Date`](/sql-reference/data-types/date) または [`DateTime`](/sql-reference/data-types/datetime) または [`DateTime64`](/sql-reference/data-types/datetime64)
- `x` — 間隔の長さを表す数値。- `unit` — 間隔の単位: YEAR、QUARTER、MONTH、WEEK、DAY、HOUR、MINUTE、SECOND、MILLISECOND、MICROSECOND、NANOSECOND。- `time_zone` — オプション。文字列としてのタイムゾーン名。- `origin` — オプション。計算の基準点(2番目のオーバーロードのみ)。

**返り値**

入力値を含む間隔の開始点を返します。[`DateTime`](/sql-reference/data-types/datetime)

**例**

**基本的な間隔の丸め**

```sql title=Query
SELECT toStartOfInterval(toDateTime('2023-01-15 14:30:00'), INTERVAL 1 MONTH)
```

```response title=Response
┌─toStartOfInt⋯alMonth(1))─┐
│               2023-01-01 │
└──────────────────────────┘
```

**基準点の使用**

```sql title=Query
SELECT toStartOfInterval(toDateTime('2023-01-01 14:45:00'), INTERVAL 1 MINUTE, toDateTime('2023-01-01 14:35:30'))
```

```response title=Response
┌─toStartOfInt⋯14:35:30'))─┐
│      2023-01-01 14:44:30 │
└──────────────────────────┘
```

## toWeek {#toWeek}

導入バージョン: v20.1

この関数は、日付または日時の週番号を返します。`toWeek()`の2引数形式では、週の開始曜日を日曜日または月曜日のどちらにするか、および戻り値の範囲を`0`から`53`または`1`から`53`のどちらにするかを指定できます。

[`toISOWeek()`](#toWeek)は、`toWeek(date,3)`と等価な互換性関数です。

次の表は、mode引数の動作を説明しています。

| Mode | 週の最初の曜日 | 範囲 | 週1は次の条件を満たす最初の週 ...  |
| ---- | ----------------- | ----- | ----------------------------- |
| 0    | 日曜日            | 0-53  | 今年の日曜日を含む    |
| 1    | 月曜日            | 0-53  | 今年に4日以上含まれる |
| 2    | 日曜日            | 1-53  | 今年の日曜日を含む    |
| 3    | 月曜日            | 1-53  | 今年に4日以上含まれる |
| 4    | 日曜日            | 0-53  | 今年に4日以上含まれる |
| 5    | 月曜日            | 0-53  | 今年の月曜日を含む    |
| 6    | 日曜日            | 1-53  | 今年に4日以上含まれる |
| 7    | 月曜日            | 1-53  | 今年の月曜日を含む    |
| 8    | 日曜日            | 1-53  | 1月1日を含む            |
| 9    | 月曜日            | 1-53  | 1月1日を含む            |

「今年に4日以上含まれる」という意味を持つmode値の場合、週番号はISO 8601:1988に従って付けられます:

- 1月1日を含む週が新年に4日以上ある場合、それが週1となります。
- それ以外の場合は、前年の最終週となり、次の週が週1となります。

「1月1日を含む」という意味を持つmode値の場合、1月1日を含む週が週1となります。
その週が新年に何日含まれるかは関係なく、1日だけであっても同様です。
つまり、12月の最終週が翌年の1月1日を含む場合、それは翌年の週1となります。

最初の引数は、[`parseDateTime64BestEffort()`](type-conversion-functions.md#parsedatetime64besteffort)がサポートする形式の[`String`](../data-types/string.md)として指定することもできます。文字列引数のサポートは、特定のサードパーティツールが期待するMySQLとの互換性のためにのみ存在します。文字列引数のサポートは将来的に新しいMySQL互換性設定に依存する可能性があり、また文字列解析は一般的に低速であるため、使用しないことを推奨します。

**構文**

```sql
toWeek(datetime[, mode[, time_zone]])
```

**エイリアス**: `week`

**引数**

- `datetime` — 週番号を取得する日付または日時。[`Date`](/sql-reference/data-types/date)または[`DateTime`](/sql-reference/data-types/datetime)
- `mode` — オプション。`0`から`9`のmodeで、週の最初の曜日と週番号の範囲を決定します。デフォルトは`0`。
- `time_zone` — オプション。タイムゾーン。[`String`](/sql-reference/data-types/string)

**戻り値**

指定されたmodeに従って週番号を返します。[`UInt32`](/sql-reference/data-types/int-uint)

**例**

**異なるmodeで週番号を取得**

```sql title=クエリ
SELECT toDate('2016-12-27') AS date, toWeek(date) AS week0, toWeek(date,1) AS week1, toWeek(date,9) AS week9
```

```response title=レスポンス
┌───────date─┬─week0─┬─week1─┬─week9─┐
│ 2016-12-27 │    52 │    52 │     1 │
└────────────┴───────┴───────┴───────┘
```
