# 算術関数



## 概要 {#overview}

算術関数は、`UInt8`、`UInt16`、`UInt32`、`UInt64`、`Int8`、`Int16`、`Int32`、`Int64`、`Float32`、または`Float64`型の任意の2つのオペランドに対して機能します。

演算を実行する前に、両方のオペランドは結果型にキャストされます。結果型は以下のように決定されます(以下の関数ドキュメントで別途指定されていない限り):

- 両方のオペランドが32ビット以下の場合、結果型のサイズは2つのオペランドのうち大きい方の次に大きい型のサイズになります(整数サイズの昇格)。例えば、`UInt8 + UInt16 = UInt32`または`Float32 * Float32 = Float64`となります。
- いずれかのオペランドが64ビット以上の場合、結果型のサイズは2つのオペランドのうち大きい方と同じサイズになります。例えば、`UInt32 + UInt128 = UInt128`または`Float32 * Float64 = Float64`となります。
- いずれかのオペランドが符号付きの場合、結果型も符号付きになります。それ以外の場合は符号なしになります。例えば、`UInt32 * Int32 = Int64`となります。

これらのルールにより、結果型はすべての可能な結果を表現できる最小の型となることが保証されます。これにより値範囲の境界付近でオーバーフローのリスクが生じますが、最大ネイティブ整数幅である64ビットを使用して計算が高速に実行されることが保証されます。この動作は、最大の整数型として64ビット整数(BIGINT)を提供する多くの他のデータベースとの互換性も保証します。

例:

```sql
SELECT toTypeName(0), toTypeName(0 + 0), toTypeName(0 + 0 + 0), toTypeName(0 + 0 + 0 + 0)
```

```text
┌─toTypeName(0)─┬─toTypeName(plus(0, 0))─┬─toTypeName(plus(plus(0, 0), 0))─┬─toTypeName(plus(plus(plus(0, 0), 0), 0))─┐
│ UInt8         │ UInt16                 │ UInt32                          │ UInt64                                   │
└───────────────┴────────────────────────┴─────────────────────────────────┴──────────────────────────────────────────┘
```

オーバーフローはC++と同じ方法で発生します。

## abs {#abs}

導入バージョン: v1.1

`x`の絶対値を計算します。`x`が符号なし型の場合は効果がありません。`x`が符号付き型の場合は、符号なしの数値を返します。

**構文**

```sql
abs(x)
```

**引数**

- `x` — 絶対値を取得する値

**戻り値**

`x`の絶対値

**例**

**使用例**

```sql title=クエリ
SELECT abs(-0.5)
```

```response title=レスポンス
0.5
```


## avg2 {#avg2}

導入バージョン: v25.11

指定された引数の平均値を計算して返します。
数値型および時間型をサポートします。

**構文**

```sql
avg2(x1, x2])
```

**引数**

- `x1, x2]` — 平均を計算する2つの値を受け取ります。

**返り値**

指定された引数の平均値を返します。戻り値の型は、互換性のある最大の型に昇格されます。

**例**

**数値型**

```sql title=Query
SELECT avg2(toUInt8(3), 1.0) AS result, toTypeName(result) AS type;
-- UInt8は比較のために64ビットに昇格する必要があるため、返される型はFloat64になります。
```

```response title=Response
┌─result─┬─type────┐
│      2 │ Float64 │
└────────┴─────────┘
```

**Decimal型**

```sql title=Query
SELECT avg2(toDecimal32(1, 2), 2) AS result, toTypeName(result) AS type;
```

```response title=Response
┌─result─┬─type──────────┐
│    1.5 │ Decimal(9, 2) │
└────────┴───────────────┘
```

**Date型**

```sql title=Query
SELECT avg2(toDate('2025-01-01'), toDate('2025-01-05')) AS result, toTypeName(result) AS type;
```

```response title=Response
┌─────result─┬─type─┐
│ 2025-01-03 │ Date │
└────────────┴──────┘
```

**DateTime型**

```sql title=Query
SELECT avg2(toDateTime('2025-01-01 00:00:00'), toDateTime('2025-01-03 12:00:00')) AS result, toTypeName(result) AS type;
```

```response title=Response
┌──────────────result─┬─type─────┐
│ 2025-01-02 06:00:00 │ DateTime │
└─────────────────────┴──────────┘
```

**Time64型**

```sql title=Query
SELECT avg2(toTime64('12:00:00', 0), toTime64('14:00:00', 0)) AS result, toTypeName(result) AS type;
```

```response title=Response
┌───result─┬─type──────┐
│ 13:00:00 │ Time64(0) │
└──────────┴───────────┘
```


## byteSwap {#byteSwap}

導入バージョン: v23.10

整数のバイトを反転します。つまり、[エンディアン](https://en.wikipedia.org/wiki/Endianness)を変更します。

以下の例は次のように計算できます:

1. 10進数の整数をビッグエンディアン形式の16進数に変換します。例: 3351772109 -> C7 C7 FB CD (4バイト)
2. バイトを反転します。例: C7 C7 FB CD -> CD FB C7 C7
3. ビッグエンディアンを仮定して結果を整数に戻します。例: CD FB C7 C7 -> 3455829959
   この関数の使用例の1つは、IPv4アドレスの反転です:

```result
┌─toIPv4(byteSwap(toUInt32(toIPv4('205.251.199.199'))))─┐
│ 199.199.251.205                                       │
└───────────────────────────────────────────────────────┘
```

**構文**

```sql
byteSwap(x)
```

**引数**

- `x` — 整数値。[`(U)Int*`](/sql-reference/data-types/int-uint)

**戻り値**

バイトが反転された`x`を返します。[`(U)Int*`](/sql-reference/data-types/int-uint)

**例**

**使用例**

```sql title=クエリ
SELECT byteSwap(3351772109)
```

```response title=結果
3455829959
```

**8ビット**

```sql title=クエリ
SELECT byteSwap(54)
```

```response title=結果
54
```

**16ビット**

```sql title=クエリ
SELECT byteSwap(4135)
```

```response title=結果
10000
```

**32ビット**

```sql title=クエリ
SELECT byteSwap(3351772109)
```

```response title=結果
3455829959
```

**64ビット**

```sql title=クエリ
SELECT byteSwap(123294967295)
```

```response title=結果
18439412204227788800
```


## divide {#divide}

導入バージョン: v1.1

2つの値 `a` と `b` の商を計算します。結果の型は常に [Float64](/sql-reference/data-types/float) です。
整数除算については `intDiv` 関数を使用してください。

:::note
`0` による除算は `inf`、`-inf`、または `nan` を返します。
:::

**構文**

```sql
divide(x, y)
```

**引数**

- `x` — 被除数
- `y` — 除数

**戻り値**

x と y の商

**例**

**2つの数値の除算**

```sql title=Query
SELECT divide(25,5) AS quotient, toTypeName(quotient)
```

```response title=Response
5 Float64
```

**ゼロによる除算**

```sql title=Query
SELECT divide(25,0)
```

```response title=Response
inf
```


## divideDecimal {#divideDecimal}

導入バージョン: v22.12

2つの小数値に対して除算を実行します。結果の値は[Decimal256](/sql-reference/data-types/decimal)型になります。
結果のスケールは`result_scale`引数(範囲`[0, 76]`の定数整数)で明示的に指定できます。指定されない場合、結果のスケールは与えられた引数の最大スケールになります。

:::note
この関数は通常の`divide`よりも大幅に低速で動作します。
精密な制御が本当に必要でない場合や高速な計算が必要な場合は、[divide](#divide)の使用を検討してください。
:::

**構文**

```sql
divideDecimal(x, y[, result_scale])
```

**引数**

- `x` — 第1の値: [Decimal](/sql-reference/data-types/decimal)。
- `y` — 第2の値: [Decimal](/sql-reference/data-types/decimal)。
- `result_scale` — 結果のスケール。型は[Int/UInt](/sql-reference/data-types/int-uint)。

**返り値**

指定されたスケールでの除算結果。[`Decimal256`](/sql-reference/data-types/decimal)

**例**

**例1**

```sql title=Query
divideDecimal(toDecimal256(-12, 0), toDecimal32(2.1, 1), 10)
```

```response title=Response
┌─divideDecimal(toDecimal256(-12, 0), toDecimal32(2.1, 1), 10)─┐
│                                                -5.7142857142 │
└──────────────────────────────────────────────────────────────┘
```

**例2**

```sql title=Query
SELECT toDecimal64(-12, 1) / toDecimal32(2.1, 1);
SELECT toDecimal64(-12, 1) as a, toDecimal32(2.1, 1) as b, divideDecimal(a, b, 1), divideDecimal(a, b, 5);
```

```response title=Response
┌─divide(toDecimal64(-12, 1), toDecimal32(2.1, 1))─┐
│                                             -5.7 │
└──────────────────────────────────────────────────┘
┌───a─┬───b─┬─divideDecimal(toDecimal64(-12, 1), toDecimal32(2.1, 1), 1)─┬─divideDecimal(toDecimal64(-12, 1), toDecimal32(2.1, 1), 5)─┐
│ -12 │ 2.1 │                                                       -5.7 │                                                   -5.71428 │
└─────┴─────┴────────────────────────────────────────────────────────────┴────────────────────────────────────────────────────────────┘
```

## intDiv {#intDiv}

導入バージョン: v1.1

2つの値 `x` を `y` で整数除算します。つまり、商を次に小さい整数に切り捨てて計算します。

結果は被除数(第1パラメータ)と同じビット幅になります。

ゼロ除算、商が被除数の範囲に収まらない場合、または最小の負数をマイナス1で除算した場合に例外がスローされます。

**構文**

```sql
intDiv(x, y)
```

**引数**

- `x` — 左辺のオペランド。- `y` — 右辺のオペランド。

**戻り値**

`x` と `y` の整数除算の結果

**例**

**2つの浮動小数点数の整数除算**

```sql title=クエリ
SELECT intDiv(toFloat64(1), 0.001) AS res, toTypeName(res)
```

```response title=レスポンス
┌──res─┬─toTypeName(intDiv(toFloat64(1), 0.001))─┐
│ 1000 │ Int64                                   │
└──────┴─────────────────────────────────────────┘
```

**商が被除数の範囲に収まらない場合**

```sql title=クエリ
SELECT
intDiv(1, 0.001) AS res,
toTypeName(res)
```

```response title=レスポンス
Received exception from server (version 23.2.1):
Code: 153. DB::Exception: Received from localhost:9000. DB::Exception:
Cannot perform integer division, because it will produce infinite or too
large number: While processing intDiv(1, 0.001) AS res, toTypeName(res).
(ILLEGAL_DIVISION)
```
