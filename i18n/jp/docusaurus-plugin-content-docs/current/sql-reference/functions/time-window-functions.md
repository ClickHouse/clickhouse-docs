## hop {#hop}

導入バージョン: v21.12

ホッピング時間ウィンドウは固定期間（`window_interval`）を持ち、指定されたホップ間隔（`hop_interval`）でホップします。`hop_interval`が`window_interval`より小さい場合、ホッピングウィンドウは重複します。したがって、レコードは複数のウィンドウに割り当てられる可能性があります。

1つのレコードが複数のホップウィンドウに割り当てられる可能性があるため、hop関数がWINDOW VIEWなしで使用される場合、この関数は最初のウィンドウの境界のみを返します。

**構文**

```sql
hop(time_attr, hop_interval, window_interval[, timezone])
```

**引数**

- `time_attr` — 日付と時刻。[`DateTime`](/sql-reference/data-types/datetime)
- `hop_interval` — 正のホップ間隔。[`Interval`](/sql-reference/data-types/int-uint)
- `window_interval` — 正のウィンドウ間隔。[`Interval`](/sql-reference/data-types/int-uint)
- `timezone` — オプション。タイムゾーン名。[`String`](/sql-reference/data-types/string)

**返される値**

対応するホッピングウィンドウの下限（含む）と上限（含まない）を返します。[`Tuple(DateTime, DateTime)`](/sql-reference/data-types/tuple)

**例**

**ホッピングウィンドウ**

```sql title=Query
SELECT hop(now(), INTERVAL '1' DAY, INTERVAL '2' DAY)
```

```response title=Response
('2024-07-03 00:00:00','2024-07-05 00:00:00')
```


## hopEnd {#hopEnd}

導入バージョン: v22.1

対応するホッピングウィンドウの排他的上限を返します。

1つのレコードが複数のホップウィンドウに割り当てられる可能性があるため、`WINDOW VIEW`なしでhop関数を使用した場合、この関数は最初のウィンドウの境界のみを返します。

**構文**

```sql
hopEnd(time_attr, hop_interval, window_interval[, timezone])
```

**引数**

- `time_attr` — 日付と時刻。[`DateTime`](/sql-reference/data-types/datetime)
- `hop_interval` — 正のホップ間隔。[`Interval`](/sql-reference/data-types/int-uint)
- `window_interval` — 正のウィンドウ間隔。[`Interval`](/sql-reference/data-types/int-uint)
- `timezone` — オプション。タイムゾーン名。[`String`](/sql-reference/data-types/string)

**返される値**

対応するホッピングウィンドウの排他的上限を返します。[`DateTime`](/sql-reference/data-types/datetime)

**例**

**ホッピングウィンドウの終了**

```sql title=Query
SELECT hopEnd(now(), INTERVAL '1' DAY, INTERVAL '2' DAY)
```

```response title=Response
2024-07-05 00:00:00
```

**構文**

```sql
hopStart(time_attr, hop_interval, window_interval[, timezone])
```

**引数**

- `time_attr` — 日付と時刻。[`DateTime`](/sql-reference/data-types/datetime)
- `hop_interval` — 正のホップ間隔。[`Interval`](/sql-reference/data-types/int-uint)
- `window_interval` — 正のウィンドウ間隔。[`Interval`](/sql-reference/data-types/int-uint)
- `timezone` — オプション。タイムゾーン名。[`String`](/sql-reference/data-types/string)

**返り値**

対応するホッピングウィンドウの下限（境界を含む）を返します。[`DateTime`](/sql-reference/data-types/datetime)

```response title=Response

```


## tumble {#tumble}

導入バージョン: v21.12

タンブリング時間ウィンドウは、固定期間（`interval`）を持つ重複しない連続したウィンドウにレコードを割り当てます。

**構文**

```sql
tumble(time_attr, interval[, timezone])
```

**引数**

- `time_attr` — 日付と時刻。[`DateTime`](/sql-reference/data-types/datetime)
- `interval` — ウィンドウの間隔。[`Interval`](/sql-reference/data-types/int-uint)
- `timezone` — オプション。タイムゾーン名。[`String`](/sql-reference/data-types/string)

**返り値**

対応するタンブリングウィンドウの下限（含む）と上限（含まない）を返します。[`Tuple(DateTime, DateTime)`](/sql-reference/data-types/tuple)

**例**

**タンブリングウィンドウ**

```sql title=Query
SELECT tumble(now(), toIntervalDay('1'))
```

```response title=Response
('2024-07-04 00:00:00','2024-07-05 00:00:00')
```
