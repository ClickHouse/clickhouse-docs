# ハッシュ関数

ハッシュ関数は、要素の決定論的な疑似ランダムシャッフルに使用できます。

Simhashは、類似した引数に対して近似したハッシュ値を返すハッシュ関数です。

ほとんどのハッシュ関数は、任意の型の任意の数の引数を受け取ります。

:::note
NULLのハッシュはNULLです。Nullable列の非NULLハッシュを取得するには、タプルでラップします:

```sql
SELECT cityHash64(tuple(NULL))
```

:::

:::note
テーブル全体の内容のハッシュを計算するには、`sum(cityHash64(tuple(*)))`(または他のハッシュ関数)を使用します。`tuple`はNULL値を持つ行がスキップされないことを保証します。`sum`は行の順序が結果に影響しないことを保証します。
:::

## BLAKE3 {#BLAKE3}

Introduced in: v22.10

BLAKE3ハッシュ文字列を計算し、結果のバイト列をFixedStringとして返します。
この暗号学的ハッシュ関数は、BLAKE3 RustライブラリによってClickHouseに統合されています。
この関数は非常に高速で、SHA-256と同じ長さのハッシュを生成しながら、SHA-2と比較して約2倍高速なパフォーマンスを示します。
FixedString(32)型のバイト配列としてBLAKE3ハッシュを返します。

**構文**

```sql
BLAKE3(message)
```

**引数**

- `message` — ハッシュ化する入力文字列。[`String`](/sql-reference/data-types/string)

**戻り値**

入力文字列の32バイトBLAKE3ハッシュを固定長文字列として返します。[`FixedString(32)`](/sql-reference/data-types/fixedstring)

**例**

**ハッシュ**

```sql title=Query
SELECT hex(BLAKE3('ABC'))
```

```response title=Response
┌─hex(BLAKE3('ABC'))───────────────────────────────────────────────┐
│ D1717274597CF0289694F75D96D444B992A096F1AFD8E7BBFA6EBB1D360FEDFC │
└──────────────────────────────────────────────────────────────────┘
```

## URLHash {#URLHash}

導入バージョン: v1.1

URLから取得した文字列に対して、何らかの正規化を用いた高速で適度な品質の非暗号化ハッシュ関数です。

このハッシュ関数には2つのモードがあります:

| モード              | 説明                                                                                                                                                                                  |
| ----------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `URLHash(url)`    | 末尾に `/`、`?`、または `#` の記号がある場合、それらを除いた文字列からハッシュを計算します。                                                                                   |
| `URLHash(url, N)` | URL階層のNレベルまでの文字列からハッシュを計算します。末尾に `/`、`?`、または `#` の記号がある場合、それらを除きます。レベルは `URLHierarchy` と同じです。 |

**構文**

```sql
URLHash(url[, N])
```

**引数**

- `url` — ハッシュ化するURL文字列。[`String`](/sql-reference/data-types/string)
- `N` — オプション。URL階層のレベル。[`(U)Int*`](/sql-reference/data-types/int-uint)

**戻り値**

`url` の計算されたハッシュ値を返します。[`UInt64`](/sql-reference/data-types/int-uint)

**例**

**使用例**

```sql title=クエリ
SELECT URLHash('https://www.clickhouse.com')
```

```response title=レスポンス
┌─URLHash('htt⋯house.com')─┐
│     13614512636072854701 │
└──────────────────────────┘
```

**指定されたレベルでのURLのハッシュ**

```sql title=クエリ
SELECT URLHash('https://www.clickhouse.com/docs', 0);
SELECT URLHash('https://www.clickhouse.com/docs', 1);
```

```response title=レスポンス
-- https://www.clickhouse.com のハッシュ
┌─URLHash('htt⋯m/docs', 0)─┐
│     13614512636072854701 │
└──────────────────────────┘
-- https://www.clickhouse.com/docs のハッシュ
┌─URLHash('htt⋯m/docs', 1)─┐
│     13167253331440520598 │
└──────────────────────────┘
```


## cityHash64 {#cityHash64}

導入バージョン: v1.1

64ビットの[CityHash](https://github.com/google/cityhash)ハッシュ値を生成します。

これは高速な非暗号化ハッシュ関数です。
文字列パラメータにはCityHashアルゴリズムを使用し、その他のデータ型のパラメータには実装固有の高速な非暗号化ハッシュ関数を使用します。
最終結果を得るためにCityHashコンビネータを使用します。

:::info
GoogleはClickHouseに追加された後、CityHashのアルゴリズムを変更しました。
つまり、ClickHouseのcityHash64とGoogleの上流のCityHashは現在異なる結果を生成します。
ClickHouseのcityHash64はCityHash v1.0.2に対応しています。
:::

:::note
計算されたハッシュ値は、異なる引数型の同じ入力値に対して等しくなる場合があります。
これは例えば、異なるサイズの整数型、同じデータを持つ名前付きおよび名前なしの`Tuple`、`Map`と対応する同じデータを持つ`Array(Tuple(key, value))`型に影響します。
:::

**構文**

```sql
cityHash64(arg1[, arg2, ...])
```

**引数**

- `arg1[, arg2, ...]` — ハッシュを計算する可変個の入力引数。[`Any`](/sql-reference/data-types)

**戻り値**

入力引数の計算されたハッシュを返します。[`UInt64`](/sql-reference/data-types/int-uint)

**例**

**呼び出し例**

```sql title=Query
SELECT cityHash64(array('e','x','a'), 'mple', 10, toDateTime('2019-06-15 23:00:00')) AS CityHash, toTypeName(CityHash) AS type;
```

```response title=Response
┌─────────────CityHash─┬─type───┐
│ 12072650598913549138 │ UInt64 │
└──────────────────────┴────────┘
```

**行順序まで正確にテーブル全体のチェックサムを計算**

```sql title=Query
CREATE TABLE users (
    id UInt32,
    name String,
    age UInt8,
    city String
)
ENGINE = MergeTree
ORDER BY tuple();

INSERT INTO users VALUES
(1, 'Alice', 25, 'New York'),
(2, 'Bob', 30, 'London'),
(3, 'Charlie', 35, 'Tokyo');

SELECT groupBitXor(cityHash64(*)) FROM users;
```

```response title=Response
┌─groupBitXor(⋯age, city))─┐
│     11639977218258521182 │
└──────────────────────────┘
```

## halfMD5 {#halfMD5}

導入バージョン: v1.1

すべての入力パラメータを文字列として[解釈](/sql-reference/functions/type-conversion-functions#reinterpretasstring)し、それぞれのMD5ハッシュ値を計算します。次にハッシュを結合し、結果として得られる文字列のハッシュの最初の8バイトを取得し、ビッグエンディアンバイト順で[UInt64](/sql-reference/data-types/int-uint)として解釈します。この関数は比較的低速です(プロセッサコアあたり毎秒500万個の短い文字列)。

代わりに[`sipHash64`](#sipHash64)関数の使用を検討してください。

この関数は可変個の入力パラメータを受け取ります。
引数はサポートされている任意のデータ型を使用できます。
一部のデータ型では、引数の型が異なる場合でも、同じ値に対してハッシュ関数の計算値が同じになることがあります(異なるサイズの整数、同じデータを持つ名前付きおよび名前なしのTuple、Mapおよび対応する同じデータを持つArray(Tuple(key, value))型)。

**構文**

```sql
halfMD5(arg1[, arg2, ..., argN])
```

**引数**

- `arg1[, arg2, ..., argN]` — ハッシュを計算する可変個の引数。[`Any`](/sql-reference/data-types)

**戻り値**

指定された入力パラメータから計算されたhalf MD5ハッシュをビッグエンディアンバイト順の`UInt64`として返します。[`UInt64`](/sql-reference/data-types/int-uint)

**例**

**使用例**

```sql title=Query
SELECT HEX(halfMD5('abc', 'cde', 'fgh'));
```

```response title=Response
┌─hex(halfMD5('abc', 'cde', 'fgh'))─┐
│ 2C9506B7374CFAF4                  │
└───────────────────────────────────┘
```

## sipHash64 {#sipHash64}

導入バージョン: v1.1

64ビットの[SipHash](https://en.wikipedia.org/wiki/SipHash)ハッシュ値を生成します。

これは暗号学的ハッシュ関数です。[`MD5`](#MD5)ハッシュ関数よりも少なくとも3倍高速に動作します。

この関数は、すべての入力パラメータを文字列として[解釈](/sql-reference/functions/type-conversion-functions#reinterpretasstring)し、それぞれのハッシュ値を計算します。
その後、以下のアルゴリズムを使用してハッシュを結合します:

1. 1番目と2番目のハッシュ値が配列に連結され、ハッシュ化されます。
2. 前のステップで計算されたハッシュ値と3番目の入力パラメータのハッシュが同様の方法でハッシュ化されます。
3. この計算は、元の入力の残りのすべてのハッシュ値に対して繰り返されます。

:::note
計算されたハッシュ値は、異なる引数型の同じ入力値に対して等しくなる場合があります。
これは例えば、異なるサイズの整数型、同じデータを持つ名前付きおよび名前なしの`Tuple`、`Map`と対応する同じデータを持つ`Array(Tuple(key, value))`型に影響します。
:::

**構文**

```sql
sipHash64(arg1[, arg2, ...])
```

**引数**

- `arg1[, arg2, ...]` — 可変個の入力引数。[`Any`](/sql-reference/data-types)

**戻り値**

入力引数の計算されたハッシュ値を返します。[`UInt64`](/sql-reference/data-types/int-uint)

**例**

**使用例**

```sql title=Query
SELECT sipHash64(array('e','x','a'), 'mple', 10, toDateTime('2019-06-15 23:00:00')) AS SipHash, toTypeName(SipHash) AS type;
```

```response title=Response
┌──────────────SipHash─┬─type───┐
│ 11400366955626497465 │ UInt64 │
└──────────────────────┴────────┘
```
