# OLAP とは何か?

[OLAP](https://en.wikipedia.org/wiki/Online_analytical_processing) は Online Analytical Processing の略です。技術的な観点とビジネスの観点という 2 つの側面から捉えられる幅広い用語です。ごく大まかに言えば、これらの単語を逆から読むと理解しやすくなります。

**Processing（処理）** — あるソースデータを処理して…

**Analytical（分析）** — …分析レポートや洞察を生成し…

**Online（オンライン）** — …それをリアルタイムに行います。



## ビジネス視点から見たOLAP {#olap-from-the-business-perspective}

近年、ビジネス関係者はデータの価値を認識し始めています。根拠なく意思決定を行う企業は、競合他社に後れを取ることが少なくありません。成功企業のデータ駆動型アプローチでは、ビジネス上の意思決定に少しでも役立つ可能性のあるすべてのデータを収集し、このデータを適時に分析できる仕組みが必要となります。ここでOLAPデータベース管理システム(DBMS)の出番となります。

ビジネスの観点から見ると、OLAPは企業が業務活動を継続的に計画、分析、報告することを可能にし、効率を最大化し、経費を削減し、最終的には市場シェアの獲得につながります。これは社内システムで実現することも、ウェブ/モバイル分析サービス、CRMサービスなどのSaaSプロバイダーに外部委託することもできます。OLAPは多くのBI(ビジネスインテリジェンス)アプリケーションの基盤技術となっています。

ClickHouseはOLAPデータベース管理システムであり、ドメイン固有のデータを分析するSaaSソリューションのバックエンドとして頻繁に使用されています。しかし、一部の企業はサードパーティプロバイダーとデータを共有することに依然として消極的であるため、社内データウェアハウスのシナリオも有効な選択肢となります。


## 技術的観点から見たOLAP {#olap-from-the-technical-perspective}

すべてのデータベース管理システムは、OLAP(Online **Analytical** Processing)とOLTP(Online **Transactional** Processing)の2つのグループに分類できます。前者は大量の履歴データに基づくレポート作成に重点を置いていますが、実行頻度は低くなります。後者は通常、継続的なトランザクションストリームを処理し、データの現在の状態を常に更新します。

実際には、OLAPとOLTPは二項対立のカテゴリではなく、スペクトラムとして捉えられています。ほとんどの実システムは通常どちらか一方に重点を置いていますが、反対の種類のワークロードも必要な場合には、何らかのソリューションや回避策を提供しています。この状況により、企業は統合された複数のストレージシステムを運用せざるを得なくなることがよくあります。これはそれほど大きな問題ではないかもしれませんが、システム数の増加は保守コストの増加につながるため、近年では単一のデータベース管理システムで両方の種類のワークロードを同等に処理できるHTAP(**Hybrid Transactional/Analytical Processing**)への移行が進んでいます。

DBMSが純粋なOLAPまたは純粋なOLTPとして始まった場合でも、競争に対応するためにHTAPの方向へ進まざるを得ません。ClickHouseも例外ではありません。当初は[可能な限り高速なOLAPシステム](/concepts/why-clickhouse-is-so-fast)として設計されており、現在でも完全なトランザクションサポートは備えていませんが、一貫性のある読み取り/書き込みや、データの更新/削除のためのミューテーションなどの機能が追加されています。

OLAPシステムとOLTPシステム間の根本的なトレードオフは依然として存在します:

- 分析レポートを効率的に作成するには、列を個別に読み取れることが重要であるため、ほとんどのOLAPデータベースは[カラムナ型](https://clickhouse.com/engineering-resources/what-is-columnar-database)です;
- 一方、列を個別に保存すると、追加やインプレース変更などの行操作のコストが列数に比例して増加します(システムが念のためにイベントのすべての詳細を収集しようとする場合、列数は膨大になる可能性があります)。したがって、ほとんどのOLTPシステムは行単位でデータを保存します。
