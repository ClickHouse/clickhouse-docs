# OLAP とは何ですか？

[OLAP](https://en.wikipedia.org/wiki/Online_analytical_processing) は Online Analytical Processing の略です。これは技術的な観点とビジネス的な観点の 2 つの視点から捉えられる幅広い用語です。ざっくり言えば、これらの単語を逆から読んでいくとイメージしやすくなります：

**Processing** — あるソースデータが処理されて…

**Analytical** — …分析レポートやインサイトが得られて…

**Online** — …それがリアルタイムで実行されます。



## ビジネスの観点から見たOLAP {#olap-from-the-business-perspective}

近年、ビジネスパーソンはデータの価値を認識し始めています。データに基づかず盲目的に意思決定を行う企業は、競合他社に後れを取ることが少なくありません。成功企業のデータ駆動型アプローチでは、ビジネス上の意思決定に少しでも役立つ可能性のあるすべてのデータを収集し、このデータをタイムリーに分析できる仕組みが必要となります。ここでOLAPデータベース管理システム(DBMS)の出番です。

ビジネスの観点から見ると、OLAPは企業が業務活動を継続的に計画、分析、報告することを可能にし、効率を最大化し、経費を削減し、最終的には市場シェアの獲得につながります。これは社内システムで実現することも、ウェブ/モバイル分析サービス、CRMサービスなどのSaaSプロバイダーに外部委託することもできます。OLAPは多くのBI(ビジネスインテリジェンス)アプリケーションの基盤技術となっています。

ClickHouseはOLAPデータベース管理システムであり、ドメイン固有のデータを分析するSaaSソリューションのバックエンドとして頻繁に使用されています。しかし、一部の企業はサードパーティプロバイダーとデータを共有することに依然として消極的であるため、社内データウェアハウスのシナリオも有効な選択肢となります。


## 技術的観点から見たOLAP {#olap-from-the-technical-perspective}

すべてのデータベース管理システムは、OLAP(Online **Analytical** Processing)とOLTP(Online **Transactional** Processing)の2つのグループに分類できます。前者は大量の履歴データに基づくレポート作成に重点を置いていますが、実行頻度は低くなります。後者は通常、継続的なトランザクションストリームを処理し、データの現在の状態を常に更新します。

実際には、OLAPとOLTPは二項対立のカテゴリではなく、スペクトラムとして捉えられています。ほとんどの実システムは通常どちらか一方に重点を置いていますが、反対のタイプのワークロードも必要な場合には、何らかのソリューションや回避策を提供しています。この状況により、企業は統合された複数のストレージシステムを運用せざるを得なくなることがよくあります。これはそれほど大きな問題ではないかもしれませんが、システムが増えると保守コストが増加するため、近年の傾向としては、単一のデータベース管理システムで両方のタイプのワークロードを同等に処理できるHTAP(**Hybrid Transactional/Analytical Processing**)へと向かっています。

DBMSが純粋なOLAPまたは純粋なOLTPとして始まったとしても、競争に対応するためにHTAPの方向へ進まざるを得ません。ClickHouseも例外ではありません。当初は[可能な限り高速なOLAPシステム](/concepts/why-clickhouse-is-so-fast)として設計されており、現在も完全なトランザクションサポートは備えていませんが、一貫性のある読み取り/書き込みや、データの更新/削除のためのミューテーションなどの機能が追加されています。

OLAPシステムとOLTPシステムの間の根本的なトレードオフは依然として存在します:

- 分析レポートを効率的に作成するには、カラムを個別に読み取れることが重要であるため、ほとんどのOLAPデータベースは[カラム型](https://clickhouse.com/engineering-resources/what-is-columnar-database)です;
- 一方、カラムを個別に保存すると、追加やインプレース変更などの行操作のコストが、カラム数に比例して増加します(システムがイベントのすべての詳細を念のため収集しようとする場合、カラム数は膨大になる可能性があります)。したがって、ほとんどのOLTPシステムは行単位でデータを保存します。
