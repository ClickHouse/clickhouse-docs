# OLAP とは何か?

[OLAP](https://en.wikipedia.org/wiki/Online_analytical_processing) は Online Analytical Processing の略です。技術的な視点とビジネス的な視点という 2 つの観点から捉えられる、幅広い意味を持つ用語です。最も大まかには、これらの単語を逆から読むとイメージしやすくなります:

**Processing** — 何らかのソースデータが処理され…

**Analytical** — …分析レポートやインサイトが得られ…

**Online** — …それがリアルタイムで行われます。



## ビジネスの観点から見たOLAP {#olap-from-the-business-perspective}

近年、ビジネスパーソンはデータの価値を認識し始めています。データに基づかない意思決定を行う企業は、競合他社に後れを取ることが少なくありません。成功している企業のデータ駆動型アプローチでは、ビジネス上の意思決定に少しでも役立つ可能性のあるすべてのデータを収集し、このデータをタイムリーに分析できる仕組みが必要となります。ここでOLAPデータベース管理システム(DBMS)の出番となります。

ビジネスの観点から、OLAPは企業が業務活動を継続的に計画、分析、報告することを可能にし、効率を最大化し、コストを削減し、最終的に市場シェアを獲得します。これは社内システムで実現することも、ウェブ/モバイル分析サービス、CRMサービスなどのSaaSプロバイダーに外部委託することもできます。OLAPは多くのBI(ビジネスインテリジェンス)アプリケーションの基盤技術です。

ClickHouseはOLAPデータベース管理システムであり、ドメイン固有のデータを分析するためのこれらのSaaSソリューションのバックエンドとして頻繁に使用されています。しかし、一部の企業はサードパーティプロバイダーとデータを共有することに依然として消極的であるため、社内データウェアハウスのシナリオも有効な選択肢となります。


## 技術的観点から見たOLAP {#olap-from-the-technical-perspective}

すべてのデータベース管理システムは、OLAP(Online **Analytical** Processing)とOLTP(Online **Transactional** Processing)の2つのグループに分類できます。前者は大量の履歴データに基づくレポート作成に重点を置いていますが、実行頻度は低くなります。後者は通常、継続的なトランザクションストリームを処理し、データの現在の状態を常に更新します。

実際には、OLAPとOLTPは二項対立のカテゴリではなく、スペクトラムとして捉えられています。ほとんどの実システムは通常どちらか一方に重点を置いていますが、反対のタイプのワークロードも必要な場合には、何らかのソリューションや回避策を提供しています。この状況により、企業は統合された複数のストレージシステムを運用せざるを得なくなることがよくあります。これはそれほど大きな問題ではないかもしれませんが、システムが増えるとメンテナンスコストが増加するため、近年の傾向としては、単一のデータベース管理システムで両方のタイプのワークロードを同等に処理できるHTAP(**Hybrid Transactional/Analytical Processing**)へと向かっています。

DBMSが純粋なOLAPまたは純粋なOLTPとして始まったとしても、競争に対応するためにHTAPの方向へ進まざるを得ません。ClickHouseも例外ではありません。当初は[可能な限り高速なOLAPシステム](/concepts/why-clickhouse-is-so-fast)として設計されており、現在も完全なトランザクションサポートは備えていませんが、一貫性のある読み取り/書き込みや、データの更新/削除のためのミューテーションなどの機能が追加されています。

OLAPシステムとOLTPシステムの間の基本的なトレードオフは依然として存在します:

- 分析レポートを効率的に作成するには、カラムを個別に読み取れることが重要であるため、ほとんどのOLAPデータベースは[カラム型](https://clickhouse.com/engineering-resources/what-is-columnar-database)です;
- カラムを個別に保存すると、追加やインプレース変更などの行操作のコストが、カラム数に比例して増加します(システムが念のためにイベントのすべての詳細を収集しようとする場合、カラム数は膨大になる可能性があります)。そのため、ほとんどのOLTPシステムはデータを行単位で配置して保存します。
