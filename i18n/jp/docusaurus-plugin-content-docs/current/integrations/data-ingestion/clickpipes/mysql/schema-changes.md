MySQL 向け ClickPipes は、ソーステーブルのスキーマ変更を検出し、場合によっては変更を自動的に宛先テーブルへ伝播します。各 DDL 操作の扱いは以下のとおりです。

[//]: # "TODO このページを、リネーム、データ型変更、TRUNCATE、互換性のないスキーマ変更への対処方法に関する挙動の説明で拡張する。"

| スキーマ変更の種類                                                                  | 挙動                                      |
| ----------------------------------------------------------------------------------- | ----------------------------------------- |
| 新しい列の追加 (`ALTER TABLE ADD COLUMN ...`)                                       | 自動的に伝播されます。スキーマ変更後にレプリケートされるすべての行で、新しい列に値が設定されます                                                                           |
| デフォルト値付きの新しい列の追加 (`ALTER TABLE ADD COLUMN ... DEFAULT ...`)        | 自動的に伝播されます。スキーマ変更後にレプリケートされるすべての行で、新しい列に値が設定されますが、既存の行では、テーブル全体をリフレッシュしないかぎりデフォルト値は反映されません |
| 既存列の削除 (`ALTER TABLE DROP COLUMN ...`)                                        | 検出はされますが、**伝播はされません**。スキーマ変更後にレプリケートされるすべての行では、削除された列には `NULL` が設定されます                                                                |
