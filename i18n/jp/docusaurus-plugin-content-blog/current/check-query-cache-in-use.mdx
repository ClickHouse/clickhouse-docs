## クエリでクエリキャッシュが使用されているかを確認するにはどうすればよいですか？ \{#how-can-i-check-that-query-cache-is-being-used-in-my-query\}

[clickhouse client](https://clickhouse.com/docs/interfaces/cli) と ClickHouse Cloud サービスを使用した次の例を参照します。

`query_cache_test` テーブルを作成します

## clickhouse client の使い方

```sql
clickhouse-cloud :) CREATE TABLE query_cache_test (name String, age UInt8) ENGINE =MergeTree ORDER BY name
```

では、`QueryCache` に関連する `TRACE` ログが次のケースでどのように異なるかを確認します。

1 回目の実行：

```
[c-red-qc-36-server-0] 2023.05.29 12:06:10.543559 [ 454 ] {3754a7fd-b786-47c1-a258-dfbc75e35a04} \<Trace\> QueryCache: クエリのエントリが見つかりませんでした SELECT name FROM query_cache_test  WHERE age > 1000 FORMAT `Null` SETTINGS
[c-red-qc-36-server-0] 2023.05.29 12:06:10.547760 [ 454 ] {3754a7fd-b786-47c1-a258-dfbc75e35a04} \<Trace\> QueryCache: クエリの結果を保存しました SELECT name FROM query_cache_test  WHERE age > 1000 FORMAT `Null` SETTINGS
```

2回目の実行時：

```
[c-red-qc-36-server-0] 2023.05.29 12:06:17.932043 [ 454 ] {a047527c-9d55-4e6b-9747-0ccad8787515} \<Trace\> QueryCache: クエリ SELECT name FROM query_cache_test  WHERE age > 1000 FORMAT `Null` SETTINGS に対するエントリが見つかりました
```

1回目の実行では、エントリが見つからなかったため（`No entry found for query SELECT...`）、ClickHouse がそのクエリ結果のエントリを保存しました（`Stored result of query SELECT...`）。

2回目の実行では、既に保存されていたエントリが見つかったため（`Entry found for query SELECT...`）、クエリはクエリキャッシュを利用しました。


## SQL だけを使用する

`clickhouse client` のトレースログを確認せず、SQL コマンドを発行するだけでも、

関連する `system` テーブルを確認することで、クエリキャッシュが使用されているかどうかを検証できます。

```sql
clickhouse-cloud :) SELECT 1 SETTINGS use_query_cache=true;

SELECT 1
SETTINGS use_query_cache = 1

Query id: a5a078c7-61e5-4036-a6f0-4d602d5b72d2

┌─1─┐
│ 1 │
└───┘

1行のデータセット。経過時間: 0.001秒。

clickhouse-cloud :) SELECT 1 SETTINGS use_query_cache=true;

SELECT 1
SETTINGS use_query_cache = 1

Query id: 322ae001-b1ab-463f-ac8d-dc5ba346f3f9

┌─1─┐
│ 1 │
└───┘

1行のデータセット。経過時間: 0.001秒。

clickhouse-cloud :) SELECT * FROM clusterAllReplicas(default,system.query_cache);

SELECT *
FROM clusterAllReplicas(default, system.query_cache)

Query id: c9b57eac-ba64-430e-8d51-8f865a13cc25

┌─query──────────────┬─result_size─┬─stale─┬─shared─┬─compressed─┬──────────expires_at─┬─────────────key_hash─┐
│ SELECT 1 SETTINGS  │         136 │     0 │      1 │          1 │ 2023-08-02 15:08:23 │ 12188185624808016954 │
└────────────────────┴─────────────┴───────┴────────┴────────────┴─────────────────────┴──────────────────────┘

1行のデータセット。経過時間: 0.005秒。

clickhouse-cloud :) SELECT * FROM clusterAllReplicas(default,system.events) WHERE event LIKE 'QueryCache%'

SELECT *
FROM clusterAllReplicas(default, system.events)
WHERE event LIKE 'QueryCache%'

Query id: d536555e-b8ab-4cd4-9741-c04e95612bec

┌─event────────────┬─value─┬─description────────────────────────────────────────────────────────────────────────────────────────────┐
│ QueryCacheHits   │     1 │ クエリキャッシュ内でクエリ結果が見つかった回数（クエリ計算が回避されました）。  │
│ QueryCacheMisses │     1 │ クエリキャッシュ内でクエリ結果が見つからなかった回数（クエリ計算が必要でした）。 │
└──────────────────┴───────┴────────────────────────────────────────────────────────────────────────────────────────────────────────┘

2行のデータセット。経過時間: 0.006秒。
```

直近の結果では、クエリ `SELECT 1 SETTINGS use_query_cache=true;` が初めて実行された際に `QueryCacheMisses` が 1 回発生し、2 回目の実行に関連する `QueryCacheHits` イベントがあることが分かります。

また、デフォルトの最大キャッシュエントリーサイズは 1,048,576 バイト（= 1 MiB）であり、結果はデフォルトでは 60 秒間しかキャッシュに保存されないことにも注意してください（たとえば `SETTINGS` で `query_cache_ttl=300` を指定すると、クエリキャッシュの結果を 5 分間保存できます）。

ClickHouse Query Cache に関するより詳細な情報は[こちら](https://clickhouse.com/docs/operations/query-cache)で確認できます。
