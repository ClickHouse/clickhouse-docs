## 質問 \{#question\}

Docker イメージを使用して ClickHouse をセットアップし、Microsoft SQL Server に接続できるようにするにはどうすればよいですか？

## Answer \{#answer\}

***この例に関する注意事項***

* ClickHouse の Docker 用 Ubuntu イメージを使用
* FreeTDS ドライバーを使用
* MSSQL Server 2012R2 を使用
* この例では Windows ホスト名は `MARSDB2.marsnet2.local`、IP アドレスは `192.168.1.133`（自身のホスト名や IP に置き換えてください）
* MSSQL インスタンス名は `MARSDB2`
* MSSQL のログインおよびデータベース ユーザーは `sql_user`

### ODBC で ClickHouse を構成する \{#configuring-clickhouse-with-odbc\}

作業ディレクトリを作成します。

```sh
mkdir ch-odbc-mssql
cd ch-odbc-mssql
```

`odbc.ini` ファイルを作成します。

```sh
vim odbc.ini
```

DSN と IP の名前を更新するには、次のエントリを追加します。

```sh
[marsdb2_mssql]
Driver = FreeTDS
Server = 192.168.1.133
```

`odbcinst.ini` ファイルを作成します：

```sh
vim odbcinst.ini
```

次の項目を追加します（`trace` は任意ですが、デバッグに役立ちます）：

```sh
[ODBC]
Trace = Yes
TraceFile = /tmp/odbc.log

[FreeTDS]
Description = FreeTDS
Driver = /usr/lib/aarch64-linux-gnu/odbc/libtdsodbc.so
Setup = /usr/lib/x86_64-linux-gnu/odbc/libtdsS.so
UsageCount = 1
```

### イメージをダウンロードし、TDS および必要な ODBC ライブラリを追加する Dockerfile を設定する \{#configure-a-dockerfile-to-download-the-image-and-add-the-tds-and-required-odbc-libraries\}

Dockerfile を作成します。

```sh
vim Dockerfile
```

次の内容を Dockerfile に追加します：

```sh
FROM clickhouse/clickhouse-server:23.10

# ODBCドライバーをインストール

RUN apt-get update && apt-get install -y --no-install-recommends unixodbc \
    && apt-get install -y freetds-bin freetds-common freetds-dev libct4 libsybdb5 \
	&& apt-get install tdsodbc
```

新しい Docker イメージをビルドします：

```sh
docker build . -t marsnet/clickhouse-odbc:23.10
```

`docker-compose.yml` ファイルを作成します：

```sh
vim docker-compose.yml
```

YAML ファイルに次の内容を追加します。

```sh
version: '3.7'
services:
  clickhouse:
    image: marsnet/clickhouse-odbc:23.10
    container_name: clickhouse-odbc
    hostname: clickhouse-host
    ports:
      - "9000:9000"
      - "8123:8123"
      - "9009:9009"
    volumes:
      - ./odbc.ini:/etc/odbc.ini
      - ./odbcinst.ini:/etc/odbcinst.ini
    restart: always
    ulimits:
      memlock:
        soft: -1
        hard: -1
      nofile:
        soft: 262144
        hard: 262144
    deploy:
      resources:
        limits:
          memory: 4g
```

コンテナを起動します。

```sh
docker compose up --detach
```

コンテナを起動すると、次のような表示になるはずです：

```sh
ch-odbc-mssql % docker compose up --detach
[+] Running 1/1
 ✔ Container clickhouse-odbc  Started
```

コンテナが実行中であることを確認します：

```sh
ch-odbc-mssql % docker ps
CONTAINER ID   IMAGE                           COMMAND            CREATED          STATUS              PORTS                                                                    NAMES
87a400b803ce   marsnet/clickhouse-odbc:23.10   "/entrypoint.sh"   57 minutes ago   Up About a minute   0.0.0.0:8123->8123/tcp, 0.0.0.0:9000->9000/tcp, 0.0.0.0:9009->9009/tcp   clickhouse-odbc
```

### ODBC 接続をテストする \{#test-odbc-connection\}

ClickHouse クライアントを使用してログインします:

```sh
./clickhouse client
```

`odbc` テーブル関数を使用して、リモートの MSSQL データベース テーブルに対する `SELECT` をテストします。

```
clickhouse-host :) SELECT * from odbc('DSN=marsdb2_mssql;port=1433;Uid=sql_user;Pwd=ClickHouse123;Database=db1', 'table1');

SELECT *
FROM odbc('DSN=marsdb2_mssql;port=1433;Uid=sql_user;Pwd=ClickHouse123;Database=db1', 'table1')

Query id: 23494da2-6e12-4ade-95fa-372a0420cac1

┌─id─┬─column1─┐
│  1 │ abc     │
│  2 │ def     │
│  3 │ ghi     │
└────┴─────────┘

3行が設定されています。経過時間: 0.188秒。 
```

`odbc` テーブルエンジンを使用してリモートテーブルを作成することも可能です。

```sql
CREATE TABLE table1_odbc_mssql
(
    `id` Int32,
    `column1` String
)
ENGINE = ODBC('DSN=marsdb2_mssql;port=1433;Uid=sql_user;Pwd=ClickHouse123;Database=db1', 'dbo', 'table1')
```

新しいリモートテーブルをテストするには、`SELECT` クエリを実行します。

```
clickhouse-host :) select * from table1_odbc_mssql;

SELECT *
FROM table1_odbc_mssql

Query id: 94724368-485d-4364-ae58-a435a225c37d

┌─id─┬─column1─┐
│  1 │ abc     │
│  2 │ def     │
│  3 │ ghi     │
└────┴─────────┘

3 rows in set. Elapsed: 0.218 sec. 
```
