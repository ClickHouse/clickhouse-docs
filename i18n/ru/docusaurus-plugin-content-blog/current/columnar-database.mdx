## Введение \{#introduction\}

Колоночная база данных хранит данные каждого столбца независимо. Это позволяет читать с диска только данные тех столбцов, которые используются в конкретном запросе. Обратная сторона состоит в том, что операции, затрагивающие целые строки, становятся пропорционально более затратными. Колоночная база данных является синонимом системы управления базами данных, ориентированной на столбцы. ClickHouse — типичный пример такой системы.

## Преимущества колоночной базы данных \{#advantages-of-a-columnar-database\}

К ключевым преимуществам колоночных баз данных относятся:

- Запросы, использующие лишь несколько столбцов из большого числа.
- Агрегирующие запросы над большими объемами данных.
- Сжатие данных по столбцам.

Колоночная база данных является предпочтительным выбором для аналитических приложений, потому что она позволяет иметь множество столбцов в таблице с запасом, но при этом не несёт затрат на неиспользуемые столбцы во время выполнения запросов на чтение (традиционная OLTP-база данных считывает все данные при выполнении запросов, поскольку данные хранятся по строкам, а не по столбцам). Колоночные базы данных спроектированы для обработки больших объёмов данных и построения хранилищ данных. Они изначально рассчитаны на масштабирование с использованием распределённых кластеров недорогого оборудования для увеличения пропускной способности. ClickHouse реализует это с помощью сочетания [распределённых](https://clickhouse.com/docs/engines/table-engines/special/distributed) и [реплицируемых](https://clickhouse.com/docs/engines/table-engines/mergetree-family/replication) таблиц.
