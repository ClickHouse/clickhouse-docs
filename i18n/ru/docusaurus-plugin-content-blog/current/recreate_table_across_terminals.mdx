## Вопрос \{#question\}

Как быстро воссоздать таблицу и её данные, используя только копирование и вставку в разных терминалах?

## Ответ \{#answer\}

Это НЕ является рекомендуемым способом переноса данных из одной базы данных в другую и НЕ должно использоваться для миграции данных в продакшн-среде.

Это лишь простой и быстрый способ воссоздать небольшой объём данных при разработке в нескольких окружениях.

1. Получите оператор CREATE TABLE с помощью команды `SHOW CREATE table`:

```sql
SHOW CREATE TABLE cookies;

SHOW CREATE TABLE cookies

Query id: 248ec8e2-5bce-45b3-97d9-ed68edf445a5

┌─statement────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────┐
│ CREATE TABLE default.cookies
(
    `id` String,
    `timestamp` DateTime
)
ENGINE = MergeTree
ORDER BY id
│
└──────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────┘

```

2. Экспортируйте данные в формате `SQLInsert`

```sql
SELECT * FROM cookies FORMAT SQLInsert;

SELECT *
FROM cookies
FORMAT SQLInsert

Query id: 383759b8-69c0-4561-ab95-f8224abc0071

INSERT INTO table (`id`, `timestamp`) VALUES ('4', '2023-03-15 16:28:46')
, ('2', '2023-03-15 16:28:41')
, ('1', '2023-03-15 16:11:02'), ('1', '2023-03-15 16:11:40'), ('1', '2023-03-15 16:11:48'), ('1', '2023-03-15 16:16:05'), ('2', '2023-03-15 16:11:06'), ('3', '2023-03-15 16:11:12'), ('3', '2023-03-15 16:11:45'), ('3', '2023-03-15 16:16:08'), ('4', '2023-03-15 16:11:14'), ('4', '2023-03-15 16:11:50'), ('4', '2023-03-15 16:16:01'), ('5', '2023-03-15 16:11:18'), ('5', '2023-03-15 16:16:11')
;

Получено 15 строк. Затрачено: 0,023 сек.
```

Обратите внимание, что на шаге 2 вам нужно заменить имя `table` на реальное имя таблицы (`cookies` в этом примере).
