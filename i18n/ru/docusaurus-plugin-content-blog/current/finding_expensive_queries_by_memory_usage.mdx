## Использование таблицы `system.query_log` \{#using-the-systemquery_log-table\}

Следующий полезный запрос показывает, какие из выполненных вами запросов использовали наибольшее количество памяти.

Пара замечаний к этому запросу:

* результаты вычисляются за последний день (`now() - toIntervalDay(1))`), но вы можете легко изменить временной интервал
* предполагается, что у вас есть кластер с именем `default`, которое является именем вашего кластера в [ClickHouse Cloud](https://console.clickhouse.cloud). Замените `default` на имя вашего кластера
* если у вас нет кластера, см. запрос в конце этой статьи

```sql
SELECT
    count() as nb_query,
    user,
    query,
    sum(memory_usage) AS memory,
    normalized_query_hash
FROM
    clusterAllReplicas(default, system.query_log)
WHERE
    (event_time >= (now() - toIntervalDay(1)))
    AND query_kind = 'Select'
    AND type = 'QueryFinish'
    and user != 'monitoring-internal'
GROUP BY
    normalized_query_hash,
    query,
    user
ORDER BY
    memory DESC;
```

Ответ будет выглядеть так:

```response
┌─nb_query─┬─user────┬─query─────────────────────────────────────────────────────────┬───memory─┬─normalized_query_hash─┐
│       11 │ default │ select version()                                              │ 46178924 │   7202516440347714159 │
│        2 │ default │ SELECT * FROM "system"."table_functions" LIMIT 31 OFFSET 0    │  8391544 │  12830067173062987695 │
└──────────┴─────────┴───────────────────────────────────────────────────────────────┴──────────┴───────────────────────┘
```

:::note
Если у вас нет таблицы `system.query_log`, скорее всего, у вас не включено логирование запросов. Ознакомьтесь с подробным описанием [настройки `query_log`](/operations/server-configuration-parameters/settings#query_log), чтобы узнать, как её включить.
:::

:::note[`query_log` содержит не только отправленные вами запросы]
Один отправленный запрос может отображаться в виде нескольких строк: исходный запрос (`is_initial_query = 1`) и производные или внутренние шаги (`is_initial_query = 0`), например вторичные запросы для распределённого выполнения или внутренние запросы, используемые для вычисления представлений. У этих внутренних строк собственные значения `query_id`, назначенные ClickHouse, и иногда они включают метку вроде `queryView...`, — поэтому не стоит считать, что каждый `query_id` совпадает с тем, который вернул ваш клиент. Чтобы отнести использование к запросу именно в том виде, в котором вы его запускали, фильтруйте по `is_initial_query = 1` или сопоставляйте `query_id` с `initial_query_id`. Подробности см. в разделе [Как определить самые дорогие запросы](/knowledgebase/find-expensive-queries#initial_query_id-vs-query_id) и в [справочнике по `query_log`](/operations/system-tables/query_log).
:::

Если у вас нет кластера, вы можете просто выполнять запросы напрямую к таблице `system.query_log`:

```sql
SELECT
    count() as nb_query,
    user,
    query,
    sum(memory_usage) AS memory,
    normalized_query_hash
FROM
    system.query_log
WHERE
    (event_time >= (now() - toIntervalDay(1)))
    AND query_kind = 'Select'
    AND type = 'QueryFinish'
    and user != 'monitoring-internal'
GROUP BY
    normalized_query_hash,
    query,
    user
ORDER BY
    memory DESC;
```
