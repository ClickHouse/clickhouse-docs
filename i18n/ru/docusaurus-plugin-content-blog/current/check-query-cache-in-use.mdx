## Как проверить, что в вашем запросе используется кэш запросов? \{#how-can-i-check-that-query-cache-is-being-used-in-my-query\}

Рассмотрим этот пример с использованием [clickhouse client](https://clickhouse.com/docs/interfaces/cli) и сервиса ClickHouse Cloud.

Создайте таблицу `query_cache_test`

## Использование клиента ClickHouse \{#using-clickhouse-client\}

(

## Использование только SQL \{#using-just-sql\}

Если просто выполнять SQL-команды, не анализируя трассировочные логи `clickhouse client`, можно проверить, используется ли кэш запросов, обратившись к соответствующим таблицам в базе `system`:

```sql
clickhouse-cloud :) SELECT 1 SETTINGS use_query_cache=true;

SELECT 1
SETTINGS use_query_cache = 1

Query id: a5a078c7-61e5-4036-a6f0-4d602d5b72d2

┌─1─┐
│ 1 │
└───┘

Получена 1 строка. Прошло: 0.001 сек.

clickhouse-cloud :) SELECT 1 SETTINGS use_query_cache=true;

SELECT 1
SETTINGS use_query_cache = 1

Query id: 322ae001-b1ab-463f-ac8d-dc5ba346f3f9

┌─1─┐
│ 1 │
└───┘

Получена 1 строка. Прошло: 0.001 сек.

clickhouse-cloud :) SELECT * FROM clusterAllReplicas(default,system.query_cache);

SELECT *
FROM clusterAllReplicas(default, system.query_cache)

Query id: c9b57eac-ba64-430e-8d51-8f865a13cc25

┌─query──────────────┬─result_size─┬─stale─┬─shared─┬─compressed─┬──────────expires_at─┬─────────────key_hash─┐
│ SELECT 1 SETTINGS  │         136 │     0 │      1 │          1 │ 2023-08-02 15:08:23 │ 12188185624808016954 │
└────────────────────┴─────────────┴───────┴────────┴────────────┴─────────────────────┴──────────────────────┘

Получена 1 строка. Прошло: 0.005 сек.

clickhouse-cloud :) SELECT * FROM clusterAllReplicas(default,system.events) WHERE event LIKE 'QueryCache%'

SELECT *
FROM clusterAllReplicas(default, system.events)
WHERE event LIKE 'QueryCache%'

Query id: d536555e-b8ab-4cd4-9741-c04e95612bec

┌─event────────────┬─value─┬─description────────────────────────────────────────────────────────────────────────────────────────────┐
│ QueryCacheHits   │     1 │ Количество случаев, когда результат запроса был найден в кэше запросов (вычисление запроса не потребовалось).  │
│ QueryCacheMisses │     1 │ Количество случаев, когда результат запроса не был найден в кэше запросов (потребовалось вычисление запроса). │
└──────────────────┴───────┴────────────────────────────────────────────────────────────────────────────────────────────────────────┘

Получено 2 строки. Прошло: 0.006 сек.
```

В последних результатах мы видим одно событие `QueryCacheMisses` для первого выполнения запроса `SELECT 1 SETTINGS use_query_cache=true;` и событие `QueryCacheHits`, связанное со вторым выполнением этого запроса.

Также имейте в виду, что максимальный размер одной записи в кэше по умолчанию составляет 1048576 байт (= 1 MiB), и по умолчанию результаты хранятся в кэше только 60 секунд (вы можете, например, использовать `query_cache_ttl=300` в `SETTINGS`, чтобы результат кэширования запроса хранился 5 минут).

Более подробную информацию о ClickHouse Query Cache можно найти [здесь](https://clickhouse.com/docs/operations/query-cache)
