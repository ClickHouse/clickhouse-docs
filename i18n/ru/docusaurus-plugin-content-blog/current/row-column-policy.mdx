## ClickHouse и ClickHouse Cloud поддерживают ограничения доступа на уровне строк и столбцов \{#clickhouse-and-clickhouse-cloud-both-support-row-and-column-level-access-restrictions\}

И ClickHouse, и ClickHouse Cloud поддерживают ограничения доступа на уровне строк и столбцов для пользователей с доступом только на чтение, необходимые для реализации модели управления доступом на основе ролей (RBAC).

[Политики строк](/operations/access-rights#row-policy-management) можно использовать для указания, какие строки будут возвращены пользователю с доступом только на чтение при выполнении запроса к таблице. ClickHouse Cloud по умолчанию настроен на использование SQL-управляемого рабочего процесса. Чтобы задействовать этот рабочий процесс, [CREATE](/sql-reference/statements/create/user) пользователя, [GRANT](/sql-reference/statements/grant) пользователю привилегии на таблицу, затем задайте соответствующую [ROW POLICY](/operations/access-rights#row-policy-management). Когда пользователь выполняет `SELECT * FROM table`, будут отображены только те строки, которые разрешены политикой.

Ограничения на уровне столбцов могут быть заданы напрямую с помощью оператора [GRANT](/sql-reference/statements/grant), чтобы предоставить доступ к таблице для пользователей и ролей. Пользователи *могут включать в запрос только* те столбцы, к которым у них есть доступ. Попытка выбрать ограниченные столбцы из таблицы, в которой у пользователя нет доступа ко всем указанным столбцам, например в запросе `SELECT * FROM table`, приведёт к ошибке о недостаточных правах доступа.
