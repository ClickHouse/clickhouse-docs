## Подробное логирование \{#verbose-logging\}

Одна из вещей, которая часто сбивает с толку новых пользователей, — это то, что ClickHouse генерирует большой объем логов даже при небольшой нагрузке. 

Это связано с тем, что уровень логирования по умолчанию по историческим причинам — `trace` (вместо `warning`, который был бы уровнем по умолчанию в других базах данных). 

Разработчики ClickHouse считают, что `trace` дает много полезной информации в случае возникновения проблем. 

С другой стороны, большие объемы логов означают, что системная таблица `system.text_log` быстро заполняется, и ее данные приходится часто сливать в фоновом режиме. 

Если база данных работает стабильно, пользователи могут перенастроить уровень логирования, как описано ниже.

## Изменить уровень логирования

Доступные уровни логирования описаны [здесь](https://clickhouse.com/docs/operations/server-configuration-parameters/settings#logger)

Чтобы изменить уровень логирования, отредактируйте файл конфигурации сервера ClickHouse (`/etc/clickhouse-server/config.xml`).

Значение по умолчанию — `trace`, но вы можете изменить его на нужный уровень. См. комментарий ниже:

```
<clickhouse>
    <logger>
        <!-- Possible levels [1]:

          - none (turns off logging)
          - fatal
          - critical
          - error
          - warning
          - notice
          - information
          - debug
          - trace
          - test (not for production usage)

            [1]: https://github.com/pocoproject/poco/blob/poco-1.9.4-release/Foundation/include/Poco/Logger.h#L105-L114
        -->
        <level>trace</level>
... Остальная часть файла конфигурации
</clickhouse>
```
