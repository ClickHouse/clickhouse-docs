## Описание \{#description\}

Эта таблица содержит информацию о параметрах запроса к серверу ZooKeeper и его ответе.

Для запросов заполняются только столбцы с параметрами запроса, а остальные столбцы заполняются значениями по умолчанию (`0` или `NULL`). Когда приходит ответ, данные из ответа добавляются в другие столбцы.

## Столбцы \{#columns\}

{/*AUTOGENERATED_START*/ }

* `hostname` ([LowCardinality(String)](/sql-reference/data-types/lowcardinality)) — имя хоста сервера, выполняющего запрос.
* `type` ([Enum8(&#39;Request&#39; = 1, &#39;Response&#39; = 2, &#39;Finalize&#39; = 3)](/sql-reference/data-types/enum)) — Тип события клиента ZooKeeper. Может принимать одно из следующих значений: Request — запрос отправлен, Response — ответ получен, Finalize — соединение потеряно, ответ не получен.
* `event_date` ([Date](/sql-reference/data-types/date)) — дата, когда произошло событие.
* `event_time` ([DateTime64(6)](/sql-reference/data-types/datetime64)) — дата и время, когда произошло событие.
* `thread_id` ([UInt64](/sql-reference/data-types/int-uint)) — Идентификатор потока, выполнившего этот запрос.
* `query_id` ([String](/sql-reference/data-types/string)) — Идентификатор запроса, в рамках которого был выполнен данный запрос.
* `address` ([IPv6](/sql-reference/data-types/ipv6)) — IP-адрес сервера ZooKeeper, использованный для выполнения запроса.
* `port` ([UInt16](/sql-reference/data-types/int-uint)) — Порт сервера ZooKeeper, использованного для выполнения запроса.
* `session_id` ([Int64](/sql-reference/data-types/int-uint)) — Идентификатор сеанса, который сервер ZooKeeper присваивает каждому соединению.
* `duration_microseconds` ([UInt64](/sql-reference/data-types/int-uint)) — Время, которое ZooKeeper затратил на выполнение запроса.
* `xid` ([Int64](/sql-reference/data-types/int-uint)) — Идентификатор запроса в рамках сеанса. Обычно это последовательный номер запроса. Он совпадает у строки запроса и соответствующей строки ответа/завершения.
* `has_watch` ([UInt8](/sql-reference/data-types/int-uint)) — Признак того, установлено ли наблюдение.
* `op_num` ([Enum16(&#39;Close&#39; = -11, &#39;Error&#39; = -1, &#39;Watch&#39; = 0, &#39;Create&#39; = 1, &#39;Remove&#39; = 2, &#39;Exists&#39; = 3, &#39;Get&#39; = 4, &#39;Set&#39; = 5, &#39;GetACL&#39; = 6, &#39;SetACL&#39; = 7, &#39;SimpleList&#39; = 8, &#39;Sync&#39; = 9, &#39;Heartbeat&#39; = 11, &#39;List&#39; = 12, &#39;Check&#39; = 13, &#39;Multi&#39; = 14, &#39;Create2&#39; = 15, &#39;Reconfig&#39; = 16, &#39;CheckWatch&#39; = 17, &#39;RemoveWatch&#39; = 18, &#39;MultiRead&#39; = 22, &#39;Auth&#39; = 100, &#39;SetWatch&#39; = 101, &#39;SetWatch2&#39; = 105, &#39;AddWatch&#39; = 106, &#39;FilteredList&#39; = 500, &#39;CheckNotExists&#39; = 501, &#39;CreateIfNotExists&#39; = 502, &#39;RemoveRecursive&#39; = 503, &#39;CheckStat&#39; = 504, &#39;TryRemove&#39; = 505, &#39;FilteredListWithStatsAndData&#39; = 506, &#39;ListRecursive&#39; = 507, &#39;SessionID&#39; = 997)](/sql-reference/data-types/enum)) — Тип запроса либо ответа.
* `path` ([String](/sql-reference/data-types/string)) — Путь к узлу ZooKeeper, указанному в запросе, или пустая строка, если в запросе не требуется указывать путь.
* `data` ([String](/sql-reference/data-types/string)) — данные, записанные в узел ZooKeeper (для запросов SET и CREATE — то, что запрос должен был записать; для ответа на запрос GET — то, что было прочитано), либо пустая строка.
* `is_ephemeral` ([UInt8](/sql-reference/data-types/int-uint)) — Создаётся ли узел ZooKeeper как временный.
* `is_sequential` ([UInt8](/sql-reference/data-types/int-uint)) — Создаётся ли узел ZooKeeper в последовательном режиме.
* `version` ([Nullable(Int32)](/sql-reference/data-types/nullable)) — Версия узла ZooKeeper, которую запрос ожидает при выполнении. Поддерживается для запросов CHECK, SET, REMOVE (имеет значение -1, если запрос не проверяет версию, или NULL для других запросов, не поддерживающих проверку версии).
* `requests_size` ([UInt32](/sql-reference/data-types/int-uint)) — Количество запросов, входящих в состав мультизапроса (это специальный запрос, состоящий из нескольких последовательных обычных запросов, которые выполняются атомарно). Все запросы, входящие в состав мультизапроса, будут иметь одинаковый xid.
* `request_idx` ([UInt32](/sql-reference/data-types/int-uint)) — Номер запроса в составе мультизапроса (для мультизапроса — 0, затем по порядку начиная с 1).
* `zxid` ([Int64](/sql-reference/data-types/int-uint)) — идентификатор транзакции ZooKeeper. Это порядковый номер, который сервер ZooKeeper присваивает в ответ на успешно выполненный запрос (0, если запрос не был выполнен/завершился ошибкой/клиенту неизвестно, был ли запрос выполнен).
* `error` ([Nullable(Enum8(&#39;ZNOWATCHER&#39; = -121, &#39;ZNOTREADONLY&#39; = -119, &#39;ZSESSIONMOVED&#39; = -118, &#39;ZNOTHING&#39; = -117, &#39;ZCLOSING&#39; = -116, &#39;ZAUTHFAILED&#39; = -115, &#39;ZINVALIDACL&#39; = -114, &#39;ZINVALIDCALLBACK&#39; = -113, &#39;ZSESSIONEXPIRED&#39; = -112, &#39;ZNOTEMPTY&#39; = -111, &#39;ZNODEEXISTS&#39; = -110, &#39;ZNOCHILDRENFOREPHEMERALS&#39; = -108, &#39;ZBADVERSION&#39; = -103, &#39;ZNOAUTH&#39; = -102, &#39;ZNONODE&#39; = -101, &#39;ZAPIERROR&#39; = -100, &#39;ZOUTOFMEMORY&#39; = -10, &#39;ZINVALIDSTATE&#39; = -9, &#39;ZBADARGUMENTS&#39; = -8, &#39;ZOPERATIONTIMEOUT&#39; = -7, &#39;ZUNIMPLEMENTED&#39; = -6, &#39;ZMARSHALLINGERROR&#39; = -5, &#39;ZCONNECTIONLOSS&#39; = -4, &#39;ZDATAINCONSISTENCY&#39; = -3, &#39;ZRUNTIMEINCONSISTENCY&#39; = -2, &#39;ZSYSTEMERROR&#39; = -1, &#39;ZOK&#39; = 0))](/sql-reference/data-types/nullable)) — Код ошибки. Может принимать множество значений, здесь приведены лишь некоторые из них: ZOK — запрос успешно выполнен, ZCONNECTIONLOSS — соединение потеряно, ZOPERATIONTIMEOUT — истек таймаут выполнения запроса, ZSESSIONEXPIRED — срок действия сеанса истек, NULL — запрос завершен.
* `watch_type` ([Nullable(Enum8(&#39;NOTWATCHING&#39; = -2, &#39;SESSION&#39; = -1, &#39;CREATED&#39; = 1, &#39;DELETED&#39; = 2, &#39;CHANGED&#39; = 3, &#39;CHILD&#39; = 4))](/sql-reference/data-types/nullable)) — Тип события наблюдения (для ответов с op&#95;num = Watch), для остальных ответов: NULL.
* `watch_state` ([Nullable(Enum16(&#39;AUTH&#95;FAILED&#39; = -113, &#39;EXPIRED&#95;SESSION&#39; = -112, &#39;CONNECTING&#39; = 1, &#39;ASSOCIATING&#39; = 2, &#39;CONNECTED&#39; = 3, &#39;READONLY&#39; = 5, &#39;NOTCONNECTED&#39; = 999))](/sql-reference/data-types/nullable)) — Статус события наблюдения (для ответов с op&#95;num = Watch), для остальных ответов — NULL.
* `path_created` ([String](/sql-reference/data-types/string)) — путь к созданному узлу ZooKeeper (для ответов на запрос CREATE); он может отличаться от указанного пути, если узел создаётся как последовательный.
* `stat_czxid` ([Int64](/sql-reference/data-types/int-uint)) — zxid изменения, в результате которого был создан этот узел ZooKeeper.
* `stat_mzxid` ([Int64](/sql-reference/data-types/int-uint)) — zxid изменения, которое последним изменило этот узел ZooKeeper.
* `stat_pzxid` ([Int64](/sql-reference/data-types/int-uint)) — идентификатор транзакции, в результате которой в последний раз были изменены дочерние элементы этого узла ZooKeeper.
* `stat_version` ([Int32](/sql-reference/data-types/int-uint)) — Количество изменений в данных этого узла ZooKeeper.
* `stat_cversion` ([Int32](/sql-reference/data-types/int-uint)) — Количество изменений в составе дочерних узлов этого узла ZooKeeper.
* `stat_dataLength` ([Int32](/sql-reference/data-types/int-uint)) — Длина поля данных данного узла ZooKeeper.
* `stat_numChildren` ([Int32](/sql-reference/data-types/int-uint)) — Количество дочерних узлов данного узла ZooKeeper.
* `children` ([Array(String)](/sql-reference/data-types/array)) — Список дочерних узлов ZooKeeper (в ответах на запрос LIST).

{/*AUTOGENERATED_END*/ }
