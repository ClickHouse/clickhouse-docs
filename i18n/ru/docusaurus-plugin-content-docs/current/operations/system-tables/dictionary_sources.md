## Описание \{#description\}

Содержит список поддерживаемых сервером источников словарей, а также встроенную документацию по каждому из них. Источник словаря определяет, откуда загружаются данные словаря; он указывается в предложении `SOURCE` запроса `CREATE DICTIONARY`.

## Столбцы \{#columns\}

{/*AUTOGENERATED_START*/ }

* `name` ([String](../../sql-reference/data-types/)) — Имя источника словаря, указанное в предложении SOURCE.
* `description` ([String](../../sql-reference/data-types/)) — Общее описание назначения источника словаря.
* `syntax` ([String](../../sql-reference/data-types/)) — Структура предложения SOURCE, используемого для указания источника. Обратите внимание, что на некоторые источники распространяется управление доступом, если словарь создается из DDL-запроса (а не из файла конфигурации сервера); подробности см. в `description` соответствующего источника.
* `examples` ([String](../../sql-reference/data-types/)) — Примеры использования.
* `introduced_in` ([String](../../sql-reference/data-types/)) — Версия ClickHouse, в которой источник впервые появился, в формате major.minor.
* `related` ([Array(String)](../../sql-reference/data-types/)) — Имена связанных источников словаря.

{/*AUTOGENERATED_END*/ }

## Пример \{#example\}

```sql title="Query"
SELECT name, syntax
FROM system.dictionary_sources
WHERE name IN ('clickhouse', 'file')
ORDER BY name
```

```text title="Response"
┌─name───────┬─syntax─────────────────────────────────────────────────────────────────────────┐
│ clickhouse │ SOURCE(CLICKHOUSE(host 'host' port 9000 user 'default' password '' db 'db' table 'table')) │
│ file       │ SOURCE(FILE(path '/path/to/file' format 'CSV'))                                  │
└────────────┴────────────────────────────────────────────────────────────────────────────────┘
```
