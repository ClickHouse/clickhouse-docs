## Описание \{#description\}

Содержит информацию обо всех записях в файловом кэше для удалённых объектов.

## Столбцы \{#columns\}

{/*AUTOGENERATED_START*/ }

* `cache_name` ([String](../../sql-reference/data-types/)) — Имя объекта кэша. Можно использовать в командах `SYSTEM DESCRIBE FILESYSTEM CACHE <name>`, `SYSTEM DROP FILESYSTEM CACHE <name>`
* `cache_base_path` ([String](../../sql-reference/data-types/)) — Путь к базовому каталогу, где хранятся все файлы кэша (кэша, идентифицируемого по `cache_name`).
* `cache_path` ([String](../../sql-reference/data-types/)) — Путь к конкретному файлу кэша, соответствующему сегменту файла в исходном файле
* `key` ([String](../../sql-reference/data-types/)) — Ключ кэша сегмента файла
* `file_segment_range_begin` ([UInt64](../../sql-reference/data-types/)) — Смещение, соответствующее началу диапазона сегмента файла
* `file_segment_range_end` ([UInt64](../../sql-reference/data-types/)) — Смещение, соответствующее концу диапазона сегмента файла (включительно)
* `size` ([UInt64](../../sql-reference/data-types/)) — Размер сегмента файла
* `state` ([String](../../sql-reference/data-types/)) — Состояние сегмента файла (DOWNLOADED, DOWNLOADING, PARTIALLY&#95;DOWNLOADED, ...)
* `finished_download_time` ([DateTime](../../sql-reference/data-types/)) — Время, когда завершилась загрузка сегмента файла.
* `cache_hits` ([UInt64](../../sql-reference/data-types/)) — Количество попаданий в кэш для соответствующего сегмента файла
* `references` ([UInt64](../../sql-reference/data-types/)) — Количество ссылок на соответствующий сегмент файла. Значение 1 означает, что в данный момент его никто не использует (единственная существующая ссылка находится в самом хранилище кэша)
* `downloaded_size` ([UInt64](../../sql-reference/data-types/)) — Размер загруженной части сегмента файла
* `kind` ([String](../../sql-reference/data-types/)) — Вид сегмента файла (используется для различения сегментов файла, добавленных как часть &#39;Temporary data in cache&#39;)
* `unbound` ([UInt8](../../sql-reference/data-types/)) — Внутренний служебный флаг
* `user_id` ([String](../../sql-reference/data-types/)) — Идентификатор пользователя, создавшего сегмент файла
* `segment_type` ([String](../../sql-reference/data-types/)) — Тип сегмента. Используется для разделения файлов данных(`.json`, `.txt` и т. д.) и служебных файлов данных(`.bin`, mark-файлы).
* `file_size` ([Nullable(UInt64)](../../sql-reference/data-types/)) — Размер файла, которому принадлежит текущий сегмент файла
