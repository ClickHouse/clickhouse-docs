## Описание \{#description\}

Содержит информацию о [квотах](../../operations/system-tables/quotas.md).

## Столбцы \{#columns\}

{/*AUTOGENERATED_START*/ }

* `name` ([String](../../sql-reference/data-types/)) — Имя квоты.
* `id` ([UUID](../../sql-reference/data-types/)) — Идентификатор квоты.
* `storage` ([String](../../sql-reference/data-types/)) — Хранилище квот. Возможные значения: &#39;users.xml&#39;, если квота настроена в файле users.xml, и &#39;disk&#39;, если квота настроена SQL-запросом.
* `keys` ([Array(Enum8(&#39;user&#95;name&#39; = 1, &#39;ip&#95;address&#39; = 2, &#39;forwarded&#95;ip&#95;address&#39; = 3, &#39;client&#95;key&#39; = 4, &#39;normalized&#95;query&#95;hash&#39; = 7))](../../sql-reference/data-types/)) — Ключ определяет, как должна разделяться квота. Если два соединения используют одну и ту же квоту и ключ, они совместно используют один и тот же объём ресурсов. Значения: [] — все пользователи используют одну и ту же квоту, [&#39;user&#95;name&#39;] — соединения с одинаковым именем пользователя используют одну и ту же квоту, [&#39;ip&#95;address&#39;] — соединения с одного и того же IP-адреса используют одну и ту же квоту. [&#39;client&#95;key&#39;] — соединения с одинаковым ключом используют одну и ту же квоту. Ключ должен быть явно передан клиентом. При использовании clickhouse-client передайте значение ключа в параметре --quota&#95;key или используйте параметр quota&#95;key в конфигурационном файле клиента. При использовании HTTP-интерфейса используйте заголовок X-ClickHouse-Quota, [&#39;user&#95;name&#39;, &#39;client&#95;key&#39;] — соединения с одинаковым client&#95;key используют одну и ту же квоту. Если ключ не передан клиентом, квота отслеживается по `user_name`, [&#39;client&#95;key&#39;, &#39;ip&#95;address&#39;] — соединения с одинаковым client&#95;key используют одну и ту же квоту. Если ключ не передан клиентом, квота отслеживается по ip&#95;address.
* `durations` ([Array(UInt32)](../../sql-reference/data-types/)) — Длительность временных интервалов в секундах.
* `apply_to_all` ([UInt8](../../sql-reference/data-types/)) — Логическое значение. Показывает, к каким пользователям применяется квота. Значения: 0 — квота применяется к пользователям, указанным в apply&#95;to&#95;list. 1 — квота применяется ко всем пользователям, кроме перечисленных в apply&#95;to&#95;except.
* `apply_to_list` ([Array(String)](../../sql-reference/data-types/)) — Список имён пользователей/ролей, к которым должна применяться квота.
* `apply_to_except` ([Array(String)](../../sql-reference/data-types/)) — Список имён пользователей/ролей, к которым квота не должна применяться.
