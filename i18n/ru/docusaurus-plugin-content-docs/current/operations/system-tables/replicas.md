## Описание \{#description\}

Содержит информацию и состояние реплицируемых таблиц, расположенных на локальном сервере.
Эту таблицу можно использовать для мониторинга. Таблица содержит по одной строке для каждой таблицы Replicated*.

## Столбцы \{#columns\}

{/*AUTOGENERATED_START*/ }

* `database` ([String](../../sql-reference/data-types/)) — Имя базы данных.
* `table` ([String](../../sql-reference/data-types/)) — имя таблицы.
* `uuid` ([UUID](../../sql-reference/data-types/)) — UUID-идентификатор таблицы.
* `engine` ([String](../../sql-reference/data-types/)) — Название движка таблицы.
* `is_leader` ([UInt8](../../sql-reference/data-types/)) — Является ли реплика лидером. Несколько реплик могут одновременно быть лидерами. Реплике можно запретить становиться лидером с помощью настройки merge&#95;tree replicated&#95;can&#95;become&#95;leader. Лидеры отвечают за планирование фоновых слияний. Обратите внимание, что записывать данные можно в любую доступную реплику, у которой есть сеанс в ZK, независимо от того, является ли она лидером.
* `can_become_leader` ([UInt8](../../sql-reference/data-types/)) — Может ли реплика стать лидером.
* `is_readonly` ([UInt8](../../sql-reference/data-types/)) — Находится ли реплика в режиме только для чтения. Этот режим включается, если в конфигурации отсутствуют разделы ClickHouse Keeper, если при повторной инициализации сеансов в ClickHouse Keeper произошла неизвестная ошибка, а также во время повторной инициализации сеансов в ClickHouse Keeper.
* `readonly_start_time` ([Nullable(DateTime)](../../sql-reference/data-types/)) — Таймстамп момента, когда реплика перешла в режим только для чтения. NULL, если реплика не находится в режиме только для чтения.
* `is_session_expired` ([UInt8](../../sql-reference/data-types/)) — Истёк ли срок действия сеанса ClickHouse Keeper. По сути, это то же самое, что и `is_readonly`.
* `future_parts` ([UInt32](../../sql-reference/data-types/)) — Количество частей данных, которые появятся в результате ещё не выполненных операций INSERT или слияний.
* `parts_to_check` ([UInt32](../../sql-reference/data-types/)) — Количество частей данных в очереди проверки. Часть помещается в очередь проверки, если есть подозрение, что она может быть повреждена.
* `zookeeper_name` ([String](../../sql-reference/data-types/)) — Имя кластера [Zoo]Keeper (возможно, вспомогательного), где хранятся метаданные таблицы
* `zookeeper_path` ([String](../../sql-reference/data-types/)) — Путь к данным таблицы в ClickHouse Keeper.
* `replica_name` ([String](../../sql-reference/data-types/)) — Имя реплики в ClickHouse Keeper. У разных реплик одной таблицы разные имена.
* `replica_path` ([String](../../sql-reference/data-types/)) — Путь к данным реплики в ClickHouse Keeper. То же, что объединение &#39;zookeeper&#95;path/replicas/replica&#95;path&#39;.
* `columns_version` ([Int32](../../sql-reference/data-types/)) — Номер версии структуры таблицы. Показывает, сколько раз был выполнен ALTER. Если у реплик разные версии, это означает, что некоторые из них ещё не выполнили все ALTER.
* `queue_size` ([UInt32](../../sql-reference/data-types/)) — Размер очереди операций, ожидающих выполнения. К таким операциям относятся вставка блоков данных, слияния и некоторые другие действия. Обычно совпадает с future&#95;parts.
* `inserts_in_queue` ([UInt32](../../sql-reference/data-types/)) — Количество вставок блоков данных, которые нужно выполнить. Обычно такие вставки реплицируются довольно быстро. Если это число велико, значит, что-то не так.
* `merges_in_queue` ([UInt32](../../sql-reference/data-types/)) — Количество слияний, ожидающих выполнения. Иногда слияния занимают много времени, поэтому это значение может оставаться больше нуля в течение длительного времени.
* `part_mutations_in_queue` ([UInt32](../../sql-reference/data-types/)) — Количество мутаций в очереди на выполнение.
* `queue_oldest_time` ([DateTime](../../sql-reference/data-types/)) — если `queue_size` больше 0, показывает, когда в очередь была добавлена самая старая операция.
* `inserts_oldest_time` ([DateTime](../../sql-reference/data-types/)) — см. `queue_oldest_time`.
* `merges_oldest_time` ([DateTime](../../sql-reference/data-types/)) — См. `queue_oldest_time`.
* `part_mutations_oldest_time` ([DateTime](../../sql-reference/data-types/)) — См. `queue_oldest_time`.
* `oldest_part_to_get` ([String](../../sql-reference/data-types/)) — Имя части, которую нужно получить с других реплик; берётся из самой старой записи GET&#95;PARTS в очереди репликации.
* `oldest_part_to_merge_to` ([String](../../sql-reference/data-types/)) — Имя результирующей части, в которую выполняется слияние, полученное из самой старой записи MERGE&#95;PARTS в очереди репликации.
* `oldest_part_to_mutate_to` ([String](../../sql-reference/data-types/)) — Имя результирующей части, до которой должна быть выполнена мутация, полученное из самой старой записи MUTATE&#95;PARTS в очереди репликации.
* `log_max_index` ([UInt64](../../sql-reference/data-types/)) — Максимальный номер записи в журнале общей активности. Этот столбец и следующие три (`log_pointer`, `total_replicas`, `active_replicas`) имеют ненулевое значение только при наличии активной сессии с ClickHouse Keeper.
* `log_pointer` ([UInt64](../../sql-reference/data-types/)) — Максимальный номер записи в журнале общей активности, которую реплика скопировала в свою очередь на выполнение, плюс один. Если log&#95;pointer значительно меньше, чем log&#95;max&#95;index, значит, что-то не так.
* `last_queue_update` ([DateTime](../../sql-reference/data-types/)) — Время последнего обновления очереди.
* `absolute_delay` ([UInt64](../../sql-reference/data-types/)) — Величина задержки текущей реплики в секундах.
* `total_replicas` ([UInt32](../../sql-reference/data-types/)) — Общее число известных реплик этой таблицы.
* `active_replicas` ([UInt32](../../sql-reference/data-types/)) — Количество реплик этой таблицы, у которых есть сеанс в ClickHouse Keeper (то есть количество работающих реплик).
* `lost_part_count` ([UInt64](../../sql-reference/data-types/)) — Суммарное количество частей, потерянных в таблице всеми репликами с момента создания таблицы. Значение сохраняется в ClickHouse Keeper и может только увеличиваться.
* `last_queue_update_exception` ([String](../../sql-reference/data-types/)) — Когда очередь содержит повреждённые записи. Это особенно важно, если ClickHouse нарушает обратную совместимость между версиями и записи журнала, созданные более новыми версиями, не могут быть разобраны старыми версиями.
* `zookeeper_exception` ([String](../../sql-reference/data-types/)) — Сообщение о последнем исключении, которое возвращается, если ошибка возникла при получении информации из ClickHouse Keeper.
* `replica_is_active` ([Map(String, UInt8)](../../sql-reference/data-types/)) — соответствие между именем реплики и признаком того, активна ли реплика.

{/*AUTOGENERATED_END*/ }

**Псевдонимы:**

* `readonly_duration` — псевдоним для `if(isNull(readonly_start_time), NULL, now() - readonly_start_time)`.
