## Описание \{#description\}

Содержит историю значений метрик из таблиц `system.metrics` и `system.events`, которые периодически сбрасываются на диск.

## Столбцы \{#columns\}

{/*AUTOGENERATED_START*/ }

* `hostname` ([LowCardinality(String)](../../sql-reference/data-types/string.md)) — Имя хоста сервера, на котором выполняется запрос.
* `event_date` ([Date](../../sql-reference/data-types/date.md)) — Дата события.
* `event_time` ([DateTime](../../sql-reference/data-types/datetime.md)) — Время события.
* `event_time_microseconds` ([DateTime64](../../sql-reference/data-types/datetime64.md)) — Время события с точностью до микросекунд.

{/*AUTOGENERATED_END*/ }

**Schema**
Эту таблицу можно настроить с разными типами schema с помощью XML-тега `<schema_type>`. Тип schema по умолчанию — `wide`; в нём каждая метрика или событие профиля хранится в отдельном столбце. Эта schema обеспечивает максимальную производительность и эффективность при чтении одного столбца.

Schema `transposed` хранит данные в формате, аналогичном `system.asynchronous_metric_log`, где метрики и события хранятся как строки. Эта schema полезна в средах с ограниченными ресурсами, поскольку снижает их потребление во время слияний.
