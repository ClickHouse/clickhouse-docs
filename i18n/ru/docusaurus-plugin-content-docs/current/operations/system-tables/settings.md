## Описание \{#description\}

Содержит информацию о настройках сеанса для текущего пользователя.

## Столбцы \{#columns\}

{/*AUTOGENERATED_START*/ }

* `name` ([String](../../sql-reference/data-types/)) — Имя настройки.
* `value` ([String](../../sql-reference/data-types/)) — Значение настройки.
* `changed` ([UInt8](../../sql-reference/data-types/)) — Показывает, была ли настройка явно задана в конфигурации или явно изменена.
* `description` ([String](../../sql-reference/data-types/)) — Краткое описание настройки.
* `min` ([Nullable(String)](../../sql-reference/data-types/)) — Минимальное значение настройки, если оно задано через ограничения. Если у настройки нет минимального значения, содержит NULL.
* `max` ([Nullable(String)](../../sql-reference/data-types/)) — Максимальное значение настройки, если оно задано через ограничения. Если у настройки нет максимального значения, содержит NULL.
* `disallowed_values` ([Array(String)](../../sql-reference/data-types/)) — Список недопустимых значений
* `readonly` ([UInt8](../../sql-reference/data-types/)) — Показывает, может ли текущий пользователь изменить настройку: 0 — текущий пользователь может изменить настройку, 1 — текущий пользователь не может изменить настройку.
* `type` ([String](../../sql-reference/data-types/)) — Тип значения, которое можно присвоить этой настройке.
* `default` ([String](../../sql-reference/data-types/)) — Значение настройки по умолчанию.
* `alias_for` ([String](../../sql-reference/data-types/)) — Имя исходной настройки, если эта настройка является псевдонимом другой настройки.
* `is_obsolete` ([UInt8](../../sql-reference/data-types/)) — Показывает, является ли настройка устаревшей.
* `tier` ([Enum8(&#39;Production&#39; = 0, &#39;Obsolete&#39; = 4, &#39;Experimental&#39; = 8, &#39;Beta&#39; = 12)](../../sql-reference/data-types/)) —
  Уровень поддержки этой функции. Возможности ClickHouse разделены по уровням в зависимости от текущего этапа их
  разработки и того, чего можно ожидать при их использовании:

- PRODUCTION: Функция стабильна, безопасна в использовании и не вызывает проблем при взаимодействии с другими функциями уровня PRODUCTION.
- BETA: Функция стабильна и безопасна. Результат ее использования вместе с другими функциями неизвестен, и корректность не гарантируется. Тестирование и отчеты приветствуются.
- EXPERIMENTAL: Функция находится в разработке. Предназначена только для разработчиков и энтузиастов ClickHouse. Функция может как работать, так и не работать, и может быть удалена в любой момент.
- OBSOLETE: Больше не поддерживается. Либо уже удалена, либо будет удалена в будущих выпусках.

{/*AUTOGENERATED_END*/ }

## Пример \{#example\}

В следующем примере показано, как получить информацию о настройках, имя которых содержит `min_i`.

```sql
SELECT *
FROM system.settings
WHERE name LIKE '%min_insert_block_size_%'
FORMAT Vertical
```

Использование `WHERE changed` может быть полезно, например, когда нужно проверить:

* Правильно ли загружены и применяются настройки из конфигурационных файлов.
* Настройки, изменённые в текущей сессии.

```sql
SELECT * FROM system.settings WHERE changed AND name='load_balancing'
```
