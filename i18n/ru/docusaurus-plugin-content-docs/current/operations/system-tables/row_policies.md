## Описание \{#description\}

Содержит фильтры для одной таблицы, а также список ролей и/или пользователей, для которых применяется эта ROW POLICY.

## Столбцы \{#columns\}

{/*AUTOGENERATED_START*/ }

* `name` ([String](../../sql-reference/data-types/)) — Имя ROW POLICY.
* `short_name` ([String](../../sql-reference/data-types/)) — Короткое имя ROW POLICY. Имена ROW POLICY являются составными, например: myfilter ON mydb.mytable. Здесь &#39;myfilter ON mydb.mytable&#39; — имя ROW POLICY, а &#39;myfilter&#39; — её короткое имя.
* `database` ([String](../../sql-reference/data-types/)) — Имя базы данных.
* `table` ([String](../../sql-reference/data-types/)) — Имя таблицы. Пусто, если ROW POLICY задана для базы данных.
* `id` ([UUID](../../sql-reference/data-types/)) — Идентификатор ROW POLICY.
* `storage` ([String](../../sql-reference/data-types/)) — Имя каталога, в котором хранится ROW POLICY.
* `select_filter` ([Nullable(String)](../../sql-reference/data-types/)) — Выражение, используемое для фильтрации в запросах SELECT.
* `is_restrictive` ([UInt8](../../sql-reference/data-types/)) — Показывает, ограничивает ли ROW POLICY доступ к строкам. Значение: • 0 — ROW POLICY определена с предложением `AS PERMISSIVE`, • 1 — ROW POLICY определена с предложением `AS RESTRICTIVE`.
* `apply_to_all` ([UInt8](../../sql-reference/data-types/)) — Показывает, что ROW POLICY заданы для всех ролей и/или пользователей.
* `apply_to_list` ([Array(String)](../../sql-reference/data-types/)) — Список ролей и/или пользователей, к которым применяются ROW POLICY.
* `apply_to_except` ([Array(String)](../../sql-reference/data-types/)) — ROW POLICY применяются ко всем ролям и/или пользователям, кроме перечисленных.

{/*AUTOGENERATED_END*/ }
