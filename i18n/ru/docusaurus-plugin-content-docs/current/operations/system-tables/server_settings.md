## Описание \{#description\}

Содержит информацию о глобальных настройках сервера, которые задаются в `config.xml`.
В настоящее время таблица показывает только настройки из первого уровня `config.xml` и не поддерживает вложенные конфигурации (например, [logger](../../operations/server-configuration-parameters/settings.md#logger)).

## Столбцы \{#columns\}

{/*AUTOGENERATED_START*/ }

* `name` ([String](../../sql-reference/data-types/)) — Имя настройки сервера.
* `value` ([String](../../sql-reference/data-types/)) — Значение настройки сервера.
* `default` ([String](../../sql-reference/data-types/)) — Значение настройки сервера по умолчанию.
* `changed` ([UInt8](../../sql-reference/data-types/)) — Показывает, была ли настройка указана в config.xml.
* `description` ([String](../../sql-reference/data-types/)) — Краткое описание настройки сервера.
* `type` ([String](../../sql-reference/data-types/)) — Тип значения настройки сервера.
* `changeable_without_restart` ([Enum8(&#39;No&#39; = 0, &#39;IncreaseOnly&#39; = 1, &#39;DecreaseOnly&#39; = 2, &#39;Yes&#39; = 3)](../../sql-reference/data-types/)) — Показывает, можно ли изменить настройку без перезапуска.
* `is_obsolete` ([UInt8](../../sql-reference/data-types/)) — Показывает, является ли настройка устаревшей.

{/*AUTOGENERATED_END*/ }

## Пример \{#example\}

В следующем примере показано, как получить информацию о настройках сервера, имена которых содержат `thread_pool`.

```sql
SELECT *
FROM system.server_settings
WHERE name LIKE '%thread_pool%'
```

Использование `WHERE changed` может быть полезно, например, когда вы хотите проверить,
правильно ли загружены настройки из конфигурационных файлов и действительно ли они применяются.

{/* */ }

```sql
SELECT * FROM system.server_settings WHERE changed AND name='max_thread_pool_size'
```
