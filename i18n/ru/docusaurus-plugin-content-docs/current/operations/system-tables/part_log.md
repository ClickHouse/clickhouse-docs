## Описание \{#description\}

Таблица `system.part_log` создаётся только в том случае, если задан параметр сервера [part&#95;log](/operations/server-configuration-parameters/settings#part_log).

Эта таблица содержит информацию о событиях, произошедших с [частями данных](../../engines/table-engines/mergetree-family/custom-partitioning-key.md) в таблицах семейства [MergeTree](../../engines/table-engines/mergetree-family/mergetree.md), таких как добавление или слияние данных.

Таблица `system.part_log` содержит следующие столбцы:

## Столбцы \{#columns\}

{/*AUTOGENERATED_START*/ }

* `hostname` ([LowCardinality(String)](/sql-reference/data-types/lowcardinality)) — имя хоста сервера, выполняющего запрос.
* `query_id` ([String](/sql-reference/data-types/string)) — Идентификатор INSERT-запроса, создавшего эту часть данных.
* `event_type` ([Enum8(&#39;NewPart&#39; = 1, &#39;MergeParts&#39; = 2, &#39;DownloadPart&#39; = 3, &#39;RemovePart&#39; = 4, &#39;MutatePart&#39; = 5, &#39;MovePart&#39; = 6, &#39;MergePartsStart&#39; = 7, &#39;MutatePartStart&#39; = 8)](/sql-reference/data-types/enum)) — Тип события, произошедшего с частью данных. Может принимать одно из следующих значений: NewPart — вставка новой части данных, MergePartsStart — слияние частей данных началось, MergeParts — слияние частей данных завершилось, DownloadPart — загрузка части данных, RemovePart — удаление или отсоединение части данных с помощью [DETACH PARTITION](/sql-reference/statements/alter/partition#detach-partitionpart). MutatePartStart — мутация части данных началась, MutatePart — мутация части данных завершилась, MovePart — перемещение части данных с одного диска на другой.
* `merge_reason` ([Enum8(&#39;NotAMerge&#39; = 1, &#39;RegularMerge&#39; = 2, &#39;TTLDeleteMerge&#39; = 3, &#39;TTLRecompressMerge&#39; = 4, &#39;TTLDropMerge&#39; = 5)](/sql-reference/data-types/enum)) — Причина события типа MERGE&#95;PARTS. Может принимать одно из следующих значений: NotAMerge — текущее событие имеет тип, отличный от MERGE&#95;PARTS, RegularMerge — обычное слияние, TTLDeleteMerge, TTLDropMerge — очистка устаревших данных. TTLRecompressMerge — повторное сжатие части данных.
* `merge_algorithm` ([Enum8(&#39;Undecided&#39; = 0, &#39;Vertical&#39; = 1, &#39;Horizontal&#39; = 2)](/sql-reference/data-types/enum)) — Алгоритм слияния для события типа MERGE&#95;PARTS. Может иметь одно из следующих значений: Undecided, Horizontal, Vertical
* `event_date` ([Date](/sql-reference/data-types/date)) — Дата события.
* `event_time` ([DateTime](/sql-reference/data-types/datetime)) — Время события.
* `event_time_microseconds` ([DateTime64(6)](/sql-reference/data-types/datetime64)) — Время события с точностью до микросекунд.
* `duration_ms` ([UInt64](/sql-reference/data-types/int-uint)) — Длительность выполнения этой операции.
* `database` ([String](/sql-reference/data-types/string)) — Имя базы данных, в которой находится часть данных.
* `table` ([String](/sql-reference/data-types/string)) — Имя таблицы, в которой находится эта часть данных.
* `table_uuid` ([UUID](/sql-reference/data-types/uuid)) — UUID таблицы, к которой относится часть данных.
* `part_name` ([String](/sql-reference/data-types/string)) — Имя части.
* `partition_id` ([String](/sql-reference/data-types/string)) — идентификатор партиции, в которую была вставлена часть данных. Столбец принимает значение `all`, если используется партиционирование по `tuple()`.
* `partition` ([String](/sql-reference/data-types/string)) — имя партиции.
* `part_type` ([String](/sql-reference/data-types/string)) — Тип части. Возможные значения: Wide и Compact.
* `part_storage_type` ([String](/sql-reference/data-types/string)) — Тип хранилища DataPartStorage. Возможные значения: Packed — все файлы хранятся в одном blob, Full — отдельный blob для каждого файла.
* `disk_name` ([String](/sql-reference/data-types/string)) — Имя диска, на котором находится часть данных.
* `path_on_disk` ([String](/sql-reference/data-types/string)) — Абсолютный путь к папке, содержащей файлы части данных.
* `rows` ([UInt64](/sql-reference/data-types/int-uint)) — Количество строк в части данных.
* `size_in_bytes` ([UInt64](/sql-reference/data-types/int-uint)) — Размер части данных на диске в байтах.
* `merged_from` ([Array(String)](/sql-reference/data-types/array)) — Массив имен исходных частей, из которых была сформирована текущая часть.
* `bytes_uncompressed` ([UInt64](/sql-reference/data-types/int-uint)) — Размер несжатой части в байтах.
* `read_rows` ([UInt64](/sql-reference/data-types/int-uint)) — Количество строк, прочитанных в ходе слияния.
* `read_bytes` ([UInt64](/sql-reference/data-types/int-uint)) — Количество байт, прочитанных в ходе слияния.
* `peak_memory_usage` ([UInt64](/sql-reference/data-types/int-uint)) — Максимальный объём оперативной памяти, использованной при слиянии
* `deduplication_block_ids` ([Array(String)](/sql-reference/data-types/array)) — Массив идентификаторов блоков, используемых для дедупликации при вставке этой части.
* `error` ([UInt16](/sql-reference/data-types/int-uint)) — Код ошибки возникшего исключения.
* `exception` ([String](/sql-reference/data-types/string)) — Текст сообщения об ошибке.
* `mutation_ids` ([Array(String)](/sql-reference/data-types/array)) — Массив идентификаторов mutation, применённых к исходной части (merged&#95;from) для событий типа MUTATE&#95;PART&#95;START и MUTATE&#95;PART.
* `ProfileEvents` ([Map(LowCardinality(String), UInt64)](/sql-reference/data-types/map)) — Все profile events, зарегистрированные в ходе этой операции.
* `projections_duration_ms` ([Map(LowCardinality(String), UInt64)](/sql-reference/data-types/map)) — Длительность слияния/перестроения каждой проекции в миллисекундах.

**Псевдонимы:**

* `ProfileEvents.Names` — псевдоним для `mapKeys(ProfileEvents)`.
* `ProfileEvents.Values` — псевдоним для `mapValues(ProfileEvents)`.
* `name` — псевдоним для `part_name`.

{/*AUTOGENERATED_END*/ }

Таблица `system.part_log` создаётся после первой вставки данных в таблицу `MergeTree`.
