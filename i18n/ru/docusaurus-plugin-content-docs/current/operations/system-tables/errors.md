Содержит коды ошибок с количеством их возникновений.

Чтобы отобразить все возможные коды ошибок, включая те, которые не возникали, установите настройку [system_events_show_zero_values](../settings/settings.md#system_events_show_zero_values) в значение 1.

Столбцы:

<!--AUTOGENERATED_START-->

- `name` ([String](../../sql-reference/data-types/)) — Имя ошибки (errorCodeToName).
- `code` ([Int32](../../sql-reference/data-types/)) — Числовой код ошибки.
- `value` ([UInt64](../../sql-reference/data-types/)) — Количество возникновений данной ошибки.
- `last_error_time` ([DateTime](../../sql-reference/data-types/)) — Время последнего возникновения ошибки.
- `last_error_message` ([String](../../sql-reference/data-types/)) — Сообщение последней ошибки.
- `last_error_format_string` ([String](../../sql-reference/data-types/)) — Строка форматирования последней ошибки.
- `last_error_trace` ([Array(UInt64)](../../sql-reference/data-types/)) — Трассировка стека, представляющая список физических адресов, по которым хранятся вызванные методы.
- `remote` ([UInt8](../../sql-reference/data-types/)) — Удалённое исключение (т. е. полученное во время выполнения одного из распределённых запросов).
- `query_id` ([String](../../sql-reference/data-types/)) — Идентификатор запроса, вызвавшего ошибку (если доступен).
<!--AUTOGENERATED_END-->

:::note
Счётчики некоторых ошибок могут увеличиваться при успешном выполнении запросов. Не рекомендуется использовать эту таблицу для мониторинга сервера, если вы не уверены, что соответствующая ошибка не является ложным срабатыванием.
:::

**Пример**

```sql
SELECT name, code, value
FROM system.errors
WHERE value > 0
ORDER BY code ASC
LIMIT 1

┌─name─────────────┬─code─┬─value─┐
│ CANNOT_OPEN_FILE │   76 │     1 │
└──────────────────┴──────┴───────┘
```

```sql
WITH arrayMap(x -> demangle(addressToSymbol(x)), last_error_trace) AS all
SELECT name, arrayStringConcat(all, '\n') AS res
FROM system.errors
LIMIT 1
SETTINGS allow_introspection_functions=1\G
```
