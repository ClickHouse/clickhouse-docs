## Описание \{#description\}

Содержит статус загрузки, информацию об ошибках и метаданные конфигурации для [пользовательских функций (UDF)](/sql-reference/functions/udf.md).

## Столбцы \{#columns\}

{/*AUTOGENERATED_START*/ }

* `name` ([String](../../sql-reference/data-types/)) — Имя UDF.
* `load_status` ([Enum8(&#39;Success&#39; = 0, &#39;Failed&#39; = 1)](../../sql-reference/data-types/)) — Статус загрузки. Возможные значения:
  * **Success** — UDF загружена и готова к использованию
  * **Failed** — Не удалось загрузить UDF (подробности см. в поле &#39;loading&#95;error&#95;message&#39;).
* `loading_error_message` ([String](../../sql-reference/data-types/)) — Подробное сообщение об ошибке, если загрузка завершилась неудачно. Пусто, если загрузка прошла успешно.
* `last_successful_update_time` ([Nullable(DateTime)](../../sql-reference/data-types/)) — Таймстэмп последнего успешного обновления. NULL, если успешных обновлений не было.
* `loading_duration_ms` ([UInt64](../../sql-reference/data-types/)) — Время загрузки UDF в миллисекундах.
* `type` ([Enum8(&#39;executable&#39; = 0, &#39;executable&#95;pool&#39; = 1)](../../sql-reference/data-types/)) — Тип UDF: &#39;executable&#39; (один процесс) или &#39;executable&#95;pool&#39; (пул процессов).
* `command` ([String](../../sql-reference/data-types/)) — Скрипт или команда для выполнения этой UDF.
* `format` ([String](../../sql-reference/data-types/)) — Формат данных для ввода/вывода (например, &#39;TabSeparated&#39;, &#39;JSONEachRow&#39;).
* `return_type` ([String](../../sql-reference/data-types/)) — Тип возвращаемого значения функции (например, &#39;String&#39;, &#39;UInt64&#39;).
* `return_name` ([String](../../sql-reference/data-types/)) — Необязательный идентификатор возвращаемого значения. Пусто, если не настроен.
* `argument_types` ([Array(String)](../../sql-reference/data-types/)) — Массив типов аргументов (например, [&#39;String&#39;, &#39;UInt64&#39;]).
* `argument_names` ([Array(String)](../../sql-reference/data-types/)) — Массив имен аргументов. Для аргументов без имени используются пустые строки.
* `max_command_execution_time` ([UInt64](../../sql-reference/data-types/)) — Максимальное время обработки блока данных в секундах. Только для типа &#39;executable&#95;pool&#39;.
* `command_termination_timeout` ([UInt64](../../sql-reference/data-types/)) — Количество секунд до отправки SIGTERM процессу команды.
* `command_read_timeout` ([UInt64](../../sql-reference/data-types/)) — Количество миллисекунд на чтение из stdout команды.
* `command_write_timeout` ([UInt64](../../sql-reference/data-types/)) — Количество миллисекунд на запись в stdin команды.
* `pool_size` ([UInt64](../../sql-reference/data-types/)) — Количество экземпляров процесса команды. Только для типа &#39;executable&#95;pool&#39;.
* `send_chunk_header` ([UInt8](../../sql-reference/data-types/)) — Отправлять ли количество строк перед каждым фрагментом данных (логическое значение).
* `execute_direct` ([UInt8](../../sql-reference/data-types/)) — Выполнять ли команду напрямую (1) или через /bin/bash (0).
* `lifetime` ([UInt64](../../sql-reference/data-types/)) — Интервал перезагрузки в секундах. 0 означает, что перезагрузка отключена.
* `deterministic` ([UInt8](../../sql-reference/data-types/)) — Возвращает ли функция один и тот же результат для одних и тех же аргументов (логическое значение).

{/*AUTOGENERATED_END*/ }

## Пример \{#example\}

Чтобы просмотреть все пользовательские функции (UDF) и их статус загрузки:

```sql
SELECT
    name,
    load_status,
    type,
    command,
    return_type,
    argument_types
FROM system.user_defined_functions
FORMAT Vertical;
```

```response
Row 1:
──────
name:           my_sum_udf
load_status:    Success
type:           executable
command:        /var/lib/clickhouse/user_scripts/sum.py
return_type:    UInt64
argument_types: ['UInt64','UInt64']
```

Найдите пользовательские функции (UDF), завершившиеся с ошибкой:

```sql
SELECT
    name,
    loading_error_message
FROM system.user_defined_functions
WHERE load_status = 'Failed';
```
