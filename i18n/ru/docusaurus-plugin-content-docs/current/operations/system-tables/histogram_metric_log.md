## Описание \{#description\}

История `system.histogram_metrics`. Снимок создается каждые `collect_interval_milliseconds`, после чего данные сбрасываются на диск.

## Столбцы \{#columns\}

{/*AUTOGENERATED_START*/ }

* `hostname` ([LowCardinality(String)](/sql-reference/data-types/lowcardinality)) — Имя хоста сервера, выполняющего запрос.
* `event_date` ([Date](/sql-reference/data-types/date)) — Дата события.
* `event_time` ([DateTime](/sql-reference/data-types/datetime)) — Время события.
* `event_time_microseconds` ([DateTime64(6)](/sql-reference/data-types/datetime64)) — Время события с точностью до микросекунд.
* `metric` ([LowCardinality(String)](/sql-reference/data-types/lowcardinality)) — Имя метрики.
* `labels` ([Map(LowCardinality(String), LowCardinality(String))](/sql-reference/data-types/map)) — Метки метрики.
* `histogram` ([Map(Float64, UInt64)](/sql-reference/data-types/map)) — Кумулятивная гистограмма: сопоставляет верхнюю границу бакета с количеством наблюдений ≤ этой границы; включает +inf в качестве последнего бакета.
* `count` ([UInt64](/sql-reference/data-types/int-uint)) — Общее количество наблюдений, равно histogram[+inf].
* `sum` ([Float64](/sql-reference/data-types/float)) — Сумма всех наблюдаемых значений.

{/*AUTOGENERATED_END*/ }

## Пример \{#example\}

```sql
SELECT event_time, metric, labels, histogram
FROM system.histogram_metric_log
WHERE metric = 'keeper_response_time_ms'
ORDER BY event_time DESC
LIMIT 1
FORMAT Vertical;
```
