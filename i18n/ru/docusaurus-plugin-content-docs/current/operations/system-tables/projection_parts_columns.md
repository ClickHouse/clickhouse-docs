## Описание \{#description\}

Эта таблица содержит информацию о столбцах частей проекций таблиц семейства MergeTree.

## Столбцы \{#columns\}

{/*AUTOGENERATED_START*/ }

* `partition` ([String](../../sql-reference/data-types/)) — имя партиции.
* `name` ([String](../../sql-reference/data-types/)) — Имя части данных.
* `part_type` ([String](../../sql-reference/data-types/)) — Формат хранения части данных.
* `parent_name` ([String](../../sql-reference/data-types/)) — Имя исходной (родительской) части данных.
* `parent_uuid` ([UUID](../../sql-reference/data-types/)) — UUID исходной (родительской) части данных.
* `parent_part_type` ([String](../../sql-reference/data-types/)) — Формат хранения исходной (родительской) части.
* `active` ([UInt8](../../sql-reference/data-types/)) — Флаг, указывающий, является ли часть данных активной
* `marks` ([UInt64](../../sql-reference/data-types/)) — Количество меток.
* `rows` ([UInt64](../../sql-reference/data-types/)) — Количество строк.
* `bytes_on_disk` ([UInt64](../../sql-reference/data-types/)) — Общий размер в байтах всех файлов частей данных.
* `data_compressed_bytes` ([UInt64](../../sql-reference/data-types/)) — Общий размер сжатых данных в части данных. Все вспомогательные файлы (например, файлы с метками) не учитываются.
* `data_uncompressed_bytes` ([UInt64](../../sql-reference/data-types/)) — Общий размер несжатых данных в части данных. Все вспомогательные файлы (например, файлы с метками) не учитываются.
* `marks_bytes` ([UInt64](../../sql-reference/data-types/)) — Размер файла с метками.
* `parent_marks` ([UInt64](../../sql-reference/data-types/)) — Количество меток в исходной (родительской) части.
* `parent_rows` ([UInt64](../../sql-reference/data-types/)) — Количество строк в исходной (родительской) части.
* `parent_bytes_on_disk` ([UInt64](../../sql-reference/data-types/)) — Общий размер в байтах всех файлов исходной (родительской) части данных.
* `parent_data_compressed_bytes` ([UInt64](../../sql-reference/data-types/)) — Суммарный размер сжатых данных в исходной (родительской) части данных.
* `parent_data_uncompressed_bytes` ([UInt64](../../sql-reference/data-types/)) — Общий размер несжатых данных в исходной (родительской) части.
* `parent_marks_bytes` ([UInt64](../../sql-reference/data-types/)) — Размер файла с метками в исходной (родительской) части данных.
* `modification_time` ([DateTime](../../sql-reference/data-types/)) — Время изменения каталога, содержащего часть данных. Обычно оно соответствует времени создания части данных.
* `remove_time` ([DateTime](../../sql-reference/data-types/)) — Время, когда часть стала неактивной.
* `refcount` ([UInt32](../../sql-reference/data-types/)) — Количество мест использования части данных. Значение больше 2 указывает на то, что часть данных используется в запросах или слияниях.
* `min_date` ([Date](../../sql-reference/data-types/)) — Минимальное значение для столбца Date, если он входит в ключ партиционирования.
* `max_date` ([Date](../../sql-reference/data-types/)) — Максимальное значение в столбце Date, если он включён в ключ партиционирования.
* `min_time` ([DateTime](../../sql-reference/data-types/)) — Минимальное значение в столбце DateTime, если этот столбец входит в ключ партиции.
* `max_time` ([DateTime](../../sql-reference/data-types/)) — Максимальное значение для столбца с типом DateTime, если такой столбец включён в ключ партиции.
* `partition_id` ([String](../../sql-reference/data-types/)) — идентификатор партиции.
* `min_block_number` ([Int64](../../sql-reference/data-types/)) — Минимальный номер среди частей данных, из которых после слияния была сформирована текущая часть.
* `max_block_number` ([Int64](../../sql-reference/data-types/)) — Максимальное количество частей данных, из которых после слияния состоит текущая часть.
* `level` ([UInt32](../../sql-reference/data-types/)) — Глубина дерева слияний. Ноль означает, что текущая часть была создана в результате вставки, а не слияния других частей.
* `data_version` ([UInt64](../../sql-reference/data-types/)) — Число, используемое для определения, какие Мутации должны быть применены к части данных (Мутации с версией выше, чем data&#95;version).
* `primary_key_bytes_in_memory` ([UInt64](../../sql-reference/data-types/)) — Объём памяти (в байтах), используемый значениями первичного ключа.
* `primary_key_bytes_in_memory_allocated` ([UInt64](../../sql-reference/data-types/)) — Объём памяти (в байтах), зарезервированный для значений первичного ключа.
* `database` ([String](../../sql-reference/data-types/)) — имя базы данных.
* `table` ([String](../../sql-reference/data-types/)) — Название таблицы.
* `engine` ([String](../../sql-reference/data-types/)) — Название движка таблицы без параметров.
* `disk_name` ([String](../../sql-reference/data-types/)) — Имя диска, на котором хранится часть данных.
* `path` ([String](../../sql-reference/data-types/)) — Абсолютный путь к папке с файлами части.
* `column` ([String](../../sql-reference/data-types/)) — Имя столбца.
* `type` ([String](../../sql-reference/data-types/)) — Тип столбца.
* `column_position` ([UInt64](../../sql-reference/data-types/)) — Порядковый номер столбца в таблице, начиная с 1.
* `default_kind` ([String](../../sql-reference/data-types/)) — Тип выражения (DEFAULT, MATERIALIZED, ALIAS) для значения по умолчанию, либо пустая строка, если оно не задано.
* `default_expression` ([String](../../sql-reference/data-types/)) — Выражение, задающее значение по умолчанию, или пустая строка, если оно не задано.
* `column_bytes_on_disk` ([UInt64](../../sql-reference/data-types/)) — Общий размер столбца в байтах.
* `column_data_compressed_bytes` ([UInt64](../../sql-reference/data-types/)) — Общий размер сжатых данных в столбце в байтах.
* `column_data_uncompressed_bytes` ([UInt64](../../sql-reference/data-types/)) — Общий размер декомпрессированных данных в столбце в байтах.
* `column_marks_bytes` ([UInt64](../../sql-reference/data-types/)) — Размер столбца с метками, в байтах.
* `column_modification_time` ([Nullable(DateTime)](../../sql-reference/data-types/)) — Время последнего изменения столбца.

{/*AUTOGENERATED_END*/ }

**Псевдонимы:**

* `bytes` — псевдоним `bytes_on_disk`.
* `marks_size` — псевдоним `marks_bytes`.
* `part_name` — псевдоним `name`.
