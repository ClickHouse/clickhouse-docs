## Описание \{#description\}

Содержит информацию о зависимых представлениях, выполняемых при выполнении запроса, например о типе представления или времени выполнения.

Чтобы начать логирование:

1. Настройте параметры в разделе [query&#95;views&#95;log](../../operations/server-configuration-parameters/settings.md#query_views_log).
2. Установите [log&#95;query&#95;views](/operations/settings/settings#log_query_views) в 1.

Период сброса данных задаётся параметром `flush_interval_milliseconds` в разделе серверных настроек [query&#95;views&#95;log](../../operations/server-configuration-parameters/settings.md#query_views_log). Чтобы принудительно выполнить сброс, используйте запрос [SYSTEM FLUSH LOGS](/sql-reference/statements/system#flush-logs).

ClickHouse не удаляет данные из таблицы автоматически. Подробности см. во вводном разделе [Introduction](/operations/system-tables/overview#system-tables-introduction).

Вы можете использовать настройку [log&#95;queries&#95;probability](/operations/settings/settings#log_queries_probability)) для уменьшения количества запросов, регистрируемых в таблице `query_views_log`.

## Столбцы \{#columns\}

{/*AUTOGENERATED_START*/ }

* `hostname` ([LowCardinality(String)](/sql-reference/data-types/lowcardinality)) — Имя хоста сервера, выполняющего запрос.
* `event_date` ([Date](/sql-reference/data-types/date)) — Дата, когда произошло последнее событие представления.
* `event_time` ([DateTime](/sql-reference/data-types/datetime)) — Дата и время завершения выполнения представления.
* `event_time_microseconds` ([DateTime64(6)](/sql-reference/data-types/datetime64)) — Дата и время завершения выполнения представления с точностью до микросекунд.
* `view_duration_ms` ([UInt64](/sql-reference/data-types/int-uint)) — Длительность выполнения представления (сумма всех его этапов) в миллисекундах.
* `initial_query_id` ([String](/sql-reference/data-types/string)) — Идентификатор исходного запроса (для распределённого выполнения запроса).
* `view_name` ([String](/sql-reference/data-types/string)) — Имя представления.
* `view_uuid` ([UUID](/sql-reference/data-types/uuid)) — UUID представления.
* `view_type` ([Enum8(&#39;Default&#39; = 1, &#39;Materialized&#39; = 2, &#39;Live&#39; = 3, &#39;Window&#39; = 4)](/sql-reference/data-types/enum)) — Тип представления. Значения: &#39;Default&#39; = 1 — представления по умолчанию. Не должно появляться в этом журнале, &#39;Materialized&#39; = 2 — materialized view, &#39;Live&#39; = 3 — live view.
* `view_query` ([String](/sql-reference/data-types/string)) — Запрос, выполненный представлением.
* `view_target` ([String](/sql-reference/data-types/string)) — Имя целевой таблицы представления.
* `read_rows` ([UInt64](/sql-reference/data-types/int-uint)) — Количество прочитанных строк.
* `read_bytes` ([UInt64](/sql-reference/data-types/int-uint)) — Количество прочитанных байт.
* `written_rows` ([UInt64](/sql-reference/data-types/int-uint)) — Количество записанных строк.
* `written_bytes` ([UInt64](/sql-reference/data-types/int-uint)) — Количество записанных байт.
* `peak_memory_usage` ([Int64](/sql-reference/data-types/int-uint)) — Максимальная разница между объёмом выделенной и освобождённой памяти в контексте этого представления.
* `ProfileEvents` ([Map(LowCardinality(String), UInt64)](/sql-reference/data-types/map)) — `ProfileEvents`, измеряющие различные метрики. Их описание можно найти в системной таблице `system.events`.
* `status` ([Enum8(&#39;QueryStart&#39; = 1, &#39;QueryFinish&#39; = 2, &#39;ExceptionBeforeStart&#39; = 3, &#39;ExceptionWhileProcessing&#39; = 4)](/sql-reference/data-types/enum)) — Статус представления. Значения: &#39;QueryStart&#39; = 1 — успешное начало выполнения представления. Не должно появляться, &#39;QueryFinish&#39; = 2 — успешное завершение выполнения представления, &#39;ExceptionBeforeStart&#39; = 3 — исключение до начала выполнения представления, &#39;ExceptionWhileProcessing&#39; = 4 — исключение во время выполнения представления.
* `exception_code` ([Int32](/sql-reference/data-types/int-uint)) — Код исключения.
* `exception` ([String](/sql-reference/data-types/string)) — Сообщение об исключении.
* `stack_trace` ([String](/sql-reference/data-types/string)) — Трассировка стека. Пустая строка, если запрос был успешно завершён.

**Псевдонимы:**

* `ProfileEvents.Names` — Псевдоним для `mapKeys(ProfileEvents)`.
* `ProfileEvents.Values` — Псевдоним для `mapValues(ProfileEvents)`.
