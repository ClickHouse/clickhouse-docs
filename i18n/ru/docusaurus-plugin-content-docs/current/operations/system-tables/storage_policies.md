## Описание \{#description\}

Содержит информацию о политиках хранения и томах, которые определены в [конфигурации сервера](../../engines/table-engines/mergetree-family/mergetree.md#table_engine-mergetree-multiple-volumes_configure).

## Столбцы \{#columns\}

{/*AUTOGENERATED_START*/ }

* `policy_name` ([String](../../sql-reference/data-types/)) — Имя политики хранения.
* `volume_name` ([String](../../sql-reference/data-types/)) — Имя тома.
* `volume_priority` ([UInt64](../../sql-reference/data-types/)) — Приоритет тома.
* `disks` ([Array(String)](../../sql-reference/data-types/)) — Список имен всех дисков, входящих в эту политику хранения.
* `volume_type` ([Enum8(&#39;JBOD&#39; = 0, &#39;SINGLE&#95;DISK&#39; = 1, &#39;UNKNOWN&#39; = 2)](../../sql-reference/data-types/)) — Тип тома — JBOD или одиночный диск.
* `max_data_part_size` ([UInt64](../../sql-reference/data-types/)) — Максимальный размер части, которая может храниться на любом из дисков тома.
* `move_factor` ([Float32](../../sql-reference/data-types/)) — Когда объем доступного пространства становится меньше этого коэффициента, данные автоматически начинают перемещаться на следующий том, если он есть (по умолчанию 0.1).
* `prefer_not_to_merge` ([UInt8](../../sql-reference/data-types/)) — Не рекомендуется использовать эту настройку. Отключает слияние частей на этом томе (это вредно и приводит к снижению производительности).
* `perform_ttl_move_on_insert` ([UInt8](../../sql-reference/data-types/)) — Отключает TTL-перемещение при INSERT части. По умолчанию (если включено), если вставляемая часть уже соответствует правилу TTL-перемещения, она немедленно помещается на том/диск, указанный в правиле перемещения.
* `load_balancing` ([Enum8(&#39;ROUND&#95;ROBIN&#39; = 0, &#39;LEAST&#95;USED&#39; = 1)](../../sql-reference/data-types/)) — Политика балансировки дисков: `round_robin` или `least_used`.

{/*AUTOGENERATED_END*/ }

## Выбор тома при `INSERT` \{#volume-selection-on-insert\}

Когда `INSERT` создает новую часть, ClickHouse выбирает целевой диск,
последовательно применяя приведенные ниже правила. Срабатывает первое правило, которое подходит **и может
зарезервировать место для части**; в противном случае (правило не применяется, нет свободного
места или превышен `max_data_part_size`) проверка продолжается со следующим
правилом.

1. **Правило TTL-перемещения** — если для вставляемых строк момент, заданный условием `TTL <expr> TO VOLUME 'X'` (или `TO DISK 'X'`),
   уже наступил, **и**
   `perform_ttl_move_on_insert = 1` (по умолчанию) на **целевом
   томе TTL** (для `TO DISK 'X'` — на томе, содержащем диск `X`), то часть
   записывается сразу в это целевое расположение. Если зарезервировать место там не удается, операция
   вставки переходит к шагам 2–4; в журнал записывается предупреждение, но `INSERT`
   не завершается ошибкой только по этой причине.
2. **`max_data_part_size`** — том отклоняет части, размер которых превышает
   его `max_data_part_size`. Эта проверка выполняется отдельно для каждого тома; она не применяется к
   резервированию на шаге 1 для `TTL ... TO DISK 'X'`, так как в этом случае операция выполняется
   непосредственно на диске.
3. **`volume_priority`** — среди оставшихся томов выбирается тот, у которого
   значение `volume_priority` минимально. Томы без явно заданного
   `<volume_priority>` упорядочиваются по их позиции в конфигурации.
4. **`load_balancing`** — после выбора тома диск внутри этого
   тома выбирается в соответствии с его политикой `load_balancing`
   (`round_robin` или `least_used`).

:::note Переопределение
Если `min_free_disk_bytes_to_perform_insert` или
`min_free_disk_ratio_to_perform_insert` не равно нулю, описанный выше порядок
не применяется. `INSERT` пытается использовать только том с наименьшим
`volume_priority` и генерирует `NOT_ENOUGH_SPACE`, если ни один диск в этом томе
не удовлетворяет пороговому значению. Вставки в базу данных `system` являются исключением.
:::

:::note
`perform_ttl_move_on_insert` считывается из **целевого тома TTL**,
а не из исходного тома. Для правила `TO DISK 'X'` этот флаг считывается
из тома, содержащего диск `X`. Установка этого параметра на любом другом томе
политики не влияет на путь вставки.
:::

Чтобы принудительно заставить вставки учитывать `volume_priority`, даже если применяется правило TTL-перемещения для уже
истекшего TTL, установите `perform_ttl_move_on_insert = 0` на
целевом томе TTL (для `TO DISK 'X'` — на томе, содержащем
диск `X`). Тогда часть сначала будет записана на том с приоритетом N, а затем
перемещена в целевой том TTL фоновой задачей перемещения (это можно наблюдать через
`system.moves`). См.
[настройку `perform_ttl_move_on_insert` для движка MergeTree](/engines/table-engines/mergetree-family/mergetree#table_engine-mergetree-multiple-volumes_configure).
