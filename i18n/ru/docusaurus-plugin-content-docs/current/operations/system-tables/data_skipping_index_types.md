## Описание \{#description\}

Содержит список типов индексов пропуска данных, поддерживаемых сервером, а также встроенную документацию для каждого типа. Тип индекса пропуска данных указывается в `TYPE` объявления `INDEX` в запросе `CREATE TABLE` и позволяет ClickHouse пропускать гранулы, которые не могут соответствовать условию запроса.

Обратите внимание, что в этой таблице перечислены доступные *типы* индексов, тогда как [`system.data_skipping_indices`](/operations/system-tables/data_skipping_indices) содержит экземпляры индексов, определённые в существующих таблицах.

## Столбцы \{#columns\}

{/*AUTOGENERATED_START*/ }

* `name` ([String](../../sql-reference/data-types/)) — Название типа индекса пропуска данных, как указано в TYPE объявления INDEX.
* `description` ([String](../../sql-reference/data-types/)) — Общее описание назначения типа индекса пропуска данных.
* `syntax` ([String](../../sql-reference/data-types/)) — Способ объявления индекса в предложении INDEX запроса CREATE TABLE.
* `examples` ([String](../../sql-reference/data-types/)) — Примеры использования.
* `introduced_in` ([String](../../sql-reference/data-types/)) — Версия ClickHouse, в которой этот тип индекса был впервые представлен, в формате major.minor.
* `related` ([Array(String)](../../sql-reference/data-types/)) — Названия связанных типов индексов пропуска данных.

{/*AUTOGENERATED_END*/ }

## Пример \{#example\}

```sql title="Query"
SELECT name, syntax
FROM system.data_skipping_index_types
WHERE name IN ('minmax', 'set')
ORDER BY name
```

```text title="Response"
┌─name───┬─syntax───────────────────────────────────────┐
│ minmax │ INDEX name expr TYPE minmax GRANULARITY n     │
│ set    │ INDEX name expr TYPE set(max_rows) GRANULARITY n │
└────────┴──────────────────────────────────────────────┘
```
