## Описание \{#description\}

Содержит сведения о существующих ограничениях во всех таблицах.

Ограничения, определённые для [временных таблиц](../../sql-reference/statements/create/table.md#temporary-tables), видны в `system.constraints` только в том сеансе, в котором они были созданы. Они отображаются с пустым полем `database`.

## Столбцы \{#columns\}

{/*AUTOGENERATED_START*/ }

* `database` ([String](../../sql-reference/data-types/)) — Имя базы данных.
* `table` ([String](../../sql-reference/data-types/)) — Имя таблицы.
* `name` ([String](../../sql-reference/data-types/)) — Имя ограничения.
* `type` ([Enum8(&#39;CHECK&#39; = 0, &#39;ASSUME&#39; = 1)](../../sql-reference/data-types/)) — Тип ограничения.
* `expression` ([String](../../sql-reference/data-types/)) — Выражение ограничения.

{/*AUTOGENERATED_END*/ }

## Пример \{#example\}

```sql
SELECT * FROM system.constraints LIMIT 2 FORMAT Vertical;
```

```text
Row 1:
──────
database:    default
table:       hits
name:        check_hits
type:        CHECK
expression:  CounterID > 0

Row 2:
──────
database:    default
table:       hits
name:        assume_positive
type:        ASSUME
expression:  WatchID > 0
```
