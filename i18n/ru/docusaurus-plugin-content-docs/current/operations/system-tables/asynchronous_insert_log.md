## Описание \{#description\}

Содержит информацию об асинхронных вставках. Каждая запись соответствует запросу INSERT, поставленному в буфер асинхронных вставок.

Чтобы начать вести журнал, настройте параметры в разделе [asynchronous&#95;insert&#95;log](../../operations/server-configuration-parameters/settings.md#asynchronous_insert_log).

Период сброса данных задаётся параметром `flush_interval_milliseconds` в разделе настроек сервера [asynchronous&#95;insert&#95;log](../../operations/server-configuration-parameters/settings.md#asynchronous_insert_log). Чтобы принудительно выполнить сброс, используйте запрос [SYSTEM FLUSH LOGS](/sql-reference/statements/system#flush-logs).

ClickHouse не удаляет данные из таблицы автоматически. Подробнее см. в разделе [Введение](/operations/system-tables/overview#system-tables-introduction).

## Столбцы \{#columns\}

{/*AUTOGENERATED_START*/ }

* `hostname` ([LowCardinality(String)](/sql-reference/data-types/lowcardinality)) — Имя хоста сервера, выполняющего запрос.
* `event_date` ([Date](/sql-reference/data-types/date)) — Дата, когда произошла асинхронная вставка.
* `event_time` ([DateTime](/sql-reference/data-types/datetime)) — Дата и время завершения асинхронной вставки.
* `event_time_microseconds` ([DateTime64(6)](/sql-reference/data-types/datetime64)) — Дата и время завершения асинхронной вставки с точностью до микросекунд.
* `query` ([String](/sql-reference/data-types/string)) — Строка запроса.
* `database` ([LowCardinality(String)](/sql-reference/data-types/lowcardinality)) — Имя базы данных, в которой находится таблица.
* `table` ([LowCardinality(String)](/sql-reference/data-types/lowcardinality)) — Имя таблицы.
* `format` ([LowCardinality(String)](/sql-reference/data-types/lowcardinality)) — Имя формата.
* `query_id` ([String](/sql-reference/data-types/string)) — Идентификатор исходного запроса.
* `bytes` ([UInt64](/sql-reference/data-types/int-uint)) — Количество вставленных байт.
* `rows` ([UInt64](/sql-reference/data-types/int-uint)) — Количество вставленных строк.
* `exception` ([String](/sql-reference/data-types/string)) — Сообщение об исключении.
* `status` ([Enum8(&#39;Ok&#39; = 0, &#39;ParsingError&#39; = 1, &#39;FlushError&#39; = 2)](/sql-reference/data-types/enum)) — Статус вставки. Значения: &#39;Ok&#39; = 0 — успешная вставка, &#39;ParsingError&#39; = 1 — исключение при разборе данных, &#39;FlushError&#39; = 2 — исключение при сбросе данных.
* `data_kind` ([Enum8(&#39;Parsed&#39; = 0, &#39;Preprocessed&#39; = 1)](/sql-reference/data-types/enum)) — Состояние данных. Значения: &#39;Parsed&#39; и &#39;Preprocessed&#39;.
* `flush_time` ([DateTime](/sql-reference/data-types/datetime)) — Дата и время, когда произошёл сброс.
* `flush_time_microseconds` ([DateTime64(6)](/sql-reference/data-types/datetime64)) — Дата и время, когда произошёл сброс, с точностью до микросекунд.
* `flush_query_id` ([String](/sql-reference/data-types/string)) — Идентификатор запроса сброса.
* `timeout_milliseconds` ([UInt64](/sql-reference/data-types/int-uint)) — Адаптивный таймаут, вычисленный для этой записи.
