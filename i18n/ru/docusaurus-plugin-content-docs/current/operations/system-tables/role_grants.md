## Описание \{#description\}

Содержит назначенные роли для пользователей и ролей. Чтобы добавить записи в эту таблицу, используйте `GRANT role TO user`.

## Столбцы \{#columns\}

{/*AUTOGENERATED_START*/ }

* `user_name` ([Nullable(String)](../../sql-reference/data-types/)) — Имя пользователя.
* `role_name` ([Nullable(String)](../../sql-reference/data-types/)) — Имя роли.
* `granted_role_name` ([String](../../sql-reference/data-types/)) — Имя роли, предоставленной роли `role_name`. Чтобы предоставить одну роль другой, используйте `GRANT role1 TO role2`.
* `granted_role_id` ([UUID](../../sql-reference/data-types/)) — Идентификатор роли.
* `granted_role_is_default` ([UInt8](../../sql-reference/data-types/)) — Флаг, показывающий, является ли `granted_role` ролью по умолчанию. Возможные значения: • 1 — `granted_role` является ролью по умолчанию, • 0 — `granted_role` не является ролью по умолчанию.
* `with_admin_option` ([UInt8](../../sql-reference/data-types/)) — Флаг, показывающий, есть ли у `granted_role` привилегия `ADMIN OPTION`. Возможные значения: • 1 — У роли есть привилегия `ADMIN OPTION`. • 0 — У роли нет привилегии `ADMIN OPTION`.

{/*AUTOGENERATED_END*/ }
