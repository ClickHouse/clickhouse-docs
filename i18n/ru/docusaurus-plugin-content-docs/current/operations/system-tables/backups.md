# system.backups

Содержит список всех операций `BACKUP` или `RESTORE` с их текущими состояниями и другими свойствами. Обратите внимание, что таблица не является персистентной и показывает только операции, выполненные после последнего перезапуска сервера.

Ниже приведена таблица в формате Markdown со столбцами name и comment:

| Column              | Description                                                                                                          |
|---------------------|----------------------------------------------------------------------------------------------------------------------|
| `id`                | Идентификатор операции, может быть передан через SETTINGS id=... или быть случайно сгенерированным UUID.            |
| `name`              | Имя операции, строка вида `Disk('backups', 'my_backup')`                                                             |
| `base_backup_name`  | Имя базовой операции резервного копирования, строка вида `Disk('backups', 'my_base_backup')`                        |
| `query_id`          | Идентификатор запроса, который запустил резервное копирование.                                                      |
| `status`            | Статус операции резервного копирования или восстановления.                                                          |
| `error`             | Сообщение об ошибке, если оно есть.                                                                                 |
| `start_time`        | Время начала операции.                                                                                              |
| `end_time`          | Время завершения операции.                                                                                          |
| `num_files`         | Количество файлов, сохранённых в резервной копии.                                                                  |
| `total_size`        | Общий размер файлов, сохранённых в резервной копии.                                                                |
| `num_entries`       | Количество записей в резервной копии, т.е. количество файлов внутри каталога, если резервная копия хранится как каталог. |
| `uncompressed_size` | Несжатый размер резервной копии.                                                                                    |
| `compressed_size`   | Сжатый размер резервной копии.                                                                                      |
| `files_read`        | Количество файлов, прочитанных во время RESTORE из этой резервной копии.                                           |
| `bytes_read`        | Общий размер файлов, прочитанных во время RESTORE из этой резервной копии.                                         |
| `ProfileEvents`     | Все события профилирования, зафиксированные во время этой операции.                                                |
