ClickHouse Cloud отправляет уведомления о критических событиях, связанных с вашим сервисом или организацией. Чтобы понять, как отправляются и настраиваются уведомления, важно учитывать несколько понятий:

1. **Категория уведомления**: Обозначает группы уведомлений, такие как уведомления о биллинге, уведомления, связанные с сервисом, и т.д. В каждой категории есть несколько типов уведомлений, для которых можно настроить способ доставки.
2. **Важность уведомления**: Важность уведомления может иметь уровень `info`, `warning` или `critical` в зависимости от того, насколько оно важно. Этот параметр не настраивается.
3. **Канал уведомления**: Канал — это способ получения уведомления, например через пользовательский интерфейс, электронную почту, Slack и т.д. Для большинства уведомлений этот параметр настраивается.


## Получение уведомлений {#receiving-notifications}

Уведомления можно получать по различным каналам. В настоящее время ClickHouse Cloud поддерживает получение уведомлений по электронной почте, через пользовательский интерфейс ClickHouse Cloud и Slack. Чтобы просмотреть текущие уведомления, нажмите на значок колокольчика в верхнем левом меню — откроется всплывающая панель. Нажатие кнопки **Просмотреть все** в нижней части панели откроет страницу с журналом активности всех уведомлений.

## Настройка уведомлений {#customizing-notifications}

Для каждого уведомления можно настроить способ его получения. Экран настроек доступен из всплывающей панели уведомлений или со второй вкладки журнала активности уведомлений.

Пользователи Cloud могут настраивать уведомления, доставляемые через интерфейс Cloud, при этом настройки применяются индивидуально для каждого пользователя. Пользователи Cloud также могут настраивать уведомления, доставляемые на их собственные адреса электронной почты, однако только пользователи с правами администратора могут настраивать уведомления, доставляемые на произвольные адреса электронной почты и в каналы Slack.

Чтобы настроить доставку конкретного уведомления, нажмите на значок карандаша для изменения каналов доставки.

## Поддерживаемые уведомления {#supported-notifications}

В настоящее время мы отправляем уведомления, связанные с биллингом (ошибка оплаты, превышение установленного порога использования и т. д.), а также уведомления, связанные с событиями масштабирования (масштабирование завершено, масштабирование заблокировано и т. д.).
