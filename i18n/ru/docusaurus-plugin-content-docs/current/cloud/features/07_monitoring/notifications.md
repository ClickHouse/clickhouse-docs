ClickHouse Cloud отправляет уведомления о критических событиях, связанных с вашим сервисом или организацией. Чтобы понять, как отправляются и настраиваются уведомления, необходимо учитывать несколько понятий:

1. **Категория уведомлений**: Обозначает группы уведомлений, такие как уведомления по биллингу, уведомления, связанные с сервисом, и т. д. В каждой категории есть несколько типов уведомлений, для которых можно настроить способ доставки.
2. **Уровень важности уведомлений**: Уровень важности уведомлений может быть `info`, `warning` или `critical` в зависимости от того, насколько важно уведомление. Это нельзя изменить.
3. **Канал уведомлений**: Канал — это способ получения уведомлений, например через UI, email, Slack и т. д. Для большинства уведомлений его можно настроить.


## Получение уведомлений {#receiving-notifications}

Уведомления можно получать по различным каналам. В настоящее время ClickHouse Cloud поддерживает получение уведомлений через электронную почту, пользовательский интерфейс ClickHouse Cloud и Slack. Чтобы просмотреть текущие уведомления, нажмите на значок колокольчика в верхнем левом меню — откроется всплывающая панель. Нажатие кнопки **Просмотреть все** в нижней части панели откроет страницу с журналом активности всех уведомлений.

## Настройка уведомлений {#customizing-notifications}

Для каждого уведомления можно настроить способ его получения. Экран настроек доступен из всплывающей панели уведомлений или со второй вкладки журнала активности уведомлений.

Пользователи Cloud могут настраивать уведомления, доставляемые через интерфейс Cloud, при этом настройки применяются индивидуально для каждого пользователя. Пользователи Cloud также могут настраивать уведомления, доставляемые на их собственные адреса электронной почты, однако только пользователи с правами администратора могут настраивать уведомления, доставляемые на произвольные адреса электронной почты и в каналы Slack.

Чтобы настроить доставку конкретного уведомления, нажмите на значок карандаша для изменения каналов доставки.

## Поддерживаемые уведомления {#supported-notifications}

В настоящее время отправляются уведомления, связанные с биллингом (ошибка оплаты, превышение установленного порога использования и т. д.), а также уведомления о событиях масштабирования (масштабирование завершено, масштабирование заблокировано и т. д.).
