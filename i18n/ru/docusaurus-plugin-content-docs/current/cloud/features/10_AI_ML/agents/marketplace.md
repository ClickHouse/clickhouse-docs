Маркетплейс агентов — это место, где можно просматривать, искать и запускать агентов, которыми с вами поделились другие сотрудники вашей организации. Здесь также представлены агенты, опубликованные вашим администратором для всей организации.

## Обзор \{#browse\}

Агенты сгруппированы по категориям — например, *Analytics*, *Operations*, *Engineering*. Переключайтесь между категориями с помощью вкладок в верхней части маркетплейса. На каждой карточке агента указаны название, описание, аватар и кнопка быстрого перехода к началу разговора.

Используйте строку поиска, чтобы найти агента по названию, описанию или ключевому слову во всех категориях.

## Откройте агента \{#open-an-agent\}

Нажмите на любую карточку агента, чтобы увидеть полную информацию о нём: инструкции, используемую им модель, подключённые инструменты и сведения о том, кто им поделился. На странице сведений вы можете:

* **Начать разговор** — открыть новый чат с агентом.
* **Сохранить копию** — если у вас есть права на редактирование, клонировать агента в свой список и изменить его.

## Опубликуйте своего агента \{#publish-your-own\}

Чтобы агент можно было найти в маркетплейсе, откройте к нему доступ с нужной областью видимости на панели [совместного использования и доступа](/cloud/features/ai-ml/agents/sharing-and-access). Уровни видимости:

* **Частный** — виден только вам. В маркетплейсе не отображается.
* **Открыт для пользователей или групп** — виден этим пользователям или группам. Появляется в их маркетплейсе.
* **Для всей организации** — виден всем в организации. Зависит от разрешений маркетплейса, заданных вашим администратором.

Перед публикацией выберите для агента категорию и напишите понятное описание — эти поля используются и для поиска, и для просмотра по категориям.

## Административные настройки \{#admin-controls\}

Администраторы организации могут:

* Определять, какие категории отображаются в маркетплейсе.
* Продвигать отдельных агентов в общий список для всей организации.
* Разрешать или ограничивать доступ к маркетплейсу для каждой роли.

Описание модели разрешений см. в разделе [совместное использование и доступ](/cloud/features/ai-ml/agents/sharing-and-access).
