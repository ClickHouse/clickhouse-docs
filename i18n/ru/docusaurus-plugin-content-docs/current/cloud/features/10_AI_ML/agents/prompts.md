Библиотека промптов — это место, где можно сохранять и повторно использовать промпты на естественном языке, которые вы вводите снова и снова. Думайте о ней как о шаблонах для поля ввода чата — это удобно, когда в разных разговорах повторяется один и тот же аналитический вопрос или инструкция по форматированию.

## Создайте промпт \{#create-a-prompt\}

Откройте панель промптов и нажмите **New prompt**. Укажите:

* **Заголовок** — то, что отображается в списке. Делайте его понятным: *&quot;Weekly active users by region&quot;* лучше, чем *&quot;WAU&quot;*.
* **Текст** — собственно текст, который будет вставлен в редактор.
* **Необязательные переменные** — плейсхолдеры в тексте, которые вы заполняете при вставке. Используйте маркеры в формате `{{name}}`; перед вставкой список предложит ввести значения.

Группируйте связанные промпты по категориям или тегам, чтобы библиотекой было удобно пользоваться по мере её роста.

## Использование промпта \{#use-a-prompt\}

В беседе откройте меню выбора промптов в поле ввода сообщения и найдите нужный промпт через поиск или просмотр списка. Если у промпта есть переменные, заполните их. Текст промпта будет вставлен в поле ввода сообщения, где его можно отредактировать перед отправкой.

## Общий доступ к промптам \{#share-prompts\}

Для промптов действует та же модель доступа, что и для агентов: по умолчанию они закрыты, ими можно делиться с конкретными пользователями или группами либо открыть доступ для всей организации. См. [общий доступ и управление доступом](/cloud/features/ai-ml/agents/sharing-and-access).

## Промпты, навыки и инструкции \{#prompts-vs-skills-vs-instructions\}

* **Промпты** — это готовые текстовые фрагменты для разового использования, которые пользователь вставляет и редактирует. Пользователь сам участвует в процессе.
* **[Навыки](/cloud/features/ai-ml/agents/builder/skills)** — это наборы инструкций, которые агент активирует самостоятельно.
* **Инструкции агента** — это постоянный системный промпт агента.

Используйте промпт, когда хотите повторно использовать формулировки, но при этом каждый раз сами контролировать окончательный текст.
