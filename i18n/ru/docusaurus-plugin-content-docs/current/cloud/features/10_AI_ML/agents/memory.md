Память сохраняет пользовательский контекст между диалогами. Вместо индексации всей истории чатов она хранит компактные структурированные записи — предпочтения, повторяющиеся факты, сведения о проектах, — которые агент может при необходимости использовать.

## Как это работает \{#how-it-works\}

Небольшой агент памяти работает параллельно с основным диалогом. Он просматривает недавние сообщения, решает, что стоит запомнить, и сохраняет записи в персональном хранилище пользователя. В следующем диалоге эти записи доступны как контекст, на который может ссылаться основной агент, поэтому вам не приходится повторяться.

Для вас это выглядит как преемственность: один раз скажите агенту, что предпочитаете вывод SQL в нижнем регистре и что ваш финансовый год заканчивается в марте, — и в будущих диалогах это будет учитываться.

## Управление памятью \{#manage-your-memories\}

Откройте панель памяти в меню учетной записи, чтобы:

* **Просматривать** записи, которые агент сохранил о вас.
* **Редактировать** любую запись, чтобы исправить или уточнить ее.
* **Удалять** записи, которые вы не хотите сохранять на будущее.

Память доступна только в рамках вашей учетной записи. Агенты других людей никогда не увидят ваши записи, и ваши агенты никогда не увидят их записи.

## Переключатель памяти \{#toggle-memory\}

В каждом чате есть переключатель памяти в его заголовке. Отключайте его для деликатных тем, которые вы не хотите сохранять, или для разовых бесед, где персонализация не нужна.

Когда память отключена, агент в рамках этого разговора не читает данные из вашего хранилища памяти и не записывает их туда.

## Когда память полезна \{#when-memory-helps\}

* Повторяющиеся соглашения: предпочтительные форматы дат, бизнес-определения, шаблоны именования.
* Контекст проекта: какой сервис или какую базу данных вы обычно запрашиваете, какие панели мониторинга для вас важны.
* Стиль общения: краткие или более разговорные ответы, с упором на код или на пояснительный текст.

## Когда память не помогает \{#when-memory-doesnt-help\}

Память — не база данных. Это не место для хранения больших объёмов справочных материалов — для этого используйте [навык](/cloud/features/ai-ml/agents/builder/skills) или добавьте эти материалы в инструкции агента. И это не механизм извлечения информации из прошлых чатов; эту роль выполняет сама история разговора.
