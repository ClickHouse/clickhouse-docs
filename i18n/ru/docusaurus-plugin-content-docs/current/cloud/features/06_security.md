# Безопасность ClickHouse Cloud

В этом документе описаны параметры безопасности и рекомендуемые практики, доступные для защиты организации и сервисов ClickHouse.
ClickHouse стремится предоставлять безопасные аналитические решения для баз данных, поэтому защита данных и целостности сервисов является для нас приоритетом.
Приведённая здесь информация охватывает различные методы, призванные помочь пользователям обеспечить безопасность своих сред ClickHouse.



## Аутентификация в консоли Cloud {#cloud-console-auth}

### Аутентификация по паролю {#password-auth}

Пароли консоли ClickHouse Cloud соответствуют стандартам NIST 800-63B: минимальная длина — 12 символов, обязательно выполнение 3 из 4 требований к сложности: заглавные буквы, строчные буквы, цифры и/или специальные символы.

Подробнее об [аутентификации по паролю](/cloud/security/manage-my-account#email-and-password).

### Единый вход через социальные сети (SSO) {#social-sso}

ClickHouse Cloud поддерживает аутентификацию через Google или Microsoft для единого входа (SSO).

Подробнее о [социальном SSO](/cloud/security/manage-my-account#social-sso).

### Многофакторная аутентификация {#mfa}

Пользователи, использующие электронную почту и пароль или социальный SSO, могут также настроить многофакторную аутентификацию с помощью приложения-аутентификатора, например Authy или Google Authenticator.

Подробнее о [многофакторной аутентификации](/cloud/security/manage-my-account/#mfa).

### Аутентификация Security Assertion Markup Language (SAML) {#saml-auth}

Корпоративные клиенты могут настроить аутентификацию SAML.

Подробнее об [аутентификации SAML](/cloud/security/saml-setup).

### Аутентификация API {#api-auth}

Клиенты могут настроить ключи API для использования с конечными точками OpenAPI, Terraform и Query API.

Подробнее об [аутентификации API](/cloud/manage/openapi).


## Аутентификация в базе данных {#database-auth}

### Аутентификация по паролю {#db-password-auth}

Пароли пользователей базы данных ClickHouse настроены в соответствии со стандартами NIST 800-63B: минимальная длина — 12 символов, требования к сложности включают прописные буквы, строчные буквы, цифры и/или специальные символы.

Подробнее об [аутентификации по паролю](/cloud/security/manage-database-users#database-user-id--password).

### Аутентификация через Secure Shell (SSH) {#ssh-auth}

Пользователей базы данных ClickHouse можно настроить для использования SSH-аутентификации.

Подробнее об [SSH-аутентификации](/cloud/security/manage-database-users#database-ssh).


## Управление доступом {#access-control}

### Управление доступом на основе ролей в консоли (RBAC) {#console-rbac}

ClickHouse Cloud поддерживает назначение ролей для управления правами доступа на уровне организации, сервиса и базы данных. Права доступа к базе данных при использовании этого метода поддерживаются только в SQL-консоли.

Подробнее об [RBAC в консоли](/cloud/security/console-roles).

### Права пользователей базы данных {#database-user-grants}

Базы данных ClickHouse поддерживают детальное управление правами доступа и ролевую модель доступа через систему предоставления прав пользователям.

Подробнее о [правах пользователей базы данных](/cloud/security/manage-database-users#database-permissions).

### Приватное подключение {#private-connectivity}

Подключайтесь к кластерам ClickHouse из AWS, GCP или Azure через приватное подключение.

Подробнее о [приватном подключении](/cloud/security/connectivity/private-networking).


## Шифрование {#encryption}

### Шифрование на уровне хранилища {#storage-encryption}

ClickHouse Cloud по умолчанию шифрует неактивные данные с использованием ключей AES 256, управляемых облачным провайдером.

Подробнее о [шифровании на уровне хранилища](/cloud/security/cmek#storage-encryption).

### Прозрачное шифрование данных {#tde}

Помимо шифрования на уровне хранилища, клиенты ClickHouse Cloud Enterprise могут включить прозрачное шифрование данных на уровне базы данных для дополнительной защиты.

Подробнее о [прозрачном шифровании данных](/cloud/security/cmek#transparent-data-encryption-tde).

### Ключи шифрования, управляемые клиентом {#cmek}

Клиенты ClickHouse Cloud Enterprise могут использовать собственные ключи для шифрования на уровне базы данных.

Подробнее о [ключах шифрования, управляемых клиентом](/cloud/security/cmek#customer-managed-encryption-keys-cmek).

## Соответствие нормативным требованиям {#compliance}

### Отчеты по безопасности и соответствию {#compliance-reports}

ClickHouse поддерживает надежную программу обеспечения безопасности и соответствия нормативным требованиям. Регулярно проверяйте наличие новых отчетов независимых аудиторов.

Подробнее об [отчетах по безопасности и соответствию](/cloud/security/compliance-overview).

### Сервисы, соответствующие требованиям HIPAA {#hipaa-compliance}

Клиенты ClickHouse Cloud Enterprise могут развертывать сервисы, содержащие защищенную медицинскую информацию (PHI), в регионах, соответствующих требованиям HIPAA, после подписания соглашения о деловом сотрудничестве (BAA).

Подробнее о [соответствии требованиям HIPAA](/cloud/security/compliance/hipaa-onboarding).

### Сервисы, соответствующие требованиям PCI {#pci-compliance}

Клиенты ClickHouse Cloud Enterprise могут развертывать сервисы, содержащие информацию о кредитных картах, в регионах, соответствующих требованиям PCI.

Подробнее о [соответствии требованиям PCI](/cloud/security/compliance/pci-onboarding).
