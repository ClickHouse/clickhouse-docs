Сервисы уровня Scale и Enterprise поддерживают автомасштабирование на основе использования CPU и памяти. Для принятия решений о масштабировании система непрерывно отслеживает использование сервиса за заданный период наблюдения. Если нагрузка поднимается выше или опускается ниже определённых пороговых значений, сервис масштабируется в соответствии с текущим спросом.

## Настройка вертикального автоматического масштабирования \{#configuring-vertical-auto-scaling\}

Участники организации с ролью **Admin** могут настраивать масштабирование сервисов ClickHouse Cloud Scale или Enterprise. Чтобы настроить вертикальное автомасштабирование, перейдите на вкладку **Settings** вашего сервиса и задайте минимальный и максимальный объём памяти, а также параметры CPU, как показано ниже.

:::note
Масштабирование сервисов с одной репликой доступно не для всех уровней.
:::

<Image img={auto_scaling} size="lg" alt="Страница настроек масштабирования" border />

Задайте для реплик значение **Maximum memory** больше, чем **Minimum memory**. После этого сервис будет масштабироваться в этих пределах по мере необходимости. Эти настройки также доступны на этапе первоначального создания сервиса. Для каждой реплики в сервисе будет выделен одинаковый объём памяти и одинаковые ресурсы CPU.

Также можно задать одинаковые значения, фактически «зафиксировав» сервис на определённой конфигурации. В этом случае сервис будет немедленно принудительно масштабирован до выбранного размера.

Важно отметить, что это отключит автомасштабирование в кластере, и сервис не будет защищён от роста потребления CPU или памяти сверх заданных значений.

:::note
Для сервисов уровня Enterprise вертикальное автомасштабирование поддерживается стандартными профилями 1:4. Пользовательские профили не поддерживают ни вертикальное автомасштабирование, ни ручное вертикальное масштабирование. Однако для таких сервисов вертикальное масштабирование можно выполнить через службу поддержки.
:::
