# Уровни ClickHouse Cloud

В ClickHouse Cloud доступно несколько уровней. 
Уровни назначаются на уровне всей организации, поэтому все сервисы внутри одной организации принадлежат одному и тому же уровню.
На этой странице рассматривается, какие уровни подходят для вашего конкретного сценария использования.

## Basic {#basic}

- Экономичный вариант с поддержкой развертывания с одной репликой.
- Подходит для использования отдельными подразделениями с небольшими объемами данных, не требующими строгих гарантий надежности.

:::note
Сервисы уровня Basic имеют фиксированный размер и не поддерживают масштабирование — ни автоматическое, ни ручное.
Для масштабирования сервисов можно перейти на уровень Scale или Enterprise.
:::


## Scale {#scale}

Предназначен для рабочих нагрузок, требующих повышенных SLA (развертывание с 2+ репликами), масштабируемости и расширенной безопасности.

- Поддерживает следующие возможности:
  - [Поддержка частных сетей](/cloud/security/connectivity/private-networking).
  - [Разделение вычислительных ресурсов](../reference/warehouses#what-is-compute-compute-separation).
  - Опции [гибкого масштабирования](/manage/scaling) (вертикальное и горизонтальное).
  - [Настраиваемые резервные копии](/cloud/manage/backups/configurable-backups)


## Enterprise {#enterprise}

Предназначен для крупномасштабных критически важных развертываний с высокими требованиями к безопасности и соответствию нормативным требованиям.

- Все возможности уровня Scale, **плюс**
- Гибкое масштабирование: стандартные профили (`соотношение vCPU:память 1:4`), а также пользовательские профили `HighMemory (соотношение 1:8)` и `HighCPU (соотношение 1:2)`.
- Обеспечивает максимальный уровень гарантий производительности и надежности.
- Поддерживает корпоративный уровень безопасности:
  - Единый вход (SSO)
  - Расширенное шифрование: для сервисов AWS и GCP. Сервисы по умолчанию шифруются нашим ключом и могут быть переведены на ключ клиента для включения управляемых клиентом ключей шифрования (CMEK).
- Позволяет планировать обновления: вы можете выбрать день недели и временное окно для обновлений как базы данных, так и облачных релизов.
- Обеспечивает соответствие стандартам [HIPAA](/cloud/security/compliance-overview#hipaa-since-2024) и PCI.
- Экспортирует резервные копии в учетную запись пользователя.

:::note
Сервисы с одной репликой во всех трех уровнях имеют фиксированный размер (`8 ГиБ`, `12 ГиБ`)
:::


## Переход на другой тарифный план {#upgrading-to-a-different-tier}

Вы можете в любой момент перейти с тарифа Basic на Scale или с Scale на Enterprise. При переходе на более низкий тарифный план потребуется отключить премиум-функции.
