## 3 апреля 2026 г. \{#april-3-2026\}

* **Более интеллектуальное автомасштабирование с рекомендателем на основе двух окон:** ClickHouse Cloud теперь использует для вертикального автомасштабирования подход с двумя окнами ретроспективного анализа: вместо одного 30-часового окна — комбинацию 3-часового «малого окна» и 30-часового «большого окна». Это сокращает задержку scale-down с 30 часов до всего 3 часов, снижая затраты на инфраструктуру для переменных рабочих нагрузок без ущерба для скорости scale-up. Подробнее см. в [статье блога](https://clickhouse.com/blog/smarter-auto-scaling).
* **Мониторинг в консоли Cloud:** Новые обзорные и инфраструктурные дашборды улучшают видимость поведения серверов ClickHouse. Пользователи с правами администратора теперь также получают уведомления по электронной почте о распространенных неполадках при использовании ClickHouse Cloud. Подробнее см. в [статье блога](https://clickhouse.com/blog/clickhouse-cloud-new-monitoring-capabilities).
* **Compute-compute separation (warehouses):** Поддержка автоматического перевода в режим простоя для основных сервисов (также называемых родительскими сервисами) постепенно становится доступной в рамках публичной предварительной версии. Если эта возможность еще не включена для вашей организации, [обратитесь в службу поддержки](https://clickhouse.com/support/program), чтобы получить доступ. Когда функция станет общедоступной, поведение по умолчанию будет таким: для существующих основных сервисов можно будет включить автоматический перевод в режим простоя, а для новых основных сервисов он будет включен по умолчанию.
* **Интеграция с каталогом данных:** ClickHouse Cloud теперь поддерживает Iceberg REST Catalog и Microsoft OneLake в интерфейсе источников данных как интеграции с каталогами озера данных. После подключения каталог отображается как база данных, что позволяет читать таблицы Iceberg напрямую, без дублирования данных.
* **Bring Your Own Cloud (BYOC) на GCP теперь общедоступен:** ClickHouse BYOC теперь имеет статус GA в Google Cloud, что позволяет развертывать сервисы ClickHouse в вашем собственном проекте GCP и соблюдать строгие требования к размещению данных. Подробнее см. в [документации](/cloud/reference/byoc/overview) и [статье блога](https://clickhouse.com/blog/byoc-gcp-ga). Вы можете [связаться с нами](https://clickhouse.com/cloud/bring-your-own-cloud), чтобы запросить доступ.
* **Уведомления о выставлении счетов для предоплаченных кредитов и изменениях способа оплаты:** ClickHouse Cloud теперь отправляет уведомления, когда в вашу организацию добавляются предоплаченные кредиты в рамках контракта с обязательством по расходам или когда обновляется способ оплаты (кредитная карта или подписка через маркетплейс). По умолчанию уведомления включены для UI и электронной почты, а также их можно настроить для отправки в Slack.

## 20 марта 2026 г. \{#march-20-2026\}

* **Пользовательский диапазон дат для Usage Breakdown:** Теперь на экране детализации использования можно просматривать расходы по всем параметрам биллинга за произвольный период.
  Выберите начальную и конечную даты (включительно), чтобы отфильтровать расходы за определенный период и скачать результаты в формате CSV.
  Максимальная длина диапазона дат — 31 день.

## 20 февраля 2026 г. \{#february-20-2026\}

* **ClickPipes:** Обратные частные конечные точки в неактивном состоянии теперь будут автоматически удаляться по истечении заданного периода ожидания. Это гарантирует, что неиспользуемые или неправильно настроенные конечные точки не будут бесконечно сохраняться в бэкенде. Подробнее см. в [документации](/integrations/clickpipes/aws-privatelink#automatic-cleanup) по автоматической очистке.

## 13 февраля 2026 г. \{#february-13-2026\}

* [Коннектор BigQuery](/integrations/clickpipes/bigquery/overview) теперь доступен в закрытой предварительной версии. Подробнее см. в этой [записи блога](https://clickhouse.com/blog/bigquery-clickpipe-private-preview) и запишитесь в [список ожидания](https://clickhouse.com/cloud/clickpipes/bigquery-connector), чтобы получить доступ.
* Мы рады сообщить о поддержке PCI-развертываний для Google Cloud. Поддерживаемые регионы:
  * GCP europe-west4 (Нидерланды)
  * GCP us-central1 (Айова)
  * GCP us-east1 (Южная Каролина)
* Параметры сбора отчетов о сбоях теперь можно настраивать на уровне организации. Ранее этот параметр был доступен только на уровне сервиса. Если отключить его на уровне организации, отправка отчетов о сбоях будет автоматически отключена для всех существующих и будущих сервисов.

## 23 января 2026 г. \{#january-23-2026\}

* ClickPipes теперь доступен в AWS `eu-west-1`. Для новых сервисов ClickHouse Cloud, созданных в этом регионе начиная с 20 января, для ClickPipes будет использоваться тот же регион. Для более старых сервисов ClickPipes по умолчанию использует `eu-central-1`. Подробнее см. в [документации](/integrations/clickpipes#list-of-static-ips).

## 16 января 2026 г. \{#january-16-2026\}

* **Фильтрация расходов на использование на уровне сервиса через API:** Теперь наш API поддерживает фильтрацию расходов на использование в вашей организации по конкретным тегам сервисов, что упрощает анализ затрат с большей детализацией.
* **Интеграция с каталогом данных:** Теперь вы можете подключить внешний каталог данных в качестве источника данных. ClickHouse Cloud поддерживает AWS Glue и Unity Catalog (скоро появится поддержка и других каталогов). После подключения каталог отображается как база данных, что позволяет читать таблицы Iceberg напрямую — без дублирования данных. Для получения доступа обратитесь в службу поддержки.
* **ClickPipes:**
  * Коннектор MongoDB получил статус Public Beta и теперь доступен как новым, так и существующим клиентам ClickHouse Cloud во всех тирах сервиса. Обзор новых возможностей см. в [посте блога](https://clickhouse.com/blog/mongodb-cdc-connector-clickpipes-beta), а чтобы начать работу, перейдите к [документации](/integrations/clickpipes/mongodb).
  * [S3 ClickPipe](/integrations/clickpipes/object-storage/s3/overview) теперь совместим с OVH Object Storage — сервисом Объектного хранилища в OVHcloud с поддержкой API S3.
