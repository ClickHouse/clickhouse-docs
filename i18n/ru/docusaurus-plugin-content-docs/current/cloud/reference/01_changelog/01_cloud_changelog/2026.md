## 3 июня 2026 года \{#2026-06-03\}

Неделя Open House 2026 принесла волну анонсов, посвящённых запуску ClickHouse там, где хранятся ваши данные, а также внедрению в платформу агентных рабочих процессов с поддержкой ИИ.

### ClickPipes теперь работает непосредственно в GCP с Private Service Connect \{#clickpipes-gcp-psc\}

Инфраструктура ClickPipes теперь развертывается в том же регионе GCP, что и ваш сервис ClickHouse Cloud, для всех новых сервисов GCP, созданных после 26 мая, включая поддержку Private Service Connect (PSC). Это устраняет расходы на межоблачный исходящий трафик, снижает задержки, позволяет хранить данные локально и дает возможность подключаться к источникам GCP через приватную сеть GCP вместо обходных решений вроде SSH-туннелирования. Существующие сервисы можно мигрировать при содействии нашей команды.

Подробнее см. в [списке статических IP-адресов ClickPipes](/integrations/clickpipes#list-of-static-ips).

### ClickPipes for GCP Pub/Sub (закрытая предварительная версия) \{#clickpipes-gcp-pubsub\}

Теперь ClickPipes может передавать данные из топиков GCP Pub/Sub напрямую в ClickHouse Cloud, предоставляя командам, работающим в GCP, прямой доступ к аналитике данных из Cloud Logging, Cloud Monitoring, GKE и других источников. Функция доступна через интерфейс ClickPipes, OpenAPI и Terraform; в рамках закрытой предварительной версии требуется [регистрация](https://clickhouse.com/cloud/clickpipes#pubsub-private-preview).

Сведения о настройке и конфигурации см. в [документации ClickPipes for GCP Pub/Sub](/integrations/clickpipes/pubsub).

### ClickHouse Agents: агентная аналитика на базе Claude (публичная бета) \{#clickhouse-agents\}

ClickHouse Agents теперь доступен в публичной бете на [ai.clickhouse.cloud](http://ai.clickhouse.cloud) — это собственная платформа агентов на базе Claude для диалогового анализа данных ClickHouse Cloud с учетом схемы. Создавайте пользовательских агентов в AgentBuilder без написания кода, подключайте MCP-серверы (Slack, Linear, Notion и другие), собирайте рабочие процессы из субагентов и генерируйте запросы, конвейеры и приложения, не выходя из диалога. Также доступны интеграция с AWS Bedrock AgentCore и управляемая платформа LibreChat.

Подробнее см. в [документации ClickHouse Agents](/cloud/features/ai-ml/agents).

### ClickHouse Postgres (публичная бета) \{#clickhouse-postgres-public-beta\}

Полностью управляемый Postgres на локальном NVMe-хранилище теперь доступен в публичной бета-версии: он обеспечивает более чем в 5 раз больше TPS, чем AWS RDS, и объединяет OLTP и OLAP в одном управляемом стеке. Нативный конвейер CDC напрямую передает данные в ClickHouse без какой-либо промежуточной инфраструктуры, а расширение с открытым исходным кодом `pg_clickhouse` позволяет выполнять запросы к таблицам ClickHouse из стандартных сеансов Postgres.

Подробнее — в [Open House 2026 Day 1: данные в реальном времени без привязки к поставщику и что команды смогут создать дальше](https://clickhouse.com/blog/open-house-2026-day-1#postgres-announcements).

### ClickStack Cloud: бессерверная обсервабилити (закрытая предварительная версия) \{#clickstack-cloud\}

ClickStack Cloud — это полностью управляемая бессерверная платформа обсервабилити на базе ClickHouse Cloud. Направьте свой OpenTelemetry Collector на управляемую конечную точку OTLP и сразу же изучайте журналы, метрики и трассировки в интерфейсе ClickStack — при этом ингестия, буферизация, масштабирование, хранение, выполнение запросов, аутентификация и RBAC уже берутся на себя сервисом. Сервис доступен в рамках ограниченной закрытой предварительной версии в AWS us-east-1.

Подробнее см. [ClickStack Cloud в закрытой предварительной версии](https://clickhouse.com/blog/clickstack-cloud-private-preview) и [лист ожидания ClickStack Cloud](https://clickhouse.com/cloud/clickstack-cloud-waitlist).

### ИИ-блокноты для Управляемого ClickStack (бета) \{#clickstack-ai-notebooks\}

ИИ-блокноты предоставляют в Управляемом ClickStack постоянно доступное и прозрачное рабочее пространство для расследования инцидентов с помощью ИИ. Сохраняйте структурированные последовательности промптов, запросов, графиков и рассуждений, создавайте ветвления для проверки нескольких гипотез и делитесь расследованиями в виде совместных командных артефактов — при этом все сгенерированные запросы и результаты остаются видимыми и редактируемыми. Доступны через левую панель навигации в интерфейсе ClickStack.

Подробнее см.:

* [Документация по ИИ-блокнотам ClickStack](/use-cases/observability/clickstack/notebooks)
* [ИИ-блокноты в бета-версии (блог)](https://clickhouse.com/blog/observability-mcp-server-ai-notebooks#ai-notebooks-in-beta)

### Горизонтальное автомасштабирование (закрытая предварительная версия) \{#horizontal-autoscaling\}

Горизонтальное автомасштабирование автоматически регулирует количество реплик в кластере ClickHouse Cloud: при резком росте спроса быстро увеличивает их число, а при снижении нагрузки постепенно сокращает — поэтому командам с переменной рабочей нагрузкой и высоким параллелизмом больше не нужно вручную подбирать размер кластера. В закрытой предварительной версии масштабирование выполняется по числу параллельных запросов; поддержка сигналов CPU и памяти запланирована для публичной беты. Доступно клиентам уровней Enterprise и Scale на ClickHouse 26.2+.

### ClickStack MCP-сервер (с открытым исходным кодом) \{#clickstack-mcp-server\}

Новый MCP-сервер ClickStack с открытым исходным кодом предоставляет внешним ИИ-агентам структурированные инструменты для расследований в области обсервабилити, построенные на ClickStack и ClickHouse. Он предоставляет семантические примитивы для журналов, метрик и трейсов — с поддержкой анализа шаблонов журналов, расследования выбросов в трейсах, корреляционного анализа и кросс-сигнальной отладки — а также может сохранять панели мониторинга, поисковые запросы и оповещения, сохраняя при этом нативный доступ к SQL для более глубокого анализа. Подключайте такие агенты, как Claude Code, Cursor, Codex или пользовательские фреймворки.

Подробнее см. в [документации по ClickStack MCP-серверу](/use-cases/observability/clickstack/mcp) и в [посте в блоге об Observability MCP-сервер + ИИ-блокнотах](https://clickhouse.com/blog/observability-mcp-server-ai-notebooks#clickstack-mcp-server).

### Исполняемые пользовательские функции \{#executable-udfs\}

{/* vale on */ }

Теперь вы можете написать функцию на Python, упаковать её в ZIP-архив, загрузить в свой кластер и вызывать из SQL как любую встроенную функцию. ClickHouse запускает пул долгоживущих изолированных процессов в песочнице и прогоняет через них строки со скоростью выполнения запроса, поэтому вашу функцию можно вызывать везде, где работает SQL: в ad-hoc запросах, JOIN и даже в materialized view, которые срабатывают при каждой вставке. Код вашей модели находится рядом с данными, а её развёртывание сводится к одному экрану загрузки в консоли Cloud.

Чтобы показать, на что это способно, мы подготовили [демо](https://github.com/ClickHouse/stock-anomaly-udf). На основе наших данных о биржевых сделках в реальном времени из StockHouse мы обучили небольшой автоэнкодер на PyTorch, который оценивает тики сделок с акциями на аномалии по мере их приёма. Это один UDF и один оператор DDL, а всё остальное — обычный SQL. Мы также показали предварительную версию UDF с доступом к сети (закрытая бета), которые позволяют UDF выполнять исходящие HTTPS-вызовы. Мы используем это, чтобы получать новости и документы, подаваемые в SEC, а затем просить Claude классифицировать, почему конкретная сделка выглядит аномальной.

Подробнее читайте в [анонсе в блоге](https://clickhouse.com/blog/executable-udfs-clickhouse-cloud-beta) или в [документации](/cloud/features/user-defined-functions)

## 18 мая 2026 г. \{#2026-05-18\}

### Клиент Python v1.0.0 \{#python-client-v1\}

Мажорный релиз [clickhouse-connect](/integrations/python), клиента Python для ClickHouse. Ключевые особенности:

* Полноценный диалект SQLAlchemy с более глубокой поддержкой SQL, специфичного для ClickHouse
* Нативный асинхронный клиент на базе aiohttp с полным паритетом возможностей, включая стриминг
* Существенные улучшения производительности для DateTime, DateTime64, вставки числовых типов фиксированной разрядности, Maps, Decimals и BigDecimals
* Переработана работа с часовыми поясами: добавлены новые параметры `tz_mode` и `tz_source`
* Нативная поддержка записи в столбцы Variant
* Ленивые импорты для необязательных зависимостей (numpy, pandas, pyarrow, polars), ускоряющие простой import в 4 раза
* Экспериментальная поддержка free-threading в Python 3.14

**Несовместимые изменения:** прекращена поддержка Python &lt; 3.9 и Pandas 1.x. Ознакомьтесь с [примечаниями к выпуску](https://github.com/ClickHouse/clickhouse-connect/releases), чтобы узнать о других несовместимых изменениях, связанных с часовыми поясами.

### Построение графиков и оповещений на основе SQL в ClickStack \{#clickstack-sql-charting-alerting\}

[ClickStack](/use-cases/observability/clickstack) теперь поддерживает создание графиков и оповещений с использованием произвольных SQL-запросов ClickHouse. Пользователи могут писать SQL напрямую в конструкторе панели мониторинга и оповещений, с полной поддержкой оконных функций, CTE, многоэтапных запросов и нативных аналитических функций ClickHouse. Оповещения на основе SQL позволяют закодировать всю логику принятия решения прямо в самом запросе, что открывает возможности для обнаружения аномалий, расчета запаздывающих средних и статистического выявления всплесков. Доступно во всех вариантах развертывания ClickStack (Cloud, Private, BYOC, OSS). Подробнее см. в разделах [Визуализации на основе SQL](/use-cases/observability/clickstack/dashboards/sql-visualizations) и [Интерпретация результатов SQL-оповещений](/use-cases/observability/clickstack/alerts#sql-result-interpretation).

## 8 мая 2026 года \{#2026-05-08\}

### Аналитика запросов Postgres (закрытая предварительная версия) \{#postgres-query-insights\}

Новая вкладка **Аналитика запросов** теперь доступна в [ClickHouse Cloud Managed Postgres](/cloud/managed-postgres) (закрытая предварительная версия). Она ранжирует шаблоны запросов по степени влияния и показывает, почему каждый из них работает медленно, в трех разделах:

* **Обзор** — состояние базы данных на одном экране
* **Шаблоны** — таблица с сортировкой по времени выполнения, CPU, ошибкам и P95
* **Выдвижная панель шаблона** — процентили задержки, CPU vs I/O, кэш vs диск, временные выгрузки на диск, нехватка параллельных воркеров и объем WAL

Основано на [pg&#95;stat&#95;ch](https://github.com/ClickHouse/pg_stat_ch), расширении Postgres с открытым исходным кодом.

## 10 апреля 2026 г. \{#2026-04-10\}

### Централизованный биллинг через маркетплейс \{#centralized-marketplace-billing\}

Теперь клиенты могут объединять биллинг нескольких организаций в рамках одной подписки через маркетплейс облачного провайдера (AWS, Azure и GCP). Ключевые обновления:

* **Совместное использование подписки через маркетплейс:** Переведите организацию с оплаты кредитной картой на существующую подписку через маркетплейс другой организации — без необходимости оформлять ещё одну подписку через маркетплейс. Объединяется только потребление по модели PAYG (предоплаченные кредиты не включаются).
* **Обновление подписок через маркетплейс:** Замените подписку через маркетплейс у организации на подписку, которую использует другая организация.
* **Резервный способ оплаты:** Теперь организации, использующие биллинг через маркетплейс, могут добавить кредитную карту в качестве резервного способа оплаты на случай, если основная подписка через маркетплейс будет отменена или истечёт.

Подробнее см. в [документации по биллингу](/manage/manage/billing/managing-payment-methods).

## 3 апреля 2026 г. \{#april-3-2026\}

* **Более интеллектуальное автомасштабирование с рекомендателем на основе двух окон:** ClickHouse Cloud теперь использует для вертикального автомасштабирования подход с двумя окнами ретроспективного анализа: вместо одного 30-часового окна — комбинацию 3-часового «малого окна» и 30-часового «большого окна». Это сокращает задержку scale-down с 30 часов до всего 3 часов, снижая затраты на инфраструктуру для переменных рабочих нагрузок без ущерба для скорости scale-up. Подробнее см. в [статье блога](https://clickhouse.com/blog/smarter-auto-scaling).
* **Мониторинг в консоли Cloud:** Новые обзорные и инфраструктурные дашборды улучшают видимость поведения серверов ClickHouse. Пользователи с правами администратора теперь также получают уведомления по электронной почте о распространенных неполадках при использовании ClickHouse Cloud. Подробнее см. в [статье блога](https://clickhouse.com/blog/clickhouse-cloud-new-monitoring-capabilities).
* **Compute-compute separation (warehouses):** Поддержка автоматического перевода в режим простоя для основных сервисов (также называемых родительскими сервисами) постепенно становится доступной в рамках публичной предварительной версии. Если эта возможность еще не включена для вашей организации, [обратитесь в службу поддержки](https://clickhouse.com/support/program), чтобы получить доступ. Когда функция станет общедоступной, поведение по умолчанию будет таким: для существующих основных сервисов можно будет включить автоматический перевод в режим простоя, а для новых основных сервисов он будет включен по умолчанию.
* **Интеграция с каталогом данных:** ClickHouse Cloud теперь поддерживает Iceberg REST Catalog и Microsoft OneLake в интерфейсе источников данных как интеграции с каталогами озера данных. После подключения каталог отображается как база данных, что позволяет читать таблицы Iceberg напрямую, без дублирования данных.
* **Bring Your Own Cloud (BYOC) на GCP теперь общедоступен:** ClickHouse BYOC теперь имеет статус GA в Google Cloud, что позволяет развертывать сервисы ClickHouse в вашем собственном проекте GCP и соблюдать строгие требования к размещению данных. Подробнее см. в [документации](/cloud/reference/byoc/overview) и [статье блога](https://clickhouse.com/blog/byoc-gcp-ga). Вы можете [связаться с нами](https://clickhouse.com/cloud/bring-your-own-cloud), чтобы запросить доступ.
* **Уведомления о выставлении счетов для предоплаченных кредитов и изменениях способа оплаты:** ClickHouse Cloud теперь отправляет уведомления, когда в вашу организацию добавляются предоплаченные кредиты в рамках контракта с обязательством по расходам или когда обновляется способ оплаты (кредитная карта или подписка через маркетплейс). По умолчанию уведомления включены для UI и электронной почты, а также их можно настроить для отправки в Slack.
