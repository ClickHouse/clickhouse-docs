ClickHouse Cloud предлагает различные каналы релизов, чтобы удовлетворить разные потребности пользователей в части стабильности, доступа к новым функциям и предсказуемости обновлений. У каждого канала свой график обновлений, и он предназначен для разных сценариев использования — как для пользователей, которым нужен мгновенный доступ к новым релизам, так и для тех, кто предпочитает откладывать обновления, чтобы получать наиболее стабильные версии.


## Подробности о каналах релизов \{#release-channel-details\}

<details>
<summary>Подробнее о каналах релизов</summary>

| Channel Name | Description | Key Considerations | Tiers Supported |
| :--- | :--- | :--- | :--- |
| **Fast (Early Release)** | Рекомендуется для непроизводственных сред. Это первый канал релизов для каждого обновления версии базы данных | Приоритет доступа к новым функциям над стабильностью.<br/>Возможность протестировать релизы в непроизводственных средах до обновления production | Basic (по умолчанию)<br/>Тарифы Scale, Enterprise |
| **Regular** | Канал релизов по умолчанию для всех сервисов с несколькими репликами.<br/>Раскатка на этом канале обычно начинается через две недели после запуска канала Fast. | Обновления по умолчанию / на весь парк сервисов.<br/>Сервисы обновляются постепенно в течение нескольких недель | Scale и Enterprise |
| **Slow (Deferred)** | Рекомендуется пользователям, менее готовым к риску, которые хотят, чтобы их сервисы обновлялись ближе к концу графика релизов.<br/>Раскатка на этом канале обычно начинается через две недели после запуска канала Regular. | Максимальная стабильность и предсказуемость.<br/>Предназначен для тех, кому требуется больше тестирования новых релизов на каналах Fast/Regular | Enterprise |

<br/>
<br/>

:::note
Все сервисы с одной репликой автоматически переводятся на канал релизов Fast.
:::

</details>

Окна плановых обновлений доступны во всех каналах релизов для сервисов в тарифе Enterprise. Эта функция позволяет настроить временное окно в заданный день недели для выполнения обновлений.

## График релизов \{#release-schedule\}

:::important Как понимать даты релизов
Даты, показанные ниже, указывают, когда ClickHouse **начинает развёртывание** по каждому каналу релизов, а не когда будет обновлён ваш отдельный сервис.

- Развёртывания автоматизированы и выполняются постепенно в течение нескольких недель
- Сервисы с настроенными запланированными окнами обновления обновляются в своё запланированное окно в течение недели после завершения развёртывания по соответствующему каналу
- Завершение развёртывания может быть отложено из‑за пауз (например, праздничных мораториев на изменения) или в связи с мониторингом состояния

Для предварительного тестирования перед обновлениями production-среды используйте канал Fast или Regular для непроизводственных сервисов и канал Slow для производственных сервисов.
:::
