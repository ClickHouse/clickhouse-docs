## 14 ноября 2025 г. {#november-14-2025}

- Мы рады сообщить, что **ClickHouse Cloud** теперь доступен в **двух новых публичных регионах**:
  - **GCP Japan (asia-northeast1)**
  - **AWS Seoul (Asia Pacific, ap-northeast-2)** — теперь также поддерживается в **ClickPipes**

  Ранее эти регионы были доступны как **приватные регионы**, теперь они **открыты для всех пользователей**.

- Terraform и API теперь поддерживают добавление тегов к сервисам и фильтрацию сервисов по тегам.


## 7 ноября 2025 г. {#november-7-2025}

- Консоль ClickHouse Cloud теперь поддерживает настройку размеров реплик с шагом 1 vCPU, 4 ГиБ.
  Эти параметры доступны как при создании нового сервиса, так и при установке минимального и максимального размеров реплик на странице настроек.
- Пользовательские профили оборудования (доступны на уровне Enterprise) теперь поддерживают режим ожидания.
- ClickHouse Cloud теперь предлагает упрощённый процесс приобретения через AWS Marketplace с отдельными вариантами для [оплаты по факту использования](https://aws.amazon.com/marketplace/pp/prodview-p4gwofrqpkltu?sr=0-2&ref_=beagle&applicationId=AWSMPContessa) и [контрактов с фиксированными затратами](https://aws.amazon.com/marketplace/pp/prodview-4qyeihstyym2s?sr=0-3&ref_=beagle&applicationId=AWSMPContessa).
- Оповещения теперь доступны для пользователей ClickStack в ClickHouse Cloud.
  Пользователи могут создавать и управлять оповещениями непосредственно в интерфейсе HyperDX для логов, метрик и трассировок без дополнительной настройки, дополнительной инфраструктуры или сервисов и без конфигурации. Оповещения интегрируются со Slack, PagerDuty и другими сервисами.
  Для получения дополнительной информации см. [документацию по оповещениям](/use-cases/observability/clickstack/alerts).


## 17 октября 2025 г. {#october-17-2025}

- **Мониторинг сервиса — панель использования ресурсов**  
  Отображение метрик использования CPU и памяти изменится: вместо среднего значения будет показываться максимальное значение использования за определенный период времени, что позволит лучше выявлять случаи недостаточного выделения ресурсов.
  Кроме того, метрика использования CPU будет показывать значение на уровне Kubernetes, которое точнее соответствует метрике, используемой системой автомасштабирования ClickHouse Cloud.
- **Внешние бакеты**  
  ClickHouse Cloud теперь позволяет экспортировать резервные копии напрямую в учетную запись вашего облачного провайдера.
  Подключите внешний бакет — AWS S3, Google Cloud Storage или Azure Blob Storage — и получите полный контроль над управлением резервными копиями.


## 29 августа 2025 г. {#august-29-2025}

- [ClickHouse Cloud Azure Private Link](/cloud/security/azure-privatelink) перешёл с использования фильтров Resource GUID на фильтры Resource ID для идентификации ресурсов. Вы по-прежнему можете использовать устаревший Resource GUID, который обратно совместим, но мы рекомендуем перейти на фильтры Resource ID. Подробности миграции см. в [документации](/cloud/security/azure-privatelink#obtaining-private-endpoint-resourceid) по Azure Private Link.


## 22 августа 2025 г. {#august-22-2025}

- **ClickHouse Connector для AWS Glue**  
  Теперь доступен официальный [ClickHouse Connector для AWS Glue](/integrations/glue) в [AWS Marketplace](https://aws.amazon.com/marketplace/pp/prodview-eqvmuopqzdg7s). Коннектор использует бессерверный движок AWS Glue на основе Apache Spark для извлечения, преобразования и загрузки данных при интеграции ClickHouse с другими источниками данных. Чтобы начать работу, ознакомьтесь с [анонсом в блоге](http://clickhouse.com/blog/clickhouse-connector-aws-glue), где описано, как создавать таблицы, записывать и читать данные между ClickHouse и Spark.
- **Изменение минимального количества реплик в сервисе**  
  Сервисы, которые были масштабированы, теперь можно [уменьшить](/manage/scaling) до одной реплики (ранее минимум составлял 2 реплики). Примечание: сервисы с одной репликой имеют пониженную доступность и не рекомендуются для использования в production-окружении.
- ClickHouse Cloud начнет отправлять уведомления о масштабировании сервисов и обновлении их версий. По умолчанию уведомления отправляются пользователям с ролью администратора. Настроить параметры уведомлений можно в соответствующем разделе настроек.


## 13 августа 2025 г. {#august-13-2025}

- **ClickPipes для MongoDB CDC теперь доступен в закрытом предварительном просмотре**
  Теперь вы можете использовать ClickPipes для репликации данных из MongoDB в ClickHouse Cloud всего за несколько кликов, что обеспечивает
  аналитику в реальном времени без необходимости использования внешних ETL-инструментов. Коннектор поддерживает непрерывную
  репликацию, а также однократную миграцию, и совместим с MongoDB Atlas и самостоятельно развёрнутыми инсталляциями MongoDB.
  Прочитайте [статью в блоге](https://clickhouse.com/blog/mongodb-cdc-clickhouse-preview) с обзором коннектора MongoDB CDC и [зарегистрируйтесь для получения раннего доступа](https://clickhouse.com/cloud/clickpipes/mongodb-cdc-connector)!


## 8 августа 2025 г. {#august-08-2025}

- **Уведомления**: Пользователи теперь будут получать уведомление в интерфейсе при начале обновления их сервиса до новой версии ClickHouse. Дополнительные уведомления по электронной почте и в Slack можно настроить через центр уведомлений.
- **ClickPipes**: В провайдер ClickHouse Terraform добавлена поддержка Azure Blob Storage (ABS) ClickPipes. См. документацию провайдера с примером программного создания ABS ClickPipe.
  - [Исправление ошибки] ClickPipes объектного хранилища, записывающие данные в целевую таблицу с движком Null, теперь отображают метрики «Total records» и «Data ingested» в интерфейсе.
  - [Исправление ошибки] Селектор «Time period» для метрик в интерфейсе по умолчанию устанавливался на «24 hours» независимо от выбранного периода времени. Это исправлено, и интерфейс теперь корректно обновляет графики для выбранного периода.
- **Межрегиональный private link (AWS)** теперь общедоступен (Generally Available). См. [документацию](/manage/security/aws-privatelink) со списком поддерживаемых регионов.


## 31 июля 2025 г. {#july-31-2025}

**Вертикальное масштабирование для ClickPipes теперь доступно**

[Вертикальное масштабирование теперь доступно для потоковых ClickPipes](https://clickhouse.com/blog/clickpipes-flexible-scaling-monitoring).
Эта функция позволяет управлять размером каждой реплики в дополнение к
количеству реплик (горизонтальное масштабирование). Страница сведений для каждого ClickPipe теперь
также включает информацию об использовании ЦП и памяти для каждой реплики, что помогает лучше
понимать ваши рабочие нагрузки и уверенно планировать операции по изменению размера.


## 24 июля 2025 {#july-24-2025}

**ClickPipes для MySQL CDC теперь в публичной бета-версии**

Коннектор MySQL CDC в ClickPipes теперь общедоступен в публичной бета-версии. Всего за несколько кликов
вы можете начать репликацию данных из MySQL (или MariaDB) напрямую в ClickHouse Cloud в реальном времени
без внешних зависимостей. Ознакомьтесь со [статьей в блоге](https://clickhouse.com/blog/mysql-cdc-connector-clickpipes-beta)
с обзором коннектора и следуйте [руководству по быстрому старту](https://clickhouse.com/docs/integrations/clickpipes/mysql),
чтобы начать работу.


## 11 июля 2025 г. {#june-11-2025}

- Новые сервисы теперь хранят метаданные баз данных и таблиц в централизованном каталоге **SharedCatalog** —
  новой модели координации и управления жизненным циклом объектов, которая обеспечивает:
  - **DDL облачного масштаба** даже при высоком уровне конкурентности
  - **Надёжное удаление и новые DDL-операции**
  - **Быстрый запуск и восстановление из режима ожидания**, так как stateless-узлы теперь запускаются без зависимостей от диска
  - **Stateless-вычисления как для нативных, так и для открытых форматов**, включая Iceberg и Delta Lake

  Подробнее о SharedCatalog читайте в нашем [блоге](https://clickhouse.com/blog/clickhouse-cloud-stateless-compute)

- Теперь поддерживается возможность запуска сервисов, соответствующих требованиям HIPAA, в регионе GCP `europe-west4`

## 30 мая 2025 г. {#may-30-2025}

- Мы рады сообщить о выходе **ClickPipes for Postgres CDC** в общий доступ
  в ClickHouse Cloud. Всего за несколько кликов вы можете реплицировать свои базы данных Postgres
  и получить доступ к молниеносной аналитике в реальном времени. Коннектор обеспечивает
  ускоренную синхронизацию данных, задержку всего в несколько секунд, автоматическое применение изменений схемы,
  полностью защищенное соединение и многое другое. Подробнее читайте в
  [блоге](https://clickhouse.com/blog/postgres-cdc-connector-clickpipes-ga).
  Чтобы начать работу, обратитесь к инструкциям [здесь](https://clickhouse.com/docs/integrations/clickpipes/postgres).

- Представлены новые улучшения дашбордов SQL-консоли:
  - Совместный доступ: Вы можете предоставить доступ к своему дашборду членам команды. Поддерживаются четыре уровня доступа, которые можно настроить как глобально, так и для каждого пользователя:
    - _Доступ на запись_: добавление/редактирование визуализаций, настройка обновления, взаимодействие с дашбордами через фильтры.
    - _Владелец_: предоставление доступа к дашборду, удаление дашборда и все остальные права пользователя с «доступом на запись».
    - _Доступ только для чтения_: просмотр и взаимодействие с дашбордом через фильтры.
    - _Нет доступа_: невозможность просмотра дашборда.
  - Для существующих дашбордов администраторы организации могут назначить себя владельцами.
  - Теперь вы можете добавить таблицу или диаграмму из SQL-консоли на дашборд прямо из представления запроса.

- Мы набираем участников для предварительного тестирования [Distributed cache](https://clickhouse.com/cloud/distributed-cache-waitlist)
  для AWS и GCP. Подробнее читайте в [блоге](https://clickhouse.com/blog/building-a-distributed-cache-for-s3).


## 16 мая 2025 г. {#may-16-2025}

- Представлена панель мониторинга использования ресурсов, которая предоставляет обзор
  ресурсов, используемых сервисом в ClickHouse Cloud. Следующие метрики
  извлекаются из системных таблиц и отображаются на этой панели:
  - Память и CPU: Графики для `CGroupMemoryTotal` (выделенная память), `CGroupMaxCPU` (выделенный CPU),
    `MemoryResident` (используемая память) и `ProfileEvent_OSCPUVirtualTimeMicroseconds` (использованный CPU)
  - Передача данных: Графики, показывающие входящий и исходящий трафик данных из ClickHouse Cloud. Подробнее [здесь](/cloud/manage/network-data-transfer).
- Мы рады объявить о запуске нового дополнения ClickHouse Cloud Prometheus/Grafana,
  созданного для упрощения мониторинга ваших сервисов ClickHouse Cloud.
  Это дополнение использует наш совместимый с Prometheus API-эндпоинт для бесшовной интеграции
  метрик ClickHouse в вашу существующую инфраструктуру Prometheus и Grafana. Оно включает
  предварительно настроенную панель мониторинга, которая обеспечивает видимость состояния
  и производительности ваших сервисов в режиме реального времени. Подробнее читайте в [блоге](https://clickhouse.com/blog/monitor-with-new-prometheus-grafana-mix-in) о запуске.


## 18 апреля 2025 г. {#april-18-2025}

- Введена новая роль уровня организации **Member** и две новые роли уровня сервиса:
  **Service Admin** и **Service Read Only**.
  **Member** — это роль уровня организации, которая по умолчанию назначается пользователям SAML SSO
  и предоставляет только возможности входа в систему и обновления профиля. Роли **Service Admin**
  и **Service Read Only** для одного или нескольких сервисов могут быть назначены пользователям
  с ролями **Member**, **Developer** или **Billing Admin**. Подробнее см.
  в разделе [«Управление доступом в ClickHouse Cloud»](https://clickhouse.com/docs/cloud/security/cloud-access-management/overview)
- ClickHouse Cloud теперь предлагает сервисы **HIPAA** и **PCI** в следующих регионах
  для клиентов тарифа **Enterprise**: AWS eu-central-1, AWS eu-west-2, AWS us-east-2.
- Введены **уведомления для пользователей ClickPipes**. Эта функция отправляет
  автоматические оповещения о сбоях ClickPipes по электронной почте, через интерфейс ClickHouse Cloud и
  Slack. Уведомления по электронной почте и через интерфейс включены по умолчанию и могут быть
  настроены для каждого пайпа. Для **Postgres CDC ClickPipes** оповещения также включают
  пороговые значения слотов репликации (настраиваются на вкладке **Settings**), конкретные типы ошибок
  и инструкции по самостоятельному устранению сбоев.
- Открыт **закрытый предварительный просмотр MySQL CDC**. Это позволяет клиентам реплицировать базы данных MySQL
  в ClickHouse Cloud в несколько кликов, обеспечивая быструю аналитику и
  устраняя необходимость во внешних инструментах ETL. Коннектор поддерживает как непрерывную
  репликацию, так и однократные миграции, независимо от того, находится ли MySQL в облаке (RDS,
  Aurora, Cloud SQL, Azure и т. д.) или в локальной инфраструктуре. Вы можете зарегистрироваться для участия в закрытом
  предварительном просмотре, [перейдя по этой ссылке](https://clickhouse.com/cloud/clickpipes/mysql-cdc-connector).
- Введена поддержка **AWS PrivateLink для ClickPipes**. Вы можете использовать AWS PrivateLink для
  установления защищенного соединения между VPC, сервисами AWS, вашими локальными
  системами и ClickHouse Cloud. Это можно сделать без передачи трафика через
  публичный интернет при перемещении данных из таких источников, как Postgres, MySQL и
  MSK на AWS. Также поддерживается межрегиональный доступ через конечные точки сервисов VPC.
  Настройка подключения PrivateLink теперь [полностью самостоятельная](https://clickhouse.com/docs/integrations/clickpipes/aws-privatelink)
  через ClickPipes.


## 4 апреля 2025 г. {#april-4-2025}

- Пользователи, работающие с сервисами Production и Development, теперь будут видеть в своих счетах стоимость использования ClickPipes и передачи данных.

### Сервисы с одной репликой {#single-replica-services}

Мы представляем концепцию «сервиса с одной репликой» как в качестве отдельного предложения, так и в составе хранилищ. В качестве отдельного предложения сервисы с одной репликой имеют ограничение по размеру и предназначены для небольших тестовых рабочих нагрузок. В составе хранилищ сервисы с одной репликой могут развертываться в больших размерах и использоваться для рабочих нагрузок, не требующих высокой доступности в масштабе, таких как перезапускаемые задания ETL.

### Улучшения вертикального автомасштабирования {#vertical-auto-scaling-improvements}

Мы представляем новый механизм вертикального масштабирования для вычислительных реплик, который мы называем «Make Before Break» (MBB). Этот подход добавляет одну или несколько реплик нового размера перед удалением старых реплик, предотвращая потерю мощности во время операций масштабирования. Устраняя разрыв между удалением существующих реплик и добавлением новых, MBB обеспечивает более плавный и менее разрушительный процесс масштабирования. Это особенно полезно в сценариях увеличения масштаба, где высокая утилизация ресурсов вызывает потребность в дополнительной мощности, поскольку преждевременное удаление реплик только усугубило бы ограничения ресурсов.

### Горизонтальное масштабирование (общая доступность) {#horizontal-scaling-ga}

Горизонтальное масштабирование теперь находится в общей доступности. Пользователи могут добавлять дополнительные реплики для масштабирования своего сервиса через API и облачную консоль. Для получения информации обратитесь к [документации](/manage/scaling#manual-horizontal-scaling).

### Изменения надёжности {#reliability-changes}

Чтобы защитить целостность данных наших пользователей, сохранив при этом основную функциональность, мы скорректировали работу этой возможности. В частности, настройка MergeTree `clean_deleted_rows` теперь устарела и больше не действует. Ключевое слово `CLEANUP` по умолчанию запрещено (для его использования необходимо включить `allow_experimental_replacing_merge_with_cleanup`). Если вы решите использовать `CLEANUP`, необходимо убедиться, что он всегда применяется вместе с `FINAL`, и вы должны гарантировать, что после выполнения `OPTIMIZE FINAL CLEANUP` не будут вставлены строки с более старыми версиями.

Этот релиз исключает единицы чтения и записи из модели ценообразования (подробности см. на [странице с ценами](https://clickhouse.com/pricing)), обновляет версию ClickHouse до 22.10, добавляет поддержку более высокого вертикального масштабирования для клиентов с самообслуживанием и повышает надежность благодаря улучшенным настройкам по умолчанию.

### Общие изменения {#general-changes-6}

- Исключены единицы чтения/записи из модели ценообразования

### Изменения конфигурации {#configuration-changes-2}

- Настройки `allow_suspicious_low_cardinality_types`, `allow_suspicious_fixed_string_types` и `allow_suspicious_codecs` (по умолчанию false) больше не могут изменяться пользователями в целях обеспечения стабильности.

### Изменения консоли {#console-changes-31}

- Увеличен максимальный лимит для вертикального масштабирования в режиме самообслуживания до 720 ГБ памяти для платных клиентов
- Улучшен процесс восстановления из резервной копии для настройки правил списка IP-доступа и пароля
- Добавлены списки ожидания для GCP и Azure в диалоговом окне создания сервиса
- Улучшена обработка ошибок при загрузке файлов
- Улучшены процессы администрирования биллинга

### Обновление до версии ClickHouse 22.10 {#clickhouse-2210-version-upgrade}

- Улучшены слияния поверх объектных хранилищ за счет смягчения порога «слишком много частей» при наличии множества крупных частей (не менее 10 ГиБ). Это позволяет хранить до петабайтов данных в одной партиции одной таблицы.
- Улучшен контроль над слиянием с помощью настройки `min_age_to_force_merge_seconds` для выполнения слияния по истечении определенного временного порога.
- Добавлен MySQL-совместимый синтаксис для сброса настроек `SET setting_name = DEFAULT`.
- Добавлены функции для кодирования кривой Мортона, хеширования целых чисел Java и генерации случайных чисел.
- См. [подробный список изменений 22.10](/whats-new/cloud#clickhouse-2210-version-upgrade) для ознакомления с полным перечнем изменений.


## 25 октября 2022 г. {#october-25-2022}

Этот релиз значительно снижает потребление вычислительных ресурсов для небольших рабочих нагрузок, снижает стоимость вычислений (подробности см. на странице [тарифов](https://clickhouse.com/pricing)), повышает стабильность за счет улучшенных настроек по умолчанию и расширяет возможности представлений Billing и Usage в консоли ClickHouse Cloud.

### Общие изменения {#general-changes-7}

- Минимальный объем выделяемой памяти для сервиса уменьшен до 24 ГБ
- Таймаут простоя сервиса уменьшен с 30 минут до 5 минут

### Изменения конфигурации {#configuration-changes-3}

- Значение max_parts_in_total уменьшено со 100 тыс. до 10 тыс. Значение по умолчанию параметра `max_parts_in_total` для таблиц MergeTree снижено со 100 000 до 10 000. Причина этого изменения в том, что мы обнаружили: большое количество частей данных может приводить к медленному запуску сервисов в облаке. Большое количество частей обычно указывает на выбор слишком детализированного ключа партиционирования, что, как правило, происходит случайно и чего следует избегать. Изменение значения по умолчанию позволит выявлять такие случаи на более ранней стадии.

### Изменения консоли {#console-changes-32}

- Расширена детализация использования кредитов в представлении Billing для пробных пользователей
- Улучшены всплывающие подсказки и справочный текст, добавлена ссылка на страницу с тарифами в представлении Usage
- Улучшен процесс переключения параметров фильтрации IP-адресов
- Добавлена кнопка повторной отправки подтверждения по электронной почте в облачную консоль
