## Роли организации {#organization-roles}

Инструкции по назначению ролей организации см. в разделе [Управление пользователями облака](/cloud/security/manage-cloud-users).

В ClickHouse доступны четыре роли уровня организации для управления пользователями. Только роль администратора имеет доступ к сервисам по умолчанию. Все остальные роли должны быть объединены с ролями уровня сервиса для взаимодействия с сервисами.

| Роль      | Описание                                                                                                                                                                                                                 |
| --------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Admin     | Выполнение всех административных действий в организации и управление всеми настройками. Эта роль назначается первому пользователю организации по умолчанию и автоматически предоставляет права администратора сервиса для всех сервисов. |
| Developer | Доступ на просмотр организации и возможность генерировать ключи API с такими же или более низкими правами.                                                                                                                        |
| Billing   | Просмотр данных об использовании и счетов, а также управление способами оплаты.                                                                                                                                                                        |
| Member    | Только вход в систему с возможностью управления настройками личного профиля. Назначается пользователям SAML SSO по умолчанию.                                                                                                                   |


## Роли сервиса {#service-roles}

Инструкции по назначению ролей сервиса см. в разделе [Управление пользователями облака](/cloud/security/manage-cloud-users).

Разрешения сервиса должны быть явно предоставлены администратором пользователям с ролями, отличными от роли администратора. Роль Service Admin предварительно настроена с правами администратора консоли SQL, но может быть изменена для сокращения или удаления разрешений.

| Роль              | Описание                 |
| ----------------- | --------------------------- |
| Service read only | Просмотр сервисов и настроек. |
| Service admin     | Управление настройками сервиса.    |


## Роли SQL-консоли {#sql-console-roles}

Инструкции по назначению ролей SQL-консоли см. в разделе [Управление назначением ролей SQL-консоли](/cloud/guides/sql-console/manage-sql-console-role-assignments).

| Роль                  | Описание                                                                                       |
| --------------------- | ---------------------------------------------------------------------------------------------- |
| SQL console read only | Доступ только для чтения к базам данных в рамках сервиса.                                     |
| SQL console admin     | Административный доступ к базам данных в рамках сервиса, эквивалентный роли базы данных по умолчанию. |
