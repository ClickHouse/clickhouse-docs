# Соответствие биллинга ClickHouse Cloud нормативным требованиям \{#clickhouse-cloud-billing-compliance\}

## Соответствие требованиям к выставлению счетов \{#billing-compliance\}

Для использования ClickHouse Cloud в вашей организации должен быть настроен
активный и действующий способ оплаты. По завершении 30‑дневного пробного периода
или исчерпании пробных кредитов (в зависимости от того, что произойдет раньше),
у вас есть следующие варианты оплаты для продолжения использования ClickHouse Cloud:

| Вариант оплаты                                      | Описание                                                                                 |
|-----------------------------------------------------|------------------------------------------------------------------------------------------|
| [Прямая оплата по мере потребления (Direct PAYG)](#direct-payg)                          | Добавьте действительную банковскую карту в учетную запись организации для оплаты по мере потребления |
| [Оплата по мере потребления через маркетплейс (Marketplace PAYG)](#cloud-marketplace-payg) | Настройте подписку с оплатой по мере потребления через поддерживаемого провайдера облачного маркетплейса |
| [Контракт с обязательствами по расходам](#committed-spend-contract) | Заключите контракт с обязательствами по расходам напрямую или через поддерживаемый облачный маркетплейс |

Если ваш пробный период завершится и ни один вариант оплаты не будет настроен
для вашей организации, все ваши сервисы будут остановлены. Если способ оплаты
по‑прежнему не будет настроен в течение двух недель, все ваши данные будут удалены.

ClickHouse выставляет счета за сервисы на уровне организации. Если мы по какой‑либо
причине не сможем обработать платеж с использованием вашего текущего способа
оплаты, вы должны обновить его до одного из трех вариантов, перечисленных выше,
чтобы избежать перерыва в работе сервисов. См. ниже дополнительные сведения о
соответствии требованиям к оплате в зависимости от выбранного способа оплаты.

### Оплата по мере потребления с использованием банковской карты \{#direct-payg\}

Вы можете оплачивать использование ClickHouse Cloud ежемесячно постфактум с
помощью банковской карты. Чтобы добавить банковскую карту, выполните следующие
[инструкции](#add-credit-card).

Ваш ежемесячный расчетный период в ClickHouse начинается в день выбора уровня
организации (Basic, Scale или Enterprise) и создания первого сервиса в рамках
организации.

Банковская карта, привязанная к учетной записи, обычно будет списана в конце
вашего ежемесячного расчетного периода, но списание может быть ускорено, если
сумма, начисленная в течение периода, достигнет 10 000 долл. США (подробная
информация о порогах оплаты приведена [здесь](/cloud/billing/payment-thresholds)).

Банковская карта, привязанная к учетной записи, должна быть действительной,
не просроченной и иметь достаточный доступный лимит для покрытия общей суммы счета.
Если по какой‑либо причине нам не удастся списать всю сумму, к вам немедленно
будут применены следующие ограничения, связанные с неоплаченным счетом:

* Вы можете масштабировать только до 120 GiB на реплику
* Вы не можете запускать свои сервисы, если они остановлены
* Вы не можете запускать или создавать новые сервисы

Мы будем пытаться обработать платеж с использованием настроенного в организации
способа оплаты в течение 30 дней. Если платеж не будет успешно выполнен в
течение 14 дней, все сервисы в организации будут остановлены. Если по истечении
30‑дневного периода оплата по‑прежнему не будет получена и мы не предоставили
продление, все данные и сервисы, связанные с вашей организацией, будут удалены.

### Оплата по мере потребления через облачный маркетплейс \{#cloud-marketplace-payg\}

Оплата по мере потребления также может быть настроена таким образом, чтобы списание
для организации производилось через один из наших поддерживаемых облачных
маркетплейсов (AWS, GCP или Azure). Чтобы подключить оплату Marketplace PAYG,
выполните следующие [инструкции](#marketplace-payg).

Аналогично Direct PAYG, ваш ежемесячный расчетный период в ClickHouse
при использовании Marketplace PAYG начинается в день выбора уровня организации
(Basic, Scale или Enterprise) и создания первого сервиса в рамках
организации.

Однако в связи с требованиями маркетплейсов мы передаем данные о начислениях
за использование по модели оплаты по мере потребления с почасовой детализацией.
Счет вам будет выставлен в соответствии с условиями вашего соглашения с этим
маркетплейсом — обычно на основе календарного месяца.

Например, если вы создаете первый сервис организации 18 января, ваш первый
расчетный период использования в ClickHouse Cloud будет длиться с 18 января до
конца дня 17 февраля. Однако первый счет от облачного маркетплейса вы можете
получить в начале февраля.

Если же ваша подписка PAYG в маркетплейсе будет отменена или не продлена
автоматически, выставление счетов будет осуществляться по банковской карте,
указанной для организации, если таковая есть. Чтобы добавить банковскую карту,
[свяжитесь со службой поддержки](/about-us/support) за помощью. Если действительная
банковская карта не была предоставлена, к вам будут применены те же ограничения,
связанные с неоплаченным счетом, что и для [Direct PAYG](#direct-payg), включая
приостановку сервисов и последующее удаление данных.

### Выставление счетов по контракту с обязательствами по расходам \{#committed-spend-contract\}

Вы можете приобрести кредиты для вашей организации по контракту с обязательствами
по расходам, выполнив следующие действия:

1. Свяжитесь с отделом продаж, чтобы купить кредиты напрямую, с вариантами оплаты,
   включающими ACH или банковский перевод. Условия оплаты будут указаны в
   соответствующей форме заказа.
2. Свяжитесь с отделом продаж, чтобы купить кредиты по подписке через один из
   поддерживаемых нами облачных маркетплейсов (AWS, GCP или Azure). Суммы начислений
   будут переданы в соответствующий маркетплейс после принятия приватного предложения
   и далее в соответствии с условиями этого предложения, но выставление счетов будет
   происходить в соответствии с условиями вашего договора с этим маркетплейсом.
   Чтобы платить через маркетплейс, выполните эти
   [инструкции](#marketplace-payg).

Кредиты, применённые к организации (например, в рамках фиксированных контрактов или
возвратов), доступны для использования в течение срока, указанного в форме заказа
или принятом приватном предложении.
Кредиты начинают расходоваться с дня их предоставления, в расчётных периодах,
основанных на дате выбора первого тарифного уровня организации (Basic, Scale или Enterprise).

Если организация **не** находится на фиксированном контракте облачного
маркетплейса и у неё заканчиваются кредиты или истекает срок их действия,
организация автоматически переключается на оплату по мере использования (Pay-As-You-Go, PAYG).
В этом случае мы попытаемся обработать платёж с помощью привязанной к организации
кредитной карты, если таковая имеется.

Если организация **находится** на фиксированном контракте облачного маркетплейса
и у неё заканчиваются кредиты, она также автоматически переключится на оплату
по мере использования (PAYG) через тот же маркетплейс на оставшийся срок подписки.
Однако если подписка не продлевается и истекает, мы затем попытаемся обработать
платёж с помощью привязанной к организации кредитной карты, если таковая имеется.

В любом из этих случаев, если нам не удаётся списать средства с настроенной
кредитной карты, будут применяться ограничения по неоплаченным счетам, описанные
выше для [оплаты по мере использования (PAYG)](#direct-payg) кредитной картой — это
включает приостановку обслуживания.
Дополнительные сведения о переходе с фиксированного контракта на оплату по мере
использования (PAYG) см. в разделе «Overconsumption» в наших
[Условиях и положениях](https://clickhouse.com/legal/agreements/terms-of-service).
Однако для клиентов с фиксированным контрактом мы свяжемся с вами по поводу любых
неоплаченных счетов до начала удаления данных. Данные не удаляются автоматически
по истечении какого-либо периода времени.

Если вы хотите добавить дополнительные кредиты до того, как ваши текущие истекут
или будут израсходованы, [свяжитесь с нами](https://clickhouse.com/company/contact).

## Как оплачивать через маркетплейсы \{#marketplace-payg\}

Если вы хотите оплачивать услуги через один из поддерживаемых нами маркетплейсов (AWS, GCP или Azure),
воспользуйтесь инструкцией [здесь](/cloud/marketplace/marketplace-billing).
По любым вопросам, связанным с биллингом в облачном маркетплейсе, обращайтесь напрямую к вашему облачному провайдеру.

Полезные ссылки для решения проблем, связанных с биллингом в маркетплейсе:
* [Часто задаваемые вопросы по биллингу AWS](https://aws.amazon.com/aws-cost-management/aws-billing/faqs/)
* [Часто задаваемые вопросы по биллингу GCP](https://cloud.google.com/compute/docs/billing-questions)
* [Часто задаваемые вопросы по биллингу Azure](https://learn.microsoft.com/en-us/azure/cost-management-billing/cost-management-billing-faq)
