# Соответствие биллинга ClickHouse Cloud требованиям



## Соблюдение требований к оплате {#billing-compliance}

Для использования ClickHouse Cloud в вашей организации должен быть настроен активный и
действующий способ оплаты. После окончания 30-дневного пробного периода или исчерпания пробных
кредитов (в зависимости от того, что произойдет раньше) для продолжения использования ClickHouse Cloud доступны следующие варианты
оплаты:

| Вариант оплаты                                        | Описание                                                                                |
| ----------------------------------------------------- | --------------------------------------------------------------------------------------- |
| [Прямая оплата по факту использования](#direct-payg)  | Добавьте действующую кредитную карту в вашу организацию для оплаты по факту использования |
| [Оплата по факту использования через маркетплейс](#cloud-marketplace-payg) | Настройте подписку с оплатой по факту использования через поддерживаемого провайдера облачного маркетплейса |
| [Контракт с фиксированными расходами](#committed-spend-contract) | Заключите контракт с фиксированными расходами напрямую или через поддерживаемый облачный маркетплейс |

Если пробный период закончится, а способ оплаты для вашей организации не настроен,
все ваши сервисы будут остановлены. Если способ оплаты не будет
настроен в течение двух недель, все ваши данные будут удалены.

ClickHouse взимает плату за сервисы на уровне организации. Если мы не сможем
обработать платеж с использованием вашего текущего способа оплаты, вы должны обновить его на один
из трех вариантов, перечисленных выше, чтобы избежать прерывания обслуживания. Ниже приведены подробные
сведения о соблюдении требований к оплате в зависимости от выбранного способа оплаты.

### Оплата по факту использования с помощью кредитной карты {#direct-payg}

Вы можете оплачивать использование ClickHouse Cloud ежемесячно постфактум с помощью кредитной карты.
Чтобы добавить кредитную карту, следуйте этим [инструкциям](#add-credit-card).

Ваш ежемесячный расчетный период для ClickHouse начинается в день выбора уровня организации
(Basic, Scale или Enterprise) и создания первого сервиса в
организации.

С кредитной карты, указанной в системе, обычно будут списываться средства в конце вашего ежемесячного
расчетного периода, но списание будет ускорено, если сумма задолженности
в течение цикла достигнет $10 000 USD (подробнее о пороговых значениях платежей [здесь](/cloud/billing/payment-thresholds)).

Кредитная карта, указанная в системе, должна быть действующей, не просроченной и иметь достаточный доступный
кредитный лимит для покрытия общей суммы счета. Если по какой-либо причине мы не сможем списать
полную сумму задолженности, немедленно будут применены следующие ограничения из-за неоплаченного счета:

- Вы сможете масштабировать только до 120 ГиБ на реплику
- Вы не сможете запустить свои сервисы, если они остановлены
- Вы не сможете запускать или создавать новые сервисы

Мы будем пытаться обработать платеж с использованием настроенного способа оплаты
организации в течение 30 дней. Если платеж не будет успешным через 14 дней, все сервисы
в организации будут остановлены. Если платеж все еще не будет получен к
концу этого 30-дневного периода и мы не предоставим продление, все данные и
сервисы, связанные с вашей организацией, будут удалены.

### Оплата по факту использования через облачный маркетплейс {#cloud-marketplace-payg}

Оплата по факту использования также может быть настроена для списания средств с организации через один из поддерживаемых облачных маркетплейсов
(AWS, GCP или Azure). Чтобы зарегистрироваться для оплаты через маркетплейс по факту использования, следуйте этим
[инструкциям](#marketplace-payg).

Аналогично прямой оплате по факту использования, ваш ежемесячный расчетный период с ClickHouse
при использовании оплаты через маркетплейс начинается в день выбора уровня организации (Basic, Scale
или Enterprise) и создания первого сервиса в
организации.

Однако из-за требований маркетплейсов мы передаем информацию о начислениях
за ваше использование с оплатой по факту на почасовой основе. Обратите внимание, что вам будет
выставлен счет в соответствии с условиями вашего соглашения с этим маркетплейсом — обычно
по календарному месячному расчетному циклу.

Например, если вы создадите свой первый сервис организации 18 января, ваш
первый расчетный период использования в ClickHouse Cloud будет длиться с 18 января до
конца дня 17 февраля. Однако вы можете получить свой первый счет от
облачного маркетплейса в начале февраля.

Однако если ваша подписка на маркетплейсе с оплатой по факту использования будет отменена или не продлится
автоматически, оплата будет переведена на кредитную карту, указанную в системе для
организации, если таковая имеется. Чтобы добавить кредитную карту, пожалуйста, [обратитесь в службу поддержки](/about-us/support)
за помощью. Если действующая кредитная карта не была предоставлена, будут применены те же ограничения из-за неоплаченного счета,
описанные выше для [прямой оплаты по факту использования](#direct-payg) — это
включает приостановку сервиса и последующее удаление данных.

### Оплата по контракту с фиксированными расходами {#committed-spend-contract}

Вы можете приобрести кредиты для вашей организации через контракт с фиксированными расходами следующими способами:


1. Обращение в отдел продаж для прямой покупки кредитов с вариантами оплаты, включая ACH
   или банковский перевод. Условия оплаты будут указаны в соответствующей форме заказа.
2. Обращение в отдел продаж для покупки кредитов через подписку на одной из поддерживаемых
   облачных площадок (AWS, GCP или Azure). Платежи будут переданы на соответствующую
   площадку после принятия частного предложения и далее в соответствии
   с условиями предложения, но счета будут выставляться согласно условиям вашего
   соглашения с этой площадкой. Для оплаты через площадку следуйте этим
   [инструкциям](#marketplace-payg).

Кредиты, применённые к организации (например, через договоры с обязательствами или возвраты средств),
доступны для использования в течение срока, указанного в форме заказа или принятом частном
предложении.
Кредиты расходуются начиная с дня их предоставления в расчётных периодах,
основанных на дате выбора первого уровня организации (Basic, Scale или Enterprise).

Если организация **не** имеет договора с обязательствами на облачной площадке и исчерпывает
кредиты или срок их действия истекает, организация автоматически переключится
на тарификацию Pay-As-You-Go (PAYG). В этом случае мы попытаемся обработать платёж
с использованием кредитной карты, привязанной к организации, если таковая имеется.

Если организация **имеет** договор с обязательствами на облачной площадке и исчерпывает
кредиты, она также автоматически переключится на тарификацию PAYG через ту же
площадку на оставшийся срок подписки. Однако если подписка
не будет продлена и истечёт, мы затем попытаемся обработать платёж с использованием
кредитной карты, привязанной к организации, если таковая имеется.

В любом из этих сценариев, если мы не сможем списать средства с настроенной кредитной карты,
будут применены ограничения для неоплаченных счетов, описанные выше для тарификации
[Pay-as-you-go (PAYG)](#direct-payg) с кредитной картой — это включает приостановку услуг.
Для получения дополнительной информации о переходе с договора с обязательствами на тарификацию PAYG обратитесь к разделу «Overconsumption» в наших [Условиях использования](https://clickhouse.com/legal/agreements/terms-of-service).
Однако для клиентов с договорами с обязательствами мы свяжемся с вами по поводу любых
неоплаченных счетов перед началом удаления данных. Данные не удаляются автоматически
по истечении какого-либо периода времени.

Если вы хотите добавить дополнительные кредиты до истечения срока действия или исчерпания существующих,
[свяжитесь с нами](https://clickhouse.com/company/contact).

### Как оплатить с помощью кредитной карты {#add-credit-card}

Перейдите в раздел «Billing» в интерфейсе ClickHouse Cloud и нажмите кнопку «Add Credit Card»
(показана ниже), чтобы завершить настройку. Если у вас возникнут вопросы,
[обратитесь в службу поддержки](/about-us/support) за помощью.

## Как оплачивать через маркетплейсы {#marketplace-payg}

Если вы хотите оплачивать через один из поддерживаемых маркетплейсов (AWS, GCP или Azure),
следуйте инструкциям, приведенным [здесь](/cloud/marketplace/marketplace-billing).
По вопросам, связанным с выставлением счетов через облачные маркетплейсы,
обращайтесь напрямую к поставщику облачных услуг.

Полезные ссылки для решения проблем с выставлением счетов через маркетплейсы:

- [Часто задаваемые вопросы по биллингу AWS](https://aws.amazon.com/aws-cost-management/aws-billing/faqs/)
- [Часто задаваемые вопросы по биллингу GCP](https://cloud.google.com/compute/docs/billing-questions)
- [Часто задаваемые вопросы по биллингу Azure](https://learn.microsoft.com/en-us/azure/cost-management-billing/cost-management-billing-faq)
