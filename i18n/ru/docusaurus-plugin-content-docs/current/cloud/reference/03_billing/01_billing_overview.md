## Часто задаваемые вопросы {#faqs}

### Что такое кредит ClickHouse (CHC)? {#what-is-chc}

Кредит ClickHouse — это единица кредита для использования ClickHouse Cloud, равная одному (1) доллару США, которая применяется на основе действующего прайс-листа ClickHouse.

:::note
Если вы получаете счета через Stripe, то увидите, что 1 CHC равен $0,01 USD в вашем счёте Stripe. Это сделано для обеспечения точного выставления счетов в Stripe из-за их ограничения на выставление счетов за дробные количества нашего стандартного SKU 1 CHC = $1 USD.
:::

### Где можно найти устаревшие цены? {#find-legacy-pricing}

Информацию об устаревших ценах можно найти [здесь](https://clickhouse.com/pricing?legacy=true).

### Как измеряются вычислительные ресурсы? {#how-is-compute-metered}

ClickHouse Cloud измеряет вычислительные ресурсы поминутно с шагом 8 ГБ оперативной памяти.
Стоимость вычислительных ресурсов варьируется в зависимости от уровня обслуживания, региона и облачного провайдера.

### Как рассчитывается дисковое хранилище? {#how-is-storage-on-disk-calculated}

ClickHouse Cloud использует облачное объектное хранилище, и использование измеряется по сжатому размеру данных, хранящихся в таблицах ClickHouse.
Стоимость хранилища одинакова для всех уровней обслуживания и варьируется в зависимости от региона и облачного провайдера.

### Учитываются ли резервные копии в общем объёме хранилища? {#do-backups-count-toward-total-storage}

Хранилище и резервные копии учитываются в стоимости хранилища и оплачиваются отдельно.
Для всех сервисов по умолчанию создаётся одна резервная копия, которая хранится в течение суток.
Пользователи, которым требуются дополнительные резервные копии, могут настроить их, добавив дополнительные [резервные копии](/cloud/manage/backups/overview) на вкладке настроек консоли Cloud.

### Как оценить степень сжатия? {#how-do-i-estimate-compression}

Степень сжатия может варьироваться от набора данных к набору данных.
Насколько она варьируется, зависит от того, насколько сжимаемы данные изначально (количество полей с высокой и низкой кардинальностью),
и от того, как пользователь настраивает схему (например, использует ли дополнительные кодеки).
Для распространённых типов аналитических данных она может составлять порядка 10x, но также может быть значительно ниже или выше.
См. [документацию по оптимизации](/optimize/asynchronous-inserts) для получения рекомендаций и этот [блог Uber](https://www.uber.com/blog/logging/) для подробного примера использования в логировании.
Единственный практический способ узнать точно — загрузить ваш набор данных в ClickHouse и сравнить размер исходного набора данных с размером, хранящимся в ClickHouse.

Вы можете использовать запрос:

```sql title="Оценка степени сжатия"
SELECT formatReadableSize(total_bytes)
FROM system.tables
WHERE name = <имя_вашей_таблицы>
```

### Какие инструменты предлагает ClickHouse для оценки стоимости запуска сервиса в облаке при наличии самостоятельно управляемого развёртывания? {#what-tools-does-clickhouse-offer-to-estimate-the-cost-of-running-a-service-in-the-cloud-if-i-have-a-self-managed-deployment}

Журнал запросов ClickHouse фиксирует [ключевые метрики](/operations/system-tables/query_log), которые можно использовать для оценки стоимости выполнения рабочей нагрузки в ClickHouse Cloud.
Подробную информацию о миграции с самостоятельно управляемого развёртывания на ClickHouse Cloud см. в [документации по миграции](/cloud/migration/clickhouse-to-cloud), а при возникновении дополнительных вопросов обратитесь в [службу поддержки ClickHouse Cloud](https://console.clickhouse.cloud/support).

### Какие варианты оплаты доступны для ClickHouse Cloud? {#what-billing-options-are-available-for-clickhouse-cloud}

ClickHouse Cloud поддерживает следующие варианты оплаты:

- Самообслуживание с ежемесячной оплатой (в USD через кредитную карту).
- Прямые продажи с годовой/многолетней оплатой (через предоплаченные «кредиты ClickHouse» в USD с дополнительными вариантами оплаты).
- Через маркетплейсы AWS, GCP и Azure (либо с оплатой по факту использования (PAYG), либо с заключением контракта с ClickHouse Cloud через маркетплейс).

:::note
Кредиты ClickHouse Cloud для PAYG выставляются в счетах единицами по $0,01, что позволяет взимать плату с клиентов за частичные кредиты ClickHouse на основе их использования. Это отличается от кредитов ClickHouse с фиксированными расходами, которые приобретаются заранее целыми единицами по $1.
:::

### Могу ли я удалить свою кредитную карту? {#can-i-delete-my-credit-card}

Вы не можете удалить кредитную карту в интерфейсе выставления счетов, но можете обновить её в любое время. Это помогает гарантировать, что у вашей организации всегда есть действующий способ оплаты. Если вам необходимо удалить кредитную карту, обратитесь в [службу поддержки ClickHouse Cloud](https://console.clickhouse.cloud/support) за помощью.

### Какова продолжительность расчётного периода? {#how-long-is-the-billing-cycle}

Выставление счетов осуществляется по месячному расчётному периоду, и дата начала отслеживается как дата создания организации ClickHouse Cloud.


### Если у меня есть активная подписка PAYG на маркетплейсе, а затем я заключаю контракт с обязательствами, будут ли мои кредиты по обязательствам расходоваться в первую очередь? {#committed-credits-consumed-first-with-active-payg-subscription}

Да. Использование оплачивается следующими способами оплаты в указанном порядке:

- Кредиты по обязательствам (предоплаченные)
- Подписка на маркетплейсе (PAYG)
- Кредитная карта

### Какие инструменты управления затратами предлагает ClickHouse Cloud для сервисов Scale и Enterprise? {#what-controls-does-clickhouse-cloud-offer-to-manage-costs-for-scale-and-enterprise-services}

- Клиенты Trial и Annual Commit автоматически получают уведомления по электронной почте при достижении определенных порогов потребления: `50%`, `75%` и `90%`. Это позволяет пользователям проактивно управлять своим использованием.
- ClickHouse Cloud позволяет пользователям устанавливать максимальный лимит автомасштабирования для вычислительных ресурсов через [Расширенное управление масштабированием](/manage/scaling), что является значительным фактором затрат для аналитических рабочих нагрузок.
- [Расширенное управление масштабированием](/manage/scaling) позволяет устанавливать лимиты памяти с возможностью управления поведением приостановки/простоя во время неактивности.

### Какие инструменты управления затратами предлагает ClickHouse Cloud для сервисов Basic? {#what-controls-does-clickhouse-cloud-offer-to-manage-costs-for-basic-services}

- [Расширенное управление масштабированием](/manage/scaling) позволяет управлять поведением приостановки/простоя во время неактивности. Настройка выделения памяти не поддерживается для сервисов Basic.
- Обратите внимание, что по умолчанию сервис приостанавливается после периода неактивности.

### Если у меня несколько сервисов, я получу счет на каждый сервис или консолидированный счет? {#if-i-have-multiple-services-do-i-get-an-invoice-per-service-or-a-consolidated-invoice}

Консолидированный счет формируется для всех сервисов в данной организации за расчетный период.

### Если я добавлю свою кредитную карту и перейду на платную версию до истечения пробного периода и кредитов, будет ли с меня взиматься плата? {#if-i-add-my-credit-card-and-upgrade-before-my-trial-period-and-credits-expire-will-i-be-charged}

Когда пользователь переходит с пробной версии на платную до окончания 30-дневного пробного периода, но при этом остаются кредиты из пробного лимита,
мы продолжаем расходовать пробные кредиты в течение первоначального 30-дневного пробного периода, а затем списываем средства с кредитной карты.

### Как я могу отслеживать свои расходы? {#how-can-i-keep-track-of-my-spending}

Консоль ClickHouse Cloud предоставляет отображение использования с детализацией по каждому сервису. Эта разбивка, организованная по параметрам использования, помогает понять затраты, связанные с каждой измеряемой единицей.

### Как получить доступ к счетам за подписку на сервис ClickHouse Cloud? {#how-do-i-access-my-invoice-for-my-subscription-to-the-clickhouse-cloud-service}

Для прямых подписок с использованием кредитной карты:

Чтобы просмотреть свои счета, выберите свою организацию на левой панели навигации в интерфейсе ClickHouse Cloud, затем перейдите в раздел Billing. Все ваши счета будут перечислены в разделе Invoices.

Для подписок через облачный маркетплейс:

Все подписки на маркетплейсе выставляются и оплачиваются маркетплейсом. Вы можете просмотреть свой счет непосредственно через маркетплейс соответствующего облачного провайдера.

### Почему даты в отчетах об использовании не совпадают с датами в счете от маркетплейса? {#why-do-the-dates-on-the-usage-statements-not-match-my-marketplace-invoice}

Выставление счетов AWS Marketplace следует циклу календарного месяца.
Например, для использования в период с 01 декабря 2024 года по 01 января 2025 года
счет формируется в период с 3 по 5 января 2025 года.

Отчеты об использовании ClickHouse Cloud следуют другому расчетному циклу, где использование измеряется
и отчитывается за 30 дней, начиная со дня регистрации.

Даты использования и счетов будут различаться, если эти даты не совпадают. Поскольку отчеты об использовании отслеживают использование по дням для данного сервиса, пользователи могут полагаться на отчеты для просмотра детализации затрат.

### Существуют ли какие-либо ограничения на использование предоплаченных кредитов? {#are-there-any-restrictions-around-the-usage-of-prepaid-credits}

Предоплаченные кредиты ClickHouse Cloud (будь то напрямую через ClickHouse или через маркетплейс облачного провайдера)
могут использоваться только в рамках условий контракта.
Это означает, что они могут быть применены на дату принятия или на будущую дату, но не за какие-либо предыдущие периоды.
Любые превышения, не покрытые предоплаченными кредитами, должны быть оплачены кредитной картой или через ежемесячное выставление счетов маркетплейса.

### Есть ли разница в ценах на ClickHouse Cloud при оплате через маркетплейс облачного провайдера или напрямую ClickHouse? {#is-there-a-difference-in-clickhouse-cloud-pricing-whether-paying-through-the-cloud-provider-marketplace-or-directly-to-clickhouse}


Нет разницы в ценообразовании между оплатой через маркетплейс и прямой регистрацией в ClickHouse.
В любом случае использование ClickHouse Cloud отслеживается в единицах ClickHouse Cloud Credits (CHCs),
которые учитываются одинаково и тарифицируются соответствующим образом.

### Как тарифицируется разделение вычислительных ресурсов? {#how-is-compute-compute-separation-billed}

При создании сервиса в дополнение к существующему
вы можете выбрать, должен ли новый сервис использовать те же данные, что и существующий.
Если да, то эти два сервиса образуют [хранилище](/cloud/reference/warehouses).
Хранилище содержит данные, к которым обращаются несколько вычислительных сервисов.

Поскольку данные хранятся только один раз, вы платите только за одну копию данных, хотя к ним обращаются несколько сервисов.
Вы платите за вычислительные ресурсы как обычно — дополнительных сборов за разделение вычислительных ресурсов / хранилища нет.
Используя общее хранилище в такой конфигурации, пользователи получают экономию как на хранении, так и на резервных копиях.

Разделение вычислительных ресурсов может сэкономить значительное количество ClickHouse Credits в некоторых случаях.
Хороший пример — следующая конфигурация:

1. У вас есть ETL-задачи, которые работают круглосуточно и загружают данные в сервис. Эти ETL-задачи не требуют большого объема памяти, поэтому могут выполняться на небольшом экземпляре с, например, 32 ГиБ оперативной памяти.

2. Специалист по данным из той же команды, у которого есть требования к нерегулярной отчетности, говорит, что ему нужно выполнить запрос, требующий значительного объема памяти — 236 ГиБ, однако ему не нужна высокая доступность, и он может подождать и повторно выполнить запросы в случае сбоя первого выполнения.

В этом примере вы, как администратор базы данных, можете сделать следующее:

1. Создать небольшой сервис с двумя репликами по 16 ГиБ каждая — это удовлетворит требования ETL-задач и обеспечит высокую доступность.

2. Для специалиста по данным вы можете создать второй сервис в том же хранилище с одной репликой на 236 ГиБ. Вы можете включить режим простоя для этого сервиса, чтобы не платить за него, когда специалист по данным его не использует.

Оценка стоимости (в месяц) для этого примера на уровне **Scale Tier**:

- Родительский сервис активен 24 часа в сутки: 2 реплики x 16 ГиБ 4 vCPU на реплику
- Дочерний сервис: 1 реплика x 236 ГиБ 59 vCPU на реплику
- 3 ТБ сжатых данных + 1 резервная копия
- 100 ГБ исходящей передачи данных через публичный интернет
- 50 ГБ межрегиональной передачи данных
