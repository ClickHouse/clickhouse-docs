ClickHouse Cloud измеряет объем входящего и исходящего трафика данных.
Сюда входит любой трафик данных в ClickHouse Cloud и из него, а также любой
трафик данных внутри региона и между регионами. Это потребление ресурсов
отслеживается на уровне сервиса. На основе этих данных клиентам
начисляются сборы за передачу данных, которые затем добавляются к их
ежемесячному счету.

ClickHouse Cloud взимает плату за:

* Исходящий трафик данных из ClickHouse Cloud в публичный интернет, включая в
  другие регионы других облачных провайдеров.
* Исходящий трафик данных в другой регион у того же облачного провайдера.

Плата не взимается за передачу данных внутри региона или использование и
передачу данных через Private Link/Private Service Connect. Однако мы
оставляем за собой право внедрить дополнительные параметры ценообразования
для передачи данных, если мы увидим модели использования, влияющие на нашу
возможность корректно тарифицировать пользователей.

Примечание: сборы облачного провайдера (Cloud Service Provider, CSP)
взимаются при использовании Private Link/Private Service Connect. Пожалуйста,
обратитесь к странице тарифов CSP для получения самой актуальной информации.

Сборы за передачу данных зависят от облачного провайдера (Cloud Service
Provider, CSP) и региона. Стоимость исходящего трафика в публичный интернет
определяется только исходным регионом. Стоимость межрегионального (или
кросс-регионального) трафика зависит как от исходного, так и от конечного
региона.

**Рекомендации по снижению затрат на передачу данных**

Существуют некоторые шаблоны использования, которые стоит учитывать при
входящей и исходящей передаче данных, чтобы минимизировать затраты на
передачу данных.

1. При входящей или исходящей передаче данных из ClickHouse Cloud по
   возможности используйте сжатие, чтобы минимизировать объем передаваемых
   данных и связанные с этим расходы.

2. Имейте в виду, что при выполнении `INSERT` по нативному протоколу с
   неинлайновыми значениями (например, `INSERT INTO [TABLE] FROM INFILE [FILE] FORMAT NATIVE`) клиенты ClickHouse забирают метаданные с серверов для
   упаковки данных. Если размер метаданных больше, чем полезная нагрузка
   самого `INSERT`, вы можете, как ни парадоксально, увидеть больше
   исходящего трафика, чем входящего, с точки зрения сервера. Если это
   неприемлемо, рассмотрите возможность инлайнинга данных с помощью
   синтаксиса `VALUES` или использования HTTP-протокола.

Таблицы ниже показывают, как сборы за исходящую передачу данных различаются
между публичным интернетом и кросс-региональной передачей в зависимости от
облачного провайдера и региона.

:::note
ClickHouse Cloud измеряет использование межрегиональной передачи по уровням,
с 1-го по 4-й, в зависимости от исходного и конечного регионов. Таблица ниже
показывает уровень для каждой комбинации межрегиональной передачи данных.
На экране Billing usage в ClickHouse Cloud вы увидите использование передачи
данных, разбитое по уровням.
:::
