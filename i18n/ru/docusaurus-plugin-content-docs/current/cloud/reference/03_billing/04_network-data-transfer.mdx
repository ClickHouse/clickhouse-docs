ClickHouse Cloud измеряет объем входящего и исходящего трафика данных.
Сюда включаются любые данные, поступающие в ClickHouse Cloud и исходящие из него, а также
объем передачи данных внутри региона и между регионами. Это использование
отслеживается на уровне сервиса. На основе этих данных для клиентов
рассчитываются платежи за передачу, которые затем добавляются к их
ежемесячному счету.

ClickHouse Cloud взимает плату за:

* Исходящий трафик из ClickHouse Cloud в публичный Интернет, включая в другие
  регионы других облачных провайдеров.
* Исходящий трафик в другой регион того же облачного провайдера.

Передача данных внутри региона, а также использование и передача данных через
Private Link/Private Service Connect не тарифицируются. Однако мы оставляем за
собой право ввести дополнительные параметры ценообразования на передачу данных,
если мы увидим модели использования, влияющие на нашу способность корректно
выставлять счета пользователям.

Примечание: облачный провайдер (Cloud Service Provider, CSP) взимает плату за
использование Private Link/Private Service Connect. Пожалуйста, обратитесь к
странице с ценами соответствующего CSP, чтобы получить самую актуальную
информацию.

Плата за передачу данных зависит от облачного провайдера (Cloud Service Provider,
CSP) и региона. Стоимость исходящего трафика в публичный Интернет зависит только
от исходного региона. Стоимость межрегионального (или кросс-регионального)
трафика зависит как от исходного, так и от целевого регионов.

**Рекомендации по минимизации затрат на передачу данных**

Существуют некоторые типовые подходы, которые следует учитывать при организации
входящего и исходящего трафика, чтобы минимизировать затраты на передачу данных.

1. При передаче данных в ClickHouse Cloud или из него по возможности используйте
   сжатие, чтобы уменьшить объем передаваемых данных и связанные с этим расходы.

2. Имейте в виду, что при выполнении `INSERT` по нативному протоколу с
   неинлайновыми значениями (например, `INSERT INTO [TABLE] FROM INFILE [FILE] FORMAT NATIVE`)
   клиенты ClickHouse получают метаданные с серверов для упаковки данных. Если
   метаданные больше, чем полезная нагрузка `INSERT`, вы можете, вопреки
   ожиданиям, увидеть больший объем исходящего трафика, чем входящего, с точки
   зрения сервера. Если это неприемлемо, рассмотрите возможность инлайнинга
   данных с помощью синтаксиса `VALUES` или использования протокола HTTP.

:::
