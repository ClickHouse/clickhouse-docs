ClickHouse Cloud учитывает объем входящего и исходящего трафика.
Это включает любые данные, поступающие в ClickHouse Cloud и выходящие из него, а также
трафик внутри региона и между регионами. Такой трафик отслеживается на уровне сервиса.
На основе этих данных клиентам начисляются платежи за передачу, которые затем
добавляются к их ежемесячному счету.

ClickHouse Cloud взимает плату за:

* Исходящий трафик из ClickHouse Cloud в публичный интернет, включая другие
  регионы других облачных провайдеров.
* Исходящий трафик в другой регион у того же облачного провайдера.

Плата не взимается за передачу данных внутри региона, а также за использование
и передачу данных через Private Link/Private Service Connect. Однако мы
оставляем за собой право ввести дополнительные параметры ценообразования
передачи данных, если увидим модели использования, которые влияют на нашу
возможность корректно тарифицировать пользователей.

Примечание: Плата облачного провайдера (Cloud Service Provider, CSP) применяется при использовании Private Link/Private Service Connect. Пожалуйста, обратитесь к странице с тарифами CSP для получения самой актуальной информации.

Тарифы на передачу данных различаются в зависимости от облачного провайдера
(Cloud Service Provider, CSP) и региона. Стоимость исходящего трафика в
публичный интернет определяется только исходным регионом. Тарифы на передачу
между регионами (или кросс-региональную) зависят как от исходного, так и от
целевого регионов.

**Рекомендации по снижению затрат на передачу данных**

Существуют некоторые типовые сценарии, которые следует учитывать при приёме
(ingress) и передаче (egress) данных, чтобы минимизировать расходы на передачу.

1. При приёме или передаче данных в ClickHouse Cloud и из него по возможности
   используйте сжатие, чтобы уменьшить объем передаваемых данных и связанные с
   этим расходы.

2. Имейте в виду, что при выполнении `INSERT` по нативному протоколу с
   неинлайновыми значениями (например, `INSERT INTO [TABLE] FROM INFILE [FILE] FORMAT NATIVE`)
   клиенты ClickHouse запрашивают у серверов метаданные для упаковки данных.
   Если метаданные больше, чем полезная нагрузка `INSERT`, вы можете, вопреки
   ожиданиям, увидеть больший объем исходящего трафика, чем входящего, с точки
   зрения сервера. Если это неприемлемо, рассмотрите возможность инлайнинга
   данных с помощью синтаксиса `VALUES` или использования протокола HTTP.

передачи, а также от облачного провайдера и региона.

данных, разбитое по уровням.
:::
