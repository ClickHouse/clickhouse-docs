ClickHouse Cloud учитывает объем входящего и исходящего трафика данных.
Сюда входит любой трафик в ClickHouse Cloud и из него, а также любой трафик
в пределах региона и между регионами. Это потребление отслеживается на уровне
сервиса. На основе этого потребления клиентам начисляется плата за передачу
данных, которая затем добавляется к их ежемесячному счету.

ClickHouse Cloud взимает плату за:

* Исходящий трафик из ClickHouse Cloud в публичный интернет, включая трафик
  в другие регионы других облачных провайдеров.
* Исходящий трафик в другой регион у того же облачного провайдера.

Плата за передачу данных в пределах одного региона, а также за использование и
передачу данных через Private Link/Private Service Connect не взимается. Однако
мы оставляем за собой право ввести дополнительные измерения тарифов на передачу
данных, если мы увидим сценарии использования, влияющие на нашу способность
корректно тарифицировать пользователей.

Примечание: Облачный провайдер (Cloud Service Provider, CSP) взимает плату за использование Private Link/Private Service Connect. Пожалуйста, обращайтесь к странице тарифов CSP для получения самой актуальной информации.

Плата за передачу данных зависит от облачного провайдера (Cloud Service Provider, CSP)
и региона. Стоимость исходящего трафика в публичный интернет зависит только от
региона-источника. Стоимость передачи между регионами (или кросс-региональной
передачи) зависит как от региона-источника, так и от региона-получателя.

**Рекомендации по снижению расходов на передачу данных**

Существует несколько сценариев использования, о которых стоит помнить при
организации входящего и исходящего трафика, чтобы минимизировать расходы
на передачу данных.

1. При организации входящего или исходящего трафика для ClickHouse Cloud
   по возможности используйте сжатие, чтобы уменьшить объем передаваемых
   данных и связанные с этим расходы.

2. Имейте в виду, что при выполнении `INSERT` по нативному протоколу с
   неинлайновыми значениями (например, `INSERT INTO [TABLE] FROM INFILE [FILE] FORMAT NATIVE`) клиенты ClickHouse получают метаданные с серверов
   для упаковки данных. Если метаданные больше, чем полезная нагрузка `INSERT`,
   вы можете, как ни парадоксально, наблюдать больший исходящий трафик, чем
   входящий, с точки зрения сервера. Если это неприемлемо, рассмотрите
   возможность инлайнинга данных с помощью синтаксиса `VALUES` или использования
   протокола HTTP.

Таблицы ниже показывают, как плата за передачу данных при исходящем трафике
меняется при выходе в публичный интернет или при кросс-региональной передаче
в зависимости от облачного провайдера и региона.

:::note
ClickHouse Cloud учитывает межрегиональное потребление в разрезе уровней — от

ниже показывает уровень для каждой комбинации межрегиональной передачи данных.
На экране Billing usage в ClickHouse Cloud вы увидите использование передачи
данных, разбитое по уровням.
:::
