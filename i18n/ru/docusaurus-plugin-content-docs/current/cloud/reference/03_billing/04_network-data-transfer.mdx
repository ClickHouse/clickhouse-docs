ClickHouse Cloud учитывает данные, переданные на вход и на выход. Это включает в себя любые данные, входящие и выходящие из ClickHouse Cloud, а также любые внутренние и межрегиональные данные. Это использование отслеживается на уровне сервиса. На основе этого использования клиенты несут расходы на передачу данных, которые затем добавляются к их ежемесячному счёту.

ClickHouse Cloud взимает плату за:
- Исходящие данные из ClickHouse Cloud в публичный Интернет, включая другие регионы других облачных провайдеров.
- Исходящие данные в другой регион у того же облачного провайдера.

Плата не взимается за внутреннюю передачу данных или использование Private Link/Private Service Connect и передачу данных. Однако мы оставляем за собой право реализовать дополнительные измерения цен на передачу данных, если увидим схемы использования, которые повлияют на нашу способность правильно взимать плату с пользователей.

Примечание: Плата облачного сервиса (CSP) применяется за использование Private Link/Private Service Connect. Пожалуйста, обратитесь к странице с ценами CSP для получения актуальной информации.

Расходы на передачу данных варьируются в зависимости от облачного сервиса (CSP) и региона. 
Цены на исходящие данные в публичный интернет основаны только на исходном регионе. 
Цены на межрегиональную (или перекрестную) передачу зависят как от исходного, так и от пункта назначения.

**Лучшие практики для минимизации расходов на передачу данных**

Есть некоторые схемы, которые следует учитывать при входящей и исходящей передаче данных, чтобы минимизировать расходы на передачу данных.

1. При входящей или исходящей передаче данных из Clickhouse Cloud используйте сжатие, где это возможно, чтобы уменьшить объем передаваемых данных и связанные с этим затраты.

2. Имейте в виду, что при выполнении INSERT через нативный протокол с неинлайновыми значениями (например, `INSERT INTO [TABLE] FROM INFILE [FILE] FORMAT NATIVE`), клиенты ClickHouse извлекают метаданные с серверов, чтобы упаковать данные. Если метаданные больше, чем нагрузка `INSERT`, вы можете наблюдать более высокий объем исходящих данных, чем входящих с точки зрения сервера. Если это неприемлемо, рассмотрите возможность инлайнинга данных с использованием синтаксиса `VALUES` или использования HTTP-протокола.

:::
