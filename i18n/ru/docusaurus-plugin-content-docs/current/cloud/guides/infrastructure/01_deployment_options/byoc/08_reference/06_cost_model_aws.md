Развертывание ClickHouse BYOC предполагает два независимых счета:

1. **Начисления ClickHouse Cloud** — ClickHouse выставляет их за ваши сервисы ClickHouse исходя из общего объема выделенной памяти.
2. **Расходы на инфраструктуру AWS** — AWS выставляет их напрямую вашей учетной записи AWS за каждый ресурс, который развертывание BYOC там создает.

На этой странице описано, как рассчитывается каждая из этих составляющих и как вместе они образуют совокупную стоимость владения (TCO).

## Начисление ClickHouse Cloud \{#clickhouse-cloud-charges\}

Начисление ClickHouse Cloud рассчитывается на основе общего объема выделенной памяти. [Свяжитесь с командой](https://clickhouse.com/cloud/bring-your-own-cloud), чтобы узнать, как это применяется к вашей конфигурации.

## Расходы на инфраструктуру AWS \{#aws-infrastructure-charges\}

AWS напрямую выставляет счета вашему аккаунту за каждый ресурс, развернутый через BYOC. ClickHouse не делает наценку и не перепродает ресурсы AWS. Полный перечень обязательных и необязательных сервисов см. в разделе [Тарифицируемые сервисы AWS](/cloud/reference/byoc/billable-aws-services).

Основные статьи расходов, обычно в порядке убывания вклада в счет BYOC:

1. **Amazon EC2** — экземпляры воркеров, на которых работают управляемые группы узлов EKS. По умолчанию используются стандартные семейства Graviton (например, `m7g`). Семейство и количество масштабируются в зависимости от объема памяти, выделенного вашему сервису, и автоскейлинга группы узлов.
2. **Amazon S3** — хранение данных таблиц ClickHouse и резервных копий в ваших бакетах. Тарификация включает плату за GB-месяц, а также за запросы и межрегиональную передачу данных.
3. **Amazon EBS** — тома gp3, подключенные к узлам-воркерам для ОС, образов контейнеров и журналов ClickHouse.
4. **NAT Gateway и межзонная передача данных** — исходящий трафик из приватных подсетей, а также трафик между зонами доступности (в развертываниях multi-AZ данные реплицируются между AZ).
5. **Amazon EKS** — фиксированная почасовая плата за control plane кластера.
6. **Elastic Load Balancing (NLB)** — тарификация за LCU-час для входящего клиентского трафика.
7. **CloudWatch Logs, Route 53, KMS, конечные точки VPC** — как правило, составляют небольшую часть общего счета, но зависят от рабочей нагрузки.

Актуальные цены AWS см. на страницах с тарифами отдельных сервисов на [aws.amazon.com](https://aws.amazon.com/pricing/).
