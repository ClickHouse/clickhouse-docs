Хотя ClickHouse известен своей скоростью и надёжностью, оптимальная производительность 
достигается в пределах определённых рабочих параметров. Например, слишком большое количество таблиц,
баз данных или кусков (parts) может негативно влиять на производительность. Чтобы этого избежать, ClickHouse
Cloud устанавливает ограничения по нескольким операционным измерениям. 
Подробности этих ограничений приведены ниже.

:::tip
Если вы упёрлись в одно из этих ограничений, возможно, вы реализуете свой сценарий
использования неоптимальным образом. Свяжитесь с нашей службой поддержки, и 
мы с радостью поможем вам доработать сценарий, чтобы избежать превышения ограничений, 
или вместе разберёмся, как мы можем увеличить их контролируемым образом. 
:::

| Dimension                     | Limit                                                      |
|-------------------------------|------------------------------------------------------------|
| **Databases**                 | 1000                                                       |
| **Tables**                    | 5000                                                       |
| **Columns**                   | ∼1000 (предпочтителен широкий формат, а не компактный)     |
| **Partitions**                | 50k                                                        |
| **Parts**                     | 100k на весь инстанс                                       |
| **Part size**                 | 150 ГБ                                                     |
| **Services per organization** | 20 (мягкий лимит)                                          |
| **Services per warehouse**    | 5 (мягкий лимит)                                           |
| **Replicas per service**      | 20 (мягкий лимит)                                          |  
| **Low cardinality**           | 10k или меньше                                             |
| **Primary keys in a table**   | 4–5, которые достаточно эффективно отфильтровывают данные |
| **Query concurrency**         | 1000 (на реплику)                                          |
| **Batch ingest**              | всё &gt; 1M будет системой разбито на блоки по 1M строк    |

:::note
Для сервисов с одной репликой максимальное количество баз данных ограничено 
100, а максимальное количество таблиц — 500. Дополнительно объём хранилища
для сервисов уровня Basic ограничен 1 ТБ.
:::
