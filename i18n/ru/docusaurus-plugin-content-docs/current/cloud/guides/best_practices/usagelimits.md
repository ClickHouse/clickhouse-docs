Хотя ClickHouse известен своей скоростью и надёжностью, оптимальная производительность
достигается в пределах определённых рабочих параметров. Например, слишком большое
количество таблиц, баз данных или партиций может негативно сказаться на производительности.
Чтобы этого избежать, ClickHouse Cloud применяет лимиты по нескольким операционным
параметрам. Подробная информация об этих ограничениях приведена ниже.

:::tip
Если вы столкнулись с одним из этих ограничений, возможно, ваш сценарий использования
реализован неоптимальным образом. Свяжитесь с нашей службой поддержки, и мы с
удовольствием поможем вам оптимизировать сценарий использования, чтобы избежать превышения
ограничений, либо вместе рассмотрим, как мы можем контролируемо их увеличить.
:::

| Измерение                     | Лимит                                                      |
|-------------------------------|------------------------------------------------------------|
| **Базы данных**               | 1000                                                       |
| **Таблицы**                   | 5000                                                       |
| **Столбцы**                   | ∼1000 (широкий формат предпочтительнее компактного)       |
| **Партиции**                  | 50 тыс.                                                    |
| **Части**                     | 100 тыс. на весь экземпляр сервиса                        |
| **Размер части**              | 150 ГБ                                                     |
| **Сервисов на организацию**   | 20 (мягкий лимит)                                          |
| **Сервисов на хранилище**     | 5 (мягкий лимит)                                           |
| **Реплик на сервис**          | 20 (мягкий лимит)                                          |  
| **Низкая кардинальность**     | 10 тыс. или меньше                                         |
| **Первичные ключи в таблице** | 4–5, которые достаточно сужают объём данных               |
| **Параллелизм запросов**      | 1000 (на реплику)                                          |
| **Пакетный приём**            | всё, что &gt; 1 млн строк, будет системой разбито на блоки по 1 млн строк |

:::note
Для сервисов с одной репликой максимальное количество баз данных ограничено
100, а максимальное количество таблиц — 500. Кроме того, объём
хранилища для сервисов уровня Basic ограничен 1 ТБ.
:::
