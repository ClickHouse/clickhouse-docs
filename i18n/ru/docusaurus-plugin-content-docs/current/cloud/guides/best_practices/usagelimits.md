Хотя ClickHouse известен своей скоростью и надёжностью, оптимальная производительность 
достигается при работе в определённых параметрах. Например, слишком большое количество таблиц,
баз данных или частей (parts) может негативно сказаться на производительности. Чтобы избежать этого,
ClickHouse Cloud применяет ограничения по нескольким операционным параметрам. 
Подробная информация об этих ограничениях приведена ниже.

:::tip
Если вы упёрлись в одно из этих ограничений, возможно, ваш сценарий использования
реализован не оптимальным образом. Свяжитесь с нашей службой поддержки, и мы с удовольствием поможем
вам скорректировать сценарий использования, чтобы избежать превышения ограничений,
или вместе рассмотрим, как можно контролируемым образом увеличить эти значения. 
:::

| Dimension                     | Limit                                                      |
|-------------------------------|------------------------------------------------------------|
| **Databases**                 | 1000                                                       |
| **Tables**                    | 5000                                                       |
| **Columns**                   | ∼1000 (предпочтителен широкий формат, а не компактный)     |
| **Partitions**                | 50k                                                        |
| **Parts**                     | 100k на весь инстанс                                       |
| **Part size**                 | 150gb                                                      |
| **Services per organization** | 20 (мягкий лимит)                                          |
| **Services per warehouse**    | 5 (мягкий лимит)                                           |
| **Replicas per service**      | 20 (мягкий лимит)                                          |  
| **Low cardinality**           | 10k или меньше                                             |
| **Primary keys in a table**   | 4–5, которые достаточно эффективно отфильтровывают данные |
| **Query concurrency**         | 1000 (на реплику)                                          |
| **Batch ingest**              | всё &gt; 1M будет системой разбито на блоки по 1M строк    |

:::note
Для сервисов с одной репликой максимальное количество баз данных ограничено 
значением 100, а максимальное количество таблиц — 500. Кроме того, объём хранилища
для сервисов базового уровня ограничен 1 ТБ.
:::
