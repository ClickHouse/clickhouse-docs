Хотя ClickHouse известен своей скоростью и надежностью, оптимальная производительность 
достигается в рамках определенных эксплуатационных параметров. Например, слишком большое количество таблиц,
баз данных или частей (parts) может негативно повлиять на производительность. Чтобы предотвратить это, ClickHouse
Cloud вводит ограничения по нескольким операционным параметрам. 
Подробности этих ограничений приведены ниже.

:::tip
Если вы достигли одного из этих ограничений, возможно, ваш сценарий использования 
реализован неоптимальным образом. Свяжитесь с нашей службой поддержки, и 
мы поможем вам скорректировать сценарий использования, чтобы избежать превышения этих ограничений, 
или вместе рассмотрим, как мы можем увеличить их контролируемым образом. 
:::

| Dimension                     | Limit                                                      |
|-------------------------------|------------------------------------------------------------|
| **Databases**                 | 1000                                                       |
| **Tables**                    | 5000                                                       |
| **Columns**                   | ∼1000 (предпочтителен широкий формат, а не компактный)     |
| **Partitions**                | 50k                                                        |
| **Parts**                     | 100k на весь экземпляр сервиса                             |
| **Part size**                 | 150 ГБ                                                     |
| **Services per organization** | 20 (мягкое ограничение)                                    |
| **Services per warehouse**    | 5 (мягкое ограничение)                                     |
| **Replicas per service**      | 20 (мягкое ограничение)                                    |  
| **Low cardinality**           | 10k или меньше                                             |
| **Primary keys in a table**   | 4–5, которые достаточно эффективно отфильтровывают данные |
| **Query concurrency**         | 1000 (на реплику)                                          |
| **Batch ingest**              | все, что &gt; 1M, будет системой разбито на блоки по 1M строк |

:::note
Для сервисов с одной репликой максимальное число баз данных ограничено 
100, а максимальное число таблиц — 500. Кроме того, объем хранилища
для сервисов уровня Basic ограничен 1 ТБ.
:::
