Хотя ClickHouse известен своей скоростью и надежностью, оптимальная производительность 
достигается в пределах определенных рабочих параметров. Например, слишком большое количество таблиц,
баз данных или партов может негативно повлиять на производительность. Чтобы предотвратить это, ClickHouse
Cloud применяет ограничения по нескольким операционным параметрам. 
Подробности этих ограничений приведены ниже.

:::tip
Если вы столкнулись с одним из этих ограничений, возможно, вы 
реализуете свой сценарий использования неоптимальным способом. Свяжитесь с нашей службой поддержки, и 
мы с радостью поможем вам оптимизировать ваш сценарий использования, чтобы избежать превышения ограничений, 
или совместно рассмотрим возможность их контролируемого увеличения. 
:::

| Параметр                      | Ограничение                                                      |
|-------------------------------|------------------------------------------------------------------|
| **Базы данных**               | 1000                                                             |
| **Таблицы**                   | 5000                                                             |
| **Столбцы**                   | ∼1000 (предпочтителен широкий формат, а не компактный)           |
| **Партиции**                  | 50 тыс.                                                          |
| **Парты**                     | 100 тыс. на весь инстанс                                         |
| **Размер парта**              | 150 ГБ                                                           |
| **Сервисов на организацию**   | 20 (мягкое ограничение)                                          |
| **Сервисов на warehouse**     | 5 (мягкое ограничение)                                           |
| **Реплик на сервис**          | 20 (мягкое ограничение)                                          |  
| **Низкая кардинальность**     | 10 тыс. или меньше                                               |
| **Первичные ключи в таблице** | 4-5, которые достаточно фильтруют данные                         |
| **Параллелизм запросов**      | 1000 (на реплику)                                                |
| **Пакетная вставка**          | всё > 1 млн будет разделено системой на блоки по 1 млн строк     |

:::note
Для сервисов с одной репликой максимальное количество баз данных ограничено 
100, а максимальное количество таблиц ограничено 500. Кроме того, хранилище
для сервисов уровня Basic ограничено 1 ТБ.
:::
