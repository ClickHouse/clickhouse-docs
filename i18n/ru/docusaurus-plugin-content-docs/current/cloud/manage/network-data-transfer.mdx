ClickHouse Cloud учитывает объем переданных данных на вход и на выход. 
Это включает любые данные, входящие в ClickHouse Cloud и исходящие из него, а также любые передачи данных внутри региона и между регионами. 
Это использование отслеживается на уровне сервиса. На основе этого использования клиенты несут расходы на передачу данных, которые затем добавляются к их ежемесячному счету.

ClickHouse Cloud взимает плату за:
- Исходящие данные из ClickHouse Cloud в общедоступный Интернет, включая другие регионы других облачных провайдеров. 
- Исходящие данные в другой регион в том же облачном провайдере.

Не взимается плата за передачу данных внутри региона или использование Private Link/Private Service Connect и передачу данных. 
Тем не менее, мы оставляем за собой право внедрять дополнительные параметры ценообразования на передачу данных, если мы увидим паттерны использования, которые влияют на нашу возможность правильно взимать плату с пользователей.

Расходы на передачу данных варьируются в зависимости от облачного провайдера (CSP) и региона. 
Цены на исходящие данные в общедоступный интернет основаны только на исходном регионе. 
Цены на межрегиональные (или кросс-региональные) передачи зависят как от исходного, так и от целевого регионов.

В таблицах ниже показано, как расходы на передачу данных за исходящие данные варьируются в зависимости от общедоступного интернета или межрегиональной передачи по облачным провайдерам и регионам.

:::note
ClickHouse Cloud учитывает межрегиональное использование в терминах категорий, от 1 до 4, в зависимости от исходного и целевого регионов. В таблице ниже показана категория для каждой комбинации межрегиональной передачи данных. На экране выставления счетов в ClickHouse Cloud вы увидите расход на передачу данных, разбитый по категориям. 
:::

:::note
Обратите внимание, что при выполнении INSERT через нативный протокол с неинлайновыми значениями (например, INSERT INTO [TABLE] FROM INFILE [FILE] FORMAT NATIVE), клиенты ClickHouse получают метаданные с серверов для упаковки данных. Если метаданные превышают размер полезной нагрузки INSERT, вы можете неинтуитивно увидеть больше исходящих данных, чем входящих с точки зрения сервера. Если это неприемлемо, рассмотрите возможность инлайновой передачи данных с синтаксисом VALUES или использования HTTP протокола.
:::
