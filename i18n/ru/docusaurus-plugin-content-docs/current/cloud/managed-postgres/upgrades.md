Managed Postgres автоматически выполняет обновление версий PostgreSQL, чтобы ваши экземпляры оставались защищёнными и актуальными. Поддерживаются как минорные, так и мажорные обновления с минимальными перебоями в работе.


## Минорные обновления версий \{#minor-version-upgrades\}

Минорные обновления версий (например, с 16.4 до 16.5) включают исправления ошибок и обновления безопасности. Они выполняются с использованием переключения на резервный узел (failover) и обычно приводят лишь к кратковременному разрыву соединения — часто всего на несколько секунд.

Для экземпляров с включёнными [standbys](/cloud/managed-postgres/high-availability) обновление сначала применяется к standby-узлу, после чего выполняется переключение (failover) для минимизации простоя.

## Обновления мажорных версий \{#major-version-upgrades\}

Обновления мажорных версий (например, с 16.x до 17.x) также требуют лишь нескольких секунд простоя и выполняются по аналогичному подходу с механизмом failover.

## Окна обслуживания \{#maintenance-windows\}

Managed Postgres поддерживает окна обслуживания, чтобы обновления и другие операции техобслуживания можно было планировать на время, когда они минимально влияют на работу ваших рабочих нагрузок. Поддержка настройки окон обслуживания в интерфейсе пользователя появится в ближайшее время. А пока свяжитесь со [службой поддержки](https://clickhouse.com/support/program), чтобы настроить окно обслуживания для вашего экземпляра.
