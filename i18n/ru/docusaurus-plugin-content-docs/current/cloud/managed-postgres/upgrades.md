Managed Postgres автоматически выполняет обновление версий PostgreSQL, чтобы ваши экземпляры оставались защищёнными и актуальными. Поддерживаются как минорные, так и мажорные обновления с минимальными перебоями в работе.

## Обновления при обслуживании \{#maintenance-updates\}

Регулярное обслуживание экземпляров PostgreSQL включает следующее:

* Минорные обновления версий (например, с 17.4 до 17.5) включают исправления ошибок и обновления безопасности движка PostgreSQL.
* Функции Managed Service. Улучшения нативной CDC, обсервабилити, pg&#95;clickhouse и других расширений.
* Патчи операционной системы и системных компонентов, включая исправления безопасности, повышение эффективности и другие улучшения.

Они выполняются с использованием переключения на резервный узел (failover) и обычно приводят лишь к кратковременному разрыву соединения — часто всего на несколько секунд.

Для экземпляров с включёнными [standbys](/cloud/managed-postgres/high-availability) обновление сначала применяется к standby-узлу, после чего выполняется переключение (failover) для минимизации простоя.

## Окна обслуживания \{#maintenance-windows\}

Окно обслуживания по умолчанию — с 14:00 до 16:00 UTC в воскресенье.
Ожидаемое время простоя в пределах этого окна — менее 1 минуты.

Для организаций уровня Enterprise Tier Managed Postgres поддерживает окна обслуживания, чтобы обновления и другие операции техобслуживания можно было планировать на время, когда они минимально влияют на работу ваших рабочих нагрузок. Поддержка настройки окон обслуживания в интерфейсе пользователя и через API появится в ближайшее время. А пока свяжитесь со [службой поддержки](https://clickhouse.com/support/program), чтобы настроить окно обслуживания для вашего экземпляра.

## Обновления основных версий \{#major-version-upgrades\}

Поддержка обновлений основных версий (например, с 17.x до 18.x) через UI и API появится в ближайшее время.
А пока свяжитесь со [службой поддержки](https://clickhouse.com/support/program), чтобы обновить свой экземпляр Managed Postgres.
