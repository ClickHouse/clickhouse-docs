## timeSeriesInstantRateToGrid \{#timeSeriesInstantRateToGrid\}

Добавлено в версии: v25.6

Агрегатная функция, которая принимает данные временного ряда в виде пар меток времени и значений и вычисляет [PromQL-подобный irate](https://prometheus.io/docs/prometheus/latest/querying/functions/#irate) для этих данных на регулярной временной сетке, задаваемой начальной меткой времени, конечной меткой времени и шагом. Для каждой точки на сетке семплы для вычисления `irate` рассматриваются в пределах заданного временного окна.

:::warning
Эта функция является экспериментальной. Чтобы её включить, установите `allow_experimental_ts_to_grid_aggregate_function=true`.
:::

**Синтаксис**

```sql
timeSeriesInstantRateToGrid(start_timestamp, end_timestamp, grid_step, staleness)(timestamp, value)
```

**Параметры**

* `start_timestamp` — Определяет начало сетки. [`UInt32`](/sql-reference/data-types/int-uint) или [`DateTime`](/sql-reference/data-types/datetime)
* `end_timestamp` — Определяет конец сетки. [`UInt32`](/sql-reference/data-types/int-uint) или [`DateTime`](/sql-reference/data-types/datetime)
* `grid_step` — Определяет шаг сетки в секундах. [`UInt32`](/sql-reference/data-types/int-uint)
* `staleness` — Задает максимальное время устаревания рассматриваемых семплов в секундах. Окно устаревания представляет собой полуинтервал, открытый слева и закрытый справа. [`UInt32`](/sql-reference/data-types/int-uint)

**Аргументы**

* `timestamp` — Метка времени семпла. Может быть отдельными значениями или массивами. [`UInt32`](/sql-reference/data-types/int-uint) или [`DateTime`](/sql-reference/data-types/datetime) или [`Array(UInt32)`](/sql-reference/data-types/array) или [`Array(DateTime)`](/sql-reference/data-types/array)
* `value` — Значение временного ряда, соответствующее метке времени. Может быть отдельными значениями или массивами. [`Float*`](/sql-reference/data-types/float) или [`Array(Float*)`](/sql-reference/data-types/array)

**Возвращаемое значение**

Возвращает значения irate на заданной сетке. Возвращаемый массив содержит одно значение для каждой точки временной сетки. Значение равно NULL, если в окне недостаточно семплов для вычисления значения мгновенной скорости изменения для конкретной точки сетки. [`Array(Nullable(Float64))`](/sql-reference/data-types/array)

**Примеры**

**Базовое использование с отдельными парами «метка времени — значение»**

```sql title=Query
WITH
    -- NOTE: the gap between 140 and 190 is to show how values are filled for ts = 150, 165, 180 according to window parameter
    [110, 120, 130, 140, 190, 200, 210, 220, 230]::Array(DateTime) AS timestamps,
    [1, 1, 3, 4, 5, 5, 8, 12, 13]::Array(Float32) AS values, -- array of values corresponding to timestamps above
    90 AS start_ts,       -- start of timestamp grid
    90 + 120 AS end_ts,   -- end of timestamp grid
    15 AS step_seconds,   -- step of timestamp grid
    45 AS window_seconds  -- "staleness" window
SELECT timeSeriesInstantRateToGrid(start_ts, end_ts, step_seconds, window_seconds)(timestamp, value)
FROM
(
    -- This subquery converts arrays of timestamps and values into rows of `timestamp`, `value`
    SELECT
        arrayJoin(arrayZip(timestamps, values)) AS ts_and_val,
        ts_and_val.1 AS timestamp,
        ts_and_val.2 AS value
);
```

```response title=Response
┌─timeSeriesInstantRateToGrid(start_ts, end_ts, step_seconds, window_seconds)(timestamp, value)─┐
│ [NULL,NULL,0,0.2,0.1,0.1,NULL,NULL,0.3]                                                      │
└──────────────────────────────────────────────────────────────────────────────────────────────┘
```

**Использование массивов в качестве аргументов**

```sql title=Query
-- it is possible to pass multiple samples of timestamps and values as Arrays of equal size
WITH
    [110, 120, 130, 140, 190, 200, 210, 220, 230]::Array(DateTime) AS timestamps,
    [1, 1, 3, 4, 5, 5, 8, 12, 13]::Array(Float32) AS values,
    90 AS start_ts,
    90 + 120 AS end_ts,
    15 AS step_seconds,
    45 AS window_seconds
SELECT timeSeriesInstantRateToGrid(start_ts, end_ts, step_seconds, window_seconds)(timestamps, values);
```

```response title=Response
┌─timeSeriesInstantRateToGrid(start_ts, end_ts, step_seconds, window_seconds)(timestamps, values)─┐
│ [NULL,NULL,0,0.2,0.1,0.1,NULL,NULL,0.3]                                                        │
└────────────────────────────────────────────────────────────────────────────────────────────────┘
```
