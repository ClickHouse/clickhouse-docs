# maxIntersections

Агрегатная функция, которая вычисляет максимальное количество пересечений внутри группы интервалов (при условии, что все интервалы пересекаются хотя бы один раз).

Синтаксис:

```sql
maxIntersections(start_column, end_column)
```

**Аргументы**

* `start_column` — числовой столбец, задающий начало каждого интервала. Если значение `start_column` равно `NULL` или 0, интервал будет пропущен.

* `end_column` — числовой столбец, задающий конец каждого интервала. Если значение `end_column` равно `NULL` или 0, интервал будет пропущен.

**Возвращаемое значение**

Возвращает максимальное количество пересекающихся между собой интервалов.

**Пример**

```sql
CREATE TABLE my_events (
    start UInt32,
    end UInt32
)
ENGINE = MergeTree
ORDER BY tuple();

INSERT INTO my_events VALUES
   (1, 3),
   (1, 6),
   (2, 5),
   (3, 7);
```

Интервалы выглядят следующим образом:

```response
1 - 3
1 - - - - 6
  2 - - 5
    3 - - - 7
```

Три из этих интервалов имеют общую точку (значение равно `4`, но конкретное значение не важно, мы считаем количество пересечений). Интервалы `(1,3)` и `(3,7)` имеют общую границу, но не считаются пересекающимися функцией `maxIntersections`.

```sql
SELECT maxIntersections(start, end) FROM my_events;
```

Ответ:

```response
3
```

Если максимальный интервал встречается несколько раз, вы можете использовать функцию [`maxIntersectionsPosition`](./maxintersectionsposition.md), чтобы определить их количество и местоположение.
