# Вложенный тип



## Nested(name1 Type1, Name2 Type2, ...) {#nestedname1-type1-name2-type2-}

Вложенная структура данных представляет собой таблицу внутри ячейки. Параметры вложенной структуры данных — имена и типы столбцов — указываются так же, как в запросе [CREATE TABLE](../../../sql-reference/statements/create/table.md). Каждая строка таблицы может соответствовать любому количеству строк во вложенной структуре данных.

Пример:

```sql
CREATE TABLE test.visits
(
    CounterID UInt32,
    StartDate Date,
    Sign Int8,
    IsNew UInt8,
    VisitID UInt64,
    UserID UInt64,
    ...
    Goals Nested
    (
        ID UInt32,
        Serial UInt32,
        EventTime DateTime,
        Price Int64,
        OrderID String,
        CurrencyID UInt32
    ),
    ...
) ENGINE = CollapsingMergeTree(StartDate, intHash32(UserID), (CounterID, StartDate, intHash32(UserID), VisitID), 8192, Sign)
```

В этом примере объявляется вложенная структура данных `Goals`, которая содержит данные о конверсиях (достигнутых целях). Каждая строка в таблице 'visits' может соответствовать нулю или любому количеству конверсий.

Когда параметр [flatten_nested](/operations/settings/settings#flatten_nested) установлен в `0` (что не является значением по умолчанию), поддерживаются произвольные уровни вложенности.

В большинстве случаев при работе с вложенной структурой данных её столбцы указываются через точку. Эти столбцы образуют массивы соответствующих типов. Все массивы столбцов одной вложенной структуры данных имеют одинаковую длину.

Пример:

```sql
SELECT
    Goals.ID,
    Goals.EventTime
FROM test.visits
WHERE CounterID = 101500 AND length(Goals.ID) < 5
LIMIT 10
```

Проще всего представлять вложенную структуру данных как набор нескольких массивов столбцов одинаковой длины.

Единственное место, где в запросе SELECT можно указать имя всей вложенной структуры данных вместо отдельных столбцов — это секция ARRAY JOIN. Для получения дополнительной информации см. раздел «Секция ARRAY JOIN». Пример:

```sql
SELECT
    Goal.ID,
    Goal.EventTime
FROM test.visits
ARRAY JOIN Goals AS Goal
WHERE CounterID = 101500 AND length(Goals.ID) < 5
LIMIT 10
```

```text
│ 1073752 │ 2014-03-17 16:38:10 │

│ 1073752 │ 2014-03-17 00:28:25 │
│ 1073752 │ 2014-03-17 10:46:20 │
│ 1073752 │ 2014-03-17 13:59:20 │
│  591325 │ 2014-03-17 22:17:55 │
│  591325 │ 2014-03-17 22:18:07 │
│  591325 │ 2014-03-17 22:18:51 │
│ 1073752 │ 2014-03-17 11:37:06 │

```

Нельзя выполнить оператор SELECT для всей вложенной структуры данных целиком. Можно только явно перечислить отдельные столбцы, которые в неё входят.

В запросе INSERT необходимо передавать все массивы столбцов, входящие во вложенную структуру данных, по отдельности (как если бы это были отдельные массивы столбцов). При вставке система проверяет, что их длины совпадают.

В запросе DESCRIBE столбцы во вложенной структуре данных перечисляются по отдельности таким же образом.

Оператор ALTER, применяемый к элементам вложенной структуры данных, имеет ограничения.
