# Логические функции

Функции, описанные ниже, выполняют логические операции над аргументами произвольных числовых типов.
Они возвращают `0` или `1` типа [`UInt8`](../data-types/int-uint.md), а в некоторых случаях — `NULL`.

Ноль в качестве аргумента интерпретируется как `false`, ненулевые значения — как `true`.

## and {#and}

Введена в версии: v1.1

Вычисляет логическую конъюнкцию двух или более значений.

Настройка [`short_circuit_function_evaluation`](/operations/settings/settings#short_circuit_function_evaluation) управляет использованием сокращённых вычислений.
Если включена, `val_i` вычисляется только в том случае, если `(val_1 AND val_2 AND ... AND val_{i-1})` равно `true`.

Например, при сокращённых вычислениях не возникает исключение деления на ноль при выполнении запроса `SELECT and(number = 2, intDiv(1, number)) FROM numbers(5)`.
Ноль в качестве аргумента считается `false`, ненулевые значения считаются `true`.

**Синтаксис**

```sql
and(val1, val2[, ...])
```

**Аргументы**

- `val1, val2[, ...]` — список из как минимум двух значений. [`Nullable((U)Int*)`](/sql-reference/data-types/nullable) или [`Nullable(Float*)`](/sql-reference/data-types/nullable)

**Возвращаемое значение**

Возвращает:

- `0`, если хотя бы один аргумент вычисляется как `false`
- `NULL`, если ни один аргумент не вычисляется как `false` и хотя бы один аргумент равен `NULL`
- `1` в остальных случаях
  [`Nullable(UInt8)`](/sql-reference/data-types/nullable)

**Примеры**

**Базовое использование**

```sql title=Запрос
SELECT and(0, 1, -2);
```

```response title=Результат
0
```

**С NULL**

```sql title=Запрос
SELECT and(NULL, 1, 10, -2);
```

```response title=Результат
ᴺᵁᴸᴸ
```


## not {#not}

Введено в версии: v1.1

Вычисляет логическое отрицание значения.
Ноль в качестве аргумента считается `false`, ненулевые значения считаются `true`.

**Синтаксис**

```sql
not(val)
```

**Аргументы**

- `val` — значение. [`(U)Int*`](/sql-reference/data-types/int-uint) или [`Float*`](/sql-reference/data-types/float)

**Возвращаемое значение**

Возвращает:

- `1`, если `val` вычисляется как `false`
- `0`, если `val` вычисляется как `true`
- `NULL`, если `val` равен `NULL`.
  [`Nullable(UInt8)`](/sql-reference/data-types/nullable)

**Примеры**

**Базовое использование**

```sql title=Запрос
SELECT NOT(1);
```

```response title=Результат
0
```


## or {#or}

Введено в версии: v1.1

Вычисляет логическую дизъюнкцию двух или более значений.

Настройка [`short_circuit_function_evaluation`](https://clickhouse.com/docs/operations/settings/settings#short_circuit_function_evaluation) управляет использованием ленивых вычислений.
Если включена, `val_i` вычисляется только в том случае, если `((NOT val_1) AND (NOT val_2) AND ... AND (NOT val_{i-1}))` равно `true`.

Например, при ленивых вычислениях не возникает исключение деления на ноль при выполнении запроса `SELECT or(number = 0, intDiv(1, number) != 0) FROM numbers(5)`.
Ноль в качестве аргумента считается `false`, ненулевые значения считаются `true`.

**Синтаксис**

```sql
or(val1, val2[, ...])
```

**Аргументы**

- `val1, val2[, ...]` — список из как минимум двух значений. [`Nullable((U)Int*)`](/sql-reference/data-types/nullable) или [`Nullable(Float*)`](/sql-reference/data-types/nullable)

**Возвращаемое значение**

Возвращает:

- `1`, если хотя бы один аргумент вычисляется как `true`
- `0`, если все аргументы вычисляются как `false`
- `NULL`, если все аргументы вычисляются как `false` и хотя бы один аргумент равен `NULL`
  [`Nullable(UInt8)`](/sql-reference/data-types/nullable)

**Примеры**

**Базовое использование**

```sql title=Запрос
SELECT or(1, 0, 0, 2, NULL);
```

```response title=Результат
1
```

**С NULL**

```sql title=Запрос
SELECT or(0, NULL);
```

```response title=Результат
ᴺᵁᴸᴸ
```


## xor {#xor}

Введено в версии: v1.1

Вычисляет логическое исключающее ИЛИ (XOR) двух или более значений.
Для более чем двух входных значений функция сначала применяет операцию XOR к первым двум значениям, затем применяет XOR к результату и третьему значению и т. д.
Ноль в качестве аргумента считается `false`, ненулевые значения считаются `true`.

**Синтаксис**

```sql
xor(val1, val2[, ...])
```

**Аргументы**

- `val1, val2[, ...]` — список как минимум из двух значений. [`Nullable((U)Int*)`](/sql-reference/data-types/nullable) или [`Nullable(Float*)`](/sql-reference/data-types/nullable)

**Возвращаемое значение**

Возвращает:

- `1` для двух значений: если одно из значений вычисляется как `false`, а другое — нет
- `0` для двух значений: если оба значения вычисляются как `false` или оба как `true`
- `NULL`, если хотя бы один из входных параметров равен `NULL`.
  [`Nullable(UInt8)`](/sql-reference/data-types/nullable)

**Примеры**

**Базовое использование**

```sql title=Запрос
SELECT xor(0, 1, 1);
```

```response title=Результат
0
```
