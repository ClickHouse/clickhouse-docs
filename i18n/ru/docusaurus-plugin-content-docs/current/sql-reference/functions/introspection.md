# Функции интроспекции {#introspection-functions}

Вы можете использовать функции, описанные в этой главе, для интроспекции форматов [ELF](https://en.wikipedia.org/wiki/Executable_and_Linkable_Format) и [DWARF](https://en.wikipedia.org/wiki/DWARF) при профилировании запросов.

:::note
Эти функции работают медленно и могут иметь последствия для безопасности.
:::

Для корректной работы функций интроспекции:

* Установите пакет `clickhouse-common-static-dbg`.

* Установите настройку [allow&#95;introspection&#95;functions](../../operations/settings/settings.md#allow_introspection_functions) в значение 1.

  По соображениям безопасности функции интроспекции по умолчанию отключены.

ClickHouse сохраняет отчёты профилировщика в системной таблице [trace&#95;log](/operations/system-tables/trace_log). Убедитесь, что таблица и профилировщик корректно настроены.

## addressToLine {#addressToLine}

Впервые появилась в версии v20.1

Преобразует адрес виртуальной памяти внутри серверного процесса ClickHouse в имя файла и номер строки в исходном коде ClickHouse.

:::note
Эта функция работает медленно и может иметь последствия для безопасности.
:::

Чтобы включить эту функцию интроспекции:

* Установите пакет `clickhouse-common-static-dbg`.
* Установите настройку [`allow_introspection_functions`](../../operations/settings/settings.md#allow_introspection_functions) в значение `1`.

**Синтаксис**

```sql
demangle(symbol)
```

**Аргументы**

* `address_of_binary_instruction` — Адрес инструкции в запущенном процессе. [`UInt64`](/sql-reference/data-types/int-uint)

**Возвращаемое значение**

Возвращает имя файла исходного кода и номер строки, разделённые двоеточием, например, `/build/obj-x86_64-linux-gnu/../src/Common/ThreadPool.cpp:199`. Возвращает имя исполняемого файла, если отладочная информация недоступна, или пустую строку, если адрес недействителен. [`String`](/sql-reference/data-types/string)

**Примеры**

**Выбор первой строки из системной таблицы `trace_log`**

```sql title=Query
SET allow_introspection_functions=1;
SELECT * FROM system.trace_log LIMIT 1 \G;
```

```response title=Response

```

**Getting a function name for a single address**

```sql title=Query

```


**Applying the function to the whole stack trace**

```sql title=Query
-- Результат обработки отображается в столбце trace_source_code_lines.

    arrayStringConcat(arrayMap(x -> addressToLine(x), trace), '\n') AS trace_source_code_lines

LIMIT 1
\G
```

## isMergeTreePartCoveredBy {#isMergeTreePartCoveredBy}

Добавлена в версии: v25.6

Функция, которая проверяет, покрывается ли часть из первого аргумента частью из второго аргумента.

**Синтаксис**

```sql
isMergeTreePartCoveredBy(nested_part, covering_part)
```

**Аргументы**

* `nested_part` — Имя ожидаемой вложенной части. [`String`](/sql-reference/data-types/string)

**Возвращаемое значение**

Возвращает `1`, если покрывает, в противном случае — `0`. [`UInt8`](/sql-reference/data-types/int-uint)

**Примеры**

**Базовый пример**

```sql title=Query
WITH 'all_12_25_7_4' AS lhs, 'all_7_100_10_20' AS rhs
SELECT isMergeTreePartCoveredBy(rhs, lhs), isMergeTreePartCoveredBy(lhs, rhs);
```
