# lead

Возвращает значение, вычисленное для строки, смещённой на offset строк вперёд относительно текущей строки в упорядоченном фрейме.
Эта функция похожа на [`leadInFrame`](./leadInFrame.md), но всегда использует фрейм `ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING`.

**Синтаксис**

```sql
lead(x[, offset[, default]])
  OVER ([[PARTITION BY grouping_column] [ORDER BY sorting_column]] | [window_name])
FROM table_name
WINDOW window_name as ([[PARTITION BY grouping_column] [ORDER BY sorting_column])
```

Для более подробной информации о синтаксисе оконных функций см.: [Window Functions - Syntax](./index.md/#syntax).

**Параметры**

* `x` — имя столбца.
* `offset` — смещение, которое нужно применить. [(U)Int*](../data-types/int-uint.md). (Необязательный параметр — по умолчанию `1`).
* `default` — значение, которое возвращается, если вычисляемая строка выходит за границы оконного фрейма. (Необязательный параметр — по умолчанию используется значение по умолчанию для типа столбца, если параметр опущен).

**Возвращаемое значение**

* значение, вычисленное для строки, которая находится на `offset` строк после текущей строки в пределах упорядоченного фрейма.

**Пример**

В этом примере используется [исторический набор данных](https://www.kaggle.com/datasets/sazidthe1/nobel-prize-data) о лауреатах Нобелевской премии, а функция `lead` применяется для получения списка последовательных лауреатов в области физики.

```sql title="Query"
CREATE OR REPLACE VIEW nobel_prize_laureates
AS SELECT *
FROM file('nobel_laureates_data.csv');
```

```sql title="Query"
SELECT
    fullName,
    lead(year, 1, year) OVER (PARTITION BY category ORDER BY year ASC
      ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING
    ) AS year,
    category,
    motivation
FROM nobel_prize_laureates
WHERE category = 'physics'
ORDER BY year DESC
LIMIT 9
```
