`clickhouse-jdbc` реализует стандартный интерфейс JDBC. Построенный на основе [clickhouse-client](/integrations/sql-clients/sql-console), он предоставляет дополнительные функции, такие как пользовательское отображение типов, поддержку транзакций и стандартные синхронные операторы `UPDATE` и `DELETE`, что позволяет легко использовать его с устаревшими приложениями и инструментами.

:::note
Последняя версия JDBC (0.7.2) использует Client-V1
:::

API `clickhouse-jdbc` является синхронным и, в общем, имеет больше дополнительных расходов (например, SQL парсинг и отображение/преобразование типов и т.д.). Рассмотрите возможность использования [clickhouse-client](/integrations/sql-clients/sql-console), когда производительность критична или если вы предпочитаете более прямой способ доступа к ClickHouse.

## Конфигурация {#configuration}

**Класс драйвера**: `com.clickhouse.jdbc.ClickHouseDriver`

**Синтаксис URL**: `jdbc:(ch|clickhouse)[:<protocol>]://endpoint1[,endpoint2,...][/<database>][?param1=value1&param2=value2][#tag1,tag2,...]`, например:

- `jdbc:ch://localhost` эквивалентен `jdbc:clickhouse:http://localhost:8123`
- `jdbc:ch:https://localhost` эквивалентен `jdbc:clickhouse:http://localhost:8443?ssl=true&sslmode=STRICT`
- `jdbc:ch:grpc://localhost` эквивалентен `jdbc:clickhouse:grpc://localhost:9100`

**Свойства соединения**:

| Свойство                 | По умолчанию | Описание                                                                                                                                                                                                                                                                                                                                                                                                                |
| ------------------------ | ------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `continueBatchOnError`     | `false` | Продолжать пакетную обработку при возникновении ошибки                                                                                                                                                                                                                                                                                                                                                                   |
| `createDatabaseIfNotExist` | `false` | Создавать ли базу данных, если она не существует                                                                                                                                                                                                                                                                                                                                                                            |
| `custom_http_headers`      |         | пользовательские HTTP-заголовки через запятую, например: `User-Agent=client1,X-Gateway-Id=123`                                                                                                                                                                                                                                                                                                                                    |
| `custom_http_params`       |         | пользовательские HTTP-параметры запросов через запятую, например: `extremes=0,max_result_rows=100`                                                                                                                                                                                                                                                                                                                                |
| `nullAsDefault`            | `0`     | `0` - обрабатывать значение null как есть и выбрасывать исключение при вставке null в не-nullable колонку; `1` - обрабатывать значение null как есть и отключать проверку null для вставок; `2` - заменять null на значение по умолчанию соответствующего типа данных как для запросов, так и для вставок                                                                                                                                                                 |
| `jdbcCompliance`           | `true`  | Поддерживать ли стандартные синхронные операции UPDATE/DELETE и фейковую транзакцию                                                                                                                                                                                                                                                                                                                                                 |
| `typeMappings`             |         | Настройка соответствия между типами данных ClickHouse и классами Java, что повлияет на результат как [`getColumnType()`](https://docs.oracle.com/javase/8/docs/api/java/sql/ResultSetMetaData.html#getColumnType-int-) так и [`getObject(Class<>?>`)](https://docs.oracle.com/javase/8/docs/api/java/sql/ResultSet.html#getObject-java.lang.String-java.lang.Class-). Например: `UInt128=java.lang.String,UInt256=java.lang.String` |
| `wrapperObject`            | `false` | Должен ли [`getObject()`](https://docs.oracle.com/javase/8/docs/api/java/sql/ResultSet.html#getObject-int-) возвращать java.sql.Array / java.sql.Struct для массива / кортежа.                                                                                                                                                                                                                                                  |

Примечание: дополнительную информацию см. в [JDBC специфической конфигурации](https://github.com/ClickHouse/clickhouse-java/blob/main/clickhouse-jdbc/src/main/java/com/clickhouse/jdbc/JdbcConfig.java).

## Поддерживаемые типы данных {#supported-data-types}

Драйвер JDBC поддерживает такие же форматы данных, как и библиотека клиента.

:::note
- AggregatedFunction - :warning: не поддерживает `SELECT * FROM table ...`
- Decimal - `SET output_format_decimal_trailing_zeros=1` в 21.9+ для согласованности
- Enum - может рассматриваться как строка и целое число
- UInt64 - сопоставляется с `long` (в client-v1)
:::

## Вставка {#insert}

:::note
- Используйте `PreparedStatement` вместо `Statement`
:::

Использование легче, но производительность ниже по сравнению с функцией ввода (см. ниже):

```java showLineNumbers
try (PreparedStatement ps = conn.prepareStatement("insert into mytable(* except (description))")) {
    ps.setString(1, "test"); // id
    ps.setObject(2, LocalDateTime.now()); // timestamp
    ps.addBatch(); // parameters will be write into buffered stream immediately in binary format
    ...
    ps.executeBatch(); // stream everything on-hand into ClickHouse
}
```

### С функцией ввода таблицы {#with-input-table-function}

Опция с отличными характеристиками производительности:

```java showLineNumbers
try (PreparedStatement ps = conn.prepareStatement(
    "insert into mytable select col1, col2 from input('col1 String, col2 DateTime64(3), col3 Int32')")) {
    // The column definition will be parsed so the driver knows there are 3 parameters: col1, col2 and col3
    ps.setString(1, "test"); // col1
    ps.setObject(2, LocalDateTime.now()); // col2, setTimestamp is slow and not recommended
    ps.setInt(3, 123); // col3
    ps.addBatch(); // parameters will be write into buffered stream immediately in binary format
    ...
    ps.executeBatch(); // stream everything on-hand into ClickHouse
}
```
- [Документация по функции ввода](/sql-reference/table-functions/input/) когда это возможно

### Вставка с заполнителями {#insert-with-placeholders}

Эта опция рекомендуется только для небольших вставок, так как она потребует длинного SQL-выражения (которое будет парситься на стороне клиента и потребует ресурсов CPU и памяти):

```java showLineNumbers
try (PreparedStatement ps = conn.prepareStatement("insert into mytable values(trim(?),?,?)")) {
    ps.setString(1, "test"); // id
    ps.setObject(2, LocalDateTime.now()); // timestamp
    ps.setString(3, null); // description
    ps.addBatch(); // append parameters to the query
    ...
    ps.executeBatch(); // issue the composed query: insert into mytable values(...)(...)...(...)
}
```

## Работа с `AggregateFunction` {#handling-aggregatefunction}

:::note
На данный момент поддерживается только `groupBitmap`.
:::

```java showLineNumbers
// batch insert using input function
try (ClickHouseConnection conn = newConnection(props);
        Statement s = conn.createStatement();
        PreparedStatement stmt = conn.prepareStatement(
                "insert into test_batch_input select id, name, value from input('id Int32, name Nullable(String), desc Nullable(String), value AggregateFunction(groupBitmap, UInt32)')")) {
    s.execute("drop table if exists test_batch_input;"
            + "create table test_batch_input(id Int32, name Nullable(String), value AggregateFunction(groupBitmap, UInt32))engine=Memory");
    Object[][] objs = new Object[][] {
            new Object[] { 1, "a", "aaaaa", ClickHouseBitmap.wrap(1, 2, 3, 4, 5) },
            new Object[] { 2, "b", null, ClickHouseBitmap.wrap(6, 7, 8, 9, 10) },
            new Object[] { 3, null, "33333", ClickHouseBitmap.wrap(11, 12, 13) }
    };
    for (Object[] v : objs) {
        stmt.setInt(1, (int) v[0]);
        stmt.setString(2, (String) v[1]);
        stmt.setString(3, (String) v[2]);
        stmt.setObject(4, v[3]);
        stmt.addBatch();
    }
    int[] results = stmt.executeBatch();
    ...
}

// use bitmap as query parameter
try (PreparedStatement stmt = conn.prepareStatement(
    "SELECT bitmapContains(my_bitmap, toUInt32(1)) as v1, bitmapContains(my_bitmap, toUInt32(2)) as v2 from {tt 'ext_table'}")) {
    stmt.setObject(1, ClickHouseExternalTable.builder().name("ext_table")
            .columns("my_bitmap AggregateFunction(groupBitmap,UInt32)").format(ClickHouseFormat.RowBinary)
            .content(new ByteArrayInputStream(ClickHouseBitmap.wrap(1, 3, 5).toBytes()))
            .asTempTable()
            .build());
    ResultSet rs = stmt.executeQuery();
    Assert.assertTrue(rs.next());
    Assert.assertEquals(rs.getInt(1), 1);
    Assert.assertEquals(rs.getInt(2), 0);
    Assert.assertFalse(rs.next());
}
```

## Настройка HTTP библиотеки {#configuring-http-library}

Коннектор ClickHouse JDBC поддерживает три HTTP-библиотеки: [`HttpClient`](https://docs.oracle.com/en/java/javase/11/docs/api/java.net.http/java/net/http/HttpClient.html), [`HttpURLConnection`](https://docs.oracle.com/en/java/javase/11/docs/api/java.base/java/net/HttpURLConnection.html) и [Apache `HttpClient`](https://hc.apache.org/httpcomponents-client-5.2.x/).

:::note
`HttpClient` поддерживается только в JDK 11 и выше.
:::

Драйвер JDBC по умолчанию использует `HttpClient`. Вы можете изменить используемую библиотеку HTTP, установив следующее свойство:

```java
properties.setProperty("http_connection_provider", "APACHE_HTTP_CLIENT");
```

Вот полный список соответствующих значений:

| Значение свойства      | HTTP библиотека        |
|---------------------|---------------------|
| HTTP_CLIENT         | `HttpClient`        |
| HTTP_URL_CONNECTION | `HttpURLConnection` |
| APACHE_HTTP_CLIENT  | Apache `HttpClient` |

## Подключение к ClickHouse с SSL {#connect-to-clickhouse-with-ssl}

Для установления безопасного JDBC-соединения с ClickHouse с использованием SSL, вам необходимо настроить ваши свойства JDBC, чтобы включить параметры SSL. Это обычно включает в себя указание свойств SSL, таких как `sslmode` и `sslrootcert`, в вашем URL JDBC или объекте Properties.

## Свойства SSL {#ssl-properties}

| Имя               | Значение по умолчанию | Допустимые значения | Описание                                                                      |
| ------------------ | ------------- | --------------- |----------------------------------------------------------------------------------|
| `ssl`                | false         | true, false     | Включить ли SSL/TLS для соединения                                     |
| `sslmode`            | strict        | strict, none    | Проверять ли SSL/TLS сертификат                                            |
| `sslrootcert`        |               |                 | Путь к SSL/TLS корневым сертификатам                                                |
| `sslcert`            |               |                 | Путь к SSL/TLS сертификату                                                      |
| `sslkey`             |               |                 | RSA ключ в PKCS#8 формате                                                         |
| `key_store_type`     |               | JKS, PKCS12     | Указывает тип или формат файла `KeyStore`/`TrustStore`                 |
| `trust_store`        |               |                 | Путь к файлу `TrustStore`                                                    |
| `key_store_password` |               |                 | Пароль, необходимый для доступа к файлу `KeyStore`, указанному в конфигурации `KeyStore` |

Эти свойства обеспечивают коммуникацию вашего Java приложения с сервером ClickHouse по зашифрованному соединению, повышая безопасность данных во время передачи.

```java showLineNumbers
String url = "jdbc:ch://your-server:8443/system";

Properties properties = new Properties();
properties.setProperty("ssl", "true");
properties.setProperty("sslmode", "strict"); // NONE to trust all servers; STRICT for trusted only
properties.setProperty("sslrootcert", "/mine.crt");
try (Connection con = DriverManager
        .getConnection(url, properties)) {

    try (PreparedStatement stmt = con.prepareStatement(

        // place your code here

    }
}
```

## Решение проблемы таймаута JDBC при больших вставках {#resolving-jdbc-timeout-on-large-inserts}

При выполнении больших вставок в ClickHouse с длительным временем выполнения, вы можете столкнуться с ошибками таймаута JDBC, такими как:

```plaintext
Caused by: java.sql.SQLException: Read timed out, server myHostname [uri=https://hostname.aws.clickhouse.cloud:8443]
```

Эти ошибки могут нарушить процесс вставки данных и повлиять на стабильность системы. Чтобы решить эту проблему, вам нужно настроить несколько параметров таймаута в ОС клиента.

### Mac OS {#mac-os}

На Mac OS следующие настройки могут быть настроены для решения проблемы:

- `net.inet.tcp.keepidle`: 60000
- `net.inet.tcp.keepintvl`: 45000
- `net.inet.tcp.keepinit`: 45000
- `net.inet.tcp.keepcnt`: 8
- `net.inet.tcp.always_keepalive`: 1

### Linux {#linux}

На Linux эквивалентные настройки могут не решить проблему сами по себе. Требуются дополнительные шаги из-за различий в том, как Linux обрабатывает настройки сокета keep-alive. Следуйте этим шагам:

1. Отрегулируйте следующие параметры ядра Linux в файле `/etc/sysctl.conf` или связанном конфигурационном файле:

- `net.inet.tcp.keepidle`: 60000
- `net.inet.tcp.keepintvl`: 45000
- `net.inet.tcp.keepinit`: 45000
- `net.inet.tcp.keepcnt`: 8
- `net.inet.tcp.always_keepalive`: 1
- `net.ipv4.tcp_keepalive_intvl`: 75
- `net.ipv4.tcp_keepalive_probes`: 9
- `net.ipv4.tcp_keepalive_time`: 60 (вы можете рассмотреть возможность уменьшения этого значения с 300 секунд по умолчанию)

2. После изменения параметров ядра примените изменения, выполнив следующую команду:

```shell
sudo sysctl -p
```

После установки этих настройек, убедитесь, что ваш клиент включает опцию Keep Alive на сокете:

```java
properties.setProperty("socket_keepalive", "true");
```

:::note
В настоящее время вам необходимо использовать библиотеку Apache HTTP Client для настройки сокета keep-alive, так как другие две поддерживаемые библиотеки HTTP клиента `clickhouse-java` не позволяют устанавливать параметры сокета. Для подробного руководства смотрите [Настройка HTTP библиотеки](/integrations/language-clients/java/jdbc-v1#configuring-http-library).
:::

В качестве альтернативы вы можете добавить эквивалентные параметры в URL JDBC.

По умолчанию таймаут сокета и соединения для драйвера JDBC составляет 30 секунд. Таймаут можно увеличить для поддержки операций вставки больших данных. Используйте метод `options` на `ClickHouseClient` вместе с опциями `SOCKET_TIMEOUT` и `CONNECTION_TIMEOUT`, как определено в `ClickHouseClientOption`:

```java showLineNumbers
final int MS_12H = 12 * 60 * 60 * 1000; // 12 h in ms
final String sql = "insert into table_a (c1, c2, c3) select c1, c2, c3 from table_b;";

try (ClickHouseClient client = ClickHouseClient.newInstance(ClickHouseProtocol.HTTP)) {
    client.read(servers).write()
        .option(ClickHouseClientOption.SOCKET_TIMEOUT, MS_12H)
        .option(ClickHouseClientOption.CONNECTION_TIMEOUT, MS_12H)
        .query(sql)
        .executeAndWait();
}
```
