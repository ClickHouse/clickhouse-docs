Рекомендуем прочитать [это руководство](/guides/inserting-data), чтобы узнать лучшие практики вставки данных в ClickHouse и оптимизировать производительность вставок.

Для пакетной загрузки данных из PostgreSQL пользователи могут использовать:

- [ClickPipes](/integrations/clickpipes/postgres) — управляемый сервис интеграции для ClickHouse Cloud.
- `PeerDB by ClickHouse` — ETL-инструмент, специально разработанный для репликации базы данных PostgreSQL как в автономно развернутый ClickHouse, так и в ClickHouse Cloud.
- [Postgres Table Function](/sql-reference/table-functions/postgresql) для непосредственного чтения данных. Как правило, это подходит, если достаточно пакетной репликации на основе известного watermark (контрольного порога), например метки времени, или если это разовая миграция. Такой подход может масштабироваться до десятков миллионов строк. Пользователям, которые планируют мигрировать более крупные наборы данных, следует рассмотреть несколько запросов, каждый из которых обрабатывает часть данных. Для каждой части можно использовать промежуточные (staging) таблицы до переноса ее партиций в итоговую таблицу. Это позволяет повторно выполнять неуспешные запросы. Более подробную информацию об этой стратегии пакетной загрузки см. здесь.
- Данные могут быть экспортированы из Postgres в формате CSV. Затем их можно вставить в ClickHouse либо из локальных файлов, либо через объектное хранилище с использованием табличных функций.
