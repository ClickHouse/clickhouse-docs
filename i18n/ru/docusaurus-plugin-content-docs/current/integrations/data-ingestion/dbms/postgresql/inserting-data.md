Рекомендуется прочитать [это руководство](/guides/inserting-data), чтобы узнать лучшие практики вставки данных в ClickHouse и оптимизировать производительность вставки.

Для пакетной загрузки данных из PostgreSQL пользователи могут использовать:

- [ClickPipes](/integrations/clickpipes/postgres) — управляемый сервис интеграции для ClickHouse Cloud.
- `PeerDB by ClickHouse` — ETL-инструмент, специально разработанный для репликации базы данных PostgreSQL как в самоуправляемый ClickHouse, так и в ClickHouse Cloud.
- [Табличную функцию Postgres](/sql-reference/table-functions/postgresql) для прямого чтения данных. Обычно это подходит, если достаточно пакетной репликации на основе известного маркера (например, метки времени) или если это разовая миграция. Такой подход может масштабироваться до десятков миллионов строк. Пользователям, желающим перенести более крупные наборы данных, следует рассмотреть несколько запросов, каждый из которых обрабатывает часть данных. Для каждой части можно использовать промежуточные таблицы до переноса её партиций в финальную таблицу. Это позволяет повторно выполнять неудавшиеся запросы. Подробности этой стратегии пакетной загрузки см. здесь.
- Данные можно экспортировать из Postgres в формате CSV. Затем их можно вставить в ClickHouse либо из локальных файлов, либо через объектное хранилище с использованием табличных функций.
