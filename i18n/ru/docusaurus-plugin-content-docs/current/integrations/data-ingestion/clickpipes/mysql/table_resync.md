# Повторная синхронизация отдельных таблиц {#resync-tables}

В некоторых случаях может быть полезно повторно синхронизировать отдельные таблицы пайпа. Примерами таких сценариев могут быть существенные изменения схемы в MySQL или переработка данных в ClickHouse.

Пока повторная синхронизация отдельных таблиц по нажатию кнопки находится в разработке, в этом руководстве описаны шаги, которые вы можете выполнить уже сейчас в MySQL ClickPipe.

### 1. Удалите таблицу из пайпа {#removing-table}

Для этого следуйте [руководству по удалению таблиц](./removing_tables).

### 2. Очистите (TRUNCATE) или удалите (DROP) таблицу в ClickHouse {#truncate-drop-table}

Этот шаг необходим, чтобы избежать дублирования данных при повторном добавлении этой таблицы на следующем шаге. Для этого перейдите на вкладку **SQL Console** в ClickHouse Cloud и выполните запрос.  
Обратите внимание, что выполняется проверка, блокирующая добавление таблицы, если таблица уже существует в ClickHouse и не пуста.

### 3. Снова добавьте таблицу в ClickPipe {#add-table-again}

Для этого следуйте [руководству по добавлению таблиц](./add_table).
