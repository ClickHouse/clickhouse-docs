# Повторная синхронизация отдельных таблиц \{#resync-tables\}

В некоторых сценариях может понадобиться повторно синхронизировать отдельные таблицы конвейера. Примеры таких случаев: серьёзные изменения схемы в MySQL или переработка модели данных в ClickHouse.

Функция повторной синхронизации отдельных таблиц одним нажатием кнопки всё ещё в разработке, поэтому в этом руководстве описаны шаги, которые позволяют сделать это уже сейчас в MySQL ClickPipe.

### 1. Удалите таблицу из пайплайна \{#removing-table\}

Для этого воспользуйтесь [руководством по удалению таблиц](./removing_tables).

### 2. Очистите или удалите таблицу в ClickHouse \{#truncate-drop-table\}

Этот шаг необходим, чтобы избежать дублирования данных, когда мы снова добавим эту таблицу на следующем этапе. Вы можете сделать это, перейдя во вкладку **SQL Console** в ClickHouse Cloud и выполнив запрос.
Обратите внимание, что реализована проверка, предотвращающая добавление таблицы, если она уже существует в ClickHouse и не пуста.

### 3. Повторно добавьте таблицу в ClickPipe \{#add-table-again\}

После этого выполните шаги из [руководства по добавлению таблицы](./add_table).
