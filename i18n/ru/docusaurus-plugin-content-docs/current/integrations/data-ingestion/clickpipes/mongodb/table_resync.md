# Повторная синхронизация отдельных таблиц \{#resync-tables\}

В некоторых сценариях может быть полезно повторно синхронизировать отдельные таблицы пайплайна. Примерами таких случаев могут быть значительные изменения схемы в MongoDB или ремоделирование данных в ClickHouse.

Пока повторная синхронизация отдельных таблиц по нажатию кнопки находится в разработке, в этом руководстве описаны шаги, которые позволяют выполнить эту операцию уже сейчас в MongoDB ClickPipe.

### 1. Удалите таблицу из пайплайна \{#removing-table\}

Для этого следуйте указаниям из [руководства по удалению таблиц](./removing_tables).

### 2. Выполните `TRUNCATE` или `DROP` для таблицы в ClickHouse \{#truncate-drop-table\}

Этот шаг необходим, чтобы избежать дублирования данных при повторном добавлении этой таблицы на следующем шаге. Вы можете сделать это, перейдя на вкладку **SQL Console** в ClickHouse Cloud и выполнив запрос.
Обратите внимание, что у нас реализована проверка, которая блокирует добавление таблицы, если таблица уже существует в ClickHouse и не пуста.

### 3. Повторно добавьте таблицу в ClickPipe \{#add-table-again\}

Для этого следуйте указаниям из [руководства по добавлению таблиц](./add_table).
