# Повторная синхронизация отдельных таблиц \{#resync-tables\}

В ряде сценариев может потребоваться повторно синхронизировать конкретные таблицы в pipe. Примерами таких случаев могут быть серьезные изменения схемы в MongoDB или переработка данных в ClickHouse.

Функция повторной синхронизации отдельных таблиц по нажатию кнопки находится в разработке, поэтому в этом руководстве описаны шаги, которые позволят выполнить это уже сейчас в MongoDB ClickPipe.

### 1. Удалите таблицу из пайплайна (pipe) \{#removing-table\}

Далее действуйте по [руководству по удалению таблиц](./removing_tables).

### 2. Выполните TRUNCATE или удалите таблицу в ClickHouse \{#truncate-drop-table\}

Этот шаг необходим, чтобы избежать дублирования данных при повторном добавлении этой таблицы на следующем шаге. Вы можете сделать это, перейдя на вкладку **SQL Console** в ClickHouse Cloud и выполнив запрос.
Обратите внимание, что у нас реализована проверка, блокирующая добавление таблицы, если таблица уже существует в ClickHouse и не пуста.

### 3. Повторно добавьте таблицу в ClickPipe \{#add-table-again\}

Для этого следуйте [руководству по добавлению таблиц](./add_table).
