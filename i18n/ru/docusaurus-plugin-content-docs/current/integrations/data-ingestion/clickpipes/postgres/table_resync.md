# Повторная синхронизация отдельных таблиц {#resync-tables}

Существуют сценарии, когда необходимо выполнить повторную синхронизацию определённых таблиц конвейера. Примерами таких случаев могут быть значительные изменения схемы в Postgres или реструктуризация данных в ClickHouse.

Хотя возможность повторной синхронизации отдельных таблиц одним нажатием кнопки находится в разработке, в данном руководстве описаны шаги, которые позволяют выполнить это в Postgres ClickPipe уже сейчас.

### 1. Удалите таблицу из конвейера {#removing-table}

Для этого следуйте [руководству по удалению таблиц](./removing_tables).

### 2. Очистите или удалите таблицу в ClickHouse {#truncate-drop-table}

Этот шаг необходим для предотвращения дублирования данных при повторном добавлении таблицы на следующем этапе. Для этого перейдите на вкладку **SQL Console** в ClickHouse Cloud и выполните запрос.
Обратите внимание, что существует проверка, блокирующая добавление таблицы, если таблица уже существует в ClickHouse и не пуста.

### 3. Добавьте таблицу в ClickPipe снова {#add-table-again}

Для этого следуйте [руководству по добавлению таблиц](./add_table).
