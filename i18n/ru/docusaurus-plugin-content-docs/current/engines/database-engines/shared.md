# Движок базы данных Shared

Движок базы данных `Shared` работает совместно с Shared Catalog для управления базами данных, таблицы которых используют без状態ные табличные движки, такие как [`SharedMergeTree`](/cloud/reference/shared-merge-tree).
Эти табличные движки не записывают постоянное состояние на диск и совместимы с динамическими вычислительными средами.

Движок базы данных `Shared` в Cloud устраняет зависимость от локальных дисков.
Это полностью работающий в оперативной памяти движок, которому требуются только процессор и память.



## Как это работает? {#how-it-works}

Движок баз данных `Shared` хранит все определения баз данных и таблиц в централизованном каталоге Shared Catalog, который использует Keeper в качестве хранилища. Вместо записи на локальный диск движок поддерживает единое версионированное глобальное состояние, разделяемое между всеми вычислительными узлами.

Каждый узел отслеживает только последнюю применённую версию и при запуске получает актуальное состояние без необходимости в локальных файлах или ручной настройке.


## Синтаксис {#syntax}

Для конечных пользователей использование Shared Catalog и движка баз данных Shared не требует дополнительной настройки. Создание базы данных выполняется как обычно:

```sql
CREATE DATABASE my_database;
```

ClickHouse Cloud автоматически назначает движок баз данных Shared. Любые таблицы, созданные в такой базе данных с использованием stateless-движков, автоматически получат преимущества возможностей репликации и координации Shared Catalog.

:::tip
Для получения дополнительной информации о Shared Catalog и его преимуществах см. раздел [«Shared catalog и движок баз данных shared»](/cloud/reference/shared-catalog) в справочном разделе Cloud.
:::
