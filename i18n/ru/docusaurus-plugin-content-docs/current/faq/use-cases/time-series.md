# Можно ли использовать ClickHouse как базу данных временных рядов? {#can-i-use-clickhouse-as-a-time-series-database}

_Примечание: Дополнительные примеры использования ClickHouse для анализа временных рядов см. в блоге [Working with Time series data in ClickHouse](https://clickhouse.com/blog/working-with-time-series-data-and-functions-ClickHouse)._

ClickHouse — это универсальное решение для хранения данных в [OLAP](../../faq/general/olap.md)-нагрузках, хотя существует множество специализированных [систем управления базами данных временных рядов](https://clickhouse.com/engineering-resources/what-is-time-series-database). Тем не менее, [ориентация ClickHouse на скорость выполнения запросов](../../concepts/why-clickhouse-is-so-fast.mdx) позволяет ему во многих случаях превосходить специализированные системы. Существует множество независимых бенчмарков на эту тему, поэтому мы не будем проводить их здесь. Вместо этого сосредоточимся на возможностях ClickHouse, которые важно использовать в данном сценарии применения.

Прежде всего, существуют **[специализированные кодеки](../../sql-reference/statements/create/table.md#specialized-codecs)**, которые оптимизированы для типичных временных рядов. Это могут быть как распространённые алгоритмы, такие как `DoubleDelta` и `Gorilla`, так и специфичные для ClickHouse, например `T64`.

Во-вторых, запросы к временным рядам часто обращаются только к недавним данным, например, за последний день или неделю. Имеет смысл использовать серверы, которые имеют как быстрые NVMe/SSD-диски, так и высокоёмкие HDD-диски. Функция [TTL](/engines/table-engines/mergetree-family/mergetree#table_engine-mergetree-ttl) в ClickHouse позволяет настроить хранение свежих «горячих» данных на быстрых дисках с постепенным перемещением на более медленные диски по мере устаревания. Агрегация или удаление ещё более старых данных также возможны, если этого требуют ваши задачи.

Хотя это противоречит философии ClickHouse по хранению и обработке исходных данных, вы можете использовать [материализованные представления](../../sql-reference/statements/create/view.md) для соответствия ещё более жёстким требованиям по задержкам или затратам.
