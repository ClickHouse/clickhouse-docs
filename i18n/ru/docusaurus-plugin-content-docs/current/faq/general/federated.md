# Поддерживает ли ClickHouse федеративные запросы?

ClickHouse обладает самой обширной поддержкой федеративных запросов и гибридного выполнения запросов среди аналитических баз данных.

Он поддерживает выполнение запросов к внешним базам данных:

- PostgreSQL
- MySQL
- MongoDB
- Redis
- любой источник данных ODBC
- любой источник данных JDBC
- любой источник данных Arrow Flight
- потоковые источники данных, такие как Kafka и RabbitMQ
- озёра данных (Data Lakes), такие как Iceberg, Delta Lake, Apache Hudi, Apache Paimon
- внешние файлы, расположенные в общем хранилище, таком как AWS S3, GCS, Minio, Cloudflare R2, Azure Blob Storage, Alicloud OSS, Tencent COS, а также в локальном хранилище, с широкой поддержкой форматов данных

ClickHouse может объединять несколько разных источников данных в одном запросе. Он также предоставляет режим гибридного выполнения запросов, совмещая локальные ресурсы с вынесением части запроса на удалённые машины.

Примечательно, что ClickHouse может ускорять запросы к внешним источникам данных без переноса самих данных. Например, агрегирующий запрос к MySQL будет выполняться быстрее при запуске в ClickHouse, потому что накладные расходы на передачу данных компенсируются более быстрым движком выполнения запросов.
