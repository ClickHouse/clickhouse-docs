# Поддерживает ли ClickHouse федеративные запросы?

ClickHouse обладает наиболее широкими возможностями для федеративных запросов и гибридного выполнения запросов среди аналитических СУБД.

Он поддерживает выполнение запросов к внешним базам данных:

- PostgreSQL
- MySQL
- MongoDB
- Redis
- любой источник данных ODBC
- любой источник данных JDBC
- любой источник данных Arrow Flight
- потоковые источники данных, такие как Kafka и RabbitMQ
- хранилища класса Data Lake, такие как Iceberg, Delta Lake, Apache Hudi, Apache Paimon
- внешние файлы, расположенные в общем хранилище, таком как AWS S3, GCS, Minio, Cloudflare R2, Azure Blob Storage, Alicloud OSS, Tencent COS, а также в локальном хранилище, с поддержкой широкого набора форматов данных

ClickHouse может объединять несколько различных источников данных в одном запросе. Также он предоставляет возможность гибридного выполнения запросов, комбинируя локальные ресурсы и перенося выполнение части запроса на удалённые машины.

Примечательно, что ClickHouse может ускорять запросы к внешним источникам данных без перемещения самих данных. Например, агрегирующий запрос к MySQL будет выполняться быстрее при запуске в ClickHouse, поскольку накладные расходы на передачу данных компенсируются более быстрой работой движка выполнения запросов.
