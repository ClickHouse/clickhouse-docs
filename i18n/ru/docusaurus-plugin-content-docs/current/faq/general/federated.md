# Поддерживает ли ClickHouse федеративные запросы?

ClickHouse обладает самой широкой поддержкой федеративных запросов и гибридного выполнения запросов среди аналитических баз данных.

Он поддерживает выполнение запросов к внешним базам данных:

- PostgreSQL
- MySQL
- MongoDB
- Redis
- любой источник данных ODBC
- любой источник данных JDBC
- любой источник данных Arrow Flight
- потоковые источники данных, такие как Kafka и RabbitMQ
- озера данных (Data Lakes), такие как Iceberg, Delta Lake, Apache Hudi, Apache Paimon
- внешние файлы, размещённые в общем хранилище, таком как AWS S3, GCS, Minio, Cloudflare R2, Azure Blob Storage, Alicloud OSS, Tencent COS, а также в локальном хранилище, с широким набором форматов данных

ClickHouse может объединять несколько разных источников данных в одном запросе. Он также предоставляет возможность гибридного выполнения запросов, сочетая локальные ресурсы и перераспределяя часть запроса на удалённые машины.

Примечательно, что ClickHouse может ускорять запросы к внешним источникам данных без перемещения самих данных. Например, агрегирующий запрос к MySQL будет обрабатываться быстрее, если запускать его в ClickHouse, поскольку накладные расходы на передачу данных компенсируются за счёт более быстрого движка выполнения запросов.
