# Поддерживает ли ClickHouse распределённый JOIN?

ClickHouse поддерживает распределённый JOIN в кластере.

Когда данные соразмещены в кластере (например, `JOIN` выполняется по идентификатору пользователя, который также является ключом шардинга), ClickHouse предоставляет возможность выполнять `JOIN` без перемещения данных по сети.

Если данные не соразмещены, ClickHouse позволяет выполнять широковещательный `JOIN`, при котором части присоединяемых данных распределяются по узлам кластера.

По состоянию на 2025 год ClickHouse не выполняет алгоритм shuffle-join, то есть не перераспределяет обе стороны `JOIN` по сети по всему кластеру в соответствии с ключами `JOIN`.
