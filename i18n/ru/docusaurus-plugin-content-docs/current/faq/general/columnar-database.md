# Что такое колоночная база данных? {#what-is-a-columnar-database}

Колоночная база данных хранит данные каждого столбца независимо. Это позволяет считывать с диска только те столбцы, которые используются в конкретном запросе. Обратная сторона — операции, затрагивающие целые строки, становятся пропорционально более затратными. Синонимом колоночной базы данных является колоночно-ориентированная система управления базами данных. ClickHouse — типичный пример такой системы.

Ключевые преимущества колоночных баз данных:

- Запросы, использующие только несколько столбцов из множества.
- Агрегирующие запросы к большим объёмам данных.
- Сжатие данных по столбцам.

Колоночная база данных является предпочтительным выбором для аналитических приложений, поскольку позволяет иметь множество столбцов в таблице на всякий случай, но не платить за неиспользуемые столбцы во время выполнения запросов на чтение (традиционная OLTP-база данных считывает все данные во время запросов, так как данные хранятся построчно, а не по столбцам). Колоночные базы данных предназначены для обработки больших данных и организации хранилищ данных, они часто изначально масштабируются с использованием распределённых кластеров недорогого оборудования для увеличения пропускной способности. ClickHouse достигает этого с помощью комбинации [распределённых](../../engines/table-engines/special/distributed.md) и [реплицируемых](../../engines/table-engines/mergetree-family/replication.md) таблиц.

Если вы хотите глубже изучить историю колоночных баз данных, их отличия от строковых баз данных и сценарии использования колоночных баз данных, см. [руководство по колоночным базам данных](https://clickhouse.com/engineering-resources/what-is-columnar-database).
