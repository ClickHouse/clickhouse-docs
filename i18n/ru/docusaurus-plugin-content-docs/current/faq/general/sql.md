# Какой синтаксис SQL поддерживает ClickHouse?

ClickHouse полностью поддерживает синтаксис SQL, включая такие возможности, как:

- SQL/JSON и тип данных JSON (SQL-2023)
- Оконные функции (SQL-2003)
- Обобщённые табличные выражения и рекурсивные запросы (SQL-1999)
- ROLLUP, CUBE и GROUPING SETS (SQL-1999)
- Полная поддержка RBAC (SQL-1999)
- Коррелированные подзапросы (SQL-1992)

Поддержка подтверждена бенчмарками TPC-H и TPC-DS, а также SQLTest.

ClickHouse реализовал многие возможности ещё до того, как они были стандартизированы ISO/IEC, такие как:

- условные агрегатные функции
- агрегатные функции `any`
- `least` и `greatest`
- `GROUP BY ALL`
- расширенное использование псевдонимов
- символы подчёркивания в числовых литералах

ClickHouse расширяет SQL, добавляя значительные улучшения для удобства работы:

- неограниченное использование псевдонимов
- псевдонимы внутри секции WITH
- комбинаторы агрегатных функций
- параметризованные агрегатные функции
- приближённые агрегатные функции
- встроенные целочисленные типы данных произвольной разрядности, десятичные числа расширенной точности
- функции высшего порядка для работы с массивами
- секция ARRAY JOIN и функция arrayJoin
- агрегация массивов
- секция LIMIT BY
- GROUP BY WITH TOTALS
- AS OF JOIN
- ANY/ALL JOIN
- естественный синтаксис для JSON
- завершающая запятая в списке столбцов
- порядок секций FROM ... SELECT
- типобезопасные параметры запросов и параметризованные представления

Некоторые из этих возможностей могут быть включены в будущие стандарты SQL, но уже сейчас доступны пользователям ClickHouse.
