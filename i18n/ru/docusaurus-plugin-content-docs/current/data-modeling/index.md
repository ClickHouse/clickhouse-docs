# Моделирование данных 

Этот раздел посвящён моделированию данных в ClickHouse и содержит следующие подразделы:

| Страница                                                        | Описание                                                                                                                                                                                      |
|-----------------------------------------------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| [Schema Design](/data-modeling/schema-design)                   | Рассматривает проектирование схемы в ClickHouse для оптимальной производительности с учётом таких факторов, как характер запросов, обновления данных, задержка и объём данных.               |
| [Dictionary](/dictionary)                                       | Объясняет, как определять и использовать словари для повышения производительности запросов и обогащения данных.                                                                              |
| [Materialized Views](/materialized-views)                       | Информация о материализованных представлениях и обновляемых материализованных представлениях в ClickHouse.                                                                                   |
| [Projections](/data-modeling/projections)                       | Информация о проекциях в ClickHouse.                                                                                                                                                          |
| [Data Compression](/data-compression/compression-in-clickhouse) | Рассматривает различные режимы сжатия в ClickHouse и то, как оптимизировать хранение данных и производительность запросов, выбирая подходящий метод сжатия для ваших типов данных и сценариев нагрузки. |
| [Denormalizing Data](/data-modeling/denormalization)            | Описывает подход к денормализации, используемый в ClickHouse, целью которого является повышение производительности запросов за счёт хранения связанных данных в одной таблице.              |
