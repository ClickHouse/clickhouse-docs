# Моделирование данных 

Этот раздел посвящён моделированию данных в ClickHouse и содержит следующие темы:

| Страница                                                        | Описание                                                                                                                                                                                      |
|-----------------------------------------------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| [Schema Design](/data-modeling/schema-design)                   | Рассматривается проектирование схемы в ClickHouse для достижения оптимальной производительности с учётом таких факторов, как характер запросов, обновления данных, задержка и объём.        |
| [Dictionary](/dictionary)                                       | Поясняется, как определять и использовать словари для повышения производительности запросов и обогащения данных.                                                                             |
| [Materialized Views](/materialized-views)                       | Информация о материализованных представлениях и обновляемых материализованных представлениях в ClickHouse.                                                                                   |
| [Projections](/data-modeling/projections)                       | Информация о проекциях в ClickHouse.                                                                                                                                                          |
| [Data Compression](/data-compression/compression-in-clickhouse) | Рассматриваются различные режимы сжатия в ClickHouse и то, как оптимизировать хранение данных и производительность запросов, выбирая подходящий метод сжатия для ваших типов данных и нагрузок. |
| [Denormalizing Data](/data-modeling/denormalization)            | Рассматривается подход к денормализации, используемый в ClickHouse, который нацелен на повышение производительности запросов за счёт хранения взаимосвязанных данных в одной таблице.        |
