## Понимание проблемы {#understanding-the-problem}

ClickHouse выдаёт ошибку «Too many parts» для предотвращения серьёзной деградации производительности. Небольшие части вызывают множество проблем: низкую производительность запросов из-за чтения и слияния большего количества файлов при выполнении запросов, повышенное потребление памяти, поскольку каждая часть требует хранения метаданных в памяти, снижение эффективности сжатия, так как меньшие блоки данных сжимаются менее эффективно, увеличенные накладные расходы на операции ввода-вывода из-за большего количества файловых дескрипторов и операций позиционирования, а также более медленные фоновые слияния, создающие дополнительную нагрузку на планировщик слияний.

## Раннее выявление проблемы {#recognize-parts-problem}

Этот запрос отслеживает фрагментацию таблиц путем анализа количества и размеров кусков во всех активных таблицах. Он выявляет таблицы с избыточным количеством или слишком маленькими кусками, которые могут требовать оптимизации слияния. Используйте его регулярно для обнаружения проблем фрагментации до того, как они повлияют на производительность запросов.

```sql runnable editable
-- Задача: Замените на реальные имена баз данных и таблиц для использования в production
-- Эксперимент: Настройте пороговые значения количества кусков (1000, 500, 100) в зависимости от вашей системы
SELECT
    database,
    table,
    count() as total_parts,
    sum(rows) as total_rows,
    round(avg(rows), 0) as avg_rows_per_part,
    min(rows) as min_rows_per_part,
    max(rows) as max_rows_per_part,
    round(sum(bytes_on_disk) / 1024 / 1024, 2) as total_size_mb,
    CASE
        WHEN count() > 1000 THEN 'КРИТИЧНО - Слишком много кусков (>1000)'
        WHEN count() > 500 THEN 'ПРЕДУПРЕЖДЕНИЕ - Много кусков (>500)'
        WHEN count() > 100 THEN 'ВНИМАНИЕ - Становится много кусков (>100)'
        ELSE 'НОРМАЛЬНО - Приемлемое количество кусков'
    END as parts_assessment,
    CASE
        WHEN avg(rows) < 1000 THEN 'ПЛОХО - Очень маленькие куски'
        WHEN avg(rows) < 10000 THEN 'УДОВЛЕТВОРИТЕЛЬНО - Маленькие куски'
        WHEN avg(rows) < 100000 THEN 'ХОРОШО - Средние куски'
        ELSE 'ОТЛИЧНО - Большие куски'
    END as part_size_assessment
FROM system.parts
WHERE active = 1
  AND database NOT IN ('system', 'information_schema')
GROUP BY database, table
ORDER BY total_parts DESC
LIMIT 20;
```
