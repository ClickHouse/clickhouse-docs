## Описание {#description}

Формат `Native` является наиболее эффективным форматом ClickHouse, поскольку он действительно «колоночный»
в том смысле, что не преобразует столбцы в строки.

В этом формате данные записываются и считываются [блоками](/development/architecture#block) в бинарном формате.
Для каждого блока последовательно записываются количество строк, количество столбцов, имена и типы столбцов, а также части столбцов в блоке.

Этот формат используется в нативном интерфейсе для взаимодействия между серверами, при использовании клиента командной строки и для клиентов на C++.

:::tip
Вы можете использовать этот формат для быстрого создания дампов, которые могут быть прочитаны только СУБД ClickHouse.
Самостоятельная работа с этим форматом может быть нецелесообразной.
:::
