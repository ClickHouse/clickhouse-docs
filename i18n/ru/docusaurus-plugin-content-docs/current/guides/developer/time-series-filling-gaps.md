# Заполнение пропусков во временных рядах

При работе с временными рядами в данных могут возникать пропуски из-за отсутствующих значений или периодов неактивности.
Обычно мы не хотим, чтобы такие пропуски присутствовали в результатах запроса. В таких случаях может пригодиться предложение `WITH FILL`.
В этом руководстве рассматривается, как использовать `WITH FILL` для заполнения пропусков во временных рядах.



## Настройка {#setup}

Представим, что у нас есть следующая таблица, которая хранит метаданные изображений, сгенерированных сервисом GenAI:

```sql
CREATE TABLE images
(
    `id` String,
    `timestamp` DateTime64(3),
    `height` Int64,
    `width` Int64,
    `size` Int64
)
ENGINE = MergeTree
ORDER BY (size, height, width);
```

Импортируем несколько записей:

```sql
INSERT INTO images VALUES (1088619203512250448, '2023-03-24 00:24:03.684', 1536, 1536, 2207289);
INSERT INTO images VALUES (1088619204040736859, '2023-03-24 00:24:03.810', 1024, 1024, 1928974);
INSERT INTO images VALUES (1088619204749561989, '2023-03-24 00:24:03.979', 1024, 1024, 1275619);
INSERT INTO images VALUES (1088619206431477862, '2023-03-24 00:24:04.380', 2048, 2048, 5985703);
INSERT INTO images VALUES (1088619206905434213, '2023-03-24 00:24:04.493', 1024, 1024, 1558455);
INSERT INTO images VALUES (1088619208524431510, '2023-03-24 00:24:04.879', 1024, 1024, 1494869);
INSERT INTO images VALUES (1088619208425437515, '2023-03-24 00:24:05.160', 1024, 1024, 1538451);
```


## Запросы по временным интервалам {#querying-by-bucket}

Мы будем анализировать изображения, созданные в период с `00:24:03` до `00:24:04` 24 марта 2023 года, поэтому создадим параметры для этих временных точек:

```sql
SET param_start = '2023-03-24 00:24:03',
    param_end = '2023-03-24 00:24:04';
```

Далее напишем запрос, который группирует данные в интервалы по 100 мс и возвращает количество изображений, созданных в каждом интервале:

```sql
SELECT
    toStartOfInterval(timestamp, toIntervalMillisecond(100)) AS bucket,
    count() AS count
FROM MidJourney.images
WHERE (timestamp >= {start:String}) AND (timestamp <= {end:String})
GROUP BY ALL
ORDER BY bucket ASC
```

```response
┌──────────────────bucket─┬─count─┐
│ 2023-03-24 00:24:03.600 │     1 │
│ 2023-03-24 00:24:03.800 │     1 │
│ 2023-03-24 00:24:03.900 │     1 │
│ 2023-03-24 00:24:04.300 │     1 │
│ 2023-03-24 00:24:04.400 │     1 │
│ 2023-03-24 00:24:04.800 │     1 │
└─────────────────────────┴───────┘
```

Результирующий набор включает только те интервалы, в которых было создано изображение, но для анализа временных рядов может потребоваться возвращать каждый 100-миллисекундный интервал, даже если в нём нет записей.


## WITH FILL {#with-fill}

Для заполнения этих пропусков можно использовать конструкцию `WITH FILL`.
Также укажем параметр `STEP` — размер заполняемых пропусков.
По умолчанию для типов `DateTime` используется значение 1 секунда, но нам нужно заполнить пропуски длиной 100 мс, поэтому зададим интервал 100 мс в качестве значения шага:

```sql
SELECT
    toStartOfInterval(timestamp, toIntervalMillisecond(100)) AS bucket,
    count() AS count
FROM MidJourney.images
WHERE (timestamp >= {start:String}) AND (timestamp <= {end:String})
GROUP BY ALL
ORDER BY bucket ASC
WITH FILL
STEP toIntervalMillisecond(100);
```

## WITH FILL...FROM {#with-fillfrom}

Однако в начале временного диапазона всё ещё остаётся пробел, который можно устранить, указав `FROM`:

```sql
SELECT
    toStartOfInterval(timestamp, toIntervalMillisecond(100)) AS bucket,
    count() AS count
FROM MidJourney.images
WHERE (timestamp >= {start:String}) AND (timestamp <= {end:String})
GROUP BY ALL
ORDER BY bucket ASC
WITH FILL
FROM toDateTime64({start:String}, 3)
STEP toIntervalMillisecond(100);
```

## WITH FILL...TO {#with-fillto}

Однако в конце временного диапазона по-прежнему отсутствуют некоторые интервалы, которые можно заполнить, указав значение `TO`.
`TO` не включает граничное значение, поэтому добавим небольшое смещение к конечному времени, чтобы оно было включено:

```sql
SELECT
    toStartOfInterval(timestamp, toIntervalMillisecond(100)) AS bucket,
    count() AS count
FROM MidJourney.images
WHERE (timestamp >= {start:String}) AND (timestamp <= {end:String})
GROUP BY ALL
ORDER BY bucket ASC
WITH FILL
FROM toDateTime64({start:String}, 3)
TO toDateTime64({end:String}, 3) + INTERVAL 1 millisecond
STEP toIntervalMillisecond(100);
```

## Накопительный подсчёт {#cumulative-count}

Предположим, что теперь мы хотим вести накопительный подсчёт количества созданных изображений по интервалам.
Это можно сделать, добавив столбец `cumulative`, как показано ниже:

```sql
SELECT
    toStartOfInterval(timestamp, toIntervalMillisecond(100)) AS bucket,
    count() AS count,
    sum(count) OVER (ORDER BY bucket) AS cumulative
FROM MidJourney.images
WHERE (timestamp >= {start:String}) AND (timestamp <= {end:String})
GROUP BY ALL
ORDER BY bucket ASC
WITH FILL
FROM toDateTime64({start:String}, 3)
TO toDateTime64({end:String}, 3) + INTERVAL 1 millisecond
STEP toIntervalMillisecond(100);
```

## WITH FILL...INTERPOLATE {#with-fillinterpolate}

Все строки, содержащие `0` в столбце `count`, также содержат `0` в столбце cumulative, хотя мы хотели бы использовать предыдущее значение из столбца `cumulative`.
Это можно сделать с помощью конструкции `INTERPOLATE`, как показано ниже:

```sql
SELECT
    toStartOfInterval(timestamp, toIntervalMillisecond(100)) AS bucket,
    count() AS count,
    sum(count) OVER (ORDER BY bucket) AS cumulative
FROM MidJourney.images
WHERE (timestamp >= {start:String}) AND (timestamp <= {end:String})
GROUP BY ALL
ORDER BY bucket ASC
WITH FILL
FROM toDateTime64({start:String}, 3)
TO toDateTime64({end:String}, 3) + INTERVAL 100 millisecond
STEP toIntervalMillisecond(100)
INTERPOLATE (cumulative);
```

Для завершения добавим столбчатую диаграмму с помощью функции `bar`, не забыв добавить новый столбец в конструкцию `INTERPOLATE`.

```sql
SELECT
    toStartOfInterval(timestamp, toIntervalMillisecond(100)) AS bucket,
    count() AS count,
    sum(count) OVER (ORDER BY bucket) AS cumulative,
    bar(cumulative, 0, 10, 10) AS barChart
FROM MidJourney.images
WHERE (timestamp >= {start:String}) AND (timestamp <= {end:String})
GROUP BY ALL
ORDER BY bucket ASC
WITH FILL
FROM toDateTime64({start:String}, 3)
TO toDateTime64({end:String}, 3) + INTERVAL 100 millisecond
STEP toIntervalMillisecond(100)
INTERPOLATE (cumulative, barChart);
```
