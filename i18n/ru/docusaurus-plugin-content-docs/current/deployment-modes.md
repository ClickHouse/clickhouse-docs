ClickHouse — это универсальная система управления базами данных, которую можно развернуть несколькими способами в зависимости от ваших потребностей. В основе всех вариантов развертывания **используется один и тот же мощный движок базы данных ClickHouse** — различается лишь то, как вы с ним взаимодействуете и где он работает.

Независимо от того, запускаете ли вы масштабную аналитическую систему в продакшене, выполняете локальный анализ данных или разрабатываете приложения, существует вариант развертывания, подходящий для вашего сценария. За счёт единообразия базового движка вы получаете одинаково высокую производительность и совместимость с SQL во всех режимах развертывания.
В этом руководстве рассматриваются четыре основных способа развертывания и использования ClickHouse:

* ClickHouse Server для традиционных клиент-серверных развертываний
* ClickHouse Cloud для полностью управляемой работы с базой данных
* clickhouse-local для обработки данных из командной строки
* chDB для встраивания ClickHouse непосредственно в приложения

Каждый режим развертывания имеет свои сильные стороны и оптимальные сценарии использования, которые мы подробно рассмотрим ниже.

## ClickHouse Server {#clickhouse-server}

ClickHouse Server представляет собой традиционную клиент-серверную архитектуру и идеально подходит для production-развертываний. Этот режим развертывания обеспечивает полные возможности OLAP-базы данных с высокой пропускной способностью и низкой задержкой запросов, которыми известен ClickHouse.

<Image img={chServer} alt='ClickHouse Server' size='sm' />

<br />

Что касается гибкости развертывания, ClickHouse Server можно установить на локальную машину для разработки или тестирования, развернуть у крупных облачных провайдеров, таких как AWS, GCP или Azure, для работы в облаке, или настроить на собственном оборудовании on-premises. Для крупномасштабных операций его можно настроить как распределенный кластер для обработки повышенной нагрузки и обеспечения высокой доступности.

Этот режим развертывания является оптимальным выбором для production-сред, где критически важны надежность, производительность и полный доступ к функциональности.


## ClickHouse Cloud {#clickhouse-cloud}

[ClickHouse Cloud](/cloud/overview) — это полностью управляемая версия ClickHouse, которая избавляет от операционных издержек, связанных с развёртыванием собственной инфраструктуры. Сохраняя все основные возможности ClickHouse Server, она дополняет их функциями, упрощающими разработку и эксплуатацию.

<Image img={chCloud} alt='ClickHouse Cloud' size='sm' />

Ключевое преимущество ClickHouse Cloud — встроенный инструментарий. [ClickPipes](/getting-started/quick-start/cloud/#clickpipes) предоставляет надёжную платформу для загрузки данных, позволяя легко подключаться к различным источникам и получать из них потоковые данные без необходимости управления сложными ETL-конвейерами. Платформа также предлагает специализированный [API для запросов](/cloud/get-started/query-endpoints), что значительно упрощает разработку приложений.

SQL Console в ClickHouse Cloud включает мощную функцию создания [дашбордов](/cloud/manage/dashboards), которая позволяет преобразовывать запросы в интерактивные визуализации. Вы можете создавать и делиться дашбордами на основе сохранённых запросов, добавляя интерактивные элементы через параметры запросов. Дашборды можно сделать динамическими с помощью глобальных фильтров, что позволяет пользователям исследовать данные через настраиваемые представления — важно отметить, что для просмотра визуализаций пользователям потребуется как минимум доступ на чтение к соответствующим сохранённым запросам.

Для мониторинга и оптимизации ClickHouse Cloud включает встроенные графики и [аналитику запросов](/cloud/get-started/query-insights). Эти инструменты обеспечивают глубокую видимость производительности кластера, помогая понять паттерны запросов, использование ресурсов и возможности для оптимизации. Такой уровень наблюдаемости особенно ценен для команд, которым необходимо поддерживать высокопроизводительные аналитические операции без выделения ресурсов на управление инфраструктурой.

Управляемый характер сервиса означает, что вам не нужно беспокоиться об обновлениях, резервном копировании, масштабировании или патчах безопасности — всё это выполняется автоматически. Это делает его идеальным выбором для организаций, которые хотят сосредоточиться на данных и приложениях, а не на администрировании баз данных.


## clickhouse-local {#clickhouse-local}

[clickhouse-local](/operations/utilities/clickhouse-local) — это мощный инструмент командной строки, который предоставляет полную функциональность ClickHouse в виде автономного исполняемого файла. По сути, это та же база данных, что и ClickHouse Server, но упакованная таким образом, что позволяет использовать все возможности ClickHouse непосредственно из командной строки без запуска серверного экземпляра.

<Image img={chLocal} alt='clickHouse-local' size='sm' />

Этот инструмент отлично подходит для специализированного анализа данных, особенно при работе с локальными файлами или данными, хранящимися в облачных хранилищах. Вы можете напрямую запрашивать файлы в различных форматах (CSV, JSON, Parquet и т. д.) с использованием SQL-диалекта ClickHouse, что делает его отличным выбором для быстрого исследования данных или разовых аналитических задач.

Поскольку clickhouse-local включает всю функциональность ClickHouse, вы можете использовать его для преобразования данных, конвертации форматов или любых других операций с базой данных, которые обычно выполняются с помощью ClickHouse Server. Хотя он в основном используется для временных операций, при необходимости он также может сохранять данные, используя тот же движок хранения, что и ClickHouse Server.

Сочетание функций удалённых таблиц и доступа к локальной файловой системе делает clickhouse-local особенно полезным для сценариев, в которых необходимо объединить данные между ClickHouse Server и файлами на локальной машине. Это особенно ценно при работе с конфиденциальными или временными локальными данными, которые не требуется загружать на сервер.


## chDB {#chdb}

[chDB](/chdb) — это ClickHouse, встроенный в качестве внутрипроцессного движка базы данных. Основная реализация выполнена для Python, но также доступны версии для Go, Rust, NodeJS и Bun. Этот вариант развертывания предоставляет мощные OLAP-возможности ClickHouse непосредственно в процессе вашего приложения, устраняя необходимость в отдельной установке базы данных.

<Image img={chDB} alt='chDB — встроенный ClickHouse' size='sm' />

chDB обеспечивает бесшовную интеграцию с экосистемой вашего приложения. Например, в Python он оптимизирован для эффективной работы с распространенными инструментами анализа данных, такими как Pandas и Arrow, минимизируя накладные расходы на копирование данных через Python memoryview. Это делает его особенно ценным для специалистов по работе с данными и аналитиков, которые хотят использовать производительность запросов ClickHouse в рамках своих существующих рабочих процессов.

chDB также может подключаться к базам данных, созданным с помощью clickhouse-local, обеспечивая гибкость в работе с данными. Это означает, что вы можете легко переключаться между локальной разработкой, исследованием данных в Python и более постоянными решениями для хранения без изменения паттернов доступа к данным.
