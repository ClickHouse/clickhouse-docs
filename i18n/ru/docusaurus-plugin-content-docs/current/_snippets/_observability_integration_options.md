## Примеры интеграции {#examples}

Внешняя интеграция позволяет организациям сохранить устоявшиеся процессы мониторинга, использовать имеющийся опыт команды работы с привычными инструментами и интегрировать мониторинг ClickHouse с общей системой наблюдаемости инфраструктуры без нарушения текущих процессов и необходимости значительных инвестиций в переобучение.
Команды могут применять существующие правила оповещений и процедуры эскалации к метрикам ClickHouse, одновременно сопоставляя производительность базы данных с состоянием приложений и инфраструктуры в рамках единой платформы наблюдаемости. Такой подход максимизирует окупаемость инвестиций в текущие системы мониторинга и обеспечивает более быстрое устранение неполадок благодаря консолидированным дашбордам и привычным интерфейсам инструментов.

### Мониторинг Grafana Cloud {#grafana}

Grafana предоставляет мониторинг ClickHouse как через прямую интеграцию плагина, так и через подходы на основе Prometheus. Интеграция через конечную точку Prometheus обеспечивает операционное разделение между мониторингом и производственными нагрузками, одновременно позволяя визуализацию в рамках существующей инфраструктуры Grafana Cloud. Руководство по настройке см. в [документации Grafana по ClickHouse](https://grafana.com/docs/grafana-cloud/monitor-infrastructure/integrations/integration-reference/integration-clickhouse/).

### Мониторинг Datadog {#datadog}

Datadog разрабатывает специализированную интеграцию через API, которая обеспечит корректный мониторинг облачного сервиса с учетом поведения простоя сервиса. В переходный период команды могут использовать подход интеграции OpenMetrics через конечные точки Prometheus в ClickHouse для операционного разделения и экономически эффективного мониторинга. Руководство по настройке см. в [документации Datadog по интеграции Prometheus и OpenMetrics](https://docs.datadoghq.com/integrations/openmetrics/).

### ClickStack {#clickstack}

ClickStack — это рекомендуемое решение для наблюдаемости ClickHouse, предназначенное для глубокого анализа системы и отладки, предоставляющее единую платформу для логов, метрик и трассировок с использованием ClickHouse в качестве движка хранения. Данный подход основан на HyperDX, пользовательском интерфейсе ClickStack, который подключается напрямую к системным таблицам внутри вашего экземпляра ClickHouse.
HyperDX поставляется с дашбордом, ориентированным на ClickHouse, с вкладками для запросов SELECT, INSERT и инфраструктуры. Команды также могут использовать синтаксис Lucene или SQL для поиска по системным таблицам и логам, а также создавать пользовательские визуализации через Chart Explorer для детального анализа системы.
Данный подход идеален для отладки сложных проблем, анализа производительности и глубокого изучения системы, а не для оповещений в реальном времени в производственной среде.

:::note
Обратите внимание, что данный подход будет пробуждать простаивающие сервисы, поскольку HyperDX выполняет запросы к системным таблицам напрямую.
:::
