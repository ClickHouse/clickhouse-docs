Обратитесь к [официальной документации](/interfaces/mysql),
чтобы узнать, как настроить сервер ClickHouse с включённым интерфейсом MySQL.

Помимо добавления записи в файл конфигурации сервера `config.xml`

```xml
<clickhouse>
    <mysql_port>9004</mysql_port>
</clickhouse>
```

также *обязательно* использовать
[Double SHA1 password encryption](/operations/settings/settings-users#user-namepassword)
для пользователя, который будет работать через интерфейс MySQL.

Генерация в командной строке случайного пароля, зашифрованного с помощью Double SHA1:

```shell
PASSWORD=$(base64 < /dev/urandom | head -c16); echo "$PASSWORD"; echo -n "$PASSWORD" | sha1sum | tr -d '-' | xxd -r -p | sha1sum | tr -d '-'
```

Результат должен выглядеть следующим образом:

```
LZOQYnqQN4L/T6L0
fbc958cc745a82188a51f30de69eebfc67c40ee4
```

Первая строка — сгенерированный пароль, а вторая — хэш, который можно использовать для настройки ClickHouse.

Ниже приведён пример конфигурации для `mysql_user`, в которой используется сгенерированный хэш:

`/etc/clickhouse-server/users.d/mysql_user.xml`

```xml
<users>
    <mysql_user>
        <password_double_sha1_hex>fbc958cc745a82188a51f30de69eebfc67c40ee4</password_double_sha1_hex>
        <networks>
            <ip>::/0</ip>
        </networks>
        <profile>default</profile>
        <quota>default</quota>
    </mysql_user>
</users>
```

Замените значение `password_double_sha1_hex` на сгенерированный вами Double SHA1-хеш.

Кроме того, рекомендуется использовать `use_mysql_types_in_show_columns`,
чтобы отображать родные типы MySQL вместо типов ClickHouse в результатах запроса `SHOW [FULL] COLUMNS`,
что позволяет BI-инструментам корректно определять схему базы данных при использовании коннекторов MySQL.

Например:

`/etc/clickhouse-server/users.d/mysql_user.xml`

```xml
<profiles>
    <default>
        <use_mysql_types_in_show_columns>1</use_mysql_types_in_show_columns>
    </default>
</profiles>
```

или назначьте его другому профилю вместо профиля по умолчанию.

Если у вас доступен исполняемый файл `mysql`, вы можете протестировать подключение из командной строки.
Используя пример имени пользователя (`mysql_user`) и пароля (`LZOQYnqQN4L/T6L0`) из приведённого выше примера, команда будет выглядеть так:

```bash
mysql --protocol tcp -h localhost -u mysql_user -P 9004 --password=LZOQYnqQN4L/T6L0
```

```
mysql> show databases;
+--------------------+
| name               |
+--------------------+
| INFORMATION_SCHEMA |
| default            |
| information_schema |
| system             |
+--------------------+
4 rows in set (0.00 sec)
Read 4 rows, 603.00 B in 0.00156 sec., 2564 rows/sec., 377.48 KiB/sec.
```

Наконец, настройте ClickHouse Server на прослушивание нужных IP-адресов. Например, в `config.xml` раскомментируйте следующие строки, чтобы принимать подключения на всех адресах:

```bash
<listen_host>::</listen_host> 
```
