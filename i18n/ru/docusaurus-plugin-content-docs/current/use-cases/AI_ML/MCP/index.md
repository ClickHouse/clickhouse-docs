[Model Context Protocol](https://www.anthropic.com/news/model-context-protocol) (MCP) — это стандарт, разработанный компанией Anthropic, который позволяет ИИ‑ассистентам без лишних усилий интегрироваться с внешними системами.
Этот протокол позволяет ИИ‑ассистентам подключаться к источникам данных, API, базам данных и многому другому в безопасном и стандартизированном формате.

MCP создаёт универсальный интерфейс между ИИ‑моделями и различными сервисами, устраняя необходимость в специализированных реализациях инструментов для каждой интеграции.
О нём можно думать как об универсальном стандарте API, разработанном специально для ИИ‑систем.

Ключевое преимущество MCP заключается в том, что библиотекам ИИ нужно реализовать поддержку протокола только один раз.
После этого все сервисы, совместимые с MCP, сразу становятся доступными, что экономит разработчикам и мейнтейнерам библиотек ИИ много времени.


## Какова архитектура MCP? {#mcp-architecture}

MCP использует клиент-серверную архитектуру:

- Клиенты (например, Claude Desktop, Cursor или VS Code) устанавливают соединения с серверами MCP. Коллекцию клиентов можно найти в репозитории GitHub [awesome-mcp-clients](https://github.com/punkpeye/awesome-mcp-clients?tab=readme-ov-file#windsurf).
- Серверы предоставляют инструменты и возможности через стандартизированные интерфейсы. Коллекцию серверов можно найти в репозитории GitHub [awesome-mcp-servers](https://github.com/punkpeye/awesome-mcp-servers).
- Модели ИИ могут использовать эти инструменты для доступа к внешним данным и функциональности при необходимости.

## Есть ли у ClickHouse MCP-сервер? {#clickhouse-mcp-server}

Да!
[ClickHouse MCP Server](https://github.com/ClickHouse/mcp-clickhouse) предоставляет следующие инструменты:

- `run_select_query` — выполнение SQL-запросов к кластеру ClickHouse.
- `list_databases` — получение списка всех баз данных в кластере ClickHouse.
- `list_tables` — получение списка всех таблиц в базе данных.
