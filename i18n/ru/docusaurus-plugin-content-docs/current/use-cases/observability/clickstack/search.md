ClickStack позволяет выполнять полнотекстовый поиск по вашим событиям (логам и трассировкам). Вы можете начать поиск, просто введя ключевые слова, которые соответствуют вашим событиям. Например, если ваш лог содержит &quot;Error&quot;, вы можете найти его, просто набрав &quot;Error&quot; в строке поиска.

Тот же синтаксис поиска используется и для фильтрации событий в Dashboards и Charts.


## Возможности поиска {#search-features}

### Синтаксис поиска на естественном языке {#natural-language-syntax}

- Поиск нечувствителен к регистру
- Поиск по умолчанию осуществляется по целым словам (например, `Error` будет
  соответствовать `Error here`, но не `Errors here`). Вы можете окружить слово
  подстановочными символами, чтобы искать по частям слова (например, `*Error*`
  будет соответствовать `AnyError` и `AnyErrors`)
- Поисковые термины могут встречаться в любом порядке (например, `Hello World`
  будет соответствовать логам, содержащим `Hello World` и `World Hello`)
- Вы можете исключать ключевые слова, используя `NOT` или `-` (например,
  `Error NOT Exception` или `Error -Exception`)
- Вы можете использовать `AND` и `OR` для объединения нескольких ключевых слов
  (например, `Error OR Exception`)
- Точное совпадение можно задать с помощью двойных кавычек (например,
  `"Error tests not found"`)

#### Поиск по столбцам/свойствам {#column-search}

- Вы можете искать по столбцам и свойствам JSON/map, используя `column:value`
  (например, `level:Error`, `service:app`)
- Вы можете искать диапазон значений, используя операторы сравнения (`>`, `<`,
  `>=`, `<=`) (например, `Duration:>1000`)
- Вы можете искать по факту существования свойства, используя `property:*`
  (например, `duration:*`)

### Ввод времени {#time-input}

- Поле ввода времени принимает запросы на естественном языке (например,
  `1 hour ago`, `yesterday`, `last week`)
- Указание одной точки во времени приведёт к поиску с этого момента и до
  текущего времени.
- Диапазон времени всегда будет преобразован в интерпретированный системой
  диапазон при выполнении поиска для упрощения отладки временных запросов.
- Вы также можете выделить столбец гистограммы, чтобы приблизить конкретный
  временной диапазон.

### Синтаксис SQL-поиска {#sql-syntax}

При необходимости вы можете переключить поле ввода поиска в режим SQL.
В этом режиме принимается любое корректное SQL-выражение WHERE для поиска.
Это полезно для сложных запросов, которые нельзя выразить с помощью синтаксиса Lucene.

### Оператор SELECT  {#select-statement}

Чтобы указать столбцы, отображаемые в результатах поиска, вы можете использовать
поле ввода `SELECT`. Это SQL-выражение SELECT для столбцов, выбираемых на
странице поиска. Псевдонимы в данный момент не поддерживаются (например, вы не
можете использовать `column as "alias"`).



## Сохранённые поисковые запросы {#saved-searches}

Вы можете сохранять поисковые запросы для быстрого доступа в дальнейшем. После сохранения ваши запросы будут отображаться в левой боковой панели, что упрощает повторное использование часто выполняемых поисковых запросов без необходимости собирать их заново.

Чтобы сохранить поисковый запрос, просто настройте условия поиска и нажмите кнопку «Сохранить». Вы можете задать сохранённому поисковому запросу понятное имя, чтобы упростить его идентификацию в будущем.

### Добавление алертов к сохранённым поисковым запросам {#alerts-on-saved-searches}

Сохранённые поисковые запросы могут отслеживаться с помощью алертов, которые уведомляют вас при выполнении определённых условий. Вы можете настроить алерты так, чтобы они срабатывали, когда количество событий, соответствующих вашему сохранённому запросу, превышает или опускается ниже заданного порогового значения. 

Дополнительные сведения по настройке и конфигурированию алертов см. в разделе [документации по алертам](/use-cases/observability/clickstack/alerts).
