## Миграция на ClickStack с Elastic {#migrating-to-clickstack-from-elastic}

Это руководство предназначено для пользователей, переходящих с Elastic Stack — в частности, для тех, кто использует Kibana для мониторинга логов, трассировок и метрик, собираемых через Elastic Agent и хранящихся в Elasticsearch. В нём описываются эквивалентные концепции и типы данных в ClickStack, объясняется, как преобразовать запросы Kibana на основе Lucene в синтаксис HyperDX, а также даются рекомендации по миграции данных и агентов для обеспечения плавного перехода.

Перед началом миграции важно понимать различия между ClickStack и Elastic Stack.

Вам следует рассмотреть переход на ClickStack, если:

- Вы обрабатываете большие объёмы данных наблюдаемости и считаете Elastic слишком дорогим из-за неэффективного сжатия и низкого использования ресурсов. ClickStack может значительно снизить затраты на хранение и вычисления — обеспечивая как минимум 10-кратное сжатие исходных данных.
- Вы сталкиваетесь с низкой производительностью поиска при масштабировании или с узкими местами при приёме данных.
- Вы хотите коррелировать сигналы наблюдаемости с бизнес-данными с помощью SQL, объединяя процессы наблюдаемости и аналитики.
- Вы используете OpenTelemetry и хотите избежать привязки к поставщику.
- Вы хотите воспользоваться преимуществами разделения хранения и вычислений в ClickHouse Cloud, обеспечивающего практически неограниченное масштабирование — с оплатой только за вычисления при приёме данных и объектное хранилище в периоды простоя.

Однако ClickStack может не подойти, если:

- Вы используете данные наблюдаемости преимущественно для задач безопасности и нуждаетесь в продукте, ориентированном на SIEM.
- Универсальное профилирование является критически важной частью вашего рабочего процесса.
- Вам требуется платформа для создания дашбордов бизнес-аналитики (BI). ClickStack намеренно имеет специализированные визуальные процессы для инженеров по надёжности сайтов и разработчиков и не предназначен для использования в качестве инструмента бизнес-аналитики (BI). Для эквивалентных возможностей мы рекомендуем использовать [Grafana с плагином ClickHouse](/integrations/grafana) или [Superset](/integrations/superset).
