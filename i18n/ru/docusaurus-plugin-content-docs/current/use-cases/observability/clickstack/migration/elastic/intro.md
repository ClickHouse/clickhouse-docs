## Миграция на ClickStack с Elastic {#migrating-to-clickstack-from-elastic}

Это руководство предназначено для пользователей, переходящих с Elastic Stack — в частности, для тех, кто использует Kibana для мониторинга логов, трассировок и метрик, собираемых через Elastic Agent и хранящихся в Elasticsearch. В нём описываются эквивалентные концепции и типы данных в ClickStack, объясняется, как преобразовать запросы Kibana на основе Lucene в синтаксис HyperDX, и даются рекомендации по миграции данных и агентов для обеспечения плавного перехода.

Перед началом миграции важно понимать различия между ClickStack и Elastic Stack.

Вам следует рассмотреть переход на ClickStack, если:

- Вы обрабатываете большие объёмы данных наблюдаемости и считаете Elastic слишком дорогим из-за неэффективного сжатия и низкой утилизации ресурсов. ClickStack может значительно снизить затраты на хранение и вычисления — обеспечивая как минимум 10-кратное сжатие исходных данных.
- Вы сталкиваетесь с низкой производительностью поиска при масштабировании или с узкими местами при приёме данных.
- Вы хотите коррелировать сигналы наблюдаемости с бизнес-данными с помощью SQL, объединяя процессы наблюдаемости и аналитики.
- Вы используете OpenTelemetry и хотите избежать привязки к поставщику.
- Вы хотите воспользоваться разделением хранения и вычислений в ClickHouse Cloud, обеспечивающим практически неограниченное масштабирование — оплачивая только вычисления для приёма данных и объектное хранилище в периоды простоя.

Однако ClickStack может не подойти, если:

- Вы используете данные наблюдаемости в первую очередь для задач безопасности и нуждаетесь в продукте, ориентированном на SIEM.
- Универсальное профилирование является критически важной частью вашего рабочего процесса.
- Вам требуется платформа дашбордов для бизнес-аналитики (BI). ClickStack намеренно имеет специализированные визуальные рабочие процессы для SRE и разработчиков и не предназначен как инструмент бизнес-аналитики (BI). Для эквивалентных возможностей мы рекомендуем использовать [Grafana с плагином ClickHouse](/integrations/grafana) или [Superset](/integrations/superset).
